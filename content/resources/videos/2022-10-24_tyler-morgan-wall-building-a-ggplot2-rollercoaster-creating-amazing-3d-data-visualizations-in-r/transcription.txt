<h3>What data works in 3D</h3>

<p><a data-t="267">So let's cover the basics.</a> <a data-t="270">What exactly goes into a 3D data visualization?</a> <a data-t="274">First, what kind of data can be used in 3D plots?</a> <a data-t="278">All you need for a 3D data is are data with at least three numeric components.</a> <a data-t="284">Here we have European flight path data from the crowdsourced open sky network.</a> <a data-t="289">Flight path data has latitude, longitude, and altitude, which intuitively correspond</a> <a data-t="294">as to X, Y, and Z, and are here plotted as 3D paths.</a></p>

<p><a data-t="299">This is a fairly straightforward and intuitive use of 3D, but you can also choose to use</a> <a data-t="302">3D aesthetics with 2D data.</a> <a data-t="306">Much like you might choose a line chart over a bar chart purely for aesthetic reasons,</a> <a data-t="310">you might also choose a 3D chart over its 2D counterpart because you can make it more</a> <a data-t="313">interesting and visually appealing.</a> <a data-t="318">This is a map of the submarine fiber optic cable network, and there's no actual 3D data</a> <a data-t="320">present, just latitude and longitude data.</a> <a data-t="325">The cables rise out of the earth where they start and fall back in where they land.</a> <a data-t="330">You could easily plot this as simple 2D paths on a world map, but I think the floating cables,</a> <a data-t="335">subtle shadows, and rotating globe really elevate what is otherwise a fairly simple</a> <a data-t="338">data set into something more relatable and tangible on a human scale, like the earth</a> <a data-t="341">is being wrapped in yarn.</a></p>

<p><a data-t="347">So many 2D plots have direct 3D equivalents, but just replacing points and lines with spheres</a> <a data-t="350">and cylinders can lead to mixed results.</a> <a data-t="354">Data floating in the void can be hard to interpret if we're just presented with a static image,</a> <a data-t="358">as objects in the foreground can block those in the background, as we see here in this</a> <a data-t="362">rendering of a morphine molecule generated with the ray molecule package.</a> <a data-t="369">But you can solve this problem by animating the 3D data.</a> <a data-t="373">The movement disambiguates the foreground from the background so that the reader can</a> <a data-t="377">fully interpret the 3D model.</a></p>

<h3>3D vs color encoding</h3>

<p><a data-t="466">So one reason you might consider using 3D instead of color is that encoding data to</a> <a data-t="470">color requires the viewer to reference a color bar to understand the ordering of the data.</a> <a data-t="475">As there's no intuitive reason why some colors correspond to lower values and some colors</a> <a data-t="480">correspond to higher.</a> <a data-t="481">Here I plotted the heights of buildings in Seattle, Washington by mapping the data to</a> <a data-t="486">color.</a> <a data-t="487">Height, on the other hand, intuitively allows you to judge which values are less than or</a> <a data-t="492">greater than others.</a> <a data-t="494">You can tell when a building here is about half the height of another without having</a> <a data-t="498">to reference an external color bar.</a> <a data-t="500">However, because 3D introduces ambiguity in the depth axis, you lose precision.</a> <a data-t="505">Thus 3D data visualizations are great when the exact numeric values aren't important</a> <a data-t="510">but comparing relative values and getting an overall sense of the distribution of data</a> <a data-t="514">is.</a></p>

<p><a data-t="516">Using both 3D and color can give you the best of both worlds.</a> <a data-t="520">Color to indicate precise numeric levels and height to give a more intuitive way to</a> <a data-t="524">compare values.</a> <a data-t="525">Here we have a visualization I made of the global population density using both color</a> <a data-t="529">and height.</a> <a data-t="531">The use of color allows the viewer to map data to direct population intervals while</a> <a data-t="535">the use of height establishes order without needing to reference a legend.</a></p>
