<h3>Why PDFs are so popular</h3>

<p><a data-t="47">Alright so just a few slides here to get started.</a> <a data-t="52">And the first thing I want to talk about is what is PDF, which stands for portable document format, particularly why is it so popular.</a> <a data-t="58">So if you've ever been in a situation where you had to share a document with somebody you probably had to stop and think and consider what is the optimal format for sharing this document.</a> <a data-t="75">For example if my documents in a Microsoft Word document and I want to share it with somebody else you probably ask some questions to yourself like does that person also have Microsoft Word</a> <a data-t="83">and if so do they have the same version or at least a relatively recent version or similar version to what I have.</a> <a data-t="88">Maybe if they don't have Microsoft Word but also work in Google Docs we're not sure.</a></p>

<p><a data-t="93">So maybe you try a different format and you want to create an HTML document.</a> <a data-t="99">So if you go to share an HTML document with somebody you have a whole new set of questions such as what internet web browser will they use to open up this document which is essentially a web page.</a> <a data-t="112">Are they going to use Safari, Firefox, Chrome and depending on what browser they use it may render slightly differently.</a> <a data-t="119">Now for a lot of these HTML formats they require some system dependencies such as JavaScript.</a> <a data-t="126">So if they go to open an HTML document and they don't have JavaScript installed will it render appropriately we don't know.</a> <a data-t="133">Also if you go to try to email an HTML document to somebody a lot of email clients will actually block emails that have HTML files as attachments so that's something else you need to consider.</a></p>

<p><a data-t="144">And then particularly important for today's demo what if we want to share a document that's created using something like Jupyter Notebooks or </a><a href="/software/quarto/">Quarto</a><a data-t="144"> or R Markdown.</a> <a data-t="154">So if we go to share one of these documents does that recipient also have R and Python installing can they even open up these documents</a> <a data-t="161">and if so do they have the same package versions and language version that I do to ensure that it's the exact same document and it's rendered the exact same way when they open it up on their system.</a> <a data-t="174">So that's a lot of questions to answer and this really is the driving force behind why PDFs were created because they are perfect for cross-platform compatibility.</a></p>

<p><a data-t="185">So if I create a PDF on my Mac computer and I go to share with somebody using a Windows machine I can be pretty confident it's gonna render exactly the same because it preserves formatting which is really important.</a> <a data-t="199">PDFs is also something known as a what you see is what you get format meaning the content won't shift reflow or break apart when open on different devices.</a> <a data-t="211">So unlike formats such as Word where formatting can change depending on the software or device things are just going to stay pretty consistent with PDFs.</a> <a data-t="221">PDFs are also by default non-editable so if you go to share with somebody they're really not going to be able to edit it.</a> <a data-t="231">Now there are some ways you can edit a PDF but usually by default they're non-editable and they're also print ready so if you receive a PDF you can typically file print and that printed output will look pretty good.</a></p>
