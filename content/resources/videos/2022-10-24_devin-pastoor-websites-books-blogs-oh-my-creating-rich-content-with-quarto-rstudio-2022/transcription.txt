<p><a data-t="0">My name is Devin, I'm a Solutions Engineer at Posit, and I'm really thrilled to be up</a> <a data-t="10">here today to talk about </a><a href="/software/quarto/">Quarto</a><a data-t="10">, because in many ways, the vision of Quarto, that reproducible</a> <a data-t="16">research is a core part of high-quality scientific communications, was a key driver in actually</a> <a data-t="22">nudging me along my career journey that ultimately has brought me to Posit.</a></p>

<p><a data-t="28">Almost 10 years ago, as a graduate student, R Markdown blew me away when all of a sudden</a> <a data-t="34">I could shortcut an enormous amount of the tedious interactions from working in R scripts</a> <a data-t="39">and actually taking the outputs of those, getting them into Word or PDF documents to</a> <a data-t="44">distribute.</a></p>

<p><a data-t="46">Now, as many graduate students are inclined to do, I also dabbled in figuring out how</a> <a data-t="52">I could use the tools I was learning, in particular R, in fun, new ways.</a></p>

<h3>From R Markdown to <a href="/software/bookdown/">bookdown</a> and <a href="/software/blogdown/">blogdown</a></h3>

<p><a data-t="60">So about that time, a new tool emerged called Gitbook, and it was really fascinating to</a> <a data-t="65">me because the idea was to take a bunch of Markdown documents, jam them through this</a> <a data-t="69">processor, and out popped a website that had navigation and styling, and it kind of looked</a> <a data-t="75">like a book.</a> <a data-t="76">And so I said, hmm, that seems kind of interesting.</a> <a data-t="79">I wonder if I can take R Markdown documents, get them to Markdown to put them through Gitbook.</a> <a data-t="85">And so I put together some really, really scary-looking code, but thankfully the </a><a href="/software/rstudio/">RStudio</a> <a data-t="91">team was also keeping an eye on such ideas, and bookdown entered the scene.</a></p>

<p><a data-t="100">This ability to build on a unit of an R Markdown document into more complex content felt like</a> <a data-t="105">a taste of opportunities to come.</a></p>

<p><a data-t="110">And so similarly, when I started playing around with what it was going to be like to build</a> <a data-t="114">a blog, I found Hugo, a static site generator that you could also make most of your content</a> <a data-t="120">with Markdown.</a> <a data-t="121">And so by using that content in Markdown, plus a little bit of metadata and configuration,</a> <a data-t="126">you could generate a professional-looking site.</a></p>

<p><a data-t="131">Again, the RStudio team responded with blogdown.</a> <a data-t="136">And it was so exciting to me, because there's actually official tools to support these janky</a> <a data-t="141">workflows that I was trying to come up with.</a> <a data-t="144">That integration between Hugo and Markdown was able to help bring content to life.</a></p>

<p><a data-t="150">But as fascinating as these experiments were, as fascinating as learning those tools were,</a> <a data-t="156">it was undeniable that there were some significant points of friction and frustration in figuring</a> <a data-t="161">it out, especially around the tooling boundaries.</a> <a data-t="165">It was never fun to see a Hugo theme that just kind of didn't work for how you wanted</a> <a data-t="169">to lay it out, or something that worked in bookdown that you couldn't put into blogdown.</a> <a data-t="175">And so as powerful as these tools were, it was clear that the separation of the communities</a> <a data-t="180">that we were trying to meld together left some gaps to make this approachable to everyone.</a></p>

<h3>Quarto unifies the ecosystem</h3>

<p><a data-t="189">But here we are today.</a> <a data-t="191">As Mine and Julie have explained, and hopefully as everyone is getting so excited about, Quarto</a> <a data-t="198">builds on the learnings of the past decade.</a> <a data-t="201">It unifies teams across language boundaries and procedural differences.</a> <a data-t="207">And in the same vein, Quarto can also unify the way you consider building those books,</a> <a data-t="213">blogs, websites, or all sorts of other complex content, so you can focus on enabling users</a> <a data-t="220">to discover, consume, and navigate in the ways that you would like to help them.</a></p>

<p><a data-t="226">And by doing so in a single, unified toolchain, it really does give you an unparalleled ability</a> <a data-t="232">to control the processing and ultimately the production of the content that you're working on.</a></p>

<blockquote><a data-t="226">And by doing so in a single, unified toolchain, it really does give you an unparalleled ability</a> <a data-t="232">to control the processing and ultimately the production of the content that you're working on.</a></blockquote>

<p><a data-t="239">Now one thing that some of you may be thinking is, this seems like a lot.</a> <a data-t="245">One tool that can do so much.</a> <a data-t="247">How may I even start learning all of the things that it can do?</a> <a data-t="252">So today, I'm going to do something that hopefully the presentation gods will be with me, and</a> <a data-t="258">I'm going to show you just how easy it is to get going with Quarto and do some pretty</a> <a data-t="263">cool stuff.</a> <a data-t="264">And hopefully also demonstrate about some of the thoughtfulness that the Quarto team</a> <a data-t="268">has put into its design.</a></p>

<h3>Live demo: creating a website</h3>

<p><a data-t="309">So, let's drop into RStudio.</a> <a data-t="313">Now, one of the things that you can always do in RStudio is if you want to go play around</a> <a data-t="318">with a somewhat recent version of RStudio, you can go in, create a new directory, and</a> <a data-t="321">Quarto is front and center.</a> <a data-t="325">But the Quarto team has wanted to make sure that Quarto is available even if you're not</a> <a data-t="329">using RStudio, if you're using VS Code or any other editor.</a> <a data-t="335">And so many to all the things that you can do in the IDE, you can also do on the command line.</a></p>

<p><a data-t="339">In this case, we want to get going with a website.</a> <a data-t="347">So in this case, I can, from the terminal, say Quarto, create project, type equals website.</a> <a data-t="351">And what this is going to do is start me off with a scaffold.</a> <a data-t="358">It has a couple of these QMD files that you've been hearing about, and it has this Quarto.yaml.</a> <a data-t="364">This is the magic, the ability to say I have the area that I'm going to write my content,</a> <a data-t="369">and I have a hub where I can control how I want that content to be rendered.</a></p>

<p><a data-t="417">Now, you could go build this today in blogdown, no problem.</a> <a data-t="422">But where things start to get exciting to me in Quarto is just how flexible you can be when you say,</a> <a data-t="428">what if I want to move things around?</a> <a data-t="431">So, in this case, you can see how this is laid out.</a> <a data-t="433">I have my website.</a> <a data-t="434">I have a title that I can change.</a> <a data-t="436">I also have a nav bar, so that top strip is a nav bar.</a> <a data-t="440">But if I instead said I want to move my content so it's on the left, right, which is actually one of the big things</a> <a data-t="446">that would differentiate for like a book, it's quite simple.</a> <a data-t="449">I could just say this is a sidebar.</a></p>
