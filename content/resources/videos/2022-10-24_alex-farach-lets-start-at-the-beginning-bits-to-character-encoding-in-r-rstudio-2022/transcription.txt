<h3>What is character encoding?</h3>

<p><a data-t="39">So what do we mean by bits to character</a> <a data-t="39">encoding, especially in relationship to NLP?</a> <a data-t="44">Well, computers don't speak the same language</a> <a data-t="44">as we do.</a> <a data-t="44">If we want to use computers as a process language, we need some way to encode</a> <a data-t="48">language so that computers understand and vice versa.</a> <a data-t="53">Both the encoding and decoding</a> <a data-t="53">process needs to be identical on both ends or the human computer translation falls apart.</a> <a data-t="59">The agreed upon method is called a protocol.</a></p>

<p><a data-t="64">So back to the question, how would you describe</a> <a data-t="64">the letter A to a computer?</a> <a data-t="68">Oh, we do so by using numbers, of course, to represent letters.</a> <a data-t="68">So instead of A, we could have one, B could be two, and so on.</a> <a data-t="74">We assign a number to represent</a> <a data-t="74">each unique character or symbol we want to communicate.</a> <a data-t="79">Computer store numbers in binary,</a> <a data-t="79">and all signals inside a computer have two possible values, zero or one.</a> <a data-t="84">Each of the</a> <a data-t="84">zero one pieces of information are called bits.</a> <a data-t="89">One bit can represent two value, two bits can</a> <a data-t="89">represent four, three bits represent eight, and so on.</a> <a data-t="94">So how many bits is needed to represent</a> <a data-t="94">256 values?</a> <a data-t="101">Using the logic just discussed, it's eight bits.</a></p>

<h3>A brief history of ASCII, Latin-1, and UTF-8</h3>

<p><a data-t="101">When computers were first developed,</a> <a data-t="101">the decision was made that a byte was the next standard unit above a bit.</a> <a data-t="106">A byte is defined as</a> <a data-t="106">eight bits.</a> <a data-t="112">In the 1960s, computer developers had one byte to work with.</a> <a data-t="112">They developed a</a> <a data-t="112">protocol to fit into this one byte called ASCII.</a> <a data-t="118">This protocol can assign a unique number to all</a> <a data-t="118">letters in the English language, plus numbers, symbols, and control characters.</a> <a data-t="123">Everything fits</a> <a data-t="123">into this one byte budget, and even better, seven of the available eight bits were needed.</a></p>

<p><a data-t="128">Over time, the need to fit more information grew larger, but luckily, there was that extra bit to</a> <a data-t="133">work with.</a> <a data-t="133">Things got standardized into Latin one, which uses a full eight bits available in</a> <a data-t="138">the single byte.</a> <a data-t="145">So then after that, a new standard was developed, UTF-8.</a> <a data-t="145">Each character in UTF-8 is</a> <a data-t="145">represented using a prefix U followed by a four digit hexadecimal number.</a> <a data-t="151">The difference between</a> <a data-t="151">binary and hexadecimal numbers are out of scope for this brief talk.</a> <a data-t="155">What's important to note</a> <a data-t="155">here though is that UTF-8 converts a code point or a single character in Unicode into a set of one</a> <a data-t="161">to four bytes and can encode all one million plus Unicode code points.</a></p>

<h3>Character encoding in R</h3>

<p><a data-t="168">Character strings in R can</a> <a data-t="168">be encoded in Latin one, UTF-8, or as raw bytes.</a> <a data-t="173">These declarations can be read or changed using</a> <a data-t="173">the encoding function.</a> <a data-t="179">The concept of native encoding is important here.</a> <a data-t="179">Natively encoded</a> <a data-t="179">strings are strings written in whatever code page the user is using.</a> <a data-t="185">As of the 4.2.0 release, R uses</a> <a data-t="185">UTF-8 as the native encoding.</a> <a data-t="191">Prior to this release, that was not the case because Windows</a> <a data-t="191">versions prior to Windows 10 didn't allow it.</a> <a data-t="197">Post Windows 10, this was allowed and R was updated to</a> <a data-t="197">take advantage of this feature.</a> <a data-t="203">This is an exciting change for us focusing on NLP and R since it</a> <a data-t="203">mitigates a lot of potential problems for common NLP tasks.</a></p>

<p><a data-t="208">It's also helpful to keep this in mind</a> <a data-t="208">when using common functions like read.csv where character encoding can be set.</a> <a data-t="216">On Linux and</a> <a data-t="216">macOS, the native encoding has been UTF-8 so this isn't an issue there.</a> <a data-t="222">Native encoding is why we</a> <a data-t="222">see unknown here as well.</a> <a data-t="227">For example, we see coffee as unknown even though it's clearly ASCII</a> <a data-t="227">compliant.</a> <a data-t="232">ASCII strings will never be marked with a declared encoding since their representation</a> <a data-t="232">is the same in all supported encodings.</a> <a data-t="238">The sys.getlocal function will tell you what your</a> <a data-t="238">native encoding is.</a> <a data-t="244">The localization information function is another helpful function.</a> <a data-t="244">And lastly,</a> <a data-t="244">we have the iconvert function which uses system facilities to convert a character vector between</a> <a data-t="250">encodings.</a></p>
