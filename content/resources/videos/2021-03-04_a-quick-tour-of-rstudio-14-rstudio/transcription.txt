<p><a data-t="0">In this video, we'll be showing you the latest in </a><a href="/software/rstudio/">RStudio</a><a data-t="0"> 1.4 and how these features can</a> <a data-t="11">benefit you and your team for credible, agile, and durable data science results.</a></p>

<h3>Visual editor and citations in R Markdown</h3>

<p><a data-t="19">The Visual Editor is one of the more exciting new options, making working with code in </a><a href="/software/rmarkdown/">RMarkdown</a> <a data-t="24">easier than ever.</a> <a data-t="26">This new interface allows you to compose actual RMarkdown using the familiar visual interface.</a> <a data-t="32">With it, you have easy access to format commands, inserting images, tables, just to name a few</a> <a data-t="41">of the options available in the visual pane.</a></p>

<p><a data-t="46">Continuing our theme for ease of use when authoring, you also have the ability to find</a> <a data-t="50">and insert citations from multiple sources, including integration with Zotero, a popular</a> <a data-t="56">free and open-source reference manager.</a> <a data-t="59">This not only makes the citation for you, but it also creates your bibliography, reducing</a> <a data-t="64">the time spent in busy work while providing research.</a></p>

<h3>Python support</h3>

<p><a data-t="68">Another thing we've spent quite a bit of time working on in RStudio 1.4 is Python, which</a> <a data-t="74">we've focused more on in our latest releases.</a> <a data-t="79">The largest feature now available is support for Python in the Environment pane.</a> <a data-t="85">This allows you to switch directly between R mode and Python mode in that pane to look</a> <a data-t="90">at data in different environments in R or immediately change the view to look at data</a> <a data-t="95">in different modules in Python.</a></p>

<p><a data-t="98">We can browse these Python modules in the pane or open it to view directly in our workspace.</a> <a data-t="104">This allows you to interact with data in Python in much the same way that you interact</a> <a data-t="109">with data in R.</a></p>

<p><a data-t="112">While RStudio is powered by R and will always be an R-first IDE, we will continue to work</a> <a data-t="117">to improve the ability to use Python alongside it, allowing teams to mix these languages</a> <a data-t="122">to suit their needs.</a></p>

<blockquote><a data-t="112">While RStudio is powered by R and will always be an R-first IDE, we will continue to work</a> <a data-t="117">to improve the ability to use Python alongside it, allowing teams to mix these languages</a> <a data-t="122">to suit their needs.</a></blockquote>

<p><a data-t="124">In addition, Python environment selections are now available, allowing you to select</a> <a data-t="129">the version of Python you would like to use or switch between at any moment, or use in</a> <a data-t="134">each project.</a> <a data-t="136">This is all powered by the </a><a href="/software/reticulate/">reticulate</a><a data-t="136"> package which enables these Python-based features</a> <a data-t="141">in RStudio 1.4.</a></p>

<h3>Editor enhancements</h3>

<p><a data-t="145">Another new feature that many users are excited about are rainbow parentheses, which helps</a> <a data-t="149">you to more easily view and decipher different sections of your code.</a> <a data-t="153">This ultimately aids in following deeply nested constructs, allowing you to view the different</a> <a data-t="158">levels and find the connected pairs.</a></p>

<p><a data-t="162">In addition, you can now use any monospace font you like to customize your coding experience,</a> <a data-t="168">allowing you to tailor the environment to your preference.</a></p>

<p><a data-t="172">Mainly designed for widescreen monitors, we also provide the ability to have multiple</a> <a data-t="176">source columns in the RStudio pane layout.</a> <a data-t="180">This allows you to take advantage of any space you may have while avoiding the opening of</a> <a data-t="184">endless windows on your screen.</a></p>

<p><a data-t="188">Also new in RStudio 1.4 is the command palette, which allows you to invoke any RStudio command</a> <a data-t="193">with just a few keystrokes, such as opening a new R Markdown document.</a> <a data-t="198">This also allows you to very easily find and learn custom shortcuts that you can use in</a> <a data-t="203">your everyday work and increase your efficiency.</a></p>

<h3>Data and configuration storage</h3>

<p><a data-t="207">One key feature that is available in both the open source and commercial server version</a> <a data-t="211">of RStudio IDE is the ability to customize where your data and configuration are stored</a> <a data-t="218">for both the users and the servers.</a> <a data-t="221">This opens up workflows for mounting configuration within containers, helping to solve the long</a> <a data-t="226">standing problem of users filling up their home directories and guiding content to where</a> <a data-t="231">it should appropriately go.</a></p>

<h3>RStudio Pro edition features</h3>

<p><a data-t="234">Moving from the open source, let's also walk through the new features that are now available</a> <a data-t="238">with the Pro Edition of RStudio 1.4.</a></p>

<p><a data-t="243">Undoubtedly the most requested and important of our recent additions is the ability to</a> <a data-t="246">authenticate RStudio Server Pro now using SAML.</a> <a data-t="251">This option along with Open IDE enables single sign-on for users and systems, with standardization</a> <a data-t="257">that provides fast yet reliable security.</a></p>

<p><a data-t="261">In addition to load balancing with the local launcher, project sharing with the launcher</a> <a data-t="265">is also now available.</a> <a data-t="267">Previously both of these popular features were not available to use at the same time,</a> <a data-t="271">but now even when you've launcherized an environment, you can still share your projects with others.</a> <a data-t="277">While it still requires some setup for permissions, these are both big features with the commercial</a> <a data-t="282">server version of RStudio IDE that are now available to use at the same time.</a></p>

<p><a data-t="288">Of note for those who need to request a GPU with SLURM, this is now possible with custom</a> <a data-t="294">SBatch parameters.</a></p>

<p><a data-t="297">Another nice new feature for the commercial server version of the RStudio IDE is the ability</a> <a data-t="302">to run Visual Studio Code sessions.</a> <a data-t="305">This is currently in beta and is great for those who want to run longer form Python programs</a> <a data-t="310">while working with other teams and projects within the RStudio Server environment.</a></p>
