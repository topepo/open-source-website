<p><a data-t="67">So why use brand.yml?</a> <a data-t="67">Let's say you're a data person in an organization and as part of your job you create reports,</a> <a data-t="73">apps, plots, dashboards, and other things.</a> <a data-t="80">All these artifacts that people at your organization</a> <a data-t="80">and maybe even people outside of your organization use.</a> <a data-t="86">And your company has particular branding,</a> <a data-t="86">it has colors, logos, fonts that it uses in its materials.</a> <a data-t="92">That branding might even be</a> <a data-t="92">immediately recognizable to people outside of your company.</a> <a data-t="101">So you as a diligent data professional</a> <a data-t="101">try to apply your company's branding across all your plots and dashboards and apps.</a></p>

<h3>Structure of a brand.yml file</h3>

<p><a data-t="281">So at this point you might have a couple of questions.</a> <a data-t="288">How do we structure this yaml file?</a> <a data-t="294">And after we've created the yaml file, how do we actually</a> <a data-t="294">apply it to our outputs to brand them?</a> <a data-t="304">So let's look at the yaml file structure first.</a></p>

<p><a data-t="304">So I'm going to give you an overview of the file structure and explain how you'd go about</a> <a data-t="308">building up your own brand.yaml file.</a> <a data-t="314">But if this seems like a lot of manual work, know that towards</a> <a data-t="314">the end of this presentation I'll show you how to create a brand.yaml with the help of an LLM.</a></p>

<p><a data-t="290">So yaml is a language often used to represent structured data in a human-readable format.</a> <a data-t="296">You often see yaml used for things like configuration files.</a> <a data-t="302">You don't need to know</a> <a data-t="302">a lot about yaml to get started creating yaml files like brand.yaml.</a></p>

<p><a data-t="329">One thing to know is that yaml uses key-value pairs.</a> <a data-t="337">So the key in this case right here is</a> <a data-t="337">meta and then this whole thing is the value.</a> <a data-t="341">And we have another key-value pair within that.</a></p>

<p><a data-t="348">And that's all you really need to know about how yaml works.</a> <a data-t="348">So now let's talk about the specifics</a> <a data-t="348">of a brand.yaml file.</a> <a data-t="357">Brand.yaml files require a particular format to brand your outputs like</a> <a data-t="357">Shiny apps and Quarto documents.</a> <a data-t="366">There are four main components.</a> <a data-t="366">Meta, which contains identity</a> <a data-t="366">information like the name of your company.</a> <a data-t="375">Logo, which is where you specify files or links to your</a> <a data-t="375">brand's logo at various sizes.</a> <a data-t="381">Color, which is where you define the colors in your brand's color</a> <a data-t="381">palette and how those colors are used.</a> <a data-t="388">And typography, which is where you define fonts</a> <a data-t="388">and other typographic choices.</a></p>

<p><a data-t="398">So again a brand.yaml file will need meta, logo, color, typography.</a> <a data-t="405">You can also specify defaults, which is where you can add additional context-specific settings</a> <a data-t="405">that might be options used by like Bootstrap in Quarto or Shiny.</a></p>

<p><a data-t="419">So let's take a closer look at the color and typography sections.</a> <a data-t="419">So first, color.</a> <a data-t="419">So the</a> <a data-t="419">color section in brand.yaml consists of two main parts.</a> <a data-t="425">There's palette, which is a set of named</a> <a data-t="425">colors specific to your brand.</a> <a data-t="431">So in this case, the key value pairs are a name of a color,</a> <a data-t="432">which this could be anything you want, and then the hex code for that color.</a></p>

<p><a data-t="438">So this brand defines a black, blue, neutral, red, and violet.</a> <a data-t="449">The second part is the theme colors.</a> <a data-t="449">And this is where you assign those specific colors to various UI elements.</a> <a data-t="456">So here we're saying</a> <a data-t="456">we want to use our neutral color for background.</a> <a data-t="464">So if you were building up this file from scratch,</a> <a data-t="464">you would first want to define the colors in your palette by taking a look at your</a> <a data-t="469">brand guidelines and figure out what colors are in your brand's color palette to define those colors</a> <a data-t="476">and then map those brand colors to particular UI elements.</a> <a data-t="481">How do you want those colors</a> <a data-t="481">to be used?</a> <a data-t="487">Where do you want them to appear?</a></p>

<p><a data-t="489">Okay, so now typography.</a> <a data-t="495">Like color, typography has two sections.</a> <a data-t="495">First, you define the fonts used by your brand.</a> <a data-t="517">And you can specify either local or online font sources.</a> <a data-t="517">So this brand.yaml uses Google fonts.</a></p>

<p><a data-t="485">Then you take those fonts and define how they are used by different typographic elements.</a> <a data-t="492">So in this case, we're using this font as the base font.</a> <a data-t="498">And then for headings, we're using</a> <a data-t="498">Montserrat with a particular weight and also assigning a color.</a> <a data-t="505">Great.</a> <a data-t="505">So, so far,</a> <a data-t="505">we've talked about why you might want to use brand.yaml, how to structure a brand file.</a></p>
