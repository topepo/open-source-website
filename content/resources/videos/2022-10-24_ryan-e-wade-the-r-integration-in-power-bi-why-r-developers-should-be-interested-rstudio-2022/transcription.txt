<h3>Introducing Power BI</h3>

<p><a data-t="264">Power BI is a data visualization tool.</a> <a data-t="270">In order to do data visualization, you need data.</a> <a data-t="270">And the data model that Power BI prefer</a> <a data-t="279">is a data model known as the star schema data model.</a> <a data-t="279">The star schema data model is made</a> <a data-t="286">up of two types of tables, a dimension table and a fat table.</a> <a data-t="286">The fat table you see here</a> <a data-t="292">in the center, and the fat table stores measures.</a> <a data-t="292">You can think of a measure as something that</a> <a data-t="297">measures and a business activity.</a></p>

<p><a data-t="305">In this particular simple example, we see that the</a> <a data-t="305">measures are price and quantity.</a> <a data-t="313">And then the fat table is connected to the dimension table.</a> <a data-t="321">And in the dimension tables, you have dimensions grouped by category.</a> <a data-t="321">So you see in this particular</a> <a data-t="327">example, we have a store dimension, a employee dimension, and so on and so forth.</a> <a data-t="335">In those different categories, you have dimensions that you can group your</a> <a data-t="340">measures and your fat table by.</a></p>

<p><a data-t="340">Another thing I want to point out about Power BI from a</a> <a data-t="345">data standpoint is how the data is stored in Power BI.</a> <a data-t="345">Power BI uses an engine called</a> <a data-t="346">a VertiPaq engine.</a> <a data-t="353">And the takeaway I want you to get from that is that the VertiPaq</a> <a data-t="353">engine, one, stores the data in memory using what is called column store indexing.</a> <a data-t="360">And</a> <a data-t="360">the takeaway you should get is that storing data in that fashion does two things.</a> <a data-t="366">It makes</a> <a data-t="366">performance, you know, some complex calculation relatively faster than what it will perform</a> <a data-t="375">if you are using another tool.</a> <a data-t="375">And in addition to that, the column store indexing increases</a> <a data-t="380">the memory footprint of your data by a lot.</a> <a data-t="380">In some cases, up to a tenth of the size of</a> <a data-t="387">the original data set.</a></p>
