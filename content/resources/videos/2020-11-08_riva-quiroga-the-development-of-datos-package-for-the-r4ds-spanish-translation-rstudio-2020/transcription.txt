<h3>How the translation project started</h3>

<p><a data-t="110">So the story I'm going to tell you starts in 2017, and as many stories in the R community,</a> <a data-t="115">it started on Twitter. So </a><a href="/people/hadley-wickham/">Hadley Wickham</a><a data-t="115"> announced that R for data science was going to be translated</a> <a data-t="122">into Chinese, French, Korean, Portuguese, Russian, Serbian. Cool. So Marcella Alfaro</a> <a data-t="127">from the Costa Rican R community asked, well, what about Spanish? And Hadley said, strangely,</a> <a data-t="133">no. And that's very strange, because Spanish is the language that had the second most amount</a> <a data-t="139">of native users, so there's a lot of Spanish-speaking R users that want to read this.</a></p>

<p><a data-t="147">So in 2018, </a><a href="/people/edgar-ruiz/">Edgar Ruiz</a><a data-t="147"> from RStudio, he forked the original repository and decided to start</a> <a data-t="155">the translation, but he didn't do nothing at all, because this is very difficult to</a> <a data-t="159">do, translate a book. So he needed a community, and he was alone in this. And a couple of</a> <a data-t="165">months later, at our OpenSciConf, Laura asked, Hadley, what about the translation? And he</a> <a data-t="171">said, no, it's not going to happen. O'Reilly is not interested in Spanish. But you can</a> <a data-t="175">do it if you want. So she spread the news around the RLadies network and on Twitter,</a> <a data-t="180">and the next week, we have the first conference call between all the people that were interested</a> <a data-t="185">in this project. And just two days after the video conference, we have our X sticker, because,</a> <a data-t="192">you know, if you don't have an X sticker, you're not a real R project. So we became</a> <a data-t="198">a sticker-driven project. We have a team of stickers, so we can start.</a></p>

<p><a data-t="202">So everyone started forking the repository, and 21 people were involved in the translation</a> <a data-t="210">from different backgrounds, from different countries, and we have another around 30 people</a> <a data-t="216">that review those translations. So this was about a 50-people effort project. So all these</a> <a data-t="222">people speak different varieties of Spanish, which is very useful, because you don't want</a> <a data-t="227">to be rude. You know, translated something in some way, and that's very offensive in</a> <a data-t="231">another part of Latin America. People from different backgrounds. There were people that</a> <a data-t="236">were from statistics, from biology, from social science, from the humanities. Also people</a> <a data-t="241">with different R experience, experts and novices, and people with different GitHub</a> <a data-t="246">experience, and this was very risky, because we forked the repository and the people forked</a> <a data-t="252">our fork, so you have around 30 people trying to manage how to contribute.</a></p>

<h3>Decisions about infrastructure and translation</h3>

<p><a data-t="259">So in this first conference, we decided two things. First, that we want to translate the</a> <a data-t="263">text and the datasets that are used in the book, because they're useful not only for</a> <a data-t="270">the book, but for teaching R, and also which infrastructure we were going to use. So we</a> <a data-t="276">used mainly GitHub and Slack. In GitHub, we have our fork of the book, we have our repository</a> <a data-t="282">with all the documentation of the project, and the repository for the package. And we</a> <a data-t="288">have responsibilities. Edgar was leading the development of the package, but I was involved</a> <a data-t="296">with taking care that we didn't break the repository, and I was involved with the translation</a> <a data-t="303">and coordinating the whole process. So GitHub is very useful for translation, because you</a> <a data-t="308">can have both versions of the text, and reviewers can put comments in between, so they can make</a> <a data-t="314">suggestions how to improve the translation. And Slack was also very useful, well, we can</a> <a data-t="320">organize this, but also because of polls, because we have to make many decisions about</a> <a data-t="326">how we are going to translate some stuff. For example, we have empty cars. We have to</a> <a data-t="332">translate the cars, and we can say cars in three different ways in Spanish. So we have</a> <a data-t="336">to decide autos, coches, or carros. So autos won, and now we have emete autos.</a></p>

<p><a data-t="343">And there's some other very difficult stuff to decide. For example, we have the pipe,</a> <a data-t="348">and in Spanish, all nouns have a grammatical gender, so we have to decide if the pipe was</a> <a data-t="353">going to be masculine or feminine. So we made a poll, and everyone voted that the pipe was</a> <a data-t="359">masculine, except me, because for me the pipe was a she. So now I have to change my mind</a> <a data-t="366">about how I thought about the pipe.</a></p>

<p><a data-t="372">So we decided that we want to translate the</a> <a data-t="378">text and the datasets, and for the text, there were previous experiences that were very helpful.</a> <a data-t="386">For example, the Carpentries has translated some lessons to Spanish, and they have this</a> <a data-t="393">guide with the agreements about how they are translating technical words, so we get inspired</a> <a data-t="399">about that. And also the Programming Historian, it's a website where we publish peer-reviewed</a> <a data-t="406">tutorials that help people from the humanities to learn digital tools, and has a version</a> <a data-t="411">in Spanish and French. So there are, like, guidelines for reviewers, for translation,</a> <a data-t="416">for authors, and that was very useful to get inspired by that.</a></p>

<h3>Translating the datasets with Datalang</h3>

<p><a data-t="421">And what about the dataset? There were two possible approaches. What we wanted is, for</a> <a data-t="426">example, we have the diamonds dataset, we want diamantes, and we wanted to have, like,</a> <a data-t="431">the variable translated and the name of the different values. And one possible approach</a> <a data-t="437">was just to, for example, rename the variables and then translate the name of the values.</a> <a data-t="442">But another approach was using Datalang, a package that Edgar Ruiz developed, that has</a> <a data-t="447">this function that's called translated data, that takes a YAML file with the translation,</a> <a data-t="453">the specification for the translation, and as a result, you get your dataset translated.</a></p>

<p><a data-t="461">So how it works, you have this YAML file, so you put which is the source packet, the</a> <a data-t="467">original dataset in English, how it's going to be named in Spanish, you specify how you</a> <a data-t="473">want to translate the variables, for example, price became precio, and the description.</a> <a data-t="479">You also explain which values you want to translate it, and you also translate the help.</a> <a data-t="486">So you have documentation in Spanish also. So you not only get the dataset translated,</a> <a data-t="493">but also you get the help page for the dataset. So that's very useful for people who are learning.</a></p>

<p><a data-t="501">So this approach of translating the book and the dataset using this packet make very easy</a> <a data-t="507">to contribute, because you can participate in a packet just by knowing how to edit a</a> <a data-t="512">YAML file.</a></p>

<h3>Finishing the package and going to CRAN</h3>

<p><a data-t="515">So last year, when we were developing the</a> <a data-t="521">packet, we sent a proposal to Luz, but the packet wasn't finished, so it became a conference-driven</a> <a data-t="528">packet. <p><a data-t="568">So we were very happy with that, but one day we were trying to do something, and we</a> <a data-t="573">realized we were not able to call the dataset that way, and we were, like, very stuck and</a> <a data-t="579">hardly remember that this function exists, delayed assign, and that helped us to solve</a> <a data-t="584">that.
