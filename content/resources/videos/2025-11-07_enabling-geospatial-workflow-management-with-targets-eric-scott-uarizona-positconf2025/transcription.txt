<h3>What targets does</h3>

<p><a data-t="53">Targets is a workflow management package for R, and it allows you to define a workflow in a series of steps.</a> <a data-t="59">So here in this kind of contrived example, we're reading in some data about penguins.</a> <a data-t="65">We are maybe fitting a model that, let's say, looks for flipper length as a function of sex,</a> <a data-t="72">and then we're plotting the model results along with the raw data.</a> <a data-t="75">And targets can figure out the relationship between those steps automatically.</a></p>

<p><a data-t="79">And then if you were to change something, like let's say I change the flipper model function to,</a> <a data-t="84">let's say, add a random effect of island.</a> <a data-t="87">Targets knows that when you run that workflow again, you can safely skip the file and data steps</a> <a data-t="94">and only needs to rerun the model step and anything downstream of it.</a> <a data-t="98">And this saves you a ton of time and makes your workflow more reproducible.</a> <a data-t="102">And it also allows you to do things like run targets that can be run independently</a> <a data-t="107">in parallel on multiple cores or on HPC.</a></p>

<p><a data-t="131">So let me explain a little bit about where this error is coming from.</a> <a data-t="134">When you read data into R with Terra, it creates these R objects that don't actually contain any data.</a> <a data-t="141">They instead contain a pointer to where that data exists.</a> <a data-t="145">And in this example, that's in my computer's memory.</a> <a data-t="148">And so if you were to save that R object out to disk as a .rds file</a> <a data-t="153">and then read it back in in a fresh R session, the pointer is still there.</a> <a data-t="157">It's pointing to the same place, but the data is not there anymore.</a> <a data-t="160">And so that pointer is not valid, and you get this error message.</a> <a data-t="164">And this is exactly what targets is doing by default.</a></p>
