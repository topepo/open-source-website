<h3>The pins package recap</h3>

<p><a data-t="251">And there's actually great news for many of you that are already</a> <a data-t="251">familiar with this particular problem, which is Darla, knowing that she's a competent</a> <a data-t="257">data scientist, knows that there's a package called Pins, which we've been developing in</a> <a data-t="263">the R community for the last year and a half, and allows her to do tasks like this, allows</a> <a data-t="268">her to retrieve data and upload data into the cloud.</a> <a data-t="274">So what is the Pins package as</a> <a data-t="274">a quick recap?</a> <a data-t="279">Well, first of all, the Pins package allows you to get a remote resource</a> <a data-t="279">into your local machine.</a> <a data-t="284">So say if you have like a remote URL with a CSV file, you can</a> <a data-t="284">create a local cache on your machine very easily.</a> <a data-t="290">Also, once you're connected to those</a> <a data-t="290">remote services like Kaggle or GitHub or RStudio Connect, you can also search those</a> <a data-t="296">services for datasets that are interesting to you, all within the Pins package.</a> <a data-t="301">And last</a> <a data-t="301">but not least, and especially important for this particular problem that we're trying</a> <a data-t="306">to solve, is the Pins package allows you to share datasets in different cloud providers</a> <a data-t="311">like Google, Kaggle, Azure, and RStudio Connect.</a> <a data-t="318">And you can do that quite easily.</a> <a data-t="318">And this</a> <a data-t="318">is something that Darla is already familiar with.</a> <a data-t="322">She knows that she can share datasets</a> <a data-t="322">with other colleagues that also use R and the Pins package.</a></p>

<p><a data-t="330">The Pins package is actually quite simple.</a> <a data-t="335">You install it from CRAN, you load the library,</a> <a data-t="335">and then you can ask it to pin a dataset, either an R object or a remote resource or</a> <a data-t="342">a local file into your local board, which is what we're doing in this particular example.</a> <a data-t="347">And then after you pin that resource locally, you can use your favorite tools to process</a> <a data-t="351">it.</a> <a data-t="356">Like in this case, we're just reading a remote JSON file, saving it locally, and</a> <a data-t="356">then using JSON Lite to read it.</a> <a data-t="361">Now, that's the simple use case of the Pins package to</a> <a data-t="361">bring things into your local machine.</a> <a data-t="365">But you can also use the Pins package to push</a> <a data-t="365">things out of your local machine.</a> <a data-t="372">And what Darla could do here potentially is register</a> <a data-t="372">a board, which is a concept from the Pins package.</a> <a data-t="377">The board that she registers could</a> <a data-t="377">be GitHub, DigitalOcean, Google Cloud, RStudio, Kaggle, Azure, or AWS S3.</a> <a data-t="387">And she can basically</a> <a data-t="387">push the dataset from something as simple of an array of 10 numbers by saying, pin one</a> <a data-t="393">to 10, name equals numbers, and then board equals, say, Google Cloud, whatever.</a></p>

<p><a data-t="400">And the dataset that she has locally can be pushed to be shared to those particular services.</a> <a data-t="407">And if she needs to get it back at any point in time, she can just say, pin get, and the</a> <a data-t="410">dataset will come back from those services into her local machine again.</a></p>

<p><a data-t="415">You can do a little bit more complex stuff.</a> <a data-t="421">Like you can say, suppose that you're collaborating</a> <a data-t="421">in an image processing project and you have ImageNet, you can retrieve a smaller version</a> <a data-t="426">of ImageNet called TinyImageNet by saying, pin this remote resource locally into my local</a> <a data-t="431">machine and we'll call it ImageNet.</a> <a data-t="437">And then what you can do is you can also say, oh, give</a> <a data-t="437">me back ImageNet, and now I want to push it to perhaps Kaggle, because maybe you did some</a> <a data-t="443">cleaning of the data or some processing.</a> <a data-t="447">So all of these are tasks that the Pins package</a> <a data-t="447">supports, including finding.</a> <a data-t="454">You can just say, pin find, and it will find the different</a> <a data-t="454">datasets that are available in the different boards.</a></p>
