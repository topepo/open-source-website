<h3>Why evals matter</h3>

<p><a data-t="154">So again, you can build an app like this</a> <a data-t="156">in something like 20 to 30 lines now.</a> <a data-t="159">These </a><a href="/software/ellmer/">ellmer</a><a data-t="159"> and ShinyChat packages are super powerful.</a> <a data-t="161">And it's really cool how far you can get</a> <a data-t="163">in such a short amount of time.</a> <a data-t="166">At the same time, once you've started to build out</a> <a data-t="169">a demo of an AI app, taking that app to production</a> <a data-t="172">requires you to be able to test it</a> <a data-t="174">and iterate on it quickly.</a> <a data-t="176">And it's difficult to do that without an ability</a> <a data-t="179">to evaluate quality and debug issues.</a></p>

<p><a data-t="182">This is where the </a><a href="/software/vitals/">Vitals</a><a data-t="182"> R package comes in,</a> <a data-t="185">which is a toolkit for LLM evaluation in R.</a> <a data-t="189">Vitals is an R port of a framework in Python</a> <a data-t="193">called Inspect AI.</a> <a data-t="195">So if you're a Pythonista, for the most part in this talk,</a> <a data-t="198">I'm going to refer to Vitals.</a> <a data-t="200">But Inspect AI uses the same language, same framework.</a> <a data-t="203">This framework is widely adopted</a> <a data-t="206">by many of the Frontier labs</a> <a data-t="208">as they're building the models themselves.</a></p>

<p><a data-t="211">And also on the Python side,</a> <a data-t="213">we have peers to ellmer and ShinyChat,</a> <a data-t="215">which are Chatless and appropriately named ShinyChat.</a> <a data-t="219">But I'll focus on Vitals throughout the rest of this talk.</a></p>

<p><a data-t="224">In the rest of the talk,</a> <a data-t="226">I want to remind you of two things.</a> <a data-t="228">The first is that if you're building an LLM app,</a> <a data-t="231">you really should be running evals on it.</a> <a data-t="234">And the second thing is that</a> <a data-t="236">if you have managed to build an LLM app,</a> <a data-t="239">you really can do it.</a> <a data-t="241">Building evals is straightforward</a> <a data-t="243">once you have an app built with ellmer.</a></p>

<p><a data-t="247">So let's go ahead and start</a> <a data-t="249">on the really-you-really-should-be side.</a> <a data-t="251">I hope to convince you</a> <a data-t="254">that you really should be evaluating your LLM product.</a></p>

<p><a data-t="255">This is a quote from Hemal Hussain,</a> <a data-t="257">who is a leader in the AI eval space.</a> <a data-t="260">He writes,</a> <a data-t="261">Like software engineering,</a> <a data-t="263">success with AI hinges on how fast you can iterate.</a> <a data-t="267">So whether you're a data scientist or a software engineer,</a> <a data-t="270">I don't know if immediately you agree with that statement,</a> <a data-t="273">but I think I might be able to convince you</a> <a data-t="276">that at some level you do</a> <a data-t="278">because you adopt tooling that allows you to iterate quickly.</a></p>

<blockquote><a data-t="261">Like software engineering,</a> <a data-t="263">success with AI hinges on how fast you can iterate.</a></blockquote>

<p><a data-t="282">So in software engineering,</a> <a data-t="284">we adopt tools that allow us to make changes,</a> <a data-t="287">like our code editors,</a> <a data-t="289">and we have autocomplete, maybe LLM-assisted autocomplete,</a> <a data-t="292">and now these agentic editors like Cloud Code</a> <a data-t="295">or </a><a href="/software/positron/">Positron</a><a data-t="295"> Assistant.</a> <a data-t="297">But in addition to these tools to make changes,</a> <a data-t="300">we have tools to evaluate quality, like unit tests.</a> <a data-t="303">So if I go and make some changes</a> <a data-t="305">in an R package I'm working on,</a> <a data-t="307">I can both test that that works as I expect it to</a> <a data-t="310">and didn't break anything else in the process.</a> <a data-t="313">And we also have tools to help us debug issues,</a> <a data-t="316">so like the debugger and rep practices,</a> <a data-t="318">and so on and so forth.</a></p>

<p><a data-t="321">In AI, we also have all sorts of tools</a> <a data-t="324">that allow us to make these sorts of changes to our apps.</a> <a data-t="327">So we can engineer the system prompt,</a> <a data-t="329">we can put together this rag tool,</a> <a data-t="332">like was under the hood in this conf chatbot app,</a> <a data-t="335">and so on and so forth,</a> <a data-t="337">and there's a lot of effort in this part of the space.</a> <a data-t="340">What about those other two steps?</a> <a data-t="342">It's mostly like a vibe check.</a> <a data-t="344">So I make some changes to the prompt,</a> <a data-t="346">and I ask a couple questions,</a> <a data-t="348">and I'm like, that seems reasonable.</a> <a data-t="351">And so we have a look-see,</a> <a data-t="353">and if you've tried to bring your AI demos to production before,</a> <a data-t="357">you know that this can kind of become</a> <a data-t="359">a whack-a-mole sort of process,</a> <a data-t="361">where you might resolve one issue,</a> <a data-t="363">and at the same time, you accidentally introduce another one.</a></p>

<p><a data-t="369">And so Vitals allows us to iterate more quickly</a> <a data-t="372">by giving us tools to evaluate quality and debug issues.</a> <a data-t="376">I'll show you in a moment that when you run evals with Vitals,</a> <a data-t="379">you get these higher-level summaries</a> <a data-t="381">of how well your app is performing on different axes,</a> <a data-t="384">and when there are issues,</a> <a data-t="386">you have the ability to drill down in the details</a> <a data-t="388">to debug what went wrong.</a></p>

<h3>Building an eval with Vitals</h3>

<p><a data-t="393">So hopefully I've convinced you that this is something worth doing.</a> <a data-t="396">Now I want to convince you that if you're a user of ellmer</a> <a data-t="400">and you've made a chatbot already,</a> <a data-t="402">you do have the ability to make some evals as well.</a> <a data-t="406">And that's because we designed Vitals</a> <a data-t="409">with ellmer specifically in mind,</a> <a data-t="411">so that once you have an app,</a> <a data-t="413">you can just plug it right into Vitals,</a> <a data-t="415">and it's plug-and-play.</a></p>

<p><a data-t="420">To talk through that process,</a> <a data-t="422">we'll talk through the three pieces that compose an evaluation,</a> <a data-t="425">which is a dataset, a solver, and a scorer.</a> <a data-t="428">And to do so, we'll use that app from SAM</a> <a data-t="431">that is the conf chatbot app.</a></p>

<p><a data-t="434">So first let's talk about a dataset.</a> <a data-t="437">Minimally, datasets are just a data frame</a> <a data-t="440">that have two columns.</a> <a data-t="442">The first column is an input,</a> <a data-t="444">and this is something that a user might ask of your app.</a> <a data-t="448">So for example, in that last slide,</a> <a data-t="450">I said, are there any talks about evals?</a> <a data-t="453">And then the second column in that data frame is a target,</a> <a data-t="456">and a target gives you corresponding grading guidance</a> <a data-t="459">for that given input,</a> <a data-t="461">or maybe even the answer itself.</a> <a data-t="464">And so a target in that last example might be,</a> <a data-t="466">yes, Simon Couch will be giving a talk</a> <a data-t="469">called, Is That LLM Feature Any Good?</a></p>

<p><a data-t="473">So I know this is a conference about serious data science,</a> <a data-t="478">and I'm an R package developer,</a> <a data-t="480">so I take data structures really seriously and yada yada.</a> <a data-t="484">You can just use Google Sheets. It's okay.</a> <a data-t="493">It's really, like, sort of painful</a> <a data-t="495">to type out these long structured text response</a> <a data-t="498">inside of a call to data.frame or tbl or trbl</a> <a data-t="501">or whatever it may be.</a> <a data-t="503">This is, like, a perfect use case</a> <a data-t="505">for something like a Google Sheet or an Excel file.</a> <a data-t="509">So again, this is just a data frame with two columns.</a> <a data-t="512">One is the input, one is the target,</a> <a data-t="514">and then you can read that into R or Python after.</a></p>

<p><a data-t="518">So for example, another possible input you might ask is,</a> <a data-t="521">are there any sessions about causal inference?</a> <a data-t="524">And your grading guidance would be,</a> <a data-t="526">yes, there's this workshop called</a> <a data-t="528">Causal Inference in R by Malcolm and Lucy.</a></p>

<p><a data-t="532">So we have our data set.</a> <a data-t="534">The next step is building a solver.</a> <a data-t="539">If you've built a chat app with ellmer,</a> <a data-t="542">you know it looks roughly something like this.</a> <a data-t="544">You can load the ellmer package</a> <a data-t="546">and then initialize a client,</a> <a data-t="548">which is just a specification</a> <a data-t="550">of how you connect to a given LLM.</a> <a data-t="553">So if I run chat open AI,</a> <a data-t="555">I've connected by default to GPT 4.1,</a> <a data-t="558">but you can connect to Anthropic or Bedrock</a> <a data-t="561">or Snowflake or Databricks</a> <a data-t="563">or whatever it may be with ellmer.</a> <a data-t="565">And then you make your changes.</a> <a data-t="567">You write out a system prompt and you incorporate tools,</a> <a data-t="570">like in this conf chatbot app,</a> <a data-t="572">this is a rag tool, and so on and so forth.</a> <a data-t="575">And once you're finished with that,</a> <a data-t="577">you just drop it into live browser.</a> <a data-t="579">This is a function in ellmer that makes use of ShinyChat</a> <a data-t="582">and puts your client into a container</a> <a data-t="584">that looks something like chatgpt.com.</a></p>

<p><a data-t="588">So when I say that evaluation with vitals is plug-and-play,</a> <a data-t="593">I mean that you can just plop your client</a> <a data-t="596">right inside to vitals, and that's your solver.</a> <a data-t="599">So you're done.</a></p>

<p><a data-t="602">So up to this point now, the data set has two columns.</a> <a data-t="605">The data set has the input</a> <a data-t="607">and then the corresponding grading guidance</a> <a data-t="610">for that input, which is the target.</a> <a data-t="612">We pass the input from the data set to the solver,</a> <a data-t="616">so we ask the solver, are there any talks about evals,</a> <a data-t="619">and we get some response back.</a> <a data-t="621">And so up to this point, we now have three data points.</a> <a data-t="626">One way that we can score this</a> <a data-t="628">is by concatenating all these together in a string.</a> <a data-t="631">You might have known I was going here.</a> <a data-t="633">And we can say you're assessing a submitted answer</a> <a data-t="636">on a given task based on a criterion.</a> <a data-t="638">Here's the task.</a> <a data-t="640">Here's the submission, which is the response from the solver.</a> <a data-t="643">And here's the criteria, which is that target grading guidance.</a> <a data-t="646">Does the submission meet the criterion?</a> <a data-t="649">And you pass that along to some LLM.</a></p>

<p><a data-t="657">The first time I saw this,</a> <a data-t="659">I definitely had this sort of reaction,</a> <a data-t="661">like, are we really serious?</a> <a data-t="663">This is the way that we're going to do this?</a> <a data-t="665">There's some strong empirical evidence</a> <a data-t="667">that this is generally something that works</a> <a data-t="669">and tends to agree well with human raters</a> <a data-t="671">when these systems are designed well.</a> <a data-t="674">So on the Vitals package website,</a> <a data-t="676">as well as the Inspect AI website,</a> <a data-t="678">there's all sorts of guidance</a> <a data-t="680">about how to put together these systems thoughtfully.</a> <a data-t="683">But for now, we'll just take it for granted</a> <a data-t="685">that, yes, this is the way that we're going to do it.</a> <a data-t="688">If you're like, absolutely not,</a> <a data-t="690">this is not the way that this is going to go,</a> <a data-t="692">there are certain solvers and LLM systems</a> <a data-t="694">that you don't need this LLM,</a> <a data-t="696">as a judge is what they call it,</a> <a data-t="698">where you use an LLM itself to score the response.</a> <a data-t="701">And so if you're like, no way, that's not a deal-breaker.</a> <a data-t="704">There's a way to make that happen.</a></p>

<h3>Running the eval and the log viewer</h3>

<p><a data-t="706">But now we have a data set, a solver, and a score.</a> <a data-t="711">The container for these three objects</a> <a data-t="713">in Vitals and Inspect is called a task.</a> <a data-t="716">And so we can use Google Sheets 4</a> <a data-t="719">to read in that spreadsheet that we put on online,</a> <a data-t="722">and then we situate the data set, the solver,</a> <a data-t="725">and the score inside of this task.</a> <a data-t="728">This task has a bunch of methods.</a> <a data-t="730">One of them is called eval,</a> <a data-t="732">which first runs all the solver inputs in parallel</a> <a data-t="736">and then scores all of them in parallel.</a> <a data-t="739">And so for these eight inputs,</a> <a data-t="741">this took something like 11 seconds, 12 seconds.</a></p>

<p><a data-t="747">In addition to a modified version of the task itself,</a> <a data-t="751">something that both Vitals and Inspect give back to you</a> <a data-t="755">is this thing called the log viewer.</a> <a data-t="757">And the log viewer is sort of a foundational tool</a> <a data-t="759">for both evaluating quality and debugging issues.</a> <a data-t="763">So at the highest level, you get these scores,</a> <a data-t="765">like the accuracy.</a> <a data-t="767">So in this case, in six of eight responses,</a> <a data-t="770">the solver from this conf chatbot app</a> <a data-t="772">was rated as being correct.</a> <a data-t="775">And we can scroll down through this app</a> <a data-t="777">until we see where some sort of issue came up.</a> <a data-t="780">So this score here was marked as incorrect</a> <a data-t="783">with a red circle.</a> <a data-t="785">And I can click into this and drill down into the details</a> <a data-t="788">to figure out what went wrong.</a></p>
