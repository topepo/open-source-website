<h3>Lesson 1: Do no harm</h3>

<p><a data-t="329">Now, Kiegan and I are going to share with you three lessons we've learned on how to be a pollinator.</a> <a data-t="333">And the first lesson is do no harm.</a> <a data-t="337">As R users and developers, it's really important for us to be really careful and intentional when we interact with others in our ecosystem and when we deal with foreign data formats.</a></p>

<p><a data-t="350">So here is one example of reading data from Stata into R. And they say that the lessons that stick with you most are the ones you learn from your mistakes.</a> <a data-t="360">And so this is one that's very dear to my heart.</a> <a data-t="363">Anyways, this example comes from the General Social Survey, or the GSS.</a> <a data-t="368">And the GSS is one of the most widely cited studies of American society.</a> <a data-t="373">It's a study that's been run by NOC for the past 50 years.</a></p>

<p><a data-t="378">And here we have an entry from the GSS codebook.</a> <a data-t="382">This variable describes the number of educations received by the respondent's mother.</a> <a data-t="387">And the codebook tells us what the unique values of this variable is and number of cases with each value.</a> <a data-t="395">And then at the bottom, we have this section called reserved codes.</a> <a data-t="399">And these are missing values.</a> <a data-t="401">So there are three different types of missing values.</a> <a data-t="404">And we have 316 cases that are missing value because they said they don't know.</a> <a data-t="410">And then we have 100 cases that are missing values because they said that they did not grow up with a female caregiver growing up.</a> <a data-t="417">So they were not asked this question.</a></p>

<p><a data-t="420">Now when we read this Stata data file into R, we see that the three different missing value types have been collapsed into a single NA.</a> <a data-t="431">And this looks bad, right?</a> <a data-t="432">Because it looks like we've just lost information.</a> <a data-t="435">So why does this happen?</a></p>

<p><a data-t="439">If folks are familiar with SAS or Stata, it's that whereas these two software have extended missing types so they can support multiple missing value types, R only has a single system missing.</a> <a data-t="452">So what do we do about this?</a> <a data-t="454">Well, it turns out that the R data object that we have, whether we've read in the data using Haven or we used the foreign package, the R data object still contains these missing value types.</a> <a data-t="467">And it takes a little bit of effort and a few more lines of code to recover the different types.</a> <a data-t="474">Folks who are interested can see Haven's convergent semantics for more information.</a></p>

<p><a data-t="479">But our point here is that when you're working with a new or foreign data format, it really is important to take a moment to learn about the idiosyncrasies of that specific format.</a> <a data-t="492">And it's part of doing no harm.</a></p>

<p><a data-t="496">Here is another example.</a> <a data-t="498">Here we have a screenshot of an Excel spreadsheet.</a> <a data-t="500">The contents don't really matter, but you can see that a few rows have been highlighted.</a> <a data-t="506">So there is data in the formatting.</a> <a data-t="510">And these could be cases that have been flagged by a team for review.</a> <a data-t="515">So some people have feelings about formatting as data.</a> <a data-t="522">But our view is that this data can be important, and it's important that we preserve this data.</a> <a data-t="530">And the great thing about R is that there are these packages, openxlsx.xl, that really helps us to retain this information in a very easy way.</a> <a data-t="541">And we should take the effort, a few lines of code to preserve this information.</a></p>

<p><a data-t="547">So the lesson here is it's going to take a bit of effort and a bit of empathy, but it's really important as pollinators that we take care with other people's data and we do no harm.</a></p>

<h3>Lesson 2: Make things that just work</h3>

<p><a data-t="561">So the second principle of how to be a pollinator is to make things that just work.</a> <a data-t="569">So what we really mean by this is we want to remove barriers for either people interacting with R or people who are using things that we're creating in R to make it so that things just work for them.</a></p>

<p><a data-t="588">So a good example of making things that just work is consider how your data are being outputted.</a> <a data-t="593">So you might take the usual approach of I'm going to share this data that I've cleaned and wrangled in R. <p><a data-t="640">So this is a really great way to make sure that the next person has an easier time.</a> <a data-t="646">They go ahead and run this script, and the data just works.</a> <a data-t="649">They don't have to go digging for the formatting and the labels and reencode everything in a way that makes sense to SAS.</a> <a data-t="656">It just works.</a> <a data-t="657">It's already there.</a></p>

<p><a data-t="660">Another example is something you've maybe run into, a message like this.</a> <a data-t="666">Looks like there's a hiccup.</a> <a data-t="667">The server-hosted dashboard is not filtering based on the uploaded file, but it's working on my local computer.</a> <a data-t="672">So maybe you've gotten this message, this panicked email.</a> <a data-t="676">Maybe you've sent this panicked email at some point.</a> <a data-t="679">It's working on one computer, but it's not working on another.</a> <a data-t="682">It's working on the server, but it's not working locally.</a> <a data-t="684">So for somebody who's not super confident in figuring out when errors are going wrong, this can be really scary.</a></p>

<p><a data-t="701">So we can remove this possible barrier by, from the start, using something like </a><a href="/software/renv/">RENV</a><a data-t="701">.</a> <a data-t="707">And there are a lot of different options, but essentially, if you don't know what RENV is, go look it up.</a> <a data-t="718">But it allows us to keep track of the package versions we're using when we're writing and implementing code and share those with any other people or environments that we're going to use this code on.</a> <a data-t="731">So the next time we go to run this code, it works on the server and it works locally.</a> <a data-t="735">It works on your machine and it works on my machine.</a> <a data-t="737">And when we come back to this code in six months, it runs exactly like it's supposed to.</a> <a data-t="742">It just works.</a></p>

<h3>Lesson 3: Promote growth</h3>

<p><a data-t="767">So the third principle is to promote growth.</a> <a data-t="770">And we really want to bring as many people into the R fold and into the R community at our organization as possible.</a> <a data-t="777">And so we do this in a couple of ways.</a> <a data-t="780">The first that we do pretty often is have newer R users interact with existing scripts and processes.</a> <a data-t="786">So maybe a more advanced R user is going to write a script or some sort of process to complete a task.</a> <a data-t="794">Maybe it's a data analysis, maybe it creates a table, maybe it creates a chart.</a> <a data-t="800">And we can have newer users take that already written code and run it with maybe changes to certain parameters or flags within that code.</a></p>

<p><a data-t="810">And this may lead to the following scenario.</a> <a data-t="814">You're trying to figure out what exactly happened to the script that you wrote.</a> <a data-t="818">So you ran lines 1 through 57, then you changed overwrite to true, then 61 through 66, then back to 37.</a> <a data-t="825">And this can be a really kind of frustrating conversation to have for both people if you're trying to figure out exactly what happened and exactly what went wrong.</a> <a data-t="835">And now the person interacting with this code who's maybe a newer R user is going to feel frustrated and a little scared.</a></p>

<p><a data-t="842">So there's a couple of ways that you can just make this a little bit easier and provide a more low-risk way for them to interact with this code.</a> <a data-t="853">So one is you can use a dialog box.</a> <a data-t="855">So something like the SVDialogs package.</a> <a data-t="859">You can write, again, one line of code, and you can have a user go run an R script, and when it gets to a certain point where you've inserted this dialog box code, a dialog box will pop up and give them the option to select yes or no.</a> <a data-t="873">So maybe it's whether you're going to include a certain set of NAs or not.</a> <a data-t="877">Maybe it's whether you're going to run this for males or females.</a> <a data-t="880">Whatever it is, it's a much easier way for them to interact with the code, and it's much less likely that you're going to run into this tracking down what lines of code got run at what time and what order.</a></p>

<p><a data-t="952">So the users are interacting with buttons in this without actually having to run the code themselves, and they're able to see exactly the products of this, these beautiful created R Markdown documents, without having to run the code themselves or dig into it.</a></p>
