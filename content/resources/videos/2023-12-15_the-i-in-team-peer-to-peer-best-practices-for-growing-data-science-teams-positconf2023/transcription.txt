<h3>The incident</h3>

<p><a data-t="92">So how bad was this incident?</a> <a data-t="101">Back in the beginning, I wrote an R package for the council, but actually it wasn't really</a> <a data-t="107">for the council.</a> <a data-t="109">My manager was the only one using it, so it was just for me.</a> <a data-t="112">And I got to do whatever I wanted with it.</a> <a data-t="114">It was great.</a> <a data-t="115">It's called Counselor, made a logo.</a></p>

<p><a data-t="120">It has things like corporate colors, database connections, templates, classic internal R</a> <a data-t="127">package.</a> <a data-t="129">And as we got more people on board using R, using GitHub, I got more people to install</a> <a data-t="135">the package and start actually using it for themselves, which was a really amazing feeling.</a></p>

<p><a data-t="142">At some point I decided that we should just completely rehaul the way that fonts were</a> <a data-t="148">sized in our </a><a href="/software/ggplot2/">ggplot2</a><a data-t="148"> theme, which is fine.</a> <a data-t="154">But it had some unintended side effects.</a> <a data-t="157">So I wrote this update, pushed it out, had all my colleagues install it.</a> <a data-t="162">And it installed, which I considered a win.</a></p>

<p><a data-t="167">However, my colleague installed the update while they were working on a very time-sensitive</a> <a data-t="173">report.</a> <a data-t="174">And for any of you that have worked with going from R Markdown to Microsoft Word, there is</a> <a data-t="183">a lot of potential frustration there with plots.</a></p>

<p><a data-t="188">So the thing is that what I did didn't break their code.</a> <a data-t="193">It just changed things.</a> <a data-t="196">So if we see this plot, where we have my plot plus theme counsel, where the font sizes are</a> <a data-t="206">correct, the title is one of the largest fonts that you see, you can read the X and Y axis</a> <a data-t="215">as well as the legend title, the axis titles, all of it makes sense.</a> <a data-t="221">And the caption is tiny, like it should be.</a></p>

<p><a data-t="225">But then without breaking the code, so there was no, like, warning or, like, clear this</a> <a data-t="231">didn't work, the plot did this, where the caption is massive, the text on the X and</a> <a data-t="240">Y axis is practically nonexistent.</a> <a data-t="244">The legend title is tiny.</a> <a data-t="245">The title title font is different.</a> <a data-t="248">And the Y axis title is rotated.</a> <a data-t="252">And this isn't none of these are bad.</a> <a data-t="256">But when you're trying to get something going, it's not fun when something changes right</a> <a data-t="263">underneath you.</a></p>

<p><a data-t="266">So what went wrong?</a> <a data-t="269">There were several things.</a> <a data-t="271">First, there was inaccurate and inadequate documentation on my part.</a> <a data-t="278">There was also a lack of communication.</a> <a data-t="280">So I didn't adequately convey the changes I was making to my colleague, and so they</a> <a data-t="285">didn't know what to expect.</a> <a data-t="289">And we also had mismatched priorities.</a> <a data-t="291">I was having fun tinkering with my package, and my colleague actually had something to</a> <a data-t="296">get out into the world.</a></p>

<p><a data-t="303">And the next question is could my manager have prevented this?</a> <a data-t="310">And the answer is no.</a> <a data-t="311">My manager could not have prevented this.</a> <a data-t="315">This is an interpersonal team thing.</a> <a data-t="319">And that's where my argument comes in.</a></p>

<h3>There is an I in team</h3>

<p><a data-t="323">There is an I in team.</a> <a data-t="326">Because teams are groups of individuals working together.</a> <a data-t="330">Note I in individuals.</a> <a data-t="334">And you, as an individual team member, can shape your team by caring for your new team</a> <a data-t="341">members and your current team members, seeing how you fit into your team—</a> <a data-t="349">sorry.</a></p>

<p><a data-t="368">Seeing how you fit into your team and changing how you work.</a> <a data-t="374">And as the turnover has happened at my job and we brought more people on board, there</a> <a data-t="378">are a few things I've learned that I'd like to share with you today.</a> <a data-t="382">And one is that when you have a new colleague coming on board, it's your responsibility</a> <a data-t="387">to bring them on board, too.</a> <a data-t="390">It's not just HR.</a> <a data-t="391">It's not just your manager.</a> <a data-t="393">It's you as well.</a></p>

<blockquote><a data-t="382">And one is that when you have a new colleague coming on board, it's your responsibility</a> <a data-t="387">to bring them on board, too.</a> <a data-t="390">It's not just HR.</a> <a data-t="391">It's not just your manager.</a> <a data-t="393">It's you as well.</a></blockquote>

<p><a data-t="395">So when you're bringing on a new teammate, three things.</a> <a data-t="400">Make the implicit explicit.</a> <a data-t="403">All knowledge is valid and valuable.</a> <a data-t="406">And document.</a> <a data-t="408">Also known as make life easier for your friends.</a></p>

<h3>Make the implicit explicit</h3>

<p><a data-t="413">So first, make the implicit explicit.</a> <a data-t="417">The story here is from a commencement speech by a fairly famous American writer in which</a> <a data-t="425">there are a couple fish swimming around.</a> <a data-t="429">In this instance, we'll say that these are you and your coworker having a great time</a> <a data-t="435">being fish.</a> <a data-t="436">A new fish comes along.</a> <a data-t="438">And then they ask a fairly innocuous question.</a> <a data-t="441">How's the water?</a> <a data-t="444">To which there's silence.</a> <a data-t="447">And then what the hell is water?</a></p>

<p><a data-t="450">And now you need to understand that this fish is having a crisis.</a> <a data-t="454">They don't know what they've been living in their entire life.</a> <a data-t="460">What does it mean to be surrounded by this thing constantly all the time?</a> <a data-t="465">And why am I just now being told that there's water?</a> <a data-t="470">So this fish is having a struggle.</a> <a data-t="474">And you don't need to be an anxious fish.</a> <a data-t="477">Don't deny that water exists.</a> <a data-t="480">And by water, I mean your workplace culture.</a></p>

<p><a data-t="485">Everyone has pet peeves and preferences.</a> <a data-t="489">Every workplace and every team within a workplace has a culture.</a> <a data-t="494">And that's not inherently bad.</a> <a data-t="496">But you need to recognize it for what it is.</a> <a data-t="500">And because otherwise, when a new person comes in, you're giving them the task of proving</a> <a data-t="507">that like, oh, no, there is a culture here, in addition to figuring out if they want to</a> <a data-t="512">stay on the team.</a> <a data-t="515">Don't give your new coworker the task of defining what water is in addition to evaluating if</a> <a data-t="521">the water is right for them.</a></p>

<p><a data-t="523">There are a few ways you can define your water, and these are just a couple starting questions.</a> <a data-t="531">What does your weekly schedule look like?</a> <a data-t="533">How much time do you spend in big meetings, one-on-one meetings, really deep focused time?</a> <a data-t="538">What does it look like for you?</a> <a data-t="540">What tools do you use every day?</a> <a data-t="543">Why?</a> <a data-t="544">And I mean every single piece of anything that you touch throughout the entire day.</a> <a data-t="549">Make a list.</a> <a data-t="551">And how do you use it?</a></p>

<p><a data-t="555">And what expectations does your workgroup have around email or message response times?</a> <a data-t="560">Same thing goes with having a status indicator in your Microsoft Teams or Slack, as well</a> <a data-t="567">as any after-hours communication expectations.</a></p>

<h3>All knowledge is valid and valuable</h3>

<p><a data-t="572">The second, all knowledge is valid and valuable.</a> <a data-t="579">No one is an empty vessel.</a> <a data-t="582">And so all of the knowledge that this new person is bringing to your team is valuable.</a> <a data-t="589">No matter where they're coming from, if they're coming from a different industry, coming straight</a> <a data-t="593">from college, coming straight from an online course, wherever they're coming from, the</a> <a data-t="598">knowledge that they have is useful.</a> <a data-t="603">And even if it seems very unrelated.</a> <a data-t="605">For whatever reason in my workplace, we've attracted a lot of PhD ecologists, which is</a> <a data-t="611">not what we do.</a> <a data-t="615">And I've learned so much from them.</a></p>

<p><a data-t="618">You also have to remember that your knowledge is not inherently superior just because you've</a> <a data-t="624">been at this workplace longer.</a> <a data-t="629">So when you're bringing this new colleague in, it's important to give them some grace.</a> <a data-t="635">Patients will prove best.</a> <a data-t="637">Whether you're introducing them to a new tool, integrating them into the broader organization</a> <a data-t="641">and getting all those social relationships sorted out.</a> <a data-t="646">Whatever it is, have patience, and it will pay off.</a></p>

<p><a data-t="650">And remember that you're also on probation.</a> <a data-t="653">In my workplace, you are hired, you have a six-month period, and at the end of that,</a> <a data-t="659">the employer can decide whether to keep you.</a> <a data-t="661">And they're on probation, but you as a team member are also on probation.</a> <a data-t="667">They're figuring out if they want to work with you.</a> <a data-t="670">And that's important, too.</a></p>

<p><a data-t="673">Also don't rely on catch-all assumptions.</a> <a data-t="676">Ask for specifics.</a> <a data-t="679">For example, two people who have Git and GitHub on their resume but have very different ways</a> <a data-t="686">of working with Git will have different experiences.</a> <a data-t="690">Someone who's on the command line is going to have a very different experience than someone</a> <a data-t="694">like me who relies on a really nice user interface to visualize branches.</a> <a data-t="701">Neither of these are necessarily better than the other one, but you need to ask for those</a> <a data-t="708">specifics rather than assuming.</a></p>

<h3>Document — make life easier for your friends</h3>

<p><a data-t="713">And finally, document, aka make life easier for your friends.</a> <a data-t="720">So an experience you may have had at breakfast this morning, not in this hotel, but whenever</a> <a data-t="726">you had breakfast, is if you were to have cereal, which comes first?</a> <a data-t="732">The cereal or the milk?</a> <a data-t="735">And there are all sorts of answers.</a> <a data-t="737">There's also questions like why cereal?</a> <a data-t="741">What kind of cereal?</a> <a data-t="742">Should it be sugary, organic, whole grain?</a> <a data-t="745">What kind of cereal?</a> <a data-t="746">And then what kind of milk?</a> <a data-t="747">Should it be milk, cow milk?</a> <a data-t="749">What kind of cow milk?</a> <a data-t="751">What kind of not cow milk?</a> <a data-t="752">There are all sorts of ways.</a></p>

<p><a data-t="756">And I'm going to be truthfully brutal with you guys.</a> <a data-t="761">It doesn't matter.</a> <a data-t="764">What does matter is that you remember the bowl in which to put your cereal and milk.</a> <a data-t="772">Documentation is the bowl.</a></p>

<blockquote><a data-t="772">Documentation is the bowl.</a></blockquote>

<p><a data-t="777">Data science is an incredibly creative field, which is one reason why I enjoy working in</a> <a data-t="782">data science.</a> <a data-t="783">There are many ways to solve a problem.</a> <a data-t="786">And by trying to force everyone into one single way, you're not taking advantage of all of</a> <a data-t="792">the gifts and knowledge that they have to bring.</a> <a data-t="797">Embrace the diversity of opinion and experience, even if it's hard for you as the existing</a> <a data-t="803">person, and it will be hard for the new person coming in.</a> <a data-t="808">Just remember whenever you're going through something to leave a trail for others to follow.</a> <a data-t="813">Add in comments in your code, every few lines.</a> <a data-t="818">Leave a trail of breadcrumbs towards the final outcome or whatever process.</a></p>

<p><a data-t="828">Documentation is also a matter of investment.</a> <a data-t="831">You're investing in tools that encourage documentation and lower the cognitive load.</a> <a data-t="838">You're also investing in the time that it takes to document, clean up, organize, etc.</a> <a data-t="844">Your work.</a> <a data-t="846">And you're investing in strong social relationships.</a> <a data-t="848">This might not seem very intuitive, but follow me.</a> <a data-t="854">Documentation is communication in written form.</a> <a data-t="858">And for good communication, you need to know your audience.</a> <a data-t="862">Kind of like if you're giving a talk at a conference.</a></p>

<p><a data-t="868">So this means that work doesn't always feel like work.</a> <a data-t="873">Schedule time to build connections and trust.</a> <a data-t="877">In person, virtual, hybrid, all of the above.</a> <a data-t="883">Social gatherings, virtual or in person, are just as valuable as that coveted, like, afternoon</a> <a data-t="891">hours of focus.</a> <a data-t="894">Really.</a> <a data-t="895">And it took me a while to kind of accept that.</a></p>

<p><a data-t="900">And try making every other team meeting a casual hangout.</a> <a data-t="905">That's something that we've done in my work group.</a> <a data-t="907">We have a weekly team meeting, and one week we talk about work.</a> <a data-t="912">The next week we talk about not work.</a> <a data-t="914">And I've gotten to know my colleagues so much more and have gained a better perspective</a> <a data-t="919">of who they are as people and the backgrounds that they have.</a> <a data-t="924">My colleagues are really great.</a> <a data-t="925">They're really interesting people.</a> <a data-t="927">And I enjoy working with them.</a></p>

<p><a data-t="930">Ultimately, at the end of the day, there is an I in team.</a> <a data-t="939">Several I's.</a> <a data-t="941">Teams are made up of individuals.</a> <a data-t="942">And it's not only the manager's job to make the team work.</a> <a data-t="947">It's your responsibility as a team member as well.</a> <a data-t="952">Onboarding is a unique and critical time when someone is coming in.</a> <a data-t="955">And you can take advantage of that time to gain a really great relationship with them</a> <a data-t="960">and set them up for success in your workplace.</a></p>
