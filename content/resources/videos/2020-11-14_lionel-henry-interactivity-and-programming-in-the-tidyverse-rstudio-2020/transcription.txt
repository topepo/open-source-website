<h3>Solving unexpected masking</h3>

<p><a data-t="375">So first about unexpected masking.</a> <a data-t="378">That's about the problem.</a> <a data-t="381">So let's say that you have created an object N in your workspace.</a> <a data-t="385">It contains a number.</a> <a data-t="386">And you have a data frame with an X column.</a> <a data-t="389">And you want to modify it, create a new column that divides the column X by the number inside</a> <a data-t="394">of N. <p><a data-t="401">And if you are using the same code with different data frames, maybe they don't contain the</a> <a data-t="406">same columns.</a> <a data-t="407">And if you get a data frame that has a column N, then the column has precedence over the</a> <a data-t="416">object in the workspace.</a> <a data-t="417">The column masks the object in the workspace, and you get the wrong result.</a></p>

<p><a data-t="424">So that's not a big problem if you are working interactively through your script doing an</a> <a data-t="430">analysis because you know what kind of objects you have in the workspace.</a> <a data-t="435">You know the columns that are in your data frames.</a> <a data-t="439">But if you are writing production code, that can be an issue because your same code will</a> <a data-t="445">be used across many different data frames from different users.</a></p>

<p><a data-t="451">So the solution here is to be explicit when you are writing production code.</a> <a data-t="455">And to do that, you use the .data pronoun or the .inf pronoun.</a> <a data-t="460">And these pronouns are available in all of the data masking functions in the tidyverse.</a> <a data-t="466">And so here we are, again, modifying the data frame, but this time we are dividing .data$X</a> <a data-t="473">by .inf$N.
