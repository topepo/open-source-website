<h3>Why use a FastAPI as a data gateway</h3>

<p><a data-t="164">For today's workflow, we will use VS Code from within Posit Workbench to develop a FastAPI in Python.</a> <a data-t="173">And to keep things nice and simple, we're going to design this API to do only one thing, and that's to serve data.</a> <a data-t="182">We'll then publish this FastAPI to Posit Connect and show how it can serve as a gateway to your data,</a> <a data-t="188">which can be accessed by multiple pieces of content using Python, R, or any other language or system.</a></p>

<p><a data-t="197">Now, many of you may be like me, and when you started your data science journey,</a> <a data-t="202">the idea of working with APIs, let alone creating your own API,</a> <a data-t="207">was so daunting that I honestly tried to avoid working with APIs at all costs.</a> <a data-t="212">Well, I'm here to tell you that they're not that scary and can actually be used to vastly improve your data science workflows.</a></p>

<blockquote><a data-t="212">Well, I'm here to tell you that they're not that scary and can actually be used to vastly improve your data science workflows.</a></blockquote>

<p><a data-t="219">Now, the acronym API stands for Application Programming Interface,</a> <a data-t="224">and they essentially allow communication between applications.</a> <a data-t="228">To help explain what APIs are, I like to think of the commonly used restaurant analogy you see on your screen here.</a> <a data-t="236">Here we have three customers that have entered a restaurant</a> <a data-t="240">and would like to order some food that the chef in the kitchen will prepare.</a> <a data-t="243">To request food, the customers need some way to communicate their order to the kitchen,</a> <a data-t="248">and rarely, if ever, the customer will interact directly with the chef.</a> <a data-t="254">So that's where the waiter or waitress comes into play.</a> <a data-t="258">They take the order from the customer and can communicate that order to the chef.</a> <a data-t="263">The chef prepares the food, which is then delivered back to the customer.</a></p>

<p><a data-t="268">So in this analogy, the waiter or waitress allows for communication between the customers and the chef.</a> <a data-t="274">Essentially, they are the API.</a></p>

<p><a data-t="278">For data scientists, APIs can be an essential workflow building block</a> <a data-t="283">that gives other users or applications or systems the ability to interact with the functions you've designed,</a> <a data-t="290">the models you've trained, or the data you've curated.</a> <a data-t="294">You also don't need to be a software engineer or computer scientist to create an API,</a> <a data-t="299">and there are numerous frameworks in both R and Python for creating APIs, including Plumber and Flask.</a> <a data-t="306">For today's workflow, we're going to focus on just FastAPI,</a> <a data-t="310">which is one of the most popular frameworks for creating APIs using Python,</a> <a data-t="315">given its simplicity and performance.</a></p>

<p><a data-t="318">Now, as I previously mentioned, the FastAPI we are going to build today and publish to Posit Connect</a> <a data-t="324">will essentially serve as a gateway to our data, which can be accessed by a variety of content.</a> <a data-t="331">But why would you want to do this?</a> <a data-t="333">Well, there are a few reasons.</a> <a data-t="335">The first reason is that it simplifies your code.</a> <a data-t="338">You no longer need to attach and read in and transform the data into each individual piece of content.</a> <a data-t="345">So this helps also prevent data duplication and also standardize data access across content.</a></p>

<p><a data-t="352">Secondly, by putting data behind an API, it makes it easily accessible to different content types.</a> <a data-t="360">For example, you can have a Jupyter Notebook built using Python</a> <a data-t="364">and a Shiny application built using R access the same data via the same API.</a> <a data-t="370">And later on in this demo, we'll demonstrate this capability.</a> <a data-t="374">Lastly, by publishing the FastAPI to Posit Connect,</a> <a data-t="378">you can control access to your data by adding an authentication layer to your APIs.</a> <a data-t="383">And this is a great option if your team is working with sensitive data.</a></p>
