<h3>Data analysis workflows</h3>

<p><a data-t="85">So let's talk a little bit about data analysis workflows.</a> <a data-t="89">And by workflows, I mean that the way that you typically engage and interact with R as you're doing your analysis process.</a> <a data-t="95">If you are a user who is new to R or if you've ever worked with somebody who is just getting started</a> <a data-t="100">with R, you typically see them using this sort of a workflow.</a> <a data-t="105">They'll be typing R commands in the console and then looking at the result and the output of the commands and then issuing</a> <a data-t="110">another command.</a> <a data-t="116">So they sort of have this ad hoc conversation with R. <p><a data-t="129">And there's actually nothing wrong with doing R this way. <p><a data-t="155">And so most people who have used R for some time have learned that instead of just entering stuff</a> <a data-t="162">ad hoc at the console most of the time, what you want to use is an R script.</a> <a data-t="167">This is sort of another level above a regular like R console thing.</a> <a data-t="173">It's instead of just entering stuff at the console, you're actually forming a script that leaves a record of what you have done.</a> <a data-t="178">And this has a huge amount of advantages over just entering code ad hoc at the console.</a></p>

<p><a data-t="187">Your analysis is now documented, like you have a record of what you did.</a> <a data-t="191">It is also now repeatable, which is to say that you can now run the same analysis over and over as you need to.</a> <a data-t="201">And it's even shareable. You can take this .R file and send it to anybody</a> <a data-t="205">and say, hey, do you want to do the analysis that I just did?</a> <a data-t="209">You know, there's no smoke and mirrors. <p><a data-t="214">So this is actually where many R users stop their exploration.</a> <a data-t="221">So R scripts, output, done.</a> <a data-t="221">But if you think about this workflow, it still actually has a lot of room for improvement.</a> <a data-t="226">For one thing, the R output is disconnected from the input.</a> <a data-t="233">Your R code, this script produces some output and it's really not connected</a> <a data-t="240">in any way to the script itself. <p><a data-t="246">Typically, this is not actually the end of the line. <h3>Introducing R Markdown</h3>

<p><a data-t="290">So to solve a lot of these problems, we introduced something a few years back called R Markdown.</a> <a data-t="294">It borrows heavily from some ideas in literate programming and it's designed to address some of the issues that I just talked about.</a> <a data-t="300">R Markdown lets you do the same kind of thing you do in an R script,</a> <a data-t="307">which is that you can write code and have a record of it, but it also allows you to</a> <a data-t="313">bring together your output and your code in a rendered document so that your document</a> <a data-t="320">itself and the code are combined.</a></p>

<p><a data-t="326">This solves both of the problems I was talking about earlier.</a> <a data-t="331">Now you can see your code and your output together. You can actually make your code</a> <a data-t="336">your final document. They're one in the same artifact.</a> <a data-t="336">And it is possible to create all kinds of very high quality professional documents from this.</a> <a data-t="340">And you typically don't need to have another tool that you export this to.</a> <a data-t="345">This can actually produce your final HTML file or PDF document or Word document or whatever that you need to email out.</a></p>

<p><a data-t="354">Even though this is a very powerful tool and it has received a lot of attention and a lot of accolades,</a> <a data-t="364">it still has some problems when it comes to interaction.</a> <a data-t="371">If you look at where we've come from, from all the way from a typical new R analysts, this is appealing because</a> <a data-t="377">it's simple.</a> <a data-t="377">By the time we get over to here, yes, it is literate. Yes, it is wonderful</a> <a data-t="383">to mix your context and your narrative and your code and produce all these wonderful</a> <a data-t="389">output formats, but something has been lost.</a> <a data-t="394">This feels easy and fun and this is beginning to get quite complicated.</a></p>

<h3>The R Notebooks philosophy</h3>

<p><a data-t="401">So, we set out to create a way of interacting with R that gave you a lot of the benefits of the literate programming model and without a lot of the complication that we have in this.</a> <a data-t="416">So, this is the interaction model for R notebooks.</a> <a data-t="416">You'll notice this is a very simple way to interact with R. You run some code, you see the output right beneath the code.</a></p>

<p><a data-t="429">Let me talk very generically about notebooks before I spend a bunch of time showing you how they work, just so you can get an idea of the philosophy.</a> <a data-t="437">We want you to be able to interact with R in kind of a single, seamless stream.</a> <a data-t="443">What we mean by that is that you don't need to have four or five windows or panes open.</a> <a data-t="448">You should be able to have a single window open in which you can cohesively interact with R.</a></p>

<p><a data-t="454">We wanted to make it possible for you to iterate quickly in your code and output, which is to say that you should be able to</a> <a data-t="458">change your code and see the change in output together in one place.</a> <a data-t="466">Again, not spread out, not in a big batch, but one line of code at a time.</a> <a data-t="471">You should be able to leave a clean and reproducible record of your analysis in really a simple text file.</a> <a data-t="477">And by this, again, we mean that as you were doing your data analysis, you're actually creating the final</a> <a data-t="483">document that you'll be using to communicate and publish your analysis.</a></p>

<p><a data-t="487">And you are also documenting exactly what you did so it can be repeated.</a> <a data-t="493">You'll be able to document your analysis with rich, literate prose.</a> <a data-t="498">By this, we mean that you can use all the power of R Markdown, which includes many very high-quality professional layouts and tools for formatting your text.</a> <a data-t="511">These things can be shared and published easily, and because it's just R Markdown, you can export it to any format that you like very quickly and easily.</a></p>

<p><a data-t="2129">So, here is... All right. So, I'm going to show you how to do this.
