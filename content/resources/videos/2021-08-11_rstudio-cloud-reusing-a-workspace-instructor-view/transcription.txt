<p><a data-t="0">It's not uncommon for us to teach the same course, but maybe with a few slight modifications.</a> <a data-t="8">Rather than creating the same course multiple times, we can actually reuse our </a><a href="/software/rstudio/">RStudio</a><a data-t="8"> Cloud</a> <a data-t="13">workspace using one of two methods.</a> <a data-t="16">We can remove all of our learners once the course is completed and reuse the workspace,</a> <a data-t="21">or we can copy the workspace for a new cohort of learners.</a></p>

<h3>Removing learners from a workspace</h3>

<p><a data-t="25">To reuse our existing workspace by removing learners, we can go to the Members tab, located</a> <a data-t="30">at the top of our workspace.</a> <a data-t="33">From here, we can navigate to a specific member and click the Delete icon here.</a> <a data-t="38">We can repeat this for any and all members in our workspace.</a> <a data-t="42">However, if we'd like to remove large numbers of learners, we'll need to use the RSCloud</a> <a data-t="47">package.</a></p>

<p><a data-t="48">It's also important to note that only admins can remove learners.</a> <a data-t="53">This means that if you'd like any of your co-instructors or teaching assistants to help</a> <a data-t="57">remove members from the shared workspace, they'll need to have their membership role</a> <a data-t="61">switched to Administrator.</a></p>

<h3>Copying a workspace</h3>

<p><a data-t="63">Rather than remove learners, we can copy our workspace.</a> <a data-t="67">Copying our workspace allows us to take all of the existing shared files to a new workspace</a> <a data-t="73">where we will be the only member.</a> <a data-t="75">Just note that any private files in our workspace will not be copied over.</a></p>
