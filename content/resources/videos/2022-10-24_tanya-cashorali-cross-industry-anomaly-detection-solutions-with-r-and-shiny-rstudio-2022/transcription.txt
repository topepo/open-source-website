<h3>What is anomaly detection?</h3>

<p><a data-t="61">So what is anomaly detection?</a> <a data-t="65">This is just the Wikipedia definition.</a> <a data-t="65">It's sometimes referred to as outlier detection, but I like to say it gets a little fancier</a> <a data-t="69">than just detecting standard deviations above or below a mean.</a> <a data-t="74">It's a way to identify rare items or events.</a> <a data-t="77">I typically use it for time series data.</a> <a data-t="81">Twitter actually developed a package, which I'll get into, and they use it to look at</a> <a data-t="85">tweets per second.</a> <a data-t="87">You can use it for CPU metrics and utilization, but I'll talk about two different industry</a> <a data-t="92">cases in which we were able to use it and had a lot of success doing so.</a></p>

<p><a data-t="100">So the R package, like I said, was developed first at Twitter, and there's some really</a> <a data-t="104">good papers that I link at the end of this.</a> <a data-t="106">The slides are available.</a> <a data-t="107">I definitely recommend it.</a> <a data-t="109">There's some good reading in there.</a> <a data-t="110">I won't have time to get into all the technical details, but I actually used a fork from HarborMaster,</a> <a data-t="117">and he made a few modifications to the package, one of which is it returns tidy data frames,</a> <a data-t="124">tibbles, and he also made the function names a little easier, snake case aliases, so you</a> <a data-t="129">can see them, ADTS for time series and ADVEC for vector.</a> <a data-t="135">So this is the package.</a> <a data-t="137">It's been phenomenal for us so far.</a></p>

<p><a data-t="141">Now just to talk about the implementation or the underlying algorithm, very briefly,</a> <a data-t="147">it builds upon an extreme studentized deviant test, just a big word for saying it does some</a> <a data-t="153">time series decomposition and robust statistics, and the paper I mentioned actually benchmarks</a> <a data-t="159">these four approaches, quant linear, a seasonal decomposition with loss, and a quant spline.</a> <a data-t="166">The one they said actually performed the best was piecewise median, so it looked like</a> <a data-t="172">stable metrics typically exhibited very little change in the median over two-week windows,</a> <a data-t="177">so it defaults to breaking your data into two-week windows and calculating a moving</a> <a data-t="181">median.</a> <a data-t="182">It also accounts for intraday and weekly seasonalities.</a> <a data-t="187">It was pretty much performing on par with these more expensive calculations.</a> <a data-t="192">Four times faster than the other ones, so that's the red line you're seeing here.</a> <a data-t="197">We've had no performance issues and have given it quite a bit of data, and we run everything</a> <a data-t="202">in real time.</a></p>

<h3>Use case 1: life sciences hardware manufacturer</h3>

<p><a data-t="360">So to get into the examples, this first one I think is pretty cool.</a> <a data-t="364">It's a hardware manufacturer in life sciences.</a> <a data-t="368">They essentially build the rapid blood testing machines in hospitals.</a> <a data-t="373">So if you go to the hospital, hopefully not, when you're there and you get your blood taken,</a> <a data-t="378">this is the company creating the devices that give you those rapid results.</a> <a data-t="382">They have sensors in them that relay quantitative metrics and help diagnose instrument failures,</a> <a data-t="387">but what this company was doing was deploying the hardware in the field and finding out</a> <a data-t="393">retroactively that there was a problem, so they would take the machine back and analyze</a> <a data-t="398">all of the data coming off the machines.</a> <a data-t="401">This seemed like a perfect opportunity to use anomaly detection.</a> <a data-t="404">We wanted to identify those problems before they went out into the field.</a></p>

<p><a data-t="409">So here's what the data and experimentation looks like.</a> <a data-t="412">We have various metrics coming off these machines.</a> <a data-t="414">There are things like protein levels and analytes, and they come off at different aggregations,</a> <a data-t="421">24 hours, 12 hours, et cetera.</a> <a data-t="425">This anomaly detection function has a bunch of parameters, and we just played around with</a> <a data-t="429">some of them and iterated on things like, what percentage of the data should we capture</a> <a data-t="433">anomalies for?</a> <a data-t="435">Should it be positive and negative direction, so above or below the expected threshold?</a> <a data-t="440">The level of significance, maybe we really need to be stringent here.</a> <a data-t="443">Maybe we can only go with 90% confidence versus 99%.</a> <a data-t="449">And then that piecewise median period that I mentioned.</a> <a data-t="452">Another thing we realized was there are some outliers that are just insane, and anomaly</a> <a data-t="457">detection would always pick up, and that's very indicative of a hardware failure, but</a> <a data-t="461">the client already knew about these.</a> <a data-t="462">They said, well, we know that these are problematic, so we actually did some outlier removal and</a> <a data-t="468">then ran the anomaly detection, and, of course, then there were no anomalies detected.</a> <a data-t="473">The input data very much makes a difference on your end results.</a></p>

<p><a data-t="478">So those are all parameters we wanted to play with, and we were literally manually analyzing</a> <a data-t="482">all of this, and it turns out putting it into a Shiny app and just allowing myself and the</a> <a data-t="488">client to play with these different parameters, we quickly got feedback on, hey, this actually</a> <a data-t="493">looks like a good threshold to use.</a> <a data-t="495">These are good parameters.</a> <a data-t="496">Let's run with these.</a> <a data-t="499">From there, we thought, well, instead of this sort of top-down approach and exploring all</a> <a data-t="503">the different options, let's alert the user on exactly where to go.</a> <a data-t="507">So we took the high-significance anomalies, highlighted them, and allowed the user to</a> <a data-t="512">then click one.</a> <a data-t="513">So I want to know more about this one, what's going on with potassium or calcium, and it</a> <a data-t="518">would bring you right back to that screen, pre-populate all the parameters, again, just</a> <a data-t="522">a nice little user experience thing, time saver, and they could dig into the details</a> <a data-t="526">a little more.</a></p>

<p><a data-t="529">The code was pretty straightforward.</a> <a data-t="531">The input data looks just like you'd expect for a time series.</a> <a data-t="534">There's a timestamp and a measure.</a> <a data-t="537">We iterated over the different significance values.</a> <a data-t="541">We also defaulted to using one year of data prior to the date range that the client is</a> <a data-t="545">actually interested in.</a> <a data-t="546">So if I care about May of 2020, let's start with data from May 1st of 2019 to get some</a> <a data-t="552">of that seasonality, and more data is better for this type of algorithm.</a></p>

<h3>Use case 2: sports and entertainment news anomalies</h3>

<p><a data-t="709">So real-world example, again, is another client that works in the sports and entertainment</a> <a data-t="713">business.</a> <a data-t="714">They actually do market research around sports technology.</a> <a data-t="718">So things like in-venue betting and understanding how to engage fans in this new technical age,</a> <a data-t="726">biometric sensors on athletes, you name it, basically any sports technology.</a> <a data-t="731">And I like to call them kind of a market research company for sports tech.</a> <a data-t="735">So what they have is a whole bunch of news articles that they collect via Feedly, DiffBot.</a> <a data-t="741">These are different third-party tools, and Leo is something that's, I believe, part of</a> <a data-t="747">Feedly, and it analyzes all the information for you.</a> <a data-t="752">It's kind of like a smart filter to bring in the information they care about.</a></p>
