<p><a data-t="0">Hello, my name is Colin Rundel, and today I'm going to tell you a little bit about a</a> <a data-t="8">package I've been working on recently called parsermd.</a> <a data-t="12">This package implements a parser for R Markdown documents using the Boost Spirit library in</a> <a data-t="16">C++ and exposes a series of functions and data structures that allow for the manipulation</a> <a data-t="22">and evaluation of R Markdown documents from within R.</a></p>

<p><a data-t="26">So to give you a little bit of context, I teach large statistical computing courses</a> <a data-t="30">and I deliver assignments to students in the form of R Markdown documents.</a> <a data-t="35">These R Markdown documents have very typical structures, so they have things like YAML</a> <a data-t="39">at the beginning, section headings to keep bits and pieces organized, instructions in</a> <a data-t="44">the form of Markdown text, and then various code chunks that include either example code</a> <a data-t="50">or areas where we want students to enter their own code to implement a solution or something</a> <a data-t="53">like that.</a></p>

<p><a data-t="55">And this works really, really well, except one of the difficulties we have is that the</a> <a data-t="58">documents contain lots of boilerplate in the form of these instructions or demo code that</a> <a data-t="63">we don't actually care about when it comes time to mark it and actually distracts from</a> <a data-t="67">the content that we are really concerned with.</a> <a data-t="69">And so part of the motivating reason for developing the parsermd package was a desire to be able</a> <a data-t="74">to take these kind of documents once the students have turned them in and then strip out all</a> <a data-t="78">of that boilerplate to get down to just the stuff that we actually care about.</a> <a data-t="82">And so today I'll show you some examples of what that looks like in practice.</a></p>

<h3>Parsing R Markdown documents</h3>

<p><a data-t="86">So at its core is this parse underscore rmd function, which reads in our Markdown document</a> <a data-t="92">and then turns it into a simple abstract syntax tree.</a> <a data-t="95">And so for our purposes right now, we don't really need to understand what that is.</a> <a data-t="99">But the key thing to see is all of those elements we just saw in that homework01.rmd are here.</a> <a data-t="104">So we see that there's the YAML is represented.</a> <a data-t="106">We have the various section headings.</a> <a data-t="108">We have the Markdown text and the code chunks.</a> <a data-t="112">And this is presented in a hierarchical way, but it's really just flat.</a> <a data-t="114">We're just inferring the hierarchy based on the various heading levels that we see within</a> <a data-t="119">the document.</a></p>

<p><a data-t="121">If this hierarchical view doesn't work for you, we can also turn this into a tidy rectangle</a> <a data-t="126">of data using the as-tbl function.</a> <a data-t="128">And so what we now have is the same representation of that data where that AST has been wrapped</a> <a data-t="133">up into a tbl.</a> <a data-t="134">We can manipulate things by looking at the various other columns that are there.</a> <a data-t="137">But that AST we were just seeing is still here.</a> <a data-t="140">It's now just in its flat form represented in that AST column on the right-hand side.</a></p>

<p><a data-t="145">So AST, what is that?</a> <a data-t="147">How do I work with it?</a> <a data-t="148">What do I care about?</a> <a data-t="149">It's a custom data structure, but it's really just a list of lists.</a> <a data-t="153">And it's all managed by S3, and it's not something that you actually need to worry about in practice</a> <a data-t="157">because the package implements a bunch of helper functions.</a></p>

<h3>Subsetting and manipulating documents</h3>

<p><a data-t="160">So if we want to subset for various elements, so in the example I have, I care about the</a> <a data-t="164">exercises and the solutions.</a> <a data-t="166">So here I can use the subset function to pull out anything that matches exercises with</a> <a data-t="170">a wildcard and then solution.</a> <a data-t="172">So this gets me all of the nodes that belong to an exercise 1, 2, or 3, and then the various</a> <a data-t="177">answers the students may have put into those particular bits of the document.</a> <a data-t="181">I can pull those out.</a></p>

<p><a data-t="184">Once I've done that, I can then also manipulate the document further.</a> <a data-t="186">So a common issue students have is they may write code that has an error, and if we were</a> <a data-t="190">to try to knit that document, we would not be able to because of the error.</a> <a data-t="194">So what we can do is we can then manipulate the code chunks to add new options.</a> <a data-t="197">So this is going to go through and set error equal to true for all of the code chunks so</a> <a data-t="201">that when we knit the document, it'll keep knitting even if there is an error at any</a> <a data-t="205">point in the document.</a></p>

<p><a data-t="207">Once we've done that, we can then, say, turn the document back in text, or we can even</a> <a data-t="212">just render it directly.</a> <a data-t="213">So in this case, I could turn it back into an HTML document and examine it.</a> <a data-t="217">And so again, this makes it very easy to then take the student's code, pull out the pieces</a> <a data-t="220">I want, and then render just the solution so that I can then mark it much more easily</a> <a data-t="224">without all of that other sort of cruft around.</a></p>

<h3>Automated feedback with templates</h3>

<p><a data-t="228">This works really well.</a> <a data-t="229">In practice, it's a little bit more difficult because we depend on the structure of the</a> <a data-t="233">document matching exactly what we expect when we're extracting things.</a> <a data-t="236">And so one of the things that I've also done in practice with this that's been able to</a> <a data-t="238">be built on top of these tools is the idea of a template, an automatic feedback for the</a> <a data-t="243">students.</a></p>

<p><a data-t="243">So an RMD template is just a simple representation of the nodes that we expect to find and the</a> <a data-t="248">various sort of labels and location references that we care about to indicate where they</a> <a data-t="253">are.</a> <a data-t="254">And it's just a simple </a><a href="/software/tibble/">tibble</a><a data-t="254"> that we can generate using the RMD template function once</a> <a data-t="257">we've subset it for the elements we want.</a> <a data-t="260">And once we have that, we can then feed it into another helper function called RMDCheckTemplate,</a> <a data-t="264">which we can then use that to compare against the student's work.</a> <a data-t="267">So here I'm comparing against homework01-student.</a> <a data-t="270">And we get a user-friendly error message out telling you exactly what's missing or what's</a> <a data-t="274">problematic about the document.</a> <a data-t="275">So in this case, the student is missing something in exercise one and exercise two.</a> <a data-t="279">And they're told explicitly what that is, either a code chunk or markdown text or what</a> <a data-t="283">it's particularly looking for in that context.</a></p>
