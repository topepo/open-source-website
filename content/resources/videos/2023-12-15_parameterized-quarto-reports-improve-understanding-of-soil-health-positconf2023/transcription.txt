<h3>Parameterized reporting 101</h3>

<p><a data-t="369">So step one is learning about parameterized reports.</a> <a data-t="373">So what is it?</a> <a data-t="375">It's essentially a template report where it's going to have placeholders where you can then</a> <a data-t="380">fill in whatever values you need.</a> <a data-t="383">So what really helped me to think of parameterized reports as functions.</a> <a data-t="388">Like really complex functions.</a> <a data-t="390">So imagine you have a Quarto file and you're trying to report something for every single</a> <a data-t="397">year.</a> <a data-t="398">So this function or this template is going to take in parameters as the function input,</a> <a data-t="404">essentially.</a> <a data-t="405">So if you were producing all these reports for each year, instead of going through and</a> <a data-t="411">changing 2019 to 2020 to 2021, you're going to put in params, dollars, and year.</a> <a data-t="418">And then the output of your function are going to be these reports for every single year</a> <a data-t="423">without copying and pasting.</a> <a data-t="425">It's amazing.</a></p>

<p><a data-t="428">So for your parameterized report, you'll have a YAML header, which stands for yet another</a> <a data-t="434">markdown language.</a> <a data-t="436">And this YAML header is going to contain your basic metadata.</a> <a data-t="439">So your title, subtitle, date, author, that fun information.</a> <a data-t="445">It'll also be where you specify your format types.</a> <a data-t="448">So in our case, we output to HTML and Word doc.</a> <a data-t="452">And this is also where you're going to set your parameters.</a> <a data-t="455">So these are key value pairs where you're naming your parameters.</a> <a data-t="459">And then you're also naming the default value that it will use.</a></p>

<p><a data-t="465">Below your YAML header is going to be your basic report content.</a> <a data-t="471">Now how do you access your parameters?</a> <a data-t="473">If you're working in </a><a href="/software/rstudio/">RStudio</a><a data-t="473"> interactively, you can run any line or code chunk.</a> <a data-t="478">And this is going to add this params object to your environment.</a> <a data-t="484">And then because this is just a list object, you can use the standard list notation and</a> <a data-t="491">access these parameters with params$farmer.</a> <a data-t="495">And then that would give you, like, Sammy Sunflower.</a> <a data-t="499">And you can also use parameters for inline code.</a> <a data-t="502">So if you enclose your expression in backtick R, your expression backtick, it will dynamically</a> <a data-t="508">calculate or render whatever your parameter is.</a> <a data-t="511">So for example, we have these are results for params$farmer, Sammy Sunflower.</a></p>
