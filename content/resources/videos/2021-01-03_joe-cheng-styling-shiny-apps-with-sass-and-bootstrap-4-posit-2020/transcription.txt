<h3>Why CSS is hard: enter Sass</h3>

<p><a data-t="641">So why is this so difficult?</a> <a data-t="645">Why does it take so much effort to just override one color?</a> <a data-t="649">This is bootstrap CSS.</a> <a data-t="656">There's a color that's being highlighted.</a> <a data-t="660">There's a hex code.</a> <a data-t="661">And this is the source code for bootstrap CSS.</a> <a data-t="665">So this is saying that this link should be blue.</a> <a data-t="669">That very specific shade of blue.</a> <a data-t="672">This color appears all throughout the CSS file 25 different times.</a> <a data-t="678">And some of them are pretty complicated.</a> <a data-t="682">So if you as a Shiny app author want to change all of those instances of blue to that specific</a> <a data-t="688">shade of orange, you would actually have to overwrite each one of those rules with custom</a> <a data-t="694">CSS.</a> <a data-t="695">And those are pretty complicated selectors that we're looking at here.</a> <a data-t="700">So clearly this is not really a tenable strategy for changing colors.</a></p>
