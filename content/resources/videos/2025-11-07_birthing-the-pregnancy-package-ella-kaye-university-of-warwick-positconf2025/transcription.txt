<h3>How it works: options and the null coalescing operator</h3>

<p><a data-t="133">So how does it work?</a> <a data-t="135">How can we make the package a pleasure to use by making it easy to call these functions</a> <a data-t="139">without needing to specify these arguments like the due date and the person</a> <a data-t="143">that aren't going to change each time you use the package?</a></p>

<p><a data-t="146">This is the function signature for howfar,</a> <a data-t="150">and you'll see that the due date and person arguments are both set by default to null.</a> <a data-t="156">And then in the function code, we have the following,</a> <a data-t="159">because we do need to have a due date to do the calculations.</a> <a data-t="162">So first of all, on line 1, it's going to look to see whether the due date</a> <a data-t="165">has been passed as an argument to the function.</a> <a data-t="168">And if it has, it's good, we're good, we're done on line 1.</a> <a data-t="171">But if it's null, as it will be by default,</a> <a data-t="174">the null coalescing operator at the end of line 1 kicks in,</a> <a data-t="178">and it was going to look for an option to see if that's been set,</a> <a data-t="182">the pregnancy due date option.</a></p>

<p><a data-t="185">If it has, again, we're good, we're done on line 2,</a> <a data-t="188">but if not, that option is null, and the null coalescing operator at the end of line 2 kicks in,</a> <a data-t="193">and we look for line 3, in which case we're going to get an error message</a> <a data-t="197">as generated by the datestop utility function.</a></p>

<p><a data-t="201">So what does that do?</a> <a data-t="203">Well, we're using rlang and </a><a href="/software/cli/">cli</a><a data-t="203">, and it's going to inform us exactly what the problem is,</a> <a data-t="210">and then it's going to tell us how we can fix it.</a> <a data-t="213">You can set the due date argument.</a> <a data-t="215">In the second bullet, it says you can set the pregnancy.dueDate option using a helper function</a> <a data-t="220">provided by the package, the setDueDate function,</a> <a data-t="223">or the recommended approach is to set options pregnancy.dueDate in your R profile,</a> <a data-t="228">because if you do that, you just need to do that once,</a> <a data-t="231">and then in every R session in the future, that R will find that due date,</a> <a data-t="235">and you can call any of the functions in the pregnancy package,</a> <a data-t="239">and you will never have to specify your due date again,</a> <a data-t="242">because it will pick it up from that option,</a> <a data-t="245">and you get a link to the documentation to provide more details.</a></p>
