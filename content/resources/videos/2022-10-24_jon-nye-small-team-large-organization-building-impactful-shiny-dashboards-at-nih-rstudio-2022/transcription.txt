<h3>Machine learning for grant classification</h3>

<p><a data-t="549">So the next example I want to give you is how we've used machine learning to classify</a> <a data-t="554">NIAID grants.</a> <a data-t="556">So just a little background on grants at NIH.</a> <a data-t="559">There are really two main types.</a> <a data-t="561">One starts at NIH.</a> <a data-t="563">So we send out a request for applications, and this is when the NIH asks the scientific</a> <a data-t="570">community to apply for funding in a very specific, well-defined area.</a> <a data-t="575">We usually use these to stimulate research in really high-priority areas.</a> <a data-t="579">The next type of application is the investigator-initiated application.</a> <a data-t="583">So instead of starting at NIH, we're starting with that investigator or scientist that's</a> <a data-t="589">running a lab somewhere, right?</a> <a data-t="591">So these make up by far the majority of our applications, greater than 85%.</a></p>

<p><a data-t="597">And so at NIH, we really have to rely on the scientific community, the creativity and ingenuity</a> <a data-t="604">that they have to come up with innovative research ideas.</a> <a data-t="608">And those ideas can be very broad and cross-cutting or very narrow and well-defined.</a> <a data-t="612">But when we get those applications, these investigator-initiated ones, we don't have</a> <a data-t="616">much information on it.</a> <a data-t="618">Besides that, it falls in the broad research areas of NIAID's mission.</a> <a data-t="624">So you can imagine that this is very difficult, right?</a> <a data-t="626">When you're dealing with thousands of applications, how do we know how much we've funded and what</a> <a data-t="632">projects we've funded in a specific research area?</a> <a data-t="635">So we get questions all the time.</a> <a data-t="636">How much have you spent on HIV vaccine research or how much have you spent on tuberculosis</a> <a data-t="643">research and what projects?</a> <a data-t="645">And we need to be able to answer those questions.</a></p>

<p><a data-t="648">So how do we do that?</a> <a data-t="650">The answer is we have a team for that.</a> <a data-t="653">So every year, NIAID gets over 9,000 grant applications, and we fund about 20% of these,</a> <a data-t="659">give or take.</a> <a data-t="660">We then have a team of about three to six people that go through these awarded grants</a> <a data-t="664">and apply codes based on the research being proposed.</a> <a data-t="667">So they have over 1,300 unique codes that they can use to describe that research.</a> <a data-t="674">And I've included a few examples here on the right.</a> <a data-t="677">So they look, are they researching a specific disease like Ebola or Zika?</a> <a data-t="681">Is the research area, you know, new therapeutics or diagnostics and so on?</a></p>

<p><a data-t="688">But you can imagine with thousands of grants, this can be challenging.</a> <a data-t="692">First, it's very time intensive.</a> <a data-t="694">So they have to go through thousands of these awarded grants and read the title, the abstract,</a> <a data-t="700">the specific aims, and describe that research.</a> <a data-t="704">Because it's so time consuming, we only have the resources to code awarded grants.</a> <a data-t="709">The 80% of applications that are unfunded, we don't have any information on.</a> <a data-t="715">And they have a constantly increasing workload.</a> <a data-t="717">So every year, we're getting more applications and giving out more awards.</a> <a data-t="721">And especially at the beginning of the pandemic, this was, we saw a huge increase in applications</a> <a data-t="725">due to COVID research.</a> <a data-t="728">And finally, because it takes so much time, this usually isn't done until the very end</a> <a data-t="731">of the year.</a> <a data-t="732">So we don't have real time data on this, which would be helpful.</a></p>

<p><a data-t="736">So we thought this was a great opportunity to use supervised machine learning for grant</a> <a data-t="741">classification.</a> <a data-t="741">We had a database of thousands of grants that had been manually coded over the past decade</a> <a data-t="746">that we could use to train machine learning models.</a> <a data-t="749">So here's an example of a grant that's proposing to research a more sensitive diagnostic for</a> <a data-t="755">tuberculosis.</a> <a data-t="757">When I looked up the codes on this grant, you could see the disease was tuberculosis.</a> <a data-t="762">The research area was diagnostics, and it was a clinically focused grant.</a></p>

<p><a data-t="766">So I just want to give you one example of how we've used machine learning to classify</a> <a data-t="771">grants based on the research area.</a> <a data-t="773">So here, it was diagnostics.</a> <a data-t="776">But there are five categories, including therapeutic, prevention, and vaccine.</a> <a data-t="781">So we took about 7,500 grants that had previously been manually coded and used them to train</a> <a data-t="789">three different machine learning models, including logistic regression, support vector machine,</a> <a data-t="793">and random forest to compare the three.</a> <a data-t="796">We then tested these models using about 3,200 grants that had also been manually coded.</a> <a data-t="803">And so what we found was the logistic regression was the most accurate, 88.5%.</a> <a data-t="808">And it also did the best at predicting this relatively small category of grants in the</a> <a data-t="812">prevention class.</a></p>

<p><a data-t="815">So we use this machine, this logistic regression model, to build a Shiny dashboard for coding</a> <a data-t="821">NIAID grants.</a> <a data-t="822">And we then expanded it to assist in coding in five additional categories.</a> <a data-t="828">So behind the scenes, once again, we have an R Markdown document that's pulling in the</a> <a data-t="832">data from multiple sources, making those machine learning predictions, and writing</a> <a data-t="837">this data to a database.</a> <a data-t="839">It's constantly getting updated with new applications, new awards.</a> <a data-t="843">And this is what our Shiny dashboard uses.</a> <a data-t="846">And once again, we wanted people to be able to collaborate.</a> <a data-t="850">So it's writing user inputs in a database for future use.</a></p>
