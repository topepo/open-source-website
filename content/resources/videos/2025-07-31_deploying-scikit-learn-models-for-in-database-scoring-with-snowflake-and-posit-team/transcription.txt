<h3>Orbital capabilities and limitations</h3>

<p><a data-t="1259">Let's jump into some of the things that.</a> <a data-t="1262">Orbital can do that Snowflake so that Orbital can do and Orbital can't do.</a> <a data-t="1266">So you might be thinking to yourself, this is awesome.</a> <a data-t="1269">I want to use Orbital for everything.</a> <a data-t="1271">Orbital is a fantastic capability, although because it is converting the SQL, there are a few limitations.</a> <a data-t="1277">As I mentioned, Orbital has a ton of capability to convert different models, convert linear regressions,</a> <a data-t="1284">logistic regressions, tree based models, XGBoost models, things like that.</a> <a data-t="1288">Those convert just fine.</a></p>

<p><a data-t="1291">But let's say you want to use a model that has some element of recursivity.</a> <a data-t="1295">Something like k-nearest neighbor.</a> <a data-t="1297">Well, SQL doesn't have recursivity.</a> <a data-t="1300">You can't write a recursive function in the SQL.</a> <a data-t="1302">So unfortunately, you can't use Orbital for that.</a> <a data-t="1305">But you can always use Snowflake's Python runtime.</a></p>

<p><a data-t="1309">The other thing to keep in mind is that Orbital does not support and probably won't support something like a neural network model,</a> <a data-t="1320">something like a </a><a href="/software/tensorflow/">TensorFlow</a><a data-t="1320"> model or a Keras model.</a> <a data-t="1321">The reason for that being is those models involve a ton of parallelization with low numeric precision,</a> <a data-t="1327">which SQL, quite frankly, these SQL engines are designed for high precision.</a> <a data-t="1332">They can parallelize.</a> <a data-t="1333">There's nothing mathematically to say that we can't build an Orbital implementation of like, let's say, a Keras model.</a> <a data-t="1343">It would be one giant case when statement, but that's mathematically something that we could do.</a> <a data-t="1349">Those models are so much better served with GPUs because SQL really doesn't run on GPUs.</a> <a data-t="1356">That's where something like Snowpark is going to come in a ton of handy because you can have GPUs attached to Snowflake and Snowpark.</a></p>

<p><a data-t="1364">But again, for those cases where I'm sure a lot of us are doing a ton of them all the time,</a> <a data-t="1369">where we're building models, building a logistic regression, building a tree based model,</a> <a data-t="1375">those take up, those deliver so much value to our customers, to our stakeholders,</a> <a data-t="1382">that being able to convert those into SQL, being able to deploy those onto Snowflake effortlessly,</a> <a data-t="1387">being able to iterate as fast as I just showed you is going to be a huge step change in your capabilities.</a> <a data-t="1393">You're going to be able to deploy models faster.
