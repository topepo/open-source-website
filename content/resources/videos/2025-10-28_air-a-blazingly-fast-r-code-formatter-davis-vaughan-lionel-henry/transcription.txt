<h3>Adoption and collaborative friction</h3>

<p><a data-t="439">If that sounds cool to you, and you're considering adopting Air, know that you will be in good</a> <a data-t="443">company.</a> <a data-t="444">We have already transitioned most of our packages over to using it, including a number of community</a> <a data-t="449">packages like Nanier and iGraph.</a> <a data-t="452">At this point, Air has been run over hundreds of thousands of lines of R code.</a> <a data-t="456">We feel pretty good about the formatting.</a> <a data-t="458">It feels pretty stable, and we're confident about it at this point.</a> <a data-t="462">And it's truly a great time to adopt this tool.</a></p>

<p><a data-t="476">So our goal with the formatter is that once you start using it, it will be hard for you</a> <a data-t="476">to go back to your previous formatter-less life.</a> <a data-t="481">And that's something that we have managed to achieve with Hadley, because now, when</a> <a data-t="485">he saves a file and he doesn't reformat, it feels broken.</a> <a data-t="490">So we knew that Hadley would be an easy win, because he cares a lot about style.</a> <a data-t="495">And actually, many of us in the organization do.</a> <a data-t="498">And I want to talk about what happens when you have a room full of programmers who all</a> <a data-t="502">care a lot about style.</a></p>

<h3>Flexibility: persistent line breaks and magic commands</h3>

<p><a data-t="682">So does that mean that with this trick, a formatter, you have absolutely no say into</a> <a data-t="688">how to format your code?</a> <a data-t="690">Well, not quite.</a> <a data-t="692">Because one of our design goals was to make it easy to still have some flexibility.</a> <a data-t="698">And that happens in two ways.</a> <a data-t="700">First, persistent line breaks and magic commands.</a> <a data-t="703">And to give you an example for persistent line breaks, here we have a couple of R expressions,</a> <a data-t="709">a list call and a ggplot pipeline.</a> <a data-t="712">And the idea is that if you insert a line break in the first element of a list call,</a> <a data-t="718">for instance, that will be taken as an instruction for the formatter to go from the flat layout</a> <a data-t="725">to the expanded layout, which is more vertical.</a> <a data-t="727">You can do the same with the ggplot pipeline.</a> <a data-t="731">So you find the first element.</a> <a data-t="733">You add a line break.</a> <a data-t="735">You save.</a> <a data-t="736">It reformats to the expanded layout.</a> <a data-t="739">That works in the opposite direction as well.</a> <a data-t="741">You can remove the line break, save, go back to flat.</a> <a data-t="745">And same for the list call.</a> <a data-t="746">Remove the line break, and it goes back to flat.</a></p>

<p><a data-t="750">So that's one of our favorite features with the formatter, because it really gives you</a> <a data-t="755">a grip on your code.</a> <a data-t="757">It's super fun to use, and we hope you will like it, too.</a></p>

<p><a data-t="763">Now another way we provide you some flexibility is magic commands.</a> <a data-t="769">So here you have an example from igraph, which is a package for specifying graphs.</a> <a data-t="774">And it uses operators, the minus and plus operators, to do that.</a> <a data-t="778">So here you have a graph specification.</a> <a data-t="781">And if you try to format this with air, there's no way that it's going to know how to do a</a> <a data-t="785">good job with this operator.</a> <a data-t="787">So that will not look like a proper graph anymore.</a> <a data-t="792">So what you can do is use a magic command.</a> <a data-t="795">In this case, it's a skip command.</a> <a data-t="797">And that's like an instruction to ignore the next line of code and to leave it as is.</a> <a data-t="804">So of course, if you use igraph a lot, you're not going to want to insert magic commands</a> <a data-t="810">everywhere.</a> <a data-t="811">So we also have a configuration option to specify which functions should be skipped</a> <a data-t="815">by the formatter.</a></p>
