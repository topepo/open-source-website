<h3>Introduction to Quarto</h3>

<p><a data-t="181">Quarto is an open-source scientific and technical publishing system built on Pandoc.</a> <a data-t="189">It's a project sponsored by Posit, taking what we've learned from creating our markdown with the goal of making it easier to create scientific documents.</a> <a data-t="197">You can write together narrative and code to produce beautiful and professional outputs.</a></p>

<p><a data-t="202">Quarto offers a unified syntax for core features and maintains consistency across all formats.</a> <a data-t="212">Features like tab sets and code folding and cross-referencing all function in the same way.</a> <a data-t="218">Additionally, Quarto provides a universal theming system based on SAS for all HTML output, using core variables defined by </a><a href="/software/bslib/">BSLib</a><a data-t="218">.</a> <a data-t="227">Quarto can be called from the terminal or the command line and is completely independent of computational systems.</a> <a data-t="234">While Quarto supports NIDAR, the engine behind our markdown, it can also support other systems.</a> <a data-t="240">And ultimately, the goal is to create dynamic outputs that are reproducible and easy to create.</a></p>

<p><a data-t="245">Quarto enables single-source publishing, where different output formats like websites, presentations, and more are created from a single source.</a></p>

<p><a data-t="254">Here's how Quarto works in the backend.</a> <a data-t="256">An engine is the tool that runs code.</a> <a data-t="258">This is the basic structure that creates documents from the source, the Quarto markdown file or the QMD file.</a> <a data-t="265">When we render a Quarto document first, the engine executes all of the code chunks and generates an intermediary markdown file that includes both the code and the code output.</a> <a data-t="275">Then that markdown file is processed by Pandoc to create the finished document.</a></p>
