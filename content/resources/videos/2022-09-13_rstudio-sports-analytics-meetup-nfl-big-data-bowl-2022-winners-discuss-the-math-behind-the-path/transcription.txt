<h3>Building the optimal path</h3>

<p><a data-t="414">Cool, thank you. So now that we know what the punt returner</a> <a data-t="418">needs to consider, and what our goals are, we want to be able to</a> <a data-t="423">find show the best path to the end zone, like we see in green</a> <a data-t="427">on the figure here. <p><a data-t="437">So the first step in building our optimal path is to find gaps</a> <a data-t="440">in the kick teams coverage. For this we use the Lonnie</a> <a data-t="443">triangulation. So for this,</a> <a data-t="465">with the punt returner going around all the players and</a> <a data-t="468">towards the sidelines, we add new gaps that connect the players</a> <a data-t="471">on the outside of the hall to the boundaries of the zone of</a> <a data-t="474">interest. <h3>Penalized expected arrival time</h3>

<p><a data-t="488">And now that we have these windows to find that the punt</a> <a data-t="492">returner can move through on his way towards the end zone, we</a> <a data-t="495">want to move towards quantifying the pressure along</a> <a data-t="498">each of these windows. And we do this using our penalized</a> <a data-t="500">expected arrival time algorithm. To give a bit of visual</a> <a data-t="504">intuition behind this algorithm, if you look at that heat line</a> <a data-t="507">in the top left of the plot here, you can see that towards</a> <a data-t="510">the sideline, it's colored green, indicating that there's</a> <a data-t="513">a lot of open space there. And it's going to take the members</a> <a data-t="516">of the kicking team a long time to get to that position. While</a> <a data-t="519">as you move closer and closer to where all the players are, you</a> <a data-t="522">that it goes from green to yellow to red, indicating that</a> <a data-t="526">there's more pressure and less time for the punt returner to</a> <a data-t="529">move through that area.</a></p>

<p><a data-t="534">And to kind of formalize this a little</a> <a data-t="534">bit more, the goal of this algorithm is to obtain the</a> <a data-t="537">minimum time we expect it to take a member of the kicking</a> <a data-t="539">team in red here to reach a target location as indicated by</a> <a data-t="544">the X here. And we're going to take a bunch of different target</a> <a data-t="547">locations. But let's just focus on this single spot right here.</a></p>

<p><a data-t="552">So with that target location, we're going to look at every</a> <a data-t="555">kicking team player on the field or every player in red, that's</a> <a data-t="558">trying to stop the punt returner from moving forward. And let's</a> <a data-t="563">say we just look at number 97, for example, to start off here.</a> <a data-t="568">So with number 97, what we're going to do is we're going to</a> <a data-t="570">create a straight line connecting him to that target</a> <a data-t="573">location. And once we do that, we're going to find all blockers</a> <a data-t="577">that we can project onto this straight line at a 90 degree</a> <a data-t="580">angle. And that's going to sort of be all of the blockers that</a> <a data-t="584">can really reasonably impede his path from getting to that target</a> <a data-t="587">location. So here we have number 82 and number 19. And for these</a> <a data-t="592">blockers, we're going to take two different measurements. So</a> <a data-t="596">first, we're going to take the length of that projection from</a> <a data-t="598">82 to the straight line, as indicated by D1 here. And that's</a> <a data-t="602">going to be a measure of the lateral distance from the kicking</a> <a data-t="605">team player. And then we're going to take the distance from</a> <a data-t="609">number 97 to that projection. And that's going to be</a> <a data-t="613">indicated by D2 here. And that's sort of the forward distance</a> <a data-t="617">from the kicking team player to the blocker.</a></p>

<p><a data-t="621">And with these two</a> <a data-t="621">measurements, we're going to attribute time penalties for</a> <a data-t="624">each of the blockers using what we call a projected Gaussian</a> <a data-t="627">kernel. And that's characterized by the equation right here and</a> <a data-t="632">the heat map below. And sort of what we're trying to do here is</a> <a data-t="635">to create an equation where we're kind of accurately</a> <a data-t="639">capturing how much a blocker is likely to impede a member of the</a> <a data-t="644">kicking team's path to that target location. Right, so you</a> <a data-t="648">can see here that when you're very close and right in front of</a> <a data-t="651">number 97, you're going to impose a very high penalty.</a> <a data-t="654">Whereas as we move further and further out, you can be a bit</a> <a data-t="657">more lateral from the kicking team player or from number 97.</a> <a data-t="662">But you you have less of a penalty overall, right? Because</a> <a data-t="665">if you're say you're right in front of number 97, like one foot</a> <a data-t="669">forward from him, but five feet to the left, then you're less</a> <a data-t="674">likely to actually catch him or impede him than if you're, say,</a> <a data-t="679">10 feet forward from him and five feet to the left.</a></p>

<p><a data-t="684">And once</a> <a data-t="684">we assign these blocker penalties, we're going to simply</a> <a data-t="687">add them all up. So with number 19, and number 82, they impose</a> <a data-t="691">a 0.54 and 0.31 second penalty on number 97, respectively,</a> <a data-t="697">giving us a total penalty of 0.85 seconds by these two</a> <a data-t="701">blockers. Then we're going to calculate the expected time to</a> <a data-t="705">the target location for number 97, given that he runs at a</a> <a data-t="709">straight line speed of seven yards per second, and that's</a> <a data-t="712">going to be 2.34 seconds. Then we add these two values up. So</a> <a data-t="716">the time that we expect him to take to get to that target</a> <a data-t="719">location unimpeded as 2.34, and the blocker penalty that we</a> <a data-t="724">expect to happen as 0.85. And that's going to give us 3.19</a> <a data-t="728">seconds for number 97 to that target. And we're going to</a> <a data-t="733">repeat this process over and over for every member of the</a> <a data-t="736">kicking team.</a></p>

<p><a data-t="739">And on the left here, you can see we have a</a> <a data-t="739">little table with the top few players here. And number 17 is</a> <a data-t="743">the quickest to get to that target location at 0.92 seconds</a> <a data-t="746">in this in this case. And that's sort of that's going to</a> <a data-t="749">be our measure of the time we expect a member of the kicking</a> <a data-t="754">team to get to that location, or our penalized expected arrival</a> <a data-t="757">time. And we're going to repeat this process over and over for</a> <a data-t="761">up to 20 evenly spaced points along each of these windows.</a></p>

<h3>A* search algorithm and optimal path</h3>

<p><a data-t="793">Great, so now that we have our mesh structure and a waiting</a> <a data-t="796">system for it, we need to find a way to get the punt returner to</a> <a data-t="799">the end zone. And to do this, we use the A star search algorithm.</a></p>

<p><a data-t="805">So it works a little bit like a GPS and just finds the safest</a> <a data-t="808">route while avoiding traffic. Our algorithm looks at a</a> <a data-t="812">heuristic function, which defines how many yards are</a> <a data-t="815">remaining from the target to the end zone, and a cost function.</a> <a data-t="820">This cost function describes the distance between the punt</a> <a data-t="823">returner and the target, the danger associated with the</a> <a data-t="826">target point, as well as how risky of a target the punt</a> <a data-t="831">returner may be willing to consider traveling to. The</a> <a data-t="835">algorithm looks at various possible routes and restarts</a> <a data-t="839">whenever the total of the heuristic and cost function</a> <a data-t="842">become too large.</a></p>

<p><a data-t="849">We can set a few constants to help us with</a> <a data-t="849">the trade off between risk and reward for the punt returner.</a> <a data-t="852">So today, we're going to really focus on the alpha term and see</a> <a data-t="855">how changing it might adjust this trade off between risk and</a> <a data-t="859">reward. So for our submission, if you have the chance to look</a> <a data-t="865">at it, we used alpha of 0.5. So we consider this to be a good</a> <a data-t="870">mixture of risk and reward for a punt returner wanting to gain</a> <a data-t="873">yards while still avoiding the tackle, avoiding injury, stuff</a> <a data-t="877">like that. If we were to change alpha to be one, the path</a> <a data-t="881">considers only yards remaining to the end zone with no care of</a> <a data-t="885">the danger of being tackled. So this would mean that you're</a> <a data-t="888">going basically straight to the end zone and hopefully that you</a> <a data-t="890">can push everyone out of the way. And then setting alpha to</a> <a data-t="894">zero finds the safest path available without considering if</a> <a data-t="898">it's really going to bring you closer to the end zone. So this</a> <a data-t="902">typically will send you along the sidelines and push you the</a> <a data-t="905">rest of the way.</a></p>

<h3>Fréchet path deviation</h3>

<p><a data-t="908">And now that we have this optimal path defined, let's just</a> <a data-t="911">take a look at a case here where we set alpha equal to 0.5. And</a> <a data-t="915">we want to compare the optimal path versus our the actual</a> <a data-t="919">observed path for the punt returner. And to do that, we</a> <a data-t="922">look over the next five yards for each of the path. So as you</a> <a data-t="925">can see on the top right here in the visual, we have sort of the</a> <a data-t="928">highlighted black and green line over the next five yards. And we</a> <a data-t="932">want to calculate the crochet distance over this time. And</a> <a data-t="936">crochet distance is a measure of difference between two paths</a> <a data-t="939">where essentially intuitively you could think of it as one</a> <a data-t="942">path being a man, the other path being a dog. And the</a> <a data-t="946">crochet distance is the minimum length of a leash you need to</a> <a data-t="949">traverse from one end of the paths to the other end.</a></p>

<blockquote><a data-t="936">crochet distance is a measure of difference between two paths</a> <a data-t="939">where essentially intuitively you could think of it as one</a> <a data-t="942">path being a man, the other path being a dog. And the</a> <a data-t="946">crochet distance is the minimum length of a leash you need to</a> <a data-t="949">traverse from one end of the paths to the other end.</a></blockquote>

<p><a data-t="954">You can see visually a bit of visual intuition behind this. With the</a> <a data-t="959">blue lines here, you can see that sort of like steps along</a> <a data-t="962">each of these paths. <p><a data-t="975">And with this crochet distance, we</a> <a data-t="975">also have to take into account a couple of other things. Just in</a> <a data-t="979">case there are cases where there's less than five yards</a> <a data-t="981">remaining in either the observed or the optimal path. So if there</a> <a data-t="985">aren't, so if both paths are more than five yards, then we</a> <a data-t="988">just simply calculate the crochet distance. And that's</a> <a data-t="990">sort of our measure of deviation from the optimal path in that</a> <a data-t="995">moment. But if either of the paths are between one and five</a> <a data-t="1000">yards, then we're going to multiply the crochet distance</a> <a data-t="1003">by five and divide it by the minimum of those two, the</a> <a data-t="1006">minimum length of those two paths to sort of scale up and</a> <a data-t="1010">account for the fact that that path, we aren't really seeing a</a> <a data-t="1013">full five yard movement. So we're trying to kind of</a> <a data-t="1017">counteract that. And if there's less than one yard on either of</a> <a data-t="1020">the paths, then we omit the calculation since we don't</a> <a data-t="1023">really have enough information to see how the punt return was</a> <a data-t="1026">moving relative to the optimal path.</a></p>
