<p><a data-t="12">So this is a feature that has been asked for a lot.</a> <a data-t="15">We know it's been requested in </a><a href="/software/rstudio/">RStudio</a><a data-t="15"> for a while, and in Positron this made the top five list of most requested features.</a> <a data-t="22">I think I'd been on the team less than a day when I found out this is what I'd be working on before I went GA in July.</a> <a data-t="27">And in March of 2025, we finally got to introduce this to Positron.</a></p>

<p><a data-t="32">You'll notice that this feature is referred to by a couple different names, Multiple Console Sessions or Multiple Interpreter Sessions.</a> <a data-t="38">It's all the same feature in case that naming confuses you a bit.</a> <a data-t="42">I'm going to be referring to it as Multiple Interpreter Sessions.</a></p>

<h3>How sessions work</h3>

<p><a data-t="45">So the first thing I want to do is walk us through how this feature works, like how you create a session, give you an overview of the session UI, and then talk about what an active session is.</a></p>

<p><a data-t="57">The main way you interact with Interpreter Sessions is through the console pane.</a> <a data-t="61">You create sessions via the Create button.</a> <a data-t="63">You've already got a session.</a> <a data-t="65">Clicking the plus button creates a copy of the active session.</a> <a data-t="67">That's the highlighted session.</a> <a data-t="69">And we'll learn a little bit more about what the active session is later.</a> <a data-t="72">But if you click the dropdown next to the plus button, you'll see a list of runtimes Positron has discovered.</a> <a data-t="77">You can pick one from there to create a session for it.</a></p>

<p><a data-t="80">So once we've got more than one session, you'll see them listed to the right of the console pane.</a> <a data-t="85">These are all the sessions that are running concurrently.</a> <a data-t="88">This means you can have each one running code at the same time.</a> <a data-t="91">We've also introduced icons to the left of the session names that indicate the code execution state.</a> <a data-t="97">So these icons are used to communicate whether a session is idle or if it's busy executing code.</a> <a data-t="102">This should make it really easy to see the state of each session at a glance.</a></p>

<p><a data-t="107">You can also rename your sessions.</a> <a data-t="110">This is handy when you need to tell sessions of the same runtime apart.</a></p>

<p><a data-t="114">So we know how to create and manage sessions.</a> <a data-t="117">Next question is, how do we know which session the IDE is using?</a> <a data-t="119">That's where the active session comes in.</a> <a data-t="122">The active session is the one you've selected from the console pane that is powering the IDE.</a> <a data-t="126">Editor helpers like Code Completions and Diagnostics use this active session.</a></p>

<p><a data-t="132">So when the active session changes, the editor context also instantly updates.</a> <a data-t="137">And it's not just editor features.</a> <a data-t="139">Other UI panes like Plots and Variables also get updated to show you results from this active session.</a></p>

<h3>Use cases for multiple sessions</h3>

<p><a data-t="146">So we've got the basics of managing sessions down.</a> <a data-t="149">Now I want to walk us through a couple different use cases</a> <a data-t="151">that really highlight how multiple sessions removes a lot of friction from multitasking workflows when compared to RStudio.</a> <a data-t="157">We're going to focus on having two concurrent sessions, but this does work for any number of sessions.</a></p>

<p><a data-t="163">I'm sure many of you have been in the situation where you have a long-running task you need to kick off</a> <a data-t="167">and a data set that you want to do some analysis on at the same time.</a> <a data-t="171">In RStudio, you'd probably have handled this by starting a background job or maybe by having two separate RStudio instances open.</a> <a data-t="178">In Positron, all you have to do is create another session from the console pane like we covered earlier.</a> <a data-t="183">You could have two sessions of the same runtime running different code.</a> <a data-t="187">The only thing you need to be aware of is which session is active when you go to execute a piece of code.</a></p>

<p><a data-t="194">And this is all with that native UI support in the IDE that you're used to.</a> <a data-t="197">You also get to keep the full state of logs, variables, execution history, and errors for each session.</a></p>

<blockquote><a data-t="197">You also get to keep the full state of logs, variables, execution history, and errors for each session.</a></blockquote>

<p><a data-t="205">When it comes to using different runtime versions of a language,</a> <a data-t="208">we know in RStudio getting multiple IDE instances up against different R versions involves setting an environment variable,</a> <a data-t="215">setting a directory alias, or even using a third-party tool.</a> <a data-t="219">But because Positron was made with the intention to natively run against different runtimes,</a> <a data-t="223">the ability to run concurrent sessions isn't limited to a single runtime version.</a> <a data-t="227">So if you're trying to test your code against two versions of R, like R4.3 and R4.6,</a> <a data-t="233">you can get both versions running at the same time.</a> <a data-t="235">And comparing outputs across both is really simple because the context of each session is just a single click away.</a></p>

<p><a data-t="241">So this can be helpful if you're a package developer, instructor,</a> <a data-t="244">or anyone who needs to maintain backwards and forwards compatibility for a project.</a></p>

<p><a data-t="250">I want to wrap up with this last idea, which is that you're not limited to your sessions being in the same language either.</a> <a data-t="256">Positron's a polyglot IDE, which means you can have sessions for different languages running side by side.</a> <a data-t="262">So if there's a Python package that makes data cleaning a breeze, but you want to use R and </a><a href="/software/shiny-r/">Shiny</a><a data-t="262"> for everything else,</a> <a data-t="267">you can create a workflow to do that pretty easily in Positron.</a> <a data-t="271">So whether it's different runtimes or even different languages,</a> <a data-t="273">multitasking should be easier with a lot less friction now.</a></p>
