<h3>Joining data sets</h3>

<p><a data-t="2610">Now normally when you want to put data sets together, you need to do something a little more involved, and that's what I call joining data.</a> <a data-t="2615">So here's a trivial example.</a> <a data-t="2615">Here's two data frames.</a> <a data-t="2615">They both talk about the Beatles.</a> <a data-t="2620">One's a list of songs and who wrote each of the songs.</a> <a data-t="2625">Then one's a list of the artists in the Beatles and what each one played or one of the instruments they played.</a> <a data-t="2630">And if you wanted to put these things together into the same data set, so you match up the instrument that each Beatle played with, you know, what song they wrote,</a> <a data-t="2636">you can do that with a left join.</a> <a data-t="2636">What left join does is it takes the names of two data frames, and then it takes some joining criteria.</a> <a data-t="2646">These would be the name or names of columns that both data sets share.</a> <a data-t="2654">And then what it will do is it'll go through the second data frame, find the pieces of information that apply to rows in the first data frame, and add them where they belong in that data frame.</a> <a data-t="2662">So order is not preserved.</a> <a data-t="2662">The second data frame's order is not preserved.</a> <a data-t="2667">First data frame, yes, that is preserved.</a> <a data-t="2667">But things are moved around to supply a play column that actually makes sense with your data frame.</a> <a data-t="2671">And at the end, you have rows in your output that describe a unified observation.</a> <a data-t="2678">John both wrote Across the Universe and played guitar.</a> <a data-t="2683">Guitar had to appear twice, because John's also in the second row, and so it appears twice.</a> <a data-t="2688">There's no information about this last row, Buddy Holly, who made Peggy Sue.</a> <a data-t="2688">So for the plays column, that gets an NA.</a></p>

<p><a data-t="2775">So we just released two new joins in the slides.</a> <a data-t="2775">You'll find them on the cheat sheet.</a> <a data-t="2775">Those are right join and outer join.</a> <a data-t="2779">So right join is like a left join, but it has the same effect as if you switch the order the data frames appear.</a> <a data-t="2786">If you put artist first and song second, that would give you the equivalent of a right join.</a> <a data-t="2791">The second data frames prioritize the first.</a> <a data-t="2791">An outer join is the complement of an inner join.</a> <a data-t="2791">An inner join only keeps the rows that appear in both data frames.</a> <a data-t="2797">An outer join keeps every row that appears in any data frame, and you get a very big data frame at the end,</a> <a data-t="2801">but that probably has a bunch of missing values.</a> <a data-t="2801">You can look those up on the cheat sheet.</a></p>
