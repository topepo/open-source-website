<h3>Hierarchical forecasting at Stripe</h3>

<p><a data-t="213">So, um, at Stripe, we're interested in forecasting as many businesses are in understanding how much payment volume are we going to have next year?</a> <a data-t="216">How many active users will we have?</a> <a data-t="218">What will our uptime be?</a> <a data-t="220">There's a number of forecasting problems that come into play that, you know, are a business advantage.</a> <a data-t="226">If we are good at them, we can set the budget, we can set our targets, we can direct our efforts appropriately.</a></p>

<p><a data-t="241">I really, as a side note, I like forecasting as a data science area, because it's not one that you always learn about in a machine learning course or spend a lot of time thinking about in your more formal education.</a> <a data-t="250">I didn't at least.</a> <a data-t="252">But now in, you know, sort of on the business side, it's an extremely practical business problem to be able to solve at the same time.</a> <a data-t="260">It's very interesting statistically.</a></p>

<p><a data-t="268">So we do what's called hierarchical forecasting.</a> <a data-t="272">This means that we want to have a forecast for Stripe overall.</a> <a data-t="276">We want to know, you know, what's our total payment volume going to be for the whole company.</a> <a data-t="279">But we also want to know what is it going to be for each region?</a> <a data-t="281">What is it going to be for each business type?</a> <a data-t="283">What's it going to be for each cohort?</a> <a data-t="284">There's all these dimensions that flow down.</a></p>

<p><a data-t="287">So you can kind of think of this as a hierarchy, right?</a> <a data-t="289">Where all of the dimensions then need to add up to the total, right?</a> <a data-t="293">We have to have a coherent hierarchy.</a> <a data-t="295">That's how we talk about it.</a></p>

<p><a data-t="300">And so we use this method where we can actually choose any forecasting method we want.</a> <a data-t="302">For each node of the hierarchy, you know, we use a lot of classical forecasting methods like ARIMA and exponential smoothing, we also experiment with other types of methods.</a> <a data-t="311">And we use those to make estimates and then we apply sort of a meta model on top of it, hierarchical reconciliation, to basically tweak each forecast as little as possible so that they all still add up to the total forecast.</a> <a data-t="325">And that then, you know, we apply some additional adjustments on top of that, and that becomes the budget for the company.</a></p>

<p><a data-t="384">Yeah, we do.</a> <a data-t="384">We're actually building a platform to do general hierarchical forecasting for any metric at Stripe.</a> <a data-t="390">And so we've expanded this.</a> <a data-t="391">We do some sales planning right now, which is a little bit financial.</a> <a data-t="396">We do it for our active users and we're looking at things like uptime as well, like how much should we expect, you know, Stripe to be live.</a> <a data-t="412">And we also do a lot in ops too, so capacity planning for our support volume, support tickets.</a></p>

<h3>Handling COVID and macroeconomic factors</h3>

<p><a data-t="1516">Most forecasting models look at historical data to predict the future, right?</a> <a data-t="1522">So that is assuming that the past is the future, right?</a> <a data-t="1525">And of course we understand that can change, right?</a> <a data-t="1528">So for example, when COVID came along, it disrupted many models, right?</a> <a data-t="1532">Nobody could have forecasted that COVID would have had the effect that it had on businesses, right?</a></p>

<p><a data-t="1565">We do include external regressors.</a> <a data-t="1566">And now of course, like we didn't predict COVID, when I look back now, forecasting was so easy pre COVID and now post COVID it's a whole different ball game.</a> <a data-t="1580">So now we do include regressors that try to account for the past COVID changes in our data.</a> <a data-t="1585">What we don't want is to continue forecasting more COVID in the future.</a></p>

<h3>Recommended books and frameworks</h3>

<p><a data-t="3360">One of my favorites is The Manager's Path by Camille Fournier.</a> <a data-t="3366">She was the CTO at Rent the Runway.</a> <a data-t="3366">And she takes you through how you can think about your career from, as a manager, from managing an intern to managing a few folks, managing a larger team, all the way sort of up to being an executive.</a> <a data-t="3376">I think that's very useful both for ICs and managers to understand how to think about your career at various levels.</a> <a data-t="3390">It's certainly one that I revisit from time to time as I've moved through my career.</a></p>

<p><a data-t="3403">The High Output Management book by Andy Grove is a classic.</a> <a data-t="3406">It's one that we reference a lot at Stripe as well.</a> <a data-t="3410">These are more management focused, but I think good resources for thinking about your career.</a></p>

<p><a data-t="3416">One framework that I have been using in career conversations recently is kind of interesting.</a> <a data-t="3418">It's called the SCARF framework.</a> <a data-t="3418">It's a neurobiology thing, and it's about how you respond to stressful situations.</a> <a data-t="3431">It's things like connectedness, autonomy, relationships, and thinking about what's important to you in your work.</a> <a data-t="3437">And so I use that and sort of ask people to talk about what resonates with them in the SCARF framework.</a> <a data-t="3443">And it's funny how different people really value different things when it comes to their work and what's important to them.</a> <a data-t="3449">And so I found that a very helpful framework for learning about folks and for folks to sort of self-identify what's important to them.</a></p>
