<h3>Falling into the same trap with programming languages</h3>

<p><a data-t="94">And even though I already made this experience and you'd think I'd know better, when it came to programming I</a> <a data-t="98">fell into the same mindset trap where I had this one programming language, in my</a> <a data-t="103">case R, where I felt like I can use this for everything and I don't need to know</a> <a data-t="108">about anything else.</a> <a data-t="113">But just like it was a mistake to think that I am only</a> <a data-t="113">someone from probability and stats and I cannot and won't do anything else, it is</a> <a data-t="118">a mistake to not appreciate that our programming languages can help you and</a> <a data-t="124">have something to share inside of your project, even if you still have one</a> <a data-t="129">preferred language.</a></p>

<p><a data-t="133">And for example, if you work on a particular problem, then for example if you work with reports, then you might want to use HTML and CSS</a> <a data-t="138">to make things look pretty.</a> <a data-t="143">Or if you want to do machine learning and AI, then Python might have something for you.</a> <a data-t="148">And if you build web apps, for example in R or </a><a href="/software/shiny-r/">Shiny</a><a data-t="148">, then it can still help you to bring in tiny nuggets of JavaScript into</a> <a data-t="153">your project to get things done faster or make it easier.</a></p>

<h3>Frameworks that transcend languages</h3>

<p><a data-t="160">And the thing is you usually like a particular language because the frameworks that are</a> <a data-t="164">implemented in that language are so nice.</a> <a data-t="169">For example, if you are into data visualization, then you might like the grammar of graphics, which in R is</a> <a data-t="172">implemented in ggplot.</a> <a data-t="177">And it turns out in Python there's a library called plot9, which does the exact same thing.</a> <a data-t="181">That means it implements the grammar of graphics.</a> <a data-t="187">And in these two packages it's even particularly nice because they use</a> <a data-t="192">the exact same code to implement this framework.</a></p>

<p><a data-t="197">And when it comes to data wrangling, you might like to use </a><a href="/software/dplyr/">dplyr</a><a data-t="197"> in R because it uses verbs like mutate</a> <a data-t="202">and chains things together with pipe.</a> <a data-t="207">And there you might have to, if you go into other languages like Python, you might have to do some switching of the</a> <a data-t="212">terminologies by using dots instead of our pipe operator and using the with</a> <a data-t="217">columns function instead of the mutate function.</a> <a data-t="223">But in general, the ideas of chaining things together can still be achieved in a different language.</a></p>
