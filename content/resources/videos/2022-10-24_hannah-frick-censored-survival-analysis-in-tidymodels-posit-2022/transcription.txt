<h3>Extending tidymodels with the censored package</h3>

<p><a data-t="159">So survival analysis is an important field of modeling and we're extending support for it in tidymodels.</a> <a data-t="165">The first milestone in this are the models themselves, to be able to specify them, fit them and predict with them.</a> <a data-t="180">And you may well be aware that R has an abundance of riches in different models contributed by different people, and it also has nearly the same abundance of riches in interfaces to these models.</a> <a data-t="208">And as you just heard, tidymodels wants to make it effective, safe and ergonomic for you, and part of that is to provide you with a consistent interface.</a> <a data-t="221">And the aspect in tidymodels that takes care of a consistent interface for models themselves is </a><a href="/software/parsnip/">parsnip</a><a data-t="221">.</a></p>

<p><a data-t="227">So it's designed to give you consistency both in how you specify and fit the model and how you predict and what you get back.</a> <a data-t="241">So you don't have to remember function names, argument names across different packages, whether or not you get a vector back or a matrix or a data frame, and what that type of prediction was called in this particular package you're using.</a> <a data-t="275">And the thing I'm really here to talk about is censored, which is a parsnip extension package.</a> <a data-t="283">So for survival models, so I'll be talking about how these design ideas play out and marry with survival analysis.</a></p>

<h3>Specifying and fitting models</h3>

<p><a data-t="289">Let's get to that first part, to specify and fit models.</a> <a data-t="299">In parsnip, you need three elements to specify your model.</a> <a data-t="305">So for one, the model type.</a> <a data-t="309">In the example here, it's a random forest, and you need a computational engine.</a> <a data-t="315">That's what you use to fit the model.</a> <a data-t="321">My example here, that's the Ranger package.</a> <a data-t="331">Often that's various R packages.</a> <a data-t="331">It can also be tools outside of R, like Stan, if you have a Bayesian model you want to fit.</a> <a data-t="345">And the third element to a parsnip model specification is the mode.</a> <a data-t="352">A random forest can be used for regression, for classification, so you need to specify that here.</a></p>

<p><a data-t="372">And these three elements are sort of very consistent in parsnip, so they will also show up in censored.</a> <a data-t="382">So what is actually new with censored?</a> <a data-t="388">Well, we have to make a few additions.</a> <a data-t="392">For one, we have a new model type, called proportional hazards, to let you fit everyone's favorite Cox model.</a> <a data-t="406">Second of all, we have a new mode to distinguish censored regression from regression.</a> <a data-t="413">And thirdly, you probably guessed it, several new engines to existing models.</a> <a data-t="420">So with that, censored covers parametric models, semi-parametric models, and tree-based models for survival analysis.</a> <a data-t="430">And the last point that I want to mention here is that we have a formula interface for all of these models.</a></p>

<p><a data-t="441">Most of the engines do that already, the glmnet package sort of famously does not, but we let you interact with that through a formula interface, and that includes a way to specify stratification.</a></p>

<h3>Predicting with censored models</h3>

<p><a data-t="610">And that's the section on specifying and fitting.</a> <a data-t="630">And the next one is obviously predicting.</a></p>

<p><a data-t="630">And in the tidymodels framework, we give you a little guarantee that your predictions will always be inside a table.</a> <a data-t="637">And the column names and types are unsurprising and predictable.</a> <a data-t="644">And the number of rows in new data and the output are the same.</a> <a data-t="650">So what's new then?</a></p>

<p><a data-t="654">Answer is new prediction types.</a> <a data-t="659">One of them for survival analysis is the survival time.</a> <a data-t="660">So I'm just taking the first three rows here of my data set to predict on them, adding in an NA in the first row just to mix things up a little, and you can get a prediction of the survival time by using the predict function and setting type to time.</a> <a data-t="676">And you get a table back, the column will always be dot pred underscore time, and it will have three rows because you put three rows in.</a> <a data-t="682">So if you have a missing value, it will have an NA rather than miraculously disappear on you.</a></p>

<p><a data-t="699">And the other big new type for survival predictions is the survival probability.</a> <a data-t="722">That is always calculated at a point in time.</a> <a data-t="728">So here in this example, I'm setting type survival, and I want the survival probability at age 10, 15, 20, and 40 years.</a> <a data-t="735">Dolphin or whale that I'm looking at.</a> <a data-t="743">And what you get back is still a table that has three rows for the three observations, but we've asked for four time points.</a> <a data-t="749">So this is actually a list column and it has little tables stashed inside of it.</a> <a data-t="756">If we want to look at one of them, we can pull it out and then we see the column that gives us the information about the time point that we're predicting on and the probability, except for in this case it's an NA, but if you look at the other one, you do get actual probabilities.</a></p>

<p><a data-t="756">And this does not return you like a classical surv object, surv fit object.</a> <a data-t="765">If you do want to access the full survival curve, you can pretty easily just dial up the number of time points you ask for.</a> <a data-t="771">So here I'm asking for survival probability predicted at age one all the way through 80, and then I can unnest that thing to sort of stack all the little tables underneath each other and I'm squeezing in the mutate to be still able to tell which rows belong to which dolphin.</a> <a data-t="805">And that gives you a data set that is ready for ggplot and you can easily do your visualization of the survival curve with a geom step.</a></p>
