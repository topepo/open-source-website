<h3>Reactivity and animation with OJS</h3>

<p><a data-t="459">So we've talked about some basic examples here, just getting a flavor for OJS.</a> <a data-t="464">Now I want to take us into some more adventurous stuff.</a> <a data-t="467">We're going to try out a few things, see a bit of what OJS can do.</a></p>

<p><a data-t="473">So here we have another observable plot.</a> <a data-t="475">This one's a map.</a> <a data-t="476">We've got some cities labeled around the world.</a> <a data-t="478">It's very similar to the previous observable plot graphic we did,</a> <a data-t="482">in the sense that you can see, again, we have our list of marks, our list of layers.</a> <a data-t="486">This time we have three.</a> <a data-t="487">We have a plot.geo, which is our coastline in dark gray.</a> <a data-t="492">Then we have a layer of dots.</a> <a data-t="495">That would be a geom point in ggplot.</a> <a data-t="498">And then we also have a text layer, which is for our labels.</a> <a data-t="502">And you can see for each one, we've mapped an X channel and a Y channel.</a> <a data-t="505">So our longitude is our X, and our latitude is our Y, just like we do with AES in ggplot2.</a></p>

<p><a data-t="512">Now this is a fine start, but I think we could make this graphic a bit more interesting.</a> <a data-t="519">So for starters, we could make it a globe.</a> <a data-t="521">And in an observable plot, that's not too hard to do at all.</a> <a data-t="524">We can add a projection here.</a> <a data-t="525">We've added an orthographic one to make it into a globe.</a> <a data-t="529">And it's centered on 50 longitude, which is going to be important in a second.</a> <a data-t="533">And then we've added a couple of extra layers on there for our grid lines and our sphere around it.</a> <a data-t="539">So it's not just land in space.</a> <a data-t="541">We can see the outline of the globe.</a></p>

<p><a data-t="545">But we've lost half our cities because half of them are on the other side of the world.</a> <a data-t="548">It would be really nice if we could move this around.</a></p>

<p><a data-t="552">So the really nice thing about observable JavaScript is that it's reactive.</a> <a data-t="557">Now, if you've ever used Shiny, for example, you'll know what reactivity is about.</a> <a data-t="562">If your plot includes something that is changing, your plot changes too.</a> <a data-t="566">And you don't need to tell it how to do that.</a> <a data-t="568">You don't need to say, you know, when now does this or when one thing does this, do all this logic.</a> <a data-t="575">You just refer to a thing that's changing and it changes as well.</a> <a data-t="579">So built into Quarto and OJS, we have the now object.</a> <a data-t="583">And that's just a ticking clock.</a> <a data-t="585">It's just number of milliseconds ticking upwards forever.</a></p>

<p><a data-t="588">But we can turn this into an angle with a bit of maths.</a> <a data-t="592">So we do a bit of dividing, a bit of modulus, some subtraction.</a> <a data-t="595">And now we've got a number between negative 180 and 180.</a> <a data-t="599">And you can see we don't need to do anything special here.</a> <a data-t="602">We just refer to now and angle is changing on its own.</a> <a data-t="606">So that's automatically updating.</a> <a data-t="608">And now we're going to take it a step further and we're going to include angle in our plot.</a> <a data-t="612">So where we had rotate 50 degrees longitude, now we have angle.</a> <a data-t="617">And so every time now changes, angle changes.</a> <a data-t="620">And every time angle changes, our plot gets redrawn just a little bit each time.</a> <a data-t="625">And when you do that finely enough, you get this animation,</a> <a data-t="628">this rotation effect as it moves around.</a> <a data-t="630">As angle slowly increases, our globe rotates just a little bit more.</a> <a data-t="635">And we get this really neat rotation effect.</a> <a data-t="637">And we haven't had to do any of this logic to tie these things together.</a> <a data-t="641">We just refer to them.</a></p>

<p><a data-t="644">We can do a lot with this technique.</a> <a data-t="646">And it doesn't just need to be animation in the sense of things changing over time.</a> <a data-t="650">My colleague Andrew Bray just presented CloseRead yesterday.</a> <a data-t="653">That's a Quarto extension we've been working on for scrollytelling in Quarto.</a> <a data-t="658">With CloseRead, you can use cr-progress instead of now.</a> <a data-t="662">And then you're animating on scroll.</a> <a data-t="664">So as you move this page down, that globe is twisting.</a></p>

<p><a data-t="671">But sometimes we might not want to animate something necessarily.</a> <a data-t="675">We might just want to conditionally update things or change them as our circumstances change.</a> <a data-t="681">Now, if you're using R or Python, you've probably seen an if-else expression.</a> <a data-t="685">We say we give it a condition.</a> <a data-t="687">And we say if the condition is true, give us this thing.</a> <a data-t="690">And if the condition is false, give us that thing.</a> <a data-t="692">And in OJS, we have that as well.</a> <a data-t="694">The condition comes first.</a> <a data-t="696">We've got a question mark, true thing, colon, false thing.</a> <a data-t="700">But the difference is in OJS, it's reactive.</a> <a data-t="703">So it doesn't just get evaluated once when the document opens.</a> <a data-t="707">It's evaluating continuously.</a> <a data-t="709">When the person opens the web page, whenever the condition changes, the thing changes as well.</a></p>

<p><a data-t="714">And we can do a lot with this.</a> <a data-t="716">So for example, we have a simple timer here at the top.</a> <a data-t="719">We don't need to worry about the code box.</a> <a data-t="721">We can just see that it's ticking between 0 and 1 every couple of seconds.</a> <a data-t="726">And if we refer to this, we can now start to make some conditional content.</a> <a data-t="729">So on the bottom left, we have some just text, markdown text.</a> <a data-t="733">And you can see that's changing as the traffic light timer does.</a> <a data-t="737">In the middle, we have graphics.</a> <a data-t="738">So we've got markdown graphics.</a> <a data-t="739">We'd normally put these in a Quarto document.</a> <a data-t="742">And we can tick between two to make like a slideshow or that kind of thing.</a> <a data-t="746">On the right, we have a really more complicated example where we're actually doing a drawing</a> <a data-t="750">of some traffic lights using basic shapes.</a> <a data-t="752">And you can actually see here that we had our condition embedded in here twice.</a> <a data-t="758">So our traffic light is ticking between, for the first one, black and orange.</a> <a data-t="762">And for the other one, green and black.</a></p>

<p><a data-t="766">So we can do a whole bunch of different things here.</a> <a data-t="768">We don't even have to use the same kind of content.</a> <a data-t="771">We can mix things.</a> <a data-t="772">So for example, we have our city population data set here.</a> <a data-t="776">And we've added it as a table.</a> <a data-t="778">A table you'd normally think of as being an output.</a> <a data-t="780">But in this case, it's also an input because we can select rows.</a> <a data-t="784">Observable, Quarto, and OJS also come with a library called Observable Inputs.</a> <a data-t="789">And this gives us a whole bunch of user interface elements that you've seen before,</a> <a data-t="792">like checkboxes, radio boxes, drop-down menus, and in this case, a table.</a> <a data-t="798">And what we're saying is that this first line up here, if we have less than three cities selected,</a> <a data-t="804">or I should say zero, one, or two cities selected,</a> <a data-t="807">we're going to do our city plot with our bar plot.</a> <a data-t="810">And if not, we're going to show an error message.</a> <a data-t="813">So you can see here, we've got an error message at the moment.</a> <a data-t="815">But if I select these, as soon as we hit three, it pops up.</a> <a data-t="819">And you can see it updating still as the data changes.</a> <a data-t="822">So this selected cities is a filtered view of our data set.</a> <a data-t="826">We've sent in our original data set.</a> <a data-t="828">And then selected cities is the filtered version based on our selection.</a></p>

<h3>Introducing Sverto for smooth transitions</h3>

<p><a data-t="866">And most of the time, this is kind of enough.</a> <a data-t="869">Honestly, most of the graphics I build with 360 Info,</a> <a data-t="872">we're either using an external charting tool,</a> <a data-t="875">or if I'm building something bespoke,</a> <a data-t="876">I'm using observable plot and observable inputs and OJS.</a> <a data-t="881">And this works really well because you can very effectively</a> <a data-t="884">and quickly iterate to find the form of your graphic</a> <a data-t="887">that's working for your audience.</a></p>

<p><a data-t="890">But there are a few times when it doesn't work so well.</a> <a data-t="892">And for me, that's often when I want to transition from A to B.</a> <a data-t="895">You know, we can kind of do simulated transitions,</a> <a data-t="900">like the globe example,</a> <a data-t="901">where we had something that's ticking up over time,</a> <a data-t="903">but it's all got to be manually calculated.</a> <a data-t="905">In OJS, when you replace one thing with another with reactivity,</a> <a data-t="909">the first thing gets destroyed immediately.</a> <a data-t="913">What I need is something that can see the start and the finish</a> <a data-t="916">and smoothly transition between the two.</a></p>

<p><a data-t="919">So OJS's reactivity is lovely, but I want to bypass it sometimes.</a> <a data-t="924">And I want more control over what I draw.</a> <a data-t="926">But I do like the declarative way I get to build graphics</a> <a data-t="929">into ggplot2 and observable plot.</a> <a data-t="931">And so the last part of our talk</a> <a data-t="933">is talking about a tool I've developed,</a> <a data-t="936">a Quarto extension that lets me use a mature web framework</a> <a data-t="940">called Svelte in Quarto.</a> <a data-t="942">It's called Sverto.</a></p>

<p><a data-t="945">The funny thing about reactivity</a> <a data-t="946">is it's actually really popular these days.</a> <a data-t="948">It's just about everywhere.</a> <a data-t="951">You've seen it in Shiny,</a> <a data-t="952">but it's also in mobile app development,</a> <a data-t="953">most Apple and Google's main frameworks</a> <a data-t="957">are reactive these days.</a> <a data-t="959">And in web development, we have React, Svelte,</a> <a data-t="961">and countless others.</a> <a data-t="962">And the thing they all have in common</a> <a data-t="964">is that you make a component,</a> <a data-t="966">you give it data or options that might change,</a> <a data-t="968">and you tell it what you want to show.</a> <a data-t="970">And then it takes care of doing the updates</a> <a data-t="973">when things change.</a></p>

<p><a data-t="976">If this sounds really familiar,</a> <a data-t="978">I think reactivity is very philosophically similar</a> <a data-t="981">to declarative graphics</a> <a data-t="982">because they're both about deciding</a> <a data-t="984">that we don't want to get bogged down in the details</a> <a data-t="986">of how we want to make changes to things</a> <a data-t="989">and updates to things.</a> <a data-t="990">We just tell it what we want to focus on</a> <a data-t="993">and it takes care of the rest.</a></p>

<blockquote><a data-t="976">I think reactivity is very philosophically similar</a> <a data-t="981">to declarative graphics</a> <a data-t="982">because they're both about deciding</a> <a data-t="984">that we don't want to get bogged down in the details</a> <a data-t="986">of how we want to make changes to things</a> <a data-t="989">and updates to things.</a> <a data-t="990">We just tell it what we want to focus on</a> <a data-t="993">and it takes care of the rest.</a></blockquote>

<p><a data-t="997">So here's Sverto.</a> <a data-t="999">This is bringing Svelte components,</a> <a data-t="1001">like charts, into Quarto.</a> <a data-t="1003">And those can be Svelte components</a> <a data-t="1004">that someone else has made, part of the library,</a> <a data-t="1006">or they can be ones you've written yourself.</a> <a data-t="1009">In this case, the Svelte component</a> <a data-t="1010">is not the whole page, it's just the chart.</a> <a data-t="1013">This is one of the really nice things about Sverto</a> <a data-t="1015">is these dropdown menus we have here on the left</a> <a data-t="1018">are actually just OJS.</a> <a data-t="1020">They're just the same observable inputs</a> <a data-t="1022">that I used previously in the presentation.</a></p>

<p><a data-t="1025">But when I change them,</a> <a data-t="1026">they are updating a dataset</a> <a data-t="1029">and then that dataset is going into the chart.</a> <a data-t="1031">And when the chart gets a change,</a> <a data-t="1033">like a changed data,</a> <a data-t="1034">it smoothly updates the things that it's drawing.</a> <a data-t="1037">And it does so knowing where things were before</a> <a data-t="1040">and where they will be next.</a> <a data-t="1041">So you get these smooth transitions.</a> <a data-t="1044">So that could be like the colors</a> <a data-t="1045">as we change from day to night,</a> <a data-t="1047">as we change from Melbourne to Brisbane,</a> <a data-t="1048">looking at historical weather,</a> <a data-t="1052">you can see that we're adding and removing points</a> <a data-t="1055">as we need to, fading them in and out,</a> <a data-t="1058">and we're moving them as well.</a> <a data-t="1059">And you can see when we go from hottest to coldest,</a> <a data-t="1061">we've also got some axes that we're using with D3.js.</a> <a data-t="1065">So yeah, we've got a bit of everything here.</a></p>
