<h3>RStudio Cloud features for teaching</h3>

<p><a data-t="466">Let's talk a little bit more about why RStudio Cloud.</a> <a data-t="468">Beyond the fact that it does not require</a> <a data-t="471">IT experience to set up,</a> <a data-t="471">it also has some really nice bells and whistles</a> <a data-t="473">that are designed particularly for teaching.</a> <a data-t="476">For example, there is the notion of a workspace</a> <a data-t="479">where you can organize your class as a workspace.</a> <a data-t="482">So either if you're teaching multiple classes</a> <a data-t="484">in the same semester,</a> <a data-t="485">or from semester to semester,</a> <a data-t="487">you're able to kind of have them organized</a> <a data-t="490">in a nicely contained way.</a></p>

<p><a data-t="493">There is a notion of roles.</a> <a data-t="494">So things like instructor, teaching assistant or student</a> <a data-t="498">can be assigned to particular roles on RStudio Cloud.</a> <a data-t="502">You can turn your project.</a> <a data-t="503">So I just showed you earlier</a> <a data-t="505">how to fire up a project in RStudio Cloud.</a> <a data-t="508">You can turn that into an assignment</a> <a data-t="509">that your students can quickly get started with.</a> <a data-t="513">And as an instructor, something I really, really like</a> <a data-t="516">is that you can peek into student projects.</a> <a data-t="519">So that is really very similar to being able to,</a> <a data-t="523">you know, be say in a computer lab with your students.</a> <a data-t="527">And if they have a question</a> <a data-t="529">kind of peeking over their shoulder,</a> <a data-t="531">and being able to do this has been particularly helpful</a> <a data-t="535">when we've had to do a bunch of remote teaching.</a></p>

<p><a data-t="538">The other thing is that you can set up a base project</a> <a data-t="541">that contains any R packages you might need,</a> <a data-t="545">or any other documentation</a> <a data-t="547">that you would like for your students to have</a> <a data-t="571">for every single one of their projects.</a> <a data-t="573">You can set that up for them</a> <a data-t="574">and modify it throughout the semester,</a> <a data-t="577">and ensure that each time a student starts a new project,</a> <a data-t="579">they're all starting with the same base project.</a></p>

<p><a data-t="583">Certain system libraries work out of the box.</a> <a data-t="586">For example, Git will work out of the box.</a> <a data-t="590">And also you can render to PDF or Word,</a> <a data-t="593">and that works out of the box as well.</a> <a data-t="595">These are generally things for local use.</a> <a data-t="598">The user would have to set up themselves,</a> <a data-t="601">or if you are teaching in a situation</a> <a data-t="604">where your students are using their local setup,</a> <a data-t="609">they would have to install for themselves.</a> <a data-t="611">And usually the installation and setup of these</a> <a data-t="614">tends to require more skills</a> <a data-t="616">than just being able to do some computing with R.</a></p>

<h3>Sharing approaches for teaching</h3>

<p><a data-t="729">So teaching with RStudio Cloud</a> <a data-t="732">beyond just using it for yourself</a> <a data-t="735">really does not require any further kind of setup than this,</a> <a data-t="738">except the fact that you probably will want</a> <a data-t="741">to prepare some projects for your students,</a> <a data-t="743">and you might be thinking about various ways</a> <a data-t="745">that you can share it with them.</a> <a data-t="747">So there are really two ways that I think about teaching</a> <a data-t="749">when I'm kind of thinking about</a> <a data-t="751">what sort of sharing approach</a> <a data-t="753">I'm going to use for my students.</a></p>

<p><a data-t="755">Often I ask myself the question,</a> <a data-t="758">is this a shorter engagement,</a> <a data-t="759">something like a workshop or a short course,</a> <a data-t="762">where I can organize content in a single RStudio project</a> <a data-t="767">or just a few projects,</a> <a data-t="769">and I have no need for keeping track of my learners.</a> <a data-t="772">So that's usually for me a setting</a> <a data-t="774">where maybe I won't be grading them,</a> <a data-t="777">or I'm not necessarily responsible</a> <a data-t="779">for keeping track of their work</a> <a data-t="781">over an extended period of time.</a> <a data-t="784">If that's the case,</a> <a data-t="785">you can simply create a project like we've shown before</a> <a data-t="788">and share that project simply by sharing its URL.</a></p>

<p><a data-t="791">If you have a longer engagement,</a> <a data-t="794">so that is perhaps a semester long course</a> <a data-t="797">or a multi-day workshop where there's lots of content.</a> <a data-t="800">And instead of kind of putting all of that</a> <a data-t="802">into a single or just a few RStudio projects,</a> <a data-t="806">you can think about it as many, many projects</a> <a data-t="809">that need to happen throughout a semester.</a> <a data-t="811">So maybe many homework assignments,</a> <a data-t="813">a final project, some lab work, so on and so forth.</a> <a data-t="817">And you have a need or desire</a> <a data-t="819">to keep track of your learners,</a> <a data-t="821">either for grading purposes,</a> <a data-t="823">or because you want to assess</a> <a data-t="825">either for your kind of enjoyment</a> <a data-t="828">or to be able to give feedback to the students,</a> <a data-t="831">how much they're interacting with both the platform</a> <a data-t="835">and also the learning materials</a> <a data-t="836">that you've prepared for them.</a> <a data-t="838">In that case, I recommend creating a workspace</a> <a data-t="841">and then inviting your students to that workspace.</a></p>

<p><a data-t="1374">Another thing is,</a> <a data-t="1376">so how do we get students into this workspace?</a> <a data-t="1380">Under the members tab within your workspace,</a> <a data-t="1384">you can see the names of your members.</a> <a data-t="1386">So I have grayed those out for privacy reasons,</a> <a data-t="1389">but you can see that I can scroll through</a> <a data-t="1392">and see who the members are of this workspace.</a> <a data-t="1395">And I can invite more folks either via an invitation.</a> <a data-t="1400">So I would need to enter their email address</a> <a data-t="1402">and send it to them, or I can use a sharing link.</a> <a data-t="1405">So I can create a sharing link,</a> <a data-t="1407">set a particular default role for that sharing link</a> <a data-t="1411">and simply share the link with them.</a> <a data-t="1414">And at some point I can reset that link if I want,</a> <a data-t="1416">if I want no further people to continue to join the workspace.</a></p>

<h3>Base projects</h3>

<p><a data-t="1549">Another perk of using this workspace setting</a> <a data-t="1552">is perhaps my favorite one</a> <a data-t="1555">is this notion of a base RStudio project.</a> <a data-t="1559">So what that is is a project that you can set.</a> <a data-t="1563">So it's an RStudio project that you can set to say,</a> <a data-t="1566">this is the image I want every single project to start with.</a> <a data-t="1568">So I can actually put, decide on the version of R</a> <a data-t="1572">I want everyone to use,</a> <a data-t="1574">the installed packages and their versions</a> <a data-t="1576">that I want everyone to use,</a> <a data-t="1578">and also any files that I want every single project</a> <a data-t="1582">in my workspace to contain.</a> <a data-t="1584">Usually for me, those are things like</a> <a data-t="1586">a statement of code of conduct,</a> <a data-t="1588">maybe some instructions for how to submit work.</a> <a data-t="1592">They're not things that change</a> <a data-t="1593">from assignment to assignment.</a> <a data-t="1595">They're things that need to stay constant.</a> <a data-t="1597">They're the sort of things you might tell your students,</a> <a data-t="1600">go read in the syllabus.</a> <a data-t="1601">So I ended up putting those there.</a></p>

<p><a data-t="1605">These base projects can be updated throughout the semester,</a> <a data-t="1609">which means that say a newer version</a> <a data-t="1611">of a package you use comes</a> <a data-t="1613">and you want to be able to teach</a> <a data-t="1614">with the newer functionality that's been implemented.</a> <a data-t="1617">Mid semester, you can update the base project</a> <a data-t="1621">so that going forward,</a> <a data-t="1622">you're using a newer version of that package</a> <a data-t="1624">and every project that's starting from that point onwards</a> <a data-t="1629">is affected by this update.</a> <a data-t="1631">But it's not retroactive and that's good</a> <a data-t="1633">because you're not gonna break old work</a> <a data-t="1635">students may have done.</a></p>

<p><a data-t="1637">I think this is also a really nice perk</a> <a data-t="1641">because when you are using kind of maybe an RStudio</a> <a data-t="1645">in the server type setup that may be shared</a> <a data-t="1649">throughout the university or throughout your department</a> <a data-t="1652">with maybe other instructors,</a> <a data-t="1653">oftentimes IT professionals would be hesitant</a> <a data-t="1656">to update this sort of base image</a> <a data-t="1659">because changes that you make could affect others as well.</a> <a data-t="1662">But by the definition of a workspace,</a> <a data-t="1666">it's only your class</a> <a data-t="1667">and your class is the only set of projects</a> <a data-t="1672">who are going to be affected by this change.</a> <a data-t="1674">So you really can do as you please.</a></p>

<h3>Tips for using RStudio Cloud</h3>

<p><a data-t="1811">So the first tip that I would mention</a> <a data-t="1815">is that changes to an assignment that you make,</a> <a data-t="1819">that after a student has started,</a> <a data-t="1821">won't propagate to their copy.</a> <a data-t="1823">So this is not like Google Docs in a way,</a> <a data-t="1826">is I think the best way to describe it.</a> <a data-t="1828">When your student makes a copy of a project</a> <a data-t="1830">you have set up for them</a> <a data-t="1832">in either of the sharing approaches,</a> <a data-t="1834">they're really taking a snapshot at that time.</a> <a data-t="1837">So if you need to change your assignment,</a> <a data-t="1839">and I'm sure we've all been there,</a> <a data-t="1841">you assign a homework assignment,</a> <a data-t="1842">then you realize something is wrong.</a> <a data-t="1844">You're gonna need to get that information</a> <a data-t="1845">to them in another way,</a> <a data-t="1847">or have them start with a new project,</a> <a data-t="1849">or have your projects be linked to a Git repository as well.</a></p>

<p><a data-t="1862">Packages in the base project are installed, but not loaded.</a> <a data-t="1865">So when you're writing your instructions,</a> <a data-t="1867">you want to remind yourself,</a> <a data-t="1871">remind your students to do library, whatever package name,</a> <a data-t="1871">because they won't be able to use it immediately</a> <a data-t="1873">in the console.</a></p>

<p><a data-t="1897">This information is always on the RStudio Cloud guide</a> <a data-t="1901">and the What's New page.</a> <a data-t="1902">RStudio Cloud is still evolving.</a> <a data-t="1905">It is not changing in terms of,</a> <a data-t="1910">it's not changing in terms of like,</a> <a data-t="1911">in a way where it's going to be a completely different UI</a> <a data-t="1915">the next day your students land in it,</a> <a data-t="1917">but there are new features being added.</a> <a data-t="1920">And so it's nice to keep up with those.</a> <a data-t="1923">And if you encounter any sort of slowness or glitches,</a> <a data-t="1926">say your students are reporting, we can't log on,</a> <a data-t="1929">there's a current system status link on the side</a> <a data-t="1932">that's super handy.</a> <a data-t="1933">It allows you to check to see if there are real outage,</a> <a data-t="1936">and if not, you can go ahead and report it.</a></p>

<p><a data-t="1939">Other tips is I strongly recommend</a> <a data-t="1943">creating an additional account.</a> <a data-t="1945">So I have this like additional account</a> <a data-t="1947">with another personal email address I had,</a> <a data-t="1950">and I invite that persona as a student,</a> <a data-t="1953">so as a contributor into my workspace,</a> <a data-t="1956">so that I can log in and see things the students see them.</a> <a data-t="1960">That can be helpful when you are kind of taking screenshots</a> <a data-t="1965">for your students as well,</a> <a data-t="1966">if that's the sort of thing you like to put</a> <a data-t="1968">in your instructions.</a></p>

<p><a data-t="1971">You can set computational resources allotted to projects.</a> <a data-t="1975">By default, you get a decent amount,</a> <a data-t="1978">but if you're working with either bigger data sets,</a> <a data-t="1982">or doing something that's more computationally intensive,</a> <a data-t="1993">you may wanna up those computational resources</a> <a data-t="1997">that are available to each project.</a> <a data-t="1999">And the one way to figure out if this is going to be needed</a> <a data-t="2004">is to actually test your assignment.</a> <a data-t="2006">So if you write answer keys, for example,</a> <a data-t="2008">you may try running them in RStudio Cloud</a> <a data-t="2010">to make sure things hold up.</a></p>

<p><a data-t="2074">I would, the peeking at your students' functionality</a> <a data-t="2075">is super nice, but from a pedagogical perspective,</a> <a data-t="2079">I would recommend that you use that feature sparingly.</a> <a data-t="2083">It's important for us to teach our students</a> <a data-t="2086">how to ask good questions without us saying,</a> <a data-t="2089">hey, just move away from the keyboard, I'll do it for you.</a> <a data-t="2092">Because that's a little bit of that.</a> <a data-t="2094">Move away from the computer, let me take over and do it.</a> <a data-t="2096">And sometimes that's exactly what a student needs.</a> <a data-t="2100">But it's useful for them to try</a> <a data-t="2102">to articulate the question first,</a> <a data-t="2104">and not go there right away.</a></p>
