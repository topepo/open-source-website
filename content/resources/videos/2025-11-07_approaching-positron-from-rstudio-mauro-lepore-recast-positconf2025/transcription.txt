<p><a data-t="0">Today I would like to talk about the transition from RStudio to </a><a href="/software/positron/">Positron</a><a data-t="0"> and the ways in which</a> <a data-t="11">you can make it less painful and more productive.</a> <a data-t="18">This talk is particularly for you if you are</a> <a data-t="18">curious about Positron or maybe you are already using it, but you still feel that </a><a href="/software/rstudio/">RStudio</a> <a data-t="23">is the place where you feel most comfortable.</a></p>

<p><a data-t="29">The situation that led to this talk is that</a> <a data-t="29">I've been using RStudio for a long time, pretty much exclusively, and I love it.</a> <a data-t="36">But</a> <a data-t="43">I've also heard great things about Positron, so I got curious, I wanted to try.</a> <a data-t="43">One of</a> <a data-t="43">the great things I've heard is that the design of Positron has been greatly inspired by RStudio,</a> <a data-t="49">so definitely something that I wanted to try.</a> <a data-t="55">And those similarities between the two ideas</a> <a data-t="55">made me think that the transition could be super cool, cruisy, maybe similar to having</a> <a data-t="61">driven a good old car for many years and then switching to a different car, but not so different</a> <a data-t="66">so that the challenge couldn't be great.</a></p>

<p><a data-t="74">Actually, what happened was a little bit more like a</a> <a data-t="74">switch between an automatic car to a manual stick for the first time.</a> <a data-t="79">A lot of the things</a> <a data-t="79">that I used to do without thinking in RStudio suddenly started taking up a lot of effort.</a> <a data-t="85">So certainly some frictions and pains.</a> <a data-t="91">So it wasn't reflecting about those pains that</a> <a data-t="91">I thought, well, the approach that I'm taking here is an approach that I've done many, many</a> <a data-t="95">times before when I want to learn a new technology, I just dive in, push through the pain, and</a> <a data-t="101">eventually you get used to it.</a> <a data-t="106">But sometimes you cannot afford that.</a> <a data-t="106">What if you are delivering</a> <a data-t="106">a product next week or the week after and you just cannot afford slowing down?</a></p>

<p><a data-t="112">So here</a> <a data-t="112">I would like to propose the idea of taking a different approach where we start the transition</a> <a data-t="119">in RStudio itself by tweaking our workflow a little bit in a few specific ways that will</a> <a data-t="125">help you get used to what's coming next.</a></p>

<h3>Three workflow tweaks</h3>

<p><a data-t="132">So here are my three tweaks, my three workflow</a> <a data-t="132">tweaks that are working well for me, and maybe some of that works for you, too.</a> <a data-t="138">First,</a> <a data-t="138">use the commands palette.</a> <a data-t="138">What is this?</a> <a data-t="138">It's this text-based interface to running commands</a> <a data-t="146">in RStudio.</a> <a data-t="152">And this thing also exists in Positron and is the main way in which you</a> <a data-t="152">are supposed to use the IDE.</a> <a data-t="158">Why would you use this?</a> <a data-t="158">Well, because that way you get used</a> <a data-t="158">to running commands in a way that is not RStudio-specific.</a> <a data-t="165">So instead of clicking on things, you just</a> <a data-t="165">type a few keywords or a few keystrokes and you filter through the commands that are available</a> <a data-t="170">to you and then hit enter and run them.</a> <a data-t="175">And along the way, you will also discover the</a> <a data-t="175">names of the commands that will be likely very similar in Positron 2.</a></p>

<p><a data-t="183">Tweak number two,</a> <a data-t="183">use the terminal more.</a> <a data-t="190">What I mean with this is basically talk to your computer in the</a> <a data-t="190">most direct and efficient way possible.</a> <a data-t="197">The terminal is both in Positron and RStudio,</a> <a data-t="197">but it's also everywhere else.</a> <a data-t="202">Even if you are using a remote server without a graphical</a> <a data-t="202">user interface, you will still have a terminal.</a> <a data-t="209">So you can still do a lot of stuff, stay productive,</a> <a data-t="209">and not depend on RStudio or any IDE.</a></p>

<p><a data-t="236">And tweak number three, of course, use AI.</a> <a data-t="240">Particularly,</a> <a data-t="240">the AI tools that are agentic and they run from a </a><a href="/software/cli/">CLI</a><a data-t="240">, from a command line interface,</a> <a data-t="246">will allow you to get stuff done without an IDE, so you spend less time in the pain zone.</a> <a data-t="253">In particular, I like Cloud Code and Gemini CLI precisely because of that.</a> <a data-t="259">Wherever there</a> <a data-t="259">is a terminal, there is this tool that will modify the source code on my behalf and not</a> <a data-t="265">by being myself the one using the programming language, but indirectly through natural human</a> <a data-t="271">language.</a> <a data-t="278">So on top of being independent of an IDE, I'm gradually independent from programming</a> <a data-t="278">language itself.</a></p>

<p><a data-t="285">And using AI effectively is in itself a huge challenge and something</a> <a data-t="285">that certainly I'm still learning.</a> <a data-t="290">But I'm now in the feeling that it's something worth</a> <a data-t="290">trying because if you do that well, then you don't need to sit on the driver's seat.</a> <a data-t="296">And</a> <a data-t="296">instead, it feels like having a driver that if you give them the right directions, will</a> <a data-t="300">take you wherever you want to go.</a> <a data-t="307">And that I think is super cool.</a> <a data-t="307">Thank you.</a></p>
