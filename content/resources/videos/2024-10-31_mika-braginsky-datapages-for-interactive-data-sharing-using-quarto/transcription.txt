<h3>Components of a data page</h3>

<p><a data-t="110">So what are the components of a</a> <a data-t="110">data page?</a> <a data-t="115">First off, you use the Redivis data platform to store your data set,</a> <a data-t="115">which for free immediately means that your data is versioned.</a> <a data-t="120">And then you use</a> <a data-t="120">our custom data pages </a><a href="/software/quarto/">Quarto</a><a data-t="120"> format to connect to this data to a Quarto website</a> <a data-t="125">that you can configure and customize.</a> <a data-t="130">At the center of this website are</a> <a data-t="130">interactive visualizations created using Observable.</a> <a data-t="134">So also you can add ggplot</a> <a data-t="134">or whatever other visualizations you want.</a> <a data-t="139">And all this results in a static</a> <a data-t="139">website so it can be hosted on GitHub pages, but it has rich interactive</a> <a data-t="143">functionality.</a></p>

<h3>Getting started</h3>

<p><a data-t="147">I'm going to walk you through more concretely the steps you</a> <a data-t="147">would follow to spin up a new data page.</a> <a data-t="152">To get started, all you would need to do</a> <a data-t="152">is use the data page GitHub template repo to create your own repo.</a> <a data-t="157">And this</a> <a data-t="157">gives you a template Quarto website that uses our custom data pages Quarto</a> <a data-t="162">project type and format.</a> <a data-t="167">And you then customize this website by adding</a> <a data-t="167">information and formatting specific to your project and connecting it to the</a> <a data-t="171">Redivis table with your data.</a></p>

<h3>What you get</h3>

<p><a data-t="176">What do you get from doing all this?</a> <a data-t="176">You get a</a> <a data-t="176">website.</a> <a data-t="181">The home page of your data page introduces your data set and features</a> <a data-t="181">these automatically created interactive visualizations.</a> <a data-t="185">So this means that users</a> <a data-t="185">can get a feel for your data right away.</a> <a data-t="190">And you can customize this through</a> <a data-t="190">configuration, through like YAML variables, or by adding like any custom</a> <a data-t="195">content you want, or by swapping out these templated visualizations.</a> <a data-t="201">And a</a> <a data-t="201">major area of focus for us is adding more configurability and more types of</a> <a data-t="205">visualizations for you.</a></p>

<p><a data-t="211">In addition to the home page, other pages include the</a> <a data-t="211">data page where there's an embedded table browser inserted through a custom</a> <a data-t="215">shortcode.</a> <a data-t="219">So users who might want to actually look at your data in more</a> <a data-t="219">detail can use this browser to preview your data and your metadata.</a> <a data-t="226">Even more</a> <a data-t="226">involved users can go to the analysis page to see an automatically generated</a> <a data-t="230">code snippet that tells them how to access the data directly from R or from</a> <a data-t="234">Python.</a> <a data-t="240">And then finally on the about page you can edit all this YAML to add</a> <a data-t="240">links, to add citation information, and then you can add whatever content you</a> <a data-t="244">want.</a> <a data-t="249">And of course since this whole thing is a Quarto website you can add</a> <a data-t="249">other pages, you can customize the styling, and use all the amazing features</a> <a data-t="252">that Quarto now has and will have in the future.</a></p>
