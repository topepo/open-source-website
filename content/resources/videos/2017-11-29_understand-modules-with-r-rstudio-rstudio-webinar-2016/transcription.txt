<p><a data-t="0">Welcome everybody, and thanks for attending today.</a> <a data-t="0">We're going to talk about some tools that you could use to write larger </a><a href="/software/shiny-r/">Shiny</a><a data-t="0"> apps</a> <a data-t="6">that are easy to think about, or to reuse Shiny code across apps.</a> <a data-t="6">And these tools are called modules.</a></p>

<p><a data-t="12">So let's start with the question, you know, what is a Shiny module?</a></p>

<p><a data-t="17">And conceptually, it's pretty easy to answer.</a> <a data-t="17">A Shiny module is a self-contained composable component of a Shiny app.</a> <a data-t="23">Self-contained means it's sort of like an R function.</a> <a data-t="23">It does something, and it does it all by itself.</a> <a data-t="29">It might take some input, it might return some output, but in the middle of the process, the app,</a> <a data-t="36">the module, doesn't need help from anything else to do its job.</a> <a data-t="42">And because of this, Shiny modules</a> <a data-t="42">are composable.</a> <a data-t="42">You could take multiple modules and put them together to make a Shiny app.</a></p>

<p><a data-t="46">And probably the easiest way to understand really what a Shiny app does, and what it looks like,</a> <a data-t="52">is to look at an actual Shiny app that uses modules.</a> <a data-t="52">So let's go to </a><a href="/software/rstudio/">RStudio</a><a data-t="52">.</a></p>

<h3>Rewriting with modules</h3>

<p><a data-t="171">Now, I've taken the time to rewrite this app using Shiny modules.</a> <a data-t="171">And when you use modules,</a> <a data-t="177">the app looks like this.</a> <a data-t="177">It's much more concise.</a> <a data-t="177">It's much smaller.</a> <a data-t="177">And what I've done is I've</a> <a data-t="182">what I've done is I've written an R script that takes the common pieces of code for each of the</a> <a data-t="187">tabs in our app and saves them to functions.</a> <a data-t="187">So here I'm creating a function called gap module UI,</a> <a data-t="193">another function called gap module.</a> <a data-t="193">Gap module does all the server side stuff to make that</a> <a data-t="197">graph connect to the slider.</a> <a data-t="197">And then in my modularized app, I load that R script.</a> <a data-t="197">And</a> <a data-t="203">instead of calling, you know, there's where I want a slider.</a> <a data-t="203">Here's where I want a graph.</a> <a data-t="203">Here's how</a> <a data-t="208">I want them to connect.</a> <a data-t="208">I just call the functions that do that for me.</a> <a data-t="208">So I'm calling gap module</a> <a data-t="212">six times once for each view of the data.</a> <a data-t="212">And then I'm calling, you know, the server side of</a> <a data-t="217">gap module six times once for each view of the data.</a> <a data-t="217">This app is much easier to understand.</a> <a data-t="223">It's</a> <a data-t="223">much easier to look at.</a> <a data-t="223">And if I want to get into any of these individual pieces, then I could look</a> <a data-t="226">at the definition of gap module UI or gap module over here.</a></p>

<p><a data-t="233">So this is the difference between a</a> <a data-t="233">modularized app and an app that's not modularized.</a> <a data-t="239">Once we took the time to write our modules,</a> <a data-t="239">we can recall them as many times as we want.</a> <a data-t="243">So I've called gap module six times here,</a> <a data-t="243">but I can also call it in a different app.</a> <a data-t="246">So this is a great way to write some code that</a> <a data-t="246">you can reuse later on.</a> <a data-t="246">You find, for example, you're always creating some sort of upload API</a> <a data-t="252">in your apps.</a> <a data-t="252">You can create that API once and then call that module across all your apps and</a> <a data-t="257">reuse it.</a></p>

<h3>Why use modules</h3>

<p><a data-t="265">So that's what modules are like conceptually.</a> <a data-t="265">Here's our module in that last</a> <a data-t="273">app.</a> <a data-t="273">Why would you use these modules?</a> <a data-t="273">Well, you could see that it's very easy to reuse this code.</a> <a data-t="273">Modules are sort of like functions.</a> <a data-t="277">Once you take the time to write them, you could call them</a> <a data-t="277">whenever you like to redo whatever that function does.</a> <a data-t="282">In this case, what it does is create</a> <a data-t="282">components of a Shiny app.</a> <a data-t="287">The next thing about modules are that they really isolate the</a> <a data-t="287">functionality in your apps.</a> <a data-t="292">Once you write your module function, all you have to think about</a> <a data-t="293">is what that function does when you see it in your app.</a> <a data-t="297">You don't have to think about</a> <a data-t="297">how it does what it does.</a></p>
