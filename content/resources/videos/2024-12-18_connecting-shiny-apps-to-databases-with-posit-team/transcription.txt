<h3>Why connect a Shiny app to a database?</h3>

<p><a data-t="49">So before we take a look at the app, let's first talk about why you would even want to</a> <a data-t="53">connect your Shiny app to a database.</a> <a data-t="57">One reason is that your organization might just store your data in a database, and you</a> <a data-t="62">want to make a Shiny app that interacts with, displays, does something with that data.</a> <a data-t="67">A second consideration is speed.</a> <a data-t="70">Even if your data originates elsewhere, you can use DuckDB to speed up operations, which</a> <a data-t="74">might be important if your app allows the user to do a lot of computations on the data.</a> <a data-t="80">A third reason is if your app's functionality relies on a shared data source.</a> <a data-t="85">As you'll see, the app we'll build today allows users to read and write to a central</a> <a data-t="89">data source.</a></p>

<h3>Deploying to Posit Connect</h3>

<p><a data-t="1157">So now let's deploy this app.</a> <a data-t="1159">One way is to click this blue Publish button, and we can publish it to Connect.</a> <a data-t="1190">Okay, and here is our published app, ready to share with others.</a></p>

<p><a data-t="1198">One thing to keep in mind is that when deploying an app that interacts with the database, you'll probably have to think about managing access.</a> <a data-t="1207">There's a couple strategies to handling database permissions.</a> <a data-t="1212">So the first is that if you are deploying to Connect, you can use the Vars pane</a> <a data-t="1218">to securely set environment variables after deploying your app.</a> <a data-t="1222">And Posit Connect will automatically provide the correct values to your app during runtime.</a></p>

<p><a data-t="1275">And now when we deploy the app, it'll get that value from Connect and be able to manage access that way.</a></p>

<p><a data-t="1282">So as you just saw, one way to deploy your app and manage access involves using environment variables in Posit Connect.</a> <a data-t="1289">This strategy relies on adding one set of credentials to Posit Connect, though.</a> <a data-t="1293">What if you want each user to authenticate with their own credentials?</a> <a data-t="1298">This could be particularly important if your database includes row-level or data-level permissions.</a> <a data-t="1304">One solution is to use OAuth integration, which is now available in Posit Connect for Databricks and Snowflake,</a> <a data-t="1311">allowing users to log in with their own accounts.</a> <a data-t="1314">With OAuth integration, you don't need to specify the database credentials used in your app.</a> <a data-t="1318">Instead, users can authenticate through the OAuth integration, and then Posit Connect will automatically handle the connection.</a> <a data-t="1325">For other database management systems, you can request user credentials through your app and then pass those credentials to the connection code.</a></p>
