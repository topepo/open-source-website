<p><a data-t="0">Our </a><a href="/software/rstudio/">RStudio</a><a data-t="0"> Cloud currently supports three project types, RStudio Projects, Projects</a> <a data-t="8">from a Git Repository, and our newest feature, Jupyter Projects.</a> <a data-t="12">This means that as instructors, we have a good deal of flexibility over the coding environment</a> <a data-t="17">we create for our learners, and that we can teach using both R and Python within the same</a> <a data-t="22">cloud-based environment.</a></p>

<p><a data-t="26">We can access each of these project types by clicking on the arrow in the right-hand</a> <a data-t="30">side of the New Projects button.</a> <a data-t="33">When we select RStudio Project, we're taken to a new, untitled project within the RStudio</a> <a data-t="38">IDE.</a></p>

<p><a data-t="49">When we select Jupyter Project, we're taken to a new, untitled project that gives us all</a> <a data-t="54">of the same functionality we expect from within a Jupyter Project, including the ability</a> <a data-t="59">to upload files, as well as create new Python notebooks using specific versions of Python.</a></p>

<h3>Cloning from a Git repository</h3>

<p><a data-t="77">We can also clone a new RStudio or Jupyter Project from a Git Repository.</a> <a data-t="82">For RStudio Projects, we simply provide the URL of the corresponding Git Repository, while</a> <a data-t="87">for Jupyter, we can accomplish this by cloning our repository from a new terminal within</a> <a data-t="92">a Jupyter Project on RStudio Cloud.</a></p>

<p><a data-t="97">RStudio support for Jupyter Projects is exciting, although at the time of this recording, the</a> <a data-t="101">feature is still in beta.</a> <a data-t="103">We would love to hear your feedback, and you can always reach out to us on the community</a> <a data-t="106">forums at community.rstudio.com.</a></p>
