<h3>Philosophy of the UI editor</h3>

<p><a data-t="999">Okay, so quickly, I just want to talk about the philosophy of the UI editor.</a> <a data-t="1004">So one thing that you might be asking at this point is,</a> <a data-t="1007">how do we avoid some of the common pitfalls of no-code solutions?</a> <a data-t="1012">Because the UI editor is, in a way, a no-code solution.</a> <a data-t="1015">And this is something that we were very careful about when we started.</a></p>

<p><a data-t="1020">And I think the simplest answer to that is that it's not a no-code editor.</a> <a data-t="1025">Fundamentally, we're writing code.</a> <a data-t="1027">The UI editor is a visual interface,</a> <a data-t="1030">but everything goes immediately back to code and vice versa.</a> <a data-t="1033">And so really, I don't think of it as no-code.</a> <a data-t="1036">I really think of it as a fancy auto-complete.</a> <a data-t="1040">You know, you might be able to tab-complete something,</a> <a data-t="1042">and it does what you want.</a> <a data-t="1043">This is essentially doing that for your UI.</a></p>

<blockquote><a data-t="1036">I really think of it as a fancy auto-complete.</a> <a data-t="1040">You know, you might be able to tab-complete something,</a> <a data-t="1042">and it does what you want.</a> <a data-t="1043">This is essentially doing that for your UI.</a></blockquote>

<p><a data-t="1047">And so we also really tried to keep it as simple as we possibly could,</a> <a data-t="1052">because one of the things that happens with no-code editors</a> <a data-t="1056">is that code is an extremely wonderful medium for expressing yourself.</a> <a data-t="1061">RStudio is really a proponent of the power of code.</a> <a data-t="1065">And so the problem with that extreme flexibility</a> <a data-t="1068">is that there's a lot of solution space that you need to make available</a> <a data-t="1073">if you really want to fully explore that.</a> <a data-t="1076">And so visually mapping to that space can be really intense.</a> <a data-t="1079">So we really carefully pulled everything back</a> <a data-t="1081">to make sure that we just let you edit your UI code.</a> <a data-t="1084">You can move things. You can set some settings.</a> <a data-t="1087">We're really not trying to go crazy and let you do everything,</a> <a data-t="1090">because we found that in a lot of tools that do that,</a> <a data-t="1093">you actually might as well just learn how to code</a> <a data-t="1095">instead of some, you know, esoteric visual editor,</a> <a data-t="1099">because at least then you know how to code.</a> <a data-t="1102">You don't have some skill that's not going to work anywhere else.</a></p>
