<h3>About CMAR and the data challenge</h3>

<p><a data-t="69">So hi, everyone.</a> <a data-t="69">My name is Danielle Dempsey.</a> <a data-t="69">I'm a research scientist at a small company</a> <a data-t="69">called the Centre for Marine Applied Research, or CMAR for short, in Nova Scotia, Canada.</a> <a data-t="82">CMAR coordinates this really great coastal monitoring program to measure ocean variables</a> <a data-t="87">like temperature, dissolved oxygen, and salinity from all around the coast of the province.</a> <a data-t="94">This data can be used by scientists to study coastal ecosystems, industry to inform their</a> <a data-t="100">site selection, government to inform their policy and management decisions, and the list</a> <a data-t="105">could go on.</a></p>

<p><a data-t="107">So this is really valuable data, but for several years, it was rarely used because it was just</a> <a data-t="116">stored on somebody's desktop in a myriad of CSV and Excel files that were really difficult</a> <a data-t="121">and time-consuming to compile into a useful format.</a></p>

<p><a data-t="126">So why was this so difficult?</a> <a data-t="129">It has to do with how the data are collected using what we call sensor strings.</a> <a data-t="135">So a sensor string consists of a rope that's anchored to the ocean floor and suspended</a> <a data-t="140">just below the surface by a buoy.</a> <a data-t="143">There are sensors attached to the rope at different depths through the water column,</a> <a data-t="148">and these sensors are recording data every one minute to one hour.</a> <a data-t="154">Each sensor string is deployed in one location for a couple of months, up to a year.</a> <a data-t="159">So they're really collecting a lot of data.</a></p>

<p><a data-t="163">And we use three different types of sensors that each report the data back in a slightly</a> <a data-t="169">different format, like you can see on the screen.</a> <a data-t="173">And you can start to appreciate how working with this data can be challenging, especially</a> <a data-t="178">because there can be seven or more sensors on each sensor string, and each of these data</a> <a data-t="183">files can be tens of thousands of rows long.</a></p>

<h3>The old copy-and-paste process</h3>

<p><a data-t="189">So part of the reason that I was hired about two years ago was to help compile all of this</a> <a data-t="195">data into a nice, tidy format that we could post online for any interested stakeholder</a> <a data-t="201">to download and use in their own analyses.</a> <a data-t="205">So about three weeks after I started this new position, I had a video call with my manager</a> <a data-t="211">so she could show me the existing process for how this data is compiled.</a></p>

<p><a data-t="218">And I can remember very vividly sitting in my home office on a beautiful Thursday afternoon</a> <a data-t="224">while she showed me this process.</a> <a data-t="226">And let me tell you, this involved copying.</a> <a data-t="231">It involved pasting.</a> <a data-t="232">It involved sorting the data by a specific column.</a> <a data-t="236">And it even involved writing down the Excel row numbers to use later on in the process.</a> <a data-t="242">It was truly horrible.</a> <a data-t="245">And I thought to myself, I will never do that.</a></p>

<p><a data-t="250">I offered to write some code to help with this, but my manager was a little bit skeptical</a> <a data-t="255">at first, particularly because at this point I wasn't very confident in my data wrangling</a> <a data-t="262">or package development skills.</a> <a data-t="265">But eventually she gave me the thumbs up and I wrote a package called strings to help automate</a> <a data-t="271">a lot of this process.</a> <a data-t="274">To be clear, the package is named after our sensor strings and has nothing to do with</a> <a data-t="278">character strings.</a> <a data-t="280">So this was maybe a bit of a naive name choice, but it's only caused one small moment of confusion</a> <a data-t="286">so far, and that's another story.</a></p>

<h3>Writing code saves time</h3>

<p><a data-t="295">Writing code will save you time.</a> <a data-t="298">Yes, you'll have to spend some time up front to develop the code, and you may have some</a> <a data-t="304">learning curves to surmount, particularly if you're new to R, new to coding, but it</a> <a data-t="308">will be worth it.</a></p>

<p><a data-t="312">So I spent a few weeks developing the backbone of the strings package, and as I said, at</a> <a data-t="322">this time I didn't have a lot of experience with data wrangling, package development,</a> <a data-t="328">working with dates and times, or importing and exporting large data files.</a> <a data-t="334">So I spent a pretty substantial amount of time looking into different packages to help</a> <a data-t="338">with those challenges.</a> <a data-t="341">And these are just some of the packages that I ended up working with.</a> <a data-t="344">You'll recognize most of them from the </a><a href="/software/tidyverse/">tidyverse</a><a data-t="344">, although there are a couple from outside of</a> <a data-t="348">that collection.</a></p>

<p><a data-t="351">And the point I want to make here is that even though I hadn't used most of these packages</a> <a data-t="355">before working on strings, I now use almost all of them in almost all of my coding projects,</a> <a data-t="363">and that's because of the time that I spent learning about them to work on the strings</a> <a data-t="366">package.</a> <a data-t="368">So this is just one small example of how the skills that you learn working on one coding</a> <a data-t="372">project are often really transferable to other projects.</a></p>

<p><a data-t="377">And in my case, the time that I spent working on the strings package also saved us time</a> <a data-t="383">for processing that coastal monitoring program data.</a> <a data-t="387">We estimate that it would have taken two years of work hours to compile all of that data</a> <a data-t="394">with the horrendous old copy and paste method.</a> <a data-t="397">Instead, within a couple of months, I had developed the package, myself and our student</a> <a data-t="404">at the time had compiled and formatted all of the data, posted it online, and generated</a> <a data-t="410">summary reports through </a><a href="/software/rmarkdown/">rmarkdown</a><a data-t="410">.</a></p>

<p><a data-t="415">It's also now really easy to train new people to compile this data using templates that</a> <a data-t="420">I wrote to go with the package.</a> <a data-t="423">Our current intern has compiled 71 deployments so far on top of all of his other work in</a> <a data-t="432">just the 11 weeks that he's been with us so far.</a> <a data-t="436">Something that would have taken over nine months of dedicated work time with the old</a> <a data-t="441">method.</a></p>

<h3>Writing code reduces errors</h3>

<p><a data-t="475">Writing code to help with your data wrangling can also help find and reduce errors in your</a> <a data-t="480">data set.</a> <a data-t="483">This will give you and your clients more confidence in the data, and also make whatever analysis</a> <a data-t="490">you plan on doing with that data more reliable.</a> <a data-t="494">And in my experience, errors can kind of creep into the data at different points along the</a> <a data-t="499">data pipeline, especially wherever humans are involved.</a> <a data-t="504">And so I've learned to write code to anticipate and sometimes even fix those errors.</a></p>

<p><a data-t="513">For the first one, after we had finished compiling all that sensor string data, I decided to</a> <a data-t="519">plot the latitude and longitude of each sensor string to get an idea of the spatial coverage</a> <a data-t="525">of the data.</a> <a data-t="527">So you can imagine my surprise when my map looks something like this.</a> <a data-t="533">So most of the sensor strings are showing up clustered around Nova Scotia, which is</a> <a data-t="538">what we would expect.</a> <a data-t="540">But there's also some sensor strings showing up way over here in Kazakhstan, which is not</a> <a data-t="546">close to Nova Scotia and not even anywhere near the ocean.</a></p>

<p><a data-t="553">After a little bit of digging, I discovered that our contractors who actually deploy these</a> <a data-t="558">sensor strings have to manually record the latitude and longitude of each location.</a> <a data-t="566">And because Nova Scotia is in the western hemisphere, longitude is typically recorded</a> <a data-t="571">as a negative value.</a> <a data-t="573">But every so often, it accidentally gets entered as a positive value, which is what's causing</a> <a data-t="578">some of these sensor strings to show up way over there in Central Asia.</a> <a data-t="584">So now, when the package first reads in this coordinate data, I'll get a warning if any</a> <a data-t="590">of those longitude values are positive.</a> <a data-t="592">So we can go in and fix that mistake before compiling all the data.</a></p>

<p><a data-t="599">A kind of more insidious error that I've seen is data from the wrong sensor in a file</a> <a data-t="607">of data that's ready to be compiled.</a> <a data-t="610">And this just happens when someone has what I call a human moment and accidentally puts</a> <a data-t="615">the wrong file in the folder.</a> <a data-t="618">In this example, data from one of the sensors on the blue sensor string has been put in</a> <a data-t="623">the folder with all of the data from sensors on the green sensor string.</a> <a data-t="628">And this issue is really twofold.</a> <a data-t="631">One, the data set for the blue sensor string will be incomplete, and two, we will get false</a> <a data-t="638">information about the area where the green sensor string was deployed.</a> <a data-t="643">So this is really a critical error to be able to find and fix.</a></p>

<p><a data-t="647">Luckily, the data files are automatically named with the serial number of the sensor.</a> <a data-t="654">And we keep track of which sensors are associated with each sensor string in deployment logs.</a> <a data-t="661">And so I've written code in the package that will give an error if there's any missing</a> <a data-t="667">extra or different files in that folder based on what we would expect from the log.</a></p>

<h3>Writing code improves traceability</h3>

<p><a data-t="678">Writing code to help automate your data wrangling can also make your workflow more traceable</a> <a data-t="683">and reproducible, especially when compared to something like copying and pasting.</a> <a data-t="688">And you can explain this to your non-coding managers with a metaphor.</a></p>

<p><a data-t="694">So copying and pasting in Excel is kind of like making a salad from just whatever is</a> <a data-t="699">left in your fridge.</a> <a data-t="701">Most of the time, it will probably taste okay.</a> <a data-t="705">But eventually you will get something that doesn't taste quite right.</a> <a data-t="710">Because you have no record of what went into that salad, it's really hard to tell what</a> <a data-t="714">exactly is making it taste funny.</a> <a data-t="718">And it's also hard to avoid making the same mistake in the future.</a> <a data-t="723">As my colleague said when I was trying this metaphor out on her, you can't take the dressing</a> <a data-t="727">back out of the salad.</a></p>

<blockquote><a data-t="723">As my colleague said when I was trying this metaphor out on her, you can't take the dressing</a> <a data-t="727">back out of the salad.</a></blockquote>

<p><a data-t="731">In contrast, writing code is more like writing and following a recipe.</a> <a data-t="737">You can follow it step by step to get a delicious salad every time.</a> <a data-t="742">And as an added bonus, you can even give this recipe to a friend so they can make their</a> <a data-t="747">own salad or help you make dinner.</a></p>

<p><a data-t="754">As a more concrete example, for another part of our coastal monitoring program, we had</a> <a data-t="760">these 65 current datasets that had to be compiled from a kind of messy text format into a tidier</a> <a data-t="767">CSV format that was easier to work with.</a> <a data-t="771">So I offered to write some code to help with this.</a> <a data-t="774">But my managers again decided to have a couple of people instead copy, paste, and format</a> <a data-t="780">in Excel.</a> <a data-t="782">This took hours and hours of several people's time.</a> <a data-t="787">But eventually they finished.</a></p>

<p><a data-t="790">And then we discovered that there had been a small mistake in how these original text</a> <a data-t="796">files were exported from the sensor software.</a> <a data-t="799">So all of the data needed to be compiled again.</a> <a data-t="805">So this time I was asked to write some code.</a> <a data-t="810">And within a few weeks, I had written a package that automated most of this workflow.</a> <a data-t="815">So now if we want to make any other changes to this process, we've actually made a couple,</a> <a data-t="822">just need to go in, tweak the code, press a button, and all of this data will be compiled</a> <a data-t="827">within just a few minutes.</a></p>

<p><a data-t="831">And so if you have a repetitive task that you've been thinking about automating, I encourage</a> <a data-t="837">you to go home after the conference and start writing some code.</a> <a data-t="841">You don't have to write a whole package in a day.</a> <a data-t="844">You can start with a function or two and just see where it goes from there.</a> <a data-t="848">And if you do get some pushback from a skeptical manager, remind them that writing code will</a> <a data-t="855">save time, reduce errors, and improve the workflow.</a> <a data-t="859">And if you do use those points and it works, please let me know, because it would make</a> <a data-t="864">my day.</a></p>

<p><a data-t="866">And the last thing I'll say is that my previously skeptical manager is now totally on board</a> <a data-t="873">and really supportive of all my coding.</a> <a data-t="876">She even showed me this super R, because she says that I've convinced her that R can do</a> <a data-t="881">anything that we put our minds to.</a> <a data-t="884">So thank you, and happy coding.</a></p>
