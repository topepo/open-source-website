<h3>The Jupyter development pattern problem</h3>

<p><a data-t="74">And I think the key problem is that Jupyter uses like</a> <a data-t="74">a fundamentally different development pattern than other computer science tools. <p><a data-t="121">But it causes a lot of problems. <h3>The R Markdown write-execute pattern</h3>

<p><a data-t="321">So, R actually had a different whole pattern to</a> <a data-t="321">notebook development, which is these write-execute notebooks. So, when you write an R Markdown</a> <a data-t="327">notebook, this is RMD style, you actually just write the sort of code and text together</a> <a data-t="334">in a single document. And just like source code, that document is kind of a rigid artifact.</a> <a data-t="339">It's just text. There's no logic in the R Markdown notebook. There's no state in the</a> <a data-t="344">R Markdown notebook. It's just text. And what the renderer does is take that document,</a> <a data-t="352">go find all the things that are labeled as code, pass that code through an interpreter,</a> <a data-t="357">and then put the code and the text back together into these output formats.</a></p>

<p><a data-t="363">And this is a really</a> <a data-t="363">wonderful pattern, because like most of the things work right away, right? So, there hasn't</a> <a data-t="373">ever been like an R Markdown Git parser, right? Because R Markdown documents are just text,</a> <a data-t="378">so they just work with Git, right? It's just like any other computer source code from when</a> <a data-t="383">there were punch cards, right? It's a text that's then interpreted. Similarly, if you</a> <a data-t="387">ever use like NuMake or any of those orchestration tools, they work really well with this code,</a> <a data-t="393">because that's kind of what they're expecting. They're expecting text that's then run through</a> <a data-t="398">some kind of interpreter. Similarly, it's really, really easy to compose R Markdown</a> <a data-t="402">notebooks. You could either copy and paste them together or parse them together in some</a> <a data-t="406">other format. It's easy to package libraries, and it's also really easy to mix languages. So,</a> <a data-t="411">you can, I think, use something like 17 different computer programming languages in an R Markdown</a> <a data-t="416">document, and they'll all play well together, because it just goes and finds the Python code,</a> <a data-t="420">runs it through a Python interpreter, goes and finds the Scala code, runs it through a Scala</a> <a data-t="423">interpreter, right? And these are all kind of well-solved problems in computer science.</a></p>

<p><a data-t="430">But there's three big problems. The first problem is that it starts with the letter R,</a> <a data-t="434">right? And so, I work with a lot of Python users, and, you know, it's just, it's their</a> <a data-t="440">least favourite letter. If you propose any kind of tool or solution, and that solution starts with</a> <a data-t="445">R, or even has an R in it, they just say no. The second sort of maybe more real problem is that it</a> <a data-t="454">requires an R runtime, right? So, R is a very niche programming language. It's not sort of</a> <a data-t="459">necessarily installed on everybody's laptop. You know, it's not installed on every server,</a> <a data-t="462">so that can be difficult. And the last one is that it has a slightly inconsistent user interface.</a> <a data-t="469">So, R Markdown is a ten-year-old technology, and over time, it's kind of been elaborated on to do</a> <a data-t="474">a lot of different things. So, like, people write their blogs in it, or my blog in R Markdown. This,</a> <a data-t="479">you know, you could write presentations in R Markdown, books, you know, and then also just</a> <a data-t="484">kind of data science reports. And over time, it's accommodated all of those use cases,</a> <a data-t="487">kind of started adding different options, and those options started drifting in terms of how</a> <a data-t="492">consistent they were.</a></p>

<p><a data-t="498">So, this is where Quarto comes in.
