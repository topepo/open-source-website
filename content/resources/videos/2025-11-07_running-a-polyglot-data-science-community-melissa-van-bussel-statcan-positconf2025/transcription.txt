<blockquote><a data-t="85">In our group, we really hold the philosophy</a> <a data-t="88">that the community aspect is far more important</a> <a data-t="92">than the programming language that someone uses.</a></blockquote>

<h3>Building a polyglot identity</h3>

<p><a data-t="182">If you ask somebody to introduce themselves,</a> <a data-t="185">especially at this conference,</a> <a data-t="187">you'll notice that people's introductions</a> <a data-t="191">usually include the programming language that they use.</a> <a data-t="195">Someone might describe themselves as an R person</a> <a data-t="198">or as a Python person in the same sentence</a> <a data-t="201">where they tell you what their name is and where they work.</a></p>

<p><a data-t="206">For a lot of us, the programming language that we use</a> <a data-t="209">can feel very core to our identity.</a></p>

<p><a data-t="214">And so if you want to create a successful Polyglot group,</a> <a data-t="218">then my recommendation to you</a> <a data-t="220">is to create branding for your group</a> <a data-t="222">that people can identify with</a> <a data-t="225">and can associate with your group.</a> <a data-t="227">But more specifically, you want to make sure</a> <a data-t="229">that that branding isn't overly specific to one language.</a></p>

<p><a data-t="235">For the R and Python user group,</a> <a data-t="238">our branding is centered around our group's acronym,</a> <a data-t="241">which is RPUG.</a> <a data-t="243">And in 2021, one of our members, Thomas Wood,</a> <a data-t="248">created a logo for the group, which is a pirate pug.</a></p>

<p><a data-t="255">This logo was created entirely using code,</a> <a data-t="258">which means it's completely reproducible</a> <a data-t="261">and also pretty easy to modify,</a> <a data-t="263">which means that it wasn't long</a> <a data-t="265">before other members of the group</a> <a data-t="269">started creating spin-offs</a> <a data-t="271">of the original logo.</a></p>

<p><a data-t="274">As of today, we've got over 30 different versions</a> <a data-t="277">of this logo, the stickers of which you can find</a> <a data-t="280">all over the place at the StatCan office in Ottawa.</a></p>

<p><a data-t="285">One of our other members, Alex McSween,</a> <a data-t="288">who's actually here at Conf,</a> <a data-t="291">created a 2025 RPUG calendar</a> <a data-t="294">where each month of the year features a different pug</a> <a data-t="298">based on that month's theme.</a> <a data-t="300">Over the last year, we've also hosted</a> <a data-t="302">a couple of merch orders that have been centered</a> <a data-t="305">around these logos.</a></p>

<p><a data-t="308">A brand isn't just a brand, and a logo isn't just a logo.</a> <a data-t="314">It's something that helps your members feel more connected</a> <a data-t="317">to something bigger than just the language that they use,</a> <a data-t="320">but more importantly, it also helps them feel connected</a> <a data-t="324">to your group.</a></p>

<h3>Creating something for everyone</h3>

<p><a data-t="325">Once your group has its own identity,</a> <a data-t="327">the next step is to make sure</a> <a data-t="329">that there's something for everyone.</a> <a data-t="332">Because a polyglot group is going to have</a> <a data-t="335">a wide variety of people, sort of by definition,</a> <a data-t="339">you wanna create a multi-park experience</a> <a data-t="342">where every person in the group</a> <a data-t="344">feels like there's something that's for them.</a></p>

<p><a data-t="350">At Disney, we've also created a multi-park experience</a> <a data-t="354">at Disney World.</a> <a data-t="356">Epcot is the theme park that's the most focused</a> <a data-t="359">on learning and exploration.</a> <a data-t="361">And in RPUG, we've got our own little version of Epcot</a> <a data-t="365">that provides our members with a number of ways</a> <a data-t="368">that they can learn new skills related to open source.</a></p>

<p><a data-t="372">For example, we have a monthly meeting</a> <a data-t="375">where we share updates about what's new</a> <a data-t="377">in the world of open source and at Statistics Canada,</a> <a data-t="382">followed by a presentation or a workshop</a> <a data-t="386">from a guest speaker.</a></p>

<p><a data-t="388">We try to make sure that these presentations</a> <a data-t="390">aren't super specific to one language</a> <a data-t="393">by choosing topics that are broadly applicable</a> <a data-t="396">to anybody who works with data.</a></p>

<p><a data-t="400">Over the last year, some of the topics that we've had</a> <a data-t="403">have included a showcase of internally developed</a> <a data-t="406">R and Python packages, a roundup of success stories</a> <a data-t="411">when working with the parquet file format,</a> <a data-t="414">and a presentation from the Statistics Canada library</a> <a data-t="417">showing our members how they can access library resources</a> <a data-t="421">to learn about open source topics.</a></p>

<p><a data-t="425">We've also got an internal website that has guides</a> <a data-t="428">and tutorials for working with R and Python</a> <a data-t="431">in a way that's specific to our workplace's</a> <a data-t="434">technical infrastructure.</a> <a data-t="437">And we've got a webpage that outlines project templates</a> <a data-t="441">and packages that have been developed</a> <a data-t="442">by the members of our community.</a></p>

<p><a data-t="447">Hollywood Studios is the theme park</a> <a data-t="449">that's all about going behind the scenes</a> <a data-t="452">and seeing the hidden work that goes into making</a> <a data-t="455">the published product before it ever ends up</a> <a data-t="457">on the big screen.</a> <a data-t="460">For us, that means giving people a safe space</a> <a data-t="463">to try things out, to make mistakes,</a> <a data-t="466">to learn, and to ask questions.</a></p>

<p><a data-t="470">We host low-stakes code challenges</a> <a data-t="473">where the prompt can either be solved</a> <a data-t="475">in just a couple of lines of code or in a lot more,</a> <a data-t="479">depending on how creative the person wants to be</a> <a data-t="481">with their solution.</a> <a data-t="483">And participants are encouraged to have fun</a> <a data-t="486">and be silly with it so that it doesn't feel like work.</a></p>

<p><a data-t="490">You can use any language that you want for these challenges</a> <a data-t="493">and this tends to be a space where people will</a> <a data-t="496">often use a language that they're less comfortable in.</a></p>

<p><a data-t="501">We've also got a number of different ways</a> <a data-t="504">that our members can ask for help with programming</a> <a data-t="506">or other technical questions.</a> <a data-t="509">We've got a very large chat on Microsoft Teams</a> <a data-t="513">where people ask questions casually throughout the day,</a> <a data-t="516">but we also have nine volunteers that offer</a> <a data-t="520">drop-in office hours on a regular basis.</a></p>

<p><a data-t="525">Because the group is polyglot,</a> <a data-t="527">each office hour clearly outlines</a> <a data-t="531">which topics the current volunteer</a> <a data-t="532">can answer questions about.</a> <a data-t="534">And it also points users to the full list of office hours</a> <a data-t="538">if they need help figuring out who the right person is</a> <a data-t="541">that they should reach out to for a specific topic.</a></p>

<h3>Fun events and Magic Kingdom moments</h3>

<p><a data-t="546">If you've ever been to Disney World,</a> <a data-t="548">then your favorite theme park is probably Magic Kingdom,</a> <a data-t="551">which is a place all about having fun just for fun's sake.</a> <a data-t="556">Over the last year, RPUG has had quite a few events</a> <a data-t="561">that have had absolutely nothing to do with programming</a> <a data-t="566">and have really just been an opportunity</a> <a data-t="568">for like-minded people to hang out together</a> <a data-t="570">and do something fun outside of work hours.</a></p>

<p><a data-t="574">We've had a couple of trivia nights</a> <a data-t="576">and recently we had an event called Beads and Brews</a> <a data-t="580">where we got together for dinner after work</a> <a data-t="583">and then people could make friendship bracelets</a> <a data-t="585">or key chains or whatever else they wanted to make.</a></p>

<p><a data-t="591">This year, we're also launching an RPUG yearbook</a> <a data-t="594">just for fun where we can put all of the pictures</a> <a data-t="597">from this year's events.</a> <a data-t="599">And we'll also have a section for RPUG awards</a> <a data-t="602">so that we can recognize the contributions of our members.</a></p>

<p><a data-t="608">Now a lot of the stuff that I've just talked about</a> <a data-t="610">might seem kind of silly,</a> <a data-t="612">especially in the context of a group that in theory</a> <a data-t="615">is supposed to be focused on data science,</a> <a data-t="619">but having these fun activities is actually really important</a> <a data-t="622">for keeping members engaged with your group.</a></p>

<p><a data-t="625">And if your group is polyglot,</a> <a data-t="628">then I'd argue that this kind of thing</a> <a data-t="630">is even more important</a> <a data-t="631">because it's these types of events and activities</a> <a data-t="634">that are going to give your members a chance to connect</a> <a data-t="638">and figure out what stuff they do have in common.</a> <a data-t="641">It's a lot harder for somebody to make a new friend</a> <a data-t="645">during a programming workshop</a> <a data-t="647">than it is for them to make a new friend</a> <a data-t="649">while they're doing something fun and casual.</a></p>

<p><a data-t="652">And the more that the members of your group</a> <a data-t="653">are friends with each other,</a> <a data-t="655">the more they're going to want to participate</a> <a data-t="656">in your group's activities as a result.</a></p>

<h3>Letting members be part of the story</h3>

<p><a data-t="662">Another way that you can increase participation</a> <a data-t="665">in your group is by taking a page</a> <a data-t="667">out of Disney World's book.</a> <a data-t="670">When you go to Disney World,</a> <a data-t="672">you don't just feel like you're watching a bunch of actors.</a> <a data-t="676">You feel like you're a character</a> <a data-t="678">that actually belongs in the story.</a> <a data-t="681">And you want to create this same kind of feeling</a> <a data-t="683">within your group.</a></p>

<p><a data-t="685">One of the easiest ways that you can do this</a> <a data-t="688">is by letting every member of the group</a> <a data-t="690">be as involved as they want to be</a> <a data-t="693">with the organization and the planning of the group.</a> <a data-t="696">And if your group is polyglot,</a> <a data-t="699">again, this is going to be even more important</a> <a data-t="701">because nobody is an expert at everything.</a> <a data-t="705">So you want to make sure you've got a wide range of voices</a> <a data-t="708">so that each member of the group feels represented</a> <a data-t="712">and welcome, regardless of which languages</a> <a data-t="714">or tools they use.</a></p>

<p><a data-t="718">In RPUG, we have the philosophy</a> <a data-t="721">that anybody who wants to contribute can.</a> <a data-t="725">There's a few different ways that we do this,</a> <a data-t="727">and one of them is the RPUG Weekly Newsletter.</a> <a data-t="730">And anybody at Statistics Canada</a> <a data-t="733">can publish something in this newsletter.</a> <a data-t="736">You actually don't even have to be a part of the group</a> <a data-t="738">to submit an article to it.</a></p>

<p><a data-t="741">What we get as a result is a weekly,</a> <a data-t="744">community-curated collection of upcoming events,</a> <a data-t="748">programming tips, and so on.</a> <a data-t="751">We also have an internal RPUG website,</a> <a data-t="754">which anyone can contribute to,</a> <a data-t="756">by submitting a merge request on GitLab.</a> <a data-t="761">And I took these numbers directly from the repository,</a> <a data-t="765">and we can see that in the last four months alone,</a> <a data-t="767">we've had 33 contributors across almost 2,000 commits.</a></p>

<p><a data-t="775">This group is grassroots,</a> <a data-t="779">it's really by users and for users,</a> <a data-t="782">which means that the people who are organizing the group</a> <a data-t="785">are also the same people who actually use the tools</a> <a data-t="788">on a regular basis,</a> <a data-t="790">rather than people from senior management, for example.</a></p>

<p><a data-t="794">We try really hard to keep things casual.</a> <a data-t="798">We try to avoid getting too bogged down with bureaucracy.</a> <a data-t="801">But with that being said,</a> <a data-t="803">of course, there is still a lot of work to be done.</a></p>

<p><a data-t="827">To be honest, it would be basically impossible</a> <a data-t="829">to have a group of 1,000 people</a> <a data-t="831">and not have some type of structure.</a> <a data-t="834">It definitely takes a village</a> <a data-t="835">to have a group this large run smoothly.</a></p>

<p><a data-t="850">The committee is open to anyone,</a> <a data-t="861">and it does tend to be the more advanced programmers</a> <a data-t="864">who join this subset.</a> <a data-t="870">Personally, I really love being part of this committee.</a> <a data-t="873">We meet every two weeks,</a> <a data-t="875">and in addition to planning some of the fun stuff</a> <a data-t="877">that I've talked about earlier today,</a> <a data-t="879">this is also the group that discusses</a> <a data-t="881">the more technical topics.</a> <a data-t="883">In particular, we talk a lot about</a> <a data-t="885">how open-source tools are being used</a> <a data-t="887">and supported in our workplace.</a></p>

<p><a data-t="891">We've found that this model where anybody can join</a> <a data-t="895">and everyone can be as involved as they want to be</a> <a data-t="898">has worked pretty well for us.</a></p>

<h3>Why you should try the polyglot approach</h3>

<p><a data-t="902">If you're in this room right now,</a> <a data-t="904">it's probably because you're somebody who cares deeply</a> <a data-t="907">about building and sustaining data communities.</a> <a data-t="911">And if that's you, then there's probably a good chance</a> <a data-t="914">that one of the following scenarios applies to you.</a></p>

<p><a data-t="940">Maybe you're in a smaller city or a smaller workplace,</a> <a data-t="944">and so the pool of potential people</a> <a data-t="947">that could possibly participate in your group</a> <a data-t="950">is just a little bit smaller.</a></p>

<p><a data-t="954">If the second or third one applies to you,</a> <a data-t="958">then I would really encourage you</a> <a data-t="959">to give the polyglot approach a chance.</a> <a data-t="964">By doing so, you're multiplying your potential members,</a> <a data-t="967">you're multiplying your potential impact,</a> <a data-t="970">and you might also learn a new programming language</a> <a data-t="973">along the way.</a></p>
