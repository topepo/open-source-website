<h3>Exploring the app code</h3>

<p><a data-t="176">This code is actually for a Streamlit app that I created several years ago when I was</a> <a data-t="180">very new to Streamlit.</a> <a data-t="182">This is an example using trauma hospital analytics from the US and it visualizes hospital locations</a> <a data-t="188">and analyzes geographic accessibility based on Kyle Walker's tutorial.</a> <a data-t="195">Let's look at the different sections.</a></p>

<p><a data-t="197">The initial lines of code import necessary libraries such as pandas and numpy, matplotlib</a> <a data-t="203">and seaborn, geopandas, and of course, Streamlit.</a> <a data-t="210">The next part configures the Streamlit page and also adds a title and a caption.</a> <a data-t="217">The next section handles the user input and data retrieval.</a> <a data-t="221">ST sidebar creates a sidebar on the web page for user controls.</a> <a data-t="225">ST select box creates a drop-down menu in the sidebar which allows the user to select</a> <a data-t="231">one of three states, California, Arizona, or Wisconsin, and the selected value is stored</a> <a data-t="236">in the state variable.</a></p>

<p><a data-t="239">Cache data is a Streamlit caching decorator.</a> <a data-t="241">It tells Streamlit to run the function below only once for a given set of inputs and that</a> <a data-t="246">the function is called again with the same inputs of retrieve the cache result instead</a> <a data-t="251">of rerunning the function which saves time.</a> <a data-t="255">Set data uses Pygris to download census track data for the selected state.</a> <a data-t="261">Load data reads a GeoJSON file named trauma.GeoJSON which contains the trauma hospital locations.</a> <a data-t="269">It then filters this data to include only the hospital in the selected data frame and</a> <a data-t="275">returns the filtered geo data frame.</a></p>

<p><a data-t="280">Finally, this part of the code structures the app's layout and performs the core geospatial</a> <a data-t="285">analysis.</a> <a data-t="287">ST columns divides the main content area into two columns and then with column defines the</a> <a data-t="295">content to be placed in each column.</a> <a data-t="298">In the first column, the display is an interactive map using the trauma geo data frame showing</a> <a data-t="304">the locations of the trauma hospitals.</a> <a data-t="307">In the second column, there is a table with specific columns of the raw trauma hospital</a> <a data-t="312">data and then also a bar chart of the histogram data visualizing the distribution and minimum</a> <a data-t="317">distances from the census tracks to trauma centers.</a></p>
