<p><a data-t="1237">Well, there's two ways that we think about this at RStudio.</a> <a data-t="1241">One is to influence how decision makers are making decisions through your model.</a> <a data-t="1248">And a great way to do that is through interactive applications.</a> <a data-t="1252">So many of you might be familiar with tools like </a><a href="/software/shiny-r/">Shiny</a><a data-t="1252">, which allow you to do that in R.</a> <a data-t="1258">But we've worked hard in the last year to ensure that multilingual teams have that same capability.</a> <a data-t="1264">So what you're looking at here is a Dash application,</a> <a data-t="1268">which is an interactive application written in Python.</a></p>

<p><a data-t="1271">And what this application is going to allow us to do is help stakeholders understand our model.</a> <a data-t="1278">So they can come in here and click through different stations</a> <a data-t="1282">and see the forecast and the location of the station.</a> <a data-t="1285">So it's a pretty simple app, but hopefully it kind of gets your wheels turning</a> <a data-t="1290">that even if you're a Python data scientist or you work with Python data scientists,</a> <a data-t="1294">they have the same ability to impact decisions by creating interactive content.</a> <a data-t="1299">And we see examples of that with Dash, with Streamlet, with Bokeh,</a> <a data-t="1303">with a wide variety of interactive Python frameworks that are supported through the RStudio stack.</a></p>

<p><a data-t="1314">So that's how you might go about influencing a person with your model.</a> <a data-t="1318">But what if you need to use your model to make a whole bunch of automated decisions?</a> <a data-t="1322">Or if you need to influence not a person, but a service?</a> <a data-t="1323">One way that's common to solve that problem is by creating an API.</a></p>

<p><a data-t="1329">And again, there's options for doing this in R and Python.</a> <a data-t="1332">So on the R side, we have tools like </a><a href="/software/plumber/">Plumber</a><a data-t="1332">.</a> <a data-t="1335">On the Python side, we have tools like Flask.</a> <a data-t="1339">And both of these can be shared just as easily through RStudio Connect.</a> <a data-t="1344">And so I'll just quickly show you what that API might entail.</a> <a data-t="1347">Again, we have all of the same controls.</a> <a data-t="1350">And so we can look at the logs of this Plumber API.</a> <a data-t="1354">We could do things like scale.</a> <a data-t="1356">If we know we're going to have thousands of requests to this API at the same time,</a> <a data-t="1360">we can specify how we want the system to handle those requests.</a></p>

<p><a data-t="1366">But at the end of the day, the idea is pretty simple.</a> <a data-t="1366">It's that anyone can come in and place a parameter of your model.</a> <a data-t="1372">In this case, we're specifying the station that we want to make a prediction for</a> <a data-t="1376">and the time horizon that we want to make a prediction for.</a> <a data-t="1380">And then those inputs are passed to our model, and the results are returned.</a> <a data-t="1385">So here we have the results of our forecast.</a> <a data-t="1388">But as you can see, that passing of inputs and outputs</a> <a data-t="1392">is done in a way that machines can understand.</a> <a data-t="1395">So here we have the output in JSON.</a> <a data-t="1397">And just above that, we have the request</a> <a data-t="1400">that our kind of interactive exploration of this API would generate.</a> <a data-t="1405">So other systems or services or software engineers</a> <a data-t="1408">can take advantage of your model at scale.</a></p>
