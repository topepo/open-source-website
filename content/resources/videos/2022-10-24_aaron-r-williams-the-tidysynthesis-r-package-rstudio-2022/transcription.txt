<h3>The value of administrative data</h3>

<p><a data-t="103">Administrative data are useful, and I'll give you two examples. The first example comes from March of 2020. It</a> <a data-t="110">was a scary time when there was a health crisis and an economic crisis unfolding at</a> <a data-t="114">the same time. One issue, though, is that a lot of our indicators for the labor market</a> <a data-t="120">are backward-looking, and so there's one indicator, one number that came from administrative data</a> <a data-t="126">that I think painted an excellent picture of the labor market. Many people had never</a> <a data-t="130">heard of it, but it ended up in a dramatic data visualization on the cover of the New</a> <a data-t="134">York Times. This is initial weekly unemployment insurance claims. If you look at the bottom</a> <a data-t="139">left, that starts in 2000, and it bounces about from 2000 to 2020, and then there's</a> <a data-t="145">this black swan event that just shows how scary of a time it was in our labor market.</a></p>

<p><a data-t="151">I'll give you another example. In 2013, Raj Chetty, Emanuel Saez, and co-authors released</a> <a data-t="159">groundbreaking research about intergenerational mobility in the United States using data from</a> <a data-t="165">40 million anonymized tax records. This was really big research that deals with how Americans</a> <a data-t="171">see themselves. We have that desire for our children to be better off than ourselves.</a> <a data-t="176">One of the things that they found is that place matters quite a bit. For example, the</a> <a data-t="182">probability that a child reaches the top quintile of the national income distribution, starting</a> <a data-t="186">from a family in the bottom quintile, is 4.4% in Charlotte, North Carolina, but 12.9% in</a> <a data-t="194">San </a><a href="/software/jose/">Jose</a><a data-t="194">, California.</a></p>

<p><a data-t="205">But some people say that this research never should have happened. In a traditional</a> <a data-t="210">model with an agency like the IRS, you do correspondence research. You have to get your</a> <a data-t="215">analysis plan approved. You may never actually see the data. You send off your code. They</a> <a data-t="220">run it on the data. They review the results and the like. What Saez and Chetty did was</a> <a data-t="225">radical and different. They actually became employees of the IRS. They had to go through</a> <a data-t="231">an extensive background check, have their fingerprints taken, go through lots of training,</a> <a data-t="231">have all their results reviewed. I've been through this process. Let me tell you, it's</a> <a data-t="235">not pleasant at all.</a></p>

<p><a data-t="241">The question is, how can we get this understanding,</a> <a data-t="247">this value, this meaning from administrative data without all this inconvenience, all while</a> <a data-t="253">protecting confidentiality? That's why we're working with synthetic data. If we can create</a> <a data-t="259">fake data that match the statistical properties of the confidential data, ways of safely validating</a> <a data-t="264">the results from the synthetic data against the confidential data, then we can conveniently</a> <a data-t="268">unlock a lot of this potential.</a></p>

<h3>The sequential synthesis approach</h3>

<p><a data-t="325">Let's consider a much simpler example for now where we have four variables and two observations.</a> <a data-t="331">That's our confidential data set. The setup of what we're trying to do is create a new</a> <a data-t="335">data set that has the exact same record layout. We're going to employ a sequential approach</a> <a data-t="340">to create this data set. The first thing we'll do is we'll sample sex with replacement. Then</a> <a data-t="346">we'll create a predictive model to predict age, conditional on sex. Then we'll create</a> <a data-t="351">a predictive model to predict wages, conditional on sex and age. Finally, a predictive model</a> <a data-t="356">to predict taxes, conditional on sex, age, and wages.</a></p>

<p><a data-t="360">A lot of predictive models, and this is where I get to express my gratitude for </a><a href="/software/tidymodels/">tidymodels</a> <a data-t="365">and all the people who have worked on this. We heard some great things about it this morning.</a> <a data-t="369">tidymodels is a comprehensive framework for doing predictive modeling in R, and we use</a> <a data-t="373">it extensively. In particular, we use </a><a href="/software/parsnip/">Parsnip</a><a data-t="373"> quite a bit, which is an interface to different</a> <a data-t="378">machine learning and predictive models, and Recipes, which is a really powerful package</a> <a data-t="382">for feature and target engineering.</a></p>

<p><a data-t="386">Now, in the example that we're working on,</a> <a data-t="390">we need to do three predictive models. I gave you another example where we need to</a> <a data-t="394">do more than 200 predictive models. So we could imagine having an R script where it's</a> <a data-t="399">model one, predict, model two, predict, model three, predict. But if you're going to go</a> <a data-t="404">all the way to 200 variables, that's going to become a real pain really quickly.</a></p>

<h3>The tidysynthesis package</h3>

<p><a data-t="405">So what tidysynthesis allows us to do is actually specify a whole sequence of predictive</a> <a data-t="410">models for generating synthetic data. It also allows us to do a few things very specific</a> <a data-t="415">to data synthesis. One is it allows us to add additional noise to predicted values for</a> <a data-t="421">extra confidentiality protection. It allows us to specify constraints in the data, things</a> <a data-t="427">like individual income can't exceed family income or interest income can't exceed your</a> <a data-t="431">personal income. It also allows us to predict from our models in different ways, and I'll</a> <a data-t="437">talk about that more in a second. So simply put, tidysynthesis, the goal is all the power</a> <a data-t="442">of tidymodels for data synthesis concisely with a few special tools.</a></p>

<blockquote><a data-t="442">Simply put, tidysynthesis, the goal is all the power of tidymodels for data synthesis concisely with a few special tools.</a></blockquote>

<p><a data-t="449">So what does this actually look like? I'm going to show you how we're going to synthesize</a> <a data-t="452">a data set today. I'm not going to synthesize unemployment insurance data. To the relief</a> <a data-t="458">of many of you, I'm not going to synthesize tax data. We could imagine synthesizing electronic</a> <a data-t="463">health records, or maybe we're a business and we're interested in unlocking information</a> <a data-t="467">from our customer data without sacrificing the confidentiality of any individual in our</a> <a data-t="472">data. <p><a data-t="481">So we're going to synthesize the Palmer penguins</a> <a data-t="485">data set. We already had a good introduction to this earlier. <p><a data-t="502">So we can start asking ourselves some questions of what this should look like. If you have 200 variables,</a> <a data-t="507">what order should we use? Well, tidysynthesis has a function that employs different heuristics</a> <a data-t="511">that you can use to determine synthesis order. <p><a data-t="534">What kind of algorithm should we use for our predictive models? <p><a data-t="573">What types of feature or target engineering should we use? In this case, this is where recipes really shines</a> <a data-t="577">in tidysynthesis. <p><a data-t="592">Finally, how are we actually going to come up with synthetic data from our models?
