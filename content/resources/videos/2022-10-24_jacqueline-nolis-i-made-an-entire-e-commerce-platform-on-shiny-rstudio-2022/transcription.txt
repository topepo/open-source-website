<h3>Passing data from R to Shiny</h3>

<p><a data-t="252">But to make this work leads a lot of engineering questions, which I'm going to enumerate now.</a> <a data-t="256">One is how do you pass the plot from R to Shiny?</a> <a data-t="260">If you think about it, Shiny, there's a lot of times where Shiny has data loaded into</a> <a data-t="261">it, right?</a> <a data-t="263">You can read data from a database in Shiny.</a> <a data-t="265">You can press the upload button on Shiny.</a> <a data-t="269">But if you think about it, there's not really an obvious way that if a Shiny server is running,</a> <a data-t="272">how can you programmatically send data to it?</a></p>

<p><a data-t="276">So the way you would do that in, like, web development land is you do an HTTP post request.</a> <a data-t="286">So an HTTP post request is a way an API or a website receives data.</a> <a data-t="290">So when you hit submit on a website, your browser is doing a post request to the server.</a> <a data-t="295">That should be done with a post request.</a> <a data-t="298">Shiny does not have a documented ability to listen for post requests.</a> <a data-t="303">Normally only Plumber does that.</a> <a data-t="305">It should have this.</a> <a data-t="306">And that's my call out to RStudio.</a> <a data-t="308">I mean, pause it.</a> <a data-t="309">I think that would be a really great feature.</a></p>

<blockquote><a data-t="298">Shiny does not have a documented ability to listen for post requests.</a> <a data-t="303">Normally only Plumber does that.</a> <a data-t="305">It should have this.</a></blockquote>

<p><a data-t="315">The good news is there's actually a number of undocumented ways to do this.</a> <a data-t="318">And so, again, what I'm trying to do is I'm trying to post the image to Shiny.</a> <a data-t="320">So here's the plot I've made.</a> <a data-t="324">And then I want Shiny to come back to me with, okay, here's the ID of that image.</a> <a data-t="329">And then the package is going to be like, okay, cool.</a> <a data-t="333">I'm then going to take the user to the browser website to actually do the preview.</a> <a data-t="336">So this is a back and forth of the post request.</a></p>

<p><a data-t="338">So there are a number of undocumented ways to do this.</a> <a data-t="342">The way I like the most is you use the brochure package by Colin Fay.</a> <a data-t="345">This is an experimental package that lets you connect multiple Shiny apps together.</a> <a data-t="349">So if you have like three different Shiny apps and you're like, man,</a> <a data-t="351">I really wish all of them were running at once,</a> <a data-t="354">you can bundle them all up into one Shiny app using brochure.</a> <a data-t="357">You can also listen to requests besides get.</a> <a data-t="360">So if you want to have your Shiny app listen for data, not just show webpages,</a> <a data-t="363">brochure will let you do that too.</a></p>

<p><a data-t="365">So this actually solves two of my problems.</a> <a data-t="369">One, it lets me listen for post data.</a> <a data-t="372">And two, I can actually have a bunch of concurrent Shiny apps,</a> <a data-t="376">one for the order upload, one for the fulfillment, one for the preview.</a> <a data-t="380">It's so new, it doesn't even have a hex sticker.</a> <a data-t="381">It also says all over it, experimental, do not use in production.</a> <a data-t="385">But here I am using in production and it's working great.</a> <a data-t="388">Thanks, Colin.</a></p>

<h3>Passing the user from Shiny to Stripe</h3>

<p><a data-t="439">So, okay, cool.</a> <a data-t="439">So we have a Shiny app.</a> <a data-t="439">It shows you the preview of the thing.</a> <a data-t="441">It got the data from a post request from R.</a> <a data-t="443">But how do we then pass the user from Shiny to Stripe, right?</a> <a data-t="446">How do I make that pay and submit button send someone to Stripe?</a> <a data-t="449">I thought this would be the hardest part of the app,</a> <a data-t="452">but it actually was surprisingly easy to make this.</a></p>

<p><a data-t="455">There are really just two steps in the payment flow.</a> <a data-t="460">First is you need Shiny to send Stripe what is actually in the shopping cart of the user,</a> <a data-t="461">which is called the Stripe session.</a> <a data-t="463">You do this via a post request.</a> <a data-t="467">By the way, it turns out GGIRL, it's like 95% just post requests bouncing around,</a> <a data-t="469">and this is another one.</a></p>

<p><a data-t="475">You then also, once you've posted the cart,</a> <a data-t="479">you then have to have the user go to the Shiny page or the Stripe page.</a> <a data-t="484">So Stripe has a JavaScript API to get users to the appropriate Stripe checkout page.</a> <a data-t="488">I refuse to learn JavaScript.</a> <a data-t="490">I will never do this.</a></p>

<p><a data-t="496">The good news is that Shiny makes it very easy, I found out,</a> <a data-t="499">to use other people's JavaScript APIs.</a> <a data-t="503">So what you do is you add tags, $script to the top of your Shiny app</a> <a data-t="505">to load in the JavaScript API,</a> <a data-t="507">and then you Shiny.customMessageHandler</a> <a data-t="507">so that when the action button is pressed for pay and submit,</a> <a data-t="510">it will call that special JavaScript.</a> <a data-t="513">So it's really just two steps.</a></p>

<h3>Fulfilling the order with webhooks and <a href="/software/callr/">callr</a></h3>

<p><a data-t="531">How do you then fulfill the order?</a> <a data-t="534">So how do you have Shiny then listen for Stripe,</a> <a data-t="536">Stripe being like, okay, cool, they paid you money.</a> <a data-t="538">Now we're going to go ahead and make the order.</a> <a data-t="541">So we do this with a POST request.</a> <a data-t="543">So we're going to have Stripe send a POST request</a> <a data-t="545">telling Shiny to order the postcards.</a> <a data-t="549">So if your server is listening for someone else's POST request,</a> <a data-t="550">that is called a webhook.</a> <a data-t="552">It's just a different name for another POST request.</a> <a data-t="554">It's just kind of going in the other direction.</a></p>

<p><a data-t="559">So we're going to use a webhook saying, hey, order XYZ or FCE6,</a> <a data-t="561">whatever the order number is, has been paid for.</a> <a data-t="564">And to make sure that people just don't know what that URL is</a> <a data-t="566">and spam it to get a million postcards,</a> <a data-t="569">we are going to add keys to ensure that the message</a> <a data-t="572">actually came from Stripe itself.</a></p>

<p><a data-t="575">So Stripe will then, so Shiny will receive the request</a> <a data-t="577">and be like, okay, cool, the order is placed.</a> <a data-t="582">And then Shiny will call the stuff to send the postcard and send the email.</a> <a data-t="584">So, and this feels like it should be straightforward, right?</a> <a data-t="587">I've already done two or three POST requests as part of this talk.</a> <a data-t="589">This is just another POST request.</a></p>

<p><a data-t="591">But here's the issue,</a> <a data-t="596">is that Stripe needs to know that the message was received really quickly.</a> <a data-t="598">So if Stripe says the order was confirmed,</a> <a data-t="600">and Stripe doesn't hear right away that the order actually,</a> <a data-t="602">like, you're Apgil, like, cool, we got the order,</a> <a data-t="604">she'll be like, oh my god, they didn't hear it.</a> <a data-t="605">I got to send it again.</a> <a data-t="607">And here's the thing you might know about Shiny,</a> <a data-t="610">is Shiny takes a while when it's running stuff.</a> <a data-t="616">But if you're not careful,</a> <a data-t="620">Stripe will ask for the order to be submitted over and over,</a> <a data-t="622">and Shiny will be like, cool, got it, cool, got it, cool, got it.</a> <a data-t="627">And the next thing you know, you have 20 POST cards sent to your house.</a></p>

<blockquote><a data-t="620">Stripe will ask for the order to be submitted over and over,</a> <a data-t="622">and Shiny will be like, cool, got it, cool, got it, cool, got it.</a> <a data-t="627">And the next thing you know, you have 20 POST cards sent to your house.</a></blockquote>

<p><a data-t="630">So the trick to solve this is you're going to use callr package.</a> <a data-t="634">And callr's a really cool package to run our processes in the background.</a> <a data-t="637">So when you get the order from Stripe,</a> <a data-t="638">you're going to use callr to send,</a> <a data-t="641">like, actually send the POST card, send the email.</a> <a data-t="642">That's all going to happen in the background,</a> <a data-t="645">so that Shiny can immediately respond to Stripe being like,</a> <a data-t="647">yeah, cool, we got it, and not be bogged down</a> <a data-t="652">by the time it takes to submit the POST card order and send the email.</a></p>
