<h3>Calendar types</h3>

<p><a data-t="202">So that's about safety.</a> <a data-t="204">Let's talk about calendars.</a> <a data-t="207">Calendars are just the idea of a way to represent a unique point in time.</a> <a data-t="212">With our date in question, we could use a calendar called year, month, day to represent</a> <a data-t="218">this date using three components, the year, the month, and the day of the month.</a> <a data-t="223">But this isn't the only way you could represent this date.</a> <a data-t="226">You could also use the year and the day of the year.</a> <a data-t="230">Or you could use one of these many other calendar types that are built into Clock.</a> <a data-t="236">If you're a finance person, you might be particularly interested in year, quarter, day, which uses</a> <a data-t="241">a true fiscal year to represent your date.</a></p>

<p><a data-t="247">These are really nice because they're all convertible to each other.</a> <a data-t="250">You can work with any particular calendar type and say you need to get the quarter out,</a> <a data-t="254">you convert to year, quarter, day, you do manipulation over there, you convert back.</a> <a data-t="258">It's obviously convertible with date and POSIX-CT as well, since those are the date time types</a> <a data-t="264">that you're most likely to start out with.</a></p>

<p><a data-t="268">The other really neat thing that I find really fun about these calendar types is that they</a> <a data-t="272">have what's known as variable precision.</a> <a data-t="275">These are all day precision calendar types at this point, but we could narrow that down</a> <a data-t="281">to month precision as needed, and you've got a built-in year, month type in Clock.</a> <a data-t="287">Similarly, you could have a built-in year, quarter type.</a> <a data-t="290">You can actually go the other way, too.</a> <a data-t="292">You can widen it out all the way to nanoseconds if you need it.</a></p>

<p><a data-t="298">The last thing I'll say is that Clock is completely compatible with some of the other packages</a> <a data-t="303">you might be familiar with that I've created, called Slider and IVs.</a> <a data-t="308">Slider is one for rolling averages, so you can use Clock types as the index to say give</a> <a data-t="313">me a rolling average looking back four or five quarters.</a> <a data-t="317">IVs is a relatively new package, you might not have heard of this one yet, but it deals</a> <a data-t="322">with date ranges, and you can use Clock types as the components of those ranges.</a></p>

<p><a data-t="329">So to sum up, LuberDate is not going anywhere, don't worry, but please try Clock for enhanced</a> <a data-t="335">safety and these powerful new types.</a> <a data-t="337">Thank you.</a></p>
