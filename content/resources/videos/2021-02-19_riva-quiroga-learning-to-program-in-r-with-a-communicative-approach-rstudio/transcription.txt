<h3>Programming languages and second language learning</h3>

<h3>The communicative approach</h3>

<p><a data-t="156">Textbooks for learning a second language always start the same.</a> <a data-t="160">You learn how to say hello, how to introduce yourself, and some simple sentences to communicate</a> <a data-t="164">meaning to others.</a> <a data-t="166">Even books for learning dead languages have this approach.</a> <a data-t="169">I had to learn a little bit of Latin as an undergrad.</a> <a data-t="173">And the first thing we learned was how to ask someone who they are and how to respond</a> <a data-t="178">to that question.</a> <a data-t="179">I even learned how to insult someone in Latin, a language in which the last native speaker</a> <a data-t="186">died centuries ago.</a></p>

<p><a data-t="189">Textbooks start this way because they are built around people's needs and knowledge,</a> <a data-t="193">not around language rules.</a> <a data-t="195">In textbooks, you learn first what you can do with a language, not how the language works.</a> <a data-t="201">In language teaching, this is called a communicative approach, and is based on the idea that learning</a> <a data-t="207">a language successfully comes from having to communicate real meaning to real people.</a></p>

<blockquote><a data-t="201">In language teaching, this is called a communicative approach, and is based on the idea that learning</a> <a data-t="207">a language successfully comes from having to communicate real meaning to real people.</a></blockquote>

<p><a data-t="212">Throughout the different lessons in a textbook, you learn new words and structures, and new</a> <a data-t="217">contexts where to use those words and structures.</a> <a data-t="221">The grammar rules behind those real-life examples are only explained after you learn</a> <a data-t="225">how to use them, and if an old if is really necessary for you to know them.</a> <a data-t="231">In textbooks, scaffolding works as a spiral.</a> <a data-t="235">You learn something in one lesson, and then come back to that content in the next lesson,</a> <a data-t="240">but now showing more ways to use what you learn, how to adapt it to different situations,</a> <a data-t="245">and how to combine the words and structures you know in new ways.</a></p>

<p><a data-t="251">It's like copy and pasting, and then adapting.</a> <a data-t="254">You learn how to use a new language structure, like how to ask a question, and then you learn</a> <a data-t="259">how to adapt it to your own needs, to solve the problems you are interested in.</a> <a data-t="265">It's this ability to adapt language structures to new situations what makes us flexible users,</a> <a data-t="272">in both natural languages and programming ones.</a></p>

<p><a data-t="275">For example, we first learn how to create one type of plot using all the default settings,</a> <a data-t="281">and then we come back to that content to add more complexity to the visualization, like</a> <a data-t="286">mapping more variables, using different geoms, or making annotations.</a> <a data-t="291">And we build those new skills over the ones we learned previously.</a> <a data-t="295">This copy, pasting, and adapting is also the strategy we use with examples we find in Stack</a> <a data-t="300">Overflow or </a><a href="/software/rstudio/">RStudio</a><a data-t="300"> community.</a> <a data-t="303">We bring to our script something that worked in another context, and then try to figure</a> <a data-t="307">out how to adapt it to our own needs.</a></p>

<p><a data-t="311">Sometimes we are not even sure why it worked, but after doing it a couple of times, we start</a> <a data-t="315">inferring what is going on behind the scenes.</a> <a data-t="319">You don't need to know all the grammar rules to be understood in a language, and you don't</a> <a data-t="324">need to know the name of all its structures to be a competent user.</a> <a data-t="329">Even native speakers don't know the name of all the parts.</a> <a data-t="332">You probably don't know what a paratactic or a hypotactic clause complex is, but you</a> <a data-t="338">definitely know how to use one.</a></p>

<p><a data-t="341">If you are a native English speaker, you know how to correctly pronounce every case of two</a> <a data-t="345">O's together in words like moon, book, floor, flood, something very, very difficult for</a> <a data-t="351">us not-native speakers.</a> <a data-t="354">But you probably don't know why they are pronounced differently.</a> <a data-t="357">The adults who raise you didn't write you a grammar book for you to acquire your native</a> <a data-t="361">language.</a> <a data-t="362">They didn't teach you the grammar rules.</a> <a data-t="366">They talked to you and let you talk.</a> <a data-t="368">They probably read you books.</a> <a data-t="370">You sang or sang together.</a> <a data-t="372">You did things with words progressively more complex.</a></p>

<p><a data-t="376">In the same way, you don't need to know all the rules of a programming language to be</a> <a data-t="380">able to do things with it.</a> <a data-t="383">What rules do you need?</a> <a data-t="384">The ones that can help you achieve the purposes you're seeking.</a> <a data-t="388">If you need to customize a plot in a very precise way because the default options are</a> <a data-t="392">not what you need, then learning how a theme works in ggplot is relevant, not before.</a> <a data-t="399">If you need to work with arrays, then learning what they are and what kind of things you</a> <a data-t="404">can do with them is relevant, not before.</a></p>

<h3>Learner personas and modern R resources</h3>

<p><a data-t="408">One more thing about textbooks.</a> <a data-t="409">They usually have learner personas.</a> <a data-t="412">If you're learning a language because you want to travel, the first chapters will probably</a> <a data-t="416">show you how to get along in the airport and ask for directions in a city.</a> <a data-t="422">If you're learning the language because you're going to study abroad, the first chapters</a> <a data-t="426">will probably be about students trying to find something in the campus or dealing with</a> <a data-t="430">university bureaucracy.</a> <a data-t="433">The same happens with programming learning resources.</a></p>

<p><a data-t="436">Fortunately, in the past couple of years, a lot of online resources have appeared to</a> <a data-t="440">learn R and other programming languages, and many of them are focused on the needs</a> <a data-t="445">of specific users.</a> <a data-t="446">So all the data and examples are connected with what those specific users need and know.</a> <a data-t="452">So taking into account the community approach when teaching programming language involves</a> <a data-t="457">doing real things from day or page one, building complexity as a spiral, revealing rules after</a> <a data-t="464">you know how and when to use them, and only the ones that can help you achieve what you're</a> <a data-t="468">looking for.</a> <a data-t="470">And taking into account who your learner personas are.</a></p>

<p><a data-t="475">In the last couple of years, a lot of R resources have been created taking into account these</a> <a data-t="479">principles.</a> <a data-t="481">Learning resources that let you experiment first and then build explanations.</a> <a data-t="486">Books that show you the whole game first and then explain the details behind every step.</a> <a data-t="492">Publications that had in mind different learner personas with different backgrounds that show</a> <a data-t="497">you how to do relevant things with relevant data for your field.</a></p>

<p><a data-t="502">But hey, what about the other books?</a> <a data-t="504">The ones that explain how R works?</a> <a data-t="506">Well, those actually are very cool books.</a> <a data-t="509">I'm a linguist myself, so I really love grammar books.</a> <a data-t="512">But for both natural and programming languages, they're not the first step to approach a language</a> <a data-t="516">when what you want is to learn how to use it and how to communicate with other people.</a></p>

<h3>Package development: documentation and vignettes</h3>

<p><a data-t="711">So if you're a package developer and want people to be happy users with your package,</a> <a data-t="716">you can think about how your error messages can give guidance.</a> <a data-t="720">But error messages are not the only part of a package that can adopt a communicative approach.</a> <a data-t="726">Let's take a look at documentation.</a> <a data-t="729">Function documentation is great.</a> <a data-t="731">Sometimes you forget the name of an argument and taking the help page can allow you to</a> <a data-t="735">solve this kind of issue.</a> <a data-t="737">But to use it, you need to know the actual name of the function you're after.</a> <a data-t="741">Otherwise it's kind of useless.</a></p>

<p><a data-t="744">What if you know the problem you want to solve, but not the name of the function that can</a> <a data-t="748">help you solve it?</a> <a data-t="751">Function documentation are like dictionaries.</a> <a data-t="754">They have the definition of the words you can use in a programming language.</a> <a data-t="758">But we don't learn a language by memorizing words from the dictionary.</a> <a data-t="762">We need to learn how to use the words, in what context they are useful, what kind of</a> <a data-t="767">meaning can we construct with them?</a> <a data-t="770">What if you don't know the name of the function you need?</a> <a data-t="773">What if you know what you want to achieve, but not what functions can be useful for that?</a> <a data-t="778">Well, here comes vignettes.</a></p>

<p><a data-t="781">Vignettes are an awesome resource to describe the problem that your package is designed</a> <a data-t="785">to solve and to show people how to solve it.</a> <a data-t="789">They are a great resource to show how to do things with your package with real example.</a> <a data-t="795">And they are easy to find using a search engine.</a> <a data-t="797">So they are also a great way to promote your package.</a> <a data-t="801">In a vignette, you can explain with details the different categories your functions are</a> <a data-t="805">divided, how you can coordinate multiple functions to solve problems, and show extended examples</a> <a data-t="812">that people can copy, paste, and adapt to meet their own needs.</a> <a data-t="817">The best part is that vignettes are easy to create using the uses package.</a> <a data-t="822">With uses-use-vignettes, you can create one.</a></p>

<p><a data-t="826">So thinking of error messages as a way to give feedback to the users of your package</a> <a data-t="831">and writing vignettes that explain how to do things with the functions you create are</a> <a data-t="835">great ways to think about package development from a communicative approach.</a></p>

<h3>Closing remarks</h3>

<p><a data-t="841">Two closing remarks.</a> <a data-t="842">Is RubyR the best programming language to learn?</a> <a data-t="847">Isn't Python better?</a> <a data-t="848">Or what about Julia?</a> <a data-t="850">With programming languages, it happens the same as with natural languages.</a> <a data-t="854">There is no one intrinsically better than the other.</a> <a data-t="857">The best programming language is the one that can help you solve more effectively the</a> <a data-t="861">problem you are trying to solve in the context you currently are.</a> <a data-t="865">Sometimes, things like processing time or number of lines of code are important to take</a> <a data-t="871">into account.</a> <a data-t="872">But sometimes they are not.</a> <a data-t="874">And only what is easier or faster for you to write, not the computer to process, is</a> <a data-t="879">more important.</a> <a data-t="881">And sometimes it's more relevant what programming language your audience can understand.</a></p>

<p><a data-t="886">The same happens with natural languages.</a> <a data-t="889">Right now, I would really prefer to be speaking in Spanish, but it's definitely more effective</a> <a data-t="894">if I speak in English.</a> <a data-t="897">Why?</a> <a data-t="897">Not because of any characteristic of English language itself.</a> <a data-t="901">Actually, for this subject in particular, Spanish is way more precise, because we have</a> <a data-t="906">more words to speak about language.</a> <a data-t="908">We make distinctions like lengua, lenguaje, idioma, all of which have different and precise</a> <a data-t="914">meanings.</a> <a data-t="915">So I need to use more words in English to explain those concepts.</a> <a data-t="919">But right now, being able to convey my message to a broader audience is more important than</a> <a data-t="924">the lexical variety or precision that Spanish offers to this subject.</a></p>

<p><a data-t="928">So which programming language to learn?</a> <a data-t="931">The one that can help you best solve the problems you are interested in, and that allows you</a> <a data-t="935">to speak with the people you are interested in communicating with at that moment.</a> <a data-t="941">And with natural language, knowing, the same way as with natural languages, knowing more</a> <a data-t="947">than one allows you to communicate with more people and do more things with words.</a></p>
