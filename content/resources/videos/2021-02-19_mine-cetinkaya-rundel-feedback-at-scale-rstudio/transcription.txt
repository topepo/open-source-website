<p><a data-t="50">As enrolments in statistics and data science courses grow, and as these courses become more and more computational, educators are faced with an interesting challenge, providing timely and meaningful feedback, particularly with online delivery of courses.</a> <a data-t="65">I'm sure we all agree that feedback should be meaningful, but traditionally, meaningful, helpful, constructive feedback requires human effort and can take a significant amount of time for large courses,</a> <a data-t="77">especially if they're under-resourced, which, let's face it, they tend to be.</a> <a data-t="82">Timeliness of feedback is just as important as meaningfulness.</a> <a data-t="88">The longer time passes between when a student turns in an assignment and when they get feedback on it, the lower the utility of that feedback.</a></p>

<p><a data-t="94">For certain assignments, like an open-ended project, this trade-off is absolutely reasonable,</a> <a data-t="100">because it's really difficult, if not impossible, to replace human feedback with something else in such an assignment.</a> <a data-t="107">But for others, there are alternatives.</a></p>

<h3>Introducing <a href="/software/learnr/">learnr</a> tutorials</h3>

<p><a data-t="109">One such option is a learnr tutorial, and if you've ever used learnr before, the tutorial probably looked a little bit different than this,</a> <a data-t="118">and that is just to say that with a little bit of CSS and theming, you can make your learnr tutorials look like whatever you like.</a> <a data-t="126">I like starting my learnr tutorials with a bit of narrative, usually a little bit longer than this about the data and the analysis we're going to work through and introduce the students to the packages,</a> <a data-t="136">and I like using the progressive reveal option, so they have to deliberately click through the material.</a> <a data-t="142">I also like providing some ready-to-run code at the beginning, so they don't need to start working on exercises right away, but they do need to start interacting with the document.</a></p>

<p><a data-t="150">Here they're working on a dataset on Airbnb listings in Edinburgh, and the first thing we're asking them to do is to take a look at the variable names.</a> <a data-t="160">It is not necessarily meaningful interactivity, but maybe it introduces them a little bit to the structure of a learnr tutorial and how to interact with it,</a> <a data-t="170">and potentially also helps build a little bit of anticipation around what the result is going to look like when they hit run code,</a> <a data-t="174">as opposed to providing them with a static document where the code output is already provided.</a> <a data-t="180">But really where learnr tutorial shines are these coding exercises, where the students can try out some code, submit their answer, and you can provide custom feedback to them.</a></p>

<p><a data-t="190">So the first exercise here says how many Airbnb listings are included in this dataset.</a> <a data-t="196">Let's imagine the student decided to look at the number of columns as opposed to the number of rows.</a> <a data-t="201">The feedback that they're going to get sounds just like the type of thing I would say to them if I was working through the exercises with them and trying to nudge them in the right direction.</a> <a data-t="210">It says, did you calculate the number of columns instead of number of rows?</a> <a data-t="215">So you can write these very precise feedback based on your anticipation about the types of mistakes students can make.</a></p>

<p><a data-t="223">And obviously, it's not possible to anticipate all possible mistakes.</a> <a data-t="227">So what if a student chooses to try out some code that I didn't anticipate they might try out?</a> <a data-t="233">For that, I like providing some canned feedback that's still constructive and still nudges them in the right direction.</a> <a data-t="237">It says something like, each observation is represented in one row. Can you remember which function we used to calculate the number of rows?</a> <a data-t="248">And finally, let's get this question correct.</a> <a data-t="252">I'm going to use the nrow function to do that.</a> <a data-t="257">Let's run our code and submit our answer.</a> <a data-t="260">You'll see that we've got the green banner, which is great, but it doesn't just say correct, well done.</a> <a data-t="267">It says a little bit more.</a> <a data-t="268">So I like using the space for the message for the correct answers to give them a little bit more information about what's to come next.</a></p>

<p><a data-t="275">So I also like following on my coding exercises with some multiple choice exercises, something that allows the students to take the code output that they saw and put it in the context of the data set that they're working with,</a> <a data-t="289">which is so important for teaching statistics and data science.</a> <a data-t="294">So what does each row in the eddybnb data set represent? That's an individual Airbnb listing.</a> <a data-t="299">And again, we can use the feedback to give them a little bit more information or something to think about.</a></p>

<p><a data-t="305">If you would like to see the code for this learnr tutorial, both in terms of creating the exercises and also the theme and the look, you can test drive it on RStudio Cloud or you can view the code on GitHub.</a> <a data-t="318">And I'll be providing links to all of this at the end of the talk.</a> <a data-t="322">For now, I'd like to talk a little bit about writing effective exercises.</a></p>

<h3>Writing effective exercises</h3>

<p><a data-t="324">So let's start with an exercise in mind.</a> <a data-t="328">The three most expensive neighborhoods in terms of mean nightly price are Newtown, Old Town, and West End.</a> <a data-t="333">Calculate the median number of reviews in these neighborhoods and arrange them in descending order.</a> <a data-t="339">One option would be to provide no scaffolding whatsoever, so an entirely empty canvas for students to work with.</a> <a data-t="345">And the thing about this is we've really not given them any direction in terms of how to get started.</a> <a data-t="350">And instead, what we've given them is this giant button that says solution.</a> <a data-t="355">And it's so tempting to simply go, all right, let me take a look at the solution, copy it to my clipboard, paste it, run my code, and submit my answer.</a> <a data-t="366">And lovely job, I got it right. But did I learn anything from this experience? Not really.</a></p>

<p><a data-t="371">This is not to say your students are all going to peek into the solution code.</a> <a data-t="375">But I think one thing to keep in mind about providing no scaffolding whatsoever is that I think there are better venues for giving such exercises.</a> <a data-t="386">So for assignments where the students already have to work in the RStudio IDE, either in an R Markdown document or in an R script, I think is a better venue for a more open-ended question where you provide no scaffolding whatsoever.</a></p>

<p><a data-t="398">Here we've given them no scaffolding, and we've also done something else where we've basically are doing this strict code-checking business.</a> <a data-t="411">So let's say that your student, with no scaffolding provided to them, actually decided to move the filter over here.</a> <a data-t="417">They group by and summarize first, and then they filter.</a> <a data-t="422">And if they run their code, they're going to get the exact same response, but their code is not marked as correct.</a> <a data-t="428">Because with the grade this code function, what is happening under the hood is that the </a><a href="/software/gradethis/">gradethis</a><a data-t="428"> package is checking the code your student wrote and the solution code you provided and looks for a one-to-one match.</a> <a data-t="445">So it says that I expected you to call summarize where you called filter, give it another try.</a> <a data-t="451">Sure, you might say that no, this is exactly how I want my students to do things, but I think that's not really entirely fair, especially because they actually got the right answer.</a> <a data-t="461">So the strict code-checking with no scaffolding whatsoever can lead to situations where students get the right answer but are not necessarily marked correct, if you will.</a></p>

<blockquote><a data-t="461">So the strict code-checking with no scaffolding whatsoever can lead to situations where students get the right answer but are not necessarily marked correct, if you will.</a></blockquote>

<p><a data-t="475">Another option would be to provide a little bit of scaffolding, but here we're still doing the strict check.</a> <a data-t="480">So here we've given them a really nice structure for how to write their code, and obviously we've made the exercise a lot easier for them,</a> <a data-t="488">but that might be okay if this is one of the first few times they're seeing these multi-line </a><a href="/software/dplyr/">dplyr</a><a data-t="488"> chains.</a> <a data-t="498">So these learnr tutorials are a great place to provide additional opportunities for kind of drill-type exercises students can work through at the beginning when they are first introduced to a new topic.</a> <a data-t="511">And you can also provide hints for them.</a> <a data-t="514">So with these hints, let's take a look at that.</a> <a data-t="517">I like writing them in a progressive way where they can scroll through and get a chance to potentially think about what their answer should look like at each stage.</a> <a data-t="528">And sure, they could at the end see the solution if you want them to, but at least it makes them think along the way as well.</a></p>

<p><a data-t="533">So let's go ahead and run this code and submit the answer.</a> <a data-t="539">I think in this structure there is no worry that they would be putting their filter function later on because we had already provided the scaffolding.</a> <a data-t="549">However, since we're doing a strict check, what if the student in the last line forgot to actually put things in descending order?</a> <a data-t="558">So let's go ahead and run that code, and obviously this is not the correct answer, but the feedback that we get tends to be not very informative when we use the grade this code function here.</a> <a data-t="569">So it's basically saying, I expected you to call desk for descending where you wrote median rating.</a> <a data-t="576">And in fact, here on line five, that statement is correct.</a> <a data-t="580">But does it actually sound like the type of thing you would say to your student if you are standing right behind them and trying to nudge them in the right direction? Not really.</a> <a data-t="589">It doesn't seem like very human-friendly feedback.</a></p>

<h3>Checking results instead of code</h3>

<p><a data-t="592">So my recommendation overall would be to provide some scaffolding and also provide some flexibility.</a> <a data-t="598">This is another approach to scaffolding where perhaps if this tutorial is coming later on, if they've already done a bunch with dplyr and you don't want to give them that whole structure,</a> <a data-t="608">you might at least give them a little bit of scaffolding to say, hey, I want you to use a dplyr pipeline for answering this question.</a> <a data-t="616">And let's go ahead and give that a try.</a> <a data-t="617">So here I am going to forget to put things in descending order and run my code.</a> <a data-t="622">And let's take a look at the feedback that I get.</a> <a data-t="628">I actually am getting human-friendly feedback here.</a> <a data-t="630">You've successfully calculated the median number of ratings, but did you forget to arrange them in descending order?</a></p>

<p><a data-t="639">How did I do this? Here, instead of checking their code, we're actually checking their results.</a> <a data-t="645">And one nice thing about doing that is, so let's go back and put the right answer here.</a> <a data-t="651">What if they made that one change where they were doing group by and summarize and then filter to have them run the code and submit their answer?</a> <a data-t="657">And in fact, this time it is marked as correct because again, we're checking the results and not the code.</a></p>

<p><a data-t="662">So what does this look like under the hood? It's a bit more to write the code checking if you're using the grade this function,</a> <a data-t="669">because you actually need to think about what are the various answers they might get and actually write some passive or fail if type statements.</a> <a data-t="681">So here I have one passive statement that says if their result is this table in this particular order, we're going to say, yep, you did the right thing.</a> <a data-t="691">But if they have put them in ascending order, then we're going to give them a specific message for that.</a> <a data-t="697">Or if they have not ordered them at all, in which case the order will be the alphabetical order of the neighborhood, we're going to give them another message.</a> <a data-t="706">And you can imagine stringing along as many of these as you think are relevant for the exercises you develop.</a> <a data-t="711">And then probably you want to write a catch-all statement that's like, not quite, take a peek at the hints.</a> <a data-t="716">If the students are making mistakes that you can't even anticipate when you write the tests, you can kind of leave the hints to take care of that.</a></p>

<p><a data-t="721">And if you use these tutorials over multiple semesters or multiple workshops, multiple learning or teaching engagements, you're probably going to learn about the various mistakes students make based on their questions.</a> <a data-t="731">And then you might iteratively build these tests to capture more mistakes that they might make.</a></p>

<h3>Collecting student data</h3>

<p><a data-t="745">So we've talked about writing these exercises. What about collecting data?</a> <a data-t="751">learnr and gradethis alone don't have some facility to allow you to collect data from students.</a> <a data-t="757">Specifically, the learnr package makes this possible, but it doesn't itself offer the functionality to do so.</a> <a data-t="763">So for that, I use a different package called LearnRHash, which hashes the student results and then allows you as the instructor to decode them.</a> <a data-t="773">So a setting where I might use this is probably similar to this, where I give them a coding exercise first.</a> <a data-t="779">It says make a histogram of the prices.</a> <a data-t="782">So let's go ahead and actually do that.</a> <a data-t="784">The data set is adbnb. The variable is price. We're going to have them make a histogram.</a></p>

<p><a data-t="807">And here are the labels x, y, and title.</a> <a data-t="827">They go ahead and run their code and they get some warnings and submit their answer.</a> <a data-t="833">It says you got it right.</a> <a data-t="835">And note that there are a couple of warnings. We'll get to those in a little bit because here our focus is a little bit different.</a> <a data-t="843">So we follow this on with some multiple choice questions.</a> <a data-t="844">Which of the following describes the shape of the distribution? It's right skewed and unimodal.</a> <a data-t="850">And let's go ahead and pick another answer for the second question. This one happens to be incorrect.</a></p>

<p><a data-t="856">Okay, so when it comes time to submitting their answers with the LearnRHash package, students can generate a hash, which is going to look very cryptic to them, which is exactly what it's meant to do.</a> <a data-t="866">They can select it and copy it.</a> <a data-t="868">And then I like embedding directly into the learnr tutorial something like a Google form or another form your institution might use, where the students can submit their information.</a> <a data-t="879">If you're using something that directly your university uses, for example, this form could ask them to authenticate first.</a> <a data-t="885">If you want to make sure that who is answering the questions is actually who you think it is.</a> <a data-t="891">And you can ask them to paste their hash that they generated here.</a> <a data-t="894">And I also like using this form to collect some free form information from the students as well.</a> <a data-t="900">So a question phrased as, write about one or two questions you didn't get right initially, but were able to solve after a few tries.</a> <a data-t="906">What was difficult about them? What did you ultimately learn?</a> <a data-t="909">And let the students write some free form answers and submit their results.</a></p>

<p><a data-t="915">On your end, what you're going to see is this hash that the students have submitted.</a> <a data-t="920">And you'll be able to decode that and be able to take a look at their multiple choice questions and the answers they gave for those.</a> <a data-t="928">And the exercises, that's the coding exercises, and the answers they gave for those as well.</a> <a data-t="933">For summative feedback for grading, I usually tend to mark the multiple choice questions, but you could choose to do both.</a></p>

<p><a data-t="937">These data are collected in a Google spreadsheet, so you can view the Google spreadsheet itself.</a> <a data-t="942">I've pre-populated it with some mock student data.</a> <a data-t="945">And what you can do is using the Google Sheets 4 package, get that data and calculate the students' scores by matching their answers to a key.</a> <a data-t="954">And then also take a look at the free form answers.</a> <a data-t="957">I like taking a look at bigrams, for example, to see what are some concepts that were mentioned many times as being difficult for the students, so you might tailor your next lesson accordingly.</a></p>

<h3>Distributing tutorials at scale</h3>

<p><a data-t="965">Lastly, let's talk about distributing these at scale.</a> <a data-t="969">One option, obviously, is to deploy these to something like ShinyAppsIO or another </a><a href="/software/shiny-r/">Shiny</a><a data-t="969"> server from the RStudio IDE with push button deploy, just clicking on publish.</a> <a data-t="979">It works out of the box. It's great.</a> <a data-t="981">But something you might want to be careful about is that if you have a large class and a strict deadline, many students might try to access the tutorial at the same time.</a> <a data-t="991">So you've got to make sure that the parameters for your deployment are set up properly in order to be able to handle that.</a> <a data-t="996">You might want to give it more memory. You might want to create more instances.</a> <a data-t="999">And at some point, if you have a really large course and you expect students to be doing things at the same time, you may need to think about using one of the paid tiers of ShinyAppsIO, and that may or may not be feasible for your setting.</a></p>

<p><a data-t="1015">So another option is to distribute them within a package.</a> <a data-t="1017">Here I'm going to give an example from the </a><a href="/software/dsbox/">dsbox</a><a data-t="1017"> package, which accompanies the data science course in a box curriculum.</a> <a data-t="1026">And yes, I am saying that you can make an R package, and you might be new to doing that, so you can use this as a skeleton for doing so.</a> <a data-t="1036">But the nice thing is really what you want to focus on is that the tutorials go in the inst folder in a subfolder called tutorials of the package.</a> <a data-t="1046">Inside that is a single folder for each one of the tutorials.</a> <a data-t="1049">This package happens to have eight of them, and inside that is the R markdown file where you developed your tutorial and any accompanying files that go along with it.</a> <a data-t="1058">Chances are you may not want to put this tutorial on CRAN, maybe it's not for wide distribution, but you can ask your students to install it directly from GitHub,</a> <a data-t="1067">or if you're using something like RStudio Cloud, you can ask them to, you can pre-install it for them.</a> <a data-t="1073">And within the RStudio IDE, they will be ready to be launched soon as the students start that RStudio session.</a> <a data-t="1081">I usually tell my students once you launch the tutorial, just make that tutorial pane, maximize it, and it looks just like they're interacting with it within the web browser.</a> <a data-t="1091">But now you're not relying on a service like </a><a href="/software/shinyapps/">ShinyApps</a><a data-t="1091">.io to be hosting your tutorial, so it really doesn't matter how many students are using it at the same time, as long as your RStudio session allows that.</a></p>

<p><a data-t="1104">The barrier to entry for getting started with learnr is pretty low, especially if you're already an R markdown user.</a> <a data-t="1110">In fact, the tutorials in the dsbox package were co-developed with students with only one semester of introductory data science under their belt, so when I say it has a low entry point, I really mean it.</a> <a data-t="1120">We all know that students get better at coding or whatever they're learning with practice.</a> <a data-t="1126">We also probably mostly agree that formative feedback that nudges learners in the right direction while they're working on the exercises, as opposed to summative feedback that comes weeks after they're done, can have positive effects on their learning and enjoyment.</a> <a data-t="1143">Plus, providing individual feedback to students, especially for exercises narrow in scope, can turn into dreadful busy work for us educators.</a></p>

<p><a data-t="1150">So while developing meaningful automated feedback can be quite tedious and time-consuming, I think that's a much more intellectually engaging way to spend your time as an instructor.</a> <a data-t="1162">It's also a great opportunity to develop software engineering skills, like writing robust tests, bringing together your pedagogical and technical expertise.</a> <a data-t="1173">You can find materials for this talk, including the source code for the learnr tutorial we worked through, and links to resources to learn more at rstd.io slash global 2021 slash mcr.</a></p>
