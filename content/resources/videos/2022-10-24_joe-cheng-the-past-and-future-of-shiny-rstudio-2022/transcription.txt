<h3>The birth of Shiny</h3>

<p><a data-t="120">July 31, 2012 was the day that we first showed Shiny to the world.</a> <a data-t="120">We were at JSM, not even a scheduled talk.</a> <a data-t="125">Brian Lewis just lent us ten minutes at the end of his talk, and we showed Shiny.</a> <a data-t="134">And if you consider that date, July 31, to be the birth of Shiny, then this is a particularly special talk for me to give to you today.</a> <a data-t="151">Because that means that Shiny is ten years old.</a></p>

<p><a data-t="169">Ten years. <p><a data-t="188">And the story of how Shiny started really begins with Danny Kaplan.</a> <a data-t="193">Danny and his students were the very first beta testers of </a><a href="/software/rstudio/">RStudio</a><a data-t="193"> IDE.</a> <a data-t="211">And Danny's feedback was so great, his fingerprints are all over the IDE even to this day.</a> <a data-t="231">You can see all the parts that he influenced with his feedback.</a> <a data-t="236">And in early 2011, Danny asked JJ for a way to build interactive apps in R that he could use to demonstrate statistical and calculus concepts.</a> <a data-t="255">And JJ being JJ, in about a week, he built this.</a> <a data-t="265">This is the manipulate feature in RStudio.</a> <a data-t="265">Sort of the proto-Shiny, if you will.</a></p>

<p><a data-t="273">And manipulate is great because it almost couldn't be simpler.</a> <a data-t="283">You give it some kind of plot-generating function or plot-generating expression and include one or more unbound variables that you want the user to be able to control.</a> <a data-t="299">Then for each of those, you describe how you want that to appear.</a> <a data-t="305">So there are a couple of, there's slider and there's lookup, a couple of things like that.</a> <a data-t="312">And this worked great and Danny ran with it and created quite a few really beautifully thought out and assembled tiny applets that illustrated these concepts.</a> <a data-t="317">And he bundled them together as part of this project, Mosaic.</a></p>

<p><a data-t="324">Now, it wasn't long, as tends to happen with software, it wasn't long before some questions started.</a> <a data-t="330">Boy, it sure would be nice to have a customizable layout.</a> <a data-t="334">Can I have those inputs show up in two columns, for example?</a> <a data-t="336">Could I have more input types?</a> <a data-t="336">Maybe multiple outputs at once instead of just one plot.</a> <a data-t="341">Maybe different kinds of outputs like tables.</a> <a data-t="341">And maybe instead of just showing things, we could also take some actions when the user clicked a button or something like that.</a> <a data-t="353">And finally, why do people have to be using RStudio IDE to use these applets?</a> <a data-t="358">It would be great to be able to take these little interactive things and move them out.</a> <a data-t="365">All of those totally reasonable. <p><a data-t="373">Now, that was something that JJ and I clearly had the technical ability to create.</a> <a data-t="380">But I felt really strongly this is not something that we should do.</a> <a data-t="387">This felt like a trap to me.</a> <a data-t="387">Because UI programming sucks.</a> <a data-t="394">This is how I honestly felt in 2012.</a> <a data-t="394">I had spent a lot of my career writing complex UIs.</a> <a data-t="402">And the reason it sucks is because of this idiom called event handlers.</a> <a data-t="411">This is the way all UI worked in those days.</a></p>

<p><a data-t="411">The way event handlers work is so simple.</a> <a data-t="418">I can explain it to you right now.</a> <a data-t="418">The idea is this. <p><a data-t="487">It can be done. <p><a data-t="533">And the problem was, as much as I hated event handlers, I couldn't come up with not only a better alternative, but any alternative.</a> <a data-t="539">I had been doing this kind of programming day in, day out for years.</a> <a data-t="544">Constantly thinking there must be a better way than this.</a> <a data-t="549">And I could not even conceive of a better way.</a> <a data-t="560">Until around May of 2012.</a> <a data-t="560">There's a lot of dates in this talk.</a></p>
