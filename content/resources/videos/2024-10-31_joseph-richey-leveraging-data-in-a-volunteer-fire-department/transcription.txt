<h3>Volunteer fire departments and data challenges</h3>

<p><a data-t="102">The majority of fire departments in the United States are like mine,</a> <a data-t="105">volunteer-based organizations.</a> <a data-t="105">The last estimate put almost 70 percent of fire departments staffed</a> <a data-t="112">completely by volunteers.</a> <a data-t="112">Volunteer fire departments are really, really important</a> <a data-t="117">because they protect a section of the country that full-time fire departments just can't.</a> <a data-t="122">It doesn't make sense to put a full-time fire department in my town of 800 people</a> <a data-t="126">like it would in a town of 8,000 people.</a> <a data-t="132">Because we're volunteers, this is not what we do for a</a> <a data-t="132">living.</a> <a data-t="137">We have day jobs, we have other responsibilities.</a> <a data-t="137">This is what we do in our</a> <a data-t="137">evenings, on weekends, and what little free time we have.</a> <a data-t="143">But just because we're volunteers doesn't</a> <a data-t="143">mean we train any less hard than the full-time firefighters.</a> <a data-t="148">We're certified to the same standard</a> <a data-t="149">and we do a lot of work to make sure that we are certified.</a></p>

<p><a data-t="156">As a volunteer fire department, we often don't have the resources that a full-time fire department has.</a> <a data-t="163">This becomes especially true when you start to look at data.</a></p>

<p><a data-t="214">So where did we start?</a> <a data-t="214">When I joined the</a> <a data-t="214">Carin Fire Department a little over a year ago, I walked into my very first Tuesday night drill.</a> <a data-t="221">We drill for two hours a week in the evenings to practice anything from how to manage a cardiac</a> <a data-t="225">arrest, to how to extricate a patient from a vehicle, or how to run some of the pumps on our</a> <a data-t="231">brush trucks.</a> <a data-t="231">When I showed up, there was a piece of paper on a clipboard and I wrote my name down.</a> <a data-t="238">This is how we used to track attendance.</a> <a data-t="238">Now if we were really, really lucky, somebody would</a> <a data-t="243">remember to write down the date of that training and what we were training that night.</a> <a data-t="248">That paper</a> <a data-t="248">would then get stuck in a filing cabinet that might or might not have looked like this, or it</a> <a data-t="252">would get shoved in the back office in a stack of papers, or it would end up who knows where.</a></p>

<p><a data-t="259">Additionally, when we'd come back from one of our calls or one of our incidents,</a> <a data-t="263">we'd get back to the station and go to a whiteboard that's hung on the wall next to our</a> <a data-t="267">ambulance.</a> <a data-t="267">On this whiteboard, we'd write the date, what we responded to, and who was at the call.</a> <a data-t="276">This would then hopefully get taken a picture of on someone's phone, it would then get texted to</a> <a data-t="281">one of the captains, and all of this data would then get entered very manually into a spreadsheet</a> <a data-t="287">that is still actually only on one laptop, and that really makes me nervous.</a> <a data-t="293">This took a lot of time, especially from this one captain that had kind of stood up to be our data</a> <a data-t="298">guy.</a> <a data-t="298">Again, a lot of time unpaid as a volunteer.</a></p>

<p><a data-t="306">Now we didn't just track this data because it was</a> <a data-t="306">just so much fun, we tracked it because we needed it.</a> <a data-t="313">As a volunteer in the state of Utah, in order</a> <a data-t="313">to respond to structure fires, wildland fires, and EMS calls and be minimally certified, it takes 64</a> <a data-t="321">hours of continuing education every year.</a> <a data-t="321">That's a lot of time, and these Tuesday evening drills we</a> <a data-t="328">have go a long way to making sure we can meet those requirements.</a> <a data-t="335">Additionally, as we plan</a> <a data-t="335">for our little community, it's really important to know how many times our services are needed.</a></p>

<h3>Designing a solution for firefighters</h3>

<p><a data-t="381">As data analysts, data scientists,</a> <a data-t="381">and whatever data person you might identify as, our job is to solve problems at its root.</a> <a data-t="388">I proposed that by decreasing repetitive work, making data entry user-friendly,</a> <a data-t="394">and leveraging the data, we could solve this problem.</a> <a data-t="401">I had to start with a question that</a> <a data-t="401">I actually thought quite a bit about the first few weeks as I began to think on this.</a> <a data-t="405">How do you get</a> <a data-t="405">firefighters to record data?</a> <a data-t="411">Firefighters and data scientists, as you might guess, come from</a> <a data-t="411">different worlds.</a> <a data-t="417">You might call a firefighter to put out this fire, and you might call a data</a> <a data-t="417">scientist to put out these fires.</a></p>

<p><a data-t="426">In order to get firefighters to record data, I knew there</a> <a data-t="426">was a couple principles that I would have to apply.</a> <a data-t="432">First, there had to be the minimum input.</a> <a data-t="432">Everybody loves sitting down and spending 15, 20, 30 minutes entering data, right?</a> <a data-t="438">Yeah.</a> <a data-t="438">If this was going to take any amount of time, it wasn't going to work.</a> <a data-t="441">It had to be</a> <a data-t="441">the minimum input.</a> <a data-t="449">Secondly, it had to be self-explanatory.</a> <a data-t="449">If we needed training every</a> <a data-t="449">time something changed in the system, or we needed technical documentation, it just wasn't going to</a> <a data-t="454">get used.</a> <a data-t="460">And finally, it had to be accessible.</a> <a data-t="460">This wasn't something that we were going to have</a> <a data-t="460">time to break out laptops on, or we had to be at the station to use the station computer to work.</a> <a data-t="465">This had to be something really agile.</a></p>

<p><a data-t="472">So, in order to solve this problem, to meet these requirements,</a> <a data-t="472">and also with no money, and also with just me to rely on, we turned to </a><a href="/software/shiny-r/">Shiny</a><a data-t="472">.</a> <a data-t="478">Shiny was a great</a> <a data-t="478">solution for this.</a> <a data-t="484">And so, I began to build a Shiny app.</a> <a data-t="484">That Shiny app quickly became three</a> <a data-t="484">Shiny apps.</a> <a data-t="491">And so, now we have a Shiny app that helps us track our attendance on Tuesday nights.</a> <a data-t="491">We have one that we record our incidents in, or our calls.</a> <a data-t="498">And finally, we have a data portal</a> <a data-t="499">used by officers and administrators to generate reports, manage rosters, trainings, etc.</a></p>
