<h3>The tax divide problem</h3>

<p><a data-t="81">But starting in 2017, researchers at the University of Chicago and journalists at the Chicago Tribune, ProPublica, and the New York Times identified a problem, and that problem was about property taxes, and specifically about the way that assessors throughout the country were assessing property values.</a></p>

<p><a data-t="99">The problem was identified using something that's called a ratio study.</a> <a data-t="102">So the assessor's job in Illinois is to have assessments that follow the market every couple of years.</a> <a data-t="107">The assessor assesses property values and adjusts them so that they're roughly in line with where the real estate market has actually gone.</a> <a data-t="115">And in a ratio study, you take the properties that have actually sold, which is roughly 5% or so of homes here in any given year, and you compare the estimated value of that property to its actual sale price.</a></p>

<p><a data-t="128">And this is a New York Times graphic, we've done some light modifications, and that ratio should be roughly 1.</a> <a data-t="137">And what the New York Times and Chicago Tribune and others have found is that in the past, an average home was right around that 1 mark.</a> <a data-t="145">That's about where it should be.</a></p>

<p><a data-t="148">But what about other homes?</a> <a data-t="149">It's really important to think about homes on the other side of the distribution, not just the average homes.</a> <a data-t="155">So here's an example of a modest home on the south side of Chicago.</a> <a data-t="159">Let's say in 2015, it sold for about $50K.</a> <a data-t="163">On the other hand of that distribution, we have a nice home on the North Shore that maybe sold for more like a million dollars.</a> <a data-t="171">And the fundamental question is, what's that assessment ratio?</a> <a data-t="174">It's going to be close to 1.</a></p>

<p><a data-t="179">So what that would mean, just to unpack that ratio a little more, if that 50K home actually was assessed at 75K, that would mean a ratio of 1.5.</a> <a data-t="192">And if that home that sold for a million dollars was assessed as if it was worth 800K, that would mean a ratio of 0.8.</a></p>

<p><a data-t="201">Here's what the New York Times actually published.</a> <a data-t="205">There was a higher ratio of assessments to sale price for the homes with lower sale values.</a> <a data-t="213">So these homes that are on the more affordable side of the price spectrum were historically, according to this reporting, over-assessed.</a> <a data-t="222">That can lead to over-taxation.</a> <a data-t="223">And on the flip side, these homes that are these higher-priced homes were also being under-assessed, leading to under-taxation.</a> <a data-t="231">That's the problem.</a></p>

<p><a data-t="232">The Chicago Tribune referred to this as the tax divide.</a> <a data-t="235">And that tax divide is why we're here.</a> <a data-t="237">So again, the issue, according to the Tribune, as they stated it, is Cook County failed to value homes accurately for years.</a> <a data-t="244">And the result was a property tax system that harmed the poor and helped the rich.</a> <a data-t="249">The problem lies with the fundamentally flawed way that the County Assessor's Office valued property.</a></p>

<blockquote><a data-t="244">And the result was a property tax system that harmed the poor and helped the rich.</a></blockquote>

<p><a data-t="257">So what's the cause?</a> <a data-t="258">It could actually be a number of interesting causes.</a> <a data-t="260">Could be issues with the characteristics data that the Assessor's Office uses to estimate property values.</a> <a data-t="266">It could be issues with the way other offices actually adjust assessed values.</a> <a data-t="271">But we're here today because we want to talk about one of those issues, and one of those is about the modeling.</a></p>

<p><a data-t="276">So recall that the Assessor's Office uses sales, again, 5% of sales in a given year, to estimate property values for everyone.</a> <a data-t="283">In the past, this was done with regression modeling.</a> <a data-t="286">This is actually a snippet of code.</a> <a data-t="290">It was closed source, SPSS, OLS-based models, and, you know, we're talking 50,000 lines of manual fixed effects, property-specific overrides, and more.</a></p>

<h3>How tidymodels powers the pipeline</h3>

<p><a data-t="579">All right, but what about tidymodels?</a> <a data-t="581">This is supposed to be a tidymodels talk.</a> <a data-t="583">So how do we actually use tidymodels?</a> <a data-t="587">Well, as I said, tidymodels has fully replaced our sort of base R modeling code base.</a> <a data-t="594">And that has been a huge, huge boon for us.</a></p>

<p><a data-t="598">It has completely abstracted away all of the complicated ML logic that used to be in that old code base, and that is stuff like cross-validation, you know, tuning hyperparameters, all sorts of things like that.</a> <a data-t="614">It has allowed us to move much, much quicker and with more confidence, so we can sort of focus on the feature engineering and data collection side of things rather than being worried about shooting ourselves in the feet at every step.</a></p>

<p><a data-t="628">And importantly for us, it has also made the code much more understandable.</a> <a data-t="634">We are a public agency, right?</a> <a data-t="637">And we want the code to be intelligible to the greatest number of people possible.</a> <a data-t="643">I think that tidymodels helps us accomplish that by putting it into a standardized framework that other people are familiar with.</a></p>

<p><a data-t="654">And it has just unlocked sort of more advanced features for us like easier hyperparameter optimization and better, cleaner feature engineering.</a></p>

<p><a data-t="667">One example of that, time-based cross-validation.</a> <a data-t="670">So this is sort of how we do our cross-validation split using our data.</a> <a data-t="674">It's like an expanding window of the training set with the last 10% held out as a validation set.</a> <a data-t="682">This is not something that would be super easy to program outside of tidymodels, although we tried.</a></p>

<p><a data-t="692">We also made an R package.</a> <a data-t="695">This is our R package called lightsnip.</a> <a data-t="697">It is basically a shim between LightGBM and </a><a href="/software/parsnip/">parsnip</a><a data-t="697"> to allow us to do all sorts of fun things similar to bonsai, if you're familiar with that package.</a> <a data-t="711">We made it because we wanted to use some of LightGBM's more advanced features.</a> <a data-t="717">Specifically, we wanted to use early stopping.</a></p>

<p><a data-t="721">So if you are training a model and after a certain point, the model is not improving in performance, LightGBM can automatically stop further training rounds and save you a lot of time.</a> <a data-t="731">In our case, implementing early stopping decreased our training time by like roughly two-thirds.</a> <a data-t="739">This was a huge boon for us as a full training run with cross-validation was taking like 60 or 70 hours.</a></p>
