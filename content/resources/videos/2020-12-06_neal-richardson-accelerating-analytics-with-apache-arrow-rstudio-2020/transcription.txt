<h3>What is Arrow?</h3>

<p><a data-t="92">So generally when I tell people that I work on the Arrow project in data science, they're</a> <a data-t="97">like, oh, yeah, Arrow, that's awesome, I've heard of that, that's really cool.</a> <a data-t="102">What is Arrow?</a></p>

<p><a data-t="106">And Arrow's a lot of things, and that kind of defies a simple explanation.</a> <a data-t="111">One way that I like to frame it in terms particularly of what it means for the R community is that</a> <a data-t="118">it's a foundation for the next generation of data frames.</a></p>

<p><a data-t="123">As anyone who's worked with bigger data in R has surely experienced, R is great and there's</a> <a data-t="128">lots of powerful tools and really expressive ways of getting work done in R, but once you</a> <a data-t="134">start getting data sets that are bigger than memory on your laptop, you have to do some</a> <a data-t="138">special tricks to make it work.</a> <a data-t="141">Often you have data sets where the data's split into multiple files, and you can work</a> <a data-t="146">with this, but it's, again, it's not as natural.</a> <a data-t="150">And many data sets have more complex data types than your base character, numeric factor</a> <a data-t="157">and what have you.</a> <a data-t="159">And of course, the more data we have, the more we want to throw all of our hardware</a> <a data-t="162">at it.</a> <a data-t="163">So it's very possible in R, but it's just not how R itself was designed.</a></p>

<p><a data-t="167">But we're in good company.</a> <a data-t="171">These are not just problems of R. So Wes McKinney, who created the Pandas library for Python,</a> <a data-t="179">ran into similar challenges there, and constraints on memory and on data types and whatnot.</a> <a data-t="189">And so Wes and some others from the data community created the Apache Arrow project</a> <a data-t="195">in 2016.</a></p>

<p><a data-t="197">You may have experienced the feather package, which came out of some of the initial explorations</a> <a data-t="202">there, the prototype of the Arrow format, which makes it really easy to write data frames</a> <a data-t="208">in R and read them in Pandas and vice versa.</a> <a data-t="212">And the idea was to draw on all these lessons that we have from R and from Pandas and from</a> <a data-t="218">the database community and build the next generation of data frame tools that were not</a> <a data-t="223">tied to a specific language or implementation.</a></p>

<p><a data-t="228">So Arrow fundamentally is two classes of things.</a> <a data-t="231">So first is the format.</a> <a data-t="233">This is the specification of how bits are arranged in memory.</a> <a data-t="239">It's columnar and it's independent of any language.</a> <a data-t="242">And then there's implementations or bindings to other implementations in currently 11 languages.</a> <a data-t="248">So the Python library and the R package, they both bind to the C++ library, which is where</a> <a data-t="255">a lot of the development happens.</a></p>

<h3>Parquet file format</h3>
