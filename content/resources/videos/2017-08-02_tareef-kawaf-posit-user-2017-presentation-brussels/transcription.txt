<h3>RStudio Connect</h3>

<p><a data-t="451">All right, so how we think about data science.</a> <a data-t="454">You guys may have seen this before.</a> <a data-t="459">You know, you import data, you tidy the data, you transform the data,</a> <a data-t="461">you visualize the data, you model the data, and you keep going.</a> <a data-t="463">And then eventually the key thing behind doing all data analysis is to communicate, right?</a></p>

<p><a data-t="477">All right, so we decided to build a product</a> <a data-t="479">based on feedback that we got from customers.</a> <a data-t="481">And just to be clear, this product is a professional product.</a> <a data-t="481">And right now it's only available on premise,</a> <a data-t="483">but ultimately in time we imagine that we will have a cloud-based solution</a> <a data-t="488">that everybody in the community can sort of leverage if they want to, right?</a> <a data-t="491">And what this product is aimed at is making it really easy</a> <a data-t="496">to go from an analysis to being able to sort of share that with the rest of the world.</a> <a data-t="499">So it's push, button, deploy of all the artifacts that you sort of create out of R.</a> <a data-t="503">It's an on-premise solution because most of our company customers</a> <a data-t="506">will not run any of their stuff outside of their firewalls.</a></p>

<p><a data-t="509">And so, again, if you want to think about it,</a> <a data-t="512">you've got, you know, you guys are the creators of analyses.</a> <a data-t="515">You may create notebooks, you have static docs, you have presentations,</a> <a data-t="519">you may have parameterized R Markdown docs, as I said earlier,</a> <a data-t="521">you can have shiny applications.</a> <a data-t="523">And ultimately your goal is to communicate this to somebody, right?</a> <a data-t="526">Somebody in the organization needs to be able to see this,</a> <a data-t="528">and so we're trying to also build in things like, you know, scheduling.</a> <a data-t="531">So if you wanted a report to run every Monday and email people out or whatever,</a> <a data-t="535">you can sort of do things like that without having to sort of write your own crontabs</a> <a data-t="539">or, you know, have a separate system call into it.</a></p>

<p><a data-t="542">Now if you want to, you can absolutely do the same thing.</a> <a data-t="544">The foundation of everything that you see here is all open source, right?</a> <a data-t="547">People can recreate this themselves at any point in time, and I expect that they will.</a></p>

<p><a data-t="552">In terms of, so again, the creators, there's artifacts, there's consumers,</a> <a data-t="556">and you guys are all familiar with the work of the creator, right?</a> <a data-t="559">So you're tidying that whole tidy, transform, visualize model,</a> <a data-t="562">but you also care about reproducibility, you care about collaboration,</a> <a data-t="566">you care about publishing, you care about protecting, right?</a> <a data-t="568">So who can see this content, when can they see it, and so on and so forth, right?</a> <a data-t="571">And then on the consumer side, they want to be able to get a hold of the data,</a> <a data-t="575">they want to be able to explore the data, they may want to personalize it,</a> <a data-t="578">say, I'm only interested in reports about North America,</a> <a data-t="581">or I'm only interested in Southern Italy, right?</a></p>

<p><a data-t="600">All right, so that's what Connect does.</a> <a data-t="602">And, you know, a lot of our work ends up being providing features</a> <a data-t="609">to answer questions for the IT organization on why R, right?</a> <a data-t="613">The data scientists are bought in, they use it, they're building really great things,</a> <a data-t="617">and then they run into these obstacles within the IT organization.</a> <a data-t="620">So, you know, we spend a lot of time trying to say, okay, you know what,</a> <a data-t="623">here's how you tune this, here's how you monitor this, here's how you scale it.</a></p>
