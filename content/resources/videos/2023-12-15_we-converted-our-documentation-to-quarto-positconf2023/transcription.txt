<h3>Context and the problem with Word</h3>

<p><a data-t="164">I'm going to start by providing some necessary context.</a> <a data-t="169">I work on a team at Statistics Canada that publishes statistics related to education</a> <a data-t="174">in Canada.</a> <a data-t="176">There's a lot of domain knowledge and subject matter, as well as methodological concepts</a> <a data-t="181">that need to be documented somewhere.</a></p>

<p><a data-t="185">This leads to a very important distinction in the title of my talk.</a> <a data-t="191">When I say the word documentation, I'm not talking about our package documentation.</a> <a data-t="197">I'm talking about documentation that defines terms like these.</a> <a data-t="202">So it's really the subject matter documentation and not the package documentation.</a> <a data-t="208">I'm talking about the sort of information that somebody might put in a wiki or on Confluence</a> <a data-t="214">or something like that.</a></p>

<p><a data-t="217">Whatever you call yours, we call ours a user guide.</a> <a data-t="221">And for us, it was previously stored as a 120-page Word document.</a> <a data-t="228">Scrolling through this document was slow, and only one person could edit it at a time,</a> <a data-t="233">which was a significant challenge for our large team.</a> <a data-t="238">Our version control was limited to the track changes feature in Word, which made it pretty</a> <a data-t="243">challenging to follow updates over time.</a></p>

<p><a data-t="249">After last year's conference, we discovered Quarto, which would address those challenges</a> <a data-t="254">that we had been facing with Word.</a> <a data-t="257">When we started this conversion, though, we were all total beginners with Quarto.</a> <a data-t="263">This meant that in order for us to effectively complete this task, we would need to use the</a> <a data-t="268">right tools.</a></p>

<h3>Tools for converting and formatting content</h3>

<p><a data-t="271">When you're working with Quarto, there are so many tools that you can use to make your</a> <a data-t="275">life easier.</a> <a data-t="278">If you're looking to convert existing text to Quarto, a good first step is to convert</a> <a data-t="286">the text to Markdown first if it's not already in that format.</a> <a data-t="292">We used the Pandoc convert function from the R Markdown package to automatically convert</a> <a data-t="298">our Word document from Word format to Markdown.</a> <a data-t="303">This provided us a really great starting point.</a></p>

<p><a data-t="307">Although this was a great starting point, it was still just a starting point.</a> <a data-t="312">There was still a lot of formatting changes that we needed to make.</a> <a data-t="317">In order to make this part of the job easier, we set up custom keyboard shortcuts in our</a> <a data-t="321">IDEs.</a> <a data-t="323">For example, we set up a shortcut that made it a little bit easier to convert URLs into</a> <a data-t="329">valid Markdown links.</a></p>

<p><a data-t="332">If you're using </a><a href="/software/rstudio/">RStudio</a><a data-t="332"> as your IDE, you can also set up code snippets, which are a great</a> <a data-t="339">way to automate code insertion for anything that you find yourself typing out very frequently.</a> <a data-t="347">This is great for when you want to use some of the cool Quarto features that take a little</a> <a data-t="352">bit longer to type out.</a></p>

<p><a data-t="357">I really recommend checking out Tom Mock's GitHub gist that contains code snippets that</a> <a data-t="362">are very useful for working with Quarto.</a> <a data-t="365">Those code snippets are provided for both RStudio and VS Code.</a></p>

<p><a data-t="371">For any syntax that's a little bit trickier to type out, there are ways to avoid doing</a> <a data-t="377">things manually.</a> <a data-t="379">Personally, I find the syntax for Markdown tables to be extremely finicky.</a> <a data-t="386">RStudio's Visual Editor comes in great handy for this, or you can also find similar tools</a> <a data-t="391">online if you're using different IDEs.</a></p>

<p><a data-t="397">For situations like these, Generative AI is another great option.</a> <a data-t="402">Both ChatGPT and GitHub Copilot have a pretty decent knowledge of Quarto syntax, but an even</a> <a data-t="409">better understanding of HTML and CSS.</a> <a data-t="413">You can use Generative AI for a variety of use cases, but you'll find it particularly</a> <a data-t="418">helpful for making your Quarto projects more aesthetic.</a> <a data-t="424">If you're looking to use these tools, I've got a video on my YouTube channel that explains</a> <a data-t="428">how to use ChatGPT and GitHub Copilot as an R programmer.</a></p>

<h3>Version control and publishing</h3>

<p><a data-t="437">Once you're done with creating the content, there's something that you can do to make</a> <a data-t="441">maintaining the content easier as well.</a> <a data-t="444">You can use Git for proper version control, and you can use either GitHub Actions or GitLab</a> <a data-t="449">Pipelines to automate the publishing of your content.</a> <a data-t="454">This will make it so that every single time you make a commit using Git, the published</a> <a data-t="459">version of your content will be automatically updated without you having to do anything.</a></p>

<p><a data-t="467">I promise that this is much easier than it sounds, and I've got videos on my YouTube</a> <a data-t="471">channel that give examples of Quarto pipelines using both GitHub Actions and GitLab Pipelines.</a> <a data-t="480">Or if you prefer to avoid the whole pipeline thing altogether, and you're just looking</a> <a data-t="485">for somewhere to host your Quarto content for free, I've also got a video that explains</a> <a data-t="490">how to use Quarto Pub to publish a website in under a minute.</a></p>

<h3>Getting help from the community</h3>

<p><a data-t="497">There are so many great tools for working with Quarto, but there are probably still going</a> <a data-t="502">to be some times when you need a little bit of extra help.</a> <a data-t="506">There's probably going to come a point where you want to do something and you don't know</a> <a data-t="510">how to do that thing.</a> <a data-t="512">Whenever that happens, you can ask the R community for help.</a></p>

<p><a data-t="518">When it comes to asking Quarto questions, there's an order of operations that I like</a> <a data-t="523">to follow.</a> <a data-t="524">Of course, you should always start by checking the Quarto documentation and googling your</a> <a data-t="529">problem to see if somebody's already made a video, a blog post, or posted on Stack Overflow</a> <a data-t="535">about it.</a> <a data-t="538">But if you can't find what you're looking for there, you can also reach out to your</a> <a data-t="542">colleagues or friends, including the ones you make while here at Conf.</a> <a data-t="547">You can also reach out to your local R user group or your R-Ladies chapter, if you have</a> <a data-t="552">one.</a></p>

<h3>Getting colleagues on board</h3>

<p><a data-t="798">Once we had a working first draft, it was time to share our project more broadly with</a> <a data-t="804">others.</a> <a data-t="806">This is where we faced our third challenge, and where you're likely to face a challenge</a> <a data-t="810">too.</a> <a data-t="812">Although you might be super passionate about Quarto and confident in its potential, it is</a> <a data-t="817">still a pretty new product, and not everyone is going to be on board on day one.</a></p>

<p><a data-t="825">Getting that support for Quarto can be challenging, and the approach that you take may differ</a> <a data-t="832">depending on your target audience.</a> <a data-t="836">With management, a good strategy is to show, not tell.</a> <a data-t="841">Rather than starting a conversation about Quarto, create a cool project using Quarto that</a> <a data-t="847">will wow people.</a> <a data-t="850">When someone inevitably asks you how you made it, this is when you can talk about what Quarto</a> <a data-t="855">is.</a></p>

<p><a data-t="859">Depending on your workplace, you may have the additional challenge of Quarto not currently</a> <a data-t="863">being on the list of installable software.</a> <a data-t="867">In our case last summer, we didn't have access to a version of RStudio that came prepackaged</a> <a data-t="873">with Quarto.</a> <a data-t="874">Instead, we needed to use VS Code as our editor while we waited for our version of RStudio</a> <a data-t="881">to be updated.</a> <a data-t="884">Another option is to use Rocker, which is a Docker image that comes prepackaged with</a> <a data-t="889">Quarto.</a></p>

<p><a data-t="891">In order to get your colleagues on board with Quarto, I highly suggest creating templates</a> <a data-t="896">that you or anyone else can use as a starting point for their project.</a> <a data-t="901">Not only is this going to prevent you from needing to reinvent the wheel every time,</a> <a data-t="905">but it'll also reduce the barriers that others might face when trying to learn a new</a> <a data-t="909">tool.</a> <a data-t="912">You can either start with a pre-existing template, for example from the awesome Quarto repo on</a> <a data-t="917">GitHub, or you can create your own.</a> <a data-t="920">For example, I created this Statistics Canada theme for the Reveal.js format.</a></p>

<p><a data-t="927">For any colleagues who don't have Git or Markdown experience, a great way to enable</a> <a data-t="932">collaboration is to use the report issue feature on your Quarto projects.</a> <a data-t="940">With just a single click of a button, a new issue can be created on the project's GitLab</a> <a data-t="945">or GitHub repo, and the necessary changes can be described in plain text using a what-you-see-is-what-you-get</a> <a data-t="951">type of text editor.</a> <a data-t="953">This then makes it very easy for the developer to go in and copy-paste the modifications</a> <a data-t="959">that need to be made with little to no modification on their end, allowing for effective collaboration</a> <a data-t="966">across the board.</a></p>

<p><a data-t="969">And finally, you can also enable Word and PDF formats for any end users who might prefer</a> <a data-t="974">those formats over HTML.</a></p>
