<p><a data-t="0">As educators, we want our students to focus on learning, not troubleshooting package installations.</a> <a href="/software/rstudio/">RStudio</a><a data-t="9"> Cloud gives us the ability to install all of the packages we'd like to use for a course or workshop using a base project.</a> <a data-t="17">This means that all of the packages we need are pre-installed in all of the projects within our shared workspace.</a> <a data-t="23">Let me show you how that works.</a></p>

<h3>Creating a base project</h3>

<p><a data-t="25">From within our workspace, we'll click on New Project.</a> <a data-t="28">This will deploy a new RStudio project for us.</a> <a data-t="31">Once we have our project, we can install all of the packages we'd like to use for our course.</a> <a data-t="37">You can either create an R script with all of the package installations, or complete the installations via the console.</a></p>

<p><a data-t="44">One benefit to installing packages via the console is that it prevents a bit of confusion for your learners.</a> <a data-t="51">This is because any R scripts or files that you create in your base project will be available to all of the learners within your workspace.</a> <a data-t="58">Therefore, it's possible that learners may run the package installation script, thinking that this is a necessary step in the process.</a></p>

<blockquote><a data-t="44">One benefit to installing packages via the console is that it prevents a bit of confusion for your learners.</a> <a data-t="51">This is because any R scripts or files that you create in your base project will be available to all of the learners within your workspace.</a> <a data-t="58">Therefore, it's possible that learners may run the package installation script, thinking that this is a necessary step in the process.</a></blockquote>

<p><a data-t="66">We can name our project anything that we'd like, and for this example, we'll name it Base by clicking on Untitled Project and typing in Base.</a> <a data-t="74">Before we can set this as the default for our workspace, we need to make it accessible to all of our learners by clicking on the Settings wheel,</a> <a data-t="81">then clicking on the Access tab, and from the Who can view this project drop-down, choosing the Everyone option.</a></p>

<h3>Assigning the base project to the workspace</h3>

<p><a data-t="88">Once our project has been created and made accessible to everyone in the workspace,</a> <a data-t="93">we can move out of the project and back to the workspace by clicking on the workspace name in the top menu.</a> <a data-t="99">From the workspace, we'll click on the Settings wheel in the top right, and from the Base RStudio Project drop-down menu, choosing our base project.</a></p>

<p><a data-t="108">This now sets up all of our learners with the packages we've installed, but that's not all.</a> <a data-t="112">We can also create a new base project at any time with additional package installations.</a> <a data-t="117">Because changes to the base project are not retroactive, any previous installations will be carried forward</a> <a data-t="124">along with the updated installations established when we create and assign a new base project.</a></p>
