<h3>What is querychat?</h3>

<p><a data-t="104">So what is querychat? Well, querychat makes it easy to query data using natural language.</a> <a data-t="111">It offers three things. One, a dropping component for Shiny. <h3>How querychat works under the hood</h3>

<p><a data-t="348">querychat is more than just a chat window sitting on top of a dataset.</a> <a data-t="353">With querychat, you never have to think about how to get to an answer.</a> <a data-t="360">You don't have to think about</a> <a data-t="360">what filters to apply or how to extract information from the filter data.</a> <a data-t="367">No, you simply ask any</a> <a data-t="367">question that comes to mind and querychat handles the rest.</a> <a data-t="373">And yeah, yeah, yeah, I hear you. You</a> <a data-t="373">could also ask all those questions in a typical LLM chat tool, but querychat is different.</a> <a data-t="379">You</a> <a data-t="379">are not relying on the LLM to invent answers or reason about the data internally.</a> <a data-t="386">No, every single</a> <a data-t="386">question is translated into SQL, and this SQL is executed on the actual dataset and returned</a> <a data-t="392">exactly as the data dictates.</a> <a data-t="400">And very important, the SQL is always shown.</a> <a data-t="400">This means you can see</a> <a data-t="400">precisely what is being run.</a></p>

<blockquote><a data-t="379">You are not relying on the LLM to invent answers or reason about the data internally.</a> <a data-t="386">No, every single question is translated into SQL, and this SQL is executed on the actual dataset and returned</a> <a data-t="392">exactly as the data dictates.</a></blockquote>

<p><a data-t="409">So that's great. All right. Four benefits of querychat.</a> <a data-t="409">One, reliability.</a> <a data-t="409">The LLM does not analyze or transform the raw data itself. It only generates SQL text.</a> <a data-t="416">querychat</a> <a data-t="416">handles the execution of that SQL via tool calling, so all the results come from the real</a> <a data-t="421">data engine and not from the model's internal guesswork.</a> <a data-t="429">Two, transparency. Every query reveals</a> <a data-t="429">the full SQL statement. Nothing is hidden, nothing is adjusted, and you always know how the answer was</a> <a data-t="435">produced.</a> <a data-t="444">Three, reproducibility. Since every SQL query is visible, analysis can be reused, shared,</a> <a data-t="444">and audited.</a> <a data-t="451">And four, safety. querychat's tools are designed with read-only actions in mind,</a> <a data-t="451">meaning the LLM is essentially unable to perform any destructive actions.</a></p>

<p><a data-t="457">All right, let's go back for a second. I mentioned querychat handles execution of the SQL via</a> <a data-t="457">something called tool calling.</a> <a data-t="468">Well, tool calling is essentially a bridge between an LLM and your R</a> <a data-t="468">session.</a> <a data-t="475">The model does not execute code. Instead, it requests your R session to execute a certain</a> <a data-t="475">function with certain inputs.</a> <a data-t="482">Once R performs the execution, the result is then passed back</a> <a data-t="482">to the model for interpretation.</a> <a data-t="489">But the LLM does need to know something about your data,</a> <a data-t="495">you know, things like which columns are there, what do they mean, and what type are they.</a></p>

<p><a data-t="496">This schema information is shared with the model, but not the raw data itself.</a> <a data-t="502">With this information,</a> <a data-t="502">it produces a SQL query as a tool call.</a> <a data-t="508">Now, to run SQL, you do need a database engine,</a> <a data-t="508">and querychat's weapon of choice is DuckDB.</a> <a data-t="515">If you have your own database, that's no problem too.</a> <a data-t="515">Whatever </a><a href="/software/dbi/">DBI</a><a data-t="515">, this is a database interface package for R, supports can be used by querychat.</a> <a data-t="522">Pretty clever, huh?</a></p>

<p><a data-t="816">And in just 10 minutes, you learned how to use querychat to build</a> <a data-t="816">your next app.</a> <a data-t="821">I want to come back to one important aspect of querychat, though, which is</a> <a data-t="821">safety.</a> <a data-t="829">Because is this safe?
