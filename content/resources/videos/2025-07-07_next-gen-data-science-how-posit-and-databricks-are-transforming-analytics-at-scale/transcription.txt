<h3>Building a Tour de France dashboard</h3>

<p><a data-t="518">This is like a very, very brief introduction, but a workflow that we see very typically is data scientists will come into an environment like Positron, they'll explore some data, and then they'll want to deliver what they've learned or some sort of insight to business users and stakeholders, and in many cases that takes the form of some sort of dashboard or interactive application.</a> <a data-t="552">Positron's a great environment for building these kinds of dashboards and interactive applications.</a></p>

<p><a data-t="558">Here's our Tour de France data, we have all this information about how long it took people to ride a given stage, what the result was, who the rider was, and so we want to present this information to our business users in a way that enables them to get their own insights from this particular dataset.</a> <a data-t="594">What we can do is we can, inside of Positron, we're going to write an interactive web application using the </a><a href="/software/shiny-r/">Shiny</a><a data-t="594"> framework in Python.</a> <a data-t="600">We can do this in Streamlit, we can do this in R, we can do this in Dash, we can do this in Gradio.</a> <a data-t="604">Choose the framework you want.</a></p>

<p><a data-t="605">I'm choosing Shiny here, and then I can click on this little play button inside the console, inside the text editor, and say, run the app.</a> <a data-t="613">What happens is Positron runs the app directly inside the IDE.</a> <a data-t="616">I don't have to open another browser window, I don't have to figure out what uvicorn is and how it's related to a unicorn, I can just run the app directly in the IDE, and it live updates.</a> <a data-t="625">If I make changes to my source code, the app refreshes, and I see those changes directly.</a> <a data-t="629">It's a very nice pattern for doing this sort of interactive application development.</a></p>

<p><a data-t="634">So let's take a look at our app for a moment.</a> <a data-t="636">We have some charts that indicate who had the most stage wins in the dataset, and other information here, who participated in the most stages.</a> <a data-t="643">But the critical thing about this app is that the only user input is this chat interface on the left-hand side.</a> <a data-t="650">And so here, as a user, I've come in and said, hey, how many total unique cyclists are there in this dataset?</a> <a data-t="656">And you'll notice that it generates this SQL query, and it says, look, there's 5,162 unique cyclists.</a></p>

<p><a data-t="662">Something really interesting is happening here.</a> <a data-t="664">We're not giving all this data to the LLM and saying, hey, tell me how many cyclists are in this data.</a> <a data-t="669">Instead, we've given the LLM the schema of our data, and then we've said, hey, given this schema, write a SQL query that will help us answer this question.</a> <a data-t="678">So it's fully transparent, and much less prone to hallucinations, because we're not asking the LLM to interpret data itself, we're asking it to write the code that will then give us the answer to the problem that we have, or the question that we pose.</a></p>

<h3>Unity Catalog and row-level permissions</h3>

<p><a data-t="899">Now, one kind of interesting piece here is you'll notice on the right-hand side, we've shown that all users who have a login can come visit this application.</a> <a data-t="909">And then down at the bottom, there's this little piece that says, hey, this application is connected to Databricks.</a> <a data-t="913">It shows me that it's connected into Azure Databricks.</a> <a data-t="916">So anytime somebody views this application, they're prompted to log in to their Azure Databricks account.</a> <a data-t="922">Now, that's critical because what that means is if you have granular permissions defined inside of Unity Catalog, if you've got a Unity Catalog filter that you've written on top of this data, that can apply all the way through.</a></p>

<p><a data-t="934">So let's illustrate this with a very infamous example, right?</a> <a data-t="938">Let's take Lance Armstrong, most infamous cyclist of all time, famous for doping violations that he later admitted to and being stripped of all seven Tour de France overall victories, as well as all UCI World Tour victories.</a></p>

<p><a data-t="951">If I come in to my application as a user, so here in Databricks, we've written a Unity Catalog filter that says, hey, any convicted cyclist for doping is invisible or not accessible to certain users.</a> <a data-t="966">So as my user, if I come in and I say, hey, application, how many stages of the Tour de France did Lance Armstrong win?</a> <a data-t="971">It comes back and says he won 24 stages.</a> <a data-t="974">But if a less-permissioned user logs in and then asks the same question, they get a very different response.</a> <a data-t="981">It says zero.</a></p>

<p><a data-t="985">And then the LLM kind of goes on a journey, because it says, I think Lance did win some stages, and so it tries a couple of times to figure out, is his name misspelled?</a> <a data-t="994">Maybe he's organized in here a little bit differently?</a> <a data-t="997">And then finally it comes in and it concludes and says, you know what?</a> <a data-t="1000">I can't find any results for Lance Armstrong, and it's probably because he was convicted of doping and all of his results were stripped.</a></p>

<p><a data-t="1007">Same application, same URL, two different users, two different answers, because they have different permissions in Unity Catalog.</a> <a data-t="1016">This allows you to use Unity Catalog as the source of truth for data and data access, and then serve an entire collection of business users with a single entry point without needing to worry about building in logic for who can see what, because Unity Catalog is already taking care of it.</a> <a data-t="1031">Not only is this best practice, it also makes the life of the developers a lot easier, because one app can now serve multiple audiences within the organization.</a></p>
