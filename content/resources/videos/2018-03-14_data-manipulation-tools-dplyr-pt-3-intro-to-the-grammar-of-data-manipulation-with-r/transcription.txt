<h3>Extracting information with select and filter</h3>

<p><a data-t="93">So when it comes to accessing information in your data frame, it can help to just extract</a> <a data-t="98">information that's already there to avoid the sort of overwhelming amounts of distraction</a> <a data-t="104">that the other variables provide. <p><a data-t="117">dplyr provides four functions, each dedicated to doing one of these things, and let's take</a> <a data-t="122">a look at them, starting with select. So, select is very simple. And you could list as many column names or as few column</a> <a data-t="142">names as you want there, and it'll bring back everything you list. So that's easy enough</a> <a data-t="146">to do in R, and you might ask, why would you use select to do it? Well, one thing that</a> <a data-t="152">select does is it provides more ways to specify which columns you want. So you'll know that</a> <a data-t="157">if you use brackets and integer subsetting, you put a negative sign in front of an integer</a> <a data-t="161">and that stands in R, for giving you everything except this integer. With select, you can</a> <a data-t="166">put a negative sign in front of a name, and you'll get back everything except that name.</a></p>

<p><a data-t="170">So here we did everything except the storm column. You could also use the colon notation</a> <a data-t="177">to get a range of variables. And you</a> <a data-t="199">can see they allow you to pull out columns whose names have a certain string in them</a> <a data-t="204">or a certain regular expression or begin with a certain word or end with a certain word</a> <a data-t="209">or is in a group of names and so on. So if you have a very large data set, selecting</a> <a data-t="214">variables out of it might be an important task, and these functions might make it very</a> <a data-t="218">easy for you to sort of search through the variable namespace in an automated fashion.</a></p>

<p><a data-t="225">Filter is analogous to select, except instead of selecting columns, it brings back rows,</a> <a data-t="230">or if your data's tidy, these would be observations. If you want to do an or</a> <a data-t="267">condition, we'll just use the Boolean operator for or and r to combine two tests into a single</a> <a data-t="272">test. That's the pipe operator. So filter works best if you know how to use logical</a> <a data-t="279">tests in R, and here's a quick list of the logical tests that exist in R. The table on</a> <a data-t="284">the right are Boolean operators. <h3>Deriving new variables with mutate</h3>

<p><a data-t="315">The mutate function takes your data frame and it returns a copy</a> <a data-t="319">of the data frame with new variables added to it. Normally these will be variables that</a> <a data-t="323">you derive from the variables that exist in your data. So before we had sort of the</a> <a data-t="328">bogus example of calculating a ratio variable from the wind and pressure variable in storms,</a> <a data-t="330">the information in that ratio variable was in the data to begin with, it just wasn't</a> <a data-t="334">on display, but you could logically derive it from the data that's there, and this is</a> <a data-t="339">how you do it with mutate. You run mutate on the data frame you want to add to or change,</a> <a data-t="346">then you give it the name of the new column you want to make, and you set that equal to</a> <a data-t="350">some expression that will make the values of the new column. You can make more than</a> <a data-t="355">one column at the same time, and you can even use a column to make a new column in the same</a> <a data-t="360">command. Make sure that the first column appears before the columns that rely on it. So here</a> <a data-t="366">inverse relies on ratio, and since ratio appears first, inverse will be able to find ratio</a> <a data-t="371">and use it to calculate inverse. This command, mutate, doesn't actually affect your original</a> <a data-t="378">data frame, it returns a data frame that has the new variables added to it. So if you want</a> <a data-t="383">to save that data frame and use those variables later, you'll need to assign it to an object.</a></p>

<p><a data-t="389">There are a lot of functions that are useful to use with mutate to make new variables.</a> <a data-t="393">This is a short list of them. They all take a vector and return a vector of the same length. So if you want</a> <a data-t="409">to add a column onto your data frame, that column's going to have to have values for</a> <a data-t="413">each row in the data frame. When you use these functions, you'll get a set of values that</a> <a data-t="418">will fit right up with your data frame. <h3>Summarizing data</h3>

<p><a data-t="437">So the last of those four functions we listed was summarize. Summarize</a> <a data-t="441">takes a data frame and it calculates summary statistics from it.
