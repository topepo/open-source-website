<h3>Origins of the flipbook project</h3>

<p><a data-t="115">So I think to understand where we</a> <a data-t="115">are with the project right now, it might be useful to look back in time to look at the</a> <a data-t="121">origins of this project. And I'd say that in 2018, we were in the era of the proto-flipbook.</a> <a data-t="129">So Garrick, Aidan, Bui, and I and Emi Tanaka were all interacting with this idea of presenting</a> <a data-t="135">code and output side-by-side and building it up incrementally. But at this time, the</a> <a data-t="143">project that the projects that we were building were very handcrafted and artisanal. So we</a> <a data-t="150">were all crafting each slide that composed each mini-flipbook one-by-one, using a lot</a> <a data-t="159">of copying and pasting.</a></p>

<p><a data-t="168">But I was excited to find these like-minded people, and so I</a> <a data-t="168">approached them. They seemed to know a lot about the tool that we were building with</a> <a data-t="173">Sharingan, and I asked them, do you want to build flipbooks together? We were all circling</a> <a data-t="183">around this idea. We thought the flipbooks could really deliver great experience to users,</a> <a data-t="190">deliver them a lot of insights about what's going on in a code pipeline. But to really</a> <a data-t="197">reach flipbooks' full potential, we realized that we had to move away from this copy-paste</a> <a data-t="204">modality. We had to move to something that unburdened the creator of flipbooks.</a></p>

<p><a data-t="211">So we moved to automation. So now we're automatically parsing a block of input code</a> <a data-t="219">and automatically reconstructing code into partial builds, and then automatically delivering</a> <a data-t="225">these partial builds onto a presentation platform.</a></p>

<p><a data-t="231">2019 was a year of exploration and refinement. We were interested in how we could use this</a> <a data-t="241">new tool, different modalities to use it, using other people's code, breaking the flipbook</a> <a data-t="248">building code that we had developed. And in 2020, you can build a flipbook in five minutes</a> <a data-t="257">with flipbook R, which is a development package on GitHub, using Sharingan as well.</a></p>
