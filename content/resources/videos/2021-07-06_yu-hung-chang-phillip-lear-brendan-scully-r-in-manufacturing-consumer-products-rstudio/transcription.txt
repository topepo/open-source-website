<h3>Decision analytics hierarchy of needs</h3>

<p><a data-t="919">So moving on to my role in the strategy team</a> <a data-t="921">is I've kind of taken a step back</a> <a data-t="921">from the individual tools and taken a higher level view</a> <a data-t="925">and trying to work on ways that Pratt can prioritize</a> <a data-t="932">decision analytics projects and best use them.</a> <a data-t="935">And one framework that's helped me think</a> <a data-t="937">about these projects is the decision analytics hierarchy</a> <a data-t="940">of needs based loosely off Maslow's hierarchy of needs.</a></p>

<p><a data-t="945">The idea here being each level of the pyramid</a> <a data-t="949">needs to be addressed before the other one</a> <a data-t="951">can even be worked on.</a> <a data-t="953">And the way I've broken it down</a> <a data-t="955">and the way I see a lot of projects at Pratt</a> <a data-t="959">are data storage and organization is the first step</a> <a data-t="965">and the most important step that needs to be addressed.</a> <a data-t="968">Data relevant to your project or your business need</a> <a data-t="973">needs to be stored and organized</a> <a data-t="975">in an effective and thoughtful way.</a></p>

<p><a data-t="980">However, storage and organization is not enough.</a> <a data-t="983">There then needs to be data accessibility in place.</a> <a data-t="987">So what's required to make the data</a> <a data-t="989">accessible to stakeholders?</a> <a data-t="991">That could be literal access, access approval,</a> <a data-t="998">having the credentials or tools</a> <a data-t="1002">you need to access the database.</a> <a data-t="1004">But it also means is it stored in a way that is accessible?</a> <a data-t="1007">Is it in, there's a team here that has a lot</a> <a data-t="1013">of repair information stored in PDFs.</a> <a data-t="1015">Well, it's stored, but it's not accessible</a> <a data-t="1017">in a meaningful way.</a></p>

<p><a data-t="1022">The last step then is when you have successful data storage</a> <a data-t="1026">and accessibility is then building the analytics tools</a> <a data-t="1031">and then implementing data-driven decisions.</a> <a data-t="1034">So how can the data then be used to serve our needs</a> <a data-t="1037">and make better decisions?</a> <a data-t="1039">A lot of times there's a lot of talk and focus</a> <a data-t="1042">on the top of the pyramid without full understanding</a> <a data-t="1045">and focus on data accessibility and data storage.</a> <a data-t="1048">So this is a framework that I find useful</a> <a data-t="1051">to level set expectations for a project</a> <a data-t="1057">and then direct the right resources</a> <a data-t="1059">to the right problems before starting a project.</a></p>
