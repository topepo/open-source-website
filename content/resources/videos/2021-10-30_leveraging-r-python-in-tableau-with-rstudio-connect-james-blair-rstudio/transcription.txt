<h3>Comparing old and new approaches</h3>

<p><a data-t="1911">So, here's a kind of a more comprehensive example.</a> <a data-t="1911">We'll take a look at this dashboard</a> <a data-t="1911">here in just a moment.</a> <a data-t="1914">But here's an example where, and this goes back to the Tableau workbook</a> <a data-t="1914">that we looked at kind of at the very beginning, where we are submitting information about a given</a> <a data-t="1919">customer and their transactions to a machine learning model that was built in Python,</a> <a data-t="1923">or generating predictions based on that input data, submitting those predictions back to Tableau,</a> <a data-t="1928">and then visualizing them in that Tableau environment.</a> <a data-t="1933">And here we see, again, the same</a> <a data-t="1933">pattern.</a> <a data-t="1933">We use this script function in our calculated fields, and then we define the path</a> <a data-t="1938">to the endpoint that we want to, or the path to the extension that we want to use, and then the</a> <a data-t="1942">values from our workbook that we want to pass into that extension.</a></p>

<p><a data-t="1949">So, let's contrast for a moment</a> <a data-t="1949">the two different approaches, right?</a> <a data-t="1953">We talked about how, in the beginning, we have tabpy, we</a> <a data-t="1953">have rserv as existing integration points for executing R and Python code within Tableau.</a> <a data-t="1960">The pattern for those extensions is what we're seeing here, right?</a> <a data-t="1966">I have some script function,</a> <a data-t="1966">and then the first argument to that script function is the full script that I want executed.</a> <a data-t="1972">In this case, it's the entire Python script that allows me to do this clustering of data points.</a></p>
