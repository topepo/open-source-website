<h3>The tidymodels framework</h3>

<p><a data-t="403">The other part I'm talking about is tidymodels, unsurprisingly.</a> <a data-t="403">It is a</a> <a data-t="403">framework for modeling and machine learning using </a><a href="/software/tidyverse/">tidyverse</a><a data-t="403"> principles.</a> <a data-t="409">So</a> <a data-t="409">what does that mean?</a> <a data-t="416">It means it has infrastructure and cover for the</a> <a data-t="416">predictive modeling workflow, so we have something for resampling, there's code</a> <a data-t="423">for pre-processing in a very elegant way, there's the models, there are the</a> <a data-t="428">performance metrics, and there's something to orchestrate all of that</a> <a data-t="431">into tuning models.</a> <a data-t="440">That's the core of it, but that's not all.</a> <a data-t="440">So it is</a> <a data-t="440">extendable.</a> <a data-t="446">We have done that a couple of times, and our community has done</a> <a data-t="446">that in parts.</a> <a data-t="452">So there is specialized stuff for spatial data, thanks to our</a> <a data-t="452">then intern Mike Mahoney.</a> <a data-t="457">There's some more specialized methods for tuning,</a> <a data-t="457">thanks to Max.</a> <a data-t="463">For recipes, there are extensions for text data, and unbalanced</a> <a data-t="463">data, and factors with a whole lot of levels, and then the bulk of extensions</a> <a data-t="469">sits around the models, where you can get specialized models for different tasks.</a> <a data-t="476">The one I'm going to point out here is the censored package, because that holds</a> <a data-t="481">the parts for survival analysis and censored regression.</a></p>

<p><a data-t="489">So that was a whole</a> <a data-t="489">bunch of lovely, very beautiful colorful hex stickers, but the point is that if</a> <a data-t="496">you do predictive modeling, there's a whole lot of things you have to do that</a> <a data-t="501">aren't actually focused on your modeling question.</a> <a data-t="507">All of the resampling, and then</a> <a data-t="507">doing your preprocessing, and your model fitting on the right resamples, and then</a> <a data-t="511">predicting, and evaluating it, and calculating the performance metrics, and</a> <a data-t="515">aggregating all of that.</a> <a data-t="515">You do that every time, but that is not what really</a> <a data-t="520">interests you in this case.</a> <a data-t="525">So a framework takes that off your plate.</a> <a data-t="525">That</a> <a data-t="525">provides the infrastructure, and you can use the parts, and you can focus on what's</a> <a data-t="531">really interesting.</a> <a data-t="538">And tidymodels is built with tidyverse design principles,</a> <a data-t="538">so if you're comfortable with the tidyverse, you should feel at home with</a> <a data-t="541">tidymodels as well.</a> <a data-t="549">It sort of follows the same care in APIs, and consistency in</a> <a data-t="549">function naming, gives you like a unified front of it, so you don't have to</a> <a data-t="555">necessarily learn individual packages.</a> <a data-t="560">If you're sitting more in a place where you</a> <a data-t="560">would like to give different ones a go to find out what fits your data, if</a> <a data-t="565">you're not having very, very specialized concerns and modeling implications.</a> <a data-t="572">So</a> <a data-t="572">that is what we try to make easier for you, so you can focus your attention on</a> <a data-t="578">what does actually fit well here, what can I get by making clever features, and</a> <a data-t="583">so on.</a></p>
