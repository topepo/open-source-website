<h3>The rise of large language models</h3>

<p><a data-t="101">Like it or not, large language models are already everywhere.</a> <a data-t="106">If you don't use them today, it's likely that you will be using them soon.</a> <a data-t="110">They're being integrated directly inside the tools that we use to do our jobs.</a> <a data-t="114">Whether we're data scientists, developers, sit in meetings all day, write emails, respond</a> <a data-t="120">to chats, Microsoft 365 Copilot is bringing language models directly into tools like Teams,</a> <a data-t="128">Outlook, Word, and PowerPoint.</a> <a data-t="130">And if you're like me, that covers a lot of what you're using day to day.</a> <a data-t="135">Google's Duet AI is doing the same thing for Google's Workspace.</a> <a data-t="140">And even autocomplete features in our text messages and emails are bringing these models</a> <a data-t="145">to our fingertips by default, whether we want them there or not.</a></p>

<p><a data-t="151">These models are a big change for all of us, developers or not, but my own experience started</a> <a data-t="157">with code.</a> <a data-t="158">Specifically, it started with me using GitHub Copilot, a large language model powered tool</a> <a data-t="165">that could help me write code.</a> <a data-t="167">This was before the craze of chat GPT, but I was still quite excited by the ability of</a> <a data-t="173">a development environment to fix my code, give me new ideas, and really just lower the</a> <a data-t="178">barrier to me making progress on a data science project.</a></p>

<h3>Text embeddings with GPT Tools</h3>

<p><a data-t="629">Okay, so this gives you the response of what a custom chatbot could look like, but of course</a> <a data-t="635">there's more that we could go do with this as well.</a> <a data-t="638">And for that, we're going to move from the GPT Studio package over to GPT Tools.</a></p>

<p><a data-t="645">A common challenge you'll find when using large language models is that their training</a> <a data-t="651">data doesn't contain the context that's important to answer the question that you just asked.</a> <a data-t="657">So giving it missing information, missing context through a technique called text embeddings</a> <a data-t="662">or a vector database, semantic search, there's a lot of different terms for this, but the</a> <a data-t="667">basic idea is that we want to give the model the information that it was missing right</a> <a data-t="671">inside the prompt so that it can provide more contextualized answers, more accurate answers</a> <a data-t="677">to our prompts.</a> <a data-t="679">To do that, we'll use the package GPT Tools, and in comparison to GPT Studio, GPT Tools</a> <a data-t="686">is more experimental.</a> <a data-t="687">It's more likely to break, but it's also where you're going to see some of the cutting-edge</a> <a data-t="693">applications of large language models inside of RStudio.</a></p>

<p><a data-t="697">So with that, let me show you how we can create text embeddings of the Tidyverse Style Guide</a> <a data-t="703">or Tidy Design Principles to extend our Hadley-Wickham chatbot.</a> <a data-t="709">We'll need to install GPT Tools, which we'll need to do from GitHub since it's an experimental</a> <a data-t="716">package and not yet on CRAN.</a> <a data-t="718">You can use the package manager, Hack, to make this easy.</a> <a data-t="723">The code on the slide shows you how, pointing to jameshwade.com.</a></p>

<p><a data-t="756">Now that we've created our embeddings, we can load the app by typing in retrieval into</a> <a data-t="761">our add-in.</a> <a data-t="762">It's been running chat-gpt with retrieval.</a> <a data-t="765">What this is going to do is when the app loads, we can type our prompt into the box here.</a> <a data-t="772">So the prompt is, in this case, what do I need to know about writing tidy functions</a> <a data-t="776">in R?</a> <a data-t="777">It's going to take that prompt and create an embedding, a vector, that it then can match</a> <a data-t="782">against the text that we just scraped.</a> <a data-t="785">The answer that we'll get back is now going to be contextualized with the Tidy Design</a> <a data-t="789">Principles.</a></p>

<h3>Keeping package documentation up to date</h3>

<p><a data-t="821">A frustration I've often found with using these large language models to help me code</a> <a data-t="827">is that they're pretty much always out of date.</a> <a data-t="832">An update will be pushed to some of my favorite packages, and when I ask for help, the answer</a> <a data-t="837">I get back from these models is old and not really what I'm wanting to use.</a> <a data-t="843">When I'm trying to push the edges of some of these capabilities, of course I want the</a> <a data-t="847">latest documentation, and we can use this text embeddings approach to keep chat-gpt</a> <a data-t="854">up to date with all of our favorite packages.</a> <a data-t="856">Let me show you how.</a></p>

<p><a data-t="859">To keep our text embeddings for our package documentation up to date, we first need to</a> <a data-t="864">list our favorite packages.</a> <a data-t="866">I have here a list of some of my favorite packages as part of the tidyverse, </a><a href="/software/tidymodels/">tidymodels</a><a data-t="866">,</a> <a data-t="872">shiny, some packages for machine learning operations, natural language processing packages,</a> <a data-t="878">and packages to interface with large language models.</a> <a data-t="882">That list we can pass to a function called getPackagesToScrape, which will look to see</a> <a data-t="887">if there have been updates to the package based off the package version.</a> <a data-t="891">If it's out of date, you can then pass it to scrapePackageSites that will take that</a> <a data-t="896">list of outdated packages, scrape the documentation, and recreate your text embeddings with the</a> <a data-t="902">up-to-date documents.</a></p>

<p><a data-t="904">It does its best to not recreate text embeddings for text it's already seen before, to be wise</a> <a data-t="911">with its token spending budget, but it isn't perfect, and this is likely to be a somewhat</a> <a data-t="916">expensive operation, I would estimate on the order of maybe a dollar or a few dollars</a> <a data-t="921">if you're using the OpenAI text embedding endpoint for this.</a> <a data-t="927">With these updated packages, you'll now get your responses coming from the querying chat</a> <a data-t="934">GPT that provides the context for the latest and greatest from your favorite packages.</a></p>
