<h3>How Workboots works</h3>

<p><a data-t="521">But if you do find yourself in a scenario where</a> <a data-t="521">you are meeting all three of these checks and you're ready to start using Workboots,</a> <a data-t="526">how does it actually work? I'm going to gloss over this very quickly, but if you'd like</a> <a data-t="532">to dig in deeper into some of the math that's underlying it, you can scan this QR code.</a> <a data-t="537">That'll take you to a little bit of an explainer.</a></p>

<p><a data-t="543">But at a very high level, Workboots takes</a> <a data-t="543">your training data, generates a whole bunch of bootstrap resamples, then it fits many</a> <a data-t="551">models to each of these resamples. Or excuse me, it fits a model to each resample so that</a> <a data-t="556">you end up with many models. Thirdly, it takes each of these many models and predicts on</a> <a data-t="562">new data. So for each observation in your new dataset, you have many predictions. And</a> <a data-t="568">then finally, it summarizes all of these with a prediction range.</a></p>
