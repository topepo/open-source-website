<h3>The tidymodels framework</h3>

<p><a data-t="876">So the tidymodels framework is a – you can think of it as analogous to the </a><a href="/software/tidyverse/">Tidyverse</a><a data-t="876">.</a> <a data-t="885">So it's a collection of R packages.</a> <a data-t="888">So if you think about the Tidyverse being used for data manipulation,</a> <a data-t="893">data visualization, reshaping of data, munging of data, dealing with CSV files,</a> <a data-t="899">dealing with dates, you know, the tidymodels framework is an analogy</a> <a data-t="906">where all these different packages are focused on a different part of the machine learning process</a> <a data-t="913">with a specific focus on supervised machine learning.</a> <a data-t="917">So if you can install the tidymodels metapackage, and then just like, you know,</a> <a data-t="923">if you have done library tidyverse and then you've gotten </a><a href="/software/ggplot2/">ggplot2</a><a data-t="923">, dplyr, tidy R,</a> <a data-t="929">if you do library tidymodels, you get a bunch of packages that get, you know,</a> <a data-t="935">attached there so that you can use the functions from them.</a></p>

<p><a data-t="940">So I would first say if it feels overwhelming to get started to know there's a bunch of packages</a> <a data-t="947">and to not be sure which functions come from which individual package,</a> <a data-t="952">I would say try to step back from that feeling of discomfort and just kind of go with it.</a> <a data-t="958">Like, just – like, don't stress out too much if you don't know which functions come from which packages.</a> <a data-t="965">That's definitely something that comes as you get more experience using something.</a></p>
