<p><a data-t="0">My name is Eric Cronstrom with the Center for Charter Schools at Central Michigan University.</a> <a data-t="12">I'm going to tell you a story about my first project after joining the Center.</a></p>

<p><a data-t="18">Now, this project was a challenge, but we were able to automate a process that had been performed</a> <a data-t="23">manually for more than a decade.</a> <a data-t="29">On its own, the project was successful, but we turned it</a> <a data-t="29">into an opportunity to rethink how the organization approaches data science.</a></p>

<p><a data-t="35">This is a story about the technology and the techniques we used to automate a task</a> <a data-t="40">that seemed too complex for procedural document generation.</a> <a data-t="47">But more importantly, this is a story</a> <a data-t="47">about how to make a difference in an organization, about how to build trust and convince leadership</a> <a data-t="53">to try the new thing.</a></p>

<h3>The Annual Performance Report</h3>

<p><a data-t="62">One of the responsibilities we have as a charter school authorizer is to hold</a> <a data-t="62">our partner schools accountable for the academic performance of their students.</a> <a data-t="66">One of the ways we do this is with the Annual Performance Report.</a></p>

<p><a data-t="72">The Annual Performance Report, or APR, is a document that summarizes how well students in</a> <a data-t="72">each of our 58 partner schools are doing academically.</a> <a data-t="84">In the spirit of transparency,</a> <a data-t="84">these reports are made publicly available on the Center's website, and they're also distributed to</a> <a data-t="92">school boards and other stakeholders as print booklets.</a> <a data-t="98">The organization demands high production</a> <a data-t="98">values for these documents, and they have to be absolutely correct.</a> <a data-t="104">And to make no bones about it,</a> <a data-t="104">they look good.</a> <a data-t="111">And these publications are the primary responsibility of my new job.</a></p>

<p><a data-t="112">So try to picture yourself at a job interview.</a> <a data-t="120">Across the table is your new potential boss.</a> <a data-t="121">He's an impressive guy with a PhD in psychometrics.</a> <a data-t="126">He's explaining how important</a> <a data-t="126">these reports are and some of what goes into making them.</a> <a data-t="133">But there's a catch.</a> <a data-t="133">These reports</a> <a data-t="133">that usually take months to produce have to go to print in seven weeks.</a> <a data-t="140">Well, to quote Richard</a> <a data-t="140">Branson, if someone offers you an amazing opportunity but you're not sure if you can do it,</a> <a data-t="146">say yes and learn how to do it later.</a></p>

<h3>The old process</h3>

<p><a data-t="154">So how do you make an APR?</a> <a data-t="154">First things first,</a> <a data-t="154">you have to get your data.</a> <a data-t="161">There are a variety of different sources.</a> <a data-t="161">Individual test scores from</a> <a data-t="161">assessment vendors, student information from the Michigan Department of Ed, demographic and</a> <a data-t="167">comparison data from the state and other public entities, and so on.</a> <a data-t="174">Once you download everything,</a> <a data-t="174">you load it into the database.</a> <a data-t="179">Sometimes.</a> <a data-t="179">Some of it just stays as flat files.</a></p>

<p><a data-t="180">After sourcing all your information, you can start producing the visuals</a> <a data-t="185">with Excel.</a> <a data-t="192">A school can have up to 23 different visualizations depending on the grade levels</a> <a data-t="192">offered.</a> <a data-t="198">And there are nine different Excel templates that produce all of the charts and</a> <a data-t="198">graphs and tables required by the APRs.</a> <a data-t="205">At this point, any data that actually made it into the</a> <a data-t="205">database gets pulled back out and copied into an Excel template.</a> <a data-t="212">Then you manually output</a> <a data-t="212">the correct visuals for each school, validating the numbers as you go.</a> <a data-t="219">One at a time for 23 visuals</a> <a data-t="219">per school for 58 schools.</a></p>

<p><a data-t="228">Now the last link in this chain was Adobe InDesign, where the actual</a> <a data-t="228">documents get produced.</a> <a data-t="233">There's a master report template that gets updated with fresh content</a> <a data-t="233">every year.</a> <a data-t="240">And finally, one last script pulls everything together to produce one PDF for each</a> <a data-t="240">school.</a></p>

<h3>Deciding to try something new</h3>

<p><a data-t="248">This is where being a new person helps a bit.</a> <a data-t="248">Hiring a new team member is an opportunity</a> <a data-t="248">to have a fresh set of eyes, who, like the proverbial boiled frog, knows right away when</a> <a data-t="254">they're dropped into hot water.</a> <a data-t="260">Our tight deadline was also a blessing in disguise,</a> <a data-t="261">since we knew there wasn't enough time to do things the way they had done before.</a></p>

<p><a data-t="266">So with only six weeks left at this point, we sat down in the conference room and hashed out our plan.</a> <a data-t="274">This is a picture I took of the whiteboard after that meeting.</a> <a data-t="281">We figured we had three different</a> <a data-t="281">potential paths forward.</a> <a data-t="287">A fully automated solution built on R and rendering documents</a> <a data-t="287">with R Markdown, or a hybrid solution where we render the visuals with R, but still use InDesign</a> <a data-t="294">to pull the documents together.</a></p>

<p><a data-t="301">I should note that up until this point, nobody on the team,</a> <a data-t="301">myself included, had used much R.</a> <a data-t="308">Excel was the tool of choice for data visualization,</a> <a data-t="309">and most of any real analysis was performed in SAS.</a> <a data-t="315">So we gave ourselves only a short amount</a> <a data-t="315">of time to experiment with R.</a> <a data-t="321">And if we couldn't have a proof of concept for production-ready</a> <a data-t="321">visuals in 10 days, we would use Excel.</a> <a data-t="328">And if within a month we couldn't have an R Markdown</a> <a data-t="328">report that met our publication standards, we would use the InDesign templates.</a></p>

<p><a data-t="334">Like I said, I hadn't used much R up until this point.</a> <a data-t="341">I was comfortable with programming.</a> <a data-t="341">My background is in software development and automation.</a> <a data-t="347">I had some exposure to R</a> <a data-t="347">just through my graduate coursework, but the only R Markdown reports I ever created were</a> <a data-t="353">academic white papers.</a> <a data-t="361">And looking back, I think it was this discussion about the risks involved</a> <a data-t="361">and being open and honest about what I knew and what I didn't know and having a backup plan</a> <a data-t="368">is how I was able to convince the team to try something new.</a></p>

<blockquote><a data-t="361">And looking back, I think it was this discussion about the risks involved</a> <a data-t="361">and being open and honest about what I knew and what I didn't know and having a backup plan</a> <a data-t="368">is how I was able to convince the team to try something new.</a></blockquote>

<h3>Building the new report</h3>

<p><a data-t="374">Now that we had a plan and we</a> <a data-t="374">convinced the organization to let us experiment, we set out to recreate this report from the ground</a> <a data-t="379">up.</a> <a data-t="386">And the first thing we needed to do was prove to ourselves that we could create a publication</a> <a data-t="386">that adheres to our identity standards using R Markdown.</a> <a data-t="392">And there was one issue that almost</a> <a data-t="392">stopped us right away.</a></p>

<p><a data-t="398">So at this point in my career, I had been writing code professionally</a> <a data-t="398">for 10 years.</a> <a data-t="405">I've taught web development courses.</a> <a data-t="405">I've taught database design at a</a> <a data-t="405">graduate level.</a> <a data-t="412">The single issue I spent the most time on for this report was getting a box</a> <a data-t="412">with our CMU maroon background color with white text using a custom font in LaTeX.</a></p>

<p><a data-t="421">Maybe that just stands out as the most difficult technical challenge</a> <a data-t="426">because almost everything else was done with ggplot and ggplot is amazing.</a></p>

<p><a data-t="432">Regardless, it's not my proudest moment.</a> <a data-t="439">More seriously though, the reason this process had</a> <a data-t="439">been performed manually for so long is that each school is a little different.</a> <a data-t="445">Different schools</a> <a data-t="445">offer different grade levels.</a> <a data-t="452">Some of our schools are K through 12.</a> <a data-t="452">Some are just high schools and</a> <a data-t="452">some are kindergarten through third grade.</a> <a data-t="460">And beyond that, we evaluate the academic performance</a> <a data-t="460">of our schools based on the terms laid out in their individual contracts.</a> <a data-t="466">The educational</a> <a data-t="466">goals of many of our schools are similar, but some are operating under unique circumstances</a> <a data-t="473">and all of those differences need to be represented in the annual performance report.</a></p>

<p><a data-t="478">When you're manually producing visuals in Excel, you can tweak things on the fly to accommodate</a> <a data-t="484">all those variations.</a> <a data-t="492">Trying to automate around all that variability presents an interesting design</a> <a data-t="492">challenge.</a> <a data-t="500">There are two main strategies we employ to accommodate this, depending on the prevalence</a> <a data-t="500">of the variation and the impact that it has on the report.</a> <a data-t="507">You can always write flow control logic</a> <a data-t="507">into your R code so your program will behave differently depending on what school's report</a> <a data-t="513">is currently being rendered.</a> <a data-t="520">This is okay if it's a unique situation that only impacts one or two</a> <a data-t="520">schools, otherwise it can get out of hand quickly.</a></p>

<p><a data-t="529">What we try to do first is to generalize an issue</a> <a data-t="529">and develop a data structure that represents all possible variations.</a> <a data-t="536">It effectively turns part of</a> <a data-t="536">a school's contract into data.</a> <a data-t="543">A good example of this is you can have two schools that both offer</a> <a data-t="543">instruction at grades K through 8.</a> <a data-t="550">School A's contract stipulates they're to be evaluated on</a> <a data-t="550">the academic achievement of their students in grades 3 through 8.</a> <a data-t="557">School B's contract</a> <a data-t="557">says they are to be evaluated based on their students in grades 3 through 7, excluding grade 8.</a> <a data-t="564">Instead of a giant if-else chain in your code to keep track of all the contract grades,</a> <a data-t="569">you can develop a data structure that represents that information.</a></p>

<h3>Meeting the deadline</h3>

<p><a data-t="575">Going through this process, we identified more problems than we were able to fix right away.</a> <a data-t="582">When you have an aggressive timeline, you do what you have to do to get the job done.</a> <a data-t="587">To be fair, we only gave ourselves six weeks to learn ggplot, markdown, and LaTeX.</a> <a data-t="594">Our code was never going to be pretty, but I'm proud of what we managed to accomplish.</a></p>

<p><a data-t="600">We met our deadline with a couple days to spare.</a> <a data-t="608">The new fully automated reports look great.</a> <a data-t="613">With the push of a button, an R driver program renders one report per school</a> <a data-t="613">from a single R markdown template.</a> <a data-t="618">And the best part is it's completely reproducible.</a> <a data-t="619">Next year, we can push the same button again and get the same reports with updated data.</a></p>

<p><a data-t="625">That is, of course, until your communications department updates their brand identity</a> <a data-t="629">standards with new fonts and new logos and new rules.</a> <a data-t="635">Such is life.</a></p>

<h3>Building on success</h3>

<p><a data-t="636">So we sent our reports out to print on time, but we knew we weren't done.</a> <a data-t="642">We used success with R in this project to convince the organization to invest</a> <a data-t="648">more into our data infrastructure.</a> <a data-t="655">We just wrapped up the implementation of our </a><a href="/software/rstudio/">RStudio</a><a data-t="655"> Connect</a> <a data-t="655">server, and we have time carved out for research and development of dashboards and other analytics</a> <a data-t="661">projects.</a> <a data-t="668">My next big project is a </a><a href="/software/plumber/">Plumber</a><a data-t="668"> API that will feed data to a web app hosted on our</a> <a data-t="668">website.</a> <a data-t="676">Gone are the days of juggling flat files between projects.</a> <a data-t="676">Now I access data in my SQL</a> <a data-t="676">server directly using R with dbPlayer, which I'm convinced works on Blackmagic.</a></p>

<p><a data-t="685">Showing results is what convinced the organization to give us the time to invest in ourselves.</a> <a data-t="692">We've started adopting best practices around data management and software development.</a> <a data-t="698">Now all of the data that we use makes it into our data warehouse, and each data pipeline has an</a> <a data-t="705">automated integration services package.</a> <a data-t="712">Because as it turns out, Excel was our ETL tool as well as</a> <a data-t="712">our DataViz tool.</a> <a data-t="719">And all of the code and assets needed to build each solution are stored in one</a> <a data-t="719">spot in source control.</a> <a data-t="724">And we're able to spend more time building and doing work that we enjoy</a> <a data-t="732">because exporting hundreds of charts from Excel one at a time is super boring.</a> <a data-t="732">And we're able to</a> <a data-t="732">spend more time contributing research and at a time when research and education is critical.</a></p>

<blockquote><a data-t="685">Showing results is what convinced the organization to give us the time to invest in ourselves.</a></blockquote>
