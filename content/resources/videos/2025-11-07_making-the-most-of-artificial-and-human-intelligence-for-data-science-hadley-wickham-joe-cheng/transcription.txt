<h3>Tool calling</h3>

<p><a data-t="571">To go a little further with elmer, you're going to need to learn a little bit more about what how LLMs work today, or at least how the APIs that we work with LLMs work.</a> <a data-t="584">And if you read anything about LLMs, there is just so much jargon, so much bullshit to wade through.</a> <a data-t="591">And I think like one of the the services that I hope that we can provide at Posit is to like demystify some of that, to like dig through some of that bullshit so that you don't have to.</a></p>

<p><a data-t="600">And the first thing I want to talk about is this really powerful idea called tools, or tool calling.</a> <a data-t="608">And tools were invented to solve a pretty major problem with LLMs, namely that they don't have any access to data about the real world.</a> <a data-t="616">So if you ask an LLM like what day is it today, like best case scenario, it's going to say I don't know.</a> <a data-t="622">Worst case scenario, it's just going to make some shit up.</a> <a data-t="626">So that's where tools come in.</a> <a data-t="626">And if you know how to program, it's really easy to define what a tool is, because a tool is just a function, an R function, a Python function, along with some metadata that tells the LLM how to use it.</a></p>

<p><a data-t="642">So let's define a tool.</a> <a data-t="642">Here I'm going to define a tool that provides today's date.</a> <a data-t="645">It's a function that's going to call the base sys.date function, which just returns today's date.</a> <a data-t="648">I'm going to give it some metadata, a name, a description, the arguments and takes, in this case none, and then I can register it.</a> <a data-t="656">And once I've done that, if I ask the LLM again what day is it today, it can now give me the correct answer.</a></p>

<p><a data-t="666">So how does this work?</a> <a data-t="666">It would be pretty surprising if every LLM provider was like running R in the back end and could somehow magically run all of my R code.</a> <a data-t="676">That's not quite what's happening.</a> <a data-t="680">So when I send this question to the LLM, it does a little bit of thinking and it realizes, okay I'm going to need to call this tool.</a> <a data-t="684">So it's going to send that request back to me.</a> <a data-t="689">Then elmer or Chatlus is going to receive that request, call that function, send the result back to the LLM, and now it can generate an answer that goes back to me, the human.</a> <a data-t="697">So this is really powerful because it allows anything you can do in your R session or in your Python session, you can now have an LLM do for you.</a> <a data-t="709">Really cool, really powerful, but also really dangerous, as you can imagine, and as I'm going to demonstrate with my next slide.</a></p>

<p><a data-t="718">So here I'm going to empower my LLM with a tool that allows it to run R code.</a> <a data-t="724">Now I'm showing this mostly for the purposes of illustration, like you probably shouldn't do this, right?</a> <a data-t="727">Because now the LLM can do anything you can do in R, and that is like pretty much anything.</a> <a data-t="734">Like you could ask this LLM to delete every file on my computer and potentially it could do so.</a> <a data-t="738">But what's really interesting, and I think one of the reasons that people are so excited about LLMs at the moment, is if I give this tool and I re-ask that initial question, what's the mean of x and y, one of their correlations, the LLM will go and actually, instead of trying to compute it incorrectly itself, it'll write a bunch of R code, which does the computation correctly, and now I get the correct results.</a></p>
