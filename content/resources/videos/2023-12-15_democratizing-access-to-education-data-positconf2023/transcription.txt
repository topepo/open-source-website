<h3>The gap between data availability and accessibility</h3>

<p><a data-t="63">So every year, a handful of government agencies do the hard and important work of collecting</a> <a data-t="68">detailed data on K-12 schools, school districts, and in colleges in the U.S.</a> <a data-t="73">The goal of the Education Data Portal is to put all of this data under a single roof by</a> <a data-t="77">providing a freely available one-stop shop for all of the major national data sets released</a> <a data-t="83">by these agencies.</a></p>

<p><a data-t="86">In many cases, these agencies have been collecting this data for a couple of decades, and as</a> <a data-t="90">you can imagine, there have been lots of changes to data formats, file structures, classifications</a> <a data-t="95">over the years.</a> <a data-t="96">So our goal is to do the hard work of harmonizing that data, that way we can do that work and</a> <a data-t="102">offload that from researchers and data scientists and other folks trying to use that data.</a> <a data-t="108">When I say we, I really mean our army of research assistants who do that hard work.</a> <a data-t="112">So much kudos to them.</a></p>

<p><a data-t="115">Ultimately, we do this to make it easier for both technical and non-technical users to</a> <a data-t="120">look at trends over time and combine data from these different sources and just to put</a> <a data-t="125">that data at their fingertips a little easier.</a></p>

<h3>Data documentation as a first-order feature</h3>

<p><a data-t="632">So the second thing that I think the education data portal does really well, that I think</a> <a data-t="636">really provides a major value-added over the existing data, is that it really treats data</a> <a data-t="644">documentation as a first-order feature, and a first-order value-add, and not as an afterthought,</a> <a data-t="649">where I think data documentation, and documentation in general, tends to fall.</a> <a data-t="654">So I was told that showing a slide with 1,400 variables wouldn't be very aesthetically pleasing, so instead</a> <a data-t="660">I'm going to focus on two things that I think really just provide a lot of value, the first</a> <a data-t="665">is that data documentation really is written for both humans and machines, and for anyone</a> <a data-t="670">who's seen Johnny Bryan's great talk on naming things, I think a lot of the same principles</a> <a data-t="674">about thinking about who your end user is, whether that is a human or a machine, and</a> <a data-t="680">think about making your work accessible to them, I think that also really applies to</a> <a data-t="684">documentation here, especially data documentation dictionaries, and the second thing is that</a> <a data-t="690">the documentation really provides the user with details on demand, and this is borrowed</a> <a data-t="694">from a design principle, and I'll talk about these in a little bit more detail now.</a></p>

<p><a data-t="701">So when I say data documentation is written for both humans and machines, here's an example</a> <a data-t="705">of a variable, degree of urbanization, I as a human think this is written for me, like</a> <a data-t="711">I think I know what it's telling me, I have the relevant information, it's pretty user-friendly</a> <a data-t="716">for someone looking at it.</a> <a data-t="723">Here's that same documentation, or data documentation, but</a> <a data-t="727">provided in a way that I think is machine-readable, so I'm making an API call to the API variables</a> <a data-t="730">metadata endpoint that I mentioned earlier, you can get the exact same information, so</a> <a data-t="735">you get the label, you get the data type, and then you've got standardized formats of</a> <a data-t="719">the values written in a way that's for machines.</a> <a data-t="724">Since there are a couple of obvious advantages of this, maybe you're someone who's trying to make hundreds of plots, and you want to</a> <a data-t="729">automate adding labels, or maybe you're someone who wants to add captions, you're building</a> <a data-t="732">a dashboard, I think there are many numbers of things that you can do that really get</a> <a data-t="736">a lot of value from having this data documentation be written, again, both for humans on the</a> <a data-t="742">left, and also in a way that can be programmatically accessed on the right.</a></p>
