<p><a data-t="549">So what is the current state of affairs?</a> <a data-t="552">Most computational science, honestly, is born in notebooks.</a> <a data-t="555">You start sort of fussing around with your data</a> <a data-t="558">at the beginning of your analysis,</a> <a data-t="560">and then it builds into a bigger project.</a> <a data-t="565">And it, well, either dies or ends in Word documents.</a> <a data-t="569">That ends up being the culminating thing</a> <a data-t="571">that you submit to a journal.</a> <a data-t="573">And for many projects, that's where things end.</a> <a data-t="576">That ends up being the published work.</a></p>

<p><a data-t="623">This notion of, I mean, the peer review</a> <a data-t="626">and publication workflow, in a way,</a> <a data-t="627">do not support notebooks as research outputs.</a> <a data-t="630">They ask for the static Word or PDF documents.</a> <a data-t="634">Additionally, more complex scenarios</a> <a data-t="639">involve a lot of manual finagling</a> <a data-t="641">to bring the project to a journal submission stage.</a> <a data-t="644">You might have a bunch of script files</a> <a data-t="647">in a variety of languages, maybe a bunch of notebooks,</a> <a data-t="649">where you bring things together.</a> <a data-t="651">But ultimately, if you don't have the style file</a> <a data-t="654">that goes along with that journal</a> <a data-t="657">that you might be submitting to,</a> <a data-t="660">you're probably going to be sort of breaking</a> <a data-t="663">the reproducibility cycle</a> <a data-t="665">and maybe manually editing that document.</a></p>
