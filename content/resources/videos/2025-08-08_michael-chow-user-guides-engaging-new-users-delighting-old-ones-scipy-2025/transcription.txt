<h3>FastAPI: task support and backwards chaining</h3>

<p><a data-t="706">So next I'm gonna look quickly at FastAPI</a> <a data-t="709">which is a really interesting tool I think with a really nice approach to user guides.</a> <a data-t="715">So the way FastAPI works is FastAPI is a tool for building, it's a web framework</a> <a data-t="724">so you can deploy web applications like this one shown in the browser.</a> <a data-t="731">The tricky thing is that for a web application is often there's something in the middle.</a> <a data-t="736">So usually the process is you might write some FastAPI code</a> <a data-t="741">and then there's this middle piece called a web server which is not necessarily FastAPI</a> <a data-t="748">and then you interact with it in the web browser.</a></p>

<p><a data-t="751">So there's actually a lot of context in which you're interacting with different things.</a> <a data-t="755">You might be in an IDE writing code.</a> <a data-t="757">You might be using something like, actually I've never said this word out loud, GUnicorn?</a> <a data-t="806">Well, I mean, one of those two is right to serve it</a> <a data-t="810">and then you might pop open your web browser to interact with it.</a> <a data-t="814">So FastAPI has a hard problem because you could be in three different contexts when you're using FastAPI.</a></p>

<p><a data-t="821">The way they handle this is that they give you a simplest whole task</a> <a data-t="826">which is essentially like doing this as fast as possible.</a> <a data-t="830">So they just hand you the code and they say copy this into a file</a> <a data-t="834">and then they just give you this special command FastAPI dev</a> <a data-t="839">which runs the web application.</a> <a data-t="842">So essentially they try to lift you past these pieces</a> <a data-t="847">so that you can get into the web browser as fast as possible.</a></p>

<p><a data-t="852">And this is a real risk that I see sometimes with these tools</a> <a data-t="856">is some tools instead will tell you about the middle thing in depth and how you can use it</a> <a data-t="863">or they're actually like, probably like five different web servers you can use</a> <a data-t="867">and some things take the chance to kind of tell you about each one</a> <a data-t="871">but I think FastAPI does a good job of just actually telling you about none of them</a> <a data-t="875">and getting you to the kind of end product.</a> <a data-t="879">You'll have time later to learn about the middle part</a> <a data-t="882">but FastAPI wants you to know what it does well for you.</a></p>

<p><a data-t="887">So this is showing a bit of what FastAPI did there.</a> <a data-t="891">So they gave you task support.</a> <a data-t="893">They just handed you the code.</a> <a data-t="895">They just handed you this special command</a> <a data-t="898">and they got you to pop open a web browser.</a> <a data-t="901">That's actually a ton of work.</a> <a data-t="903">I think that FastAPI got you super far</a> <a data-t="907">and I think if you're new to programming or new to web applications</a> <a data-t="912">they got you through a ton of stuff really fast.</a></p>

<p><a data-t="958">The other interesting thing is their sidebar structure looks like this.</a> <a data-t="961">This is a bit in the weeds but they talk a lot about concepts</a> <a data-t="966">but rather than...
