<p><a data-t="537">So if we could reimagine what actually this day one looks like, we'd want to go off into the cloud.</a> <a data-t="545">So we want to go towards some kind of cloud computing.</a> <a data-t="548">Because the steps here, right, are you would have to log into an environment, you'd have to select a project, and then you'd have to start exploring.</a> <a data-t="598">These three steps, super quick, all done inside of a web browser.</a></p>

<p><a data-t="604">But what if we could drop one of the steps?</a> <a data-t="608">When I say drop one of the steps, I mean, just gently modify.</a> <a data-t="612">So instead of having the student find the project, you could just use a badge.</a> <a data-t="618">So the badge would effectively take you into the login screen, if they're not already logged in.</a> <a data-t="623">And then you immediately get them into an environment that's pre set up for you to go off and explore.</a> <a data-t="630">This feature of Pulsate Cloud, or in this case, RStudio Cloud before, was really, really appealing.</a></p>

<h3>How Quarto WebR works</h3>

<p><a data-t="851">Instead of having to go out to post-cloud, what we can do is we can use a compute server.</a> <a data-t="857">Or not, we can use the compute server, but we can move away from this paradigm of needing a compute server, where we have R running on that server, and then we have this linked connection between R and then the student's computer.</a> <a data-t="906">Instead, what we can do is we can move over to just using any kind of web server that if it can serve a file, so that is it can allow a student to download a file, they have a working copy of R right in their web browser without sort of us worrying about it.</a></p>

<p><a data-t="931">So this then takes us to the third huge breakthrough, and that is WebR itself.</a> <a data-t="936">So on the left-hand side, you can see some R code where you're sort of exploring how to iterate through items in a list.</a> <a data-t="946">And then on the right side, you can sort of see the same code that would allow you to work with R, except directly inside of a web browser.</a> <a data-t="957">So this allows us, again, to just move away from really needing a compute server and just sort of having a server that we can use to download files, where if the student has the file, then basically they can use R right in their web browser.</a></p>

<p><a data-t="1021">So it might seem a little bit complicated.</a> <a data-t="1025">So let's sort of try and show what this unification path really is between these three technologies.</a> <a data-t="1025">So to start, we have Quarto right inside of RStudio.</a> <a data-t="1030">Next, we're going to go ahead and press that render button, and lo and behold, it's going to render into an HTML document.</a> <a data-t="1039">Now, again, everything that you're seeing inside of the blue rectangle, that's all been powered by Quarto.</a></p>

<h3>Installing and configuring the extension</h3>

<p><a data-t="1169">And in terms of making this available, the Quarto WebR extension really only needs to be made available on the person's computer that's rendering the document.</a> <a data-t="1180">So that means that you do need to install a piece of software.</a> <a data-t="1200">The easiest way here is just to go on the lower left-hand side of your RStudio window, press the terminal tab, which is right next to your console, and then type a quick install command of Quarto add codeless forward slash Quarto dash WebR, press enter.</a> <a data-t="1222">And you have two quick prompts to verify that you do indeed want to install the extension.</a></p>

<p><a data-t="1227">So on that end, right, we now sort of have it installed.</a> <a data-t="1236">What we can do is we can sort of make our existing HTML documents inside of Quarto dynamic just by sort of changing four things.</a> <a data-t="1245">The first part that we have to change is we have to modify the documents YAML to incorporate an engine parameter.</a> <a data-t="1253">So if we say engine knitter, this makes sure that all of the code cells are going to be run using knitter as the compute engine.</a> <a data-t="1260">The alternative here is everything would go through Jupyter, which would require you to install more software.</a> <a data-t="1265">From there, we need to add a filter.</a> <a data-t="1272">This filter, again, is that Quarto extension along us to incorporate WebR.</a> <a data-t="1281">And then the third part here is we switch the code cell type from R to WebR dash R.</a> <a data-t="1285">So once those three are done, we can just render the document.</a> <a data-t="1292">And lo and behold, we get an interactive document that we can use just about anywhere that there is a static web server host to share our content and have students learn.</a></p>
