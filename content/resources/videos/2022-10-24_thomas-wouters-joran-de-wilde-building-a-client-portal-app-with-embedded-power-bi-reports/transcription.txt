<h3>Embedding Power BI in Shiny</h3>

<p><a data-t="509">If you want to use a dashboard that was written in Power BI in our Shiny application the first question that arised for us was whether or not we should.</a> <a data-t="518">We could also just migrate the existing visuals to our Shiny application.</a> <a data-t="523">This way Power BI serves more as a design tool where we kind of see what works and what doesn't and the final visuals are recreated in our Shiny application.</a></p>

<p><a data-t="534">This is an interesting approach if you have dashboards that don't evolve a lot, that don't change a lot and you have people who feel comfortable working in R and Shiny.</a> <a data-t="485">The option of embedding is more interesting if you have dashboards that do change and evolve a lot.</a> <a data-t="497">This way you can update your dashboards without having to republish your app every time.</a> <a data-t="502">Furthermore you can have a team that's really dedicated on maintaining the Power BI dashboards who don't need to have any Shiny knowledge.</a></p>

<blockquote><a data-t="502">Furthermore you can have a team that's really dedicated on maintaining the Power BI dashboards who don't need to have any Shiny knowledge.</a></blockquote>

<p><a data-t="512">For our case we were definitely in the second category where we had a team of people who didn't necessarily possess the correct amount of Shiny knowledge and we had dashboards that changed and evolved a lot.</a></p>

<p><a data-t="526">How does embedding a Power BI dashboard work exactly?</a> <a data-t="530">The first thing we do is write the application and the general structure of the application itself in R and Shiny.</a> <a data-t="534">Next we use our Power BI and its drag and drop features to create a nice looking dashboard and thereafter we simply embed the dashboard via a nice and clean way into our app.</a> <a data-t="546">This way we get a portal that thanks to a unified UX and style, it's very difficult for the customers to notice where your Power BI dashboard ends and our Shiny application begins.</a></p>

<p><a data-t="565">Of course to be able to embed a Power BI dashboard you'll need a few things.</a> <a data-t="570">First of all of course the Power BI desktop software because with this software you'll be creating your Power BI dashboards.</a> <a data-t="574">You'll also definitely need the Power BI service because after all that is where your reports will be published to.</a> <a data-t="582">With the Power BI service you could technically embed your Power BI dashboards already using iFrames however if you want to embed it using the Power BI API you'll also need Power BI dedicated capacity which you can either get via Power BI Premium or Power BI Embedded.</a> <a data-t="600">Keep in mind however that this option will be a bit more pricey but I hope to convince you in the next slide why this extra cost actually might be worth it.</a></p>

<p><a data-t="610">So the easiest solution is using iFrames to embed your Power BI dashboard however this comes with some very big disadvantages.</a> <a data-t="620">First you'll have to authenticate again in each iFrame which means that the customer will have to log in for each dashboard that it wants to see and there are also no interaction possible between your Power BI dashboard and your Shiny app.</a> <a data-t="642">These disadvantages are negated by using API embedding because thanks to the Power BI API you can integrate an automatic authentication and with the API you can also catch all the interactions which means that you can actually have your Shiny app communicate with your Power BI dashboard.</a> <a data-t="664">This way you can create bookmarks, apply and delete filters and many many many more options.</a></p>

<h3>Technical implementation</h3>

<p><a data-t="669">How does one embed a Power BI dashboard into an R Shiny app you may ask?</a> <a data-t="675">Well fortunately for us Microsoft offers a lot of examples on how to embed a Power BI dashboard using the API.</a> <a data-t="685">Unfortunately however there is no clear example of how to do it using R Shiny.</a> <a data-t="690">Lucky for us and probably unlucky for others there's JavaScript to the rescue.</a> <a data-t="690">Via the Power BI JavaScript library you can actually embed a Power BI dashboard using JavaScript.</a></p>

<p><a data-t="704">This means that your R Shiny app and the JavaScript files will have to communicate with each other.</a> <a data-t="709">On the R Shiny website there's already a very nice overview of how to set up the interactions between your R and your JavaScript.</a> <a data-t="719">A brief overview, so send custom meshes can be used in your R application to trigger JavaScript functions or JavaScript code.</a> <a data-t="731">These triggers can be read and caught using the add custom message handler in your JavaScript files and to send input back from your JavaScript to your Shiny application the Shiny dot on input change button can be used.</a></p>

<p><a data-t="742">A quick example of how we use these functions in our app.</a> <a data-t="748">So when a user navigates in your R Shiny app to a page where there should be a Power BI report we will call the send custom message function which will call the embed API function in your JavaScript files.</a> <a data-t="761">With this custom message we'll also send some parameters for our report.</a> <a data-t="768">Next in the Power BI embed dot JS JavaScript file the add custom message handler will catch this call embed API and using some magic JavaScript code we can actually and using the library we can embed the Power BI dashboards at a specific container in our Shiny app.</a> <a data-t="783">When it's done rendering we will let our app know that the report has been rendered by using the set input value and our Shiny app can continue with its next step or steps in the process.</a></p>

<p><a data-t="797">To realize the authentication and the more complex interactions some more complex embedding steps are needed.</a> <a data-t="804">We won't go into too much detail in this talk but I can share with you already the tip of the embedding iceberg.</a> <a data-t="812">For authentication we use the Azure auth package this together with the </a><a href="/software/httr/">HTTR</a><a data-t="812"> package for retrieving the embed token made it possible to integrate the authentication within our Shiny app so no more two different authentications necessary.</a> <a data-t="839">The Power BI API was then used to create bookmarks to apply and delete filters and so on and so on.</a></p>

<p><a data-t="841">I want to conclude with a brief overview of how to embed a Power BI dashboard into a Shiny app.</a> <a data-t="849">For everyone just tuning in this is probably one of the most important slides of our talk.</a> <a data-t="854">So we start by getting an access token via Azure with this access token we can get the embed URL from the Power BI API and with this URL together with JavaScript we can render our Power BI reports in a container in our app.</a> <a data-t="872">Keep in mind however that you will need Power BI embedded capacity for this to work.</a></p>
