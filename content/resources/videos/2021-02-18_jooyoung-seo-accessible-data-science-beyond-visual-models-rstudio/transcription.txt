<h3>Data science beyond vision</h3>

<p><a data-t="196">Now that you got how I interact with computer, let's talk about data science.</a> <a data-t="203">Data science involves a cycle of interaction with data, such as importing, tidying, transforming,</a> <a data-t="216">visualizing, and modeling.</a> <a data-t="219">And it feels like data science is highly vision-dominant.</a> <a data-t="223">Yes, it could be.</a></p>

<p><a data-t="233">But data science is more than what is seen.</a> <a data-t="239">If data science was restricted to only vision domain, that would mean that people who cannot</a> <a data-t="247">employ vision sense can't do data science at all.</a> <a data-t="253">Very fortunately, however, that is not true, because we have R, which enables blind people</a> <a data-t="261">see data beyond visual models.</a></p>

<blockquote><a data-t="233">But data science is more than what is seen.</a> <a data-t="239">If data science was restricted to only vision domain, that would mean that people who cannot</a> <a data-t="247">employ vision sense can't do data science at all.</a> <a data-t="253">Very fortunately, however, that is not true, because we have R, which enables blind people</a> <a data-t="261">see data beyond visual models.</a></blockquote>

<p><a data-t="266">I believe R is one of the most accessible data science environments because of the following</a> <a data-t="273">benefits.</a> <a data-t="275">First, command line interface.</a> <a data-t="279">As opposed to point-and-click graphical user interface, which requires mouse input interaction,</a> <a data-t="287">R is a command line environment.</a> <a data-t="291">This means that you can do everything within terminal using keyboard only.</a> <a data-t="297">This is a good benefit for keyboard users like me.</a></p>

<p><a data-t="301">Second, the beautiful interplay between reproducibility and accessibility.</a> <a data-t="309">R Markdown is a plain text-based document for reproducible report that can be written</a> <a data-t="317">with any text editor and can be rendered into multiple formats, including HTML, Word, PDF,</a> <a data-t="326">EPUB, and more.</a> <a data-t="329">These outputs, in many cases, are made readily accessible to assistive technologies.</a> <a data-t="337">Especially, the HTML output is super accessible.</a></p>

<p><a data-t="343">Last but not least, there are a number of accessible packages out there, thanks to its</a> <a data-t="350">open-source nature.</a></p>

<h3>Shiny accessibility improvements</h3>

<p><a data-t="946">I'm going to talk a little bit about Shiny.</a> <a data-t="951">I'm super excited to announce that Shiny is getting accessible.</a> <a data-t="956">With the amazing support of Winston, Carson, Barrett, and other Shiny team members, I was</a> <a data-t="963">working for making Shiny accessible for my internship activity at RStudio.</a> <a data-t="968">As a result, you will see many accessibility improvements in the next Shiny release, version</a> <a data-t="975">1.6 or higher.</a> <a data-t="979">If you want to test the development version, that's available on GitHub, and you can install</a> <a data-t="985">it through remote package.</a></p>

<p><a data-t="990">Some significant improvements are as follows.</a> <a data-t="994">First, we've added PayPal's bootstrap accessibility plugin under the hood.</a> <a data-t="1002">So alert, tooltip, popover, model dialogue, dropdown, tab panel, collapse, and carousel</a> <a data-t="1012">elements are made keyboard accessible.</a></p>

<p><a data-t="1025">Next, there are many significant improvements made for Shiny widgets.</a> <a data-t="1035">Select input, especially selectize input, file input are now fully keyboard accessible.</a> <a data-t="1045">Font awesome and glyphic icon now produces appropriate labels based on icon names for</a> <a data-t="1053">screen readers.</a> <a data-t="1056">Media buttons and checkbox group input are properly grouped together for assistive technologies.</a> <a data-t="1067">Date input and date range input are properly labeled.</a> <a data-t="1075">And now dynamic contents within all the output and update input are auto announced to screen</a> <a data-t="1083">readers and refreshable bread display.</a></p>

<p><a data-t="1091">There are also some great enhancements for semantic accessibility.</a> <a data-t="1097">We can pass alt text to render plot function to specify alternative text description for</a> <a data-t="1105">screen readers.</a> <a data-t="1108">It could be a static text.</a> <a data-t="1112">Not object is given by default.</a> <a data-t="1116">Or you can even use reactive function for alt term to create dynamic text that is useful</a> <a data-t="1124">when you want to create reactive alt text in response to user input.</a></p>

<p><a data-t="1132">And you can explicitly set the lang code, language code, used within your Shiny app</a> <a data-t="1142">by passing lang parameter to float page function.</a> <a data-t="1148">This helps search engine parsers and assistive tech better identify document language.</a> <a data-t="1157">Last but not least, semantic landmarks have been applied to main panel as well as sidebar</a> <a data-t="1164">panel for screen readers to quickly navigate through Shiny apps in a logical manner.</a></p>
