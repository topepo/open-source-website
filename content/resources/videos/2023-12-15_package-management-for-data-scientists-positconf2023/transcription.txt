<h3>What is a package?</h3>

<p><a data-t="107">So what is a package?</a> <a data-t="107">The way I kind of define a package is it's an encapsulated</a> <a data-t="107">piece of code or software.</a> <a data-t="113">These are shared or installed to enhance or extend existing</a> <a data-t="113">functionality.</a> <a data-t="119">So in R, you use install.packages.</a> <a data-t="119">In Python, you might use pip install or conda or</a> <a data-t="119">pipenv.</a> <a data-t="126">You have a lot more options.</a> <a data-t="126">And then some examples, we use </a><a href="/software/ggplot2/">ggplot2</a><a data-t="126"> to create graphs and</a> <a data-t="126">plots and requests to make HTTP requests.</a></p>

<p><a data-t="134">So to make sure everyone's on the same page, we install</a> <a data-t="134">a package using install.packages.</a> <a data-t="141">We use a library command in R to import a package, and then we can</a> <a data-t="141">use that package in our code to run the functions, in this case, creating a scatterplot.</a> <a data-t="148">Same thing</a> <a data-t="148">in Python, we install a package.</a> <a data-t="154">This happens at the command line in Python using pip.</a> <a data-t="154">We can import</a> <a data-t="154">that package, and then we can use it in our code.</a></p>

<h3>Conflicting versions</h3>

<p><a data-t="161">So the first challenge that I want to talk about</a> <a data-t="161">is the conflicting versions challenge.</a> <a data-t="166">We talked about this a little bit with snapshots, but</a> <a data-t="167">packages generally use other packages.</a> <a data-t="173">So this is an example of a dependency tree.</a> <a data-t="173">It's a little</a> <a data-t="173">dated, but the concepts are still there.</a> <a data-t="179">Say we want to install some package.</a> <a data-t="179">We call that a direct</a> <a data-t="179">dependency.</a> <a data-t="186">That package, just like we want to enhance our software, it wants to enhance itself</a> <a data-t="192">and not have to do all the little things.</a> <a data-t="192">So it has indirect dependencies.</a> <a data-t="192">The thing that we really</a> <a data-t="199">want to focus on here is packages that have many lines going to them.</a> <a data-t="199">So those are packages with a</a> <a data-t="205">number of reverse dependencies like we see down here in the bottom.</a></p>

<p><a data-t="205">So with that information,</a> <a data-t="206">we can look at a conflicting versions example.</a> <a data-t="206">So say we have package A, and that relies on some</a> <a data-t="212">version of Rcpp, call it 1.0.6.</a> <a data-t="212">This is all good and dandy until we want package B,</a> <a data-t="220">which requires Rcpp 1.0.7.</a> <a data-t="220">So that probably won't work.</a> <a data-t="220">But what should we do here?</a> <a data-t="226">We can say</a> <a data-t="226">we'll just always use Rcpp 1.0.6 for package A, and we'll use 1.0.7 for package B.</a> <a data-t="239">A lot of</a> <a data-t="239">languages won't actually allow that, but even if they did, it would be pretty hard to manage.</a> <a data-t="243">We could potentially upgrade package A, but that will depend if the author has actually upgraded</a> <a data-t="249">package A in GitHub or on CRAN.</a> <a data-t="257">And then we could try downgrading package B, but you run into the</a> <a data-t="257">same problem.</a> <a data-t="266">That author might not have a version that works with Rcpp 1.0.6.</a></p>

<p><a data-t="266">So this fictitious</a> <a data-t="266">example comes from Stack Overflow.</a> <a data-t="274">I found this annotation hub installation of Rcpp 1.0.6 is</a> <a data-t="274">already loaded, but you know you need 1.0.7.</a> <a data-t="281">So looking at the solutions from Stack Overflow,</a> <a data-t="282">first one user recommended using </a><a href="/software/renv/">renv</a><a data-t="282">.</a> <a data-t="290">That ended up working for the person who ran into the issue,</a> <a data-t="290">but at the end of the day that was just downgrading annotation hub.</a> <a data-t="294">The next user said, update all of the installed packages, delete all of your other files, create</a> <a data-t="299">a new environment, and that solves your problem.</a> <a data-t="299">I don't know if you want to do that every single</a> <a data-t="304">time.</a> <a data-t="310">And then same thing, upgrade R, which is basically a new environment, and then everything</a> <a data-t="310">should work for you again when you install all your packages.</a></p>

<p><a data-t="315">So we should know how to solve</a> <a data-t="315">these things, but we should learn to prevent them in the first place.</a> <a data-t="321">And I have renv on this slide,</a> <a data-t="322">but maybe it should say slushy.</a> <a data-t="329">We should use tools to create reproducible environments.</a> <a data-t="329">So</a> <a data-t="329">we have renv lock, we have requirements.txt in Python, and then we should use snapshots for</a> <a data-t="335">package installation.</a> <a data-t="342">So mini-cran is one R package that you can use to grab subsets of cran.</a> <a data-t="342">Bandersnatch is one in Python, and then it was mentioned a couple times previously,</a> <a data-t="347">but we also have posit public package manager.</a> <a data-t="353">If you haven't seen that before,</a> <a data-t="353">this is what the view kind of looks like as you're selecting a snapshot that you</a> <a data-t="356">want to always install packages from.</a> <a data-t="361">This works really well on the R ecosystem,</a> <a data-t="361">because those packages on any given date are more likely to be compatible with one another,</a> <a data-t="366">and then it works really well in Python, because if a package is randomly deleted from PyPI,</a> <a data-t="370">which happens sometimes when authors get mad, then you also get those.</a></p>

<h3>System dependencies</h3>

<p><a data-t="374">So the next challenge I want to talk about are system dependencies.</a> <a data-t="380">Like packages also use other packages, packages also use other software.</a> <a data-t="387">So you can have R that</a> <a data-t="387">relies on Python using </a><a href="/software/reticulate/">reticulate</a><a data-t="387">, but you can have R and Python that rely on a bunch of different</a> <a data-t="394">pieces of software.</a> <a data-t="394">So your Python code might only work on Windows, your R code might only</a> <a data-t="400">work with a specific version of C++, Python might require OpenSSL, whatever it is,</a> <a data-t="406">and so this can make it pretty hard to move these things around.</a></p>

<p><a data-t="411">So as an example again, say we want to install the cryptography package in Python.</a> <a data-t="419">So we run pip install cryptography, and then we get some esoteric error that we don't have</a> <a data-t="425">something installed.</a> <a data-t="434">So what should we do now?</a> <a data-t="434">We can google how do you install GCC and then try</a> <a data-t="434">this command again and hope it works.</a> <a data-t="439">We can try to find a binary that's pre-installed and maybe</a> <a data-t="439">that will solve our problem.</a> <a data-t="446">And this again comes from Stack Overflow, I actually found a ton of</a> <a data-t="446">these.</a> <a data-t="452">And then looking at the solutions, if you're on Windows, you'll need to make sure you</a> <a data-t="452">have OpenSSL installed.</a> <a data-t="458">For Debian or Ubuntu, you need to run some other command.</a> <a data-t="460">And there were more solutions for Mac OS, RHEL, Fedora, Raspberry Pi, etc.</a></p>

<p><a data-t="466">So again, we can prevent this from happening in the first place.</a> <a data-t="471">We can use tools to easily manage</a> <a data-t="471">and recreate environments.</a> <a data-t="471">We can use Conda, Docker, PipM, even Workbench works for this.</a> <a data-t="477">We can use pre-compiled binaries when possible.</a> <a data-t="485">So PyPI will serve wheels which differ by package,</a> <a data-t="485">so some packages will have pre-compiled wheels and other packages won't.</a> <a data-t="489">CRAN generally serves R packages built for Windows and Mac OS.</a> <a data-t="495">R-Universe also</a> <a data-t="495">serves packages built for Windows and Mac OS.</a> <a data-t="501">Public Package Manager adds some Linux distributions,</a> <a data-t="501">but in some of these cases, we still need system dependencies installed for things to work.</a> <a data-t="507">There's no kind of silver bullet here.</a></p>
