<h3>Fairness is about our beliefs</h3>

<p><a data-t="150">So this kind of gets at the first point that I want to make in this talk, the first point of three, which is that fairness is not just about statistical behavior.</a> <a data-t="162">And folks that are working in machine learning who are used to being able to operationalize our beliefs somewhat straightforwardly into evaluation metrics, we really need to take a step back and realize that when we're assessing models for what we believe to be fairness, we need to realize that we're coming to this evaluation task with beliefs.</a> <a data-t="200">And those beliefs are going to translate into whether we think our model is doing well or not.</a> <a data-t="209">So the same model parameters can result in behavior that feels totally benign when situated in one context and deeply unjust in another.</a></p>

<h3>Defining fairness is hard</h3>

<p><a data-t="348">The second hard part that I want to try to underscore is that defining fairness is really hard.</a> <a data-t="355">The translation of our values and our morally held beliefs into these mathematical measures that we can evaluate is not at all a trivial task, and it doesn't do anything to resolve the differences in our morally held beliefs.</a></p>

<p><a data-t="378">So I'll try to argue that back to this home value example.</a> <a data-t="382">So again, this is a bad model.</a> <a data-t="384">It's a bad model in kind of a nice way where all the errors are correlated with each other.</a> <a data-t="391">And so we have statistical techniques to correct for this.</a> <a data-t="394">So one kind of argument that a lot of people have in what makes a fair model is that a fair model should at least be performant by our usual metrics.</a> <a data-t="406">So earlier in this session, we mentioned R-squared and RMSE.</a> <a data-t="411">So let's see if we can get this model as performant as possible with respect to those metrics and see if that brings us closer to something that feels fair to us.</a></p>

<p><a data-t="418">So again, because these are correlated in a really nice way, we can apply a statistical technique called calibration.</a> <a data-t="424">If we're lucky, we end up with a plot that looks something like this, where the errors are independently and identically distributed.</a> <a data-t="432">And so maybe this is better.</a> <a data-t="434">Let's take a look and make sure that those errors are independently and identically distributed.</a> <a data-t="438">What that means is that the variance and the mean of the errors is constant across the outcome, the distribution of the outcome.</a> <a data-t="447">So it looks to be just about the case.</a> <a data-t="449">The errors are similar for a house that's $100,000 and $500,000.</a> <a data-t="455">And to me, that feels like exactly the problem.</a> <a data-t="458">If I'm the owner of an $100,000 house, and I'm just as likely to receive an error of $50,000 in the assessment of my home as somebody who owns a $500,000 house, that error as a percentage is much more impactful to me.</a></p>

<p><a data-t="493">So maybe we can evaluate this model from a different perspective.</a> <a data-t="497">The percentage error across the distribution, if that's consistent, maybe that's a better model for me in terms of what I think is fair.</a> <a data-t="509">So we can train a model that optimizes the mean percentage error across the distribution.</a> <a data-t="514">It looks something like this.</a> <a data-t="515">To me, that feels better.</a> <a data-t="517">But at the same time, people that are so used to seeing these metrics like r squared or root mean squared error, that they might see that our model is worse.</a> <a data-t="527">The errors are larger across the distribution.</a> <a data-t="530">And so we see that those two interpretations of fairness are in conflict with each other.</a> <a data-t="535">And they arose from two different worldviews about how we should be taxing people and how errors should be distributed across the distribution.</a></p>

<p><a data-t="549">So defining fairness is hard.</a> <a data-t="551">This is kind of one of the most popular quotes from a recent meta-analysis in the field.</a> <a data-t="557">Definitions of fairness are not mathematically or morally compatible in general.</a> <a data-t="562">For the mathematicians in the room, there actually is a proof here.</a> <a data-t="566">It's called the impossibility theorem.</a> <a data-t="568">And it basically says that if we do indeed live in a world where there are disparities between groups, then different definitions of fairness are impossible to satisfy more than a small fixed set of them.</a></p>

<h3>Thinking about the whole system</h3>

<p><a data-t="588">The last thing that I want to talk about, for practitioners of machine learning, we're very used to this process of evaluating our models with respect to performance metrics.</a> <a data-t="600">And much of the work of machine learning fairness as a research field has focused on what it looks like to measure that performance with metrics.</a> <a data-t="609">But I want to argue that thinking about the whole system is just as important because the metrics evaluate the model, but the model is situated in a much broader system.</a> <a data-t="622">And so I'll try to argue that in this same problem context.</a></p>

<p><a data-t="625">Let's think about how the predictions will be used.</a> <a data-t="628">In one model, we could say that you take your assessed value of the home, and then you multiply it by some fixed number, and that decides what your property tax is.</a> <a data-t="638">I found this 0.9% on some web page in the King County verse, and then I went to a different web page and found a different one.</a> <a data-t="648">So if you live in the area, you probably know what this number actually is or if there's an analogous number.</a> <a data-t="655">But the idea here, at least to me, our model that is trained on the mean percentage error seems to be the most performant model in that case or the most fair model.</a></p>

<p><a data-t="676">Where I live in Chicago and in most places in the US, there's something that we call the homeowner exemption, where if you live in the home that you own, some fixed amount of that assessed value is sort of wiped out.</a> <a data-t="691">So we're shifting the distribution of the errors down, and then the rest of that value is taxed at a fixed percentage.</a> <a data-t="700">And so to me, the R squared or the root mean squared error, that seems to me like a reasonable metric to use in that case, because it does better on the other end of the distribution.</a> <a data-t="713">That mean shift of the errors is more impactful for me.</a> <a data-t="716">If I own a home, it's $100,000.</a> <a data-t="718">If I own several homes that I don't live in, then the errors are sort of averaged across the homes.</a> <a data-t="725">And so it's not as important to me to get those errors spot on in that case.</a></p>

<p><a data-t="731">Another thing that might impact our gut reaction to what is actually fair is the change in the value from last year.</a> <a data-t="737">So if I own a $500,000 house, and this assessment from the previous year was $200,000 under, and now my prediction is accurate, and I'm paying $2,000 or $3,000 more than I expected to, is that change in the amount that I'm paying unfair?</a> <a data-t="751">And so we even need to think about what that model looked like last year if we want to train a fair model.</a></p>

<p><a data-t="761">So I've tried to argue here that the metrics evaluate the model, but the model is one part of a larger system.</a> <a data-t="769">And that larger system is just as important to us in aligning the way that we talk about evaluating machine learning models with our actual morally held beliefs.</a></p>

<h3>Wrapping up</h3>
