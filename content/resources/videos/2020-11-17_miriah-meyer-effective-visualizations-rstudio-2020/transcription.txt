<h3>Designing new visualization techniques</h3>

<p><a data-t="171">All right, so to begin, let's talk a little bit about the idea of designing new and innovative</a> <a data-t="177">visualization techniques.</a> <a data-t="180">And here I want to talk to you about, I want to tell you about a project that I worked</a> <a data-t="184">on with a colleague of mine, Bang Wong, who is in Boston.</a> <a data-t="187">He's a really brilliant visual designer.</a></p>

<p><a data-t="190">And Bang and I were working with a group of biologists who were studying yeast, or more</a> <a data-t="194">specifically, they were interested in the process of metabolism in yeast.</a> <a data-t="199">Now this group was trying to understand how different species evolved a similar process</a> <a data-t="206">for metabolism because this actually has implications for our understanding of how diseases like</a> <a data-t="211">cancer work.</a></p>

<p><a data-t="213">Now the main kind of data that this group was working with was something called gene</a> <a data-t="217">expression, which is really just a measurement of how much a gene is turned on or off in</a> <a data-t="222">a cell.</a> <a data-t="222">And the group was collecting gene expression for many different species of yeast and under</a> <a data-t="226">different experimental conditions.</a></p>

<p><a data-t="230">Now their main method for visualizing this data is a heat map, and this is in fact one</a> <a data-t="236">from theirs.</a> <a data-t="237">And I can see Will in the front sort of tilting his head, and I'm sorry, I will never do that</a> <a data-t="242">again.</a> <a data-t="243">So, yeah, so, but a heat map is the predominant way that these biologists were looking at</a> <a data-t="249">their gene expression data.</a></p>

<p><a data-t="251">So Bang and I began to engage these scientists in conversations around what worked well for</a> <a data-t="258">them with this visualization, but really what was really problematic.</a> <a data-t="262">And what they said is that they wanted to be able to make very fine-scale, nuanced decisions</a> <a data-t="267">based upon changes in gene expression, and they found that in these heat maps it was</a> <a data-t="272">nearly impossible to see that kind of detail of the data.</a></p>

<p><a data-t="277">So Bang and I stepped back and we said, okay, so we need to redesign a heat map, but how</a> <a data-t="281">in the world do we start even approaching this problem of coming up with something new?</a> <a data-t="286">And so we decided to turn to a fundamental principle in visualization design, which is</a> <a data-t="290">that spatial encoding is the most effective encoding channel that we have.</a> <a data-t="296">So by spatial encoding, what I mean is by positioning marks on a common scale, such</a> <a data-t="300">as in a scatter plot, or using the length of some sort of mark like we do in a bar chart.</a></p>

<p><a data-t="306">So okay, so this is a foundational principle.</a> <a data-t="309">We can ask many visualization designers about this, but as a researcher, how do we actually</a> <a data-t="316">know that this is a good principle to go by?</a> <a data-t="320">And part of it we know through controlled studies that have been done looking at different</a> <a data-t="324">encoding channels, the first of which was done by the statisticians Cleveland and McGill</a> <a data-t="328">back in the 80s, when they did a controlled lab experiment where they ran participants</a> <a data-t="333">through a series of tasks using different encodings like color, position, angle, and</a> <a data-t="337">so on, and they found that the spatial encoding techniques significantly outperformed other</a> <a data-t="343">types of visual encodings.</a></p>

<p><a data-t="345">Now this study was replicated a number of years ago by Jeff Hare and Mike Bostock, only</a> <a data-t="352">this time they did it on Mechanical Turk, so they were reaching hundreds and hundreds</a> <a data-t="355">of people, and the results that they got were very similar to what Cleveland and McGill</a> <a data-t="361">had done in their studies.</a> <a data-t="364">And so it's in studies like this that we're able to say with some confidence that certain</a> <a data-t="368">kinds of visual encoding channels are more effective for certain kinds of tasks.</a></p>

<p><a data-t="374">And based on this, we can think about our basic encoding channels and rank order them.</a> <a data-t="381">So for example, here what we're looking at on the left-hand side are encoding channels</a> <a data-t="385">that we have for encoding quantitative data, and on the right-hand side are the basic encoding</a> <a data-t="390">channels for categorical attributes.</a> <a data-t="391">And you'll notice at the top, both of them have spatial encoding as the highest-ranked</a> <a data-t="397">one.</a></p>

<p><a data-t="398">So as visualization researchers, we take studies, we do studies on a variety of things like</a> <a data-t="403">basic visual encoding channels, but we also combine it with things that we learn from</a> <a data-t="407">design practice and years and years of anecdotal evidence to get to ideas like this.</a></p>

<p><a data-t="417">So when Beng and I, we decided, okay, so let's take this principle and let's think about,</a> <a data-t="422">well, what does that mean in terms of the data we have?</a> <a data-t="424">And we decided, okay, well, the data we have is actually time series, so let's look at</a> <a data-t="428">line charts.</a></p>

<p><a data-t="430">So we went off in an illustrator, we mocked up every single variation that we could possibly</a> <a data-t="434">think of for how to show a line chart, and we printed it out on a big poster, and what</a> <a data-t="438">we noticed was the one over here on the right, these little filled frame charts, really made</a> <a data-t="443">the shape of those line charts pop out.</a> <a data-t="446">And so we incorporated that idea into a new technique that we called a curve map that</a> <a data-t="451">we implemented as part of a bigger multi-view system that we deployed to our collaborators.</a></p>

<p><a data-t="457">Now using this new curve map technique, our collaborators said that they were able to</a> <a data-t="463">much more quickly see nuanced things that they knew about the data, but they also saw</a> <a data-t="467">things that they had never known were there, which led to follow-up experiments and the</a> <a data-t="472">discovery of some new scientific concepts that they weren't aware of before.</a></p>

<blockquote><a data-t="477">So in this way, I want to stress that this idea is that we can take ideas about perceptual</a> <a data-t="482">principles and other things and use those as a way to brainstorm and sort of a springboard</a> <a data-t="487">for thinking about new ways to encode our own data.</a></blockquote>

<h3>The visualization design process</h3>

<p><a data-t="499">Okay, the second point, the visualization design process.</a> <a data-t="504">So here is really about saying, okay, well, we have to create visualizations, but there's</a> <a data-t="508">a whole lot of stuff we have to do before that, which is even figuring out what is it</a> <a data-t="513">that I want to visualize in my data.</a> <a data-t="518">And this is one of the things that I personally have spent a lot of time on and care about</a> <a data-t="524">a lot.</a></p>

<p><a data-t="525">So I'm going to start with a little flavor of what it's like to be a visualization researcher</a> <a data-t="530">like myself.</a> <a data-t="531">So we have two people, a viz person and a biologist.</a> <a data-t="534">We might start by saying, oh, what is it you want to visualize?</a> <a data-t="538">And the biologist might say something like, well, from patterns of conservation, we want</a> <a data-t="542">to visualize the mechanisms that influence gene regulation.</a> <a data-t="545">And to me, it sounds like this.</a></p>

<p><a data-t="549">So it turns out the ginger effect isn't just for debugging.</a> <a data-t="553">It also occurs all the time in visualization design.</a> <a data-t="556">And as a side note, beware of </a><a href="/people/jenny-bryan/">Jenny Bryan</a><a data-t="556"> scooping your punch lines.</a></p>

<p><a data-t="577">But I am just going to plow ahead anyway.</a> <a data-t="581">Thanks, Jenny.</a></p>

<p><a data-t="591">So but it's true.</a> <a data-t="591">That same experience about only really like seeing a few things at a time is absolutely</a> <a data-t="597">the case when we jump into these new projects.</a> <a data-t="600">What I'm really interested is in how we get from a very semantically rich like task like</a> <a data-t="606">we have here to the knowledge that we can then design a visualization like this to support.</a></p>

<p><a data-t="612">So fundamental to this process is identifying something I like to call proxies, which are</a> <a data-t="618">the partial and imperfect data representation of that semantically rich thing that the analyst</a> <a data-t="624">actually cares about.</a> <a data-t="636">The high level goals of a data analyst is actually rarely captured directly in the data.</a> <a data-t="642">And if it was, you probably wouldn't need visualization anyway.</a></p>

<p><a data-t="646">So instead, what we have to spend a lot of time doing is thinking about what are the</a> <a data-t="649">things in my data that I can </a><a href="/software/infer/">infer</a><a data-t="649"> something from that will then actually let me answer</a> <a data-t="657">the question I have.</a> <a data-t="658">And so let me give you an example of what I mean by this.</a></p>

<p><a data-t="662">Let's say that I wanted to identify good film directors because I don't know, Sundance is</a> <a data-t="666">happening right now in Salt Lake and I'm a journalist and I need to identify some good</a> <a data-t="670">film directors.</a> <a data-t="671">Okay.</a> <a data-t="673">So I go off and I scrape some data from IMDb about movies.</a> <a data-t="677">So now I have this data set that includes lots of information about movies such as how</a> <a data-t="680">much money they've made, their ratings and so on.</a></p>

<p><a data-t="685">But now going back to my question, my question is really about film directors, but I have</a> <a data-t="689">information about movies.</a> <a data-t="691">So what do I do?</a> <a data-t="693">And this is where I like to think about this notion of how do I select a good proxy in</a> <a data-t="697">my data for my question.</a></p>

<p><a data-t="699">And so my colleague Danielle Fisher and I came up with this very simple mechanism for</a> <a data-t="702">doing this, which is we break a task down into three things.</a> <a data-t="706">Into the action, which is the thing you want to do.</a> <a data-t="710">Into the object, which is the items you want to take that action on.</a> <a data-t="715">And then finally, the measure, which is the value that you care about for those objects.</a></p>

<p><a data-t="718">And so if we do this, let's look at our tasks.</a> <a data-t="721">So the action is identifying.</a> <a data-t="723">What do I want to identify?</a> <a data-t="725">Film directors.</a> <a data-t="727">Now this is where I know that I have a data set about movies, not film directors, but</a> <a data-t="731">I could say if I can learn something about movies, I can infer something about directors.</a> <a data-t="735">So I'm going to choose movies as my proxy for director.</a> <a data-t="739">And what do I want to know?</a> <a data-t="740">I want to know if they're good.</a></p>

<p><a data-t="742">And now, you know, I don't have an attribute labeled good in my spreadsheet.</a> <a data-t="747">So this is where I think all the time in data science, we are constantly making decisions</a> <a data-t="751">that are inherently subjective and really specific for the one, for the, whatever the</a> <a data-t="755">view is on the problem you're tackling.</a> <a data-t="758">So in this case, good could be movies that make a lot of money.</a> <a data-t="761">It could be movies that reach a broad demographic.</a> <a data-t="764">But for now, I'm going to keep it simple and just say it's good equals movies with high</a> <a data-t="767">IMDB ratings.</a></p>

<p><a data-t="770">So by identifying my proxies, I can now translate my task into something, identify movies with</a> <a data-t="775">high IMDB ratings, which is actually something I can design a visualization or other sort</a> <a data-t="781">of analysis to tackle.</a></p>

<p><a data-t="784">But it turns out that actually being able to do this translation and being able to find</a> <a data-t="788">these proxies is a really challenging process and takes a lot of time.</a> <a data-t="792">And in our group, what we do is we spend a lot of time really immersing ourselves with</a> <a data-t="797">the people that we're working with in order to better understand the kinds of challenges</a> <a data-t="800">they face and the things that they want to do.</a></p>

<p><a data-t="804">And some colleagues at the University of Calgary just this past year formalized this notion</a> <a data-t="808">of immersion by calling it design through immersion and talking about some of the sorts</a> <a data-t="813">of benefits that we get when we do transdisciplinary kind of research and bring sort of big tent</a> <a data-t="821">style approach to how we problem solve and in this case do data analysis.</a></p>

<h3>Visualizations as probes</h3>

<p><a data-t="835">And then the last thing I want to talk about here is the notion of visualizations as probes.</a> <a data-t="839">I'm going to do this by talking about another project that I worked on that was headed up</a> <a data-t="845">by a former student of mine, Nina McCurdy, where we were working with a group of public</a> <a data-t="849">health experts at USAID who were studying Zika, the spread of Zika in Latin America</a> <a data-t="854">and all the effects, notably microcephaly in babies.</a></p>

<p><a data-t="859">And so during this project, Nina spent six months at the USAID working closely with the</a> <a data-t="864">health experts there.</a> <a data-t="866">And it had all the makings of a straightforward visualization project.</a> <a data-t="870">The collaborators had data.</a> <a data-t="872">They had tried to visualize their data and failed and so they were super excited to work</a> <a data-t="875">with us.</a> <a data-t="876">And so Nina left and we dug in.</a></p>

<p><a data-t="879">Now, over the course of the study, Nina did a lot of rapid prototyping and ultimately</a> <a data-t="884">designed a tool that used best practices for how we can incorporate all sorts of tabular</a> <a data-t="891">data with geospatial information.</a> <a data-t="895">She evaluated this tool with various stakeholders throughout that time and the response was</a> <a data-t="901">overwhelmingly positive, that this was a great tool for the data that they had.</a></p>

<p><a data-t="907">And yet, when we tried to engage our immediate collaborators in incorporating the tool as</a> <a data-t="913">part of their workflow, we noticed a lot of hesitation and we were really concerned about</a> <a data-t="918">this.</a> <a data-t="919">And as we probed deeper into this hesitation, we came to understand that even though the</a> <a data-t="924">tool was a good representation of their data, the data was not a good representation of</a> <a data-t="929">what they knew to be true about the spread of Zika on the ground.</a></p>

<p><a data-t="934">So this first came up in a discussion of a choropleth that looked like this one showing</a> <a data-t="939">Zika cases at a national level.</a> <a data-t="941">So as you'll see here, Brazil is in dark red, indicating a relatively high percentage of</a> <a data-t="945">cases, whereas Colombia is in a lighter orange, indicating a relatively lower percentage.</a> <a data-t="951">Now, one of our collaborators, when she saw this, she noted, well, Brazil reports all</a> <a data-t="956">cases, whereas Colombia only reports cases after a thorough investigation.</a></p>

<p><a data-t="962">The implication of this comment was that the different ways in which these countries were</a> <a data-t="968">reporting Zika was leading to an inaccurate and possibly misleading picture of what was</a> <a data-t="973">actually happening.</a> <a data-t="975">So we pivoted to focus on this problem and found that the data was littered with discrepancies</a> <a data-t="980">like this.</a></p>

<p><a data-t="981">And then in the case of Zika data, this came from the way that the data was collected and</a> <a data-t="986">processed and reported.</a> <a data-t="989">And it was this distributed heterogeneous data generation pipeline that occurred differently</a> <a data-t="994">in every country that led to all these discrepancies.</a></p>

<p><a data-t="998">And now, you know, all was not lost.</a> <a data-t="1001">So even though these problems with the data were not included in the data set, the experts</a> <a data-t="1006">working with it had deep and intimate knowledge about them.</a> <a data-t="1012">And these discrepancies, it turns out, were shaped by the individual country's political,</a> <a data-t="1017">cultural, economic, geographic, and demographic context.</a></p>

<p><a data-t="1022">And so we'd get discrepancies like the union in Region X goes on strike often and doesn't</a> <a data-t="1027">report Zika data.</a> <a data-t="1029">Or country Y recently overhauled its surveillance system, leading to a sudden increase in detected</a> <a data-t="1035">cases.</a></p>

<p><a data-t="1036">So in thinking about this, we formalized this notion into what we called implicit error,</a> <a data-t="1041">which is measurement error that is inherent to a given data set, assumed to be present</a> <a data-t="1045">and prevalent, but not explicitly defined or accounted for.</a> <a data-t="1049">And so we were able to characterize aspects of implicit error, as well as then design</a> <a data-t="1054">an annotation mechanism to help experts externalize.</a></p>

<p><a data-t="1059">Nina implemented this annotation mechanism back into our tool, which we deployed back</a> <a data-t="1064">out so that these health experts could start annotating their data in order to share information</a> <a data-t="1070">with their colleagues, as well as to start to build up a database of contextual information</a> <a data-t="1074">around that initial data set.</a></p>

<p><a data-t="1078">So although this work was grounded in Zika health data, we suspect that this type of</a> <a data-t="1084">implicit error is prevalent in many, many other domains.</a></p>

<p><a data-t="1093">Okay, so this was an example where we were able to use visualization, not as necessarily</a> <a data-t="1097">the end result, but as a conversation starter about challenges that the group was having</a> <a data-t="1102">with the data, and then ultimately use visualization as a sort of mechanism to help analysts externalize</a> <a data-t="1109">things that they knew that wasn't captured in that data set.</a></p>

<blockquote><a data-t="1093">Okay, so this was an example where we were able to use visualization, not as necessarily</a> <a data-t="1097">the end result, but as a conversation starter about challenges that the group was having</a> <a data-t="1102">with the data, and then ultimately use visualization as a sort of mechanism to help analysts externalize</a> <a data-t="1109">things that they knew that wasn't captured in that data set.</a></blockquote>
