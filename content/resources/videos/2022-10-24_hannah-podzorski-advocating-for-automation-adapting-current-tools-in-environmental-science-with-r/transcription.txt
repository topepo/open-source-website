<h3>Reducing activation energy</h3>

<p><a data-t="97">So in chemistry, activation energy is the energy needed to get from one side of a chemical</a> <a data-t="101">reaction to another.</a> <a data-t="104">And you can reduce the activation energy by applying a catalyst.</a> <a data-t="108">So when I want to move my team from their current workflow to an automated workflow,</a> <a data-t="108">I can reduce the activation energy by rephrasing how I pitch automation, starting small, and</a> <a data-t="115">being a little bit creative.</a></p>

<p><a data-t="118">So let's start with the pitch, and most importantly, that word automation.</a> <a data-t="122">It can be a little vague and field dependent, but what I mean when I say automation is the</a> <a data-t="127">streamlining of a workflow by reducing unnecessary human interactions.</a> <a data-t="132">So my team has more time to spend on greater value tasks.</a> <a data-t="138">Most times when I pitch automation, I don't even use the word automation.</a> <a data-t="141">I say let's streamline this workflow.</a></p>

<blockquote><a data-t="138">Most times when I pitch automation, I don't even use the word automation.</a> <a data-t="141">I say let's streamline this workflow.</a></blockquote>

<p><a data-t="145">So let's talk about the difference between workflow that I traditionally see, which I'm</a> <a data-t="148">going to call for the purposes of this talk reactionary workflow and automated workflow.</a> <a data-t="154">So let's say we have four work products, A, B, C, and D. A might be our database, B might</a> <a data-t="160">be our tables full of summary stats, C is our figures, and D is the text that goes into</a> <a data-t="166">our final report.</a> <a data-t="168">Say we have to update our database.</a> <a data-t="170">Maybe we know that we need to update our database, like we need to go out and collect more data,</a> <a data-t="174">or maybe we don't know and maybe someone found an error in unit conversions at the last minute.</a></p>

<p><a data-t="181">Well with this change, now we have to manually update all of the workflow downstream.</a> <a data-t="187">In comparison, an automated workflow, if you have to make that one change to your database,</a> <a data-t="193">you can use functions or scripts to automate the processing of that one edit and apply</a> <a data-t="198">it to all of your work products downstream.</a> <a data-t="200">And this is, I know, a very simple definition of what automation is and streamlining a workflow,</a> <a data-t="206">but I like to lay it out like this because it makes the pros or benefits of using automation</a> <a data-t="211">very obvious.</a></p>

<h3>Automating Pro-UCL for environmental statistics</h3>

<p><a data-t="466">So now that I've got my team a little bit on board, I've started with these smaller tasks,</a> <a data-t="471">now I can take on bigger tasks that might require a little bit more of initial investment</a> <a data-t="477">and delayed gratification, so I'm going to talk just about that, a specific example where</a> <a data-t="482">my team decided to automate software developed by a regulatory agency.</a> <a data-t="488">So that software is Pro-UCL, Pro-UCL was developed by the U.S. Environmental Protection Agency</a> <a data-t="494">or EPA, and it specializes in performing statistical analysis on left-sensor data.</a></p>

<p><a data-t="500">In environmental sciences, we have a lot of left-sensor data, whenever we're measuring</a> <a data-t="504">the concentration of some compound in water or soil or </a><a href="/software/air/">air</a><a data-t="504">, depending on our methods and</a> <a data-t="510">our equipment, we have something called a detection limit, and below that detection</a> <a data-t="513">limit, we don't know where that concentration will sit, it can be anywhere between zero</a> <a data-t="518">and our detection limit, so there's a lot of uncertainty around that.</a> <a data-t="522">So Pro-UCL takes that uncertainty into account when doing simple statistics like means and</a> <a data-t="527">standard deviations, as well as more complex statistics like distributions and upper confidence</a> <a data-t="533">limits.</a></p>

<p><a data-t="534">So Pro-UCL is definitely, it's solving a problem for us, but it has some less than great design</a> <a data-t="541">features, one of which is it will crash if you put too much data into it, and by too</a> <a data-t="547">much I mean about 10,000 records, which for the type of projects we're working on, which</a> <a data-t="551">require us to analyze hundreds of compounds in a variety of environmental media, we were</a> <a data-t="557">having to run Pro-UCL multiple times.</a> <a data-t="560">And because of that, we decided we wanted to try and automate it, and we decided to</a> <a data-t="564">try and automate it by using a mouse macro, so in our case it's called mini-mouse macro,</a> <a data-t="573">and basically automate the clicking of the software itself.</a></p>

<p><a data-t="577">So we wrote a function in R that allows us to subset that data into more manageable chunks,</a> <a data-t="586">and then you run the function, and it creates an input file for our mouse macro, and then</a> <a data-t="591">it runs that file.</a> <a data-t="593">So the mouse macro input is just a simple text file.</a> <a data-t="596">I used some command line code to open up the mouse macro and load in the input file, and</a> <a data-t="601">then it just clicks through the software for you.</a> <a data-t="605">So here it's using the text string we gave it to input the location of the input file.</a> <a data-t="610">It's clicking through all of the dropdowns we'll need, and then it's loading in the compounds</a> <a data-t="615">we want to look at, and finally, it'll save that file to the text string I gave it for</a> <a data-t="620">the output file.</a></p>

<p><a data-t="623">And then once it senses that there's information in that file, it'll close out of Pro-UCL,</a> <a data-t="628">and if necessary, run it again.</a> <a data-t="629">So we'll loop this over and over and over again.</a> <a data-t="631">And that might have seemed like a very simple process.</a> <a data-t="633">Why would you automate it?</a> <a data-t="635">Well, that was a very small data set, and there's actually a lot of computation time</a> <a data-t="639">that has to go into it when you're doing larger data sets.</a> <a data-t="642">So for small projects, running Pro-UCL through the automation like this can take 15 to 30</a> <a data-t="648">minutes, but for bigger projects, we're talking just automating like this, six to eight hours.</a> <a data-t="654">So it's definitely saving us time that way.</a></p>

<p><a data-t="657">But writing the code to automate the clicking of my mouse through the software was actually</a> <a data-t="660">the easy part.</a> <a data-t="662">The hard part was deciphering the output file.</a> <a data-t="667">So here's what Pro-UCL gives you as an output file.</a> <a data-t="670">It's a formatted Excel sheet.</a> <a data-t="673">This is just a small subsection of the statistics it gives you and only one compound.</a> <a data-t="677">So once again, for our sites, we're looking at hundreds of compounds, and we might even</a> <a data-t="681">have to group our data spatially, so we'll need additional information.</a></p>

<p><a data-t="688">And so before we automated, what we did was just have someone go through and copy and</a> <a data-t="692">paste the values we were really interested in.</a> <a data-t="695">And that took quite some time.</a> <a data-t="697">So instead, we automated it, and now we create tables both unformatted and formatted in Excel</a> <a data-t="702">that my team can use for their analysis and their final report.</a></p>
