<h3>R validation in pharma</h3>

<p><a data-t="832">So, GSK, we do R validation.</a> <a data-t="834">A lot of how we do R validation can actually be found on the R Validation Hub, which I think is part of the — is either a POSIT working group or is loosely related.</a> <a data-t="852">So, how we do it is really kind of taken from there in terms of how we do it, the R Validation Hub.</a></p>

<p><a data-t="870">We were a — we, being GSK, were a strong participant in figuring out how to do it.</a> <a data-t="875">And then we worked with basically our QA department to make sure that we're checking all the boxes that we need to hit in order to say that, yes, we believe that this is validated.</a></p>

<p><a data-t="902">So, within a biostats organization, you basically have two things that you need to accomplish to move assets or drugs through the pipeline.</a> <a data-t="922">Basically, one, can you tell what — can you trust the efficacy of a drug and can you trust the safety of a drug or an asset?</a> <a data-t="943">And how you do that is you write statistical code in order to be able to prove that, yes, we trust these statistical tests that we've written.</a></p>

<p><a data-t="965">In order to do that, you need to do something that's called validation for your software, which is essentially to say we have tested the software and we believe that all the outputs from this software is correct.</a> <a data-t="982">How you do validation is defined by the organization, but it's really important for you to be able to trust your software.</a></p>

<p><a data-t="989">So, there's a lot of testing that goes into place in order to get that level of confidence within an organization that says, yes, this software can be trusted.</a> <a data-t="1002">And there's different nuances for how you do the testing.</a> <a data-t="1004">So, for example, if you're using </a><a href="/software/ggplot2/">ggplot2</a><a data-t="1004"> for creating outputs of graphics, things like that, you can generally test it through code coverage as well as basically doing a couple simple simulations and saying, okay, does it actually create the output that we would expect if you were to put it — basically create that output?</a></p>

<p><a data-t="1037">But then there's also output — there's validation from a statistical correctness point of view.</a> <a data-t="1044">So, for example, if we're writing a piece — a package to do linear regression or doing a linear regression — or using a package for linear regression, how do you test for that statistical correctness?</a> <a data-t="1060">And that's a little bit more of a thornier problem to figure out, but it tends to require like a lot of different simulations, things like that, in order to be able to say, yes, from a statistical correctness point of view, we trust this piece of software.</a></p>

<p><a data-t="1081">So, what we do is we have it — we host it on Workbench, and what we do basically is we create what's known as a frozen R environment, which is basically an environment that has a set number of packages that we have gone through the validation process with and said, yes, we trust these packages.</a> <a data-t="1106">So, then what we do is then we create basically a frozen R — I don't know.</a> <a data-t="1112">It's not truly a container, but basically an execution script where we can then basically just take and install that in any system that we choose.</a> <a data-t="1123">So, the goal is to make it system agnostic.</a></p>

<p><a data-t="1139">If we want to do it on, I don't know, pick your data science Workbench, Domino, I don't know, whatever, different things like that, you're able to do it, but trying to make it system agnostic.</a></p>

<p><a data-t="1157">Yeah, I mean, there's like two levels to the validation.</a> <a data-t="1161">It always seems like validating the packages inside of R and then validating the installation of R itself.</a> <a data-t="1172">And that latter one is the one that, you know, we've had the most challenges with, you know, how do you prove that R is actually R, is sort of the philosophy, the spirit of what FDA wants to see, and we find that surprisingly difficult.</a></p>

<p><a data-t="1951">So I actually don't believe in having people redefine themselves.</a> <a data-t="1959">I actually think about how do you help someone like learn a tool or add a tool to their toolkit in order to like enhance the way that they're working?</a> <a data-t="1970">Because at the end of the day, you are not going to turn like a statistician into a programmer or a computer scientist or things like that unless they really want to become one.</a></p>

<p><a data-t="1979">So I think the goal is like you don't actually try to redefine them.</a> <a data-t="1999">I think you try to help them add tools to their toolkit in order to be able to use it in order to enhance their work.</a> <a data-t="2013">So like teaching like a statistician Git, like you have to explain why it's important for them to know, you know, kind of the hit by the bus rule, if anyone's familiar with that.</a></p>
