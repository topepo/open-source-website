<h3>How conversations with LLMs work</h3>

<p><a data-t="592">Great, so now we're going to talk about how conversations with LLMs work.</a> <a data-t="592">So they're at a high level at first and then you'll see some code.</a> <a data-t="611">So you probably talked to an LLM before, maybe it was ChatGPT or Gemini or something like that, but what's actually going on when you talk?</a></p>

<p><a data-t="617">So you might have done something like this where you ask an LLM a question or you tell it to do something and then you get a response back.</a> <a data-t="625">And you keep asking questions and then ChatGPT or whatever LLM, it keeps responding and it kind of feels like you're having a conversation with the person.</a> <a data-t="637">So how is this happening?</a> <a data-t="637">At a general level, this entire conversation is happening by HTTP requests and responses.</a> <a data-t="645">And this isn't too different from just sort of normally interacting with the internet.</a> <a data-t="650">HTTP requests power the internet at a high level.</a> <a data-t="650">So when you visit a website, your browser sends an HTTP request to the server that's hosting that website.</a> <a data-t="659">The server then responds to that request by sending back the HTML of the webpage.</a> <a data-t="663">And then your browser does a bunch of other stuff to get like images and things, but that's generally what's happening.</a> <a data-t="668">And it's not really that different with an LLM.</a> <a data-t="668">So talking to an LLM like ChatGPT also happens via these HTTP requests.</a> <a data-t="676">You send a message to ChatGPT, you're sending an HTTP request to OpenAI.</a> <a data-t="681">OpenAI processes that request, it runs the model to get the response, and then it sends that response back to you.</a></p>

<p><a data-t="700">But when you send a message to an LLM, you're not just sending text, you're actually sending text with a specific role attached to it.</a> <a data-t="710">So there are three roles that messages can have, and understanding these is pretty important to working with LLMs programmatically.</a> <a data-t="716">So the first is the user role.</a> <a data-t="716">This is attached to what like you as the person typing is sending.</a> <a data-t="724">And then there's the assistant role, which is attached to the text that the LLM is sending back to you.</a> <a data-t="729">This might seem like a small detail, but these roles are kind of fundamental to how you control and guide LLM behavior.</a> <a data-t="739">So there's three roles.</a> <a data-t="739">So the third one is the system role or the system prompt.</a> <a data-t="745">This is kind of your instruction manual for the LLM.</a> <a data-t="745">It's where you as the developer can guide the model's behavior, tell it what constraints it needs to follow, tell it what kind of format you want to see the responses in, that kind of thing.</a> <a data-t="756">And we'll talk a lot more about system prompts in a later section.</a></p>

<p><a data-t="763">And this isn't really a thing that you as a user, something like ChatGPT would see.</a> <a data-t="768">Like you don't have access to the ChatGPT system prompt, you can't see it.</a> <a data-t="773">It's kind of behind the seams, but it is particularly powerful because it persists across this entire conversation, and it's going to shape how the assistant responds to every user message.</a> <a data-t="783">Okay.</a> <a data-t="783">So again, there's three roles, the system prompt, the user, and the assistant.</a></p>
