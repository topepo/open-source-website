<p><a data-t="704">And so at a higher level, you know, while we've been working on accelerating the data science stack, there's been this broader trend at the large tech company level of thinking about how companies like Meta and Google and Microsoft and Amazon, like how we build large scale data management systems.</a> <a data-t="716">And so the work that we've done in Arrow, there's been similar thinking around all the different pieces that go into building these large scale data warehouses.</a> <a data-t="729">So last year, working with the folks at Meta, they had built many query processing engines within Facebook's data platform, Meta's data platform.</a> <a data-t="736">And they said, well, you know, what if we could create like a reusable execution engine that we can consolidate all of our query processing and all of these different systems onto a common layer, make it interoperable with Arrow.</a> <a data-t="751">It could be used for stream processing, batch processing, machine learning, and all the things that go into building state-of-the-art systems at Meta.</a></p>

<p><a data-t="764">So the core ideas here is that approaching the problem from the Arrow ethos that we want to build systems that use open standards and open protocols and that are designed around the ideas of modularity, reuse, and interoperability</a> <a data-t="776">so that we can have these off-the-shelf components like DuckDB or DataFusion that, you know, provide accelerated execution.</a> <a data-t="782">But that we also have some of the other layers like the storage systems, the file formats, which are also open standards and are available off the shelf where we can mix and match the components and we can build new custom data processing systems in much less time than we could in the past.</a> <a data-t="805">And at the same time, because we're not starting, when you're building a new system, you're not starting from scratch and trying to leverage all of the cutting-edge capability that's available in modern hardware that you can leverage the advances that are being developed within each of these component systems.</a></p>

<p><a data-t="1027">Another interesting thing I just wanted to make you aware of is that these modular, these kind of new execution engines are being used to accelerate legacy systems like Apache Spark.</a> <a data-t="1044">And so Spark has enormous adoption and isn't going anywhere.</a> <a data-t="1048">There's also open source database systems like Presto and Trino.</a> <a data-t="1050">Trino is a fork of Presto and maybe it's called PrestoDB.</a> <a data-t="1055">And so the Meta folks have been working on accelerating Presto with their Velox project called Prestissimo.</a> <a data-t="1063">Data Fusion just spun out from Arrow and Apple and others are using Data Fusion to accelerate Spark.</a> <a data-t="1068">That's called the Comet Project.</a> <a data-t="1068">Intel is working on accelerating Spark with Velox.</a> <a data-t="1074">It's called Gluten.</a> <a data-t="1074">And I expect that we'll see more and more of these modular acceleration projects to take existing systems, keep the API the same, swap out the execution engine to make things faster, more interoperable, and more efficient.</a></p>

<p><a data-t="1088">So this all, you know, but I back, you know, again, there's Spark.</a> <a data-t="1093">Spark is the, you know, probably the most successful distributed big data system.</a> <a data-t="1098">And just a few minutes ago, I told you the big data was dead.</a> <a data-t="1098">And so it turns out that actual big data does still exist.</a> <a data-t="1104">And while you can do a lot with DuckDB, you want to be able to write, essentially right size the framework that you're using to the size of your data set.</a> <a data-t="1117">And so the ideal thing would be you can use things like DuckDB up to a certain data scale, call it like the one terabyte scale.</a> <a data-t="1123">But as soon as you have truly big data, it's not a hardship for you to switch over to using Spark or some other type of distributed execution system.</a></p>
