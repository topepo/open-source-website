<h3>Templates and slide structure</h3>

<p><a data-t="1344">I want to address some frequently asked questions.</a> <a data-t="1346">There's a question about getting started, about using templates and presentations that we get frequently.</a> <a data-t="1355">So this is the getting started page.</a> <a data-t="1357">We come down here to templates.</a> <a data-t="1361">You can see that the way you set up a template is with this reference doc.</a> <a data-t="1368">And the template supports four of the many layouts that are in a PowerPoint template.</a> <a data-t="1377">These are the layouts that you want to include in your template for R Markdown documents.</a></p>

<p><a data-t="1385">The first one is the title layout.</a> <a data-t="1388">So this will give you the title and the subtitle on this slide.</a> <a data-t="1392">The next one is the section header.</a> <a data-t="1396">And the section header will only render the header and not the content.</a> <a data-t="1405">The title and content slide, which you typically think of as the main slide, and then the two-content layout, which allows you to do columns.</a></p>

<p><a data-t="1414">Now, layouts are important because Markdown doesn't allow you to programmatically change the design or size of your presentation.</a> <a data-t="1427">So what it does is Pandoc actually ships with a 4x3 vanilla template that is used to render presentations by default.</a> <a data-t="1439">So if you want to use a widescreen format, you're going to have to create a widescreen template and include that as a custom template here in your reference doc.</a> <a data-t="1450">If you want to change the background or if you want to change the layout, the placeholders in any way, that's all going to happen with the template.</a></p>

<p><a data-t="1463">The next thing I want to talk about is structuring the presentation.</a> <a data-t="1467">This gets a little detailed, but I think it's important to go through.</a> <a data-t="1472">PowerPoint presentations have a hierarchy of title, section, and content, whereas Markdown documents have a hierarchy of headers and contents.</a> <a data-t="1481">So in order for the PowerPoint presentation to get structured, Pandoc has to determine which Markdown header level should be used for the PowerPoint slide level.</a></p>

<p><a data-t="1502">So for example, in this Markdown document, you have a hierarchy of headers that go from 1 to 5.</a> <a data-t="1510">And this header is the one that Pandoc is going to use as the slide level by default.</a> <a data-t="1519">And it's going to use it as a slide level because it's the first slide that has content associated with it.</a> <a data-t="1529">So everything that gets rendered above this is going to be turned into section slides, and everything below this is going to be turned into subheaders on the content slide.</a></p>

<p><a data-t="1547">Now if you didn't want this to be the content slide and you wanted this to be the slide level, you would change slide level to 2 because 2 would match up to the two hashtags on this R Markdown section header.</a> <a data-t="1568">Okay, so you probably don't have to worry too much about this because the defaults should work well for you in most cases.</a> <a data-t="1575">If you want to avoid this issue altogether, you can just use the same level header for everything, and then everything will be rendered as content slides.</a></p>
