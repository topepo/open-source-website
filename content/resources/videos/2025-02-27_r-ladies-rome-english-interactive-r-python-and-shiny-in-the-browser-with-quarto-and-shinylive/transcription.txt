<h3>WebAssembly, WebR, and Pyodide</h3>

<p><a data-t="497">And so that's why we are entering the world</a> <a data-t="499">of interactive R and Python in the browser.</a> <a data-t="503">So running code directly in the browser</a> <a data-t="505">opens the door for many new exciting uses</a> <a data-t="509">of data science on the web.</a></p>

<p><a data-t="512">And the two primary technologies</a> <a data-t="514">that we'll be talking about is,</a> <a data-t="517">or we'll be talking about several technologies,</a> <a data-t="519">but these are brought to you thanks to WebR and Pyodide.</a> <a data-t="524">But before we talk about them,</a> <a data-t="526">let's talk about another tool called WebAssembly.</a></p>

<p><a data-t="531">So WebAssembly, or sometimes called WASM,</a> <a data-t="536">is a low-level, high-performance binary instruction</a> <a data-t="541">and it allows compiled languages like C or C++</a> <a data-t="545">to run in web browsers at near native speeds.</a> <a data-t="550">So C and C++ code is compiled by something called Emscripten.</a> <a data-t="555">Emscripten takes the C or C++ code as input.</a> <a data-t="560">It produces WASM as a result.</a> <a data-t="563">And once it's in WASM,</a> <a data-t="565">you can run it in any browser that supports WASM,</a> <a data-t="569">which in modern browsers is actually most of them.</a></p>

<p><a data-t="574">Python has had a browser solution</a> <a data-t="577">for a while called Pyodide.</a> <a data-t="579">So what Pyodide does is compiles</a> <a data-t="581">the Python interpreter CPython to WebAssembly</a> <a data-t="585">using Emscripten, and then it makes it possible</a> <a data-t="588">to run Python in the browser.</a></p>

<p><a data-t="592">And relatively more recently,</a> <a data-t="594">now R has a similar solution with WebR.</a> <a data-t="598">So WebR is the WebAssembly version of R</a> <a data-t="601">that has been compiled to run in the browser.</a> <a data-t="604">It's an open-source project primarily driven by Posit,</a> <a data-t="607">and thanks to the amazing work of many folks,</a> <a data-t="610">but especially George Stagg and </a><a href="/people/lionel-henry/">Lionel Henry</a><a data-t="610">,</a> <a data-t="613">it's now possible to run R code directly in a web browser</a> <a data-t="617">without the need of traditional servers to execute the code.</a> <a data-t="621">So essentially what that means</a> <a data-t="623">is a user can visit a website and run R code directly,</a> <a data-t="627">so without needing R installed on their device.</a> <a data-t="631">All that is required is a web host server</a> <a data-t="634">like Netlify or GitHub Pages.</a></p>

<p><a data-t="640">So let's see what this actually looks like.</a> <a data-t="641">This is a web-based interactive R console</a> <a data-t="646">that's included in the WebR source repo</a> <a data-t="649">as a demonstration of integrating WebR</a> <a data-t="652">in a wider web application.</a> <a data-t="654">So it provides a simple R environment in your web browser</a> <a data-t="657">and includes things like code input,</a> <a data-t="659">output, code editing, file management, and graphics.</a></p>

<p><a data-t="664">So if I click this, this is the URL,</a> <a data-t="668">and I can share this with others.</a> <a data-t="670">I could share this with you,</a> <a data-t="671">and you may notice it kind of looks like RStudio.</a> <a data-t="674">They did a really nice job designing it</a> <a data-t="677">so that if you are an RStudio user, it looks very familiar.</a> <a data-t="681">You have your console down here and your plots here,</a> <a data-t="685">and even though it looks like RStudio, though,</a> <a data-t="688">remember, this is completely being run in the browser.</a> <a data-t="692">There was nothing installed from our end.</a> <a data-t="695">There was nothing that we had to set up,</a> <a data-t="697">and I can do something like install packages </a><a href="/software/ggplot2/">ggplot2</a><a data-t="697">.</a></p>

<p><a data-t="711">Dangerous of live coding.</a> <a data-t="711">There we go.</a> <a data-t="712">So it says installing packages,</a> <a data-t="714">and you'll notice it says downloading WebR package.</a> <a data-t="717">We'll talk a little bit more about that,</a> <a data-t="719">but even though it looks familiar,</a> <a data-t="721">again, everything is being done in the browser.</a> <a data-t="724">I can go up here into my R script and do library ggplot2,</a> <a data-t="730">run that, and then I do ggplot ntcars as x equals mpg,</a> <a data-t="735">ntcars as x equals mpg, y equals hp, plus geom point,</a> <a data-t="748">and then it produces the plot.</a></p>

<h3>Packages in WebR and Pyodide</h3>

<p><a data-t="772">So in terms of Pyodide,</a> <a data-t="777">it includes a package manager</a> <a data-t="779">that allows the use of Python packages.</a> <a data-t="782">You can also install Python wheels</a> <a data-t="784">or load custom Python packages with Pyodide.</a> <a data-t="787">And then on the R side,</a> <a data-t="789">there's over 14,000 R packages available for WebR.</a> <a data-t="793">A bit of a caveat.</a> <a data-t="794">So the way that this works</a> <a data-t="796">is that R packages have to be compiled for Wasm.</a> <a data-t="799">That's why the installed package</a> <a data-t="801">looked a little bit different.</a></p>

<p><a data-t="803">And so the 14,000 number comes from 14,000 R packages</a> <a data-t="809">that were compiled to Wasm without error,</a> <a data-t="811">but they haven't all necessarily been tested.</a> <a data-t="813">So again, just a bit of a caveat there,</a> <a data-t="817">but still a lot of packages are available,</a> <a data-t="821">especially the ones that are very popular and well-known</a> <a data-t="824">like tidy models and </a><a href="/software/tidyverse/">tidyverse</a><a data-t="824"> and things like that.</a> <a data-t="827">So when you are installing packages,</a> <a data-t="830">by default, the WebR install function</a> <a data-t="833">will use the public repository</a> <a data-t="835">hosted at repos.rwasm.org.</a></p>

<p><a data-t="839">And then say like you also want to</a> <a data-t="844">use your own personal package for use in WebR.</a> <a data-t="849">So there is a project called R Universe.</a> <a data-t="851">It's a project by R OpenSci</a> <a data-t="854">that experiments with ideas for improving the publication</a> <a data-t="857">and discovery of research software in R.</a> <a data-t="860">And R Universe builds Wasm binaries for R packages</a> <a data-t="864">for use in WebR applications.</a> <a data-t="866">So if you have a custom package,</a> <a data-t="868">the easiest way to get up and running</a> <a data-t="871">is to create an R Universe repo for your packages.</a> <a data-t="874">And then the system will automatically build R binaries</a> <a data-t="878">for different things, including WebAssembly.</a> <a data-t="881">And then you can use that in WebR.</a></p>

<h3>Introduction to Quarto</h3>

<p><a data-t="879">And so there are other options.</a> <a data-t="885">There's an rwasm package and things like that,</a> <a data-t="888">but just wanted to highlight that is the recommended</a> <a data-t="892">easiest or quickest way to get started.</a></p>

<p><a data-t="897">In terms of how to use WebR,</a> <a data-t="899">if you go to the WebR documentation linked here,</a> <a data-t="903">it's very robust and very detailed.</a> <a data-t="906">But if you're like me, you look at the front page</a> <a data-t="909">and it says, install the package, npm i WebR.</a> <a data-t="914">And you're like, okay.</a> <a data-t="916">And then import the WebR class.</a> <a data-t="918">And it has this.</a> <a data-t="919">And then you realize like, oh, this is JavaScript.</a> <a data-t="922">So what if you don't know JavaScript like me?</a> <a data-t="926">It's the edge.</a> <a data-t="927">So WebR, it doesn't seem to be as quick and easy</a> <a data-t="930">to use for those who do.</a> <a data-t="932">But if you don't, what options do you have?</a> <a data-t="936">And that is where Quarto comes in.</a></p>

<p><a data-t="940">Quarto is a next generation open source publishing system.</a> <a data-t="945">So it is often called the next evolution</a> <a data-t="948">or the next generation of our markdown.</a> <a data-t="951">So it's expanded upon our markdown.</a> <a data-t="953">What we have done at Posit is taken everything</a> <a data-t="956">that we learned over the past 10 years</a> <a data-t="959">to develop a system like Quarto.</a> <a data-t="962">One of the things about Quarto, it's inherently multilingual.</a> <a data-t="966">So right now it supports R, Python, Julia and Observable,</a> <a data-t="971">but it's been designed so that we can add new languages</a> <a data-t="974">in the future.</a></p>

<p><a data-t="977">Another thing that we've learned is R markdown</a> <a data-t="979">has a lot of packages that does a lot of things.</a> <a data-t="983">If you are an R markdown user,</a> <a data-t="984">you might know like use shringen if you wanna do slides</a> <a data-t="987">and </a><a href="/software/bookdown/">bookdown</a><a data-t="987"> if you wanna do books.</a> <a data-t="990">But Quarto has a shared expression</a> <a data-t="992">for all these core features.</a> <a data-t="994">So what that means is if you do a tab set in a Quarto book,</a> <a data-t="998">it looks the same as a tab set in a Quarto presentation.</a> <a data-t="1002">And the idea behind Quarto is to create dynamic content</a> <a data-t="1006">with Python and R and Julia and Observable.</a> <a data-t="1010">And then another thing is it enables</a> <a data-t="1013">single source publishing.</a> <a data-t="1015">So the idea of creating notebooks or dashboards</a> <a data-t="1018">or books, et cetera, from one source.</a></p>

<p><a data-t="1021">And I will say there are many great resources about Quarto.</a> <a data-t="1025">We're not gonna talk about everything about Quarto.</a> <a data-t="1028">At this time, I'm happy to share the resources.</a> <a data-t="1031">I would say that the Quarto documentation site</a> <a data-t="1034">is absolutely incredible.</a> <a data-t="1036">It is very detailed and how to get started,</a> <a data-t="1039">how to use Quarto.</a> <a data-t="1041">I gave a talk a couple of years ago at R Ladies St. Louis</a> <a data-t="1044">that I'm happy to share that went more into detail</a> <a data-t="1047">in terms of like, you know, getting set up in RStudio</a> <a data-t="1050">and rendering documents and things like that.</a> <a data-t="1052">But for today, I'm just going to be talking</a> <a data-t="1055">about the most critical parts of Quarto</a> <a data-t="1057">that you need to know in order to use WebR</a> <a data-t="1061">and Pyodide in Quarto documents.</a></p>

<p><a data-t="1069">So first, the anatomy of a Quarto document.</a> <a data-t="1073">So a Quarto document has an extension QMD.</a> <a data-t="1075">The first part that we're going to talk about</a> <a data-t="1077">is what is called the YAML heading.</a> <a data-t="1078">So it is denoted by these three dashes</a> <a data-t="1081">and then a key value pairs within these dashes.</a> <a data-t="1084">So the key, for example, can be title</a> <a data-t="1087">and the value is ggplot2-demo.</a> <a data-t="1090">And the YAML heading contains the metadata</a> <a data-t="1094">for your document.</a> <a data-t="1095">So in this case, it's saying,</a> <a data-t="1097">I want to produce an HTML format.</a></p>

<p><a data-t="1100">I can provide additional options like code full true</a> <a data-t="1104">means like the code within the document</a> <a data-t="1107">should collapse so that it's not immediately visible</a> <a data-t="1110">unless somebody clicks it to open it up.</a> <a data-t="1114">And so the YAML heading, just to reiterate,</a> <a data-t="1117">is the kind of metadata of your document.</a></p>

<p><a data-t="1122">The next part to talk about is the markdown text.</a> <a data-t="1125">So markdown is a lightweight way of writing formatted text</a> <a data-t="1130">and in a way that it looks very readable.</a> <a data-t="1134">You can do things like, for example, here in line eight,</a> <a data-t="1138">the two pound signs mean this is a heading.</a> <a data-t="1141">So this will be rendered as a heading.</a> <a data-t="1143">You can do things like embed images, links, tables,</a> <a data-t="1148">format your text, all sorts of things.</a> <a data-t="1151">But the really nice thing about markdown</a> <a data-t="1153">is that once you learn it, it's easily readable</a> <a data-t="1158">when you say, put it up on GitHub.</a> <a data-t="1162">It doesn't come out in a very nested,</a> <a data-t="1165">difficult to read format.</a> <a data-t="1166">It looks like text.</a></p>

<h3>Shinylive: Shiny apps without a server</h3>

<p><a data-t="2119">So we've talked about Quarto documents</a> <a data-t="2121">and executable R chunks and things like that,</a> <a data-t="2124">but you might be wondering, well, what about Shiny?</a> <a data-t="2128">And if you are new to Shiny,</a> <a data-t="2130">Shiny allows you to create web applications</a> <a data-t="2133">without needing to know JavaScript, HTML,</a> <a data-t="2136">or anything like that.</a> <a data-t="2138">You can just do it in pure R or pure Python,</a> <a data-t="2140">which is pretty amazing</a> <a data-t="2143">if you don't know JavaScript like me.</a></p>

<p><a data-t="2146">But say you've created your Shiny dashboard,</a> <a data-t="2149">very beautiful, ready to go.</a> <a data-t="2152">And then you need to share it with somebody.</a> <a data-t="2155">So the traditional Shiny architecture</a> <a data-t="2159">for deploying a Shiny app</a> <a data-t="2161">involves a separate server and a client.</a> <a data-t="2165">And we've mentioned the word server several times today,</a> <a data-t="2169">but essentially a server is a machine</a> <a data-t="2171">that's connected to the internet</a> <a data-t="2173">that runs 24 seven ready to run your Shiny app.</a> <a data-t="2176">And the server runs R or Python and Shiny,</a> <a data-t="2181">and clients connect via the web browser.</a> <a data-t="2185">And each client keeps an open web socket connection</a> <a data-t="2188">as long as they are using the application.</a></p>

<p><a data-t="2202">And in terms of hosting Shiny apps,</a> <a data-t="2204">there are a lot of different options</a> <a data-t="2206">for hosting your server.</a> <a data-t="2208">So for example, you can have RStudio server on your laptop</a> <a data-t="2212">and say you need some scalability</a> <a data-t="2214">or additional security for your app,</a> <a data-t="2216">you can get Posit Connect,</a> <a data-t="2218">which is an enterprise version from Posit</a> <a data-t="2221">and for hosting your Shiny apps.</a> <a data-t="2226">There are also cloud-based solutions.</a> <a href="/software/shinyapps/">Shinyapps</a><a data-t="2228">.io has been around for a long time.</a> <a data-t="2232">There is a new product called Posit Connect Cloud</a> <a data-t="2236">that lets you publish Shiny</a> <a data-t="2238">and also all sorts of kind of Python frameworks as well.</a> <a data-t="2244">And then like things like Hugging Face,</a> <a data-t="2247">you can roll your own if you would like.</a></p>

<p><a data-t="2250">And then in terms of choosing what the right tool is,</a> <a data-t="2254">it's a very wide spectrum</a> <a data-t="2256">in terms of the number of users.</a> <a data-t="2258">For some people, we might just be sharing our apps</a> <a data-t="2260">with one or two other people.</a> <a data-t="2262">For other people, we might be talking thousands.</a> <a data-t="2265">And then there are some folks in between,</a> <a data-t="2267">like a professor, an academic,</a> <a data-t="2270">might have a few hundred folks looking at their apps.</a> <a data-t="2273">But what if you don't have the infrastructure</a> <a data-t="2276">or the resources to set up either of these kind of options?</a></p>

<p><a data-t="2283">Another kind of hosting service are web hosting services.</a> <a data-t="2287">And they offer a lot of scalability and features</a> <a data-t="2290">and perhaps a low price point,</a> <a data-t="2294">like GitHub Pages or Netlify or Quarto Pub,</a> <a data-t="2297">but the issue is they can't run traditional Shiny apps.</a> <a data-t="2301">And the reason is that these are called static web hosting.</a> <a data-t="2304">They're very good in terms of reach and affordability.</a> <a data-t="2307">You don't have to think about servers anymore.</a> <a data-t="2310">So when you publish, say, a Quarto website,</a> <a data-t="2313">you just give them your rendered Quarto document files.</a> <a data-t="2318">But there is no way to run Shiny apps</a> <a data-t="2321">on these kind of servers,</a> <a data-t="2323">on these kind of web hosting services.</a></p>
