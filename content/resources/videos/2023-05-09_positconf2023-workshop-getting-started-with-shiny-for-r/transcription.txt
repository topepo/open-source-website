<p><a data-t="29">The first is building a solid foundation.</a> <a data-t="32">While Shiny apps are written in R code, they use a reactive programming paradigm, which</a> <a data-t="36">is very different from what most R users are experienced with when they're developing R</a> <a data-t="41">code themselves.</a> <a data-t="42">Throughout this workshop, we will help you develop a thorough understanding of this reactive</a> <a data-t="45">paradigm, how you implement it, how you debug applications using it, so that you are prepared</a> <a data-t="50">to write your own Shiny apps once you finish the workshop.</a></p>

<p><a data-t="54">We also want to help you putting it all together.</a> <a data-t="56">Throughout the course of the day, we'll be working on the iterative development of a</a> <a data-t="59">complete and real-world application that includes complex user interactions and internal functionality.</a> <a data-t="65">We will be starting with a very basic app, and we will then be incrementally adding functionality</a> <a data-t="70">and introducing new core concepts of Shiny along the way as we add features to that application.</a> <a data-t="75">We'll take time to discuss how the added features interact with the existing functionality of</a> <a data-t="78">the app, which is an incredibly important but often overlooked aspect of developing</a> <a data-t="83">Shiny apps that is often hard to pick up from the documentation alone.</a></p>
