<h3>Why use tidymodels?</h3>

<p><a data-t="651">So for this portion of the presentation, instead of walking through the notebook that does the training and the evaluating, I'll leave it to folks to look through the notebooks that are on that GitHub page after the presentation.</a> <a data-t="667">And instead, I'll try to answer the question of why you might be interested in using tidymodels over other modeling frameworks that are out there.</a></p>

<p><a data-t="681">The first reason why you should be interested in using tidymodels is that tidymodels is consistent in a way that other modeling frameworks and modeling packages are not.</a> <a data-t="692">An example that I'll use to call this out here, we can start off by showing the code to fit a linear model using the lm function with tidymodels and a linear model using only the lm function.</a> <a data-t="710">On the left hand side, there's just one line of code. <p><a data-t="735">On the right hand side, the code does the same thing, except the tidymodels wrapper has been dropped on top of that function.</a> <a data-t="744">The code reads, we'll use a linear regression using the lm function as our computational engine and we'll fit that model using the same dataset and the same preprocessing formula, mpg, as a function of everything.</a> <a data-t="763">When you see this, you might think it seems a whole lot easier to just go ahead and use lm, it's only one line of code, and I hear you.</a> <a data-t="774">Where we really see the value of tidymodels, though, is when we start to introduce complexity into the modeling pipeline.</a></p>

<p><a data-t="782">So let's say you're using the lm function and it's not doing everything you need it to do, and you want to add, for example, a regularization term to your linear regression.</a> <a data-t="793">If you're not familiar with that, it doesn't matter to understand what I'm about to demonstrate here.</a> <a data-t="799">I want you to look at that line 3 in the tidymodels portion of the code where I've set the engine to lm.</a> <a data-t="807">If I want to transition that code to instead use glmnet, all that I need to do is change the portion of the code that said lm to instead say glmnet.</a> <a data-t="820">And you can see on the left hand side, it was much more than just one of the lines of code that needed to change.</a> <a data-t="828">Instead, here, glmnet, for its first argument, is going to take all of the predictors and they can only be a matrix.</a> <a data-t="836">So thankfully, here, in mtcars, all of the predictors are numbers anyway, and so they're one homogenous type and we can easily convert it with as.matrix.</a> <a data-t="848">And then instead of the name of the outcome variable like it was in lm, the outcome here is specified as the actual vector of values in the mtcars dataset.</a> <a data-t="862">So that's mtcars dollar sign mpg.</a></p>

<p><a data-t="866">To transition that code, I needed to know not only that glmnet existed and was able to perform regularization,</a> <a data-t="874">but I needed to look at the documentation to see how that function wanted the predictors to be formatted,</a> <a data-t="881">how it wanted the outcome to be formatted, whether it needed any additional arguments,</a> <a data-t="887">which of the functions in the package I needed to use to do so.</a> <a data-t="891">And again, in tidymodels, all I needed to do was change the value of that engine argument.</a> <a data-t="899">And the story goes the same way with any other engine you can imagine.</a></p>

<p><a data-t="1003">Another reason to use tidymodels is safety.</a> <a data-t="1008">tidymodels is designed to safeguard you from common modeling pitfalls.</a> <a data-t="1014">One of the things that tidymodels is designed to protect you from is data leakage.</a> <a data-t="1019">A 2023 review found data leakage to be a widespread failure mode in machine learning-based science.</a> <a data-t="1026">Data leakage is when information from the set you're going to use, or the portion of the data that you're going to use to evaluate your model, somehow leaks into the portion of the data that you're using to train your model.</a> <a data-t="1042">If a model is being evaluated on data that it's seen already, then it will perform better than on data that it hasn't seen before.</a> <a data-t="1053">And so it will seem like your model is performing really well, but it's actually not, and you'll only learn that the model isn't performing well once it's deployed.</a></p>

<p><a data-t="1063">So in our example, in the mutagen example, if our model seems to be able to classify drugs as either mutagens or non-mutagens really, really well,</a> <a data-t="1076">if we go ahead and deploy that, thinking that the model is performing perfectly,</a> <a data-t="1082">then scientists will mistakenly interpret the results from that model and take them at face value.</a></p>

<p><a data-t="1093">Another common failure mode is overfitting.</a> <a data-t="1098">Overfitting leads to analysts believing models are more performant than they actually are.</a> <a data-t="1104">So this is really similar to that first failure mode, where in the case of an overfit model, in our mutagen example,</a> <a data-t="1112">the model is going to perform really, really well on the portion of the data that was used to train the model.</a> <a data-t="1120">And if we don't have tools to then evaluate the model on data that it hasn't seen before,</a> <a data-t="1127">then that model might just learn every little intricacy of the training data and won't generalize well to data that it hasn't seen before.</a></p>

<p><a data-t="1137">A last component of safety that tidymodels make sure to protect its users from is irreproducibility.</a> <a data-t="1147">Implementations of the same machine learning model give differing results in many popular implementations of machine learning algorithms,</a> <a data-t="1158">resulting in irreproducibility of modeling results.</a> <a data-t="1162">So each of the citations listed in the bottom here in order are reports on data leakage and overfitting and irreproducibility.</a> <a data-t="1172">And tidymodels is designed to safeguard you from all of them.</a> <a data-t="1176">We developed this software so that it's really hard to do the things that are wrong and that an analyst would commonly want to do,</a> <a data-t="1185">and makes the things that are usually difficult but pay off in the long run, it makes those things really easy.</a></p>

<p><a data-t="1196">Another reason why you might want to use tidymodels is that tidymodels is designed to be communicable.</a> <a data-t="1203">All tidymodels objects, or at least many of them, have visualization functions called autoplot that are associated with them.</a> <a data-t="1212">So if I make a receiver operating curve with the tidymodels, which is a tool used to evaluate models,</a> <a data-t="1222">I can pass that object to autoplot and see what that curve looks like in 2D space.</a> <a data-t="1228">I can also pass, for example, the results of trying to tune a random forest algorithm.</a> <a data-t="1234">Again, I just pass that result to the autoplot function, and we have built-in visualization procedures for that model object.</a></p>

<p><a data-t="1244">The same story goes for iterative search results.</a> <a data-t="1248">Things like Bayesian optimization, simulated annealing,</a> <a data-t="1252">these are advanced machine learning techniques that are built into tidymodels.</a> <a data-t="1258">When I use tidymodels to generate these results, I can understand them by quickly plotting them.</a></p>

<p><a data-t="1266">Another example to use tidymodels is completeness.</a> <a data-t="1270">There's built-in support for 99 machine learning models once you load the tidymodels packages,</a> <a data-t="1277">and there are even more packages if you load extensions that are contributed by the community.</a> <a data-t="1281">tidymodels is also complete with reference to data preprocessing techniques.</a> <a data-t="1292">With the three main packages for preprocessing that are loaded when you load tidymodels,</a> <a data-t="1297">we load in 102 different ways to process data that comes in raw,</a> <a data-t="1303">and turn it into something that a model knows what to do with.</a></p>

<p><a data-t="1311">I've hinted at this already, but again, tidymodels is extensible.</a> <a data-t="1314">If you can't find the machine learning technique you need,</a> <a data-t="1317">which is either the machine learning model or the preprocessing technique,</a> <a data-t="1322">you can add it yourself.</a> <a data-t="1323">We have APIs that are designed to make it really easy to contribute your own extensions</a> <a data-t="1330">that are designed with your problem context in mind.</a></p>

<p><a data-t="1336">The last reason I'll list here is deployability,</a> <a data-t="1339">and this will bring us into that final step three that we'll drop into the live demo for.</a> <a data-t="1345">tidymodels is tightly integrated with Vetiver,</a> <a data-t="1348">which is a free and open-source package for model deployment.</a> <a data-t="1351">And Vetiver is configured to quickly and easily and securely ship a model to Posit Connect,</a> <a data-t="1359">which is part of the suite of products in Posit Team.</a></p>
