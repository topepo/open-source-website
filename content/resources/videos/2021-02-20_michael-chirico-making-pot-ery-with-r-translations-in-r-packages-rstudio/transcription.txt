<h3>Introducing PoTools</h3>

<p><a data-t="113">The whole time I was working on this, I was kind of taken aback by how much friction there</a> <a data-t="118">is in the process.</a> <a data-t="118">There's a lot of pain points in having to learn a whole new language, which</a> <a data-t="118">is this language of get text and pot files, po files, mo files, all these things, which</a> <a data-t="124">you don't really need to understand that much to add translations to your package.</a> <a data-t="129">So over</a> <a data-t="129">the last couple of months, I've been working on a package called PoTools.</a> <a data-t="133">The goal is to</a> <a data-t="133">really eliminate as much of those frictions as possible to make it easier to add translations</a> <a data-t="137">to the packages.</a> <a data-t="141">And as of right now, there's only one user-facing function.</a> <a data-t="141">It's called</a> <a data-t="141">translate package.</a> <a data-t="147">And it does all the legwork for you of adding together all those get text</a> <a data-t="147">spacing utilities and just walks you through adding translations to all the messages it</a> <a data-t="151">finds with your package.</a></p>

<h3>Setting up your package for translation</h3>

<p><a data-t="158">So with that in mind, we really need to think about how to set your</a> <a data-t="158">package up to be translated in the first place.</a> <a data-t="163">There are some general rules about how to</a> <a data-t="163">do development on your project that will make it easier for translation.</a> <a data-t="167">The first</a> <a data-t="167">one is about templating.</a> <a data-t="174">Here, the stop message that we see, it's something very common to</a> <a data-t="174">see and it's used all the time in my own development.</a> <a data-t="179">But in terms of translation, it makes things</a> <a data-t="179">a bit harder.</a> <a data-t="184">What a translator would see from the first instance is three strings.</a> <a data-t="184">Sound, columns, comma, but, and are needed.</a> <a data-t="209">And out of context, those things are very</a> <a data-t="209">hard to translate.</a> <a data-t="214">And there's some issues with duplication where it would be even harder</a> <a data-t="214">to translate.</a> <a data-t="220">If it's in the templated form, the translator is free to rearrange things,</a> <a data-t="220">which will have to be done in, for example, Japanese, which has a totally different grammar.</a></p>
