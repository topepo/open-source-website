<h3>Software vs. statistical performance</h3>

<p><a data-t="151">Now it's true that all software products have to be maintained, I don't think I, at least</a> <a data-t="155">I, have never been able to write a piece of software that does not require some kind of</a> <a data-t="160">maintenance, but machine learning models are unique in that they have both software</a> <a data-t="166">characteristics or properties, and also statistical characteristics or properties.</a> <a data-t="172">So let's think about something that does not have any statistical properties, it's just</a> <a data-t="177">software, it's just code, like an example of this would be a fairly straightforward</a> <a href="/software/shiny-r/">shiny</a><a data-t="182"> app.</a></p>

<p><a data-t="182">Unlike that, something that's only software, when it comes to a model, you could entirely</a> <a data-t="188">retrain the model with new data and just totally change its statistical properties,</a> <a data-t="195">and you could not change its software properties at all.</a> <a data-t="198">So we have different kinds, like machine learning models are complex products in that they have,</a> <a data-t="204">they are both software products and statistical products.</a></p>

<p><a data-t="208">Whether we're talking about the software or the statistical properties, what we're really</a> <a data-t="211">talking about is performance.</a> <a data-t="213">And one of the just most basic prerequisites for model maintenance is monitoring that</a> <a data-t="219">performance.</a> <a data-t="220">So when we monitor, when we measure model performance, this lets us say, my model is</a> <a data-t="226">performing well.</a> <a data-t="228">But it turns out, if you hear someone say that, or you're having a conversation about</a> <a data-t="231">that, people can mean different things when they say this.</a></p>

<p><a data-t="258">Another person, when they say my model is performing well, they may mean something different.</a> <a data-t="263">So this person, when she is saying this, she means that the model returns predictions that</a> <a data-t="268">are close to the true values for whatever it is that we're trying to predict here.</a> <a data-t="274">So I am, I consider myself by profession largely a data practitioner.</a> <a data-t="280">And I bet in this audience, that is true of most of you as well.</a> <a data-t="283">And as data practitioners, we focus on monitoring this kind of statistical performance.</a></p>

<h3>Data drift and concept drift</h3>

<p><a data-t="364">There's some vocabulary around these kind of problems.</a> <a data-t="367">You may have heard this term model drift.</a> <a data-t="369">I love this term.</a> <a data-t="371">It's very evocative to me.</a> <a data-t="372">Model is drifting off, away from where I need it to be.</a> <a data-t="376">So that's a generic or general term.</a> <a data-t="379">And there are also more specific terms, and we're going to talk about two.</a> <a data-t="382">The first one is data drift, and the second one is concept drift.</a></p>

<p><a data-t="386">So let's say that I run a laundry service in my city, and I'm going to build a model</a> <a data-t="391">to predict whether someone's going to be my customer.</a> <a data-t="395">I use inputs, features, predictors to the model.</a> <a data-t="399">Maybe I use things like how many loads of laundry a household needs to do a month.</a> <a data-t="405">Maybe income.</a> <a data-t="405">Maybe whether someone lives in a house or an apartment.</a> <a data-t="408">I use these inputs to train a model and get a predicted probability that someone will</a> <a data-t="413">be my customer.</a></p>

<p><a data-t="416">Data drift is about shift, drift, change in those inputs.</a> <a data-t="422">And when I use the model, compared to when I developed the model.</a> <a data-t="427">So how do you measure this?</a> <a data-t="428">How do you know you have it?</a> <a data-t="430">Using tools you are probably very familiar with, things like summarization, visualization.</a> <a data-t="435">Let's look at a possible example.</a></p>

<p><a data-t="438">Let's say on the X axis here is the number of loads of laundry that a household needs</a> <a data-t="444">to do every month.</a> <a data-t="446">So I built my model, I'm using it in my business, I'm going along, and I'm going to keep track</a> <a data-t="451">of the distribution that I see as I'm using it.</a> <a data-t="454">To know if I'm experiencing model drift, I need to compare it to what I had when I trained</a> <a data-t="460">the model.</a> <a data-t="461">So if it looks like this, I might think I'm a-okay.</a> <a data-t="465">Those that look like they're drawn from the same distribution, I'm okay.</a> <a data-t="469">I'm talking about comparing this visually, but you can also use any of the statistical</a> <a data-t="474">tests that are appropriate to the kind of data you have to do this kind of measurement.</a> <a data-t="479">If by contrast I see something like this, then I know something has changed.</a> <a data-t="483">The distribution of this thing I'm using as a feature has changed.</a> <a data-t="487">If it's important in the model, if it changes, that means the outputs from my model will</a> <a data-t="492">change as well.</a> <a data-t="492">I'm going to get a different distribution of predictions than I would have thought.</a> <a data-t="497">So data drift, to understand if you have data drift, you need to monitor your inputs.</a></p>

<p><a data-t="503">Concept drift is about the relationship between the input and the output, and concept drift</a> <a data-t="508">means that is changing over time.</a> <a data-t="510">So let's say in the city where I have my laundry service, a bunch of new apartments are being</a> <a data-t="516">built and all the new apartments have laundry units inside of the apartments.</a> <a data-t="521">If house versus apartment is a feature that I use in my model, the fact that the apartments,</a> <a data-t="528">the nature of them is changing means I will have concept drift.</a> <a data-t="532">The relationship between this feature and my output is changing over time.</a> <a data-t="536">So with concept drift, it's not enough only to monitor your inputs, but we also need to</a> <a data-t="542">monitor your outputs.</a></p>

<p><a data-t="544">This is a place that vetiver comes in because we're now talking about specific machine learning</a> <a data-t="549">statistical kind of performance, and either in R or Python, we have functions that say</a> <a data-t="554">you have some laundry service monitoring data.</a> <a data-t="557">You can use the functions to compute the metrics, measuring the statistical performance of your</a> <a data-t="562">model at some given time aggregation with the metrics that are important to you in your</a> <a data-t="569">laundry service business that you're clearly running.</a> <a data-t="572">So vetiver also has functions to help you get started with these kind of basic plots,</a> <a data-t="577">and because everything you have access to is code, you can extend these, you can add</a> <a data-t="582">on to these here.</a></p>

<h3>Feedback loops</h3>

<p><a data-t="584">Now one thing that it's important to know as we start talking about model monitoring</a> <a data-t="590">is that the fact that you put a model into a system actually can cause those things we</a> <a data-t="597">were just talking about, data drift and or concept drift.</a> <a data-t="600">We talk about this, we call this feedback loops.</a></p>

<p><a data-t="603">So what's happening here is the user, if a user takes some action as a result of a prediction,</a> <a data-t="610">or if you have users like correcting predictions, or some systems produce kind of feedback automatically.</a> <a data-t="617">To be a little more concrete about this, let's walk through a couple of examples.</a></p>

<p><a data-t="621">Many of us, I expect, are customers of a streaming service like Netflix or Hulu.</a> <a data-t="627">So in those apps, you will see recommended pieces of content that's driven by machine</a> <a data-t="632">learning models.</a> <a data-t="633">And if you click, if you choose one of those recommendations, that's a signal to the system</a> <a data-t="641">that that prediction was good, that data is collected and then used to improve the model.</a> <a data-t="648">This is an example of a good feedback loop.</a></p>

<p><a data-t="651">Sorry to report that not all feedback loops are good.</a> <a data-t="655">There's a really interesting case study from Stripe where they had a data set of credit</a> <a data-t="659">card transactions and they trained a model to predict which ones were fraudulent.</a> <a data-t="666">They then took that model, which looked great, and they put it into their system in a way</a> <a data-t="671">such that they blocked the credit card transactions.</a> <a data-t="674">They actually never became credit card transactions.</a> <a data-t="677">Some of you may be seeing where this is going.</a> <a data-t="680">So they, you know, you were using the model, as happens with models in production, it gradually</a> <a data-t="686">started doing a little bit worse.</a> <a data-t="688">When they went to try to retrain their model, they did not, they had not captured information</a> <a data-t="694">to be able to treat them like credit card transactions anymore.</a> <a data-t="697">So they actually could not retrain their model.</a> <a data-t="700">So they had to go back, do new data engineering so that they captured information in a different</a> <a data-t="705">way and put these data sets together to be able to retrain their model.</a></p>

<p><a data-t="710">This is an example of a feedback loop that's bad for a, like a single company, but we actually</a> <a data-t="716">have evidence of feedback loops that can be bad for us as society as a whole.</a> <a data-t="720">If you take data from, say, arrests and try to train a model to predict, say, what areas</a> <a data-t="726">of a town or city are going to have more crime in them and then take some intervention based</a> <a data-t="732">on that prediction, you can end up with a feedback loop, depending on the nature of</a> <a data-t="737">the intervention, that makes certain areas actually worse off.</a> <a data-t="742">You can make certain neighborhoods have worse experience than if you didn't use a model</a> <a data-t="747">at all.</a></p>
