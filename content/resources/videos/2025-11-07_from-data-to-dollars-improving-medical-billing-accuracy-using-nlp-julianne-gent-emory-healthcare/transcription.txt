<h3>Evaluating solutions</h3>

<p><a data-t="134">So, of course, our office said, of course we can.</a> <a data-t="136">So, we actually considered a couple of solutions.</a></p>

<p><a data-t="140">The first was to require clinicians to review</a> <a data-t="143">every single note they've written that day</a> <a data-t="145">and double-check for billing accuracy.</a> <a data-t="148">Well, we just talked about that we need</a> <a data-t="149">to meet physicians where they're at.</a> <a data-t="151">We're not gonna add yet another tedious task</a> <a data-t="153">to their already long stack of tasks to do</a> <a data-t="157">and take away from patient care.</a></p>

<p><a data-t="159">So, we're gonna go ahead and eliminate that.</a> <a data-t="162">The second one we looked at was called a SMART phrase,</a> <a data-t="166">and it is a functionality that is provided</a> <a data-t="168">by our electronic medical record or EMR software vendor.</a> <a data-t="173">So, basically what this does,</a> <a data-t="175">it allows clinicians to type in the name of a template</a> <a data-t="178">that consists of a lot of commonly used phrases.</a> <a data-t="180">For example, here it says critical care billing statement.</a> <a data-t="184">And then it automatically adds it to their note.</a> <a data-t="187">So, the data typed into this phrase,</a> <a data-t="190">where you can see with the three asterisks</a> <a data-t="192">would be where you put the billing time,</a> <a data-t="194">is actually stored very cleanly in a data warehouse</a> <a data-t="197">where we can extract that data.</a></p>

<p><a data-t="200">However, we found two large issues with this approach.</a> <a data-t="204">First, SMART phrase templates are just one</a> <a data-t="207">of many documentation tools that physicians can use,</a> <a data-t="211">that they're optional, they're not required,</a> <a data-t="213">and many physicians don't use it.</a> <a data-t="215">Many physicians like to write their own text in freehand.</a> <a data-t="218">And secondly, we recently found that a lot</a> <a data-t="222">of clinician workflow is doing the old copy and paste</a> <a data-t="225">of notes from a previous,</a> <a data-t="228">text from a previous note into a new note.</a> <a data-t="231">This happens a lot when care is not changed</a> <a data-t="233">from day to day with a patient</a> <a data-t="235">and it saves them a lot of time and extra typing.</a> <a data-t="237">But when this happens, the data that I showed</a> <a data-t="241">in that asterisk in that phrase is no longer extractable.</a> <a data-t="244">It is no longer put into that data warehouse.</a></p>

<h3>Why not an LLM?</h3>

<p><a data-t="248">So, I know what a lot of data scientists</a> <a data-t="250">in this room are probably thinking.</a> <a data-t="252">This is just untextured, sorry, excuse me.</a> <a data-t="256">This is unstructured text.</a> <a data-t="258">Just use an LLM.</a> <a data-t="260">We got asked this a lot by a lot of physicians.</a> <a data-t="262">Can't you just use AI?</a> <a data-t="263">What's this LLM?</a> <a data-t="265">Well, we have a couple of limitations,</a> <a data-t="268">especially, specifically to healthcare.</a></p>

<p><a data-t="271">So, there are three main challenges we face</a> <a data-t="273">when we thought about using an LLM.</a> <a data-t="275">The first being cost.</a> <a data-t="276">We all know that LLMs are not cheap,</a> <a data-t="279">but they also have a large infrastructure cost.</a> <a data-t="282">So, if you're trying to run an LLM on a daily basis,</a> <a data-t="284">requires powerful GPUs, a dedicated cloud space,</a> <a data-t="289">which is something maybe a large university,</a> <a data-t="291">a pharmaceutical company or tech company</a> <a data-t="293">could have and support,</a> <a data-t="296">but not a small to medium-ish healthcare system.</a></p>

<p><a data-t="300">Secondly,</a> <a data-t="304">even if we somehow got the cost,</a> <a data-t="306">both financially and infrastructurally,</a> <a data-t="308">one of the issues is timeliness.</a> <a data-t="311">Anything related to healthcare quality,</a> <a data-t="313">so that means anything patient safety,</a> <a data-t="315">or anything that could get audited</a> <a data-t="316">by a government official or a regulatory branch,</a> <a data-t="320">like as we saw earlier,</a> <a data-t="322">the auditor coming for medical fraud,</a> <a data-t="324">can come at any time, at any point of the year,</a> <a data-t="327">and can come multiple times.</a> <a data-t="328">So, we were given a turnaround time</a> <a data-t="330">of about a week and a half.</a> <a data-t="331">And there is not really a good feasible way in LLM</a> <a data-t="334">we'd be able to train, validate in that timeframe</a> <a data-t="339">to get ready for any potential auditing.</a></p>

<p><a data-t="342">And finally, the biggest one is HIPAA compliance.</a> <a data-t="346">So, for those of you who are not from the U.S.,</a> <a data-t="349">HIPAA is a federal law</a> <a data-t="351">that protects patients' healthcare privacy and data</a> <a data-t="353">that is taken very, very seriously.</a> <a data-t="356">So, if we wanted to use an LLM</a> <a data-t="358">that's already on market, like ChatGPT or Copilot,</a> <a data-t="362">if we put healthcare data into that LLM,</a> <a data-t="367">we would be violating HIPAA.</a> <a data-t="370">And could have devastating consequences</a> <a data-t="371">to a healthcare system.</a> <a data-t="372">So, all in all, we scrapped the LLM.</a></p>

<blockquote><a data-t="342">And finally, the biggest one is HIPAA compliance.</a> <a data-t="356">So, if we wanted to use an LLM</a> <a data-t="358">that's already on market, like ChatGPT or Copilot,</a> <a data-t="362">if we put healthcare data into that LLM,</a> <a data-t="367">we would be violating HIPAA.</a></blockquote>

<h3>Building the NLP tool</h3>

<p><a data-t="375">So, because we scrapped three of our main tools,</a> <a data-t="379">we decided, why don't we just make our own?</a> <a data-t="382">If you want it done right, do it yourself.</a></p>

<p><a data-t="389">So, how we built this tool.</a> <a data-t="390">So, here's an example.</a> <a data-t="393">Don't worry, healthcare people, this is fake data.</a> <a data-t="395">We're not breaking HIPAA today.</a> <a data-t="398">So, here's an example of what a typical</a> <a data-t="401">hospital intensive care unit clinical note would look like.</a> <a data-t="404">You can see it's all free text.</a> <a data-t="406">It's incredibly messy.</a> <a data-t="407">Tons of random numeric data.</a></p>

<p><a data-t="411">Luckily, our EMR vendor that I talked about earlier</a> <a data-t="414">stores all of this free text in a data warehouse,</a> <a data-t="417">which we can extract via SQL code.</a> <a data-t="420">And we can also get a lot of other important information,</a> <a data-t="422">like who wrote the note and when.</a> <a data-t="425">And so, now we have this messy free text data</a> <a data-t="428">in a cleaner format.</a> <a data-t="430">So, we can now see Meredith Gray,</a> <a data-t="432">our favorite dramatic physician,</a> <a data-t="434">wrote this note for this patient on New Year's Day.</a></p>

<p><a data-t="435">So, we could theoretically take that SQL output,</a> <a data-t="438">copy and paste it into an Excel file,</a> <a data-t="440">and import it into R.</a> <a data-t="442">But we're data people.</a> <a data-t="443">We want things to be automated.</a> <a data-t="444">We want to reduce the manual process as much as possible.</a> <a data-t="448">So, instead, we utilized the </a><a href="/software/odbc/">odbc</a><a data-t="448"> package</a> <a data-t="451">to connect the SQL data warehouse directly to R.</a></p>

<p><a data-t="456">And so, in the DB get query function down below,</a> <a data-t="460">you can actually copy and paste your SQL query.</a> <a data-t="464">And then, you can now directly import the data</a> <a data-t="467">from the SQL data warehouse directly into R</a> <a data-t="470">in one stop shop, one motion.</a> <a data-t="472">No copy and pasting, no Excel needed.</a></p>

<p><a data-t="476">So, the next step was to create a data set of keywords</a> <a data-t="479">that we find in these notes</a> <a data-t="481">that are specifically related to billing.</a> <a data-t="483">So, we actually worked with our critical care clinicians.</a> <a data-t="486">They looked at several notes of their colleagues</a> <a data-t="489">and kind of tagged the most commonly used phrases</a> <a data-t="491">or the phrases that we would need to have in a note</a> <a data-t="494">for regulatory purposes that are required.</a> <a data-t="497">And we put these into a keyword data set.</a> <a data-t="501">So, I'm showing probably about the first 15 or so,</a> <a data-t="503">but there's tons because there's so much variation</a> <a data-t="506">in ways that clinicians can bill and put that phrasing in.</a></p>

<p><a data-t="510">So, the reason we also include the minutes</a> <a data-t="512">or min information is because when you saw</a> <a data-t="515">in the free text note previously,</a> <a data-t="517">there's lots of numeric values.</a> <a data-t="519">There's telephone numbers, there's labs, there's </a><a href="/software/vitals/">vitals</a><a data-t="519">,</a> <a data-t="521">there's PIC numbers, which is a physician number.</a> <a data-t="525">And so, we wanna make sure that</a> <a data-t="527">because we aren't using an LLM,</a> <a data-t="529">we want the tool to be able to pull the number</a> <a data-t="532">and know that the number is the number of minutes</a> <a data-t="534">and not a blood pressure or a heart rate.</a></p>

<p><a data-t="540">And so, finally, we coded the function</a> <a data-t="543">that serves as our NLP, and we were actually able</a> <a data-t="545">to use this with some very basic and ready-to-use</a> <a data-t="548">and readily available tools from Posit.</a> <a href="/software/tidyverse/">Tidyverse</a><a data-t="552">, Dipler, and stringr.</a> <a data-t="554">That is all we needed to make a basic NLP tool</a> <a data-t="557">for our purposes.</a></p>

<p><a data-t="559">So, basically, what this does is that it first determines</a> <a data-t="562">if there's a number present before,</a> <a data-t="565">but also within one of the keyword or phrases,</a> <a data-t="568">and extracts it into its own column.</a> <a data-t="570">And if it recognizes that there's no number present before,</a> <a data-t="574">that's also within the phrase.</a> <a data-t="575">It'll see if there's a number that's afterwards,</a> <a data-t="577">but still within that phrase.</a> <a data-t="579">That was in the keyword data set.</a></p>

<p><a data-t="582">We then apply this function</a> <a data-t="583">to the entire clinical note data set.</a> <a data-t="586">So, if we apply the results of our NLP,</a> <a data-t="589">natural language processing tool, to our example,</a> <a data-t="593">we can now see that there is a column called billing time</a> <a data-t="597">that has correctly extracted the number of 35 minutes.</a> <a data-t="601">And it knows that none of the other numbers</a> <a data-t="603">in that free text were the billing time.</a></p>

<p><a data-t="609">So, finally, we then created an outlier variable</a> <a data-t="612">that we can identify potential documentation errors.</a> <a data-t="615">So, at our practice, we found it would be pretty rare</a> <a data-t="618">for a clinician to bill less than 10 minutes of time,</a> <a data-t="621">or more than 100 minutes in a single note.</a> <a data-t="625">So, we use these values as outlier thresholds.</a> <a data-t="628">And then, if the billing time is outside of these thresholds,</a> <a data-t="631">or if it doesn't exist,</a> <a data-t="633">because that is another issue we want to identify,</a> <a data-t="636">if there is no billing information in the note,</a> <a data-t="638">the column is then marked as a one,</a> <a data-t="640">and then zero otherwise, classic binary variable.</a> <a data-t="644">So, this allows our end users, our clinicians,</a> <a data-t="646">to filter the data to identify those outliers quickly.</a></p>

<h3>Monthly process and future plans</h3>

<p><a data-t="722">So, how do we currently use this tool?</a> <a data-t="724">So, following our successful validation,</a> <a data-t="727">for the most part, we still are in the process</a> <a data-t="729">of continuous validation, as I will point out here.</a> <a data-t="733">The tool actually went live in February 2025.</a> <a data-t="736">So, here's our monthly process.</a> <a data-t="739">We first apply the tool</a> <a data-t="741">to the previous month's clinical notes.</a></p>

<p><a data-t="744">Then the new data is sent</a> <a data-t="746">to critical care leadership for manual review.</a> <a data-t="749">Sometimes they like to look at some certain outliers,</a> <a data-t="752">maybe if a patient is really complex,</a> <a data-t="755">sometimes that justifies a very high billing time,</a> <a data-t="757">and so they just want to make sure</a> <a data-t="758">that everything is up to snuff.</a></p>

<p><a data-t="761">So, then critical care leadership</a> <a data-t="764">will notify the clinicians if they have notes</a> <a data-t="767">that are within those outliers.</a> <a data-t="770">And if they find that the physician</a> <a data-t="772">hasn't made that many errors,</a> <a data-t="773">or if the errors are pretty minor,</a> <a data-t="774">maybe they put 40 instead of 400,</a> <a data-t="777">or something kind of obvious,</a> <a data-t="780">the clinicians will then be notified</a> <a data-t="782">and the documentations are fixed.</a> <a data-t="784">However, if we find that there's maybe a physician or two</a> <a data-t="787">that is constantly missing documentation,</a> <a data-t="790">or if they're constantly making very inaccurate statements</a> <a data-t="795">all of the time, they then get flagged</a> <a data-t="797">for further investigation.</a></p>

<p><a data-t="800">And then finally, critical care leadership</a> <a data-t="803">notifies our team if they have found</a> <a data-t="805">any valuable uncaptured phrases</a> <a data-t="808">related to billing text that we can then add</a> <a data-t="812">back to our keyword data set, and the process continues.</a></p>

<p><a data-t="817">So, the last part I want to talk about</a> <a data-t="819">is our expansions and our future analytics plans.</a> <a data-t="822">So, we want to actually expand this to other specialties.</a> <a data-t="826">Right now, it is only in critical care,</a> <a data-t="829">but we got asked to expand the tool to surgical services</a> <a data-t="833">to use for their operation or procedure notes.</a> <a data-t="837">So, we've been trialing an NLP tool for that instance</a> <a data-t="839">that has had pretty similar success</a> <a data-t="842">as our previous NLP tool.</a></p>

<p><a data-t="845">And we also like to run some additional analytics</a> <a data-t="847">to see if people are even correcting their notes.</a> <a data-t="851">You know, we create this tool for physicians,</a> <a data-t="854">but we actually want to know if it's being used,</a> <a data-t="856">if it's valuable, if it's just extra manual work for them,</a> <a data-t="860">or if it's going to require some additional education</a> <a data-t="863">or other processes.</a></p>

<p><a data-t="867">So, in short, our team utilized</a> <a data-t="869">currently available POSIT tools</a> <a data-t="871">to create a natural language processing tool</a> <a data-t="874">that successfully identified billing mistakes</a> <a data-t="876">in clinical notes.</a> <a data-t="877">And we also proved that this tool could be developed</a> <a data-t="880">with readily available and cost-effective tools</a> <a data-t="884">that can be reproducible among organizations</a> <a data-t="886">where access to LLM technology is limited.</a> <a data-t="890">So, now I feel like our clinicians</a> <a data-t="892">don't have to worry so much</a> <a data-t="894">about whether they're making accidental keystrokes</a> <a data-t="896">and whether or not that's going to affect their medical license</a> <a data-t="899">or causing their patients financial harm,</a> <a data-t="901">and they can go back to doing what they do best,</a> <a data-t="903">keeping us safe and healthy.</a> <a data-t="905">Thank you, everyone.</a> <a data-t="907">I'd like to open the floor for questions.</a></p>
