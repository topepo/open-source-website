<h3>What is deep learning?</h3>

<p><a data-t="628">OK, so now let's get into a little bit of what, because deep learning is probably the main thing people do with TensorFlow.</a> <a data-t="634">And so I think it's important to understand kind of what is it?</a> <a data-t="636">What is it useful for?</a> <a data-t="636">What is it not useful for?</a> <a data-t="639">Should we in the R community, how much should we care about it?</a> <a data-t="641">And kind of how does it work?</a></p>

<p><a data-t="650">So at a really, really high level, deep learning is taking some input, transforming it into some output via successive layers of representation.</a> <a data-t="658">And when I say this really abstract to say input and output, what I mean there is like observations or x data and predictions.</a> <a data-t="664">So it's x to y, but how do we do that?</a> <a data-t="668">We do that by taking original input, in this case, a grayscale image of a handwritten number 4.</a> <a data-t="675">That's our input, and our output is a prediction about what the digit is, the number 4.</a> <a data-t="679">We have to successfully transform that input until we get close to the output.</a> <a data-t="683">That's the basic mechanic of how deep learning models work.</a></p>

<p><a data-t="687">So I'm talking about layers.</a> <a data-t="688">I showed four layers there.</a> <a data-t="689">What exactly is a layer?</a> <a data-t="692">And I'm not going to talk about the mechanics of, maybe some of you have seen this before, of neurons and how they talk to other neurons and all that stuff.</a> <a data-t="698">You can really just think of a layer, and all you really need to consider a layer as, it's a function.</a> <a data-t="703">It's a data transformation function.</a> <a data-t="704">It does a geometric transformation of data, and it's parameterized by a set of weights and a bias, just like a linear equation.</a> <a data-t="710">So you can think of each one of these as just a successive transformation of data.</a> <a data-t="715">So that's all a deep learning model is, is taking some data and then chaining together a bunch of transformations of the data until we actually get an output, or in this case, a prediction.</a></p>

<p><a data-t="727">So when I say representations, what am I talking about?</a> <a data-t="729">We really want to take our data and transform it in a way that's closer to the domain of prediction that we're looking for.</a> <a data-t="738">So this is a really simple example.</a> <a data-t="739">I've got raw data.</a> <a data-t="740">I've got a bunch of points, and I want to determine whether a point, be able to predict whether a point is black or white.</a> <a data-t="747">If I change the coordinate system, all of a sudden, the problem becomes really simple, because everything with x greater than zero is a black point, and everything with x less than zero is a white point.</a></p>

<p><a data-t="759">So the idea is we want to transform our data so it's easier and easier to get close to the actual output that we're looking for, the prediction that we're looking for.</a> <a data-t="767">If you work with conventional machine learning models, you'll know this as sort of feature engineering, where we're trying to transform our data into a form that works better for the prediction task.</a> <a data-t="778">And in deep learning, the feature engineering is actually done in the layers.</a> <a data-t="780">The feature engineering is actually learned rather than hand coded.</a></p>

<p><a data-t="787">So back to this example of the handwritten digits, and I'll explain a little bit later how this works.</a> <a data-t="793">But basically, we're taking raw data, and these layers are trying to filter out irrelevant information, like how the image actually looks.</a> <a data-t="802">And it's trying to find other things, like where are edges, what are the angle of the edges, and get closer and closer to the domain of output or prediction.</a> <a data-t="810">So you can think of it as sort of an information distillation pipeline.</a></p>

<p><a data-t="815">And so this gives us an intuition about where the word deep comes from.</a> <a data-t="819">Why do we call this deep learning?</a> <a data-t="820">It's not because it gives us deeper insight or deeper models.</a> <a data-t="825">It's really just talking about the fact that there's multiple layers of representation.</a> <a data-t="829">A traditional machine learning model might have some feature engineering, and then one or two layers.</a> <a data-t="833">A deep learning model could have 20 layers, or 30 layers, or 100 layers.</a> <a data-t="838">And so maybe a more accurate term for deep learning might be like layered representation learning, or hierarchical representation learning.</a> <a data-t="844">The deep learning maybe implies more than it should, but it's really just talking about the stacking up of layers.</a></p>

<p><a data-t="851">And so what has this method achieved?</a> <a data-t="856">You can see from the list here, it's done really, really well on a lot of perceptual tasks, like speech recognition and image classification.</a> <a data-t="862">And then those, in turn, have been composed together to do things like trying to build autonomous driving systems, to build reinforcement learning systems that play games.</a> <a data-t="872">So it's achieved a lot, this fairly straightforward mechanism.</a></p>

<p><a data-t="875">And to give you an intuition for why it might be able to do that, if you think about a paper ball that's crumpled up, and you think about trying to write a linear equation for how to uncrumple the paper ball, it would be really, really difficult to do that.</a> <a data-t="892">But if you decompose it into a set of simpler geometric transformations, i.e. layers, then it's actually straightforward.</a> <a data-t="895">So a human being might just take that paper ball, and in 30 or so really simple operations, uncrumple it.</a> <a data-t="900">And that's really how deep learning models work, is that they learn these simple geometric transformations that compose together can do very, very complex transformations, and therefore solve very complex functions, or fit very complex functions.</a></p>

<h3>Frontiers of deep learning</h3>

<p><a data-t="1361">So computer vision is kind of the poster child for what deep learning has accomplished.</a> <a data-t="1365">There's a competition that ran from 2010 to 2017 called the ImageNet Challenge, where machine learning researchers tried to build models that would predict what images are.</a> <a data-t="1375">And the images were like dog, cat, table, chair.</a> <a data-t="1377">So when they started the competition in 2010, no entrants used deep learning, and they had 71.8% prediction accuracy.</a> <a data-t="1383">In 2012, the very first team that used deep learning entered the competition, and that team beat the rest of the field by 11%.</a> <a data-t="1392">And that basically ignited everybody to start looking at deep learning for computer vision.</a> <a data-t="1397">And by the time we got to 2017, in a period of seven years, the accuracy in computer vision went from 71.8% to 97.3%, which is phenomenal.</a> <a data-t="1405">That, more than anything, got people excited about deep learning.</a></p>

<p><a data-t="1410">And actually, the competition is not going to exist anymore after 2017, because this particular task is basically considered solved.</a> <a data-t="1418">So clearly, it works really well in computer vision and image classification, and you see a lot of applications for it there.</a></p>

<h3>Layer types in Keras</h3>

<p><a data-t="2345">So layers is kind of the core construct in Keras.</a> <a data-t="2348">And there's actually 65 different layers available, and you can create your own layers.</a> <a data-t="2350">This is just a sampling of some of them.</a> <a data-t="2352">I'm going to talk categorically about some of the types of layers that there are.</a></p>

<p><a data-t="2356">There's dense layers, which are kind of the staple of neural networks.</a> <a data-t="2359">These work the same way that traditional neural networks that were written about and used in the 70s work.</a> <a data-t="2367">And they're kind of a staple of every deep learning model.</a> <a data-t="2371">Usually, if you're using more sophisticated layers at the end of the model, you'll have these dense layers.</a> <a data-t="2373">And these dense layers are really just a bunch of weights and biases that are applied and cascade through other dense layers.</a></p>

<p><a data-t="2384">Convolutional layers are used for computer vision, most commonly.</a> <a data-t="2389">And they're basically trying to find spatial features.</a> <a data-t="2396">And they're trying to find spatial features that, when they learn, like, for example, what an edge is or what an eyeball is, that knowledge is actually transferable to other parts of the image.</a> <a data-t="2407">So if I see an eye in a certain quadrant of the image, and then I see later on an eye in another quadrant, I can still recognize that as an eye.</a> <a data-t="2412">So the reason it's convolutional is we're building up these filters that recognize patterns.</a> <a data-t="2416">And then we're convolving, sliding the filter across the entire image.</a></p>

<p><a data-t="2436">Recurrent layers are basically layers that have some memory or state.</a> <a data-t="2438">And so if you're dealing with sequence data, like time series data or text, where it matters not only what I'm seeing now, but what I've seen in the past, recurrent layers are able to maintain state.</a> <a data-t="2446">So they're oftentimes used in these sequence-oriented applications.</a> <a data-t="2449">And then embedding layers, for those of you who do natural language processing, is a way of vectorizing text.</a> <a data-t="2455">So one way to vectorize text would be to say, I've got an instance of the word cat, an instance of the word dog, and they're just classes.</a> <a data-t="2463">It's just cat versus dog.</a> <a data-t="2464">But another way would be, I've got a vector that represents semantic relationships between words.</a> <a data-t="2469">So in one of the dimensions of that, one of the axes of that vector, I've got a thing that says, oh, cat and dog are both animals.</a> <a data-t="2478">So embedding layers do this kind of vectorization.</a></p>

<p><a data-t="2488">And you can actually learn the embedding jointly with building the model, or you can load a pre-trained word embedding that we conventionally use, like word2vec.</a> <a data-t="2494">You can load one of those embeddings into an embedding layer.</a> <a data-t="2497">So there's just a few of the layer types, but again, there's 65 layers.</a> <a data-t="2500">And a lot of what doing deep learning is is trying to figure out what is the right composition of those layers and behavior of those layers to get a model that works well.</a></p>

<h3>Workflow tools and GPU training</h3>

<p><a data-t="2929">Kind of first and foremost is using GPUs.</a> <a data-t="2944">Some deep learning models work great on CPUs.</a> <a data-t="2951">Ones that use convolution or recurrent neural networks tend to be very slow on CPUs.</a> <a data-t="2953">So if you go to our TensorFlow for R website, there's a bunch of resources about different ways you can use GPUs.</a> <a data-t="2958">One is if you have a high-end NVIDIA graphics card, you can use a GPU on your local system.</a> <a data-t="2964">There's cloud services that let you do batch jobs that use GPUs.</a> <a data-t="2970">You can set up a cloud server with RStudio Server on Google Compute or Amazon or Azure.</a> <a data-t="2977">And there's even ways to actually have a virtual cloud Linux desktop that has a GPU.</a></p>

<p><a data-t="2991">We have another package called </a><a href="/software/tfruns/">tfruns</a><a data-t="2991">, stands for TensorFlow runs.</a> <a data-t="2993">And the idea here is that because a huge amount of experimentation is required with deep learning models, you're going to run a ton of training jobs.</a> <a data-t="3002">Instead of using the source function to run those jobs, you use a function called training run.</a> <a data-t="3006">And what training run does is it records everything about that job.</a> <a data-t="3009">The source code, the output, the performance of the model, the hyperparameters you use for the model.</a> <a data-t="3014">And then you basically have a data frame at the end for all the model runs that you've ever done.</a> <a data-t="3019">And then you can compute on that data frame to try to understand better what's working well and not working well.</a></p>

<p><a data-t="3026">It also gives you reports on each runs that summarize, again, the code, the model, the metrics, the performance of the model.</a> <a data-t="3031">So every single time you do a training run, one of these reports is generated.</a> <a data-t="3035">And then you can compare runs.</a> <a data-t="3036">So if you find, okay, I've examined my data frame.</a> <a data-t="3039">I found that these are my best two performing models.</a> <a data-t="3041">What was different about them?</a> <a data-t="3042">And you can see in this case, it's showing a source code diff saying I changed two things in the source code.</a> <a data-t="3046">And that accounts for that difference in performance.</a></p>

<p><a data-t="3052">And there's another piece called flags, which basically says if there's important aspects of my model, it's good to externalize those, get them out of my source code so that I can kind of pass these flags to my script as a parameter.</a> <a data-t="3061">So you can see I'm defining a set of flags here.</a> <a data-t="3068">And then I'm using the flags when I define a layer.</a> <a data-t="3071">And then when I train the model, I'm passing one of the flags.</a> <a data-t="3072">They're basically kind of command line parameters for your script that let you easily vary things.</a></p>

<p><a data-t="3079">And then that in turn enables this construct called a tuning run that says, okay, I've got two types of dropout.</a> <a data-t="3084">In this case, I'm gonna vary dropout from 0.2 to 0.4.</a> <a data-t="3092">And I want you to try every combination of those two hyperparameters and then tell me what the best model is.</a> <a data-t="3099">And if the number of combinations gets very, very large, you know, into the thousands, then you can actually sample from that and only run 50 training jobs rather than thousands of them to try to evaluate what the best model is.</a> <a data-t="3113">But these sort of tools end up being critical to getting deep learning models that perform well because you have to try so many different things.</a></p>

<p><a data-t="3125">The Cloud ML package is our interface to Google Cloud ML.</a> <a data-t="3128">And Google Cloud ML basically lets you train models, sort of, you sort of submit batch jobs to Cloud ML and then the model trains.</a> <a data-t="3132">And what's really cool about it is that they have GPUs and they actually have some really, really fast GPUs and even machines that have multiple fast GPUs available.</a> <a data-t="3142">And they also have a service for doing this hyperparameter tuning that I just talked about.</a> <a data-t="3146">So the idea is you want to work from the comfort of your laptop with smaller down-sampled data.</a> <a data-t="3149">And then when you want to actually do your training, that's going to take three hours.</a> <a data-t="3153">You submit it to Cloud ML and then it takes 20 minutes, hopefully, instead of three hours.</a></p>
