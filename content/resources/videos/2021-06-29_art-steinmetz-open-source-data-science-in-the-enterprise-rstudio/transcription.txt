<h3>Open source for enterprise-level data science</h3>

<p><a data-t="147">Open source is absolutely appropriate for enterprise-level data science.</a> <a data-t="151">It's a great way to boost productivity as it can empower all the interested</a> <a data-t="156">parties in the organization.</a> <a data-t="162">At my company, there were many people that I, as a senior manager,</a> <a data-t="162">didn't even know about who were working on developing projects using open source tools.</a> <a data-t="169">So you can let a thousand flowers bloom.</a> <a data-t="169">Now, importantly, such freedom can be done with</a> <a data-t="176">enterprise-level curation.</a> <a data-t="176">You can do this in a controlled way.</a> <a data-t="182">Now, of course, IT likes that</a> <a data-t="182">because you don't have a million people launching different things which are</a> <a data-t="188">unaudited or unvalidated, but you can still have the same creativity of the team.</a></p>

<p><a data-t="196">Tools like </a><a href="/software/rstudio/">RStudio</a><a data-t="196"> really help provide this full production life cycle within a curated</a> <a data-t="202">environment that is enterprise-grade.</a> <a data-t="210">There are so many people contributing to that community.</a> <a data-t="210">There are so many tremendous packages out there.</a> <a data-t="215">You can be comfortable that if you're using a</a> <a data-t="215">commercial software that has been audited and vetted and has gone through a process before</a> <a data-t="221">it was released into the wild, that is greater than what might happen in the open source community.</a> <a data-t="226">But at the same time, in the open source community, the crowdsourcing and the enormous</a> <a data-t="232">number of people that are contributing means that the quantity of tremendous enterprise-level</a> <a data-t="238">packages or tools is much, much larger.</a> <a data-t="245">There are tools that can help companies</a> <a data-t="245">curate, audit, and validate all of those tools that are out there.</a> <a data-t="250">Packages like RStudio</a> <a data-t="252">let you do that and have a walled garden where the appropriateness and quality of all the packages</a> <a data-t="260">is monitored within the organization.</a></p>

<h3>Building support for open source in an organization</h3>

<p><a data-t="268">My personal experience, building support for open source software in an organization means</a> <a data-t="274">starting small.</a> <a data-t="274">What is very powerful, I think, are small, gee-whiz demonstration projects.</a> <a data-t="284">Take this in small bites, projects with quick turnaround that demonstrate value.</a> <a data-t="288">Then you're</a> <a data-t="288">going to have people looking over your shoulder saying, how can I do that?</a> <a data-t="294">So you don't have to</a> <a data-t="294">start out with a large investment.</a> <a data-t="300">But in my experience, the power of these tools is such that</a> <a data-t="300">you very quickly have people saying they want in.</a> <a data-t="307">They want to try to do some of this stuff</a> <a data-t="307">themselves.</a></p>

<h3>Do executives need to know how to code?</h3>

<p><a data-t="356">Certainly not.</a> <a data-t="356">Executives don't need to know how to code to manage a data science team.</a> <a data-t="363">I was a coder</a> <a data-t="364">and it did help me be a champion with some credibility in the organization.</a> <a data-t="370">But it's</a> <a data-t="370">absolutely not necessary.</a> <a data-t="370">And you can't really expect that or make that a requirement.</a> <a data-t="376">You know, the higher you go in any organization, the more you are going to be responsible for</a> <a data-t="381">supervising people who are experts in their domain and know more about that than you do.</a></p>
