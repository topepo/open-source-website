<h3>RStudio Cloud overview</h3>

<p><a data-t="151">So RStudio Cloud.</a> <a data-t="153">Briefly, RStudio Cloud is the RStudio IDE, available as a service that you can access from anywhere you have an internet connection and a modern web browser.</a> <a data-t="159">It doesn't require any IT support or local installation, and you can sign up for an account and you can use it today.</a> <a data-t="165">We created RStudio Cloud to make it easy to do, share, teach, and learn data science using R,</a> <a data-t="168">and in support of that mission, we've built a ton of extra features to make it easier to organize work, teach classes, learn R, and share analyses.</a></p>

<p><a data-t="175">So first, a little bit of basic terminology.</a> <a data-t="178">In RStudio Cloud, the fundamental unit of work is a project.</a> <a data-t="182">It encapsulates your R code packages and data files and provides isolation from other analyses.</a> <a data-t="188">So this is analogous to projects in a local installation of the RStudio IDE.</a> <a data-t="192">A space is a place for a group of people to organize and collaborate on work.</a> <a data-t="196">In addition to a personal workspace that everyone gets automatically, you can also create spaces with roles and permissions to control project sharing and visibility for space members.</a> <a data-t="210">So not coincidentally, these spaces work well for courses, sections, and workshops.</a></p>

<p><a data-t="215">Spaces have four different possible roles.</a> <a data-t="218">The creator of a space is automatically assigned the admin role.</a> <a data-t="221">This role is recommended for instructors.</a> <a data-t="223">Admins have total control over the space and can see and edit all projects in it.</a> <a data-t="228">Next we have moderators.</a> <a data-t="229">This role works well for TAs if you have them.</a> <a data-t="231">Like an admin, a moderator can see and edit all projects in a space regardless of visibility.</a> <a data-t="237">So more on that in just a minute.</a> <a data-t="239">The contributor role is a good choice for students.</a> <a data-t="241">Contributors can create new projects in the space.</a> <a data-t="243">And viewers can see the space, but they can't create new projects in it.</a> <a data-t="249">So this role could work well for someone who is auditing the class, for example.</a></p>

<p><a data-t="253">Whether or not a project can be seen by someone else is controlled by its visibility.</a> <a data-t="257">A project can be public, that is, shared with everyone, or private.</a> <a data-t="260">Private projects in a space can be viewed by the author and also the admins and moderators of the space.</a> <a data-t="265">Private projects are indicated with a lock icon like that.</a></p>

<h3>Additional features and best practices</h3>

<p><a data-t="963">Okay, so the workflow that I just described makes good use of copy projects via assignments.</a> <a data-t="968">However, you might want your students to create new projects themselves.</a> <a data-t="971">If they're pulling code from an existing Git repository, for example, the new project menu has an option for that.</a> <a data-t="976">Git is really great.</a> <a data-t="977">This option will bring up a dialog to enter a repository URL, and RStudio Cloud will create a new project for it.</a> <a data-t="983">Once the project is created, you may have to log in, but once you do, you'll have all of your source from Git right there.</a></p>

<p><a data-t="988">One final helpful thing, you can set up your space such that all new projects created in it will include a default set of packages and files.</a> <a data-t="995">This is done by selecting a base project in the space settings.</a> <a data-t="999">You can actually take advantage of this feature when creating assignments for your class.</a> <a data-t="1002">If your assignments all need the same set of packages, say you want the </a><a href="/software/tidyverse/">tidyverse</a><a data-t="1002"> in all of them,</a> <a data-t="1006">you can create a project in your space with those packages installed and set it as the base project.</a> <a data-t="1011">All new projects will have those packages installed automatically.</a> <a data-t="1015">Note that base projects are not retroactive and won't apply to previously created projects.</a> <a data-t="1020">You can learn more about this in the guide.</a></p>

<p><a data-t="1021">I'm sorry to tease a tantalizing bit of setup like this, but I'm really short on time.</a> <a data-t="1025">RStudio Cloud currently has a few limits to be aware of.</a> <a data-t="1026">RStudio Cloud is still an alpha product.</a> <a data-t="1027">We do not currently support simultaneous collaborative editing.</a> <a data-t="1031">If your students need to collaborate on work in this way, we suggest using a Git-based repository for code.</a> <a data-t="1035">Using source control is not a terrible idea regardless.</a> <a data-t="1038">Currently projects are limited to one gig of RAM, and by default you can create one additional private space with up to 10 members and 25 projects.</a> <a data-t="1045">If you need extra spaces or extra capacity, please reach out to us, particularly if you want to use RStudio Cloud for teaching, and we will do our best to accommodate you.</a></p>

<p><a data-t="1053">Some best practices in summary.</a> <a data-t="1055">First, use RStudio Cloud for teaching.</a> <a data-t="1058">I am biased, but I think it's really great.</a> <a data-t="1062">Second, use spaces to get the helpful features that we've just talked about, like assignments, which takes me to third.</a> <a data-t="1069">Make use of assignments and spaces.</a> <a data-t="1071">This will make sure that your students do not inadvertently lose work because they forgot to create a permanent copy.</a> <a data-t="1076">Also because only public projects can be assignments, this ensures that your students will have access to the project you intended to share with them.</a> <a data-t="1083">Third, take advantage of space rules and permissions to make sure students can only see what they should.</a> <a data-t="1089">And finally, test things out ahead of time.</a> <a data-t="1091">Make sure you're not going to hit RAM limits and that you have the capacity that you need.</a></p>
