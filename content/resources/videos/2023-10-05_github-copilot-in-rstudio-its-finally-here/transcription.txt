<h3>The generative loop: context, intent, output</h3>

<p><a data-t="745">So, Copilot is another generative AI tool, and it can predict output text, or in this case, more specifically, code.</a> <a data-t="762">But importantly, generative AI doesn't understand anything.</a> <a data-t="765">It's just a prediction engine.</a> <a data-t="767">It's just trying to get to that next word and continue finishing out a sentence or a script.</a> <a data-t="774">So, in this case, to get the most out of generative loop, we can think about it in three different ways.</a> <a data-t="779">We have the context, which is what prompts or what code or what comments have been provided.</a> <a data-t="784">Basically, what all is inside the script that it can use.</a> <a data-t="788">The intent, which is what I actually think in my head of what I want to do,</a> <a data-t="792">but I have to codify that or write it into my scripts to get the most out of it.</a> <a data-t="797">And then I have the output, which is actually what Copilot will return in this example.</a></p>

<p><a data-t="802">So, these three components in my mind are the generative loop.</a> <a data-t="805">I have the intent initially about what I'm trying to do,</a> <a data-t="808">the context about what prompts I then provide to Copilot,</a> <a data-t="812">and the output of, again, what is actually returning.</a> <a data-t="815">So, to get the most out of this, I can try and supply better context,</a> <a data-t="819">which means it gets closer to my intent of what I want to do</a> <a data-t="823">and leads to a better output or a better prediction to help solve my problem.</a></p>

<p><a data-t="828">So, this is really what I meant by, like, scoping the problem and a prompt.</a> <a data-t="833">The prompt is really anything in your script, like code snippets, names of files,</a> <a data-t="838">names of data frames, little comments throughout,</a> <a data-t="842">and other things that have been loaded in the environment.</a> <a data-t="846">To simplify this even further, I have my little buzzword for the day, which is S2C,</a> <a data-t="852">or making your problems simple, specific, and use comments throughout your document.</a> <a data-t="857">Using this approach, you can generally be productive with tools like Copilot</a> <a data-t="862">and, you know, allow it to use, be closer to what you're actually intending to do</a> <a data-t="868">and get better outputs.</a></p>
