<h3>Structuring research as an R package</h3>

<p><a data-t="115">So the particular solution that our team</a> <a data-t="115">has ended up with is to place the project within an R package.</a> <a data-t="119">And</a> <a data-t="119">I'll make that concrete.</a> <a data-t="123">So senior members of the team, grad students</a> <a data-t="123">and faculty, set up a simple package.</a> <a href="/software/devtools/">DevTools</a><a data-t="128"> create just like you</a> <a data-t="128">would normally do.</a> <a data-t="133">And then we set up all of the research tools as if</a> <a data-t="133">it's part of that package.</a> <a data-t="136">And that includes the project management,</a> <a data-t="136">which we'll go into.</a></p>

<p><a data-t="140">This means that when your junior members, those</a> <a data-t="140">undergraduates or new grad students set up and want to start using</a> <a data-t="143">this, all they have to do is run DevTools load all and they get</a> <a data-t="146">everything into their environment.</a> <a data-t="151">This allows us to set up things that</a> <a data-t="151">require no new tools.</a> <a data-t="154">And it's going to look very familiar.</a> <a data-t="154">So if you</a> <a data-t="154">see our good old friend </a><a href="/software/rstudio/">RStudio</a><a data-t="154">'s file explorer, the package stuff is</a> <a data-t="160">the normal package stuff.</a> <a data-t="163">And all we're doing really is adding a few</a> <a data-t="163">more folders and a few more functions that let us do the research</a> <a data-t="166">stuff.</a> <a data-t="171">Stuff being the technical word, of course.</a></p>

<p><a data-t="171">So what we're really</a> <a data-t="171">doing, then, is using the R package to handle all of the project</a> <a data-t="174">management part.</a> <a data-t="178">And we create a series of functions.</a> <a data-t="178">So, for example,</a> <a data-t="178">if you want to create a new analysis, there's an analysis function.</a> <a data-t="182">Or</a> <a data-t="182">if you want to set up a pull request on GitHub, we automate that.</a> <a data-t="188">Things</a> <a data-t="188">like the dataverse, which in the social sciences is a big place to share</a> <a data-t="191">data that you've created.</a> <a data-t="196">Or because, again, these are inexperienced</a> <a data-t="196">people that we're trying to teach how to do research, we want to be able</a> <a data-t="199">to peer review their work internally.</a> <a data-t="202">And so we set up a bunch of</a> <a data-t="202">functions that handle that.</a> <a data-t="206">These can be tested and documented so that</a> <a data-t="206">the students don't have to learn anything new.</a> <a data-t="209">They can just use the</a> <a data-t="209">tools that they're already seeing in their classes.</a></p>
