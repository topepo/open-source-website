<h3>blogdown internals and helper functions</h3>

<p><a data-t="2481">All right.</a> <a data-t="2481">So I also want to mention some of the internals of blogdown.</a> <a data-t="2481">So basically, as I said, blogdown is pretty much the combination of Hugo and R Markdown.</a> <a data-t="2486">But for Hugo, it can only process Markdown files using another Markdown engine named Black Friday.</a> <a data-t="2505">You can know more about Black Friday from the Hugo website.</a> <a data-t="2505">So Hugo does not use Pandoc.</a> <a data-t="2505">So that is pretty much the whole motivation of blogdown.</a> <a data-t="2506">We want to use Pandoc's Markdown syntax instead of Black Friday's.</a> <a data-t="2514">So the way that blogdown works is that we will compile your R Markdown files to HTML files and pass the results to Hugo, and Hugo will pick the output from R Markdown and Pandoc and build the website.</a> <a data-t="2534">So it will basically bypass the Markdown engine in Hugo.</a></p>

<p><a data-t="2534">And there are a few helper functions in this package.</a> <a data-t="2558">So as I have already mentioned, the new site function, which you can create a new website, potentially using a different theme other than the default theme.</a> <a data-t="2559">So if you have not installed Hugo before, you can use the install Hugo function.</a> <a data-t="2568">And if you want to install an additional theme, you can use the install theme function.</a> <a data-t="2569">And if you want to automatically rebuild and reload your website, you can use the serve site function or equivalently using the RStudio add-in serve site.</a></p>

<p><a data-t="2585">And there's also a new post function, which I think is also very handy.</a> <a data-t="2585">The new post function can create a new blog post for you.</a> <a data-t="2585">And let me show you a quick example of the new post function.</a> <a data-t="2611">So basically, you can either use that function or the add-in named new post.</a> <a data-t="2611">So if you click this new post add-in, you can just fill out all the information related to your post.</a> <a data-t="2611">For example, you can say hello, blogdown, and you can choose the date of that post.</a> <a data-t="2636">And you can decide the subdirectory to put that post.</a> <a data-t="2636">And you can change the file name if you want.</a> <a data-t="2636">And there are many other things for you to tweak.</a> <a data-t="2650">And you can interactively select categories or tags.</a> <a data-t="2650">And of course, you can add new categories by just typing here.</a> <a data-t="2650">And then you can add a new category there.</a> <a data-t="2669">And you can also select tags there.</a> <a data-t="2669">And then you can decide whether you want your post to be markdown or rmarkdown file.</a> <a data-t="2680">Then you can click done.</a> <a data-t="2680">And blogdown will call the function new post to create a new post and open that automatically in RStudio.</a> <a data-t="2680">So now you can just start writing the content of your blog post.</a> <a data-t="2680">And I think that one is very useful.</a></p>
