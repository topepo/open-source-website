<h3>Quarto projects overview</h3>

<p><a data-t="260">This is where Quarto comes in handy, it's the next generation of R Markdown, I won't go too deep into what Quarto is, I think everybody is aware at this point.</a> <a data-t="268">But with Quarto projects, it allows you to create various project types from a single source.</a> <a data-t="273">And when I talk about project types, I'm talking about an organization of files.</a> <a data-t="276">So we're familiar with reports, and manuscripts, and presentations.</a> <a data-t="280">This could also be a Quarto website, or Quarto blog.</a> <a data-t="283">And then when I talk about project formats, I'm talking about how it's going to be output in the end, an HTML, a PDF, or a .docx.</a></p>

<p><a data-t="287">So a general setup with Quarto, as with any project in </a><a href="/software/rstudio/">RStudio</a><a data-t="287">, you need a primary folder.</a> <a data-t="293">I'll go over my folder structure in just a minute.</a> <a data-t="295">These Quarto projects are the directories, they're the central location for all of your folders, your images, your scripts, and then any output that you're going to generate.</a> <a data-t="304">This is a key concept for this presentation.</a> <a data-t="306">We're going to do three different projects today in this talk.</a> <a data-t="310">We're going to generate a report, we're going to generate a manuscript, and we're going to generate a presentation with templates, all from the same source code.</a></p>

<p><a data-t="318">So Quarto projects, they have this shared YAML for metadata.</a> <a data-t="322">Each of the project types typically have a different setup for their YAML.</a> <a data-t="325">I've got examples in here.</a> <a data-t="327">But when you're in RStudio, all you have to do is click the create project button.</a> <a data-t="331">It gives you an option to include a directory name and change where your working directory is going to be at.</a> <a data-t="335">And then finally, in project type, you can start setting these things up.</a> <a data-t="338">So Quarto website, blog, book, manuscript, they're all listed there.</a> <a data-t="341">It's very easy to click them.</a> <a data-t="342">There's also terminal commands to set these up.</a> <a data-t="345">It's pretty straightforward.</a></p>

<p><a data-t="346">Whenever you start a new Quarto document, it's going to give you a template file, usually index.qmd.</a> <a data-t="352">It comes with some markdown code, and there's an example below.</a> <a data-t="355">It also includes a very basic YAML header.</a> <a data-t="358">The title is the only thing in this one, and then an example R code chunk.</a> <a data-t="363">You can change the language if you want to.</a> <a data-t="364">If you don't want to use R and you want to use Python, that's supported.</a> <a data-t="367">So is Julia and a couple other languages.</a> <a data-t="369">Toolbar gives you very easy access to editing tools without knowing markdown syntax.</a> <a data-t="375">And this is just the basic layout.</a> <a data-t="377">So you can continue to add chunks and markdown content as you need.</a></p>

<p><a data-t="381">The YAML header is very, very important.</a> <a data-t="383">This is a versatile human-readable data serialization language.</a> <a data-t="388">This is where you configure how your document is going to populate.</a> <a data-t="392">So you'll see throughout my examples today, a lot of front matter.</a> <a data-t="394">The title, the subtitle, date, author, et cetera.</a> <a data-t="397">The important sections are format, which designates the output format.</a> <a data-t="401">This is also where I designate all of my templates.</a> <a data-t="404">So if you're interested in, oh, my author usually generates manuscripts for a very specific journal.</a> <a data-t="410">I use the Public Library of Science in my example.</a> <a data-t="412">You can download that template, and then it will automatically output to that when you render the PDF.</a> <a data-t="418">There's a bunch of YAML options.</a> <a data-t="419">I included a link to go explore more.</a></p>

<h3>Shortcodes for modularity</h3>

<p><a data-t="488">Now, importantly, neither of these things are coded directly within my report.</a> <a data-t="494">You'll see that I use what's called shortcodes.</a> <a data-t="497">So in a nutshell, the goal of this is to maintain quality and consistency through modularity.</a> <a data-t="502">Quarto shortcodes are special markdown directives that generate our content from another file.</a> <a data-t="507">So if you look back at my file directory, all of my actual QMD files are in this analysis folder.</a> <a data-t="512">I'm not doing anything within the Quarto markdown.</a> <a data-t="514">It's not hosted locally.</a> <a data-t="516">It's just pulling from this QMD.</a> <a data-t="518">All the data is in one separate file.</a> <a data-t="523">It's not being generated and brought into R each time I want to do a report.</a></p>

<p><a data-t="528">And then down here, you can see there's two types of shortcodes I use.</a> <a data-t="532">The first one is embed, which directly embeds that Quarto markdown file or Jupyter Notebook into my report, my presentation, my manuscript.</a> <a data-t="539">It's just going to run the code that's there.</a> <a data-t="542">And if you're not careful, it will copy in everything that's in that QMD.</a> <a data-t="545">So everything that's generated will be put into your document.</a> <a data-t="549">You can specify what you want, though, using hashtags.</a> <a data-t="552">And then include is a direct copy-paste.</a> <a data-t="555">So it's not embedding the QMD.</a> <a data-t="557">It is just grabbing the code and then running it in your main file.</a></p>

<p><a data-t="561">The example of the include shortcode is on line one.</a> <a data-t="566">I didn't specify any particular figure, so it's going to throw everything that's in my data processing QMD, everything that's being output into my documents.</a> <a data-t="574">The second one is an example of embed.</a> <a data-t="576">And you'll see that there's a hashtag.</a> <a data-t="579">That's where I specified, I only want figure one.</a> <a data-t="581">There's a bunch of other figures in there that I was doing as part of EDA, but this saves time and processing and makes sure you only grab what you want.</a></p>

<h3>Advanced features and conclusion</h3>

<p><a data-t="879">So to wrap up, there's a bunch of advanced features that I didn't want to spend time going into because I could spend a whole lecture on them.</a> <a data-t="887">But the first one is inline programming.</a> <a data-t="889">So again, instead of copy-pasting values from tables or from figures, which can change, you know, I go back and update the analysis code, suddenly the mean has changed by two units.</a> <a data-t="900">Just use inline programming.</a> <a data-t="901">So that can make sure that no matter what format you're outputting to, that value is being pulled from my actual analysis.</a> <a data-t="907">It's not being copy-pasted from somebody else.</a></p>

<p><a data-t="910">Second one is everything's dynamically updating based on the quarto.yaml file.</a> <a data-t="914">So under execute, there is a freeze function and typically it's set to true.</a> <a data-t="918">If you set it to auto, it will automatically re-render files if changes are detected.</a> <a data-t="925">So let's say you want to render a report and everything's fine.</a> <a data-t="931">You don't want to keep re-rendering that processing file because it takes time and computational power, especially if nothing's changing.</a> <a data-t="937">Only change if somebody has gone in there and edited something.</a></p>
