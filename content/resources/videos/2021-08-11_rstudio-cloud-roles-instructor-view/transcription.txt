<p><a data-t="0">From within an </a><a href="/software/rstudio/">RStudio</a><a data-t="0"> Cloud workspace, there are four possible roles that an individual</a> <a data-t="6">can occupy.</a> <a data-t="8">As the creator of the workspace, we are given the default status of administrator, and we</a> <a data-t="12">cannot change that status.</a></p>

<p><a data-t="16">Administrators have full privileges within a workspace.</a> <a data-t="19">Administrators can manage the workspace and all of the projects, including their own,</a> <a data-t="24">and also view all of the projects within a workspace, including private projects.</a></p>

<p><a data-t="30">Moderators have all of the same privileges as an administrator, except that moderators</a> <a data-t="34">cannot add or remove learners from the workspace.</a> <a data-t="38">This means that moderators can manage all of the projects as well as view all of the</a> <a data-t="42">projects within a workspace.</a></p>

<p><a data-t="45">Contributors can create and manage their own projects, and they can also view any projects</a> <a data-t="49">that have been shared with them.</a> <a data-t="51">As an administrator or moderator, we can also control whether or not contributors can see</a> <a data-t="56">the names of other members in the workspace, make their projects visible to all members,</a> <a data-t="61">and change project resources.</a></p>

<p><a data-t="64">Viewers have the most restrictive permissions and can only view projects that have been</a> <a data-t="68">shared with the workspace or with them as an individual.</a> <a data-t="72">As an administrator or moderator, we can also control whether or not a viewer can see</a> <a data-t="76">the names of other members in the workspace.</a></p>
