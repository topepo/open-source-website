<h3>What is xaringan?</h3>

<p><a data-t="34">So wait, what?</a> <a data-t="35">What is xaringan?</a> <a data-t="37">First of all, it's an R package that you can install from Crayon.</a> <a data-t="40">Second, it's a way to write an R markdown with some extra syntax, and then have your</a> <a data-t="44">R markdown, R code, and outputs, plots, widgets, all rendered into slides.</a> <a data-t="50">The slides are web pages in HTML, and the great thing about that is that you can use</a> <a data-t="55">CSS to layout and style your slides.</a> <a data-t="57">But the downside is that you have to use CSS to layout and style your slides.</a> <a data-t="62">So CSS is a little tricky to wrap your mind around, but I think that xaringan makes it</a> <a data-t="66">a little more fun.</a></p>

<h3>CSS and the web basics</h3>

<p><a data-t="68">So let's first learn a little bit about CSS and the web.</a> <a data-t="72">CSS styles are rules about how elements on the screen should appear.</a> <a data-t="76">The most common way to link a rule to an element is to use a class.</a> <a data-t="80">In CSS, these start with a dot.</a> <a data-t="83">Xaringan has an extra little syntax that makes it easier to reference these CSS classes.</a> <a data-t="87">You write the class name, then a pair of brackets, and then you put text or whatever</a> <a data-t="91">you want inside the brackets.</a> <a data-t="94">Here we use the complement class for the whole sentence, and the RStudioBlue class for just</a> <a data-t="99">the word </a><a href="/software/rstudio/">RStudio</a><a data-t="99">.</a></p>

<p><a data-t="101">Inside a rule, there are CSS properties where you can set a property equal to a value, like</a> <a data-t="107">font family to monospace, or color to tomato.</a> <a data-t="111">Notice that because RStudioBlue has its own color, and it's closer to the word RStudio,</a> <a data-t="116">it wraps around RStudio.</a> <a data-t="118">It doesn't turn tomato.</a></p>

<p><a data-t="121">The second thing to know about the web is that everything is in a box.</a> <a data-t="125">The block boxes take up a full line of horizontal space.</a> <a data-t="129">In Markdown, their brackets span more than one line.</a> <a data-t="132">Inline boxes fit within block boxes and don't break the flow.</a> <a data-t="136">In Markdown, their brackets are on the same line, like with RStudioBlue.</a></p>
