<p><a data-t="0">Hi, I'm John Helveston, and thanks for tuning in to my lightning talk on how to make R-powered</a> <a data-t="11">surveys with formr.</a> <a data-t="13">If you're someone who has ever had to make a survey, chances are you've probably used</a> <a data-t="17">something like SurveyMonkey or Google Forms, and these platforms work great for simple</a> <a data-t="22">surveys.</a> <a data-t="23">But what if you needed to make something that was a little more complex than what these</a> <a data-t="27">platforms offer?</a></p>

<h3>What formr can do</h3>

<p><a data-t="52">So what can formr do?</a> <a data-t="54">Well, I kind of think of it like SurveyMonkey plus RMarkdown.</a> <a data-t="58">You can use it to create simple surveys, but you can also use R code to generate a much</a> <a data-t="62">richer set of features.</a> <a data-t="65">For example, you could create interactive charts that survey respondents could then</a> <a data-t="69">experience while taking the survey.</a></p>

<p><a data-t="73">Or you could use R code to generate dynamic feedback.</a> <a data-t="77">For example, images like this were generated with ggplot code for each respondent in a</a> <a data-t="82">longitudinal study, showing them their risk preferences for different activities based</a> <a data-t="87">on their answers to previous questions.</a></p>

<h3>How formr works</h3>

<p><a data-t="90">So at this point, you might be wondering how all this works.</a> <a data-t="92">So let's dig into some of the details.</a> <a data-t="95">Surveys in formr are built using open source components.</a> <a data-t="99">The text and code for each survey are input using Google Sheets.</a> <a data-t="103">And once the survey is live, R code is computed using OpenCPU.</a> <a data-t="107">This makes it really easy to make your work as transparent as you'd like.</a></p>

<p><a data-t="112">So here's an example Google Sheet.</a> <a data-t="114">Each row is an item that will be displayed on the survey, and each column controls what</a> <a data-t="118">will be shown.</a> <a data-t="120">The label column is where you insert markdown or code chunks that you want to display in</a> <a data-t="125">the survey.</a> <a data-t="127">The type column is where you set the type of each question to show.</a> <a data-t="131">You can choose from a long list of options like multiple choice questions, text entry,</a> <a data-t="136">or simply just display some text or images.</a></p>

<p><a data-t="140">Depending on the type of question, you may also have some options for respondents that</a> <a data-t="144">you want them to choose from, like in a multiple choice question.</a> <a data-t="148">And you can also customize the look and feel with some styling options.</a> <a data-t="152">It's kind of analogous to an RMarkdown document, where the style and question type columns</a> <a data-t="157">are sort of like the YAML, and the label and choice columns contain the content that you</a> <a data-t="161">want to display.</a></p>

<p><a data-t="164">Once you've got your spreadsheet ready, you can then log in to formr.org, input the link</a> <a data-t="168">to your Google Sheet, and formr will convert it into a survey.</a> <a data-t="173">The example I just showed generates a survey with some multiple choice questions about</a> <a data-t="177">apples that look like this in a web browser.</a> <a data-t="181">Once your survey is live, you can then use the associated formr package to go import</a> <a data-t="186">results in a nicely formatted data frame directly into R.</a></p>

<p><a data-t="216">So, for example, if I wanted to know how people felt about the price or freshness of different</a> <a data-t="220">apples, I could ask them to choose from profiles like this, and then in each choice question,</a> <a data-t="226">I'll randomly vary the values that I'm showing them.</a> <a data-t="230">Because I'm working in R, I can predefine all those profiles in a data frame.</a> <a data-t="234">So, in this example, the three profiles shown are generated from the first three rows of</a> <a data-t="239">this data frame.</a></p>
