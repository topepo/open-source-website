<h3>Benefits of RStudio Pro on SageMaker</h3>

<p><a data-t="31">So let's jump into this. First off, we're really excited.</a> <a data-t="35">This is a true kind of partnership</a> <a data-t="36">that we have with the Amazon team and the SageMaker team.</a> <a data-t="40">And we're really excited to kind of be able to offer</a> <a data-t="42">our commercial offering of RStudio</a> <a data-t="45">along with Amazon and SageMaker's platform.</a> <a data-t="48">So enabling our customers to do this really powerful machine</a> <a data-t="51">learning and enhance their workloads</a> <a data-t="53">beyond what's available on their local laptops.</a></p>

<p><a data-t="58">Ultimately, we can break down kind of the benefits of SageMaker</a> <a data-t="62">with RStudio Pro into four topics.</a> <a data-t="64">And we'll kind of dive into these a bit deeper.</a></p>

<p><a data-t="67">So number one, RStudio Pro on SageMaker is a managed service.</a> <a data-t="70">So this massively simplifies the administration.</a> <a data-t="74">And you don't have to have a robust IT team</a> <a data-t="77">to manage over everything.</a> <a data-t="79">It's managed through the SageMaker platform as a hosted service.</a></p>

<p><a data-t="83">Additionally, especially compared to a local laptop,</a> <a data-t="86">SageMaker with RStudio Pro provides this flexible compute.</a> <a data-t="90">And you don't have to figure out how to spin up new servers.</a> <a data-t="93">You can actually just use a dropdown within RStudio</a> <a data-t="95">to select different sizing of your environment</a> <a data-t="97">for both RAM and CPU, et cetera.</a></p>

<p><a data-t="101">Additionally, the SageMaker platform</a> <a data-t="102">comes with strict security and compliance benefits.</a> <a data-t="105">So things like SOC2, FedRAMP, HIPAA,</a> <a data-t="108">and additional other kind of security confirmations and compliance.</a> <a data-t="112">It's really useful and needed for our customers</a> <a data-t="115">in banking or pharma or health care, for example.</a></p>

<p><a data-t="119">And then lastly, SageMaker also allows customers</a> <a data-t="121">to use their existing cloud budgets.</a> <a data-t="123">So for a lot of customers, they actually have budgets</a> <a data-t="126">that are set aside for purchasing compute through cloud like AWS.</a> <a data-t="130">And you can use some of that budget to purchase licenses</a> <a data-t="132">for RStudio Pro that is then used inside SageMaker.</a> <a data-t="136">Again, alleviating some of the struggles with purchasing</a> <a data-t="140">or figuring out how to get this budget approved.</a></p>

<p><a data-t="143">Overall, you can kind of have this nice little table here</a> <a data-t="146">that compares pure SageMaker to RStudio Pro</a> <a data-t="149">in addition to the core SageMaker platform.</a></p>

<p><a data-t="152">Both SageMaker and RStudio Pro on SageMaker</a> <a data-t="154">are flexible computing environments.</a> <a data-t="157">So actually backed by these ephemeral EC2 instances</a> <a data-t="160">and Docker images that are loaded into it.</a> <a data-t="163">Both have a dedicated home directory for persistent files</a> <a data-t="166">or other things like that.</a> <a data-t="168">And then the SageMaker platform is a little bit different.</a> <a data-t="171">It's dedicated for persistent files, R and Python package installs,</a> <a data-t="174">configuration settings, different things</a> <a data-t="176">that you want your user kind of experience to be the same.</a></p>

<p><a data-t="180">The main difference at this point is that SageMaker</a> <a data-t="183">is really like a hosted version of Jupyter or JupyterLab.</a> <a data-t="186">And then RStudio adds the additional RStudio IDE experience</a> <a data-t="191">on top of SageMaker's compute.</a></p>

<p><a data-t="194">For SageMaker with JupyterLab, you're primarily using Python.</a> <a data-t="197">So you can use packages like Boto3</a> <a data-t="199">to interact with AWS and different services.</a> <a data-t="202">And then in R, there's the pause R package</a> <a data-t="205">that allows you to do very similar things</a> <a data-t="207">and interact with all sorts of AWS services,</a> <a data-t="210">in addition to SageMaker.</a></p>

<p><a data-t="213">And then lastly, both SageMaker and RStudio Pro inside SageMaker</a> <a data-t="216">allow you to leverage specific SageMaker machine</a> <a data-t="219">learning capabilities.</a> <a data-t="221">Both are really using the SageMaker Python SDK</a> <a data-t="224">or software developer kit.</a> <a data-t="226">In SageMaker, you primarily call it natively through Python.</a> <a data-t="230">And then in RStudio, you actually wrap some of the commands in R</a> <a data-t="233">using the same SageMaker Python SDK,</a> <a data-t="236">but wrapping it with the </a><a href="/software/reticulate/">reticulate</a><a data-t="236"> R package,</a> <a data-t="239">allowing you to call Python directly from R.</a></p>

<p><a data-t="242">So really, the environment is the same for both of them,</a> <a data-t="245">but providing kind of a different user interface</a> <a data-t="247">with the same packages and persistent files behind the scenes.</a></p>

<p><a data-t="278">Additionally, because SageMaker is a hosted environment</a> <a data-t="281">and the AWS team is actually managing the infrastructure for you,</a> <a data-t="286">users who lack IT support</a> <a data-t="288">or don't necessarily want to commit the resources</a> <a data-t="291">needed to maintain a fully custom environment</a> <a data-t="294">can go through this with kind of a ready to go,</a> <a data-t="296">spin it up kind of very quickly environment.</a></p>

<h3>Additional benefits and trade-offs</h3>

<p><a data-t="330">As far as some of the additional benefits of RStudio Pro on SageMaker,</a> <a data-t="335">since again, RStudio is a managed service,</a> <a data-t="338">customers don't have to really worry about managing installations</a> <a data-t="341">of R and Python or even installing a Posit Workbench</a> <a data-t="344">slash RStudio Pro itself in the environment.</a> <a data-t="347">It's already kind of ready to go.</a></p>

<p><a data-t="349">Additionally, our software product called Launcher</a> <a data-t="352">is working behind the scenes with SageMaker internally</a> <a data-t="355">that allows you to create these individually sized</a> <a data-t="359">and isolated sessions.</a> <a data-t="360">So you can have a gigantic session, run a model,</a> <a data-t="363">leave it running and then go to a much smaller session</a> <a data-t="366">with say like eight gigs of RAM</a> <a data-t="368">and then do some computation there or exploratory data analysis</a> <a data-t="371">while your large model training is occurring.</a></p>

<p><a data-t="375">Additionally, RStudio Pro on SageMaker</a> <a data-t="377">has direct integration with identity access management through AWS.</a> <a data-t="382">So you can use a lot of the other AWS services</a> <a data-t="384">without needing to reauthenticate</a> <a data-t="386">or move around a whole bunch of credentials,</a> <a data-t="388">kind of log in once and use a lot of other different things.</a></p>

<p><a data-t="392">You can also use, again, these pre-allocated cloud budgets.</a> <a data-t="395">So it's pretty often that we see some of these customers</a> <a data-t="398">have a lot of money tied up into their cloud budgets,</a> <a data-t="402">and they can use a small portion of that</a> <a data-t="404">to purchase these RStudio Pro licenses to be used within SageMaker.</a></p>

<p><a data-t="408">And then lastly, reiterating that for some of our compliant</a> <a data-t="412">and secure needs of customers,</a> <a data-t="415">that SageMaker already has SOC2, FedRAMP, HIPAA compliance</a> <a data-t="419">and is adding additional compliance kind of certifications over time.</a></p>

<p><a data-t="426">I do want to call out some of the trade-offs or differences,</a> <a data-t="428">though, between a self-managed Posit Workbench</a> <a data-t="431">that includes RStudio Pro as well as JupyterLab and VS Code</a> <a data-t="435">versus RStudio Pro on SageMaker.</a></p>
