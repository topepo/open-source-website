<h3>Why use a listing?</h3>

<p><a data-t="292">So to sort of sum up what a listing is, a listing is an automatically generated list of content.</a> <a data-t="296">You didn't have to do any work to create that list,</a> <a data-t="296">you just had to tell Quarto where to look for this content.</a> <a data-t="303">It's styled via a template, so we saw that you could say type just default, and you get kind of like a blog style listing.</a> <a data-t="308">Type grid gives you this grid layout, and then type table is very literally a table of all these links.</a> <a data-t="314">And a listing in Quarto can be included on any page, so you just create the page and then add that listing item to the YAML, and you get a listing.</a></p>

<p><a data-t="325">And the way that Quarto does this, the way that it's generating the links on that listing page, is it actually goes in and looks at those files and looks at the front matter of them.</a> <a data-t="336">So in those files, in that blog directory, those Quarto files have things like the title set, the description set, the author set, an image set in that YAML,</a> <a data-t="343">so that Quarto can generate that nice little link to the content.</a></p>

<p><a data-t="348">And you might be asking why would you want to use a listing?</a> <a data-t="353">Listings are really great when you've got a very large collection of content, so you don't want to sort of by hand create links to all these things.</a> <a data-t="358">They're also really good for things that grow, collections or lists that grow.</a> <a data-t="363">So if you've got something like a list of projects that you're adding to over time, a listing makes a lot of sense because you can set it up once, and then it's just a matter of adding in the content when you want to expand that list.</a></p>

<p><a data-t="372">So what you've seen is a listing is something that's really common with blogs.</a> <a data-t="372">You'll see them out there a lot where people list, it's like a list of blog posts.</a> <a data-t="382">But listings can be really useful for other kinds of lists.</a> <a data-t="382">So I'm going to show you a few examples.</a> <a data-t="387">Here's an example on Isabella Velasquez's homepage where she's using a listing to list out her projects, so sort of like a project portfolio.</a> <a data-t="400">Here's another example, this is from Megan Hall, and this is a listing of talks.</a> <a data-t="408">So rather than sort of creating the talks list by hand, each one of these is a piece of content that has a little bit more details if you click on it, but the listing is being used to lay out that list of talks.</a> <a data-t="420">And then here's another one from Mikhail Kanui, which is a listing of publication.</a> <a data-t="424">So if you're in the academic sector, this is something you often need to have.</a> <a data-t="424">And each one of these listings is actually a link to a published paper on a publisher's website.</a></p>
