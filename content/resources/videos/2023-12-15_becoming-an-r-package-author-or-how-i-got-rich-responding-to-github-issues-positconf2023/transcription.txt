<h3>Encoding domain knowledge into a package</h3>

<h3>Why every R user should consider contributing</h3>

<p><a data-t="834">So what I hope to sort of get across here is that there's lots of different ways to contribute to packages.</a> <a data-t="840">And I think this map of sort of how I did it, starting with GitHub issues, just asking them,</a> <a data-t="846">then maybe responding to GitHub issues that you see, maybe level up and get into fixing little bugs or adding small new features,</a> <a data-t="854">and then eventually taking what you know outside of R package development and try to insert it into the package framework</a> <a data-t="861">is, like, really powerful, and there's a lot of good reasons for why I think everyone who's an R user might want to do this.</a></p>

<p><a data-t="868">And I think of them along sort of two dimensions here.</a> <a data-t="871">The first is becoming a tool builder.</a> <a data-t="873">So you're going to improve your R programming skills markedly.</a> <a data-t="877">When you start thinking about writing functions, you're learning about abstracting your code and writing modular and generalizable code.</a> <a data-t="883">You're going to learn all about unit tests and how important those are.</a> <a data-t="886">You're also going to learn a lot about Git and GitHub.</a> <a data-t="889">Maybe if you're a solo R user, you might use Git just for version control on your own.</a> <a data-t="893">But once you're in the package development world, you're all about collaboration.</a> <a data-t="897">And so you're going to learn really quickly about forks and clones and merge requests and merge conflicts and all that fun GitHub stuff.</a> <a data-t="904">And I think ultimately what this does is make you more desirable to an employer because you can say, not only do I know how to use R,</a> <a data-t="911">I'm also an R package developer.</a> <a data-t="914">Also, I know how to use Git and GitHub really well.</a></p>

<p><a data-t="918">The other domain that I kind of think of this on is on community contributions.</a> <a data-t="923">So you, through this, are going to be able to meet other R users online.</a> <a data-t="929">I've never actually met Kyle in person.</a> <a data-t="931">But as you can see, we've communicated a lot on Twitter, on GitHub, and other places.</a> <a data-t="934">And I kind of think of him as a friend.</a> <a data-t="938">You're also going to be able to grow and sustain the open source community.</a> <a data-t="941">So as R users, we all have benefited so much from so many people who have volunteered their time to contribute to the open source community.</a> <a data-t="951">And so me, now, as a tidycensus co-author, I'm able to give back in just a little bit, a little way,</a> <a data-t="958">and show my appreciation for that community and help grow it and help sustain it.</a> <a data-t="962">And then you might get really cool opportunities like being able to stand up on the stage and speak to you at posit::conf.</a></p>
