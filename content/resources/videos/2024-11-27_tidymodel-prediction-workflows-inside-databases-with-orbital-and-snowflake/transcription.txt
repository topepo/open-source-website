<h3>What is orbital?</h3>

<p><a data-t="34">First up, what is orbital?</a> <a data-t="36">Orbital is an incredible package.</a> <a data-t="38">It was introduced at this year's PositConf.</a> <a data-t="40">I'd highly encourage you to go check out the talks.</a> <a data-t="43">They're now available on YouTube.</a> <a data-t="44">Emil Hviefelt from the tidymodels team gave that talk.</a></p>

<p><a data-t="48">The goal of orbital is to enable running predictions of tidymodels workflows directly inside databases.</a> <a data-t="56">So what you can think of is kind of almost like a three-step process.</a> <a data-t="59">First up, use the tidymodels tools that you're familiar with to create an R model.</a> <a data-t="66">You can use tools like Recipes, </a><a href="/software/parsnip/">Parsnip</a><a data-t="66"> to create that model, and then use orbital.</a> <a data-t="74">And what orbital does is it takes all the steps of that tidymodel and converts it into SQL</a> <a data-t="80">that a database like Snowflake can understand.</a></p>

<p><a data-t="83">You can then use Snowflake, use any other database to either run the predictions of that model,</a> <a data-t="91">or you can actually use that SQL to then actually deploy the model directly to Snowflake</a> <a data-t="97">or any other database as a native object.</a></p>

<p><a data-t="105">Just a quick refresher for those of you who aren't familiar with workflows</a> <a data-t="110">or maybe haven't used it in a while.</a> <a data-t="112">Workflows is a part of the tidymodels package, tidymodels universe.</a> <a data-t="116">And what it lets you do is bundle together your data pre-processing steps,</a> <a data-t="122">your modeling steps, and post-processing steps into one R object.</a> <a data-t="126">So you can combine all the things that are needed to run your model,</a> <a data-t="131">that can feature engineering, model fitting, post-processing, into a single portable R object.</a></p>

<p><a data-t="138">And the secret sauce of orbital is that it does that conversion over the entire workflow object.</a> <a data-t="145">So it'll convert all of your feature engineering steps.</a> <a data-t="149">It'll convert your model fitting steps.</a> <a data-t="151">It'll convert basically anything that you can fit into workflows.</a> <a data-t="154">You can put into orbital, and then that'll turn it into SQL that can be then run on a database.</a></p>

<h3>Why use orbital?</h3>

<p><a data-t="163">You might be asking yourself, why would I use orbital?</a> <a data-t="165">Why would I want to put my model fitting into a database?</a></p>

<p><a data-t="173">Number one, putting your model fitting in your database allows you to share your models with anyone super easily.</a> <a data-t="179">Because you're taking these models, converting them to SQL,</a> <a data-t="183">what you're doing is you're putting them into a language that your database can understand,</a> <a data-t="188">and you're putting them into a language that basically almost anyone can access.</a> <a data-t="191">So if you can take that model, you can take your model, turn it using orbital into SQL,</a> <a data-t="196">deploy it onto Snowflake, anyone else on your team, whether they're using something like Python,</a> <a data-t="202">or whether they're using something that doesn't use code at all,</a> <a data-t="205">something like Power BI, can now access the outputs of your model.</a></p>

<p><a data-t="212">And number two, probably most importantly, it makes your model really, really, really, really, really fast.</a> <a data-t="218">One of the really cool things about databases is they have a ton of compute power behind them.</a> <a data-t="222">They're really good at doing things like feature engineering, like data preprocessing.</a> <a data-t="227">And especially because your data is already there,</a> <a data-t="230">one of the things that fitting your models directly in the databases does is it removes the step of having to download data onto a different machine,</a> <a data-t="239">download it onto an R machine.</a> <a data-t="242">Instead, the model fitting can be done directly in the database, which tends to make that make the model fitting incredibly fast.</a></p>

<blockquote><a data-t="212">And number two, probably most importantly, it makes your model really, really, really, really, really fast.</a> <a data-t="218">One of the really cool things about databases is they have a ton of compute power behind them.</a> <a data-t="222">They're really good at doing things like feature engineering, like data preprocessing.</a></blockquote>
