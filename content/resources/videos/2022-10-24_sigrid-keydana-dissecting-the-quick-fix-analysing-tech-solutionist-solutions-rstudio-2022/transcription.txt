<p><a data-t="0">Hello and welcome to my talk, Dissecting the quick fix, analysing tech-solutionist solutions.</a></p>

<p><a data-t="12">Let me start with a definition.</a> <a data-t="15">What is tech-solutionism?</a> <a data-t="18">Tech-solutionism is the tendency to use technology to solve a societal, political, organisational, environmental, what have you, problem.</a> <a data-t="26">A few examples should make this more clear.</a></p>

<h3>Three examples of tech-solutionism</h3>

<p><a data-t="30">Example number one, riots.</a> <a data-t="33">What's the problem?</a> <a data-t="34">Well, suppressed population groups, for example, ethnic minorities, are starting to mobilise.</a> <a data-t="42">What do we observe?</a> <a data-t="42">This is more likely to happen when it's hot, very hot.</a> <a data-t="49">What is the solution?</a> <a data-t="49">Well, just install climatisation throughout.</a> <a data-t="55">And what do we gain?</a> <a data-t="55">We don't have to actually address those groups' demands.</a></p>

<p><a data-t="59">Example two, mass shootings.</a> <a data-t="61">What is the problem?</a> <a data-t="61">Since 2014, on average, there has been more than one mass shooting a day in the US.</a> <a data-t="69">What do we observe?</a> <a data-t="69">You can't always see if someone is carrying a gun.</a> <a data-t="75">What's the solution?</a> <a data-t="75">Well, just install advanced surveillance technology.</a> <a data-t="80">And what do we gain?</a> <a data-t="80">We don't have to change the laws related to gun ownership.</a></p>

<p><a data-t="85">Example number three, climate change.</a> <a data-t="90">What is the problem?</a> <a data-t="90">Climate change is rendering parts of the earth inhabitable and strongly impacts living conditions and others.</a> <a data-t="99">What's our observation?</a> <a data-t="99">Time is passing and we are less and less on track to keep the situation under control.</a> <a data-t="105">What's the solution?</a> <a data-t="105">Well, we just move to Mars or somewhere else in space.</a> <a data-t="111">What do we gain?</a> <a data-t="111">Well, we survive and get to have fun, if that's the kind of thing we find fun, right?</a></p>

<h3>Why cynicism isn't enough</h3>

<p><a data-t="119">So yeah, I think by now it should be pretty clear what I'm talking about here.</a> <a data-t="125">On the other hand, you probably also have noticed I got more and more into some kind of cynical tone, like disgust, cynicism, and pure cynicism and just talking that way doesn't bring us very far.</a> <a data-t="120">Why?</a></p>

<p><a data-t="121">Well, on the one hand, we won't notice less latent solutions, quote, unquote, when it's less clear.</a> <a data-t="131">Like, for example, talking about climate change, what about things like geoengineering?</a> <a data-t="136">Isn't that easy?</a> <a data-t="138">And also, we're not never going to reach those people who don't already agree with us anyway, right?</a> <a data-t="146">So I don't think this is very useful.</a></p>

<p><a data-t="149">On the other hand, I think what we should try is to understand what is so seductive about this thing.</a> <a data-t="157">Well, like the word already says, it solves something.</a> <a data-t="161">Good, right?</a> <a data-t="163">It solves a problem.</a> <a data-t="166">Often it solves a hard problem, a problem that otherwise we wouldn't know how to solve, how to address even.</a> <a data-t="173">It solves a problem we would have to make sacrifices for.</a> <a data-t="177">And we don't want that, right?</a> <a data-t="179">And often it just solves a problem we're terribly scared about.</a></p>

<h3>A framework of W questions</h3>

<p><a data-t="186">So what I'm going to try to, in this talk is, or what I think we need is some kind of framework to analyze those proposals, and a framework which does not just ask the technical questions, because this is something we are already doing to some degree.</a> <a data-t="227">But I think the technical aspect is not enough, and so I'm going to try to propose a very simple framework, a framework of W questions, like what, where, when, why, who, to whom, how.</a> <a data-t="243">It's not a W anyway, but to address this in a systematic way.</a> <a data-t="248">And there I want to focus on the more neglected aspects.</a></p>

<p><a data-t="254">But as I'm going to focus on two of these aspects, let me say what I mean by the rest of them.</a> <a data-t="271">First what.</a> <a data-t="272">What does the solution do?</a> <a data-t="274">And this should include the complete technical workflow, like from where do we get the data from, how do we train our machine learning models, how do we evaluate them.</a> <a data-t="237">And in this area, we already have some things like data set documentation, model documentation, which is all great, but which is all the technical side of things.</a></p>

<p><a data-t="289">Then, where, when, and to whom will the solution be applied?</a> <a data-t="294">And this should include the complete geographic, social, political, et cetera context.</a> <a data-t="301">Then, how will the solution be applied?</a> <a data-t="304">And here, really, the details matter.</a> <a data-t="307">Next, who is implementing the solution?</a> <a data-t="311">There, as you see, I want to talk about more.</a> <a data-t="315">And also, why is the solution being developed?</a> <a data-t="319">There also, I would like to go into some detail.</a></p>

<p><a data-t="323">So in other words, in this talk, I want to go into the why and the who, because in my view, these are the two aspects which are being neglected most in this area.</a></p>
