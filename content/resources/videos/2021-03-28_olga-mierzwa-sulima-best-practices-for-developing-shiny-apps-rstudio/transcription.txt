<p><a data-t="48">So how should you organize the code inside your shiny app?</a> <a data-t="48">First answer is use shiny modules.</a> <a data-t="57">So shiny modules let you decompose your application.</a> <a data-t="57">So you can follow a don't repeat yourself coding principle.</a> <a data-t="63">They offer encapsulation, modularity, reusability, and</a> <a data-t="74">allow you to test the whole components.</a> <a data-t="81">They allow you to organize your app in the server</a> <a data-t="81">and the UI part, which in essence mean that you can define your own pair of server and</a> <a data-t="90">UI logic and use it inside your app or embed it in the other module.</a></p>

<p><a data-t="101">So secondly, you can also organize your code using </a><a href="/software/r6/">R6</a><a data-t="101"> classes.</a> <a data-t="101">And an R6 class is a modern</a> <a data-t="110">and fast and simple implementation of object-oriented programming in R.</a></p>

<p><a data-t="143">So how do you organize your code using R6 classes?</a> <a data-t="143">So you can keep each class in a separate</a> <a data-t="150">script with a name similar to the class name.</a> <a data-t="150">You can store the script in a folder separated</a> <a data-t="158">from the other code.</a> <a data-t="158">You can import the code using a package or as in here in the example,</a> <a data-t="167">using a use function from the modules package.</a> <a data-t="176">And then you can initialize the class with a new</a> <a data-t="176">method, which is ultimately available for all the classes and attach it to some object.</a></p>

<p><a data-t="184">And the third option is to use modules from the honey.</a> <a data-t="184">And I would like to avoid some</a> <a data-t="197">confusion with shiny modules.</a> <a data-t="197">So the modules I'm going to talk right now is something completely</a> <a data-t="204">different from shiny modules.</a> <a data-t="204">So the modules are the organizational unit for the source code,</a> <a data-t="213">and they can contain shiny modules or R6 classes or just functions you define.</a></p>

<p><a data-t="221">And the cool thing about them is that they enforce rigor when defining the dependencies.</a> <a data-t="227">It means that you have to explicitly declare which functions or packages your code is using and you have to import them.</a></p>

<p><a data-t="264">So to put it all together, shiny modules are six classes and modules.</a> <a data-t="264">So shiny modules, they let you reuse UI and the server part of the components.</a> <a data-t="270">R6 classes, they let you build objects that have a single responsibility and implement given logic.</a> <a data-t="276">And behind modules, they encapsulate dependencies and let you organize files that could be shiny modules and R6 classes as a separate unit.</a></p>
