<h3>Making the menu collapsible with CSS</h3>

<p><a data-t="55">The first thing you can do is to set the app such that the menu is collapsible.</a> <a data-t="59">In Shiny, by default, when collapsible is set to true,</a> <a data-t="63">the menu is collapsed and the screen size is 940 pixels or less.</a> <a data-t="67">Here, I used CSS and media queries to style the menu icon</a> <a data-t="70">and to reduce the font size of the tab names and tab background color.</a></p>

<p><a data-t="75">CSS is used to style HTML elements and media queries enable web page content</a> <a data-t="80">to adapt to different screen sizes and resolutions.</a> <a data-t="83">Think of media queries as eval statements.</a> <a data-t="85">For example, if the screen size is less than 480 pixels,</a> <a data-t="89">typical of mobile phones, convert the background color to pink</a> <a data-t="92">or the h1 font size to 8 pixels.</a> <a data-t="95">We use breakpoints with media queries to set points</a> <a data-t="97">where websites adjust to the width of devices.</a></p>

<h3>Hiding elements and using shinybrowser</h3>

<p><a data-t="100">Next, you can choose to remove some objects</a> <a data-t="103">so that they are not visible on some devices.</a> <a data-t="105">In this example, we removed the icons and introduced a button on the mobile view.</a> <a data-t="110">You can do this by setting display as none in CSS.</a> <a data-t="114">I also used media queries to edit the topography of the paragraphs for the mobile view.</a></p>

<p><a data-t="119">Shiny browser, a package by Dinatali,</a> <a data-t="121">lets you easily detect a user's web browser information in Shiny.</a> <a data-t="125">One of the caveats is that the width and the height of the browser window</a> <a data-t="128">are only reported once when the detect function is initially called,</a> <a data-t="132">so you'd need to refresh the page to get the new dimensions.</a></p>

<p><a data-t="136">You can use shiny browser to reduce the font size of different things</a> <a data-t="139">such as plot titles, axis text and labels.</a> <a data-t="143">You can have a function that generates your graph</a> <a data-t="145">with an argument of type of device.</a> <a data-t="147">For example, if your device is mobile,</a> <a data-t="148">we want the title to be of size 14, otherwise size 18.</a> <a data-t="152">Then on the UI, insert the detect function and on the server,</a> <a data-t="155">define a reactive element that detects the type of device based on the device's width</a> <a data-t="160">and use this as the graph's function argument.</a></p>

<p><a data-t="164">And this is what I did in the Kwanzaa tab.</a> <a data-t="166">The axis text size in the desktop view is a bit larger than that in the mobile view.</a> <a data-t="172">You can use the same principle to change the orientation of the graph</a> <a data-t="176">as seen on the tattoo tab.</a> <a data-t="177">On the mobile view, the chart is flipped.</a></p>
