<h3>Making your model reusable: R package infrastructure</h3>

<p><a data-t="391">So let's go through these questions one by one.</a> <a data-t="393">Can someone use and improve on my model?</a> <a data-t="397">There's a lot of work in the R space now in terms of reproducibility.</a> <a data-t="400">RNV is a great option, Docker is a great option.</a> <a data-t="403">The one I actually like to use is making it a package, and I'm not talking about making it for CRAN, but I think there's two elements of packages that I think are really underrated and help kind of build things going forward.</a> <a data-t="415">One of them is adding a description file, and the other one is wrapping code into functions.</a></p>

<p><a data-t="415">Adding a description file, one of the things that adding a description file does, it forces you or asks you to add a license.</a> <a data-t="421">And a license is really important, especially when you're doing public work, because it tells people what you're okay with them doing with your work, right?</a> <a data-t="430">So if you want them to, if you're okay with them doing anything with that, say so.</a> <a data-t="433">Add an MIT license, that's what that's basically there for.</a> <a data-t="437">If you want them to only use this work to help extend public work, you can use a license like GPLv3, right?</a> <a data-t="442">So just choosing the license helps communicate that you want them to use that work.</a> <a data-t="447">And by sharing that work to them with that license, people can understand what you're okay with them doing with it.</a></p>

<p><a data-t="454">The other thing that it does is it makes dependencies, it makes installation of dependencies really easy, because you're forced to list all the dependencies, and if you're able to also list the minimum versions, it makes things like remotes or Pac or Pac-Man work, because they can read the description file from GitHub and then install all the dependencies for you while you're installing this package, or this package.</a> <a data-t="476">And even if it doesn't have any functions in it, it'll still install the dependencies for you, and then you can use that in, you know, running the script after you've cloned it on GitHub.</a> <a data-t="484">Of course, Useless has this covered, so you don't actually need to type out a description file.</a> <a data-t="489">It's got four functions that you need to know to do exactly what I'm talking about.</a> <a data-t="492">Use description, use MIT license, use package for each package that you're using, and once you've added all the packages, you use latest dependencies, and it's gonna version lock the version that's on your machine.</a> <a data-t="502">So it'll put that in as the minimum version, and then, you know, you set this up for success.</a></p>

<p><a data-t="510">The other thing that I like to do, this is kind of the next level, is to wrap logic into functions.</a> <a data-t="515">It's really easy for users.</a> <a data-t="517">So from your end of things, what you need to do is convert the hard-coded variables and data frames to arguments.</a> <a data-t="522">Extract function is a cool feature in </a><a href="/software/rstudio/">RStudio</a><a data-t="522">, Ctrl-Alt-X, and then if you copy and paste or if you select the entire code chunk, and you press this shortcut, I think it's Command-Option-X.</a> <a data-t="534">I don't use a Mac, but it's related.</a> <a data-t="537">It'll automagically create a function and put the skeleton of your code in a function and take all the variables and put them into arguments.</a> <a data-t="545">So this already basically does it for you, and then the other thing you can do is add some usage notes.</a> <a data-t="549">And I say documentation here because documentation scares people, but really understanding how you use logic, how you use the function, and what the function's meant to do will help other people kind of understand where you're going with it.</a> <a data-t="560">And the goal is to make it easy for users to run, right?</a> <a data-t="564">So our users love functions.</a> <a data-t="566">They understand exactly what to do when you give them a function, you put arguments into it, and then you kind of go from there.</a></p>
