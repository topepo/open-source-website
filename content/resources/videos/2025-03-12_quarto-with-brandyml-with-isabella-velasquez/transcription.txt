<h3>Challenges of branding and what brand.yml solves</h3>

<p><a data-t="507">With that being said, branding can come with some challenges.</a> <a data-t="511">So one challenge is excessive manual effort.</a> <a data-t="515">So the idea of applying colors and fonts and logos across reports and apps and dashboards,</a> <a data-t="522">this all takes time and is prone to errors.</a> <a data-t="525">And especially if you're an R user, you might find yourself a bit overwhelmed with the number</a> <a data-t="531">of options that you have to apply branding to your plots or your documents and your reports,</a> <a data-t="539">et cetera.</a></p>

<p><a data-t="540">Another thing is that brands can be difficult to update.</a> <a data-t="544">So if you think about after you've applied all those colors and fonts and logos across</a> <a data-t="548">reports and apps and dashboards, if something needs to be updated, it can be really hard</a> <a data-t="554">to manage that across all your products consistently.</a> <a data-t="559">And speaking of consistency, you want to make sure that all the contributors follow</a> <a data-t="564">brand guidelines across everything that you're creating.</a> <a data-t="568">And that can also be challenging to manage, depending on their familiarity and their tool</a> <a data-t="573">sets and things like that.</a></p>

<p><a data-t="577">So that's why when brand.yaml was released, it was so exciting.</a> <a data-t="582">It provided another option for people looking to create beautifully themed documents with ease.</a> <a data-t="590">So its tagline, brand.yaml's tagline, is unified branding with a simple YAML file.</a> <a data-t="596">And the idea is straightforward.</a> <a data-t="599">Every company has brand guidelines, or most companies have brand guidelines.</a> <a data-t="603">And brand.yaml translates them into a YAML file that can style slides and websites and</a> <a data-t="608">shiny apps and more.</a> <a data-t="610">And this approach ensures that branding is consistent across different projects.</a> <a data-t="615">And the goal is to structure a YAML file in a way that makes sense to both those referencing</a> <a data-t="621">it for brand consistency and to developers using it to theme their work.</a></p>

<p><a data-t="628">And so the way that it works is you take your brand guidelines and you translate them into</a> <a data-t="633">structured YAML.</a> <a data-t="634">We'll talk more about what YAML is in a little bit.</a> <a data-t="638">So you create this one single structured YAML file that Quarto and Shiny can pick up.</a> <a data-t="643">And so it ensures that you have consistent branding across your outputs.</a> <a data-t="649">And the idea, speaking about what we mentioned earlier, is you don't have to use different</a> <a data-t="653">systems to change the appearance of your documents.</a> <a data-t="657">So will you need a CSS?</a> <a data-t="659">Would you need to learn </a><a href="/software/bslib/">BSLib</a><a data-t="659">?</a> <a data-t="662">Really the answer is maybe.</a> <a data-t="663">The idea behind brand.yaml is to offer good enough theming.</a> <a data-t="667">It might not perfectly match your company's guidelines, but it will get you very close.</a></p>

<p><a data-t="681">And so to specify a brand, you create something called a brand.yaml file in the root directory</a> <a data-t="691">of your Quarto project.</a> <a data-t="694">And if you've created a Quarto project before, you know it comes bundled with an underscore</a> <a data-t="698">Quarto.yaml file.</a> <a data-t="701">So you put your brand.yaml file next to your Quarto.yaml file.</a> <a data-t="706">And then Quarto will automatically detect the brand.yaml file and apply the branding</a> <a data-t="711">to all supported document formats in the project.</a> <a data-t="716">And then another advantage is that you can use brand.yaml in conjunction with other theming</a> <a data-t="722">files, like if you need additional tweaking with CSS.</a></p>

<p><a data-t="729">And in terms of where it works, if you're using it in Quarto, you need version 1.6 or</a> <a data-t="736">above.</a> <a data-t="737">So please make sure that you're running 1.6 or above.</a> <a data-t="741">And if you're not, you can download it from the Quarto website or install it the way that</a> <a data-t="745">you usually do with Quarto.</a> <a data-t="747">And more specifically in Quarto, the formats that are available are HTML, Dashboard, </a><a href="/software/revealjs/">RevealJS</a><a data-t="747">,</a> <a data-t="755">HR Slides, and Typst, which are PDF files.</a> <a data-t="760">You can also use brand.yaml in </a><a href="/software/shiny-python/">Shiny for Python</a><a data-t="760"> version 1.2 and above.</a> <a data-t="766">And very recently, BSLib version 0.9 was released.</a> <a data-t="771">And with that, now brand.yaml also works with R Markdown and Shiny for R.</a></p>
