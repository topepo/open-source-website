<h3>The problem: making statistics engaging</h3>

<p><a data-t="101">When teaching something that has the potential to be a little bit dry, what is it that makes</a> <a data-t="106">it difficult to motivate learners?</a> <a data-t="109">Within our own local academic circles, Hassa and I had gotten the impression that many</a> <a data-t="114">people saw statistics as either something unintuitive, scary, or very boring, and we</a> <a data-t="121">wanted to see if we could do something about that.</a></p>

<p><a data-t="123">So we thought, let's teach a really cool stats seminar at our university, and we were pretty</a> <a data-t="130">excited about this idea, but for reasons of red tape, our idea for teaching got squashed,</a> <a data-t="138">but we decided that we still wanted to make that content and put it online.</a></p>

<p><a data-t="143">But then that introduces a second problem, which is that we now lose that precious face-to-face</a> <a data-t="148">time with students in a classroom that usually adds an additional hurdle for engagement,</a> <a data-t="155">unless we could come up with a way to keep that content compelling.</a> <a data-t="159">And so we thought that we could try this by nesting our statistics lessons inside of a</a> <a data-t="164">story, and we all know that for a good story, you need good characters.</a></p>

<h3>Good characters: the teacup giraffe</h3>

<p><a data-t="170">So really early on in our process, we started to brainstorm some heroes that could play</a> <a data-t="175">the protagonists throughout our statistical content that we were going to be building.</a> <a data-t="181">And ultimately, we came up with the teacup giraffe, which is our fictional little species</a> <a data-t="187">named for its tiny stature.</a></p>

<p><a data-t="190">And so now we had this hero for all of our stats modules, and we could begin to make</a> <a data-t="194">up data about it and build a story around it, and so I just want to give you a small</a> <a data-t="199">sampler of what this narrative is like.</a></p>

<p><a data-t="203">So there are two distinct populations of teacup giraffes, each inhabiting their own</a> <a data-t="208">remote islands.</a> <a data-t="210">And as the learner coming to our site, you play the role of the graduate student, and</a> <a data-t="215">you've been tasked with collecting data about this mysterious little species, which the</a> <a data-t="222">most sought-after data is about their height, which on both islands is variable.</a> <a data-t="228">Teacup giraffes also really love eating celery.</a> <a data-t="231">So by assessing how much of it that they eat, you now have an additional variable to measure.</a> <a data-t="237">And all the while that you're collecting this data, you deal with really demanding emails</a> <a data-t="242">from your advisor back at home, who needs GG plots and other results for these data,</a> <a data-t="247">and he needs them now.</a></p>

<p><a data-t="250">But hopefully, teacup giraffes make that a little bit less stressful.</a></p>

<p><a data-t="256">So with this, now you've gotten a little bit of a sampler of the teacup giraffe's world,</a> <a data-t="261">and this is actually what served as our backdrop for our equations, our code, and all of our</a> <a data-t="267">exercises that we built.</a> <a data-t="269">And actually, we went a little bit overboard in creating this world, because we wrote an</a> <a data-t="275">entire backstory that was completely separate from the stats content, and it's all about</a> <a data-t="282">how the giraffes were discovered, a little bit about their silly behavior.</a> <a data-t="287">And at this point, you're probably thinking it's reaching the level of absurdity, and</a> <a data-t="292">I actually agree with you.</a></p>

<p><a data-t="294">But we hope that all of this storyboarding would really help to suck learners in, and</a> <a data-t="298">not only learners, because as content creators, we also got sucked into the teacup giraffe</a> <a data-t="304">world, quite literally.</a></p>

<p><a data-t="276">Okay, but silliness aside, if you do have some creative muscle that you'd like to flex,</a> <a data-t="316">then creating your own good characters is a great way to do that with your educational</a> <a data-t="319">content.</a> <a data-t="321">All of the character elements that I've shown you so far here are illustrations that I made</a> <a data-t="325">on an iPad Pro using the Procreate app, but in case drawing your own characters is not</a> <a data-t="330">a possibility, there are many other resources where you can find really great high-res and</a> <a data-t="335">open-source images, and one of those is unsplash.com, so I point you there for really strong visuals.</a></p>

<p><a data-t="302">Okay, so usually at this point of introducing the project to someone, most people think</a> <a data-t="308">that we've actually made a children's book and not a stats website.</a> <a data-t="313">But our teacup giraffes do have some more utility than just ramping up the delight meter.</a> <a data-t="318">We specifically created our character with our lessons in mind, so in particular, we</a> <a data-t="324">could make things like statistical concepts, like the variance, a little bit less abstract,</a> <a data-t="329">because not only do our teacup giraffes provide strong visuals for word problems, like, for</a> <a data-t="334">example, asking what percentage of giraffes in our population will be shorter than this</a> <a data-t="339">little guy here, but maybe a more subtle benefit is that we can use our character to link to</a> <a data-t="346">otherwise less intuitive concepts.</a> <a data-t="348">For example, by using the spots, the color of the spots on our characters to show and</a> <a data-t="354">illustrate the concept of the sum of squares and how you build that, which is a precursor</a> <a data-t="358">for calculating the variance.</a></p>

<h3>Good play: interactivity in R Markdown</h3>

<p><a data-t="362">So with good characters, we now had one pillar in place to help make our content feel a little</a> <a data-t="367">bit more friendly.</a> <a data-t="369">And the next step was to include good play.</a> <a data-t="373">And by that I mean taking full advantage of the HTML output that you get with our markdown</a> <a data-t="378">by inserting movement and interaction so that the learner actually has something that they</a> <a data-t="382">can do on your site.</a> <a data-t="384">And I just want to share five examples of packages and tools that we found most useful</a> <a data-t="388">for this.</a></p>

<p><a data-t="390">The first one is Plotly.</a> <a data-t="392">So Plotly lets you create clickable and interactive plots so that a learner can see a concept</a> <a data-t="397">play out in real time.</a> <a data-t="399">This, for example, is a Plotly plot that we used to explain why the equation for sample</a> <a data-t="404">variance needs to be divided by n minus 1, and we, of course, explained this in the text</a> <a data-t="409">before showing this plot.</a></p>

<p><a data-t="412">And another good thing that you can use for good play is the Gifsky package, which lets</a> <a data-t="417">you make GIFs.</a> <a data-t="418">And so once you've installed Gifsky, you can include an animation hook in the code chunk</a> <a data-t="425">settings, and this will stitch together any plots that are output from that code chunk.</a> <a data-t="431">So this GIF here was made by throwing several ggplots in one code chunk and then looping</a> <a data-t="435">through all of them with something like either a per function or an apply function.</a> <a data-t="443">But you can actually use the Gifsky hook to make GIFs out of regular images as well, so</a> <a data-t="447">you're not just limited to ggplots.</a> <a data-t="450">This is a GIF that we made to represent very abstractly the process of going from cross</a> <a data-t="455">products to covariances and then standardizing those to become correlation scores.</a></p>

<p><a data-t="463">But the most important part of good play that we wanted to give our learners was the ability</a> <a data-t="467">to go through some stats methods by hand, or in other words, we wanted learners to have</a> <a data-t="472">a chance to break apart functions, write their own functions, at least ones to kind of eliminate</a> <a data-t="477">that statistical black box feeling, and in order to do that, we needed interactive console</a> <a data-t="482">windows in our site.</a></p>

<p><a data-t="486">So our solution was to take a </a><a href="/software/learnr/">LearnR</a><a data-t="486"> window and embed it into our R Markdown website.</a> <a data-t="493">Here's an example of how this works.</a> <a data-t="495">So if you have not seen the LearnR package before, LearnR documents are R Markdown documents</a> <a data-t="501">that can be used as interactive tutorials, and they usually exist as standalone things</a> <a data-t="506">or standalone apps, but what's really cool about them is that they have these built-in</a> <a data-t="510">little consoles, so you can preload code into them for a learner or text.</a> <a data-t="515">You can also preload solutions, which a user can then run and see their output, and that's</a> <a data-t="521">great.</a></p>

<p><a data-t="534">So one of the problems, however, is that when a user is using a LearnR document in the iframe,</a> <a data-t="540">it's a little tricky to get it to resize appropriately when the user is interacting with it, and</a> <a data-t="545">so a solution to get it to resize appropriately is to use a little bit of JavaScript, and</a> <a data-t="551">in particular something called an iframe resizer, which does exactly what it sounds like.</a> <a data-t="556">So to our knowledge, this is the first time that someone's applied an iframe resizer to</a> <a data-t="561">a LearnR window that's inside of an iframe, and the process for it is a little bit involved,</a> <a data-t="565">so I'm not going to cover the details here, but I have created a step-by-step guide that</a> <a data-t="569">shows you how to do this in your own R Markdown documents, and you can find that on my blog,</a> <a data-t="575">which is at deseret.rbind.io.</a></p>

<p><a data-t="577">And LearnR windows are not the only things that you can embed, of course.</a> <a data-t="581">There are two more quick pieces of good play that I'd like to share with you that are considerably</a> <a data-t="585">easier to embed in your site.</a> <a data-t="587">So first there's a regular </a><a href="/software/shiny-r/">shiny</a><a data-t="587"> app like this one, which we used to have learners guess</a> <a data-t="591">correlation values, and using the same iframe approach, you can also embed quiz questions,</a> <a data-t="599">and we haven't yet done this in our project, but we look forward to asking really tough</a> <a data-t="603">quiz questions like this one.</a></p>

<p><a data-t="613">So by using Plotly, Gifski, LearnR, and Shiny, we could make our R Markdown site so much</a> <a data-t="617">more interactive, and there's still one more piece missing that I'd like to share, though,</a> <a data-t="621">about our site, and that is good design.</a></p>

<h3>Good design: layout and styling</h3>

<p><a data-t="621">And so by this I mean the page layout and styling that's distinct from the content,</a> <a data-t="626">but brings good vibes and a sense of polish to your product.</a> <a data-t="631">So essentially judging a book by its cover or by extension, judging a site by its HTML</a> <a data-t="638">and CSS.</a> <a data-t="639">And while this is, of course, not the best way to measure the quality of your content,</a> <a data-t="644">the degree of styling that you give your site not only changes a learner's first impression</a> <a data-t="649">when they first see your page, but it can also improve readability, navigation, and</a> <a data-t="654">the user's general experience as they go through your content.</a></p>

<p><a data-t="658">So out of the box, here is what our R Markdown site might have looked like just using the</a> <a data-t="662">default settings for an R Markdown website.</a> <a data-t="665">And this is a decent start, but I want to share three things that I think deliver some</a> <a data-t="669">pretty big impact.</a> <a data-t="671">And the first is decluttering.</a></p>

<p><a data-t="674">So we do this by getting rid of borders and highlights and numbers that are not absolutely</a> <a data-t="680">essential to our output.</a> <a data-t="682">And then we can add a whole lot more white space so things don't feel so cramped.</a> <a data-t="687">So around the table of contents, between paragraphs and headers.</a> <a data-t="691">We can also constrain the width of the body of the text.</a> <a data-t="695">And finally, we can increase font size and the line height of the paragraph, too.</a> <a data-t="700">And so once we've made these changes, we get a lot more breathing room on the page</a> <a data-t="705">and it's easier to read.</a></p>

<p><a data-t="707">Most of these changes do involve a little bit of CSS, but if that scares you, one non-CSS</a> <a data-t="713">thing that you can do in your R Markdown document, straight in the document itself, is just by</a> <a data-t="717">adding backslashes or line breaks on their own lines.</a> <a data-t="723">And adding that space in between is manually something that goes a long way when it comes</a> <a data-t="729">to making more breathing room.</a></p>

<p><a data-t="732">The second thing I want to share about good design is setting the visual ambiance for</a> <a data-t="737">the site by adding a big image to the top, which in the lingo is called a hero image.</a> <a data-t="742">And so you can do this if you're familiar with the R Markdown setting for the includes</a> <a data-t="747">before body option, and you can just insert a file in there that references some big</a> <a data-t="752">image to go at the very top of your page.</a></p>

<p><a data-t="764">And finally, making use of built-in features of Pandoc, which is what runs under the hood</a> <a data-t="770">of R Markdown, to make things like note boxes.</a> <a data-t="773">So these can help organize your page content and help call out things like instructions</a> <a data-t="778">or notes or critical thinking questions for your learners.</a> <a data-t="781">Here we've used them to encase the learning objectives for this particular module.</a> <a data-t="786">And the step-by-step process for making these note boxes is also something that you can</a> <a data-t="790">find on my blog.</a></p>

<p><a data-t="795">So there's some other things that you can tweak as well.</a> <a data-t="797">Things like adding a logo, Google fonts, you can customize your nav bar.</a> <a data-t="802">And these and many other good design elements I've shown you today are things that, again,</a> <a data-t="806">involve some CSS.</a> <a data-t="808">If your experience with CSS in the past has been either difficult or nonexistent, I hope</a> <a data-t="814">that this does not stop you.</a></p>
