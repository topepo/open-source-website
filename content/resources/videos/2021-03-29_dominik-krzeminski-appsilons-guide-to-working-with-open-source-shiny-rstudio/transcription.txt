<h3>Popular Shiny extensions</h3>

<p><a data-t="126">So there is no way that I could cover all of them in detail during my 10-minute slot,</a> <a data-t="132">but I decided to cherry pick just a few, without which I can't even start thinking about</a> <a data-t="139">starting developing my own Shiny applications.</a></p>

<p><a data-t="142">So let's kick off with ShinyJS, which is a package from Dean Attali that lets you perform</a> <a data-t="150">common, useful JavaScript operations in Shiny, like hiding elements, disabling inputs,</a> <a data-t="155">resetting values, or alerting some variables that I often use for debugging myself.</a> <a data-t="161">The cool thing, though, is that you don't really need to know any JavaScript to start using</a> <a data-t="166">that, because everything is wrapped in R functions that are nicely documented, so it's</a> <a data-t="171">quite easy to get started.</a></p>

<p><a data-t="173">Then we have Shiny Widgets, which is basically a collection of extra components to enhance</a> <a data-t="181">your Shiny applications.</a> <a data-t="182">The repository contains a big selection of the components that are usually nicely</a> <a data-t="187">documented with little code snippets, such that you can copy and paste the code and adjust it</a> <a data-t="193">for your own needs.</a> <a data-t="194">Some of my favorite elements are, for example, picker input, those fancy checkbox button,</a> <a data-t="200">or horizontal multiple choice selector.</a></p>

<p><a data-t="204">Then we have small but rather powerful package, Shiny CSS Loaders, that helps you with</a> <a data-t="211">adding loading animations to the output instead of greening them up.</a> <a data-t="216">So I mentioned to you already that Shiny uses this power of R for data analysis, but</a> <a data-t="223">certain actions can take a while, so it's better to provide our user with some feedback that</a> <a data-t="229">there's something happening in the background.</a> <a data-t="232">So you can do it with a simple one-function call with Spinner that adds to your Shiny</a> <a data-t="239">those little animations.</a></p>

<p><a data-t="242">OK, and I wouldn't be myself if I haven't mentioned about some packages that help you style</a> <a data-t="247">your apps.</a> <a data-t="247">As in Appsilon, we care a lot about how the apps look.</a> <a data-t="252">So starting from the left, we have BS4Dash that provides you with bootstrap for admin LTE</a> <a data-t="260">skin for your dashboards.</a> <a data-t="262">Then we have Shiny Mobile that helps you create mobile-ready Shiny apps that look almost</a> <a data-t="268">like native apps for iOS and Android.</a> <a data-t="271">And then on the right, we have our own Shiny Semantic that helps you to attach an</a> <a data-t="276">alternative look to your Shiny applications.</a></p>

<p><a data-t="280">OK, and I mentioned that I just scratched the surface here, so there is even more fantastic</a> <a data-t="287">Shiny extensions that you can find on this Shiny Awesome repository that I discovered</a> <a data-t="292">recently.</a> <a data-t="303">And this little repository summarizes and categorizes various Shiny extensions</a> <a data-t="307">such that you can always find something that you need there.</a> <a data-t="310">I encourage you to check it out.</a></p>

<h3>How to contribute to open source Shiny</h3>

<p><a data-t="315">OK, but using all those Shiny extensions is just like one way how you can take advantage of</a> <a data-t="320">Shiny open sourceness, but you can help yourself with building those tools.</a></p>

<p><a data-t="323">So first of all, you can start from something as simple as reporting a bug or bringing some</a> <a data-t="330">new idea to the table by creating a new issue on GitHub repository or emailing an author</a> <a data-t="338">of the code.</a> <a data-t="338">But remember to be respectful with that, as very often open source code is</a> <a data-t="343">created by enthusiasts, or as in our case, this is something that we do outside of our</a> <a data-t="349">daily work with clients.</a></p>

<p><a data-t="351">Then, if you feel comfortable enough actually to fix some bug or contribute with a new</a> <a data-t="359">idea to a certain package, feel free to do some coding and create a new pull request.</a> <a data-t="365">But remember first about reading and contributing guidelines, as they usually contain</a> <a data-t="371">useful information with what to do or not to do on a given repository.</a> <a data-t="376">Remember about testing all of your changes so that you see that you didn't break all the</a> <a data-t="382">previous functionalities of the code and always ask politely for a review.</a> <a data-t="387">It's good to have another pair of eyes to check the quality of your code.</a></p>

<h3>Creating your own package</h3>

<p><a data-t="393">OK, but now if you still see that there is no solution to your problem available, then</a> <a data-t="402">you can move on and think about creating your own package.</a> <a data-t="406">And remember that for starters, it doesn't really need to be perfect.</a> <a data-t="410">Some solution is better than no solution whatsoever.</a> <a data-t="414">And always you may find someone in the community who would be interested in collaborating and</a> <a data-t="420">moving the project forward if you don't really have time to do all the coding on your own.</a></p>

<p><a data-t="424">And then a couple of tips from me.</a> <a data-t="428">Remember about adding tests to your package from the early beginning, as it will really spare you</a> <a data-t="434">so much troubles at the later stage of the development.</a> <a data-t="438">Remember about documenting your code.</a> <a data-t="441">For you, it might be quite obvious how to use your own functions, but not necessarily for the</a> <a data-t="445">users of your package.</a> <a data-t="447">And recently, it became a good practice to add interactive examples to your repository such that</a> <a data-t="455">users can test what this package actually can help them with before they even download it.</a> <a data-t="462">And then comes the fun part.</a> <a data-t="464">So remember about publishing it, sharing with others to get credits and acknowledgements.</a> <a data-t="469">It's a really satisfying feeling.</a></p>
