<p><a data-t="265">How did, I'm so curious how Apache factored in too.</a> <a data-t="268">Like I never, so I like know the name Apache and obviously a lot of the Apache, but I'm curious what was kind of the benefits you saw in the strategy.</a></p>

<p><a data-t="278">The Apache Software Foundation or the ASF.</a> <a data-t="281">So it's one of the best known open source foundations.</a> <a data-t="285">They provide like a home for open source projects.</a> <a data-t="289">They provide like marketing and some developer infrastructure, legal protections.</a> <a data-t="294">You know, they manage like trademark and copyright and that sort of thing, code licensing.</a> <a data-t="299">They were started in the 1990s when the world of open source was very different.</a></p>

<p><a data-t="303">So Linux was only a few years old at that time and everyone was freaked out about getting sued by Oracle or Sun or whoever were the major, you know, players in that day.</a> <a data-t="313">Because open source was just really as a thing was, was the free software movement.</a> <a data-t="317">It was still, still very new.</a> <a data-t="318">But as the ASF evolved, it has become essentially a neutral home for open source projects that have corporate contributors that are looking to establish a neutral, like a Switzerland like neutral ground for governance.</a></p>

<p><a data-t="334">So that contributors can develop the project in a way that is open and transparent and to try to at least as much as possible mitigate some of their conflicts of interest.</a> <a data-t="345">And so, you know, so when a company is looking to develop a project but to give up enough control where that maybe even their direct competitors can join as contributors.</a> <a data-t="358">And, you know, you don't want to be looking around and say like what's your motivation or like what are you hiding?</a> <a data-t="363">Like, you know, what's going on in these backroom discussions?</a></p>

<p><a data-t="366">So in the ASF, the rule is that you can't have, you can't make decisions in private or in backroom discussions.</a> <a data-t="374">Like if you want to make a decision, you have to have the discussion openly and in public in a way where everybody can participate.</a> <a data-t="381">And so this, this helps level the playing field.</a> <a data-t="384">But it also helps people who maybe don't work at your company.</a> <a data-t="387">It gives them visibility into how you're thinking, what you're doing, why you're doing it and gives them a way to also get involved.</a></p>

<p><a data-t="395">And so as a way to build community, I think the ASF has played, played a pretty, you know, pretty important role.</a></p>

<p><a data-t="422">So ASF is a combination of a permissive license that's corporate friendly.</a> <a data-t="427">So the Apache 2.0 license includes, you know, do whatever you like with the software, but also don't sue us.</a> <a data-t="433">And like don't get, don't get software patents and then sue us over the software patents.</a> <a data-t="437">So it has protections like for all of those, all of those things to make the corporate, you know, pointy haired lawyers happy that their bases are covered with respect to IP.</a></p>

<p><a data-t="449">But yeah, it's like, but beyond the licensing, there's just this community, community over code is like the motto of the ASF.</a> <a data-t="457">Like this sort of community first mindset of like how you communicate, how you make decisions, how you treat each other, like having a code of conduct, being respectful.</a> <a data-t="466">And, but how you, how you treat each other is also like, you know, that, that you treat people with respect in terms of like, you know, you don't just show up and like try to steamroll a pull request or say like, you know, I'm the authority on this matter.</a> <a data-t="439">And this is the way that I think it should be done.</a> <a data-t="461">You actually have to argue from first principles why your approach to the problem is, you know, is the right one.</a></p>

<h3>Building the data explorer</h3>

<p><a data-t="790">So like a data viewer is like a really fast kind of like glimpse into your data that you can like filter.</a> <a data-t="815">And so we, as we, you know, initially brainstormed on it, like because RStudio has a data viewer component.</a> <a data-t="822">So when you have a data frame loaded up in RStudio, you can click on the data frame in the variables pane, and it will pop open this data viewer window.</a> <a data-t="833">It has some basic filtering capabilities.</a> <a data-t="835">It has some search capabilities.</a> <a data-t="837">But so we started from that place.</a></p>

<p><a data-t="838">We're like, well, we'd like to essentially have table stakes, like deliver what exists in RStudio, but also sky's the limit.</a> <a data-t="846">What else can we build?</a> <a data-t="847">What have we always dreamed of?</a> <a data-t="848">What have we always dreamed of having?</a></p>

<p><a data-t="850">And so, you know, for me, like so often I've found myself looking at the data while I'm writing code to clean it and manipulate it.</a> <a data-t="860">And so I wanted to have the thing that I could apply selective filters, or I could sort, or I could search, but that would live update while I'm writing code and modifying the data set.</a> <a data-t="874">And so I wanted that like live update experience.</a> <a data-t="877">I want something that has really smooth, infinite scroll, both horizontally and vertically.</a> <a data-t="882">I've often found myself with these like massive, you know, 10,000 column or 100,000 column data sets.</a> <a data-t="887">I'm like, why can't we have a data explorer that is infinitely scrollable horizontally and vertically?</a></p>

<p><a data-t="899">And of course, you know, seeing some of the innovations that have come out of the business intelligence world, having things like, you know, spark lines, you know, summary statistics, like being able to have like the statistical visualizations like just immediately available.</a> <a data-t="915">We're really influenced by, you know, I'm a big fan of this tool called RealData.</a> <a data-t="920">So it's Mike Driscoll's kind of latest company powered by DuckDB.</a></p>

<p><a data-t="924">But I also want to like really basic stuff like, you know, when you're in data, if you're in VS Code and I'm sure you've been in VS Code and you have like a data file, like shouldn't you be able to just open the data file and look at it?</a> <a data-t="936">So I wanted that like instant preview experience.</a> <a data-t="939">And so like we use DuckDB to build that.</a> <a data-t="943">You can just click on a parquet file and it opens and everything works.</a> <a data-t="948">And it's, you know, it's a really big parquet file.</a> <a data-t="951">It may not be instantaneous, but it's really snappy.</a></p>

<p><a data-t="956">And so I'm proud of what we build.</a> <a data-t="958">And it's a nice feeling, you know, to cross over, to build something that's at the intersection of like backend engineering because there's clearly like a lot of like, you know, you need to have knowledge of the internals of like how Pandas works and how Polars works.</a> <a data-t="1012">And our data frames and how DuckDB works to create this thing that you can deliver that snappy kind of intuitive thing, something that's easy to use but also is fast.</a></p>
