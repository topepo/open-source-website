<h3>Reframing ggplot internals as data wrangling</h3>

<p><a data-t="302">So here is the outline for the rest</a> <a data-t="302">of my talk.</a> <a data-t="308">So first I'll introduce this reframing of ggplot internals as data wrangling, which</a> <a data-t="308">will also provide context and motivation for my package ggtrace.</a> <a data-t="313">And then I'll demonstrate</a> <a data-t="313">how we can put this idea into practice using a simple example, using functions from ggtrace,</a> <a data-t="319">starting with an example of a bar plot, and then returning to our box plot annotation</a> <a data-t="323">problem and then building up the solution from scratch.</a></p>

<h3>Introducing ggtrace</h3>

<p><a data-t="538">So ggtrace is a package that allows us, the users, to drop</a> <a data-t="538">inside, like parachute down this internal pipeline at any point we want and intercept</a> <a data-t="542">the data at that step and interact with it using the kind of data wrangling tools that</a> <a data-t="547">we're already familiar with.</a> <a data-t="552">So ggtrace comes with a family of workflow functions in the</a> <a data-t="552">form of ggtrace action value, which all take three arguments, the ggplot object, the ggproto</a> <a data-t="559">method, and when to interact with that method.</a> <a data-t="565">So this talk will showcase just two functions</a> <a data-t="565">from the inspect workflow, ggtrace inspect args and ggtrace inspect return, which lets</a> <a data-t="570">us take a snapshot of a layer's data as it goes in and out of ggproto methods.</a></p>

<p><a data-t="576">So for</a> <a data-t="576">example, if we want to look at the state of a layer's data in the before stat stage, then</a> <a data-t="582">we can use ggtrace inspect args to intercept the data that was passed into layer compute</a> <a data-t="587">statistic when it is first called for our plot, which gives us the box plot layers after</a> <a data-t="592">stat data.</a> <a data-t="597">And then likewise, if we want to look at the layers after stat, or sorry, that</a> <a data-t="597">was the before stat data.</a> <a data-t="601">If we want to look at the after stat data, we need to use ggtrace</a> <a data-t="601">inspect return instead to intercept the data when it is returned by that same method, which</a> <a data-t="606">gives us the box plot layers after stat data.</a></p>
