<h3>WebAssembly and <a href="/software/shinylive/">shinylive</a></h3>

<p><a data-t="2261">Another lesson that we've learned, which is that this is WebAssembly is very, very cool.</a> <a data-t="2268">So, WebAssembly is what makes shinylive possible.</a> <a data-t="2271">And so, if you don't know what shinylive is, I'll give you a really brief overview.</a> <a data-t="2275">So, it is Shiny, currently Shiny for Python, without a server running Python.</a> <a data-t="2282">So, normally when you deploy a Shiny application, you need a server that runs Python or R.</a> <a data-t="2289">And then that serves the application.</a> <a data-t="2291">And the user loads their web browser and they connect to that server.</a></p>

<p><a data-t="2297">With shinylive, Python runs inside of the browser itself because it's compiled to WebAssembly and browsers can run programs that are compiled to WebAssembly.</a> <a data-t="2309">So, you don't need a separate server running R or Python.</a> <a data-t="2313">And this allows for deployment on static web hosts, which can scale to extreme amounts of traffic for extremely low cost.</a> <a data-t="2326">So, that's great if you have an application that for certain classes of applications, if you need to handle a lot of traffic and they don't have some certain needs, which I'll talk about in just a minute, shinylive is really great for that.</a></p>

<p><a data-t="2343">So, it's not appropriate for all use cases.</a> <a data-t="2345">Like, you can't keep any secrets.</a> <a data-t="2347">If somebody can run your application on their computer, you know, you don't want to put any, like, secret authorization tokens or anything like that in there because they'll have access to them.</a> <a data-t="2355">You can't make secure database connections.</a> <a data-t="2357">And it's not, you know, it's not necessarily good for large datasets because you might not want to transfer large amounts of data to each user's browser.</a></p>

<p><a data-t="2394">But so, even though it has those limits, it's still the really good thing about it is that you can use the same knowledge of Shiny to create shinylive applications or traditional Shiny applications, whichever is appropriate.</a> <a data-t="2405">It's the same, you know, you're just writing a Shiny application.</a> <a data-t="2409">There's just a few different things that you can do between regular Shiny and shinylive applications.</a> <a data-t="2412">So, you can deploy it whichever way is appropriate.</a> <a data-t="2417">And this gives you much more range in the space of, like, the web applications that you can create.</a></p>
