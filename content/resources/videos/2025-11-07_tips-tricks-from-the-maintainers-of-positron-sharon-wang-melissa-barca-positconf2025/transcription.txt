<h3>Pairing with Positron Assistant</h3>

<p><a data-t="595">With that, let's move on to talking about</a> <a data-t="598">effectively pairing with Positron Assistant.</a> <a data-t="601">Positron Assistant is an AI client</a> <a data-t="604">that provides LLM integration with Positron</a> <a data-t="607">through chat and inline completions.</a> <a data-t="610">It's great for helping with debugging, refactoring,</a> <a data-t="613">or getting suggestions for the next steps</a> <a data-t="615">that you should take in your project.</a></p>

<p><a data-t="617">Two concepts that you should really understand</a> <a data-t="620">to make the best use of Assistant are context and tools.</a> <a data-t="624">Context refers to the information</a> <a data-t="627">that Positron Assistant provides to the LLM alongside your prompt.</a> <a data-t="631">Positron Assistant provides context about files and folders,</a> <a data-t="635">and it also provides your in-memory data from your session.</a> <a data-t="640">So this includes your loaded variables, plots pane,</a> <a data-t="643">active console sessions, and session state.</a> <a data-t="647">With your permission, Assistant can even run code</a> <a data-t="649">directly in your active session.</a></p>

<p><a data-t="653">You can take advantage of this context</a> <a data-t="655">by asking Assistant to plot data frames for you</a> <a data-t="658">or tweak things like the title or access labels on your plots.</a></p>

<p><a data-t="664">You can view what context Assistant is passing to the LLM</a> <a data-t="667">in your chat context.</a> <a data-t="669">So here we ask Assistant about the current plot</a> <a data-t="672">with the console session enabled as context.</a> <a data-t="675">Assistant has access to review the plot</a> <a data-t="677">as well as the code that generated the plot.</a> <a data-t="680">And you can see this because the open eye icon</a> <a data-t="686">is open next to the context.</a></p>

<p><a data-t="690">What about closing the eye? What if we want it closed?</a> <a data-t="694">You can close the eye.</a> <a data-t="695">You can control exactly what context Assistant sends to the LLM.</a> <a data-t="699">So whether you have security concerns</a> <a data-t="701">or the context is just irrelevant to your prompt,</a> <a data-t="704">you can close your eye by clicking the eye icon.</a> <a data-t="710">So in this example, we've removed access</a> <a data-t="712">to the console session context.</a> <a data-t="715">Assistant is aware that you have a plot,</a> <a data-t="718">but it cannot see the plot</a> <a data-t="719">nor the code that generated the plot.</a></p>

<p><a data-t="724">Providing relevant contextual information</a> <a data-t="726">can yield better results from AI models,</a> <a data-t="729">so there may be even more context</a> <a data-t="731">that you want to provide to Assistant.</a> <a data-t="734">You can do this by clicking add context</a> <a data-t="737">to attach additional context such as files and folders,</a> <a data-t="740">and you can even add additional interpreter sessions as well.</a></p>

<p><a data-t="746">Much of this context is provided to the LLM</a> <a data-t="748">through tool calling.</a> <a data-t="750">So for example, if you ask a question</a> <a data-t="752">about the current plot,</a> <a data-t="754">Positron Assistant will call the get plot tool,</a> <a data-t="757">and that will allow the active plot image</a> <a data-t="759">to be included with your question.</a> <a data-t="762">Without the get plot tool,</a> <a data-t="763">the AI model isn't actually able to see your image.</a></p>

<p><a data-t="769">If you want to check if Positron Assistant has called a tool,</a> <a data-t="772">you can keep an eye out for this green check mark</a> <a data-t="775">followed by some text describing an action.</a> <a data-t="778">So in this case, we have viewed the active plot.</a> <a data-t="783">By default, Positron will call tools</a> <a data-t="785">that seem relevant to your prompt,</a> <a data-t="788">but it can be helpful to attach tools as context.</a> <a data-t="791">So if you want Assistant to utilize specific tools</a> <a data-t="795">with your prompt,</a> <a data-t="796">you can click on add context in the chat,</a> <a data-t="799">and then tools,</a> <a data-t="800">and you'll be able to pick a tool to include as your context.</a></p>

<p><a data-t="806">Let's now take a look at a couple other tools</a> <a data-t="808">available with Positron Assistant.</a> <a data-t="811">The get an R help page tool</a> <a data-t="813">looks at your current R session and loaded packages</a> <a data-t="816">and retrieves help and vignettes content for that package.</a> <a data-t="821">The AI model is then provided with this content</a> <a data-t="824">so that you can get guidance</a> <a data-t="826">directly based on the package's documentation.</a></p>

<p><a data-t="831">Another tool Positron Assistant provides</a> <a data-t="833">is the get changed files tool,</a> <a data-t="835">which retrieves your file modifications from Git.</a> <a data-t="839">If you want to use this tool,</a> <a data-t="840">include hashtag changes in your message to Assistant.</a> <a data-t="845">So for example, here we've asked Assistant,</a> <a data-t="848">tell me about the hashtag changes I've made.</a></p>

<p><a data-t="851">So if Assistant can understand my changes,</a> <a data-t="854">does that mean it can just write the commit message for me?</a> <a data-t="858">Yeah, you could ask Assistant,</a> <a data-t="860">generate a commit message for my hashtag changes in the chat,</a> <a data-t="864">or here we've added axolotl and kangaroo to our code,</a> <a data-t="869">and from the source control panel,</a> <a data-t="871">we've clicked the sparkles button in the commit message box,</a> <a data-t="875">and Positron Assistant has generated</a> <a data-t="877">the commit message for us.</a></p>

<p><a data-t="881">Calling back to attach in context,</a> <a data-t="883">you can also select source control,</a> <a data-t="886">and this allows you to choose a specific commit</a> <a data-t="889">to include as context.</a> <a data-t="891">Sometimes, like, a lot of code can change in a commit</a> <a data-t="894">if you're, like, changing a bunch of files,</a> <a data-t="897">10,000 lines have changed,</a> <a data-t="898">or maybe it's just code that you're unfamiliar with,</a> <a data-t="901">so you can attach that commit as context</a> <a data-t="903">and then ask Assistant to help you understand</a> <a data-t="906">or summarize those changes.</a></p>
