<h3>Les deux goûts de TidyEval : sélection et action</h3>

<p><a data-t="152">Nous allons parler des deux goûts de TidyEvaluation que nous avons dans </a><a href="/software/tidyverse/">Tidyverse</a><a data-t="152">.</a> <a data-t="158">Nous avons ici quatre fonctions de </a><a href="/software/dplyr/">dplyr</a><a data-t="158">, Mutate, GroupBy, Select et Filter.</a> <a data-t="165">Elles ressemblent un peu à celles-ci.</a> <a data-t="168">Elles sont utilisées de la même manière.</a> <a data-t="170">Mais il y en a une qui est différente des autres.</a> <a data-t="173">C'est Select.</a> <a data-t="174">Select prend une sélection.</a> <a data-t="175">Toutes les autres fonctions prennent des actions.</a></p>

<p><a data-t="180">Ce que nous voulons par action, c'est que nous obtenions ce blueprint</a> <a data-t="184">et qu'on l'utilise pour créer de nouveaux vecteurs.</a> <a data-t="187">Ensuite, nous modifions le cadre de données.</a> <a data-t="191">Par contre, avec la sélection, quelque chose de différent s'est produit.</a> <a data-t="195">Nous avons matché les colonnes à leur position dans le cadre de données</a> <a data-t="200">et nous avons réorganisé le cadre de données.</a> <a data-t="203">Il n'y a pas de création de vecteurs.</a></p>

<p><a data-t="207">Dans les actions, quand nous obtenons le blueprint,</a> <a data-t="211">c'est un code R normal.</a> <a data-t="215">Mais dans la sélection, il y a quelques fonctions spéciales.</a> <a data-t="219">Pour commencer, la fonction C et les opérateurs de colonnes et de minus</a> <a data-t="225">comprennent les positions et les noms des colonnes.</a> <a data-t="229">C'est ainsi que vous pouvez très facilement spécifier un rang de colonnes.</a> <a data-t="233">Par exemple, avec l'opérateur de colonnes.</a></p>

<p><a data-t="237">Vous pouvez aussi utiliser les opérateurs de sélection,</a> <a data-t="240">qui sont très utiles car ils connaissent les variables</a> <a data-t="243">avec lesquelles vous travaillez actuellement.</a> <a data-t="246">Par exemple, vous pouvez sélectionner toutes les colonnes</a> <a data-t="249">qui finissent avec une suffixe,</a> <a data-t="251">ou vous pouvez les matcher contre un ragaphone.</a> <a data-t="253">C'est très utile quand vous avez des frames de données bien organisées</a> <a data-t="257">avec des noms de colonnes consistants avec des suffixes et des préfixes.</a> <a data-t="260">Cela fait très facile de sélectionner les colonnes.</a></p>

<p><a data-t="263">Parfois, les verbes d'action et de sélection</a> <a data-t="266">apparaissent fonctionner de la même façon.</a> <a data-t="269">Ici, nous avons Select Height et Transmute Height.</a> <a data-t="273">Et nous obtenons exactement le même résultat.</a> <a data-t="276">Mais c'est utile de savoir si vous avez un verbe d'action</a> <a data-t="280">ou un verbe de sélection,</a> <a data-t="282">car si vous changez un peu l'entrée,</a> <a data-t="284">ici, je supplie un.</a></p>

<p><a data-t="287">Pour le verbe de sélection,</a> <a data-t="290">puisqu'il comprend les positions des colonnes,</a> <a data-t="292">il signifie sélectionner la première colonne.</a> <a data-t="294">Pour le verbe d'action,</a> <a data-t="296">il créera un nouveau vecteur de votre entrée.</a> <a data-t="299">Dans ce cas-ci, on le recycle pour la taille de la frame de données</a> <a data-t="303">et on obtient une colonne d'un seul.</a></p>

<h3>Le cas de Group By</h3>

<p><a data-t="307">Qu'est-ce qu'il y a de Group By ?</a> <a data-t="309">C'est un cas intéressant, car la plupart des utilisateurs</a> <a data-t="312">pensent que c'est un verbe de sélection.</a> <a data-t="315">Mais en réalité, c'est un verbe d'action.</a> <a data-t="318">Si vous essayez de passer un groupe de sélection,</a> <a data-t="322">vous obtiendrez cette erreur</a> <a data-t="324">où les variables ne sont pas trouvées.</a></p>

<p><a data-t="328">Nous l'avons fait un verbe d'action</a> <a data-t="331">car c'est très utile pour créer</a> <a data-t="334">ces nouveaux vecteurs sur la flèche</a> <a data-t="337">où vous discriminez entre les groupes.</a> <a data-t="339">Dans cet exemple,</a> <a data-t="341">je groupe par les observations</a> <a data-t="345">dont la taille est au-dessus d'un certain threshold.</a> <a data-t="348">Ensuite, je les résume sur la flèche</a> <a data-t="350">et j'obtiens un bon tableau de résumés.</a></p>

<p><a data-t="354">Qu'est-ce que vous pouvez faire</a> <a data-t="356">si vous avez un verbe d'action comme Group By</a> <a data-t="359">et que vous voulez vraiment passer une sélection ?</a> <a data-t="362">Vous pouvez utiliser les variants At, les verbes scopés.</a> <a data-t="367">Nous avons MutateAt, SummarizeAt, GroupByAt</a> <a data-t="371">et un tas d'autres.</a></p>

<p><a data-t="374">Ils prennent une sélection</a> <a data-t="376">à l'intérieur de cette petite fonction Vars</a> <a data-t="378">qui est une fonction qui fait le coding,</a> <a data-t="381">qui obtient le blueprint et qui le passe au GroupByAt.</a> <a data-t="384">Au-delà de ça, vous pouvez passer une sélection.</a> <a data-t="386">Ici, dans cet exemple, nous groupons</a> <a data-t="388">par toutes les colonnes qui finissent avec couleur.</a></p>

<h3>Créer des fonctions TidyEval avec le dot</h3>

<p><a data-t="393">Voyons donc comment nous pouvons créer des valeurs idéales.</a> <a data-t="398">Il y a trois défis ici.</a> <a data-t="400">D'abord, il y a la nécessité de coder les entrées</a> <a data-t="405">pour obtenir les sélections et les actions des utilisateurs.</a> <a data-t="408">Ensuite, il y a la nécessité de modifier ces sélections ou actions</a> <a data-t="412">et de les passer vers d'autres valeurs idéales.</a></p>

<p><a data-t="417">La deuxième est plus complexe que la première.</a> <a data-t="421">C'est là que vous devez comprendre toute la théorie idéale.</a> <a data-t="424">Vous devez utiliser les outils idéaux comme Bang Bang.</a> <a data-t="427">Nous ne parlons pas de ça.</a> <a data-t="429">Aujourd'hui, nous allons nous concentrer</a> <a data-t="431">sur les premières et les troisièmes défis.</a> <a data-t="434">Pour ce faire, nous allons passer le dot.</a></p>

<p><a data-t="440">Ici, nous avons la fonction la plus simple</a> <a data-t="443">que vous pouvez créer en passant le dot.</a> <a data-t="447">Nous prenons typiquement un frame de données,</a> <a data-t="449">puis nous prenons le dot,</a> <a data-t="451">et nous le passons à une autre fonction idéale.</a> <a data-t="455">Cela signifie que tous les arguments</a> <a data-t="458">que l'utilisateur apporte à travers le dot</a> <a data-t="461">sont maintenant la responsabilité de nos fonctions</a> <a data-t="464">dans dplyr, Tidyer ou les autres grammaires Tidyverse</a> <a data-t="467">pour faire le codage et créer les actions ou les sélections.</a> <a data-t="471">Vous n'avez donc pas à faire quelque chose</a> <a data-t="473">et vous réservez les deux défis 1 et 3 en même temps.</a></p>

<p><a data-t="479">C'est un peu limité, mais ça peut être très utile</a> <a data-t="482">et nous allons voir trois exemples.</a> <a data-t="485">D'abord, nous allons créer un verbe de sélection avec dplyr.</a> <a data-t="489">Ensuite, nous allons voir comment nous pouvons transformer</a> <a data-t="492">ce verbe de sélection en un verbe d'action.</a> <a data-t="495">Ensuite, nous allons utiliser Tidyer pour complexifier le pipeline.</a></p>

<p><a data-t="502">Comme je l'ai mentionné, la plupart des verbes de dplyr</a> <a data-t="504">ont une suffixe variante avec un at,</a> <a data-t="507">et ils prennent des sélections,</a> <a data-t="508">ce qui peut être très utile pour voir</a> <a data-t="510">si vous voulez créer un verbe de sélection</a> <a data-t="512">en passant le dot.</a></p>

<p><a data-t="516">Deux d'entre eux sont un peu différents des autres</a> <a data-t="518">parce qu'ils prennent une sélection,</a> <a data-t="520">mais ils prennent aussi une fonction.</a> <a data-t="524">Ils appliquent la fonction à chaque colonne</a> <a data-t="527">qui fait partie de la sélection.</a> <a data-t="528">C'est un peu comme l'application L</a> <a data-t="530">ou la mappage de la fonction par.</a></p>

<p><a data-t="534">Ça marche comme ça.</a> <a data-t="535">Vous passez un frame de données,</a> <a data-t="537">vous passez une sélection, c'est comme ça,</a> <a data-t="539">et puis une fonction qui peut aussi être par lambda,</a> <a data-t="542">avec la notation de la formule.</a> <a data-t="546">Ça applique le corps de la fonction</a> <a data-t="548">à chaque colonne de la sélection,</a> <a data-t="550">et c'est comme si vous aviez créé</a> <a data-t="552">cet enregistrement plus long.</a></p>

<h3>Rassembler les résultats avec <a href="/software/tidyr/">TidyR</a> Gather</h3>
