<h3>What makes dplyr powerful</h3>

<p><a data-t="155">I think a big question is what dplyr is really doing that's so useful.</a> <a data-t="161">And to really understand, we have to go back to Hadley's 2014 talk where he introduced dplyr.</a> <a data-t="167">And he mentions that analysts sort of have two bottlenecks,</a> <a data-t="171">a cognitive bottleneck and a computational bottleneck.</a> <a data-t="174">The computational one's the one we think of most often,</a> <a data-t="178">which is as the data becomes bigger, it takes longer to run the code.</a></p>

<p><a data-t="182">But intriguingly, Hadley mentions he thinks a lot about the cognitive bottleneck,</a> <a data-t="186">which is how should we think about the data and describe what we want to do to the data or code.</a> <a data-t="193">And dplyr aims at this cognitive bottleneck to help people sort of focus their thoughts</a> <a data-t="199">and to give people strategies for data analysis.</a> <a data-t="202">It does this by kind of slimming down, taking this big space and slimming down the options that people have.</a> <a data-t="209">So there are five simple verbs.</a> <a data-t="212">And all of these critically can combine with an operator group by.</a> <a data-t="216">All of these take a data frame and return a data frame.</a> <a data-t="220">And he mentions that overall, it's a very constrained design,</a> <a data-t="224">especially compared to his previous tool, Plyar.</a></p>

<blockquote><a data-t="182">But intriguingly, Hadley mentions he thinks a lot about the cognitive bottleneck,</a> <a data-t="186">which is how should we think about the data and describe what we want to do to the data or code.</a></blockquote>

<p><a data-t="228">So Siuba aims to sort of let people capture those dplyr-like thoughts and keep them</a> <a data-t="235">while writing dplyr syntax in Python and then doing the computation in Pandas.</a> <a data-t="242">So the gist is Siuba lets you transfer your thoughts from R to Python.</a></p>

<h3>Three roadblocks when moving from dplyr to pandas</h3>

<p><a data-t="251">All right, so I'm going to try to show three motivating cases where it can be tricky to transfer dplyr thoughts to Pandas.</a> <a data-t="259">Three roadblocks that people hit.</a> <a data-t="261">And I know it's tempting to see dplyr in Pandas. It's pretty similar.</a> <a data-t="265">So in this example, they're doing roughly the same thing.</a> <a data-t="268">And they're using basically the same code.</a> <a data-t="270">So it's a group summarize.</a> <a data-t="272">We're calculating the average of a column.</a> <a data-t="275">And we're renaming the result average HP.</a> <a data-t="278">And we get roughly the same output.</a></p>

<p><a data-t="282">But it's also worth noting, if you take this somewhere like Twitter or look around,</a> <a data-t="287">you'll see some pretty scathing reviews.</a> <a data-t="289">So dplyr has forever ruined Pandas for me.</a> <a data-t="292">Or Chelsea notes, every picture shows Pandas in chaos.</a> <a data-t="298">And I think it's worth noting these people, I think they like Pandas.</a> <a data-t="301">So this isn't a straight critique of Pandas.</a> <a data-t="303">But it can be really frustrating when you try to go from one tool and the way it structures thinking to another.</a></p>

<p><a data-t="312">And so I would say these three cases try to bring that to the surface, what's going on.</a> <a data-t="318">The first roadblock is select.</a> <a data-t="320">So select is really a very basic action.</a> <a data-t="323">You can choose columns.</a> <a data-t="325">You can drop columns.</a> <a data-t="326">You might match certain columns or rename a column.</a> <a data-t="329">And the trick is that Pandas uses different names for all these actions.</a> <a data-t="334">So there are four different methods, one for each action.</a></p>

<p><a data-t="338">The other thing is that sometimes they have kind of funky arguments you need to pass.</a> <a data-t="343">So you might have to say axis equals one to say, oh, I want to do this to the columns.</a> <a data-t="347">Or you might have to say columns equals.</a> <a data-t="349">Those are kind of different ways of saying the same thing.</a> <a data-t="353">The last is that sometimes you have to use sort of intense programming constructs, like a lambda function,</a> <a data-t="359">if you want to do things like match on a string.</a> <a data-t="363">In contrast, dplyr uses the same verb for all these actions, select.</a> <a data-t="368">And it's really easy to change between what you're doing,</a> <a data-t="372">to go from keeping a column to dropping a column, or even to combine these actions together.</a></p>

<p><a data-t="378">And so this is a case, I think, of where sort of changing what you do requires changing gears, right,</a> <a data-t="384">and shifting between these different functions and methods in Pandas.</a> <a data-t="389">The second example is, I would say, groupby.</a> <a data-t="393">And this is really, I would say grouping is really a critical activity in data analysis in dplyr.</a></p>

<p><a data-t="400">And the Pandas docs describe some translations from R to Pandas.</a> <a data-t="405">And it looks pretty straightforward, at least for these cases.</a> <a data-t="408">But actually, I would say groupby in dplyr and Pandas use radically different underlying grammars.</a> <a data-t="416">So in this example, on the left, I'm showing a filter in Pandas or a mutate.</a> <a data-t="423">And note there are two ways of doing it, or multiple ways, I should say.</a> <a data-t="428">So you can use query, that's the top left, or eval on the top right,</a> <a data-t="433">or an alternative way to filter on the bottom left is these blocks,</a> <a data-t="437">or to mutate a sign on the bottom right.</a></p>

<p><a data-t="442">Critically, we can ask, what happens when we try to groupby before these operations?</a> <a data-t="449">And the answer is that none of them work.</a> <a data-t="453">And the trick is a groupdata in Pandas, a dataframe groupby,</a> <a data-t="459">actually doesn't have even any of these methods on it to perform these operations,</a> <a data-t="465">except in one case where the operation just sort of fails.</a> <a data-t="470">And this is probably really surprising, I think, for dplyr users.</a> <a data-t="474">The trick ends up being you have to shift the groupby from outside of the verb to inside it.</a> <a data-t="481">So you basically have to move groupby into your column operations.</a> <a data-t="486">And again, this isn't necessarily bad,</a> <a data-t="488">and it means Pandas can still do the same computations as dplyr.</a> <a data-t="492">It's just, I think, surprising for dplyr users,</a> <a data-t="495">and for me challenging to do when I'm trying to analyze data quickly.</a></p>

<p><a data-t="500">The third roadblock, I'd say, is SQL.</a> <a data-t="502">So dplyr users, if you're like me, working in industry,</a> <a data-t="508">I lean on dplyr and </a><a href="/software/dbplyr/">dbplyr</a><a data-t="508"> being able to generate SQL heavily.</a> <a data-t="512">So you can just swap out your data source to SQL,</a> <a data-t="515">and dplyr will generate a SQL query for you.</a> <a data-t="519">In contrast, if you're using Pandas, you just need to switch to writing SQL.</a> <a data-t="524">And that's what I see happen quite a bit in industry is once it hits the database,</a> <a data-t="529">people just switch and start coding SQL.</a> <a data-t="531">So your computation changes, and how you code ends up changing as a result.</a></p>

<p><a data-t="537">So to recap, dplyr is really powerful.</a> <a data-t="541">It uses one verb, select, for all the cases for grabbing columns.</a> <a data-t="547">For groupby, you have this sort of really nice expressive syntax</a> <a data-t="552">where you can modify the full table action.</a> <a data-t="555">So you can have a grouped mutate or a grouped filter or group summarize.</a> <a data-t="561">In contrast, Pandas is computationally powerful,</a> <a data-t="565">and it's packed with options.</a> <a data-t="567">But that means that you might have to do a little bit more work up front</a> <a data-t="570">during the analysis.</a> <a data-t="572">And this makes it great for engineering,</a> <a data-t="574">but maybe a big challenge, I think, for exploratory data analysis</a> <a data-t="578">or quick analyses.</a></p>
