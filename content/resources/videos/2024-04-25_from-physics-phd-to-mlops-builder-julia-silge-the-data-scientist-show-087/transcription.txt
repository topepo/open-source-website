<p><a data-t="538">So the project that I work on now, it's called Vetiver.</a> <a data-t="543">And if you are like a perfume person or a candle person,</a> <a data-t="547">you may have heard that word, Vetiver.</a> <a data-t="548">So it's a stabilizing ingredient in perfumery.</a> <a data-t="552">So the idea is like in a perfume,</a> <a data-t="555">you might have like very,</a> <a data-t="556">lots of volatile kind of fragrances,</a> <a data-t="558">but Vetiver is one that's a stabilizing ingredient.</a> <a data-t="558">And so the metaphor here is that Vetiver, the project,</a> <a data-t="562">is for stabilizing your models.</a> <a data-t="564">For making your models might be these more,</a> <a data-t="566">these more volatile things,</a> <a data-t="569">but Vetiver helps you have confidence</a> <a data-t="571">in the reliability of the model.</a></p>

<p><a data-t="573">So it's a project for Python and for R.</a> <a data-t="575">And what it focuses on is you have a model trained</a> <a data-t="579">and what do you need to do to maintain it in production,</a> <a data-t="583">reliably and efficiently.</a> <a data-t="584">So the three main sort of tasks and approach</a> <a data-t="587">are versioning your model.</a> <a data-t="588">So that's, oh, I trained it this week and I got this result,</a> <a data-t="592">but I trained it last week and I got these results.</a> <a data-t="594">Like how do I reliably know where,</a> <a data-t="597">or keep those models versions really organized</a> <a data-t="599">in like a model registry.</a> <a data-t="601">So versioning is the first one.</a> <a data-t="602">The second one is deploying.</a> <a data-t="604">So that's that process of getting the model</a> <a data-t="606">out of the computational environment where you developed it.</a> <a data-t="611">Maybe think of that as like your laptop,</a> <a data-t="612">or maybe you might have developed it</a> <a data-t="614">in like a server kind of environment,</a> <a data-t="616">but you developed it somewhere</a> <a data-t="617">and then you need to lift it out</a> <a data-t="619">with all the like computational pieces that it needs</a> <a data-t="623">and you put it somewhere else</a> <a data-t="625">so that it can be in a production environment</a> <a data-t="627">where it's integrated into the infrastructure of your org.</a></p>

<p><a data-t="631">So picture that as like one way</a> <a data-t="636">that you can do that is with Docker.</a> <a data-t="637">There are other ways as well that Vetiver supports,</a> <a data-t="639">but that might be a good way to think of it.</a> <a data-t="642">So instead of it's on my laptop,</a> <a data-t="644">you're like, ah, it's in a Docker container</a> <a data-t="647">and is ready to make predictions</a> <a data-t="649">because we have captured all the software requirements.</a> <a data-t="652">So versioning is the first piece.</a> <a data-t="654">So think of like a model registry.</a> <a data-t="656">Deploying is the second piece.</a> <a data-t="658">So think about getting it off your laptop,</a> <a data-t="660">maybe like into a Docker container.</a> <a data-t="662">And then the last piece is model monitoring.</a> <a data-t="665">So this is, the model is in production.</a> <a data-t="667">It's serving predictions in whatever is appropriate</a> <a data-t="671">for your organization.</a> <a data-t="672">And the, but you need to know,</a> <a data-t="674">is the model performing as you expect it to?</a> <a data-t="677">Is, do you, is it starting to degrade?</a> <a data-t="681">Is it starting, are you having that sort of model drift</a> <a data-t="684">so that you need to address it?</a> <a data-t="685">So that third piece is the monitoring and maintenance piece.</a></p>

<h3>MLOps challenges and user research</h3>

<p><a data-t="716">So before we started deciding</a> <a data-t="719">exactly what would be the shape</a> <a data-t="721">of the MLOps tool we would build,</a> <a data-t="723">I did a round of user interviews.</a> <a data-t="726">I went to both people who were customers of Posit,</a> <a data-t="730">so these are people who use our professional products,</a> <a data-t="733">and also people in the open source ecosystem,</a> <a data-t="736">people who maybe are users of Scikit-learn</a> <a data-t="740">and our tidy models, or people who are more,</a> <a data-t="743">who are not our customers.</a> <a data-t="744">So I did rounds of user interviews</a> <a data-t="746">and I had an organized, like I designed an interview</a> <a data-t="748">so that we could really understand</a> <a data-t="750">what do people like about the tools</a> <a data-t="753">they're trying to use now?</a> <a data-t="754">What are the deficits that exist?</a></p>

<p><a data-t="772">Yeah, one thing that I feel like</a> <a data-t="772">was just hammered home to me,</a> <a data-t="775">both through my own interviews</a> <a data-t="776">and through the reading that I did,</a> <a data-t="778">in many organizations,</a> <a data-t="780">the people who are responsible for this</a> <a data-t="784">are the people who are also developing the model.</a> <a data-t="788">So in many places,</a> <a data-t="790">if you're someone who develops a model,</a> <a data-t="792">and by that I mean that process</a> <a data-t="793">of hyperparameter tuning, evaluation,</a> <a data-t="795">like how do I train the model?</a> <a data-t="797">They are often the people who also are responsible</a> <a data-t="801">for operationalizing that model.</a> <a data-t="804">But many of the tools that are out there for MLOps</a> <a data-t="807">are really built with a software engineer user in mind,</a> <a data-t="812">and don't so much really acknowledge</a> <a data-t="815">the iterative exploratory nature</a> <a data-t="818">of what data science is like,</a> <a data-t="820">what machine learning is like.</a></p>

<h3>Survey data challenges at Stack Overflow</h3>

<p><a data-t="1027">Yeah, it is, so working on the Stack Overflow</a> <a data-t="1027">developer survey was honestly quite stressful at the time,</a> <a data-t="1034">partly because it was fairly public.</a> <a data-t="1035">It was fairly public.</a> <a data-t="1036">People would pay quite a bit of attention to it,</a> <a data-t="1039">and it was enormous.</a> <a data-t="1041">On the years that I worked on the survey,</a> <a data-t="1043">it would have on the order of 70,000 responses,</a> <a data-t="1048">90,000 responses, and so it's an online opt-in survey.</a> <a data-t="1053">So in this way, it's different from surveys</a> <a data-t="1057">that you might hear about</a> <a data-t="1058">through a government statistical organizations,</a> <a data-t="1061">where they want a sample that is reflective</a> <a data-t="1064">of the population of that country,</a> <a data-t="1067">and they work very hard so that the people</a> <a data-t="1070">who take the survey are, they know how to map</a> <a data-t="1074">from the responses they get to the population.</a></p>

<p><a data-t="1222">So we tried to prioritize understanding differences.</a> <a data-t="1227">Yeah, I think that makes sense.</a></p>
