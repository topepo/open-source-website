<p><a data-t="0">Hi, my name is John Harmon.</a> <a data-t="0">I run the R4DS online learning community.</a> <a data-t="0">Our community operates primarily through a Slack team, and we invite R learners at all skill levels to join us</a> <a data-t="18">at r4ds.io slash join.</a> <a data-t="26">Among other things, we host book clubs for R-related books.</a> <a data-t="26">Whenever we start a new cohort for one of the book clubs, we need to find a time that works for</a> <a data-t="31">as many of the people interested in reading the book as we can.</a></p>

<p><a data-t="37">We used to use a generic free online app for everyone to sign up.</a> <a data-t="44">Everyone would type in their name, choose their available</a> <a data-t="44">times and days, and then set a password in case they wanted to edit their selections.</a> <a data-t="50">Sometimes people would sign up twice, not remembering that John who was signed up already</a> <a data-t="55">was them.</a> <a data-t="60">At least one time, someone had a typo in their name, so when they came back</a> <a data-t="60">to change their available hours, it counted as a new person.</a> <a data-t="64">And of course, sometimes</a> <a data-t="64">we legitimately had multiple people with the same name signing up, so I didn't have any</a> <a data-t="69">way to sort out who was who and how many duplicates we had.</a> <a data-t="76">And even when everything went smoothly,</a> <a data-t="76">I didn't have an easy way to connect the information in that app to their Slack user, so I couldn't</a> <a data-t="82">contact people directly to let them know that a cohort was being created at a time</a> <a data-t="87">they'd chosen.</a></p>

<h3>Building shinyslack</h3>

<p><a data-t="94">So I figured we already all have logins to the R4DS Slack.</a> <a data-t="94">What if we made our own app in </a><a href="/software/shiny-r/">Shiny</a><a data-t="94"> and then connected that to the Slack API?</a> <a data-t="100">Then the app could talk to our Slack and connect each user to their identity on R4DS.</a> <a data-t="108">I posted the idea on our Slack, and one of our users, Priyanka Gagnija, volunteered to make the initial app</a> <a data-t="114">minus the login, and then I focused on the login part.</a> <a data-t="120">I'd already done some work with Yanni CD on his Slack teams package, which allows you to talk to Slack through R.</a> <a data-t="126">You can find that and Yanni's whole Slackverse family of packages on GitHub at YanniCD.</a></p>

<p><a data-t="135">My goal was to take what I did there with interactive Slack authorization, make it work in Shiny,</a> <a data-t="142">and then wrap that up into an easy-to-use package.</a> <a data-t="148">So this is what that looks like.</a> <a data-t="155">If you aren't already logged in, it asks you to log in using Slack.</a> <a data-t="155">From that initial screen, you're taken to Slack's permission screen to connect to your Slack account, and then</a> <a data-t="159">it returns you to the Shiny app.</a> <a data-t="164">Here we just use the connection to identify the user, but I have another app where we use it to make sure the user has permission to see certain</a> <a data-t="168">messages, and I'm working on one that will help users ask better questions and then post</a> <a data-t="174">those questions as them.</a></p>

<h3>Using the package</h3>

<p><a data-t="179">So it works.</a> <a data-t="179">Now let's see how I did on the easy-to-use part of the goal.</a> <a data-t="186">You can install this from GitHub at R4DS slash Shiny Slack.</a> <a data-t="186">This is one of the ways we can launch a normal Shiny app.</a> <a data-t="191">We define a UI, define a server function,</a> <a data-t="191">and wrap them in Shiny's Shiny app function.</a> <a data-t="197">Now we'll use Shiny Slack to add a Slack login to that app.</a> <a data-t="204">We replace Shiny's Shiny app with Shiny Slack's Shiny Slack app.</a> <a data-t="211">We also need to give it the code that identifies our Slack team.</a> <a data-t="217">And that's it.</a> <a data-t="217">Well, kinda.</a></p>

<blockquote><a data-t="217">And that's it.</a> <a data-t="217">Well, kinda.</a></blockquote>

<p><a data-t="231">So you probably want to deploy your app to something like </a><a href="/software/shinyapps/">shinyapps</a><a data-t="231">.io, not just run the app locally.</a> <a data-t="235">And to do that, you'll need your own Slack app that has permission to</a> <a data-t="241">send people to apps hosted on your Shiny apps account.</a> <a data-t="247">When I first started working on this, setting up the Slack side of things was really complicated.</a> <a data-t="253">I was kind of dreading explaining this part.</a> <a data-t="258">I mean, I just did that thing showing you how easy it is to set up the Shiny side of this, and this part was going to ruin that.</a> <a data-t="264">But then something magical happened.</a> <a data-t="264">Slack added an option to create a Slack app from something called a manifest.</a> <a data-t="270">So I added a function over in Slack teams, create custom app to make that easy.</a> <a data-t="275">There's a vignette explaining it, but basically you call create custom app with a couple arguments and copy paste the result into Slack's app creation form.</a> <a data-t="286">And with that, you can wrap your Shiny app with Shiny Slack and integrate it with your Slack team.</a></p>
