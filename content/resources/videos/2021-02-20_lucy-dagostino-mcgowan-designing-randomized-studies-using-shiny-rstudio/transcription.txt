<h3>Informed consent</h3>

<p><a data-t="55">So informed consent.</a> <a data-t="55">Informed consent is a common process when involving human subjects in a trial.</a> <a data-t="60">Often you hand</a> <a data-t="60">people a piece of paper, and they either say that they agree to be in a study or they don't.</a> <a data-t="66">And so instead of that, we can actually have the entire process completed online from the</a> <a data-t="69">comfort of your participants' browsers.</a> <a data-t="74">And so we're basically going to leverage Shiny</a> <a data-t="74">such that if the user consents, they're sent to the study.</a> <a data-t="79">And if they don't, then they're</a> <a data-t="79">sent back to the homepage.</a></p>

<p><a data-t="86">And so in Shiny, we often work with two components, the user</a> <a data-t="86">interface, what the user sees, and the server, what runs behind the scenes in R.</a> <a data-t="92">And so here's</a> <a data-t="92">some code for the user interface to be able to incorporate the consent here.</a> <a data-t="98">And so first,</a> <a data-t="98">you can incorporate the consent text via an HTML document, for example.</a> <a data-t="105">And then you can</a> <a data-t="105">use an action button function to create a nice action button that will either send the</a> <a data-t="109">participant to the study if they agree, or back to the homepage if not.</a></p>

<p><a data-t="116">So on the server</a> <a data-t="116">side, we need to check whether the user clicked yes or no, and then send them somewhere accordingly.</a> <a data-t="122">And so this function will check if the user clicked yes, and if they did, it's going to</a> <a data-t="126">open a new window and send them to the link.</a> <a data-t="131">I'm doing this using render UI, so I need</a> <a data-t="131">to make sure to update the user interface with that UI output so that this actually</a> <a data-t="136">works.</a> <a data-t="136">And so that's what this is doing here, this UI output.</a> <a data-t="142">And then OpenStudy was the ID</a> <a data-t="142">that I gave that output in the server.</a> <a data-t="147">So now that will actually send them to the link.</a></p>
