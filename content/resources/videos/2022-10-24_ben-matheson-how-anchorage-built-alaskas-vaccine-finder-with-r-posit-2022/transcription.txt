<h3>Making it more stable</h3>

<p><a data-t="575">But as we moved out of those really crazy days of the early pandemic, we wanted to turn</a> <a data-t="579">this into something more stable.</a> <a data-t="581">I'd spent enough time trying to fix a scraper at 9 p.m. or 6 a.m., and one of the big changes</a> <a data-t="587">was that we started with one major data source, and that's scraping this system called PrepMod.</a> <a data-t="593">But all of a sudden, we were bringing in all sorts of other data.</a> <a data-t="596">We've got pharmacy APIs.</a> <a data-t="597">We've got some open-source project that brings in pharmacy chains.</a> <a data-t="601">Vaccines.gov came on board, and we accessed some of that CDC data.</a> <a data-t="606">And then all of a sudden, this simple project turns into kind of a complex project with</a> <a data-t="610">six different opportunities for things to break.</a></p>

<p><a data-t="613">And what I was definitely overdue to do was to turn all these into modules so that when</a> <a data-t="618">something breaks, you can turn it off.</a> <a data-t="620">When something needs to be troubleshooted, you can target that in its own individual</a> <a data-t="624">file and its own individual function.</a> <a data-t="627">And it made it from a brittle website into a less brittle website.</a></p>

<p><a data-t="633">I also tried to bring in more error handling.</a> <a data-t="637">Errors would break the website in the early days for sure.</a> <a data-t="639">So I brought in a bunch more try-catch statements so that you can run a statement, and if it</a> <a data-t="644">successfully executes, you get your data back.</a> <a data-t="646">But if and when that error comes, we can still return some valid data.</a> <a data-t="650">We can still keep the rest of the sources online, and we don't break the system just</a> <a data-t="654">because those errors did come.</a></p>
