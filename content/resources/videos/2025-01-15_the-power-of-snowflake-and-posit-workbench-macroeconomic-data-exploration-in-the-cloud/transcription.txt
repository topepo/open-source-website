<p><a data-t="874">If you've used the tidymodels framework or have read about it or are familiar with it to some extent, one of the advantages</a> <a data-t="881">that that framework offers is it gives you the ability to not only train and tune a model itself,</a> <a data-t="887">but to also embed within that model definition any pre-processing that takes place prior to the</a> <a data-t="892">model fit. <p><a data-t="923">What Orbital does is it will take this trained R model object that we have</a> <a data-t="934">and convert that object into native SQL, which means that in the context of Snowflake, I can</a> <a data-t="941">take this model definition that typically would require me to have access to an R runtime in order</a> <a data-t="946">to run and generate new predictions, and instead I can turn that definition into a SQL statement</a> <a data-t="953">that I can then store within Snowflake as a view perhaps or a stored procedure, and now all of a</a> <a data-t="959">sudden my model execution and prediction is not happening in R session somewhere, it's happening</a> <a data-t="965">in a Snowflake warehouse, and it's happening lightning fast because it's part of Snowflake's</a> <a data-t="970">native dialect.</a></p>

<p><a data-t="975">We've seen dramatic results with a number of different customer accounts where model prediction pipelines that previously took</a> <a data-t="987">days, hours or days, now take seconds or minutes, and it's been a dramatic time savings because</a> <a data-t="993">we're able to take advantage of the full compute capability of what Snowflake warehouses have to</a> <a data-t="998">offer.</a></p>
