<h3>What Flex Dashboards are</h3>

<p><a data-t="90">So the idea here behind Flex Dashboards is pretty simple.</a> <a data-t="94">It's just a way to display graphics, tables, a little bit of navigation, a little bit of layout organization</a> <a data-t="101">to create an administrative dashboard for looking at data.</a></p>

<p><a data-t="107">The delivery of Flex Dashboards is the same as any dashboard.</a> <a data-t="112">It's your portal into a dataset.</a> <a data-t="115">But on the back end, Flex Dashboards have some things that make it attractive to you, the developer.</a> <a data-t="120">And those things include, first of all, a Flex Dashboard is created with R Markdown.</a> <a data-t="126">So if you know how to use R Markdown, you can create Flex Dashboards.</a></p>

<p><a data-t="130">The syntax is the same.</a> <a data-t="130">And if you don't know how to use R Markdown,</a> <a data-t="133">Markdown syntax is so easy to learn that by the end of this webinar you'll be able to make your own Flex Dashboards.</a> <a data-t="139">For the most part.</a> <a data-t="139">There might be some things I'm not going to repeat about R Markdown here.</a></p>

<blockquote><a data-t="133">Markdown syntax is so easy to learn that by the end of this webinar you'll be able to make your own Flex Dashboards.</a></blockquote>

<p><a data-t="144">The other benefit of using R Markdown is, once you create your Flex Dashboard,</a> <a data-t="149">it's just an HTML document, an HTML document that was created with R Markdown.</a> <a data-t="153">And you can use any of the channels that you already use to publish R Markdown documents</a> <a data-t="157">to publish your Flex Dashboard.</a> <a data-t="157">So you can put in RPUBs, for example,</a> <a data-t="160">or maybe RStudio Connect, or even </a><a href="/software/shinyapps/">ShinyApps</a><a data-t="160">.io to publish your Dashboard</a> <a data-t="164">and get it out there for other people to see.</a></p>

<h3>What Flex Dashboards do for you</h3>

<p><a data-t="167">The other thing about Flex Dashboards is that they're going to do some work for you in the background.</a> <a data-t="172">Flex Dashboards automatically resize the components inside the Dashboard to fit your user's browser window.</a> <a data-t="178">Or even if your user looks at the Dashboard on a mobile device,</a> <a data-t="180">it'll do resizing to make sure that the Dashboard looks intelligent on their mobile device as well.</a> <a data-t="185">So that's all things you don't have to worry about.</a> <a data-t="185">Flex Dashboard is going to do it for you.</a></p>

<p><a data-t="190">And since Flex Dashboards are R Markdown documents,</a> <a data-t="194">and since R Markdown can deliver anything that you can create with R,</a> <a data-t="198">Flex Dashboards can also deliver anything that you create with R.</a> <a data-t="202">You can make it with R, you can display it in your Dashboard.</a> <a data-t="205">And that's a wide variety of things.</a> <a data-t="205">You have your interactive HTML widgets,</a> <a data-t="208">that's JavaScript-based visualizations.</a> <a data-t="208">You have your base graphics, last graphics,</a> <a data-t="212">ggplot2 graphics, put tables in there.</a> <a data-t="215">And then we'll see that Flex Dashboards even come with some of their own bells and whistles</a> <a data-t="218">that you can use inside your Dashboards.</a></p>

<p><a data-t="221">And then finally, like we highlighted, Flex Dashboards are Shiny compatible.</a> <a data-t="226">So if you want to have some deep reactivity in your Dashboard,</a> <a data-t="229">if you want to provide R scripts and R instructions that your users can trigger</a> <a data-t="234">just by pushing a button and moving a slider, you can do that.</a> <a data-t="237">You can really curate their analysis experience inside this Dashboard</a> <a data-t="242">by combining Flex Dashboards with Shiny.</a></p>
