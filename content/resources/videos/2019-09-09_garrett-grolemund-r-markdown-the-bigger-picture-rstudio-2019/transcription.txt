<h3>What's really causing the replication crisis</h3>

<p><a data-t="310">You can solve this problem, but you need to spot the cause first.</a> <a data-t="316">I can tell you what academia thinks is the cause, because the American Statistical Association</a> <a data-t="320">published an article on it last fall.</a> <a data-t="322">It's right here next to the cover story on R, which, by the way, is also a very good</a> <a data-t="325">read, and the ASA used the metaphor of a cargo cult to explain what's going on.</a></p>

<p><a data-t="330">So do you know what a cargo cult is?</a> <a data-t="332">In World War II, after World War II, in isolated South Pacific islands, cargo cults developed.</a> <a data-t="339">During the war, natives who lived on these islands saw soldiers arrive and build airfields</a> <a data-t="343">and radio towers and whatnot, and then miraculously, from the natives' perspective, planes descended</a> <a data-t="348">from the sky, laden with cargo for the war effort, and a lot of that cargo made its way</a> <a data-t="352">into the hands of the natives, who found it very, very useful.</a> <a data-t="355">But the war ended, and the planes stopped coming.</a></p>

<p><a data-t="357">So in some places, the natives reconstructed landing fields, radio towers, radar dishes,</a> <a data-t="364">the things that they had seen the soldiers use to try to summon the planes back.</a> <a data-t="367">They didn't understand the original technology, but they assumed if they did something that</a> <a data-t="371">looked similar, they could get similar results.</a> <a data-t="373">Well that's the metaphor the ASA uses.</a> <a data-t="375">The ASA says that many applications of statistics are cargo cult statistics.</a> <a data-t="380">Practitioners go through the motions with scant understanding.</a> <a data-t="383">In other words, the people analyzing data just don't know what they're doing.</a></p>

<h3>Confusing science with math</h3>

<p><a data-t="536">So let me tell you what I think is going on.</a> <a data-t="538">And I could do it with this example.</a> <a data-t="539">This is just one of the pages from that Fisher example we were looking at.</a> <a data-t="541">If you look closely at this page, you'll see things like formulas, algebraic variables,</a> <a data-t="548">factorial signs.</a> <a data-t="549">It looks like math.</a> <a data-t="551">Why might that be a problem?</a></p>

<p><a data-t="553">Well, let's do a thought experiment.</a> <a data-t="556">Which set of words do you associate with math?</a> <a data-t="561">I'm going to guess that's not hypotheses, messy, best guess, or discover, unless you're</a> <a data-t="566">very new to math or very advanced at math.</a></p>

<p><a data-t="574">The beauty of math is that it's so precise.</a> <a data-t="577">It's so logically certain.</a> <a data-t="579">You can prove things with math.</a> <a data-t="581">And that's why we love to use it when we can.</a> <a data-t="583">But now think of science, the sort of roll your sleeves up, get your hands dirty and</a> <a data-t="586">make an experiment science.</a> <a data-t="588">You're always working with hypotheses that only represent your best guess.</a> <a data-t="592">At any point in the future, your hypotheses could be revised due to new data or completely</a> <a data-t="597">overturned.</a> <a data-t="599">Your only road to glory in science really relies on discovering something that no one</a> <a data-t="602">else has documented yet.</a> <a data-t="604">With science, you cannot prove things.</a></p>

<p><a data-t="607">These two systems are complementary.</a> <a data-t="609">Math as a form of logic can prove things for you, but only things that are already</a> <a data-t="613">present in your premises or implied by your definitions.</a> <a data-t="616">Math can't tell you if those definitions could correspond well with reality.</a> <a data-t="620">But science can.</a> <a data-t="621">That's science's job.</a> <a data-t="622">Science helps you pick the most pragmatic definition, and it helps you keep track of</a> <a data-t="626">whether or not that definition corresponds to your current state of knowledge.</a> <a data-t="630">But science can never prove that that definition is correct.</a> <a data-t="633">It will always be an estimate or a guess.</a></p>

<h3>Reproducibility as the solution</h3>

<p><a data-t="740">But scientists know what to do.</a> <a data-t="741">It's in your DNA.</a> <a data-t="741">We were taught it all in grade school, and I have a very useful metaphor you could take</a> <a data-t="744">away, and that is the age of exploration when navigators were going out there and discovering</a> <a data-t="748">new continents.</a> <a data-t="749">They were searching for parts of the world that had not yet been observed, and that is</a> <a data-t="752">exactly what scientists do.</a></p>

<p><a data-t="754">So think of someone like Christopher Columbus.</a> <a data-t="756">He sailed across the sea, he discovered the new world, and he came back to Spain.</a> <a data-t="760">Did he offer a logical proof that the new world existed?</a> <a data-t="763">No, that wouldn't make any sense.</a> <a data-t="765">He offered a map, and the map spoke for itself.</a> <a data-t="769">If other explorers followed the map and got to the new world, well, that spoke for itself.</a> <a data-t="774">If they followed the map and didn't get to the new world, then that would speak for itself.</a> <a data-t="777">And that's what scientists should do.</a> <a data-t="779">We should create maps of the things we find for other scientists to follow, and then let</a> <a data-t="784">the destination speak for itself when they get there.</a></p>

<p><a data-t="788">Now scientists are actually pretty good at this, and if you look at the typical scientific</a> <a data-t="792">report, the thing where you start with your hypothesis, you talk about your methods, materials,</a> <a data-t="796">and then the results, that is a very good map that goes from the questions you're asking</a> <a data-t="800">to the data that you collect.</a> <a data-t="802">And back in the days when data was almost synonymous with conclusions, that sufficed.</a> <a data-t="808">But now in the days of data science, there's a second arc in the journey.</a> <a data-t="812">You have to go from your data to your conclusions, and it's not straightforward.</a> <a data-t="816">You need to provide your fellow scientists a map that can take them along the way that</a> <a data-t="819">you went.</a></p>

<p><a data-t="822">You can see why this is sort of difficult, because a map that other scientists could</a> <a data-t="826">use to reproduce this journey would require some tough things.</a> <a data-t="829">They would need your data.</a> <a data-t="831">They would need the code you use to analyze your data, and they would need the software</a> <a data-t="834">required to run that code, and also they need the reasoning that you use to make every decision</a> <a data-t="839">along the way and to interpret the results at the end.</a> <a data-t="842">It's an impossible-sounding combination of things.</a></p>

<p><a data-t="846">But this is exactly what R Markdown lets you put in a single report.</a> <a data-t="850">R Markdown is a free R package that provides an authoring format for data science.</a> <a data-t="856">It's a plain text document that allows you to put things there.</a></p>
