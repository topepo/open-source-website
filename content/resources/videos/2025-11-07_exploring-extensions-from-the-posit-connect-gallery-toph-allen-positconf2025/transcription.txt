<h3>Runtime version scanner</h3>

<p><a data-t="244">So another thing that we need to deal with is making sure that stuff's up to date.</a> <a data-t="244">Now,</a> <a data-t="244">I don't know about you, but I know that I updated to a newer version of Python at some</a> <a data-t="250">point, and I've definitely updated some of the things on Connect since I updated Python.</a> <a data-t="256">So those things might be running Python 3.10, but there's sort of this dark matter of other</a> <a data-t="261">things on Connect that might be running Python 3.9, and if it's removed from the server,</a> <a data-t="265">I don't want them to go down.</a> <a data-t="270">And so the question is, like, do I have to look at each</a> <a data-t="270">one to figure out which version of Python is running on the server?</a></p>

<p><a data-t="277">Turns out, so there's</a> <a data-t="277">an extension in the gallery called the Runtime Version Scanner.</a> <a data-t="284">And the Runtime Version Scanner,</a> <a data-t="285">you can look at the runtimes that different bits of, that you can see all of the stuff</a> <a data-t="291">that you've published to Connect.</a> <a data-t="291">You can pick a version of R, Python, and or </a><a href="/software/quarto/">Quarto</a> <a data-t="296">from a dropdown menu, and then see just the things that are running that version or lower.</a> <a data-t="305">And you can also see the view count over a time period that you can select.</a> <a data-t="312">And so that</a> <a data-t="312">can help you prioritize the most important things and maybe highlight some things that</a> <a data-t="315">you can just delete and not have to worry about.</a></p>

<h3>Package vulnerability scanner</h3>

<p><a data-t="321">So one other type of maintenance task here is accounting for vulnerabilities.</a> <a data-t="328">And I often</a> <a data-t="328">just don't have much ambient awareness at all of if there are any vulnerabilities in</a> <a data-t="332">the packages that I'm using.</a> <a data-t="337">Like if I'm working on something new and I'm setting it up, like,</a> <a data-t="337">I'll install the latest version of, you know, FastAPI or something, but I'm not necessarily</a> <a data-t="343">going to update that whenever I'm updating this, you know, this dashboard that I'm working</a> <a data-t="348">on, because that could cause a bug or, you know, introduces more busy work.</a></p>

<p><a data-t="353">And so I guess there are two problems here.</a> <a data-t="358">The first problem is just which of the things</a> <a data-t="358">that I've got on Connect is running the version that has the vulnerability?</a> <a data-t="365">And then just</a> <a data-t="365">in general, how can I monitor to see if there is anything running a vulnerability?</a> <a data-t="371">So you</a> <a data-t="371">may be detecting a pattern here.</a> <a data-t="374">There's a package vulnerability scanner that we have</a> <a data-t="374">up in the gallery.</a> <a data-t="382">And the package vulnerability scanner lets you see which of your, the things</a> <a data-t="382">that you've published to Connect has a package which has a reported vulnerability.</a> <a data-t="388">We actually</a> <a data-t="388">pull in vulnerability data on packages from Posit Package Manager.</a> <a data-t="394">And you can also dive</a> <a data-t="394">into an individual piece of content to see details on the vulnerability itself, as well</a> <a data-t="401">as the version of the package that it was fixed in.</a> <a data-t="407">So now, looking at this, I'll know</a> <a data-t="407">which bits of content I need to go update and what version of the package to update</a> <a data-t="414">to.</a> <a data-t="419">And even without getting a notification from someone, I can come check this to see</a> <a data-t="419">if there are any vulnerabilities on Connect in my content.</a></p>

<p><a data-t="426">And so just to take a quick look back, the four extensions that I've described, which</a> <a data-t="432">are to do with package, with, I guess, maintenance of long-running apps.</a> <a data-t="441">We've got a usage metrics</a> <a data-t="441">dashboard, which helps us understand what's most important.</a> <a data-t="446">We've got a content health</a> <a data-t="446">monitor, which lets us pick an important piece of content and get alerts if it goes</a> <a data-t="452">down.</a> <a data-t="458">And then we have two, the runtime version scanner and the package vulnerability scanner,</a> <a data-t="458">which help with those maintenance tasks.</a> <a data-t="466">So I'm going to show you some of this live on</a> <a data-t="466">a Connect, on a Connect server.</a></p>
