<h3>What is Pandas?</h3>

<p><a data-t="818">Speaking of history, let's jump in.</a> <a data-t="818">There's a possibility that people out there listening don't know what Pandas is.</a> <a data-t="853">Maybe for that crew, we could introduce what Pandas is.</a></p>

<p><a data-t="538">Absolutely. <h3>Apache Arrow</h3>

<p><a data-t="4254">Yeah, so around the mid-2010s, 2015,</a> <a data-t="4254">I started working at Cloudera,</a> <a data-t="4254">which is a company that was one of the pioneers</a> <a data-t="4254">in the big data ecosystem,</a> <a data-t="4238">and I had spent several years working on,</a> <a data-t="4238">five, six years working on Pandas,</a> <a data-t="4243">and so I had gone through the experience</a> <a data-t="4245">of building Pandas from top to bottom,</a> <a data-t="4250">and it was this full stack system</a> <a data-t="4250">that had its own mini query engine,</a> <a data-t="4253">all of its own algorithms and data structures</a> <a data-t="4256">and all the stuff that we had to build from scratch.</a></p>

<p><a data-t="4261">And I started thinking about,</a> <a data-t="4261">what if it was possible to build</a> <a data-t="4263">some of the underlying computing technology</a> <a data-t="4266">like data readers, like file readers,</a> <a data-t="4269">all the algorithms that power the core components of Pandas,</a> <a data-t="4274">like group operations, aggregations, filtering, selection,</a> <a data-t="4278">all those things.</a> <a data-t="4279">What if it were possible to have a general-purpose library</a> <a data-t="4283">that isn't specific to Python,</a> <a data-t="4285">but is really, really fast, really efficient,</a> <a data-t="4288">and has a large community building it</a> <a data-t="4291">so that you could take that code with you</a> <a data-t="4294">and use it to build many different types of libraries,</a> <a data-t="4296">not just data frame libraries,</a> <a data-t="4298">but also database engines and stream processing engines</a> <a data-t="4302">and all kinds of things.</a></p>

<p><a data-t="4354">And one of the problems we realized we needed to solve,</a> <a data-t="4357">this was like a group of other open-source developers and me,</a> <a data-t="4357">was that we needed to create a way to represent data</a> <a data-t="4401">that was not tied to a specific programming language</a> <a data-t="4405">and that could be used for a very efficient interchange</a> <a data-t="4409">between components,</a> <a data-t="4411">and the idea is that you would have this immutable,</a> <a data-t="4414">like this kind of constant data structure,</a> <a data-t="4417">which is like it's the same in every programming language,</a> <a data-t="4420">and then you can use that as the basis</a> <a data-t="4422">for writing all of your algorithms.</a></p>

<p><a data-t="4430">So we started with building the Arrow format</a> <a data-t="4433">and standardizing it,</a> <a data-t="4434">and then we built a whole ecosystem of components,</a> <a data-t="4439">like library components</a> <a data-t="4440">and different programming languages</a> <a data-t="4442">for building applications that use the Arrow format,</a> <a data-t="4446">so that includes not only tools for building</a> <a data-t="4451">and interacting with the data,</a> <a data-t="4453">but also file readers,</a> <a data-t="4454">so you can read CSV files and JSON data and Parquet files,</a> <a data-t="4458">read data out of database systems.</a> <a data-t="4501">Wherever the data comes from,</a> <a data-t="4502">we want to have an efficient way to get it into the Arrow format.</a></p>

<p><a data-t="4507">And then we moved on to building data processing engines</a> <a data-t="4510">that are native to the Arrow format,</a> <a data-t="4513">so that Arrow goes in, the data's processed,</a> <a data-t="4516">Arrow goes out.</a> <a data-t="4518">So DuckDB, for example,</a> <a data-t="4519">supports Arrow as a preferred input format</a> <a data-t="4522">and DuckDB is more or less Arrow-like in its internals.</a></p>

<p><a data-t="4549">And so if you were building Pandas now,</a> <a data-t="4550">you could build a Pandas-like library</a> <a data-t="4553">based on the Arrow components in much less time,</a> <a data-t="4557">and it would be fast and efficient</a> <a data-t="4559">and interoperable with the whole ecosystem</a> <a data-t="4562">of other projects that use Arrow.</a></p>

<p><a data-t="4600">And it turns out that, as is true</a> <a data-t="4604">with many open-source software problems,</a> <a data-t="4608">that many of these problems are,</a> <a data-t="4611">the social problems are harder</a> <a data-t="4612">than the technical problems.</a> <a data-t="4613">And so if you can solve the kind of people coordination</a> <a data-t="4618">and consensus problems,</a> <a data-t="4620">solving the technical issues</a> <a data-t="4621">is much easier by comparison.</a></p>

<h3>Modern hardware and performance</h3>
