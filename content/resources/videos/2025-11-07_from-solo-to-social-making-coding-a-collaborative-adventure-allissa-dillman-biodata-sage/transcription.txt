<h3>Three pillars: trust, accessibility, emotional resonance</h3>

<p><a data-t="223">So how can we create spaces, environments, and places where things are more social?</a> <a data-t="229">That things are about the doing, the building of the thing versus the doing of the thing?</a> <a data-t="233">And I've kind of come up with these three basic concepts that kind of happen over and</a> <a data-t="238">over again in these kinds of spaces.</a> <a data-t="241">And the first is trust.</a> <a data-t="243">Right?</a> <a data-t="244">To create a safe, collaborative environment where people feel comfortable asking questions,</a> <a data-t="251">expressing interest, and also just saying frustrations.</a> <a data-t="256">You also want things to be accessible, right?</a> <a data-t="258">You want to lower those barriers of entry, make it not so scary, have people have a starting</a> <a data-t="263">point so they don't feel like they're drowning.</a> <a data-t="265">And also, and more importantly, and actually this kind of came out in our previous speaker,</a> <a data-t="269">right?</a> <a data-t="270">This idea of, like, you want things to kind of speak to people, right?</a> <a data-t="272">This emotional resonance.</a> <a data-t="273">I want to feel a connection, I want to be connected to my passion, and I want to be</a> <a data-t="277">engaged.</a></p>

<p><a data-t="278">So with these three concepts in mind, there are a couple of different ways that you can</a> <a data-t="284">start to create community spaces using these concepts.</a> <a data-t="290">And there's sort of three event types to me that really lend themselves well to this idea</a> <a data-t="295">of trust, accessibility, and emotional resonance.</a> <a data-t="299">The first is think-pair-share.</a> <a data-t="301">So the idea of think-pair-share is this cooperative learning strategy, and it starts with individual</a> <a data-t="307">reflection first, and then you're paired together, and you kind of discuss the problem, and then</a> <a data-t="312">you come out to a larger group, and then you're kind of discussing, like, the things that</a> <a data-t="315">you've learned.</a> <a data-t="316">Then you have data storytelling, and here we have, like, these very bite-sized projects,</a> <a data-t="323">and they're really focused on the idea of, like, data to meaning, so they're really focused</a> <a data-t="328">on visualization, not necessarily so much on the coding themselves.</a> <a data-t="333">And then finally, you have the sort of more traditional tried-and-true hackathons, right?</a> <a data-t="337">Fast-paced team-building, problem-solving, and that sort of thing.</a></p>

<h3>Think-pair-share with high school students</h3>

<p><a data-t="339">And I'm going to break down each one of these, and I'm going to kind of have a flow throughout</a> <a data-t="345">of how you can interweave trust, accessibility, and emotional resonance for each one of these</a> <a data-t="350">event types with a story, an example.</a> <a data-t="354">So the first I'm going to talk about is think-pair-share, and this story is going to be about a group</a> <a data-t="359">of high school students that I worked with.</a> <a data-t="362">It was a completely virtual and mostly asynchronous for most of the summer, and then we kind of</a> <a data-t="366">got together at the end of the summer in person, and the first thing I noticed when we were</a> <a data-t="371">together in person was, while they really did understand, like, the functions and kind</a> <a data-t="376">of what they did, they got really stuck on, like, how do I actually apply this in the</a> <a data-t="382">real world to, like, real things?</a> <a data-t="383">So that was the first thing.</a> <a data-t="385">The other thing was they also really didn't want to ask questions, right?</a> <a data-t="388">They'd been asynchronous and online the entire time, so nobody wanted to say anything.</a> <a data-t="394">Nobody wanted any questions.</a> <a data-t="395">They were very quiet.</a></p>

<p><a data-t="397">And I thought, like, a think-pair-share might be a great way to sort of hit both of these</a> <a data-t="401">issues, and so the first thing I did is before we even did the think-pair-share, I actually</a> <a data-t="409">asked folks, like, hey, what are your goals?</a> <a data-t="412">What are your interests?</a> <a data-t="413">What are your hobbies?</a> <a data-t="414">Like, what do you want to get out of this?</a> <a data-t="415">I asked, like, a whole set of questions, and then I actually built coding challenges around</a> <a data-t="420">these problems of interest.</a> <a data-t="424">Then I provided a bunch of different coding examples in a bunch of different formats,</a> <a data-t="430">and so that way each pair got to pick the challenge that really kind of fit them, and</a> <a data-t="437">they got to roll with that.</a></p>

<p><a data-t="440">The other thing that was really important to me is to really hit home this idea of mistakes</a> <a data-t="445">are learning opportunities, not failures, and I even demoed a bunch of my old code from</a> <a data-t="451">grad school that is full of terrible, terrible mistakes, and kind of walked through, like,</a> <a data-t="456">and here's what I've learned about that, and here's, you know, how I've built new things,</a> <a data-t="460">and it was great that I did these mistakes, because now I get to do more cool things.</a> <a data-t="464">But also all questions are valuable.</a> <a data-t="466">Honestly, a lot of the times when people ask me questions, I'm like, oh, I never even thought</a> <a data-t="471">about it.</a> <a data-t="472">Let's try it together.</a> <a data-t="473">Like, it's a great space to learn together, but also, like, more often than not, if you</a> <a data-t="476">have a question, somebody else has pretty much the same question.</a> <a data-t="479">So why not learn it together?</a></p>

<p><a data-t="483">The other thing that I really like personally about think-pair-share is the think part,</a> <a data-t="489">and you know, this is meant to be, like, the space where we really get to slow down and</a> <a data-t="493">process things internally, but I like it a lot, because I'm dyslexic, so I'm not a</a> <a data-t="479">fast reader, and often what would happen, like, when I was trying to learn things, like,</a> <a data-t="484">you would get shown some code, and everyone would already have the solution, and I'm like,</a> <a data-t="486">I'm still reading what's happening.</a> <a data-t="488">I don't know where we are.</a> <a data-t="490">So like the think part really does have it baked into there.</a> <a data-t="493">You have this pause time, this down time.</a> <a data-t="495">Let's sit with our thoughts, slow down for a second, see what this means.</a></p>

<p><a data-t="514">The other thing is that I built prompts for discussion, so not coding prompts, but actual</a> <a data-t="520">discussion prompts, because I really wanted them to focus on this idea of, like, problem-solving</a> <a data-t="527">and different ways to problem-solve and how to work through the challenge together, rather</a> <a data-t="534">than, like, oh, let's get really nitty-gritty on the code.</a> <a data-t="536">And then I kind of further brought that home for the group discussion, so we celebrated</a> <a data-t="539">process over product, so what I mean by that is, like, we really celebrated, like, that's</a> <a data-t="543">a really cool way to think about that, or I didn't even think about, like, using it</a> <a data-t="547">that way.</a> <a data-t="548">This is really great.</a> <a data-t="549">And it was, again, less about, like, the code itself.</a> <a data-t="552">I mean, but they were writing code, learning code, building code.</a></p>

<p><a data-t="558">One thing I really loved about this is, so after we did the think-pair-share, I had always</a> <a data-t="562">had virtual office hours throughout the whole summer.</a> <a data-t="565">Almost none of the students ever came and just sat there and did nothing for an hour.</a> <a data-t="570">But after this event, they were coming to virtual office hours, and I was like, this</a> <a data-t="574">is great.</a> <a data-t="575">And then when we would do live demos, they were asking questions.</a> <a data-t="579">But the other cool thing is this was the third year I'd run this program, but the first year</a> <a data-t="583">I'd done this, and they actually started their final projects early, like, before they needed</a> <a data-t="588">to start them.</a> <a data-t="589">So I have a hunch that maybe they felt a lot more confident in getting things done,</a> <a data-t="594">even sort of, like, without guidance.</a></p>
