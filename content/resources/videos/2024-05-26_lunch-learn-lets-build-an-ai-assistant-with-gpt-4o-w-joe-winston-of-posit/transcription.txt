<h3>Introduction to <a href="/software/shiny-python/">Shiny for Python</a></h3>

<p><a data-t="1118">So what is Shiny?</a> <a data-t="1119">Shiny is a framework for making interactive web apps in pure Python.</a> <a data-t="1123">You do not need to know HTML, CSS, or JavaScript.</a> <a data-t="1127">If you don't want to, just pure Python, you're already halfway there if you know how to code in Python.</a> <a data-t="1133">But if you do know HTML, CSS, and JavaScript, you can absolutely use them to enhance Shiny by customizing it and extending it, which is something that I spent a lot of time doing this week, and I can't wait to show you the results in a few moments here.</a></p>

<p><a data-t="1148">Shiny is specifically designed for data-related applications.</a> <a data-t="1153">There's a lot of visualization built in that it works with Matplotlib and Plotly and is designed to work with data frames.</a> <a data-t="1161">So you can use it for a lot of different things, but it is especially well-suited for data-related applications.</a></p>

<p><a data-t="1172">The heart of Shiny is a reactive programming model.</a> <a data-t="1176">If you have not heard that term before, you don't have to worry about it right now except to know that this approach is really good at making your apps efficient because only a minimum amount of code needs to run every time you interact with your app in order to get your outputs to be correct.</a> <a data-t="1210">And it's also a very good programming paradigm for avoiding the kinds of bugs that are usually pretty common in UI programming.</a></p>

<p><a data-t="1228">It is an open-sourced framework, MIT-licensed, and after you've written your Shiny app, you can either run them locally or you can deploy them in a variety of ways in the cloud or on-prem.</a> <a data-t="1239">We have a number of free and paid cloud options as well as open-source and enterprise on-prem options, and all that is described in the Shiny for Python documentation I'll link to at the end.</a></p>

<p><a data-t="1259">On-prem means on-premises, meaning you're running it on your own server that you manage, whether that is in your bedroom or in your data center or in a cloud of your choice.</a> <a data-t="1271">But if you're managing the server, then that's called on-prem, whereas the cloud offerings, you don't even know that a server exists.</a></p>
