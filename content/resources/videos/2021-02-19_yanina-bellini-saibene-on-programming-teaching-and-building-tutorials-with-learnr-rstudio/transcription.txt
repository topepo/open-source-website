<p><a data-t="0">En el año 83, Argentina volvía a la democracia y yo, con seis años, usaba por primera vez</a> <a data-t="15">una computadora. Fue amor a primera vista. Yo vivía en un pueblito rural de 2.000 habitantes</a> <a data-t="22">del interior de mi país y mi escuela primaria había comprado una computadora. Una sola</a> <a data-t="28">para toda la escuela y la única en todo el pueblo. Todos los sábados a la mañana, entre</a> <a data-t="35">mis nueve y catorce años, mis padres me llevaban 80 kilómetros para que yo pudiera aprender</a> <a data-t="41">programación. No teníamos un vehículo propio y no había transporte público, pero como</a> <a data-t="47">mi papá es mecánico, algunos de sus clientes nos prestaban sus vehículos para que pudiéramos</a> <a data-t="52">hacer ese viaje.</a></p>

<p><a data-t="59">A los 15 años tuve que empezar a trabajar y mi primer trabajo fue</a> <a data-t="59">enseñar computación. Desde ese día no paré de enseñar y sigo aprendiendo informática,</a> <a data-t="65">lo que me trae hoy aquí a charlar con ustedes para contarles sobre dos de mis pasiones,</a> <a data-t="72">enseñar y programar. Soy Janina Bellini Saibene, investigadora del Instituto Nacional de Tecnología</a> <a data-t="78">Agropecuaria, soy organizadora y parte del Global Team de R-Ladies, soy cofundador y</a> <a data-t="84">chair de LatinAR, soy parte de MyAR y cofundadora de MetaDocencia. Hoy vamos a aprender sobre</a> <a data-t="91">el paquete </a><a href="/software/learnr/">learnr</a><a data-t="91"> para hacer tutoriales interactivos y espero que podamos compartir el mismo entusiasmo</a> <a data-t="98">que yo sentía en esos viajes los sábados a la mañana.</a></p>

<h3>¿Qué es learnr?</h3>

<p><a data-t="103">La sabiduría popular del altiplano andino nos dice que si escucho algo me lo olvido,</a> <a data-t="112">si veo algo me acuerdo, pero si hago algo lo aprendo. El paquete learnr facilita la</a> <a data-t="119">conversión de un documento R Markdown, como este que vemos a la izquierda, en un</a> <a data-t="125">tutorial interactivo, como este que vemos a la derecha. Podemos generar tutoriales de</a> <a data-t="130">estadística, ciencia de datos, programación o de lo que ustedes se imaginen. Este tipo</a> <a data-t="136">de tutoriales tienen contenido junto con componentes interactivos que nos permiten verificar y</a> <a data-t="143">reforzar la comprensión.</a></p>

<blockquote><a data-t="112">si veo algo me acuerdo, pero si hago algo lo aprendo.</a></blockquote>

<p><a data-t="149">Como vemos en este tutorial de ejemplo, el contenido se puede</a> <a data-t="149">presentar en forma de texto, imágenes y diagramas. También podemos agregar videos, gráficos</a> <a data-t="156">y personalizar el estilo de nuestro tutorial. Contamos a la izquierda con una barra de</a> <a data-t="162">contenido y de progreso que nos permite saltar entre los diferentes temas y saber hasta dónde</a> <a data-t="168">avanzamos.</a></p>

<p><a data-t="174">learnr nos permite aprender haciendo al transformar los chunks de códigos en ejercicios</a> <a data-t="174">interactivos. Por ejemplo, este ejercicio ya nos presenta el código para realizar un</a> <a data-t="180">gráfico. Si nosotros presionamos en el botón Run Code, el resultado se mostrará debajo. Si</a> <a data-t="188">nosotros cambiamos algo en el código, como por ejemplo intercambiamos las variables de los ejes</a> <a data-t="193">X e Y y volvemos a ejecutar, ese cambio se verá reflejado en el resultado. Y si cometemos un</a> <a data-t="200">error, como por ejemplo nos olvidamos de indicar cuál es la variable del eje X y lo ejecutamos,</a> <a data-t="207">se nos mostrará un mensaje indicándonos ese error.</a></p>

<p><a data-t="215">Este tipo de ejercicio permite que los</a> <a data-t="215">y las estudiantes pongan a prueba sus conocimientos y reflexionen sobre lo que aprendieron. También</a> <a data-t="221">podemos generar preguntas de opción múltiple, indicando cuál o cuáles de las respuestas son</a> <a data-t="228">correctas y seleccionando un mensaje diferente para darle a nuestros estudiantes según la</a> <a data-t="235">opción que hayan elegido.</a></p>

<p><a data-t="241">Finalmente, otra de las grandes ventajas que tiene este tipo de tutorial</a> <a data-t="241">es que se pueden entregar como un paquete de R y entonces al instalarlo nos van a aparecer en</a> <a data-t="248">la pestaña Tutorial dentro de </a><a href="/software/rstudio/">RStudio</a><a data-t="248"> o bien se pueden publicar como una página web. De esta</a> <a data-t="255">manera compartir este material es muy sencillo.</a></p>

<h3>Cómo generar un buen tutorial</h3>

<p><a data-t="263">La pregunta ahora es ¿cómo generar un buen</a> <a data-t="263">tutorial? Les voy a contar mi propia receta secreta basada en las enseñanzas de este libro.</a> <a data-t="269">Podemos pensar que este tipo de material educativo tiene dos componentes principales,</a> <a data-t="275">el contexto y el contenido, y el contexto va a influenciar fuertemente el contenido final.</a></p>

<p><a data-t="281">Para empezar a definir este contexto la primera pregunta que tengo que contestar es ¿para quién</a> <a data-t="288">estoy generando este tutorial? ¿Son personas novatas, son personas principiantes? ¿Qué tienen</a> <a data-t="295">que saber para poder aprovechar lo que quiero enseñarles? ¿Tienen necesidades específicas como</a> <a data-t="302">por ejemplo de accesibilidad? El primer paso para iniciar el diseño es conocer a mi audiencia y las</a> <a data-t="310">personas tipo son una muy buena herramienta para esta etapa en particular.</a></p>

<p><a data-t="318">La segunda pregunta que</a> <a data-t="318">tengo que contestar es ¿qué quiero que aprendan con este tutorial? Y aquí tengo que definir qué voy</a> <a data-t="324">a enseñar. Tengo que escribir mis objetivos de aprendizaje. En esta etapa en particular a mí me</a> <a data-t="331">resulta muy útil generar un mapa conceptual con todos los conceptos que quiero enseñar y cómo</a> <a data-t="337">se relacionan entre sí.</a></p>

<p><a data-t="345">Mi tercer paso es buscar y ver si ya existen lecciones sobre este tema.</a> <a data-t="353">Analizar cómo enseñan otras personas, qué tipo de herramienta utilizan, cuál es la secuencia en</a> <a data-t="353">que presentan los contenidos, puede ahorrarnos mucho tiempo y darnos muy buenas ideas. Un punto</a> <a data-t="360">extra además si ese material está compartido de una forma que puede ser reusable.</a></p>

<p><a data-t="368">Finalmente y</a> <a data-t="368">relacionado con este punto anterior tengo que contestar la pregunta ¿qué voy a dejar que otras</a> <a data-t="374">personas hagan con este material? Así que aquí tengo que seleccionar una licencia de uso que sea</a> <a data-t="380">bien clara y explícita así tanto mis alumnos como mis colegas pueden saber cómo usar mi material y</a> <a data-t="387">hacerlo de la forma correcta.</a></p>

<p><a data-t="394">Sí, ya sé, hay muchísimo trabajo que hacer antes de escribir</a> <a data-t="394">una sola línea de código. No, no te preocupes, tengo un paquete que te va a ayudar con estas</a> <a data-t="399">tareas. Desde MetaDocencia generamos un paquete para learnr con plantillas en español donde además</a> <a data-t="407">te sugerimos qué secciones y qué partes puede tener un tutorial. Vamos a ver cómo funciona.</a></p>

<h3>Plantillas de MetaDocencia para learnr</h3>

<p><a data-t="413">Necesitamos tener instalados el paquete learnr que se encuentra en CRAN y la versión en desarrollo</a> <a data-t="419">que se encuentra en GitHub del paquete learnr es que contienen las plantillas en español.</a> <a data-t="424">Luego para generar un nuevo tutorial voy a seleccionar la opción File, New File, R Markdown</a> <a data-t="432">y allí voy a elegir From Template. Del listado de plantillas que aparecen voy a elegir Tutorial</a> <a data-t="438">Interactivo, Secciones.</a></p>

<p><a data-t="445">Este nuevo documento que se crea ya en el encabezado nos indica que va a ser</a> <a data-t="445">un tutorial de learnr y aparecen varias opciones con sus parámetros y qué significan cada uno de</a> <a data-t="452">ellos. A su vez si vemos ya hay un contenido precargado con secciones y sugerencias. Vamos a</a> <a data-t="461">ejecutar este documento para ver que además hemos personalizado cómo se presentan la fuente, los</a> <a data-t="469">colores, los links con la identidad visual de MetaDocencia. Pueden encontrar cómo personalizar</a> <a data-t="475">esto utilizando nuestro archivo CSS que se instala junto con el paquete.</a></p>

<p><a data-t="483">Ahora bien, esta</a> <a data-t="483">primera estructura que les proponemos que todos sus tutoriales tengan presenta como primera opción</a> <a data-t="490">los objetivos de aprendizaje con la idea que una persona que se encuentra con nuestro tutorial en</a> <a data-t="496">una primera vista pueda saber qué se supone que tiene que aprender con él mismo. Además cada vez</a> <a data-t="503">que vean el recuadro con la manzanita estamos presentando ahí un ejemplo y también ponemos</a> <a data-t="509">disponible links a lugares donde aprender más sobre esta herramienta o con ejemplos generados</a> <a data-t="516">por otra persona.</a></p>

<p><a data-t="521">El segundo contenido que le sugerimos que presente son las personas tipo,</a> <a data-t="521">de nuevo para que quien va a tomar el tutorial sepa en quién se pensó cuando se generó ese</a> <a data-t="529">contenido y estamos poniendo links a varias definiciones de personas tipo hechas por RStudio</a> <a data-t="536">y por MetaDocencia.</a></p>

<p><a data-t="545">Finalmente presentamos la licencia de uso y aquí explicamos la diferencia</a> <a data-t="545">entre las licencias que son para el material educativo, la explicación, las slides, los vídeos</a> <a data-t="552">y el código fuente y también presentamos un ejemplo. Finalmente como una parte opcional de</a> <a data-t="560">estas herramientas presentamos un mapa conceptual con un ejemplo y links a dónde ir aprender más</a> <a data-t="567">sobre mapas conceptuales y dónde se pueden ver y utilizar varios mapas conceptuales ya generados</a> <a data-t="575">en relación a la ciencia de datos y la enseñanza de R.</a></p>

<p><a data-t="582">En caso que ustedes vayan a agregar esta</a> <a data-t="582">sección se recomienda ponerla después de los objetivos de aprendizaje. Finalmente las otras</a> <a data-t="589">dos secciones que aparecen en esta plantilla de tutorial presentan un ejemplo de un ejercicio con</a> <a data-t="596">y sin código y de una pregunta de opción múltiple.</a></p>

<h3>Generando el contenido: ejercicios y preguntas</h3>

<p><a data-t="603">Definido el contexto tenemos que generar el</a> <a data-t="603">contenido de la lección hay muchos procesos para generar lecciones de forma efectiva al menos vamos</a> <a data-t="610">a tener que contestar dos preguntas cuál va a ser la secuencia de presentación de ese contenido y</a> <a data-t="617">cómo vamos a ayudar a nuestros estudiantes a chequear la comprensión de cada tema. Las evaluaciones</a> <a data-t="624">formativas presentadas en diferentes momentos a lo largo del tutorial son una excelente herramienta</a> <a data-t="630">para lograr ese objetivo de reflexionar sobre lo aprendido chequear los conceptos aprendidos y</a> <a data-t="638">permitir validar el avance y el progreso que vamos haciendo.</a></p>

<p><a data-t="647">Los ejercicios interactivos y las</a> <a data-t="647">preguntas de opción múltiple son las dos herramientas que nos presenta learnr para ayudarnos en esa</a> <a data-t="652">tarea. Un chunk de código se convierte en un ejercicio cuando le doy un nombre y ponemos el</a> <a data-t="660">parámetro exercise en true. A partir de ahí learnr genera una caja de código con un botón</a> <a data-t="666">de run code y de start over que puede tener código o estar vacía. Si uno escribe el código vamos a</a> <a data-t="673">tener funcionalidades muy útiles como por ejemplo sugerencias para completar lo que estamos</a> <a data-t="678">escribiendo o nos va a marcar también posibles errores de sintaxis.</a></p>

<p><a data-t="687">A estos ejercicios como bien</a> <a data-t="687">les comentaba yo puedo presentarlos en blanco o puedo presentarlos con código y ese código puede</a> <a data-t="694">estar completo, puede ser un código funcional o puede estar incompleto. Los que nos presenta un</a> <a data-t="701">abanico de posibilidades para realizar ejercicios. Yo puedo presentar un código funcional y pedirle</a> <a data-t="708">a los alumnos que lo cambien para que haga otra cosa. Puedo presentar un código que tiene un</a> <a data-t="713">error y pedirles a los alumnos que lo corrijan o puedo presentar un código al cual le faltan</a> <a data-t="720">partes como un ejercicio desvanecido y el cual los alumnos tienen que completar correctamente.</a></p>

<p><a data-t="729">Estos ejercicios interactivos tienen también la posibilidad que uno le agregue consejos o pistas</a> <a data-t="736">y soluciones. Para hacer eso generamos otro chunk de código que va a tener el mismo nombre pero al</a> <a data-t="743">cual le agregamos la palabra guión del medio hint. Si queremos más de uno luego de la palabra hint</a> <a data-t="750">vamos a poner guión del medio 1, guión del medio 2 y así según la cantidad de pistas que querramos</a> <a data-t="757">darle a los estudiantes. Para agregar una solución vamos a escribir otro chunk de código de vuelta con</a> <a data-t="764">el mismo nombre y en este caso será guión del medio solution. Todo lo que pongamos ahí adentro</a> <a data-t="771">se va a presentar en un botón que se llama hint en esta caja de código y el cual vamos a poder</a> <a data-t="777">ir navegando. El último que se presenta es la solución. Tenemos la posibilidad también de copiar</a> <a data-t="783">al portapapel lo que dice en cada pista y pegarlo en el código para no tener que tipearlo desde cero.</a></p>

<p><a data-t="791">El otro elemento que tenemos para poder realizar evaluaciones formativas son las preguntas de</a> <a data-t="797">opción múltiple. Para esto learnr nos presenta la función quiz, question y answer. Ahí podemos</a> <a data-t="806">determinar cuál es el texto de la pregunta, cuáles son las respuestas, con el parámetro correct</a> <a data-t="812">igual a true vamos a indicar cuál es la correcta y en el parámetro message vamos a poder escribir</a> <a data-t="818">el texto que queremos dar como feedback a nuestros estudiantes. También tenemos la posibilidad de</a> <a data-t="824">indicar si vamos a permitir que el estudiante vuelve a intentar en caso que se haya equivocado</a> <a data-t="830">con el parámetro allow retry igual a true y tenemos en el caso de las preguntas la posibilidad de</a> <a data-t="837">cambiar el texto que tienen los botones de enviar respuesta e intentar de nuevo permitiéndonos</a> <a data-t="843">ponerlo en nuestro lenguaje. Esperemos que pronto tengamos esa funcionalidad también en los</a> <a data-t="849">ejercicios de código.</a></p>

<p><a data-t="856">Las respuestas de opción múltiple son una muy buena opción para detectar</a> <a data-t="856">errores comunes que los alumnos suelen cometer cuando aprenden un nuevo tema. Si nosotros las</a> <a data-t="864">diseñamos adecuadamente e incluimos en las respuestas incorrectas este tipo de error común</a> <a data-t="870">podremos detectarlo y el feedback que le entreguemos en los mensajes va a ser mucho más</a> <a data-t="877">significativo para ayudarlos a aprender bien el tema y poder continuar en su camino de aprendizaje.</a></p>
