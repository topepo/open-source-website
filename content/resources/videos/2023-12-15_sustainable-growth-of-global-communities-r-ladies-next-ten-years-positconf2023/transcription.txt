<h3>Good programming practices for community management</h3>

<p><a data-t="134">So, we're people who love to code.</a> <a data-t="136">And we were thinking in the last year,</a> <a data-t="138">what if we use good programming practices to manage our programming community?</a> <a data-t="143">For example, modularity, refactoring, and testing.</a></p>

<p><a data-t="147">So, what about modularity?</a> <a data-t="149">You don't want a function that is that big.</a> <a data-t="152">What you want to do is to make it in small chunks,</a> <a data-t="155">so it's flexible, you can maintain it easily, and you can develop new features.</a> <a data-t="160">So, what about communities?</a> <a data-t="162">You want to do the same.</a> <a data-t="163">But this is about workload, obviously.</a> <a data-t="166">You don't want just one person taking all the work.</a></p>

<p><a data-t="172">But it's not just about workload.</a> <a data-t="174">There are other things involved.</a> <a data-t="174">For example, if you have a team that's working on their website,</a> <a data-t="178">and they're only working on their website, they don't have to take care about other tasks,</a> <a data-t="181">they can focus their effort on feature development.</a> <a data-t="185">So, making your website cooler.</a></p>

<p><a data-t="188">And it's also important in terms of participation,</a> <a data-t="191">because not everyone can engage as an organizer or as part of the global team in the same way.</a> <a data-t="196">Not everyone has the same time patterns to commit.</a> <a data-t="199">For example, you may have some volunteers that have three hours a week</a> <a data-t="203">that they can devote only on Friday afternoon.</a> <a data-t="206">But you have other people that prefer to work on small chunks of time.</a> <a data-t="211">So, they can maybe go to the computer two times a day</a> <a data-t="214">and work there around 10 and 20 minutes.</a></p>

<p><a data-t="217">So, if you have a community that can offer people different ways to engage,</a> <a data-t="222">they can be part of the team and help you and all your community with the work.</a> <a data-t="228">And there are some tasks where you need people that have these time patterns.</a> <a data-t="232">For example, you want someone moderating a community</a> <a data-t="235">that engages not just one day, three hours,</a> <a data-t="238">but that's there many times a week.</a></p>

<p><a data-t="241">This is also important for the ways that people can participate.</a> <a data-t="245">So, sometimes you want to participate in a community for learning new things.</a> <a data-t="249">So, if you want to learn how to code, more about coding,</a> <a data-t="252">you might want to participate, for example, in the website team.</a> <a data-t="256">But if you want to, I don't know, engage in things that are related to social skills,</a> <a data-t="263">maybe you want to be part of the mentoring team of our ladies.</a> <a data-t="266">So, this gives space for everyone.</a></p>

<p><a data-t="269">So, one thing that we are for sure going to keep working towards</a> <a data-t="273">is have a modular global team.</a> <a data-t="275">So, this can help everyone to participate in meaningful ways.</a></p>

<h3>Refactoring workflows</h3>

<p><a data-t="281">So, what about refactoring?</a> <a data-t="283">Sometimes you have code that works, and we want to keep it that way.</a> <a data-t="287">We don't want to touch it.</a> <a data-t="289">And the same happens with projects.</a> <a data-t="291">So, sometimes you have a project that you really love,</a> <a data-t="293">you really care about the vision of that project,</a> <a data-t="296">and you really don't care the kind of work that involves to keep the project alive.</a> <a data-t="301">And that's because you have the vision of the project in front of you.</a> <a data-t="304">So, you're really committed to that.</a></p>

<p><a data-t="307">But it's good to think about how we can refactor our workflows</a> <a data-t="311">as community organizers to make everyone participate in meaningful ways.</a> <a data-t="316">So, refactoring your workflows is not only about making your community more efficient,</a> <a data-t="321">but it's also about making the time that people are committing</a> <a data-t="324">as volunteers in your community more meaningful.</a> <a data-t="327">Because you don't want people committing time that can be,</a> <a data-t="331">that's only about repetitive tasks that can be automated.</a> <a data-t="334">You want them doing relevant things.</a></p>

<blockquote><a data-t="316">So, refactoring your workflows is not only about making your community more efficient,</a> <a data-t="321">but it's also about making the time that people are committing</a> <a data-t="324">as volunteers in your community more meaningful.</a></blockquote>

<p><a data-t="337">So, as an example, we can explain what happened with the R-Ladies directory.</a> <a data-t="343">So, the R-Ladies directory is an initiative</a> <a data-t="346">that's very related to R-Ladies mission</a> <a data-t="349">that help us showcasing what R-Ladies do around the world.</a> <a data-t="353">And it's an excellent antidote to all my panels.</a></p>

<p><a data-t="383">So, the directory, this is how the directory used to look on the webpage.</a> <a data-t="389">So, to add your entry to the directory,</a> <a data-t="392">you can, there's a form there that you can use.</a> <a data-t="395">And there used to be this message.</a> <a data-t="398">If you want to update your profile, just fill the same form</a> <a data-t="401">and drop us a line in the comment box saying you want to update your profile.</a> <a data-t="406">And there was this extra option.</a> <a data-t="408">Or you can just email us at speakers at rlates.org</a> <a data-t="412">with any relevant information about yourself.</a> <a data-t="415">Easy. <p><a data-t="425">But what happens behind the scenes with this workflow?</a> <a data-t="428">So, you have a spreadsheet, and people start submitting their entries.</a> <a data-t="433">And you have to take this information and put it on the website.</a> <a data-t="437">And you work in these two entries.</a> <a data-t="438">So, you're going to put a color to remember that those are already on the website.</a> <a data-t="443">And you keep working.</a> <a data-t="444">And then this one is not really well,</a> <a data-t="446">so you're going to work in this one afterwards.</a> <a data-t="449">So, you put another color in there.</a> <a data-t="450">So, you remember that this one has a problem.</a> <a data-t="452">And you keep working.</a> <a data-t="454">And then you realize that this entry,</a> <a data-t="456">in the comment they put this is an update,</a> <a data-t="458">so you have to treat this one differently.</a> <a data-t="460">And then an email arrives,</a> <a data-t="463">and you have to take the information from the email</a> <a data-t="465">and put it here so it can go to the directory.</a></p>

<p><a data-t="671">So, now the task, the focus of the task is on the review,</a> <a data-t="675">not in copy-pasting.</a> <a data-t="677">The task can be scheduled,</a> <a data-t="679">and people have time for planning features for the website.</a> <a data-t="682">For example, how to filter the entries that are there</a> <a data-t="686">on some other awesome stuff that the website</a> <a data-t="688">and the director team are always thinking about.</a></p>

<h3>Testing workflows through onboarding</h3>

<p><a data-t="721">So, a way to discover that your workflows need</a> <a data-t="728">a little bit of refactoring is testing your workflows.</a> <a data-t="733">So, we always want to ensure that our code behaves as expected.</a> <a data-t="739">So, we want to do the same with our workflows.</a> <a data-t="741">We want our workflows to behave as expected.</a></p>

<p><a data-t="745">So, I'm going to tell a little story.</a> <a data-t="747">I became part of the global team two years ago</a> <a data-t="750">as part of the Slack monitoring team.</a> <a data-t="756">And this is a task that I jump into very easily</a> <a data-t="759">because I was already a person who was using Slack.</a> <a data-t="763">I know how the code of conduct of our ladies work</a> <a data-t="766">because I was an organizer.</a> <a data-t="768">So, it was very easy to me to know how to perform this task.</a></p>

<p><a data-t="772">And a couple of months later,</a> <a data-t="774">there were people needed for the meetup team,</a> <a data-t="777">people who manage the meetup accounts of all the chapters.</a> <a data-t="781">And I fell.</a> <a data-t="782">I wasn't able to successfully start doing this task.</a> <a data-t="786">And I was wondering why,</a> <a data-t="787">because there's very good documentation.</a> <a data-t="789">So, there's all the goal of the task,</a> <a data-t="792">all things that a meetup account manager needs to do</a> <a data-t="795">was very well explained.</a> <a data-t="797">But I didn't know how to become the person</a> <a data-t="800">that can use the documentation.</a> <a data-t="802">So, there was a gap between me and the documentation</a> <a data-t="804">that I didn't experience when I was working in Slack</a> <a data-t="807">because I already was familiar with that.</a></p>

<p><a data-t="810">And I discovered that the problem</a> <a data-t="813">was not with the technical infrastructure,</a> <a data-t="816">but with the social infrastructure.</a> <a data-t="818">So, for example, to use that documentation,</a> <a data-t="820">I needed some credentials,</a> <a data-t="822">but I didn't know who I need to ask for the credentials.</a> <a data-t="826">And the person who gave the credentials</a> <a data-t="828">didn't know that I was in need of them.</a> <a data-t="831">So, we were basically testing in production.</a> <a data-t="835">And the problem of testing,</a> <a data-t="836">this is absolutely valid, you can't do this.</a> <a data-t="838">But the problem of testing in production</a> <a data-t="842">is that in some cases,</a> <a data-t="844">what happened is something failed.</a> <a data-t="847">And in this occasion, with perspective,</a> <a data-t="849">I know that what failed was the workflow.</a> <a data-t="852">But at that moment, I thought that I was failing</a> <a data-t="854">because I wasn't able to complete the task</a> <a data-t="857">that I was expected to do.</a></p>
