<h3>What is TensorFlow?</h3>

<p><a data-t="21">First, a quick recap on what's TensorFlow.</a> <a data-t="25">Well, TensorFlow is an open source platform for machine learning.</a> <a data-t="30">It's especially useful for deep learning.</a> <a data-t="34">So, it has fast implementations of most common operations in deep learning, like convolutions, for example.</a> <a data-t="44">And it's very efficient for both CPUs, GPUs, and even TPUs, which is the Google hardware for deep learning.</a> <a data-t="56">It provides automatic differentiation, which is also very useful for deep learning.</a> <a data-t="102">And it's really production-ready.</a> <a data-t="105">So, inside the TensorFlow ecosystem, there are multiple ways to deploy your models to even mobile devices, cloud platforms, and all of this.</a></p>

<p><a data-t="83">Well, TensorFlow is mostly a Python project with a lot of things in the ecosystem.</a> <a data-t="91">And I'm going to talk about the TensorFlow for R package.</a> <a data-t="95">And we are spread across multiple R packages.</a></p>

<p><a data-t="100">There is the TensorFlow R package, which provides mostly the basic access to the TensorFlow model and installation functions and things like this.</a> <a data-t="113">There is the Keras R package, which wraps the tf.keras model, which is the recommended way to start using TensorFlow.</a> <a data-t="124">It's like a higher-level API to use TensorFlow.</a> <a data-t="128">And </a><a href="/software/tfdatasets/">tfdatasets</a><a data-t="128">, which wraps the tf.data model, which uses it to load and preprocess that data for deep learning.</a> <a data-t="140">And we have a lot of other packages in the ecosystem, like tf.hub, tf.probability, tf.js, tf.autograph, and tf.auto.keras, which I'm going to talk about.</a></p>

<h3>TensorFlow 2.0 and eager execution</h3>

<p><a data-t="154">So, what's new?</a> <a data-t="156">There's a lot of packages, and things are moving fast.</a> <a data-t="160">So, we have support for TensorFlow 2.0.</a> <a data-t="166">So, before 2.0, TensorFlow was working like you had to define all your graph computation, and then you executed it.</a> <a data-t="182">And this is a programming model that is hard to think about, usually.</a> <a data-t="190">And with 2.0, it's much easier because of the eager execution.</a> <a data-t="196">So, before, you needed to write code like this.</a> <a data-t="199">So, you write all computation, and then you executed it.</a> <a data-t="204">And now, it works just like normal R arrays.</a></p>

<p><a data-t="211">Other changes in 2.0 are like API cleanup and things like this.</a> <a data-t="219">There is this new package by Thomas Kalinowski, which is called tf.autograph, which allows us to write conditional loops.</a> <a data-t="230">And like if, and while, and for statements.</a> <a data-t="235">Just like an R if statement, which is much easier.</a> <a data-t="240">And this is very efficient, also, for TensorFlow.</a> <a data-t="244">So, before, you needed to write code like this, with all functional conditionals.</a> <a data-t="252">And now, you can just write an if statement, and it will be converted to TensorFlow code.</a></p>

<h3>Feature spec interface in tfdatasets</h3>

<p><a data-t="262">In tfdatasets, which allows to load and preprocess that data, we have this new feature spec interface.</a> <a data-t="272">And this is useful for tabular data, and for using tabular data in deep learning models.</a> <a data-t="279">So, it works like, it has a similar interface to like recipes, the tidy models package.</a> <a data-t="290">And so, you just define what transformations you want to do on your tabular data.</a> <a data-t="296">And then, you fit this specification, which means it will find all possible vocabulary and like scaling,</a> <a data-t="305">or find the normalizing constants for your numeric variables, for example.</a> <a data-t="312">And then, you can just use this layer dense features function in Keras.</a> <a data-t="320">And this will make all the transformations directly on the TensorFlow graph.</a> <a data-t="328">So, it's much easier to deploy this kind of models.</a></p>

<p><a data-t="336">Next, we have some minor changes in tfdatasets.</a> <a data-t="340">Like, if you ever use a data set map, you needed to write a function to like per map.</a> <a data-t="349">And now, you can use the per style lambda functions, which like simplifies a bit the code.</a> <a data-t="359">And you can also pass, if you used tfdatasets before,</a> <a data-t="365">you needed to use the make iterator one-shot function to create an iterator for your TensorFlow data set.</a> <a data-t="374">And now, you can pass this directly to Keras, and it will just work.</a></p>

<h3>tf.hub and pre-trained models</h3>

<p><a data-t="382">Another cool new package is the tf.hub package,</a> <a data-t="387">which allows us to use any pre-trained models from this website here, tf.hub.dev.</a> <a data-t="396">And so, people train their models in large data sets,</a> <a data-t="404">and provide them the pre-trained models in a useful format.</a> <a data-t="410">And then, you can just use it as a new Keras layer.</a> <a data-t="415">So, you have the URL for a model.</a> <a data-t="419">In this case, it's the mobile net model.</a> <a data-t="423">And you have this layer hub function that just takes the model URL.</a> <a data-t="431">And then, so, this model is taking an image and returning a feature vector representing this image.</a> <a data-t="442">And you can just plug a dense layer, for example, to make a classification using these pre-trained models.</a> <a data-t="451">It also includes models for text and video.</a></p>

<p><a data-t="458">tf.hub also provides recipes integration, the tidy models recipes.</a> <a data-t="466">So, if you are familiar with recipes, you have all these step functions.</a> <a data-t="473">And there's a new step, pre-trained text embedding.</a> <a data-t="477">And you can just plug a pre-trained text model, for example.</a> <a data-t="495">For example, here, it's a pre-trained model in the Google News data set,</a> <a data-t="515">which is a very large data set of news.</a> <a data-t="520">And it will convert raw text to a feature vector that you can just use in your machine learning model.</a> <a data-t="529">And then, you can feed logistic regression, for example, using this feature vector.</a></p>
