<h3>Lessons learned</h3>

<p><a data-t="993">So what did we learn?</a> <a data-t="995">What did we learn from this experience,</a> <a data-t="997">from this experiment?</a> <a data-t="999">And the biggest takeaway for me was</a> <a data-t="1002">the power of domain-specific languages.</a> <a data-t="1006">Using SQL as this sort of intermediary</a> <a data-t="1010">between the chatbot, the human, and the data app</a> <a data-t="1014">worked out fantastically, because SQL</a> <a data-t="1017">is pretty easy for LLMs to write these days, at least</a> <a data-t="1021">the really frontier models.</a> <a data-t="1023">And it's relatively easy for humans to read and reason about.</a> <a data-t="1027">I mean, I don't do a lot of SQL.</a> <a data-t="1029">And I write Python and R code every single day.</a> <a data-t="1032">But I would rather be debugging those SQL statements</a> <a data-t="1036">than debugging a bunch of pandas code,</a> <a data-t="1040">or well, </a><a href="/software/dplyr/">dplyr</a><a data-t="1040"> is pretty good.</a> <a data-t="1042">But in general, it just constrains</a> <a data-t="1043">what's possible to express in a way that makes it</a> <a data-t="1046">easy to read and reason about.</a></p>

<p><a data-t="1049">And it's super powerful and flexible.</a> <a data-t="1051">Like, with just SQL, we were able to do all those things</a> <a data-t="1055">that you saw in this demo.</a> <a data-t="1057">And compared to executing straight Python or R that's</a> <a data-t="1061">being written by an LLM based on a prompt</a> <a data-t="1065">from a possibly malicious user and then executed on my server,</a> <a data-t="1069">yeah, I'd rather stick to SQL.</a></p>

<p><a data-t="1072">And this experiment was using SQL.</a> <a data-t="1076">But I'm really excited to experiment further with maybe</a> <a data-t="1080">we can use Vega-Lite as a DSL for creating visualizations,</a> <a data-t="1084">or use CSS Grid as a DSL for doing different kinds</a> <a data-t="1087">of dashboard layouts.</a> <a data-t="1088">So you could ask questions in the future like,</a> <a data-t="1091">please draw me a scatterplot.</a> <a data-t="1093">And instead of getting a bunch of Python or R,</a> <a data-t="1097">you get, again, something that's easy for you</a> <a data-t="1100">to read and reason about.</a></p>
