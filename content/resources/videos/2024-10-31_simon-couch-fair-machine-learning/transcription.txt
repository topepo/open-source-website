<h3>Defining fairness</h3>

<p><a data-t="288">So, the second hard part about machine learning fairness is defining what it actually is.</a> <a data-t="296">And to demonstrate that, we'll return to this example that I showed you earlier.</a></p>

<p><a data-t="301">So, I said earlier that this is a bad model, and I'll give you that.</a> <a data-t="301">And so, maybe our first reflex to make this model act more fairly is to make it act more performant by our usual metrics of model performance, like r-squared or root-mean-squared error.</a></p>

<p><a data-t="320">So, our proposal now for what a fair model is, is a model that's good by other usual measures of performance.</a></p>

<p><a data-t="329">Because those errors are super-correlated, we can apply calibration, and if we're lucky, we end up with predictions that look like this, where we have a fair model, predictions that look like this, where errors are independently and identically distributed, meaning that their mean invariance is the same across the distribution of that true outcome variable.</a> <a data-t="358">Okay, so maybe we feel that this machine learning model is more fair than the first one we saw.</a> <a data-t="366">Just to check that those errors are iid, we can take a look at the residuals, and it does seem like that distribution is similar across the true outcome variable.</a></p>

<p><a data-t="374">But let's not forget what those units are on the y-axis.</a> <a data-t="381">We're saying that a homeowner of a home that's $100,000 is going to be subject to similar residuals as a homeowner of a home that's $500,000 in their absolute magnitude.</a> <a data-t="394">So, they're just as likely to receive a residual of $50,000 plus or minus, and that error to them is probably much more impactful than it is to somebody on the other end of that distribution.</a></p>

<p><a data-t="408">So, to me, that maybe leads to a different proposal of what a fair machine learning model looks like in this context.</a> <a data-t="414">What if the percentage of the error was similar across the distribution of true outcome values?</a> <a data-t="420">So, we're talking about a 2% to 3% difference instead of a $10,000 to $20,000 difference.</a> <a data-t="426">We're speaking in terms of percentages rather than absolute magnitude.</a> <a data-t="432">If we're training a model where we can adjust the loss function that we're optimizing for, we might end up with a model that looks like this, where the absolute magnitude of the errors are larger for the more expensive homes, but the percentage error is similar across the distribution.</a></p>

<p><a data-t="452">But if I'm a homeowner of, in this case, a $500,000 home, and suddenly I'm subjected to these $200,000 residuals that I didn't used to see, that feels pretty unfair to me.</a> <a data-t="465">So, what we see here is that we start with these morally held beliefs about fairness, and we map them onto mathematical measures.</a> <a data-t="471">But the fact that they're mathematical measures doesn't necessarily mean they're any more compatible with each other than our morally held beliefs are.</a> <a data-t="485">So, defining fairness is hard.</a> <a data-t="485">Those definitions of fairness are not mathematically or morally compatible in general.</a></p>

<h3>The model is part of a larger system</h3>

<p><a data-t="492">A lot of software for machine learning fairness focuses on implementing as many fairness metrics as possible.</a> <a data-t="507">What I want to emphasize here is that machine learning metrics or fairness metrics only allow us to evaluate the behavior of a model, but we need to think about the whole system that the model is situated inside of.</a></p>

<p><a data-t="524">So, in this example that I was just talking about, how would these predictions of what the sale price of a home would be actually be used?</a> <a data-t="532">One tax system, I propose that you just take that assessed value for the sale price of the home and multiply it by a fixed percentage, and that's the property tax that you pay.</a> <a data-t="546">And to me, the most fair model that we've proposed so far under that system is one where we have a similar percentage error across the distribution.</a></p>

<p><a data-t="556">Where I live in Chicago, we call this the homeowner exemption, where if you live in the home that you own, you start with that assessed value and you subtract off some constant number, and then the results of that calculation is then multiplied by that fixed percentage, and that's the property tax that you pay.</a> <a data-t="575">So, if we think back to that graph, we're sort of mean shifting the errors down such that the owner of an $100,000 house is not as likely to experience that very positive residual or the overvaluation of their home.</a> <a data-t="589">If I am the owner of many $100,000 homes, I'm not allowed or allotted this exemption, but the errors are sort of averaged across the homes that I own, and so everybody ends up with a similar error on that end of the distribution.</a></p>

<p><a data-t="610">So, in that case, a model that's trained with a usual performance metric like R squared or root mean squared error, that feels fine to me.</a> <a data-t="618">But even if I could predict the sale price to the dollar, if last year my valuation was half of what it is this year, even though this year it's right on the dot, is that change in the assessed value unfair in itself?</a> <a data-t="633">Because I wasn't expecting evaluation that high.</a> <a data-t="641">So, these are all things that if we measured only with fairness metrics, we wouldn't be factoring in other components of this machine learning system that trigger this gut reaction that we have that reflects our morally held beliefs.</a></p>
