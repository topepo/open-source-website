<h3>What is MLOps?</h3>

<p><a data-t="1018">So a lot of these practices maybe weren't enough.</a> <a data-t="1021">And I also learned that if you develop models,</a> <a data-t="1024">you can operationalize them.</a> <a data-t="1026">And so this is making sure that your models are</a> <a data-t="1029">in some sort of larger ecosystem.</a> <a data-t="1034">Maybe that's on Git.</a> <a data-t="1036">Maybe that's shared with your teammates in some way,</a> <a data-t="1038">making sure that the model isn't just on your computer.</a></p>

<p><a data-t="1044">So I learned that you want to bring your model</a> <a data-t="1047">outside your computer.</a> <a data-t="1049">And you want to learn kind of the best practices</a> <a data-t="1051">of how to do that as well.</a> <a data-t="1054">And so those best practices for machine learning operations</a> <a data-t="1058">to operationalize your model are called MLOps.</a></p>

<p><a data-t="1062">And so there's a lot of hype around this word</a> <a data-t="1065">that I find online.</a> <a data-t="1067">It's like, I don't even know what MLOps is.</a> <a data-t="1069">You can read all of these different startups.</a> <a data-t="1072">And you read their documentation.</a> <a data-t="1073">And you don't really know what they're doing at the end of it.</a> <a data-t="1075">It's kind of stressful to get started.</a> <a data-t="1077">But here is a kind of one-liner.</a> <a data-t="1080">MLOps is a set of practices to deploy and maintain</a> <a data-t="1084">machine learning models in production</a> <a data-t="1086">reliably and efficiently.</a></p>

<blockquote><a data-t="1080">MLOps is a set of practices to deploy and maintain</a> <a data-t="1084">machine learning models in production</a> <a data-t="1086">reliably and efficiently.</a></blockquote>

<p><a data-t="1088">So where we had those best practices like setting a seed</a> <a data-t="1092">or splitting your training and testing data to build a model,</a> <a data-t="1096">MLOps are the same different set of practices.</a> <a data-t="1100">But it's a set of practices to make sure</a> <a data-t="1103">that the model, when it leaves your computer,</a> <a data-t="1106">can still be reproducible and shared</a> <a data-t="1109">in a safe way with others.</a></p>

<p><a data-t="1113">In my first job, when I was starting to learn about MLOps</a> <a data-t="1119">and learn how to deploy models, I</a> <a data-t="1122">was struggling because there was a set of tools that</a> <a data-t="1126">were really based on Kubernetes.</a> <a data-t="1130">And it felt like I almost had to be like a cloud expert</a> <a data-t="1135">and then also like a DevOps expert</a> <a data-t="1139">and maybe kind of a systems architect and also a data scientist.</a> <a data-t="1142">And that felt like way too much.</a></p>

<p><a data-t="1145">These practices are hard.</a> <a data-t="1147">It gets models off your computer into some sort of larger system.</a> <a data-t="1153">And sometimes tools don't necessarily</a> <a data-t="1156">feel ergonomic for data scientists.</a> <a data-t="1160">It felt like these tools were not really</a> <a data-t="1163">made for people with the skill set</a> <a data-t="1165">that I had coming from a purely data science background.</a></p>

<p><a data-t="1169">And that is kind of how vetiver was created.</a> <a data-t="1177">I ended up changing career paths from doing data science</a> <a data-t="1181">to building data science tools and making these MLOps tools</a> <a data-t="1183">to feel like an extension of what people are already doing.</a></p>

<p><a data-t="1188">So I do work for Posit, the company that was formerly known as RStudio.</a> <a data-t="1193">Vetiver is funded by Posit because I'm an affiliate for them.</a> <a data-t="1198">So it's no surprise that while I'm showing you Python code</a> <a data-t="1201">and while I primarily write in Python,</a> <a data-t="1204">there is also an R version of vetiver as well.</a> <a data-t="1207">The code actually looks almost identical.</a> <a data-t="1210">And that was on purpose, especially</a> <a data-t="1213">realizing that people sometimes might</a> <a data-t="1214">have to work in different languages</a> <a data-t="1216">or collaborate with people who are using different languages.</a> <a data-t="1221">We wanted to make tools that feel super easy to go back</a> <a data-t="1224">and forth so it doesn't feel like you</a> <a data-t="1225">have to start from zero if you're</a> <a data-t="1228">moving between languages.</a></p>

<p><a data-t="1231">So part of the idea of vetiver is</a> <a data-t="1233">that there's this life cycle where</a> <a data-t="1236">you collect and understand and evaluate your model.</a> <a data-t="1241">But it wants to fill in the gap of how</a> <a data-t="1244">to get this model out into the real world,</a> <a data-t="1247">making predictions from real world data</a> <a data-t="1250">in a really safe, reproducible way.</a></p>

<p><a data-t="1258">So some of the tasks that vetiver helps with</a> <a data-t="1267">is to version your models, to deploy your models,</a> <a data-t="1271">and to monitor your models.</a> <a data-t="1273">And those are the three main tasks vetiver focuses on.</a> <a data-t="1277">There are many, many other best practices</a> <a data-t="1281">that envelops in cases.</a> <a data-t="1284">But we really wanted to narrow down what, when I say we,</a> <a data-t="1286">it's me and the vetiver team, wanted</a> <a data-t="1291">to narrow down what we thought was important for people who</a> <a data-t="1293">are just starting to deploy models, who are just</a> <a data-t="1295">trying to figure out what this world is</a> <a data-t="1297">and what it means for them.</a></p>

<h3>Versioning models with pins</h3>

<p><a data-t="1301">So the first step of this is versioning.</a> <a data-t="1305">And when people think of versioning,</a> <a data-t="1307">it's normally in the context of Git,</a> <a data-t="1311">where you have some sort of central repository of files</a> <a data-t="1316">that people can read and write to, like push and pull to.</a> <a data-t="1318">But people actually version a lot of things.</a> <a data-t="1320">And they normally version things pretty badly, actually.</a></p>

<p><a data-t="1326">I think of how I am building models in my past.</a> <a data-t="1331">And I would version.</a> <a data-t="1332">I would start with a model.</a> <a data-t="1333">And I save it probably as a RDS file or as a Joblib file.</a> <a data-t="1338">So I save my model.</a> <a data-t="1340">I name it model, or maybe something a little better.</a> <a data-t="1343">And then maybe you update some of your features</a> <a data-t="1347">when you get new data.</a> <a data-t="1348">Then you re-save it, and it's model final.</a> <a data-t="1350">And then maybe you realize that it</a> <a data-t="1353">needs a little bit more tweaking.</a> <a data-t="1355">And so you have your model vinyl final.</a> <a data-t="1357">And you guys see where this goes.</a> <a data-t="1359">You end up with all of these versions of one object</a> <a data-t="1365">that maybe you need information from multiple files of these.</a> <a data-t="1371">But it's not really scalable.</a></p>

<p><a data-t="1386">Versioning really is the foundation for success</a> <a data-t="1390">in machine learning deployments.</a> <a data-t="1393">And also maybe not just for machine learning deployments,</a> <a data-t="1395">but a lot of reproducibility problems.</a> <a data-t="1398">A well-versioned system, a well-versioned file</a> <a data-t="1402">is normally going to be easier to share.</a> <a data-t="1405">It's going to be easier for you to share with yourself</a> <a data-t="1408">six months from now as well.</a></p>

<p><a data-t="1411">So when we think about our ideal versioning system,</a> <a data-t="1413">it would be nice if all of these files</a> <a data-t="1416">kind of lived in a central location</a> <a data-t="1420">so you don't have to search between different directories,</a> <a data-t="1423">maybe if they were discoverable by a team,</a> <a data-t="1426">so something that's not just on your computer,</a> <a data-t="1429">but something that can be hosted in a cloud environment</a> <a data-t="1431">between people.</a> <a data-t="1433">And it's also really nice</a> <a data-t="1435">if things can be loaded right into memory</a> <a data-t="1439">so you don't have to, like,</a> <a data-t="1441">the problem I have with Git sometimes with data</a> <a data-t="1444">is you'll have to go download the data</a> <a data-t="1447">if you can't in an easy way,</a> <a data-t="1449">load it onto your local computer,</a> <a data-t="1452">open up RStudio, load the data in that.</a></p>

<p><a data-t="1469">So something that helps vetiver out</a> <a data-t="1472">is actually another package called pins.</a> <a data-t="1476">And we're going to do, like, a very small pins side quest.</a> <a data-t="1479">You don't actually have to install pins to use vetiver,</a> <a data-t="1483">but they play together really nicely</a> <a data-t="1486">and they're kind of built alongside each other.</a></p>

<p><a data-t="1497">So pins is a package that publishes data,</a> <a data-t="1501">models, other R objects,</a> <a data-t="1503">and it makes it easy to share them</a> <a data-t="1505">between projects and with your colleagues.</a> <a data-t="1510">Some things that might be considered good uses for pins</a> <a data-t="1514">are, like, an ETL pipeline that will store a model</a> <a data-t="1520">or summarize data sets maybe once a day.</a></p>

<p><a data-t="1527">Pins, though, is not meant for multiple writing things back and forth.</a> <a data-t="1535">One thing that we see people try to do,</a> <a data-t="1538">and then I just kind of want to get ahead of,</a> <a data-t="1540">is don't try to make like a Google form with a pin.</a> <a data-t="1544">These are just files.</a> <a data-t="1546">So if you're trying to have multiple people</a> <a data-t="1548">like update the same file at once,</a> <a data-t="1550">it might get corrupt.</a> <a data-t="1551">But if one person or if, say, some sort of system</a> <a data-t="1555">is reading and writing this pin one at a time</a> <a data-t="1559">or you want to share something for multiple people to ingest,</a> <a data-t="1562">pins is good.</a> <a data-t="1563">Please don't make a Google format of it.</a> <a data-t="1564">I can tell you right now,</a> <a data-t="1568">I've had some horror stories from that.</a></p>

<p><a data-t="1571">But what works really well is people have like GitHub</a> <a data-t="1574">Actions running where it will update a notebook</a> <a data-t="1575">or something like that,</a> <a data-t="1579">and pins is kind of a perfect solution for them.</a> <a data-t="1582">And pins is built to be like super easy to use.</a></p>

<p><a data-t="1627">So it's good for pipelines where things is read and or updated.</a> <a data-t="1636">Pins is not really meant to have multiple people</a> <a data-t="1642">writing data to it at once.</a> <a data-t="1644">You can have as many people as you want reading from it,</a> <a data-t="1647">but you can't have multiple writes.</a> <a data-t="1649">That is kind of a downside of pins.</a></p>

<p><a data-t="1703">So this is the temporary board.</a> <a data-t="1718">If you want to change it to be a S3 board,</a> <a data-t="1722">it would just be board underscore S3</a> <a data-t="1725">or board underscore Azure, whatever.</a> <a data-t="1728">So it's super easy to like switch between boards</a> <a data-t="1730">or something like that.</a></p>

<p><a data-t="1735">And this is where vetiver comes in.</a> <a data-t="1739">So let's say we are in our data science workflow.</a> <a data-t="1742">We have created our model.</a> <a data-t="1744">It's called RFPipe.</a> <a data-t="1748">This is like a random forest pipeline that I created.</a> <a data-t="1750">And we're ready to deploy it.</a> <a data-t="1753">So we want to create this deployable model object</a> <a data-t="1755">called a vetiver model.</a> <a data-t="1757">And our inputs into this vetiver model</a> <a data-t="1761">are going to be the model itself that we've created.</a> <a data-t="1764">And then we're going to give it a name as well.</a></p>
