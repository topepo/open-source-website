<h3>The feedback bot</h3>

<p><a data-t="379">So come the next project that I think is,</a> <a data-t="382">at least for me, a little more inspired,</a> <a data-t="384">a little bit different than what I've seen out there,</a> <a data-t="386">and something that I've enjoyed working on,</a> <a data-t="389">and now I can say that my students</a> <a data-t="391">have sort of enjoyed it as well,</a> <a data-t="392">so I'll share some details about that.</a></p>

<p><a data-t="452">Leveraging, I already have to write</a> <a data-t="455">really meticulous rubric items.</a> <a data-t="457">My university uses Gradescope, which if you've used it,</a> <a data-t="460">you have to have well-defined rubric items.</a> <a data-t="463">And I have a large number of TAs</a> <a data-t="466">with varying experience levels,</a> <a data-t="468">grading a large number of students' work,</a> <a data-t="470">so I need to make sure that they are sticking</a> <a data-t="473">to a very well-defined rubric.</a> <a data-t="475">And these are a pain to write, to be perfectly honest,</a> <a data-t="478">but that work is already getting done no matter what,</a> <a data-t="481">so I've been thinking, how else can we leverage this?</a></p>

<p><a data-t="485">And also, hopefully, shifting resources.</a> <a data-t="488">Could AI also help TAs redistribute their time a little bit</a> <a data-t="492">towards higher value and more enjoyable for them</a> <a data-t="495">touch points with students?</a> <a data-t="497">And away from repetitive and error-prone tasks,</a> <a data-t="501">which is grading, most of which go unread.</a> <a data-t="504">Because guess what happens when the homework</a> <a data-t="506">is done mostly by AI?</a> <a data-t="508">No one reads the feedback because they're like,</a> <a data-t="510">well, I don't know, I don't actually know</a> <a data-t="512">what's in there in the first place.</a> <a data-t="514">So now, a week after I turn this in,</a> <a data-t="516">why would I read the feedback written for me?</a></p>
