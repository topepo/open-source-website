<p><a data-t="1552">Let me tell you about our PBC report.</a> <a data-t="1555">This is a report that Posit makes every two years</a> <a data-t="1558">that tracks our progress toward our mission.</a> <a data-t="1563">And it is primarily a PDF document.</a> <a data-t="1567">Here's the cover of that document.</a> <a data-t="1569">Inside, there's some pages</a> <a data-t="1571">that have this dark blue hex background.</a> <a data-t="1573">There's some pages that have a white background</a> <a data-t="1575">but with some full bleed photos.</a> <a data-t="1577">Text is laid out in two columns.</a> <a data-t="1579">Some pages have a banner at the top.</a> <a data-t="1581">There's plots in there.</a> <a data-t="1584">This year, we made that report with Quarto.</a></p>

<p><a data-t="1588">And if you've made PDFs with Quarto before,</a> <a data-t="1590">you know that they go through LaTeX on that journey.</a> <a data-t="1594">But we did not make this report by going through LaTeX.</a> <a data-t="1599">We used Typst.</a> <a data-t="1601">Typst is a new open source framework for producing PDFs.</a> <a data-t="1605">And it's already supported by Quarto.</a> <a data-t="1608">To use Typst, you simply swap out format PDF for format Typst.</a></p>

<p><a data-t="1615">We think Typst is the future of PDF</a> <a data-t="1617">because it's really easy to get started.</a> <a data-t="1620">Quarto bundles Typst,</a> <a data-t="1622">so there's no separate install to manage.</a> <a data-t="1625">You can do it today.</a> <a data-t="1629">Typst is really fast.</a> <a data-t="1630">So rendering a PDF via Typst</a> <a data-t="1633">is much faster than rendering a PDF via LaTeX.</a> <a data-t="1637">We think it's easier to learn.</a> <a data-t="1639">There's great documentation on the Typst site.</a> <a data-t="1642">And the syntax feels a lot more familiar</a> <a data-t="1644">to people that are coming from languages</a> <a data-t="1646">like R or Python or Julia.</a> <a data-t="1648">And it's very customizable.</a> <a data-t="1651">So our report being an example of that.</a> <a data-t="1653">It took a little work to get that customization,</a> <a data-t="1655">but you can produce really beautiful documents.</a></p>

<p><a data-t="1660">So we think it's the future of PDF.</a> <a data-t="1661">It maybe isn't quite the present of PDF.</a> <a data-t="1664">Typst is still developing,</a> <a data-t="1665">and our support in Quarto is still developing too.</a> <a data-t="1669">But I do want to show you something really cool</a> <a data-t="1672">you can do in Typst right now.</a></p>

<p><a data-t="1675">So take a look at this table.</a> <a data-t="1676">This is an HTML table.</a> <a data-t="1678">It's built using the GreatTables package in Python.</a> <a data-t="1680">And an example from Grant Chalmers.</a> <a data-t="1683">It's on the GreatTables documentation site.</a> <a data-t="1687">And I think it's a beautiful table.</a></p>

<p><a data-t="1691">And GreatTables and </a><a href="/software/gt/">GT</a><a data-t="1691"> and a lot of table packages</a> <a data-t="1694">really focus on absolutely beautiful HTML output.</a> <a data-t="1698">So what happens if you switch to Typst?</a> <a data-t="1703">Well, behind the scenes,</a> <a data-t="1704">Quarto takes all the CSS</a> <a data-t="1706">that makes that HTML table really beautiful,</a> <a data-t="1709">and it converts it to Typst properties.</a> <a data-t="1713">And what that means is when you get your PDF out,</a> <a data-t="1716">that table looks just as good as it did in HTML,</a> <a data-t="1720">but now it's in a PDF.</a> <a data-t="1723">And not only that, like Typst is sort of good enough</a> <a data-t="1726">to know that, hey, this table's pretty long.</a> <a data-t="1728">It's not going to fit on one page.</a> <a data-t="1730">I'll break it over two pages,</a> <a data-t="1732">and I'll repeat the headers and footers</a> <a data-t="1733">and make it all just work for you.</a> <a data-t="1736">It's really cool.</a></p>

<h3>WebR and Quarto Live</h3>

<p><a data-t="1868">Hello, my name is George Stagg</a> <a data-t="1870">and I'm the lead developer of the WebR system,</a> <a data-t="1872">a way to run R code directly inside your web browser.</a></p>

<p><a data-t="1877">So I've been working for Posit now for about two years</a> <a data-t="1881">and I'm still really excited to tell people that.</a> <a data-t="1884">Posit's mission of making great open source software</a> <a data-t="1887">really resonates with me.</a> <a data-t="1889">I've been making open source software for over 10 years now.</a> <a data-t="1901">I started in research software developing for HPC</a> <a data-t="1906">and over that time my approach to open source</a> <a data-t="1909">has always been no matter what you do,</a> <a data-t="1911">no matter what decisions I make,</a> <a data-t="1913">they must always be in the interest of the user.</a></p>

<p><a data-t="1916">And it was that philosophy that originally led me</a> <a data-t="1918">to create WebR.</a> <a data-t="1920">At the time I was supporting a group of students</a> <a data-t="1922">who were remote up and down the UK</a> <a data-t="1925">and we needed to evaluate R code</a> <a data-t="1928">in a consistent environment and capture those results.</a> <a data-t="1931">To me this felt like the best solution for the job.</a></p>

<p><a data-t="1935">If you've not seen WebR before,</a> <a data-t="1937">here's an example of a WebR application.</a> <a data-t="1940">It's a website you can visit</a> <a data-t="1941">and when you do an R console pops up</a> <a data-t="1944">and you can type code in and you can do visualizations</a> <a data-t="1947">and you can really experiment with R</a> <a data-t="1950">in a very easy way, very quickly.</a> <a data-t="1952">This is really great for new users of R</a> <a data-t="1955">who don't have something like RStudio installed.</a> <a data-t="1957">And Hadley earlier talked about the idea</a> <a data-t="1959">of code first data science.</a> <a data-t="1961">What could be more code first than opening a web browser</a> <a data-t="1964">and typing in some R code?</a></p>

<p><a data-t="1966">So how does this work?</a> <a data-t="1968">The main technology is called WebAssembly</a> <a data-t="1972">and what WebAssembly is,</a> <a data-t="1973">is a portable binary code format.</a> <a data-t="1975">So what does that mean?</a> <a data-t="1978">Really this just means that</a> <a data-t="1979">if you take a piece of software and compile it,</a> <a data-t="1981">you get a big list of instructions to execute.</a> <a data-t="1984">The word portable here</a> <a data-t="1985">just means that those instructions,</a> <a data-t="1987">the binary is the same on every machine.</a> <a data-t="1990">And that's done by running that binary inside a web browser.</a> <a data-t="1993">It's really designed to run</a> <a data-t="1995">high performance applications in this way.</a> <a data-t="1997">It does work with most modern web browsers on many devices.</a></p>

<p><a data-t="2002">And one of the things it's really great at is security</a> <a data-t="2006">because it's really designed to run on the web.</a> <a data-t="2008">It runs your code in a containerized sandbox environment.</a> <a data-t="2011">In a web browser,</a> <a data-t="2014">you interact with WebAssembly through JavaScript</a> <a data-t="2017">and that has positives and negatives.</a> <a data-t="2021">It means that WebR itself is a JavaScript application.</a> <a data-t="2024">So you need to write JavaScript to use it.</a> <a data-t="2027">And a positive of that is that</a> <a data-t="2029">it means it works great with other web technologies.</a> <a data-t="2032">A downside of that is that</a> <a data-t="2033">you probably don't love writing JavaScript code.</a> <a data-t="2036">At least not in the same way</a> <a data-t="2037">that the people in front of me love writing R code</a> <a data-t="2040">or they love writing Python code.</a></p>

<p><a data-t="2042">So for that reason,</a> <a data-t="2044">in addition to working on WebR,</a> <a data-t="2046">we produce tools built upon WebR to make your job easier.</a></p>

<p><a data-t="2053">Tools like </a><a href="/software/shinylive/">ShinyLive</a><a data-t="2053">.</a> <a data-t="2055">At PositConf last year,</a> <a data-t="2057">Joe Tang had a talk introducing ShinyLive for R</a> <a data-t="2060">built upon the WebR technology.</a> <a data-t="2063">And over the years since then,</a> <a data-t="2065">we've been continuing to update the system.</a> <a data-t="2067">There's been several releases.</a> <a data-t="2069">And you can use ShinyLive now in a web browser</a> <a data-t="2073">to interact with live Shiny examples.</a> <a data-t="2076">You can export existing apps using an R package.</a> <a data-t="2080">Or you can use a Quarto extension</a> <a data-t="2081">to embed Shiny applications</a> <a data-t="2083">directly into your Quarto documents.</a></p>

<p><a data-t="2086">Again, the website example is great.</a> <a data-t="2088">You can go on to a URL and type some Shiny code</a> <a data-t="2091">and it just appears.</a> <a data-t="2093">And I've actually seen this used at workshops</a> <a data-t="2096">to teach people how to use Shiny</a> <a data-t="2097">and to share examples.</a></p>

<p><a data-t="2100">And that takes me on to one of the demographics</a> <a data-t="2104">that have really taken ShinyLive and run with it.</a> <a data-t="2106">And that's the demographic of educational content educators.</a> <a data-t="2110">They've really embraced the idea</a> <a data-t="2113">of WebAssembly-powered educational content.</a> <a data-t="2117">And for that reason,</a> <a data-t="2118">we thought to ourselves,</a> <a data-t="2119">well, is there more we can do?</a> <a data-t="2121">Can we get more WebAssembly into Quarto documents?</a> <a data-t="2125">And the answer to that is yes.</a> <a data-t="2126">And I've been working on a new extension</a> <a data-t="2129">for Quarto over the last six months.</a></p>

<p><a data-t="2134">And we're calling it Quarto Live.</a> <a data-t="2136">Quarto Live is a way to get interactive code examples</a> <a data-t="2141">in R in your Quarto documents.</a> <a data-t="2144">You can directly interact with that code</a> <a data-t="2147">and you can create code exercises</a> <a data-t="2149">so that if you're an educator,</a> <a data-t="2151">you can create very easy ways</a> <a data-t="2153">for students to interact with your code</a> <a data-t="2156">and get quality feedback.</a></p>

<blockquote><a data-t="2136">Quarto Live is a way to get interactive code examples</a> <a data-t="2141">in R in your Quarto documents.</a> <a data-t="2144">You can directly interact with that code</a> <a data-t="2147">and you can create code exercises</a> <a data-t="2149">so that if you're an educator,</a> <a data-t="2151">you can create very easy ways</a> <a data-t="2153">for students to interact with your code</a> <a data-t="2156">and get quality feedback.</a></blockquote>

<p><a data-t="2160">We've designed Quarto Live to be really, really easy</a> <a data-t="2163">to use both for authors</a> <a data-t="2165">and for the students and learners.</a></p>

<p><a data-t="2168">So what does it look like?</a> <a data-t="2170">So let's say you have a Quarto document</a> <a data-t="2171">and it looks a bit like this.</a> <a data-t="2172">You've got some R code</a> <a data-t="2174">and it's producing a visualization.</a> <a data-t="2175">Here we have some </a><a href="/software/air/">air</a><a data-t="2175"> quality data</a> <a data-t="2177">from 1970s New York</a> <a data-t="2180">and it's plotting some temperatures</a> <a data-t="2181">and some ozone levels.</a> <a data-t="2183">Now, let's say you want to turn this</a> <a data-t="2185">into an interactive example.</a> <a data-t="2186">Much like in Charlotte's examples,</a> <a data-t="2188">you change the format into a live format</a> <a data-t="2192">and you change your R block into a web R block.</a> <a data-t="2195">And if you have the Quarto Live extension installed,</a> <a data-t="2197">that's it.</a> <a data-t="2197">You do nothing else.</a> <a data-t="2198">It just, it works.</a></p>

<p><a data-t="2201">You can see the output looks pretty much identical.</a> <a data-t="2204">That's by design.</a> <a data-t="2205">And the only real difference</a> <a data-t="2207">is that your source code above your output</a> <a data-t="2209">is now an editable code block</a> <a data-t="2211">that you can interact with</a> <a data-t="2212">and click the buttons to rerun your code.</a> <a data-t="2215">Now, this is great for students</a> <a data-t="2216">because someone who's interacting</a> <a data-t="2220">with this content for the first time</a> <a data-t="2222">may be interested.</a> <a data-t="2223">They want to know more.</a> <a data-t="2224">They want to play with this example.</a> <a data-t="2226">Previously, they would have to copy that code</a> <a data-t="2227">and open our studio</a> <a data-t="2228">and switch to a different window.</a> <a data-t="2230">But now they can literally just change the code</a> <a data-t="2233">to plot wind instead of temperature,</a> <a data-t="2235">hit the button,</a> <a data-t="2235">and it's there in the same page</a> <a data-t="2237">inside their document.</a></p>

<p><a data-t="2240">By the way,</a> <a data-t="2241">one of the things I'm really excited about this</a> <a data-t="2243">is that those are no longer pictures</a> <a data-t="2245">that are distributed with your document.</a> <a data-t="2247">They're generated dynamically on the fly</a> <a data-t="2250">every time you open that document.</a> <a data-t="2252">That means they're kind of reproducible by default, right?</a> <a data-t="2255">Because the fact that you can see that image at all</a> <a data-t="2258">means the R code in your document</a> <a data-t="2260">has executed successfully</a> <a data-t="2262">and produced some output.</a></p>

<p><a data-t="2266">If you're an educator</a> <a data-t="2267">and you want to augment these code examples</a> <a data-t="2270">into an exercise,</a> <a data-t="2271">you give it an exercise label.</a> <a data-t="2274">At that point,</a> <a data-t="2274">you can then add things</a> <a data-t="2276">to make that exercise more and more interactive.</a> <a data-t="2279">Here's an example</a> <a data-t="2280">where we're asking a student to plot some data, right?</a> <a data-t="2284">And what we're going to do</a> <a data-t="2285">is we're going to augment that with some setup code.</a> <a data-t="2287">So we create another WebR block.</a> <a data-t="2289">We tell it it's a setup block</a> <a data-t="2291">and this is the exercise that it's linked to.</a> <a data-t="2294">And what we've done</a> <a data-t="2295">is we've set up that data set for the students</a> <a data-t="2297">so they don't have to worry about it.</a> <a data-t="2298">They don't have to worry about loading that data.</a> <a data-t="2301">It's just there.</a> <a data-t="2301">They can just directly answer that question</a> <a data-t="2304">without having to worry about the extra parts.</a> <a data-t="2305">And that's because a setup block</a> <a data-t="2306">will run automatically</a> <a data-t="2308">every time when that student hits the run button.</a></p>

<p><a data-t="2313">So let's say the student tries to run their code.</a> <a data-t="2315">They see, okay, there's a blank.</a> <a data-t="2316">I need to answer this question.</a> <a data-t="2318">But they're struggling.</a> <a data-t="2319">They're new to this.</a> <a data-t="2320">They're not quite sure what's going on.</a> <a data-t="2321">Can we help?</a> <a data-t="2323">Well, we could add a hint</a> <a data-t="2325">and we do that using a quarter block.</a> <a data-t="2327">Here's a quarter block with the class hint</a> <a data-t="2329">and we've, again, linked it to our exercise</a> <a data-t="2331">using that same shared exercise label.</a> <a data-t="2334">Inside there, you can write your hint</a> <a data-t="2336">and you'll notice it's not added</a> <a data-t="2337">to the document straight away.</a> <a data-t="2339">Instead, QuartoLive adds a button</a> <a data-t="2341">to the exercise saying hint</a> <a data-t="2343">and when the student clicks it,</a> <a data-t="2345">then they get the hint text containing your block.</a> <a data-t="2348">And if the student's still struggling,</a> <a data-t="2349">you can do the same thing with a solution block.</a> <a data-t="2351">You can write your Quarto code inside a solution block</a> <a data-t="2354">and when the student hits that button,</a> <a data-t="2355">they see that solution text.</a></p>

<p><a data-t="2358">What's really great about this</a> <a data-t="2359">is that this is just Quarto syntax.</a> <a data-t="2361">There's nothing new here.</a> <a data-t="2363">All of the tools that you know how to write in Quarto</a> <a data-t="2367">to make great content will work inside these blocks.</a> <a data-t="2370">Tools like call-out blocks,</a> <a data-t="2372">tools like collapsible panels,</a> <a data-t="2375">syntax highlighted code blocks,</a> <a data-t="2378">code annotations to really explain</a> <a data-t="2380">how a piece of code works.</a> <a data-t="2381">All of these things work inside a solution block</a> <a data-t="2384">and a hint block.</a> <a data-t="2384">So you can make all of these tools</a> <a data-t="2386">work with your exercises straight away.</a></p>

<p><a data-t="2389">Now that the student has seen the solution,</a> <a data-t="2391">they can then obviously continue</a> <a data-t="2392">to interact with the code exercise,</a> <a data-t="2396">paste in that code and run it</a> <a data-t="2397">and ah, I'd recognize that logo anyway.</a></p>
