<h3>What is shinylive?</h3>

<p><a data-t="27">So what is shinylive?</a> <a data-t="27">When we do our typical Shiny deployment, we have one centralized server, or many centralized</a> <a data-t="34">servers, but we have a server that is detached from the user's computers.</a> <a data-t="39">And this is where all your beef of your server computations are done, and it makes it so</a> <a data-t="47">that you can have your secrets for your databases and everything like that.</a> <a data-t="51">And then the user just gets this very thin UI within their browser.</a></p>

<p><a data-t="57">But shinylive kind of flips that up on its head and says, hey, I actually want to take</a> <a data-t="63">this computation, and I want to distribute it to all of the users so that I don't have</a> <a data-t="69">a centralized server.</a> <a data-t="70">But instead, I send R, and I send the computations to the server's browser.</a> <a data-t="76">And then it runs within their own machine on their own compute cycles, and they can</a> <a data-t="82">look at all of the code within the app there.</a></p>

<p><a data-t="87">First off, saying this makes it seem like it's pretty trivial to do, but there's been</a> <a data-t="92">years of work been done to try to make this possible to even allow R to run in the</a> <a data-t="97">browser.</a> <a data-t="103">If I would have told myself 10 years ago that R could run in the browser, I</a> <a data-t="107">would have laughed at you, and I still laugh because it's a crazy idea that just happens</a> <a data-t="111">to work today.</a></p>

<blockquote><a data-t="103">If I would have told myself 10 years ago that R could run in the browser, I</a> <a data-t="107">would have laughed at you, and I still laugh because it's a crazy idea that just happens</a> <a data-t="111">to work today.</a></blockquote>

<p><a data-t="114">And now, since R is within each user's browser, we don't need that centralized server.</a> <a data-t="119">It is now distributed and infinitely horizontally scalable as long as you can send out</a> <a data-t="125">the static resources.</a></p>
