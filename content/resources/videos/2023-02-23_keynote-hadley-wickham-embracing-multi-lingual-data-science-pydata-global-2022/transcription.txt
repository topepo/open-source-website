<h3>Methods belong to functions, not classes</h3>

<p><a data-t="1077">Next, I wanted to talk a little bit more about the pipe.</a> <a data-t="1077">Because if you look at this pipeline that I show you earlier and kind of squint your eyes a little bit, you might say, oh, well, couldn't we just write that using like method chaining with Python?</a> <a data-t="1090">Like very, very similar, right?</a> <a data-t="1090">Just basically changing that pipe for a dot at each point.</a> <a data-t="1098">But there's something that's profoundly different about that pipe.</a> <a data-t="1102">And that is at each point, each element of that pipeline can come from a different package.</a></p>

<p><a data-t="1120">And the thing that makes this work is kind of surprising and mind blowing.</a> <a data-t="1120">And when I first encountered R, it took me several years to really understand.</a> <a data-t="1125">And that is that in R, methods belong to functions, not classes.</a> <a data-t="1135">So I don't want to go into all the details of this, but if you kind of think about object-oriented systems, every object-oriented system has these functions that you call and classes that you use and the methods are sort of the intersection.</a> <a data-t="1146">You can think of it as sort of a 2D table.</a> <a data-t="1146">That you've got functions and you've got classes.</a> <a data-t="1152">And most programming languages kind of organize their OOP system like row-wise and R kind of organizes it column-wise.</a></p>

<p><a data-t="1159">And for a long time, this was pretty quirky.</a> <a data-t="1159">Like two languages used this, Common Lisp and R.</a> <a data-t="1161">But recently, another programming language has come along that also uses this style of programming language, and that's Julia.</a> <a data-t="1172">And I think it's one of the things that Julia folks really attribute to the ease of reusing code in Julia is this style of object-oriented programming.</a> <a data-t="1179">It just seems to fit particularly with kind of like scientific computing and data science so well.</a></p>

<p><a data-t="1193">So, the consequences of this are pretty profound.</a> <a data-t="1193">This ability component of that pipeline can come from a different package.</a> <a data-t="1200">It means that we don't need these really big kind of monolithic packages like Pandas, like Scikit-learn.</a> <a data-t="1204">That when you're assembling a pipeline, you can assemble pieces from all over the place, which means the life of software developers is much easier because they can develop these small reusable pieces in isolation without having to consider this giant other package.</a></p>

<h3>Package development and CRAN</h3>

<p><a data-t="1242">Okay, so we've talked about two weird things, about two surprising things about R.</a> <a data-t="1246">The first one, first-class abstract and vector use in environments, which allows you to create these domain-specific languages.</a> <a data-t="1252">For data science, we have the fact that methods belong to functions, not classes, which allows us to write pipelines that combine tools from many, many different places.</a> <a data-t="1259">And the last thing I want to talk about is kind of hard to describe, kind of crazy at first sight, and that's like when you install a package in R, generally like 99% of the time, you just install it into some like a global repository of packages on your computer.</a> <a data-t="1278">Because when you're working with R, most projects don't need a virtual environment, and most people using R can just share the same set of packages across all of their projects, and things by and large just work.</a></p>

<h3>Open source tools for multilingual data science</h3>

<p><a data-t="1877">So to finish up, I want to talk about a few of the open source tools that we're currently working on that are specifically designed to support multiple languages.</a> <a data-t="1886">And the first one I want to talk about is </a><a href="/software/quarto/">Quarto</a><a data-t="1886">, which is a tool designed to help you publish stuff, to make documents.</a> <a data-t="1892">The key idea of Quarto is you're going to start with some document.</a> <a data-t="1899">Those two types of documents you might start with.</a> <a data-t="1899">You might start with something like a Jupyter Notebook, where you have text, where you have code, and you have the results of that code.</a> <a data-t="1910">Or you might start with something like an R Markdown, more like an R Markdown document, now a Quarto Markdown document that just has the text and the code, and you need to run that code.</a> <a data-t="1922">And that code can be in Python or R or Julia or, you know, pretty much anything.</a> <a data-t="1928">In both cases, we're going to end up with a Markdown document.</a></p>

<p><a data-t="1928">Markdown is a really great, we think, like, format because it's a text format.</a> <a data-t="1935">It's easily diffable.</a> <a data-t="1935">You can check it as a commit, you can share it with people, easy to write by hand if you need to.</a> <a data-t="1941">But even more importantly than that, Markdown gets you into the Pandoc ecosystem.</a> <a data-t="1947">And if you haven't heard of Pandoc before, it's this really amazing tool for doing document transformations, basically.</a> <a data-t="1953">And by building on top of Pandoc, that allows Quarto to produce things like Word documents and PowerPoint presentations and PDFs and HTML files.</a> <a data-t="1966">You can even publish, kind of fully roundtrip, to publish a notebook from the end of it, if your input is something else, or even in some cases it's useful to kind of roundtrip, because some of the tools I'll talk about shortly.</a></p>

<p><a data-t="2006">So Quarto, like, helps you turn your words and your code into some document that you can share with others.</a> <a data-t="2011">It also provides tools for when you've got, like, a bunch of documents and you want to convey some bigger story, whether that's through a website or a blog or a book.</a></p>

<p><a data-t="2025">Now, if you're an R user, this might look kind of familiar.</a> <a data-t="2025">And that's no surprise, because Quarto was very much inspired by our 10 years of experience with R Master, which really, I think, convinced us that this core idea of literate programming for data science is the right approach.</a> <a data-t="2038">That you want to be able to create reproducible documents that, when you, when the data changes, you can rerun your code, combine it with your pros, and get a new Word document or HTML page or PowerPoint slide.</a> <a data-t="2058">But in the time that since R Markdown was created, like, the number of languages used for data science has increased.</a> <a data-t="2062">It continues to increase.</a></p>

<p><a data-t="2068">And so while, like, R Markdown did support other languages, you could absolutely use Python inside of R Markdown today, you still have to have R installed.</a> <a data-t="2075">And that's not, like, super compelling, right?</a> <a data-t="2075">If you're a Python user and you want to create a nice document, you don't want to install R.</a> <a data-t="2079">So the goal of Quarto is it's ground up reimagining.</a> <a data-t="2084">We're going to make it fully multi-language, fully multi-engine.</a> <a data-t="2091">So regardless of what language you're doing your data science in, whether it's R or Python or Jupyter or something that doesn't even exist today, Quarto is going to support that.</a> <a data-t="2100">Quarto is going to be the engine that helps you combine your pros and your code and your results to create some beautiful documents to share with others.</a></p>

<p><a data-t="2117">And generally, one of the goals of Quarto is to help make it easier to write scientifically, to do scientific writing inside of Markdown.</a> <a data-t="2122">And some of you might have seen this kind of humorous depiction of the tools you might currently use to write scientific documents.</a> <a data-t="2129">So, you know, you can start with, you might use Word, which starts really simple, but rapidly gets very, very complex.</a> <a data-t="2142">As soon as you add in citations or complex figures, it gets really, really complicated, really, really quickly.</a> <a data-t="2147">You could, of course, use LaTeX, which starts hard and stays hard.</a> <a data-t="2155">Once you've got the ability to do a LaTeX document, you've mastered the art of Googling for LaTeX syntax, it doesn't get that much harder, but very, very hard and not accessible to most people.</a> <a data-t="2169">So Markdown is a great way to start, but if you're using Markdown for scientific writing, you kind of rapidly hit some limitations, like you're going to have to bring your own tooling for combining multiple files, you're going to have to figure out the new syntax for the various types of things you want to do in scientific writing.</a></p>

<p><a data-t="2188">And so the idea of Quarto, the aspiration for Quarto is to really start as easy as Word and only get a little bit harder over time.</a> <a data-t="2197">And Quarto does that by building a bunch of extensions on top of Pandoc, extensions for things like citations.</a> <a data-t="2204">So you can just copy and paste a DOI in and you automatically get a nicely formatted citation.</a> <a data-t="2211">Rich cross-references between longer documents, equations using LaTeX math, complex diagrams and multi-fixed panels, all of these things that you need when writing, creating scientific documents.</a></p>

<p><a data-t="2222">And the thing that's great about all of these things is they're built on top of this extension mechanism provided by Pandoc.</a> <a data-t="2234">And the neat thing about this is that this extension mechanism is written on top of the abstract syntax tree of the Markdown documents, written on top of the semantics of the document, not on the text format.</a> <a data-t="2245">So if you've ever tried to do some of these things, introduce new syntax by hand with regular expressions, it starts off like it's doable if you want to add one extension, maybe.</a> <a data-t="2255">If you want to add two, you start to hit all of these weird edge cases and it's very, very hard to get fully correct.</a> <a data-t="2261">So Pandoc provides this really cool extension mechanism.</a> <a data-t="2266">The downside of that extension mechanism is you're going to have to learn a little bit about Lua.</a> <a data-t="2272">It's a relatively rare programming language, but it's designed to be used in exactly this situation, embedded inside other tools.</a> <a data-t="2278">So if you've used any other programming language before, whether it's R or Python or something else, you're going to be able to pick up Lua very, very quickly.</a></p>

<p><a data-t="2289">And then on top of those Pandoc extensions, Quarto provides an extension mechanism that makes it really easy to document your extension, to publish your extension, to share your extension with other people.</a> <a data-t="2314">So once you've figured out how to do this, the whole community can build on top of it.</a> <a data-t="2319">So here are some examples of Quarto used today.</a> <a data-t="2328">Personally, I use it for pretty much everything I write, all the books I write are in Quarto.</a> <a data-t="2333">But in the Python ecosystem, you might be interested in these two projects that use it.</a> <a data-t="2333">Wes McKinney's Python for Data Analysis uses it, and the Practical Deep Learning course from Fast.ai uses it as well.</a> <a data-t="2344">And if you want to see more examples, there are many, many more, of course, on the Quarto website.</a></p>

<h3>Vetiver for MLOps</h3>

<p><a data-t="2692">The last thing I want to talk about today is Vetiver, which is a set of tools for R and Python for MLOps.</a> <a data-t="2699">So that of course begs the question, what is MLOps?</a> <a data-t="2699">And there are a lot of competing and conflicting definitions in play today, but kind of our framing of MLOps is a set of practices to deploy and maintain ML models in production, both reliably and efficiently.</a> <a data-t="2712">And here we're thinking mostly about kind of human efficiency.</a> <a data-t="2712">How do we make this as easy as possible for the data scientists?</a></p>

<p><a data-t="2724">So Vetiver stays out of the initial model building process where you collect data and understand and clean it and visualize it and try a bunch of models, evaluate them.</a> <a data-t="2736">You bring your own, whether that's XGBoost in R or Scikit-learn in Python, Vetiver is going to support you no matter what you do.</a> <a data-t="2742">Vetiver starts to play when you want to get that model into production.</a> <a data-t="2746">And if you're going to put a model in production, we really believe the first thing you need to do is version that model.</a> <a data-t="2750">And that's really important because it allows you to track how it changes over time.</a> <a data-t="2755">And if you accidentally screw up, you can easily revert the previous version.</a></p>

<p><a data-t="2763">Then Vetiver is going to help you deploy the model.</a> <a data-t="2763">In the most general case, this is basically developing a Docker, creating a Docker container that runs a server that exposes a JSON API that other services can hit with data and get predictions back out.</a> <a data-t="2776">If you're using PositConnect, our tool that helps data scientists communicate with others in the organization, this all gets handled behind the hood and you can deploy your model with one line of code.</a> <a data-t="2787">Importantly, this doesn't just deploy the model, it also deploys some documentation.</a> <a data-t="2794">Currently, the documentation for the API, like how you actually use it.</a> <a data-t="2799">But in the future, this also is going to support publishing documentation about the model itself through the popular model card framework.</a> <a data-t="2809">And then finally, you're going to want to monitor that model.</a> <a data-t="2809">And because you've got all these versions of the previous model, very easy to track what's going on, what's happening to your model.</a></p>

<p><a data-t="2820">If that sounds interesting, I highly recommend attending the talk by my colleague, </a><a href="/people/isabel-zimmerman/">Isabel Zimmerman</a><a data-t="2820">, Practical ML Ops with Vetiver Models, which is happening in one day, two and a half hours ago.</a></p>
