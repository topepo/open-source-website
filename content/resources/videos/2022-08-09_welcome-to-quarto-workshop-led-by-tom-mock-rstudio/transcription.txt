<p><a data-t="2342">But then we can also add text to this.</a> <a data-t="2344">So rather than just executing code, we can add Markdown text and do things like adding</a> <a data-t="2348">alternative text to images or bolding the text or italicizing text or adding headers or</a> <a data-t="2353">subsections.</a> <a data-t="2355">But overall, you can still read this and it's plain text format, but it's indicating this</a> <a data-t="2360">is the Markdown or the markup to generate an output format in bold or italics or other</a> <a data-t="2365">things.</a></p>

<p><a data-t="2367">The metadata or that YAML at the top is what's controlling the overall rendering</a> <a data-t="2372">process.</a> <a data-t="2373">So this tells you what is the output format I want to do?</a> <a data-t="2375">What kind of table of contents do I want to do?</a> <a data-t="2378">What are all the different things that I want to do across the entire document as it</a> <a data-t="2382">goes?</a> <a data-t="2382">And again, this can affect things like the code or just the content, the rendering process</a> <a data-t="2387">or how it appears.</a> <a data-t="2389">And it's usually read across by all the different components.</a> <a data-t="2391">So it interacts with Pandoc and interacts with Quarto and interacts with Knitter and</a> <a data-t="2395">affects all the different processes over time.</a></p>
