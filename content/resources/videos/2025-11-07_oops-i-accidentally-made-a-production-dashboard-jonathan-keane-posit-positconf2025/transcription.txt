<h3>Modularity and testing</h3>

<p><a data-t="362">So, what do we do now that we know we've got something that we have to maintain? We really</a> <a data-t="366">need to invest time in it. <p><a data-t="393">So, what is modularity? It is taking a long script that's just kind of a very procedural</a> <a data-t="400">set of code. <p><a data-t="462">And the best part of composability and decomposing things is that it makes testing</a> <a data-t="466">these things a lot easier. <h3>Continuous integration</h3>

<p><a data-t="559">Okay. Now that you've got your tests, you can run them locally. That's great.</a> <a data-t="563">That will confirm that you don't have a bug. And that's what this person is doing, running a bunch</a> <a data-t="567">of tests all the time. They make a change, they run a test. But if you have code that's going to</a> <a data-t="573">be shipped off somewhere, hosted on a service and run somewhere else, you can't just test on your</a> <a data-t="577">local laptop because you might have installed specific dependencies or something and not</a> <a data-t="581">realize that's actually important to how you're actually running the code in production. And so,</a> <a data-t="586">one of the first things that you need to do when you're testing is have your tests be run on some</a> <a data-t="590">other computer. And so, you could call a colleague and be like, hey, can you run these tests for me?</a> <a data-t="597">And that will work a few times. And so, ultimately,</a> <a data-t="606">what you want to do is find a way to run that test on some computer that's somewhere else. It's not</a> <a data-t="611">your computer. And that's what cloud laptops are for. And that's exactly what continuous integration</a> <a data-t="617">is. It's a container, which is a laptop in the cloud somewhere. <p><a data-t="628">And that is effectively what continuous integration is. But if you're in a larger organization,</a> <a data-t="668">you might see other things like Travis CI or Circle CI or Jenkins. And other teams might</a> <a data-t="673">already have these set up. And you can hook into these.
