<h3>Initial step in FDA: smoothing</h3>

<p><a data-t="1014">So once you have measured your functional observations, I think what you've noticed is that it's not truly a smooth, continuous function that we're analyzing because we never actually observe this true process because we measure it at set time points or points in the domain.</a> <a data-t="1026">It's always a discretized function.</a> <a data-t="1026">So that's why smoothing is often the first step of preprocessing.</a> <a data-t="1032">And in many cases, it can be a requirement for like PCA or for a functional PCA.</a></p>

<p><a data-t="1037">So why do we have to do this?</a> <a data-t="1037">Well, again, with a discrete curve.</a> <a data-t="1037">So X is actually our true process that we want to explore, but we actually observe Y because when we go to measure the points, we try to measure at different TJs, the value of this process.</a> <a data-t="1051">There's error likely involved, measurement error, process error.</a> <a data-t="1058">And so we don't observe the true process.</a> <a data-t="1058">We observe something with error that we need to convert to our functional observation to analyze.</a> <a data-t="1065">And so the common approaches are penalized splines in some fashion, Fourier series for periodic data, cubic B-splines or some other regression spline for non-periodic data.</a> <a data-t="1078">We play with the bias-variance tradeoff with a smoothing parameter lambda.</a> <a data-t="1083">And an important note is that you could do this all at once.</a> <a data-t="1083">So you could take all your observations, smooth them all together, or you can actually treat them and do them individually, which we'll see we have to do in our case.</a></p>

<p><a data-t="1098">And just to give an example of this penalization and smoothing here.</a> <a data-t="1098">So I've generated from the sine function some points.</a> <a data-t="1106">So the gray circles, I think it's 100 or maybe 200 points within the domain minus three to three, equally spaced.</a> <a data-t="1113">So you see that there is a process underlying this.</a> <a data-t="1113">There is a functional relationship across these points, but we don't have a great, we wouldn't want to connect these points.</a> <a data-t="1118">So how do we actually estimate the true function, the black curve?</a> <a data-t="1125">So if we pick a lambda that's too small, we undersmooth.</a> <a data-t="1125">And as it approaches zero, you basically interpolate the points.</a> <a data-t="1131">So you have high variance, low bias.</a> <a data-t="1132">So we probably wouldn't want to use that red curve.</a> <a data-t="1139">Green, if we pick a lambda too big, we oversmooth.</a> <a data-t="1139">So we actually typically will miss valleys and peaks, which you observe here.</a> <a data-t="1145">So here we have very low variance, but high bias.</a> <a data-t="1145">So the best is, you know, we can pick this through trial and error or some cross-validation technique.</a> <a data-t="1158">But when you have an appropriately smoothed function with Fourier series, you see it's actually hard to discern the blue dashed line compared to black, which tells us that, see, we were able to use splines to capture our true, the underlying process here.</a></p>
