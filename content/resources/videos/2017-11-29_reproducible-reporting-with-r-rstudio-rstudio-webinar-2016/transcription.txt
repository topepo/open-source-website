<h3><a href="/software/packrat/">PackRat</a>: dependency management for R</h3>

<p><a data-t="2135">Thanks Yihui.</a> <a data-t="2137">So now I'm going to talk about PackRat which is a dependency management system for R.</a> <a data-t="2143">So given everything that Jeff and Yihui have taught us about trying to write reproducible research.</a> <a data-t="2150">One thing that we want to be able to do is take the R environment that we use to produce these reports and try and preserve that over time.</a> <a data-t="2158">So PackRat is an R package that helps us manage these R projects, R package dependencies in a way that is reproducible.</a></p>

<p><a data-t="2169">So when you form an analysis you're going to use different R packages.</a> <a data-t="2175">Say you got them from CRAN or if you live on the bleeding edge maybe you got them from GitHub.</a> <a data-t="2180">But when you do an analysis or write an R project you want to make sure that you can save these packages over time.</a> <a data-t="2186">Say you publish this as research and one or two years down the line you need to run that again.</a> <a data-t="2193">You want to make sure you can reproduce that exact same results with the exact same environment that you used originally when forming the analysis.</a> <a data-t="2202">Another thing about this is you want your projects to be isolated.</a> <a data-t="2207">So in the current state of the R world when you install a package it goes into your global library.</a> <a data-t="2213">So every project that you write on your computer has access to that package and that's often useful.</a> <a data-t="2221">In some occasions you want to be able to have a private library for each project in case one project depends on one version of a package and another project depends on another version of a package.</a> <a data-t="2234">So PackRat gives you a means of isolating these package dependencies within a library for each project.</a> <a data-t="2240">And last of all we want this to be portable so that if you need to collaborate with people on alternative machines they can build the same R environment using the same system from your PackRat project.</a> <a data-t="2255">So PackRat makes it easy to install those packages for a particular R project.</a></p>

<p><a data-t="2260">Part of the reason why the PackRat development was motivated is for those of you who have either been in research for a while or done some work and come back to it a year later.</a> <a data-t="2273">You've probably come back to your code and asked first of all who wrote this garbage code?</a> <a data-t="2279">Unfortunately one of the side effects of becoming a better programmer is realizing that the code you wrote before is probably not as good.</a> <a data-t="2287">And unfortunately if you come back to code a year later unless you wrote it in a reproducible nicely commented way you're going to struggle to recollect exactly what you did and how you did it.</a></p>

<p><a data-t="2303">Another thing is you might have used say Lattice which is an R package for generating plots.</a> <a data-t="2308">You used that to make some figures before.</a> <a data-t="2310">But now when you try to run it with your new version of R, say you updated it recently, all of a sudden your plotting code is just throwing errors at you.</a> <a data-t="2320">For some reason a Lattice update has broken your code and you know your code didn't change but Lattice did.</a> <a data-t="2327">So now you have to figure out what changed and how you can fix your code or find the old version of Lattice.</a> <a data-t="2332">Say you used the NLME package for fitting non-linear mixed effects models.</a> <a data-t="2338">And when you ran it before everything went fine.</a> <a data-t="2341">You showed the results to your boss or you published it as research.</a> <a data-t="2345">The model convergence criteria everything worked okay.</a> <a data-t="2349">You come back and try to re-run it a year later and something has changed underneath NLME and suddenly it's complaining about model convergence.</a> <a data-t="2357">I guess something changed in there and how they are fitting the parameters and this makes you feel pretty uncomfortable.</a> <a data-t="2365">Suddenly the environment you used to produce the results before, you can't reproduce that with this new environment with the updated version of NLME.</a></p>

<p><a data-t="2375">Another one is not every R package you use is going to be on CRAN or GitHub or an external repository.</a> <a data-t="2382">Say you got it from collaborators who put it and they just share it through email or you got on a USB stick from someone.</a> <a data-t="2390">You really don't want to have to go and hunt that back down.</a> <a data-t="2393">It'd be really nice if we were able to just keep that package in a project and save it over time.</a> <a data-t="2399">So PackRat helps us solve this problem by one, we make sure that package sources are stored alongside your project.</a> <a data-t="2406">And we track the package version so that if you update R but you need to return to an old version of a R project, you can roll back and reinstall that package for some reason.</a> <a data-t="2419">So the way I like to think about it is PackRat lets you build the sandbox in which an analysis can run and it helps you ensure that this sandbox will persist over time.</a></p>

<blockquote><a data-t="2419">So the way I like to think about it is PackRat lets you build the sandbox in which an analysis can run and it helps you ensure that this sandbox will persist over time.</a></blockquote>

<p><a data-t="2428">And as I said before, one of the benefits but also problems with the current system for R packages is that everything gets installed into a global library.</a> <a data-t="2441">So all projects on your system have access to the same library.</a> <a data-t="2445">But for a particular version of R, only one version of a particular package can be installed at a time.</a> <a data-t="2450">This is problematic if you have multiple projects on your system and one depends on a current version of a package, say R3P.</a> <a data-t="2458">And another one depends on a newer version of R3P which maybe doesn't work with your older one.</a> <a data-t="2464">But you want to leverage the features of that for new projects.</a> <a data-t="2468">So we want to be able to isolate these package dependencies for each project and PackRat helps us do that.</a> <a data-t="2475">So PackRat does this by giving each project its own private PackRat library.</a> <a data-t="2480">So if you don't know what a library is in R parlance, a library is just a folder of installed R packages.</a> <a data-t="2488">So when you make a call to the library function which is kind of poorly named because it loads a package from a library.</a> <a data-t="2501">Say we use library </a><a href="/software/dplyr/">dplyr</a><a data-t="2501"> to load the dplyr package from the library.</a> <a data-t="2507">R will attempt to load dplyr by default from that global library.</a> <a data-t="2512">If you use PackRat to manage an R project on the other hand, PackRat will ensure that R loads that package from a private library instead.</a></p>

<p><a data-t="2521">The other nice thing is because we do it this way with PackRat, it doesn't interfere with any normal workflows.</a> <a data-t="2530">Any calls to install.packages, </a><a href="/software/devtools/">devtools</a><a data-t="2530"> install github, bioclite if you're a bioconductor person.</a> <a data-t="2537">These will also get installed into the private library.</a> <a data-t="2540">So you can ensure that installing new packages for one project does not break any analysis you have for another project.</a> <a data-t="2548">And because PackRat collects all of these project dependencies and it ensures that it is portable across different systems.</a> <a data-t="2558">So if you're working with collaborators, you can ensure everyone working on a project is using the same version of any R packages your project depends on.</a> <a data-t="2567">So you don't have to worry about, oh you're getting a bug, is it because you have the newest version and I'm using this different version of this package.</a> <a data-t="2574">When people collaborate on a PackRat project, you ensure everyone works in the same environment.</a></p>
