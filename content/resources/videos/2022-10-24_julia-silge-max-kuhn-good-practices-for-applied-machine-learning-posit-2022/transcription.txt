<h3>Model deployment with vetiver</h3>

<p><a data-t="1114">In my experience and talking with folks about their modeling, one of the parts of the whole modeling analysis that has been the least ergonomic, that has involved the most pain, is the process of getting your model off your machine.</a> <a data-t="1135">You know, you would spend time training, tuning, developing a beautiful, accurate, appropriate model, and then when it's time to deploy that model, to put that model into production, there's been a real lack of tools for people who are excellent model developers to be able to take that model and get it off where it needs to go.</a></p>

<p><a data-t="1200">Maybe some of you are sitting here and you're like, what does that even mean, the word production?</a> <a data-t="1207">That's a great question, because there isn't some industry standard definition for what production means.</a> <a data-t="1215">I like to use the definition or the idea that putting a model into production is taking your trained model, getting it out of the computational environment where you trained it, and putting it somewhere where it is useful to a wider group of people.</a> <a data-t="1248">So by that definition, taking a model and putting it in a </a><a href="/software/shiny-r/">shiny</a><a data-t="1248"> app and letting people interact to get predictions, we can say that's production.</a></p>

<p><a data-t="1261">The industry standard these days for putting models into production for model deployment is to create restful APIs so that you can scale your API, you can have it as a microservice, and we have this convenient way to take a model that you have trained and put it in a computational environment where it's accessible to anyone in your organization who needs it.</a></p>

<p><a data-t="1280">I've been working on a new framework for model deployment and other MLOps tasks called vetiver.</a> <a data-t="1289">So if you are into perfumery or fancy candles like I am, you may have seen this word, this word vetiver.</a> <a data-t="1296">So vetiver is a stabilizing ingredient in perfume.</a> <a data-t="1296">It takes the more volatile fragrances and it stabilizes them.</a> <a data-t="1304">So in this metaphor, your model is this more volatile thing.</a> <a data-t="1311">You have used different hyperparameters, you have used different data, and vetiver helps you stabilize it so you can version it, deploy it, and monitor it.</a></p>

<p><a data-t="1325">Let's walk through that a little briefly.</a> <a data-t="1325">So any modeling process starts with you collecting data, having access to some kind of data.</a> <a data-t="1331">The first thing you need to do is to understand, clean, you need to engage in the process of exploratory data analysis.</a> <a data-t="1345">There are great tools in Python and R for you to approach those tasks.</a> <a data-t="1354">Next, you want to train your model.</a> <a data-t="1354">You want to train, tune, evaluate, and again, there are amazing open source tools like what we are talking about to be able to approach those kinds of tasks.</a> <a data-t="1367">So far, at this point, that becomes less true as we move around this cycle.</a></p>

<p><a data-t="1375">Once your model is trained, you need to have a reliable scheme to version that model so that you can know which version of my model was used over which time with which data.</a> <a data-t="1390">Once your model is versioned, you need to deploy it.</a> <a data-t="1390">And after it's deployed, your job is not done.</a> <a data-t="1390">You need to also monitor your model.</a> <a data-t="1400">You need to measure regularly the statistical characteristics of how your model is performing so that you can make adjustments, decide to retrain, head down that path.</a> <a data-t="1416">So, the tasks on the right side of this diagram are ones that there are great open source tools for.</a> <a data-t="1423">The tasks on the left side are ones where that has not been true, and this is where vetiver sits.</a> <a data-t="1429">So, vetiver is a framework for versioning, deploying, and monitoring your model.</a></p>

<p><a data-t="1437">Now, you can probably guess from this slide that vetiver is not just for tidymodels.</a> <a data-t="1437">It supports many kinds of models in R, and vetiver is not just for R.</a> <a data-t="1443">vetiver was designed from the beginning to have feature parity between R and Python.</a> <a data-t="1451">You can use the tools that you want to use to understand and clean your data, to train your model, and then vetiver can support these ML ops tasks for both Python and R.</a></p>
