<h3>How webpages work</h3>

<p><a data-t="262">And the way we're going to do this is with a strategy that relies on the nature of webpages.</a> <a data-t="269">So just to quickly review how webpages work, every website that you visit is stored as a set of instructions on a web server somewhere.</a> <a data-t="279">You visit it through a web browser on your computer, and the first thing that happens when you try to access that website is your computer sends a request to the server that hosts the website.</a></p>

<p><a data-t="315">If you were to look at the HTML document that your web browser receives, it's just text.</a> <a data-t="322">Text instructions.</a> <a data-t="323">It's formatted a particular way, but everything that appears on that webpage will be mentioned somewhere here in the instructions, and this is the kernel of our strategy for web scraping.</a></p>

<h3>HTML structure</h3>

<p><a data-t="483">Let's look at the structure of HTML very quickly.</a> <a data-t="485">HTML organizes content by placing tags around the content.</a> <a data-t="490">Here's an example tag.</a> <a data-t="492">This piece of HTML would create a link to github.com, and the link would appear in a document as just the word github.</a> <a data-t="499">If you open this in a web browser, that word would probably be blue or something, so you know that there's a link and you can click on it.</a></p>

<p><a data-t="505">The tag here in this piece of code is A. It stands for anchor.</a> <a data-t="509">It's for web links, and the tag has a name, and it starts right after the less than sign.</a> <a data-t="515">This tag also has an attribute.</a> <a data-t="517">In this case, the attribute is named href, and the attribute has a value, and the value is, in this case, the name of the website we want to link to, and then github itself is the content of the tag.</a></p>

<p><a data-t="527">When you're scraping data, you could scrape any of these things out of the HTML, but most often you'll want the actual content that's in your webpage or in the HTML.</a></p>

<p><a data-t="539">Within any webpage, the tags are organized in a hierarchy.</a> <a data-t="557">This webpage here begins with an HTML tag, and then the next level of the hierarchy is a head tag and a body tag, which sort of divides the webpage into two, so we can visualize this as a tree.</a> <a data-t="570">Under the head tag, we have some other tags, title, links, scripts, and under the body tag, we have some tags.</a> <a data-t="576">Those tags also have their own sub-tags, for example, p and span here, and beneath p, we have a b tag, and then we have some tables with their own items.</a></p>

<p><a data-t="588">Now if you wanted to find a piece of content in this simple webpage, for example, if you want to find the word here, all you would need to know is that it occurs inside the b tag, and the b tag's in this location, and then you can search for that b tag and extract this content, and you'd have the word here.</a> <a data-t="604">This is sort of what we're going to do when we scrape data out of our webpages.</a></p>

<p><a data-t="610">If we want to find Kristen Bell's name, she's the lead actress here, we would need to figure out which HTML tag surrounds her name in the code.</a> <a data-t="619">In this case, since I have the source code up, I'll search for her name, and I found it.</a> <a data-t="623">Here, she's the star of the show, so her name's mentioned in several places, but I'm most interested in this piece here, because if I can extract Kristen Bell in this table, then I should be able to extract every other actor or actress in the table, so we're looking at the cast table.</a></p>

<p><a data-t="638">I've already searched through this webpage a little bit, and you can see here's cast, so this is the table, and here's Kristen Bell.</a> <a data-t="645">This is referring to her picture, which appears in the table.</a> <a data-t="648">Here's her actual name in the table, and we can see that it's contained in the span tag, so we know which tag surrounds Kristen Bell, but in this particular webpage, there are many, many span tags that are not just used for Kristen Bell's name.</a></p>

<h3>CSS selectors</h3>

<p><a data-t="696">Let's look at how we could zoom in or be a little more targeted when we search for Kristen Bell.</a> <a data-t="701">We're going to use more than just the HTML tag to describe and locate her name in this document, and we're going to do that by taking advantage of another technology that's used with webpages, and that's CSS.</a></p>

<p><a data-t="659">So if you're not familiar with CSS, here's a very brief introduction.</a> <a data-t="664">If you have an HTML document with those tags you're looking at, you'll be able to create a page that has text and links and paragraphs and stuff, but it's going to look very plain because HTML doesn't really contain all that much.</a></p>

<p><a data-t="719">So if you consider a webpage you might have visited, like shiny.</a><a href="/software/rstudio/">rstudio</a><a data-t="719">.com, if you look just at the HTML of the webpage, it would build something that looks like this, a plain white document.</a> <a data-t="729">And if you've ever tried to load a webpage with a really slow connection, you might have actually seen a webpage that looked like this.</a> <a data-t="735">Your computer was just trying to help you out and show you what it can before the rest of the information related to the webpage showed up.</a> <a data-t="741">That information would be CSS.</a></p>

<p><a data-t="743">You combine CSS with an HTML document, you get a styled webpage.</a> <a data-t="747">So this is how the </a><a href="/software/shiny-r/">shiny</a><a data-t="747">.rstudio.com webpage looks if you visit it, and you'll notice it has the same components, it has the same text and the same web links and stuff, but they all look different.</a> <a data-t="759">They have a different style, and it's that style that really makes the visual experience that you want to have when you visit the webpage.</a></p>

<p><a data-t="767">If we understand how CSS works, we can use it to scrape data as well as style webpages.</a> <a data-t="772">So if you look at an example CSS file, you'd see something that looks a little bit like this.</a> <a data-t="777">These are sets of instructions.</a> <a data-t="779">Between the brackets here we have actual pieces of styling.</a></p>

<p><a data-t="794">The other part of the CSS, the part that we'll rely on, are selectors.</a> <a data-t="802">So where it says spanner.num or table.data, this tells your web browser which parts of the HTML document to apply the styling to.</a> <a data-t="812">For example, that color FFFFF, that's only going to be applied to span tags.</a> <a data-t="818">We've already seen some span tags.</a> <a data-t="820">So any span tag in that hypothetical webpage will have a certain color.</a></p>

<p><a data-t="834">But these selectors are a way to specify specific elements in the webpage to then apply styling to.</a> <a data-t="841">But this idea of specifying a specific element in a webpage is how you're going to extract specific elements as data.</a> <a data-t="849">We use the same system.</a> <a data-t="850">So let's look at the basics of this system.</a></p>

<p><a data-t="856">Here is an HTML tag.</a> <a data-t="858">It's a span tag.</a> <a data-t="859">Notice this tag has a class and an ID attribute.</a> <a data-t="862">It has some content.</a> <a data-t="863">Content here says shiny.</a> <a data-t="865">Here's our class, here's our ID.</a></p>

<p><a data-t="870">And then here's a selector that we can use in CSS to describe this tag.</a> <a data-t="875">The word span in the selector will refer to every span tag that exists in your HTML document.</a> <a data-t="881">And since this is a span tag, it will refer to this as well.</a> <a data-t="883">So any styling we group under the span tag will be applied here.</a></p>

<p><a data-t="887">But we could also be more specific in the way we apply our styling.</a> <a data-t="892">We could apply our styling based on class.</a> <a data-t="894">So this span tag has a class big name.</a> <a data-t="897">If we want to create a CSS selector that says find all the elements who have the class big name, we do that by putting a period and then big name.</a> <a data-t="906">So that period prefix tells your web browser that this selector is a class selector.</a> <a data-t="913">It matches everything that has the class big name.</a></p>

<p><a data-t="917">And if we put that period together with span, now we're saying we want to match everything that is both a span element and a big name class.</a> <a data-t="925">So it's even more specific.</a> <a data-t="928">And then we could also be very, very precise.</a> <a data-t="930">And if we set an ID name for a tag, then we can refer to that tag by its ID with a hashtag.</a> <a data-t="937">So hashtag shiny would refer to anything that has the ID shiny.</a> <a data-t="940">But normally when you write a webpage, only one thing will have that ID.</a></p>

<p><a data-t="945">So those selectors we saw there refer to different things.</a> <a data-t="948">And the way they refer to them is with the prefix that is put in front of them.</a> <a data-t="952">The prefixes that you could use are no prefix, which refers to the tag name, a dot, which refers to the class name, and a hashtag that refers to an ID.</a></p>

<p><a data-t="962">Now will there be a quiz on this later?</a> <a data-t="964">Yeah, kind of.</a> <a data-t="965">You don't actually need to remember all this.</a> <a data-t="967">I'm going to show you a tool that does it for you.</a> <a data-t="968">But you will need to be familiar with this if you want to understand what's going on.</a></p>

<p><a data-t="976">So which CSS end identifiers are associated with Kristen Bell's name?</a> <a data-t="980">Well, if we go back over here, we see it's a span that has the class item prop, and item prop is just another attribute.</a> <a data-t="984">So we're looking at span and class item prop.</a> <a data-t="987">So that should narrow things down a little bit for us.</a></p>

<h3>Extracting HTML tables</h3>

<p><a data-t="1381">So just to review, we download HTML with readHTML.</a> <a data-t="1386">The function readHTML just takes the URL of the webpage you want to download.</a> <a data-t="1390">If you're connected to the internet, and this is a valid URL, R will go fetch the HTML for that webpage.</a> <a data-t="1395">Next, we extract its specific nodes with HTML nodes.</a> <a data-t="1399">You give HTML nodes the names of the website that you downloaded, the object you saved it to, that is, and then a CSS selector that helps it select specific nodes from that webpage.</a> <a data-t="1411">Then finally, you use one of these functions to extract specific types of content from all the nodes that you just collected.</a></p>

<p><a data-t="1417">We did this, and we got this list of content, but we scraped too much information, because we have things here beyond the actors and actresses' names.</a> <a data-t="1426">This is pretty common when you are scraping data from a webpage.</a></p>

<h3>Recap</h3>

<p><a data-t="1998">Alright, so let's recap, when you're scraping data from the web, the strategy that you're going to use is you're going to pull the data right out of the HTML that the webpage is contained in, and you're going to rely on the HTML CSS structure of that webpage to refer to the data that you want, since HTML and CSS structure is uniform across webpages within a site, you can use that structure to automate the retrieval of data across that site.</a></p>

<p><a data-t="2041">The RMS package lets you enact this strategy, you download the data using read HTML, this collects the entire website, you then pull out just the tags that you want, using HTML nodes and CSS selectors.</a> <a data-t="2057">You can use the Selector Gadget to get those selectors, but now I'm getting ahead of myself.</a> <a data-t="2061">Finally, once you have the nodes, you extract the content with the helper function, which is normally HTML text.</a> <a data-t="2068">If your node is a table, you can extract the whole table as a R-friendly data frame with HTML table, and then you can use the Selector Gadget widget to find useful selector combinations to get just the information that you want.</a> <a data-t="2078">This is the best way to go about it, because there's not necessarily going to be much rhyme or reason, or intuition to follow about which CSS selectors best describe what you want.</a></p>
