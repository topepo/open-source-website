<h3>What is Quarto?</h3>

<p><a data-t="123">So ultimately, we'll answer that question by saying, what is Quarto?</a> <a data-t="127">So Quarto is the next generation version of our Markdown for everyone.</a> <a data-t="132">And that everyone kind of has an asterisk to it in terms of not 6 billion people or</a> <a data-t="135">whatever is around in the world, but for really any data science language or any future language.</a> <a data-t="140">As JJ talked about in his keynote this morning, we want to build a 100-year company.</a> <a data-t="145">And while that today might mean working in R and Python and Julia and JavaScript and</a> <a data-t="148">other languages, in the future, we want kind of a language agnostic tool that can work</a> <a data-t="153">with anything, anything that becomes popular in the future.</a></p>

<p><a data-t="157">To make it a bit more clinical instead of just saying the next generation of our Markdown,</a> <a data-t="161">Quarto if we define it is an open source scientific and technical publishing system built on Pandoc.</a> <a data-t="167">So Pandoc is a command line tool that's also used by our Markdown.</a> <a data-t="171">So again, there's going to be a lot of familiarity as we go through these slides, and we'll build</a> <a data-t="174">up a mental model of how Quarto is a little bit different than our Markdown and some of</a> <a data-t="178">the improvements.</a></p>

<p><a data-t="180">Quarto itself is a command line interface, or what is called a </a><a href="/software/cli/">CLI</a><a data-t="180">.</a> <a data-t="183">So you can access Quarto via your terminal, whether in </a><a href="/software/rstudio/">RStudio</a><a data-t="183"> or any text editor or any</a> <a data-t="188">terminal on your computer.</a> <a data-t="189">In this example, I'm just calling Quarto dash dash help to see what can I do with Quarto</a> <a data-t="194">once I've installed it on my machine.</a></p>

<p><a data-t="197">It'll print out something like this, and I've abbreviated it so we can see the slides nice</a> <a data-t="200">and big, but the commands here that we're looking at are I can use Quarto render to</a> <a data-t="205">render documents from the plain text format out to, say, HTML or PDF or to a website.</a> <a data-t="211">I can use Quarto preview, which will actually render the document and then maintain a background</a> <a data-t="215">web server so as I make changes, it shows those in realtime on the actual rendered output.</a> <a data-t="221">And because it's a publishing system, I can not only render it locally, but I can publish</a> <a data-t="225">it to the web, places like GitHub pages or Netlify or something else we'll talk about</a> <a data-t="229">in the future called Quarto pubs.</a></p>

<h3>How Quarto differs from <a href="/software/rmarkdown/">rmarkdown</a></h3>

<p><a data-t="233">To really kind of build up our mental model, though, and help differentiate a little bit</a> <a data-t="236">between R Markdown and Quarto, we need to kind of talk about what R Markdown is and</a> <a data-t="241">what is happening behind the scenes.</a> <a data-t="243">When you use R Markdown in your daily work, you're probably just thinking, oh, yeah, I</a> <a data-t="247">open up an R Markdown inside RStudio, type some code, type some text, and then I</a> <a data-t="251">render it and I get something beautiful out.</a></p>

<p><a data-t="254">Behind the scenes are things that are happening here in the middle is actually a little bit</a> <a data-t="257">of kind of magic.</a> <a data-t="258">It's not real magic.</a> <a data-t="259">It's code.</a> <a data-t="260">But it's things that are happening as intermediary steps.</a> <a data-t="263">So when you render your R Markdown, the R code is evaluated by a package called Knitter.</a> <a data-t="268">That will evaluate your R code, generate graphics, generate tables, embed HTML widgets and all</a> <a data-t="273">the other magic that you're building.</a> <a data-t="275">This generates an intermediary Markdown file or a .md file.</a> <a data-t="280">That is then passed to Pandoc, which actually converts a plain text MD into something like</a> <a data-t="285">a report, so a Word document, a PDF report or HTML page, or a presentation, just like</a> <a data-t="291">the presentation I'm giving today, something in HTML, or a more complex project like a</a> <a data-t="295">website or a book.</a></p>

<p><a data-t="298">So this process is great and it's been around for almost ten years.</a> <a data-t="301">You know, Knitter is an old package, it's robust, we've been using it a long time.</a> <a data-t="305">Something to note, though, is that it has this hard dependency on R. So to use R Markdown,</a> <a data-t="310">whether you're using R or other languages, you have to have R, you have to have Knitter,</a> <a data-t="314">you have to have R Markdown.</a> <a data-t="316">And some of the formatting or preprocessing that occurs to the document actually occurs</a> <a data-t="320">with literal R code written in the R Markdown package.</a></p>

<p><a data-t="324">For Quarto, we have a similar diagram, and for some of you, if you're not looking closely,</a> <a data-t="328">this might look like the exact same workflow.</a> <a data-t="331">But there's a few differences we'll note.</a> <a data-t="333">For one, rather than using a .RMD, I'm now using a QMD, or a Quarto Markdown document.</a> <a data-t="339">It's going to feel almost identical to R Markdown, which is great.</a> <a data-t="343">And we're still using Knitter.</a> <a data-t="345">So Knitter will evaluate our R code from our QMD, generate an intermediary Markdown file,</a> <a data-t="350">but now we have another word added to Pandoc.</a> <a data-t="353">We have Pandoc with Lua filters.</a> <a data-t="355">So we've moved any preprocessing or postprocessing into Quarto, that command line interface, or</a> <a data-t="361">to affect them with Pandoc, rather than having to call them from R.</a></p>

<p><a data-t="365">What this does is still allows us to create the same reports, presentations, and projects</a> <a data-t="369">that are available, but we don't have to have R in the system to use this.</a> <a data-t="374">You could use Quarto on a fresh computer and not have R anywhere in it, and just render</a> <a data-t="378">Markdown as opposed to code.</a> <a data-t="380">So people could use it for scientific communication without actually having to execute code.</a></p>

<blockquote><a data-t="380">So people could use it for scientific communication without actually having to execute code.</a></blockquote>

<p><a data-t="387">But because, again, we're trying to become this company that is building for all of data</a> <a data-t="391">science and building for scientific communication in general, we want to do more than just evaluate</a> <a data-t="396">for R code.</a> <a data-t="398">So Quarto also extends to other languages.</a> <a data-t="401">So Quarto can now use Jupyter and Jupyter kernels as the engine rather than just R and Knitter.</a> <a data-t="407">So you can imagine Jupyter kernels include things like Julia or Python.</a> <a data-t="411">We also have JavaScript you can execute via Quarto.</a> <a data-t="414">So you have this availability of not just being limited to R, but collaborating with</a> <a data-t="418">other people in Python, or if you're a Python dev yourself, also using Python within your</a> <a data-t="424">Quarto documents.</a></p>
