<p><a data-t="18">If you are trying to create a predictive model, you'd really like to use all of the variables</a> <a data-t="23">in your dataset as that's what's likely to create the most accurate and successful predictions.</a> <a data-t="31">But you may soon run into a problem because many of the functions within R that are used</a> <a data-t="37">to create these models require that all of the input data are continuous numbers.</a> <a data-t="44">Now this makes sense because many of the computations that go on in the background work on linear</a> <a data-t="50">algebra and matrices, which inherently can only have numbers in them, but that might</a> <a data-t="55">be a problem for you because some of your data might be categorical variables and you</a> <a data-t="60">don't want to have to drop them.</a></p>

<h3>The problem with categorical variables</h3>

<p><a data-t="62">You might try to replace each category with a number, 1, 2, 3, which might work if your</a> <a data-t="68">data is ordinal, but if your data doesn't have any inherent order to it, this might</a> <a data-t="76">not work.</a> <a data-t="78">If one of your categories is color, how are you going to choose numbers for black, brown,</a> <a data-t="84">yellow, or green?</a></p>

<p><a data-t="87">You might try to create dummy variables, but this actually can cause big data problems</a> <a data-t="93">really quick.</a> <a data-t="95">Dummy variables will create a new variable for each level of your data and, for example,</a> <a data-t="104">if one of your categories is United States ZIP Code, you're about to add 42,000 new variables</a> <a data-t="111">to your data and that's probably not what you want.</a></p>

<h3>Introducing categorical embeddings</h3>

<p><a data-t="115">That leads me to categorical embeddings.</a> <a data-t="119">If you're familiar with word embeddings, such as Word2Vec, this is actually a very similar</a> <a data-t="125">idea, except instead of an embedding for each word in language, you're embedding each category</a> <a data-t="132">in your data.</a></p>

<p><a data-t="136">What happens is that every category is replaced by a vector of numbers.</a> <a data-t="141">The vector can be length 4, it can be length 50, it really depends on the situation, but</a> <a data-t="146">these numbers will replace your categorical variable for any future modeling.</a></p>

<p><a data-t="153">Now, categorical variables are most commonly fit by neural networks, which is why within</a> <a data-t="160">R, many of the most common ways of creating them are using the Keras and </a><a href="/software/tensorflow/">TensorFlow</a><a data-t="160"> packages</a> <a data-t="167">or the new Torch package.</a> <a data-t="169">However, I find the simplest way to do it is with the embed package, which is one of</a> <a data-t="175">the add-on packages in the tidy models suite of packages.</a></p>
