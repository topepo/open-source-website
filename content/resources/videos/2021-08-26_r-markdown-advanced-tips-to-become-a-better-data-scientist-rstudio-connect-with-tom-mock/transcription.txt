<h3>Literate programming</h3>

<p><a data-t="250">The first one, and I think the obviously one of the core workflows that you're going to do in</a> <a data-t="255">R Markdown, is literate programming. The goal here is that you're going to capture your code,</a> <a data-t="262">text, or comments, and your output in a single document. So you have essentially like this</a> <a data-t="267">reproducible workflow of capturing your code, along with the output, along with any kind of</a> <a data-t="273">comments about what the code is doing, or the outputs that you're seeing, all together. So you</a> <a data-t="277">can share one document, or one report with someone if you wanted to do that. But you can also</a> <a data-t="281">reproduce it, because you have the source code.</a></p>

<p><a data-t="288">The idea of literate programming has been around</a> <a data-t="288">for a long time. Donald Knuth has this nice quote about literate programming as a programming</a> <a data-t="295">paradigm. And the explanation of it is that it's logic in a natural language, such as English,</a> <a data-t="301">interspersed with snippets of macros and traditional source code, from which the</a> <a data-t="305">compilable source code can be generated. Now when we think about source code in R,</a> <a data-t="309">we have code chunks. So you have these little portions here, where you might recognize some</a> <a data-t="314">of these, your favorite ggplot, some </a><a href="/software/dplyr/">dplyr</a><a data-t="314"> to filter a data set. But then with R Markdown,</a> <a data-t="320">you can also intersperse actual text, you know, write about it, just as you would in a word</a> <a data-t="325">processor. But now you get the benefit of doing code and text together.</a></p>

<p><a data-t="332">The other component I just want to bring up here is that there is the YAML header, which allows you to define certain metadata.</a> <a data-t="337">So up here at the top, you can say, I want my output to be an HTML document. So this will,</a> <a data-t="343">when it's rendered, when you finish it and you click render, it will actually create an HTML</a> <a data-t="348">object that you can open up in a web browser. And you can switch this out to do</a> <a data-t="353">Schrodingen or any other type of things or presentation like I'm doing today.</a></p>

<p><a data-t="359">Now going a step further beyond just that plain text, visual R Markdown was introduced in RStudio</a> <a data-t="364">1.4. So this has been around for six or seven months, but there's still lots of people who</a> <a data-t="369">may not have heard about it. So I wanted to cover it today. You can essentially think of this as</a> <a data-t="374">not only do you have the ability to look at the source code and the raw Markdown, you can also</a> <a data-t="380">view it in a way where it shows you the actual rendered outputs. So when you create a table,</a> <a data-t="386">it shows you a table. When you include a graphic, it shows you the graphic. You don't just have to</a> <a data-t="391">believe that, okay, this code should bring in this image. It will actually do that live inside</a> <a data-t="396">RStudio.</a></p>

<p><a data-t="404">In addition to that, it can do things with LaTeX or with real-time spell checking or</a> <a data-t="409">all the different source code that you still want to be using. And you get a nice header up here</a> <a data-t="409">that allows you to add things like bold, italics, cross-throughs, bullet points, tables, all those</a> <a data-t="416">things with insertion. But it still allows you to do that with code and actually adds the code into</a> <a data-t="422">it. I'll be walking through this in a second here live.</a></p>

<p><a data-t="434">And then the last part is that R Markdown and literate programming is not just for R. You can do things like include Python via </a><a href="/software/reticulate/">reticulate</a><a data-t="434">. You can use</a> <a data-t="439">SQL inside code chunks to actually write raw SQL code. You can include CSS or JavaScript for some</a> <a data-t="445">of your front-end development work. And you can include other things for bash, RCPP, SAM, and a</a> <a data-t="452">lot of other different languages. So altogether, Knitter provides kind of interfaces to 52 possible</a> <a data-t="458">languages. So you can do a lot in R Markdown, even though a lot of what I'll be showing today will be</a> <a data-t="463">focused on R.</a></p>
