<p><a data-t="0">My quarantine fad of choice this last summer was gardening.</a> <a data-t="12">Among other things, I was working with some basil, and a friend of mine sent me a recipe for pesto.</a> <a data-t="17">Turns out that they're one of those good at cooking types, so instead of talking tablespoons or cups, this recipe was all in grams.</a> <a data-t="23">Seeing that I don't own a scale, I realized that I had stumbled upon a totally real-world, definitely not contrived need for machine learning.</a> <a data-t="35">I know how tall the plants are, what species they are, and some other yada yada about the concentration of some sort of chemical, and I need to estimate my basil mass.</a> <a data-t="48">Thankfully, the previous tenant of 10 years has been collecting measurements on this same plot, so I have some solid data to work with.</a></p>

<p><a data-t="58">So what's our strategy?</a> <a data-t="60">Maybe there's no need for anything fancy.</a> <a data-t="62">We could start out with a linear regression.</a> <a data-t="63">Or maybe I'm the kind of feller that uses k-n-n in the regression setting.</a> <a data-t="68">We could run with that.</a> <a data-t="68">Or maybe I'm feeling trendy and decide that my best move is a neural network.</a> <a data-t="76">Or... Okay, hear me out.</a> <a data-t="76">What if I could figure out some way to combine those outputs?</a> <a data-t="82">I want to come up with some sort of model that generates predictions that are informed by each of these constituent models.</a> <a data-t="88">This is the premise behind ensembling.</a></p>

<p><a data-t="93">Ensembling is a machine learning technique that has been shown to increase predictive performance in a variety of settings.</a> <a data-t="98">In a community survey conducted over the summer by the </a><a href="/software/tidymodels/">tidymodels</a><a data-t="98"> team, we learned that users felt that the number one priority for future development in this ecosystem should be ensembling.</a> <a data-t="109">This December, we are delighted to announce the initial release of stacks, a package for tidymodels-aligned model ensembling.</a></p>

<h3>How stacks works</h3>

<p><a data-t="114">When you're coding up an ensemble with stacks, most of the code you'll end up writing isn't specific to stacks at all.</a> <a data-t="123">I'll skip over the code that generates these models, but the gist is that I've created three model specifications here using functionality from the tidymodels core.</a> <a data-t="135">Packages like </a><a href="/software/parsnip/">parsnip</a><a data-t="135">, workflows, recipes, and tune.</a> <a data-t="135">The linear regression spec here just defines one model definition.</a> <a data-t="141">There aren't any hyperparameters to tune.</a> <a data-t="141">For this KNN model spec, though, I'm tuning over several possible numbers of neighbors, so this spec actually defines four model definitions.</a> <a data-t="153">Finally, for this neural network model, I'm tuning over six combinations of numbers of hidden layers and dropout rates, so this spec defines six model definitions.</a></p>

<p><a data-t="163">In stacks, each of these eleven model definitions are called candidate members, or candidates for short.</a> <a data-t="168">These candidates are the building blocks for an ensemble.</a> <a data-t="174">The first step to putting together a model stack is the stacks initialization function, which works kind of like the ggplot function from </a><a href="/software/ggplot2/">ggplot2</a><a data-t="174">.</a> <a data-t="180">It lays the foundation called a data stack for the model to be built on top of.</a></p>

<p><a data-t="186">Now, the first core verb in stacks is add candidates.</a> <a data-t="193">It takes in a candidate and adds it.</a> <a data-t="196">We can iteratively call the function as many times as we need to to add all of the candidates.</a> <a data-t="199">Under the hood, add candidates collects predictions on a subset of the training data from each candidate and appends them as columns to the data stack.</a> <a data-t="210">The second core verb is blend predictions, which fits a regularized linear model on the data stack, predicting the true outcome using the predictions from each of the candidate members.</a> <a data-t="220">We call the coefficients of this linear model stacking coefficients.</a> <a data-t="225">A stacking coefficient of zero means that the predictions for the relevant candidate will not ultimately influence the model stack's predictions.</a> <a data-t="231">Candidate members with non-zero stacking coefficients earn the title of members, and we can fit them on the whole training data using the fitMembers function.</a></p>

<p><a data-t="242">And that's it.</a> <a data-t="244">The output of fitMembers is ready to predict on new data.</a> <a data-t="247">Fingers crossed that I'll nail this pesto recipe.</a></p>
