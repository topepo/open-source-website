<p><a data-t="0">One of the challenges that we've seen time and time again is that there's often a gap</a> <a data-t="12">between data scientists and those stakeholders they're hoping to inform.</a> <a data-t="18">And that gap a lot of times is the result of different mediums.</a> <a data-t="23">And so that creates a problem for us is that</a> <a data-t="23">if we just wait for decision makers to proactively go visit our work, we're not actually closing</a> <a data-t="28">that gap.</a></p>

<p><a data-t="33">But that's one problem.</a> <a data-t="33">We also see something else kind of interesting happen</a> <a data-t="40">as data science teams try to close this gap.</a> <a data-t="40">And so one thing, as we said, is that maybe</a> <a data-t="45">the dashboards fall by the wayside.</a> <a data-t="45">Another thing that can often happen is that stakeholders</a> <a data-t="50">will get excited.</a> <a data-t="50">They'll say, great, this insight makes sense.</a> <a data-t="50">Can you bring it to my</a> <a data-t="57">medium?</a> <a data-t="57">Can you copy and paste those predictions into my spreadsheet?</a> <a data-t="57">Or can you send me a</a> <a data-t="57">screenshot of that plot so I can put it in my presentation?</a> <a data-t="62">And that leads to a lot</a> <a data-t="62">of painful, labor-intensive work.</a></p>

<h3>The solution: proactive email notifications</h3>

<p><a data-t="69">But I'd like to propose for you today an alternative,</a> <a data-t="69">a solution that we've found a lot of success using ourselves here at </a><a href="/software/rstudio/">RStudio</a><a data-t="69">.</a> <a data-t="74">And so what</a> <a data-t="74">does this solution entail?</a> <a data-t="74">Instead of creating a dashboard, what we like to do is create</a> <a data-t="79">a proactive notifier or something that is going to alert me to the information that</a> <a data-t="86">I need to know.</a> <a data-t="90">And in our case, we spend most of our time inside of our inboxes.</a> <a data-t="90">So</a> <a data-t="90">that's where our alerts go is through email.</a> <a data-t="94">And so that's what we're going to talk about</a> <a data-t="94">today is how you can use a rich set of R packages to allow you to create these proactive notifications.</a></p>

<p><a data-t="102">Oftentimes these notifications need to meet the expectations of our stakeholders for how</a> <a data-t="106">they're formatted.</a> <a data-t="106">So we'll look at some new packages that are available to help you precisely</a> <a data-t="111">format these results.</a> <a data-t="116">And then finally, we don't want to solve this dashboard fatigue</a> <a data-t="116">problem by introducing another type of fatigue, which would be email fatigue.</a> <a data-t="120">And so we're</a> <a data-t="120">going to talk about how you can only send these alerts on conditions so that instead</a> <a data-t="125">of sending emails every day, you can send emails more specifically when there's a problem</a> <a data-t="130">at hand.</a> <a data-t="134">We also don't want to introduce data science fatigue.</a> <a data-t="134">So we're going to talk about</a> <a data-t="134">how we can achieve these proactive notifications in an automated way.</a></p>
