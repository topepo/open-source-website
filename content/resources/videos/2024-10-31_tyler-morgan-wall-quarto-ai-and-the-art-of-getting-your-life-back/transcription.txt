<h3>Switching to Quarto</h3>

<p><a data-t="188">So enter Quarto.</a> <a data-t="191">Quarto builds static sites so there's no constant parade of security updates.</a> <a data-t="195">However, customizing your website is harder because you have to do so manually.</a> <a data-t="199">And while in the past that meant having to learn languages like CSS and JavaScript,</a> <a data-t="203">the development of large language models, which henceforth are referred to as AI,</a> <a data-t="209">has greatly lowered this barrier to entry.</a></p>

<h3>Automating deployment with Quarto render hooks</h3>

<p><a data-t="411">However, having converted my website to Quarto,</a> <a data-t="414">I now face the next issue, which is actually getting it online.</a> <a data-t="418">Because in WordPress, I could just press the button Publish Post,</a> <a data-t="422">and my content was online.</a> <a data-t="424">I wanted the equivalent of that Publish Post button in </a><a href="/software/rstudio/">RStudio</a><a data-t="424">,</a> <a data-t="428">and for me, that meant never having to touch a terminal and being able to do this entirely from my Quarto project.</a> <a data-t="434">So how could I do that?</a></p>

<p><a data-t="461">For example, in my website, I store my media into non-standard directories,</a> <a data-t="465">posts and videos located within my posts directory.</a> <a data-t="470">And when I render my Quarto website, these are not automatically copied over.</a> <a data-t="474">So all the first two post-render hooks do is copy those over to my site directory.</a> <a data-t="478">I then call an R script called transferSite that does exactly what it sounds like.</a> <a data-t="485">But note the one little argument at the end of this script.</a> <a data-t="489">What exactly is that doing?</a></p>

<h3>Customizing your Quarto website with AI</h3>

<p><a data-t="563">So AI is a great tool for learning new and well-established utilities</a> <a data-t="572">such as rsync or popular languages like CSS and JavaScript.</a> <a data-t="577">And that is what brings me to the last part of my talk,</a> <a data-t="581">customizing your website with AI.</a> <a data-t="584">While transitioning from a server-generated framework like WordPress</a> <a data-t="588">to a static framework like Quarto</a> <a data-t="590">brings many performance and maintenance improvements,</a> <a data-t="593">it does also limit some of the cool dynamic features of your website.</a> <a data-t="596">And by moving from a massively popular tool like WordPress</a> <a data-t="600">to a relatively new tool like Quarto,</a> <a data-t="602">you lose a lot of the great community resources and tools</a> <a data-t="605">that have been developed to help you go through that process.</a></p>

<p><a data-t="610">So the development of AI, however, has vastly evened this playing field.</a> <a data-t="615">The current batch of AI tools are in large part trained on web-based data,</a> <a data-t="619">and we are trying to build a website.</a> <a data-t="621">So while AI is not good at things necessarily like data science</a> <a data-t="625">or being really good at R or critical thinking or basic mathematics,</a> <a data-t="630">it is pretty good at asking you things like,</a> <a data-t="633">hey, generate the CSS for me to make my submit form button</a> <a data-t="637">sashay across the page when I press it.</a></p>

<p><a data-t="639">So let's do that.</a> <a data-t="641">Not dancing buttons, but let's add some nice decorative 3D effects</a> <a data-t="645">to our Quarto main page.</a> <a data-t="647">So here are the Quarto postcards</a> <a data-t="652">for each one of my blog posts on my old website.</a> <a data-t="654">I wanted some sort of subtle 3D effect like that on my Quarto main page.</a> <a data-t="659">So let's look and see what we're starting with in Quarto.</a> <a data-t="662">So this is the default Quarto postcards,</a> <a data-t="665">completely static with a white background.</a> <a data-t="667">They're fine.</a> <a data-t="668">I wanted to write some CSS to add some subtle animations</a> <a data-t="672">and that sort of 3D pop-out effect when a reader mouths over the card.</a></p>

<p><a data-t="676">To do so, I iterated back and forth with ChatGPT,</a> <a data-t="681">adding the generated CSS to my Quarto style.css file</a> <a data-t="685">and then provided feedback with each iteration,</a> <a data-t="688">which honed in on the effect I was after.</a> <a data-t="692">We made small adjustments to the existing CSS</a> <a data-t="699">to get closer to what I wanted.</a> <a data-t="701">And note that nowhere in my prompts was I using technical language</a> <a data-t="704">specific to CSS to request what I wanted.</a> <a data-t="706">I was using plain descriptive language,</a> <a data-t="708">like tilting a piece of brushed metal</a> <a data-t="711">rather than asking for specific CSS features or classes.</a> <a data-t="715">And that's one of the really strengths of AI</a> <a data-t="718">is that it's very good at translating your plain language request</a> <a data-t="721">into specific features and functionality in these languages.</a></p>

<p><a data-t="727">And even if it doesn't get the implementation exactly right,</a> <a data-t="730">it will often get you about 90% there</a> <a data-t="732">just by pointing out the features relevant to your response.</a> <a data-t="737">And the risk of AI giving you the wrong answer,</a> <a data-t="739">which is always present,</a> <a data-t="741">can be mitigated if you have the ability to quickly check for correctness,</a> <a data-t="744">which you can do here simply by rendering the page.</a> <a data-t="747">So after 10 minutes, back and forth the chat GPT,</a> <a data-t="750">this was the final result,</a> <a data-t="752">a nice little 3D effect on my Quarto landing page</a> <a data-t="755">made entirely with AI.</a></p>

<h3>Building an image carousel</h3>

<p><a data-t="757">However, this was a relatively easy ask of AI</a> <a data-t="762">since what I was asking for was a relatively common</a> <a data-t="764">and simple decorative effect.</a> <a data-t="766">How could the AI handle something with user interactivity</a> <a data-t="770">and imposed design constraints?</a> <a data-t="772">So that brings me to the second effect I wanted AI to assist me on,</a> <a data-t="776">which is an image carousel.</a> <a data-t="778">So on my old website, I had an image carousel</a> <a data-t="781">that faded through the seven most recent posts</a> <a data-t="783">at the top of the main page,</a> <a data-t="785">as well as showed the title</a> <a data-t="789">and put an image selector at the bottom</a> <a data-t="791">that allowed you to select and go between the images.</a></p>

<p><a data-t="795">So in WordPress, this was actually generated dynamically</a> <a data-t="798">every single time you went to the website.</a> <a data-t="800">The page would query the server,</a> <a data-t="802">it would generate the most recent posts,</a> <a data-t="804">and then some JavaScript would transform that</a> <a data-t="806">into an image carousel.</a> <a data-t="808">So how could I recreate this in Quarto?</a> <a data-t="811">To do so, I would need two elements.</a> <a data-t="813">First, I would need a way to query the most recent posts</a> <a data-t="816">when I rendered the website,</a> <a data-t="818">and then I would need some JavaScript to transform it</a> <a data-t="820">into an image carousel.</a></p>

<p><a data-t="822">And thankfully, I didn't have to use AI for the first portion</a> <a data-t="824">because I took a workshop in Quarto</a> <a data-t="826">at this very conference two years ago</a> <a data-t="828">that gave me the solution for what I needed,</a> <a data-t="830">which is document listings.</a> <a data-t="832">Document listings allow you to generate a feed</a> <a data-t="834">of categorized and recent posts and add them to any Quarto page.</a> <a data-t="837">And so here, the Quarto main page</a> <a data-t="840">consists of a main listing of all posts,</a> <a data-t="842">but you can add as many additional listings as you want.</a> <a data-t="845">So here I added an additional listing above the first</a> <a data-t="848">of the most recent seven posts.</a></p>

<p><a data-t="851">Now I need a way to transform these</a> <a data-t="854">into the image carousel.</a> <a data-t="856">So the first thing we can do entirely in Quarto</a> <a data-t="858">is just get them to the right size.</a> <a data-t="860">And I adjust it to take the full width of the main page,</a> <a data-t="863">of the main content area on the main page,</a> <a data-t="865">and then I adjust the image height</a> <a data-t="867">via the image height argument,</a> <a data-t="869">and then to our desired final image height for our carousel.</a> <a data-t="873">And then I just need to write some JavaScript</a> <a data-t="876">to transform those into an image carousel.</a></p>

<p><a data-t="880">So that was an animated mock-up of what I was attempting to do</a> <a data-t="883">with each post card.</a> <a data-t="886">So this process was a bit more involved</a> <a data-t="889">than the CSS one, unsurprisingly,</a> <a data-t="891">so I'm not going to go through the full series of prompts.</a> <a data-t="893">But what I will do is share with you</a> <a data-t="895">four useful prompting strategies</a> <a data-t="897">that I developed as I was working with AI.</a></p>

<p><a data-t="901">So the first, and unsurprisingly, I think,</a> <a data-t="903">is it's very good to give the AI</a> <a data-t="906">the exact source code you want it to manipulate</a> <a data-t="909">so that it has access to the structure</a> <a data-t="911">and classes in your page.</a> <a data-t="913">So here I just copied and pasted the listing HTML</a> <a data-t="916">directly into my prompt</a> <a data-t="918">and then asked it to transform it into an image carousel.</a></p>

<p><a data-t="921">Second, as you iterate,</a> <a data-t="923">it can often be good to constantly remind the AI</a> <a data-t="927">what the current status of your script looks like.</a> <a data-t="930">So even if it just generated for you a few prompts prior,</a> <a data-t="934">AIs can lose focus over time due to their limited context window,</a> <a data-t="937">and I found if I didn't constantly remind the AI</a> <a data-t="940">what the current status was, it would kind of forget</a> <a data-t="943">and make really bad mistakes.</a> <a data-t="945">And by providing it that current version,</a> <a data-t="947">it actually ended up much more likely</a> <a data-t="949">to give me a relevant response.</a></p>

<p><a data-t="951">Third, if you have visual details</a> <a data-t="953">you want the AI to adjust,</a> <a data-t="955">it can often be helpful to provide a screenshot of the issue</a> <a data-t="958">as well as the description of it with your prompt.</a> <a data-t="961">So here I had successfully transformed</a> <a data-t="964">the post title into a semi-transparent gray overlay</a> <a data-t="966">on the image.</a> <a data-t="968">However, it had a very small few pixel gap</a> <a data-t="971">at the bottom that I wanted removed.</a> <a data-t="973">So I provided the screenshot and a description of the image,</a> <a data-t="976">or a description of the problem,</a> <a data-t="978">and the AI was able to issue a fix.</a></p>

<p><a data-t="980">And then finally, sometimes the AI can seemingly get stuck</a> <a data-t="983">and be unable to solve a problem solely by prompting.</a> <a data-t="986">So, for example, I managed to get the AI</a> <a data-t="989">to come up with an almost complete implementation</a> <a data-t="992">of my image carousel, but there were one or two issues</a> <a data-t="995">that it could not seem to solve just solely by prompting.</a> <a data-t="998">So at that point, I just started asking the AI</a> <a data-t="1001">to explain the purpose of a specific snippet of code,</a> <a data-t="1004">and then once I understood what the code was doing,</a> <a data-t="1008">I just went in and solved the problem myself.</a></p>

<p><a data-t="1011">So AI can be middling at best when actually writing code,</a> <a data-t="1016">but I found that it's actually very good</a> <a data-t="1018">and very consistent at explaining the purpose</a> <a data-t="1021">of specific snippets of code.</a> <a data-t="1024">So combining the breadth of AI's knowledge</a> <a data-t="1028">with your far superior problem-solving skills</a> <a data-t="1032">can really supercharge your development speed.</a></p>

<blockquote><a data-t="1011">So AI can be middling at best when actually writing code,</a> <a data-t="1016">but I found that it's actually very good</a> <a data-t="1018">and very consistent at explaining the purpose</a> <a data-t="1021">of specific snippets of code.</a></blockquote>

<p><a data-t="1035">And while I didn't escape entirely unscathed</a> <a data-t="1038">from boring and tedious front-end work,</a> <a data-t="1040">the final result was pretty much exactly what I had in mind.</a> <a data-t="1043">In fact, I liked it a little better</a> <a data-t="1045">than the original WordPress version</a> <a data-t="1047">because this one actually rotated between images</a> <a data-t="1050">rather than just faded, as you'd think something</a> <a data-t="1052">called an image carousel would.</a></p>
