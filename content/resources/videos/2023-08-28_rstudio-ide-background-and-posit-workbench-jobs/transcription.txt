<p><a data-t="0">Now at their core, background and workbench jobs are very similar in that they allow you to run</a> <a data-t="5">scripts in a separate session.</a> <a data-t="11">This is a great feature for any long running scripts you may have,</a> <a data-t="18">including for example a machine learning training set.</a> <a data-t="18">It also allows you to run multiple jobs</a> <a data-t="23">simultaneously.</a></p>

<p><a data-t="23">Now the main difference between background jobs and workbench jobs is that</a> <a data-t="30">background jobs run as a child process within the same session and workbench jobs run as a</a> <a data-t="36">new session either on the local server or off host if you're running in a multi-server or</a> <a data-t="42">clustered environment.</a></p>
