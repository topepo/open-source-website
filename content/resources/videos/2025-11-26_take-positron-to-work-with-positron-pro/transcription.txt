<h3>Positron vs. Positron Pro</h3>

<p><a data-t="71">One of the most common questions we're asked about Positron is, when should I pay for Positron?</a> <a data-t="78">Just like with RStudio, we distribute two versions of Positron, a free, source-available</a> <a data-t="84">desktop application for Mac, Windows, or Linux that's called Positron Desktop, or</a> <a data-t="90">more often, just Positron, and Positron Pro, a commercially licensed, native server version</a> <a data-t="98">of Positron that's accessible via browser and only available in Posit Workbench.</a></p>

<p><a data-t="108">What are the key differences in features and capabilities between Positron and Positron Pro?</a> <a data-t="113">Well, I've tried to list everything worth mentioning here, but the key message is that</a> <a data-t="117">our goal is and has always been that the Positron development experience you know and love is</a> <a data-t="123">fully available in the free desktop application.</a> <a data-t="128">Unlike other IDEs, we don't want you to have to pay for a Pro version of our software to</a> <a data-t="132">unlock a better development experience that's tailored for data science.</a></p>

<p><a data-t="138">In the free desktop version of Positron, you can code natively in both R and Python,</a> <a data-t="144">install OpenVSX-compatible extensions, customize your layout with dedicated panes for data</a> <a data-t="150">science work, push and pull code from version control, preview data apps, and even publish</a> <a data-t="157">to Posit Connect and use Positron's AI capabilities without needing a Positron Pro license.</a></p>

<p><a data-t="165">With Positron Pro, you get the same data science development experience you know and</a> <a data-t="169">love from the free version of Positron.</a> <a data-t="173">You also get to benefit from the capabilities built into Posit Workbench, our professional</a> <a data-t="177">data science workbench that includes Positron, RStudio, VS Code, and JupyterLab.</a> <a data-t="185">Things like single sign-on, managed credentials for accessing enterprise data in AWS, Snowflake,</a> <a data-t="191">or Databricks, scalable and customizable compute, monitoring and auditing tools for</a> <a data-t="198">IT and compliance, and dedicated support from Posit engineers when something goes wrong.</a></p>

<p><a data-t="219">When you have several colleagues who also use Positron, or who prefer RStudio, VS Code,</a> <a data-t="224">or JupyterLab for their development work.</a> <a data-t="228">When IT has concerns about the security and management of local desktop installations</a> <a data-t="232">of data science software, or when they want greater monitoring and auditability of your</a> <a data-t="237">use of that software.</a> <a data-t="240">When you and your colleagues are working with datasets that are larger than your laptop</a> <a data-t="243">can handle, and you need quick, easy access to more power.</a> <a data-t="248">Or when your development work is critically important, and you need quick access to support</a> <a data-t="252">when something goes wrong.</a></p>

<h3>Workbench jobs and audited jobs</h3>

<p><a data-t="1127">Another one of my favorite features inside of Posit Workbench are what we call Workbench jobs.</a> <a data-t="1133">If you have code that takes a really long time to execute, for example, you're training</a> <a data-t="1137">a machine learning model, you're running a really long ETL process, or you're just doing</a> <a data-t="1141">something else that takes a really long time to run, and you'd like to run that in the</a> <a data-t="1145">background while you're doing other work.</a> <a data-t="1150">That's what Workbench jobs are for.</a></p>

<p><a data-t="1157">So if I open the Posit Workbench menu here in my left sidebar, down here under the Workbench</a> <a data-t="1162">jobs submenu, I can click Run Job.</a> <a data-t="1167">Just open this menu, which will let me run a Workbench job.</a> <a data-t="1170">So here I can choose a script and a working directory.</a> <a data-t="1173">Let's go inside of our AirQuality directory.</a> <a data-t="1178">We're going to run our load data script.</a></p>

<p><a data-t="1182">And this small resource profile is fine, because I don't really care how long this</a> <a data-t="1187">takes to execute.</a> <a data-t="1188">But I could also give it one of these larger profiles if I wanted to.</a> <a data-t="1194">And similarly, give it a different image if I wanted to, as well as specify a different</a> <a data-t="1198">R version than the default, and provide any other arguments I might want.</a> <a data-t="1201">Once this looks good, I can hit Start.</a> <a data-t="1209">Then I can monitor that job in the same menu that I opened up moments ago.</a> <a data-t="1212">As it's running, or when it's completed, I can click on this and see some information</a> <a data-t="1218">about that job.</a></p>

<p><a data-t="1225">And critically, if you work in a regulated industry, or you do analyses where it's really</a> <a data-t="1235">important to verify the validity of the environment in which you're doing this analysis, we have</a> <a data-t="1241">a new kind of job in Workbench called audited jobs, which in addition to logging some basic</a> <a data-t="1245">information about this job, who ran it, when it was run, some of the settings about the</a> <a data-t="1251">environment, you can optionally record even more detailed information, like the versions</a> <a data-t="1256">of R and Python that were used, the operating system, any environment variables that were</a> <a data-t="1261">used, and even a digital signature to verify that this specific analysis was run in this</a> <a data-t="1266">specific environment.</a></p>
