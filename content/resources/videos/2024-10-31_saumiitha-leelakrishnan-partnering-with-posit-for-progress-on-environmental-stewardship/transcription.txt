<h3>Step one: grasp your needs</h3>

<h3>Advancing with machine learning</h3>

<p><a data-t="508">So now we have a web application.</a> <a data-t="513">Engineers can use it.</a> <a data-t="515">They can derive insights from it.</a> <a data-t="517">They have action items to work on.</a> <a data-t="521">So what is the next step?</a> <a data-t="523">Is that all there is?</a> <a data-t="524">No, we would have to advance and adapt.</a></p>

<p><a data-t="528">So in this example, I would like to say</a> <a data-t="530">that the engine serial numbers that</a> <a data-t="533">have been identified to have presumably not accumulated</a> <a data-t="539">enough of that IOMPA ratio I mentioned earlier,</a> <a data-t="542">you need some key attributes like duty cycle</a> <a data-t="545">and application so that the engineering teams can</a> <a data-t="549">go and analyze the data.</a> <a data-t="550">So how do I find the duty cycle?</a> <a data-t="552">So before then, what's a duty cycle?</a> <a data-t="555">So duty cycle is a definition of history of speed and load</a> <a data-t="560">conditions over which an engine is</a> <a data-t="562">operating over a period of time for any selected application.</a></p>

<p><a data-t="567">So I wanted to find the duty cycle with the engineering data</a> <a data-t="571">that was available in hand with me.</a> <a data-t="574">In that case, I jumped into using</a> <a data-t="577">the simplest and the popular unsupervised machine learning</a> <a data-t="580">model, which is k-means clustering.</a> <a data-t="583">I utilized this Python library for it.</a> <a data-t="586">So I worked with subject matter experts</a> <a data-t="589">to identify five engine features based on the characteristics</a> <a data-t="595">to determine the duty cycle.</a> <a data-t="597">So it was idle percent, torque curve percent,</a> <a data-t="601">motoring percent, key cycles per hour,</a> <a data-t="604">and drive cycle average percent.</a></p>

<p><a data-t="607">So once I identified these engine features,</a> <a data-t="612">how I used it in the model is that how this k-means</a> <a data-t="616">clustering model works is that it first randomly assigns</a> <a data-t="620">a data point to one of your clusters,</a> <a data-t="623">and then you compute the centroid of each cluster.</a> <a data-t="626">Then there is reassignment of your centroid of each data</a> <a data-t="629">point to the nearest centroid of your cluster.</a> <a data-t="632">And this continues going until there</a> <a data-t="634">is no changing of your clusters happening.</a> <a data-t="637">Thus, finally, you're able to identify your duty</a> <a data-t="640">cycle of each of the engine serial numbers out there.</a></p>

<p><a data-t="644">So that will give an insight on what conditions</a> <a data-t="648">do you think those engines were not</a> <a data-t="651">able to accumulate enough of the IOMPR ratio.</a></p>
