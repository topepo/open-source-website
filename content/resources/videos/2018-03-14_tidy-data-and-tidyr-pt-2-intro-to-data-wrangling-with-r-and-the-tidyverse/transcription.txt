<p><a data-t="346">And so we get a vector that has, at the end, just as many values as the original vectors.</a> <a data-t="352">Now this is great if you put your data in such a way that each variable is in its own column,</a> <a data-t="357">and then each row contains a single observation.</a> <a data-t="360">Because what element-wise operations will do is it will make sure that the values of one observation</a> <a data-t="365">are only paired with other values from the same observation.</a> <a data-t="368">Which as a data scientist is just what you want.</a></p>

<p><a data-t="392">So that's sort of a roundabout way to say that there's one structure that just works well with the natural quirks of R,</a> <a data-t="399">and if you put your data into that structure before you begin to analyze your data,</a> <a data-t="403">you'll have a much quicker, more efficient experience when you work with that data.</a> <a data-t="407">And I'm going to call that data structure tidy data.</a></p>

<blockquote><a data-t="407">And I'm going to call that data structure tidy data.</a></blockquote>

<p><a data-t="411">This is a term invented by or popularized by Hadley, the author of </a><a href="/software/tidyr/">tidyr</a><a data-t="411"> and dplyr.</a> <a data-t="417">And what it amounts to is you take your data,</a> <a data-t="419">and you make sure that each variable is saved in its own column, which we saw storms did.</a> <a data-t="425">Make sure each observation is saved in its own row.</a> <a data-t="429">I'm not going to go into what an observation is here, but it's both obvious and puzzling at the same time.</a> <a data-t="437">And then finally, you want to put basically all the data that you plan to work with in the same data table</a> <a data-t="443">before you start working with it.</a> <a data-t="445">Otherwise, you'll be recombining your data every time you make some sort of adjustment to it.</a></p>

<p><a data-t="450">So if you make your data tidy, with variables and columns, observations and rows,</a> <a data-t="456">and the type of data in its own table, it will be easy to access your variables,</a> <a data-t="461">and you'll automatically preserve observations as you do things to your data.</a> <a data-t="465">So that's sort of the goal. tidyr is the means to get there.</a> <a data-t="469">The package that picks up where reshape2 left off, if you are a user of reshape or reshape2,</a> <a data-t="477">and tidyr has two main functions, gather and spread, that do similar things as cast and melt</a> <a data-t="483">if you're coming from the reshape world.</a> <a data-t="485">And then there's several other functions we'll look at in there.</a> <a data-t="488">But these are the main ones, and let's take a peek at them.</a></p>

<h3>Separate and unite</h3>

<p><a data-t="932">So the tidyr package also has two other functions that I think are useful, but not always going to be that useful.</a> <a data-t="938">If you looked at the STORMS dataset, and you're very persnickety, you might notice that it actually has three more variables in there.</a> <a data-t="943">You could extract a year variable from the date, a month variable, and a day variable.</a> <a data-t="947">And those variables are all combined into one column here, the date column.</a></p>

<p><a data-t="952">What separate does is it takes a single column and it splits it up into multiple columns, based on some sort of separating character.</a> <a data-t="959">So this is really easy with dates, because dates normally have some sort of separator between year, month, and day.</a> <a data-t="965">And if we ran this code above here, we'd get a new data frame where the date column has been split into a year, month, and day, three columns.</a></p>

<p><a data-t="974">If I save that as STORMS2, then I could use UNITE, which does the inverse.</a> <a data-t="978">It takes several columns and puts them together in the same column, adding a separator between the values of the previous columns.</a> <a data-t="984">So that's a great way to recreate some dates, although in this case those dates would be character strings.</a> <a data-t="989">You need to reparse them if you want them to behave as dates in R, but that's another story.</a></p>

<p><a data-t="996">So just to recap about the tidyr package, it primarily works with the layout of your data and reshapes the layout.</a> <a data-t="1005">You can use the gather function and the spread function to change the shapes back and forth.</a> <a data-t="1009">They're complementary, and then you can use separate and unite to split up and unite columns.</a> <a data-t="1013">They're also complementary.</a> <a data-t="1015">This package is designed basically to get data into that tidy format, which is the most efficient way to work with it in R.</a></p>
