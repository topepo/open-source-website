<h3>How Query Chat works under the hood</h3>

<p><a data-t="1157">So, how does query chat work under the hood?</a> <a data-t="1159">It's safe because all it has access to is select statements.</a> <a data-t="1163">So, we're not writing to a database.</a> <a data-t="1165">We are not, you know, changing anything.</a> <a data-t="1168">It is only a select statement.</a> <a data-t="1170">We can think it's reliable because as long as it's writing correct SQL code, which LLMs are actually really good at writing SQL code, all the execution is handled by the database engine.</a> <a data-t="1181">So, we don't have to worry about it sort of filtering data in a weird way or hallucinating an entire data frame because it's only writing the SQL.</a> <a data-t="1194">The database execution executes on the SQL.</a> <a data-t="1198">We can verify because we can literally see the SQL.</a></p>
