<h3>Behavioral science and the BID framework</h3>

<p><a data-t="193">Well, with behavioral science,</a> <a data-t="195">which is the study of how people think, feel, and act.</a> <a data-t="200">Typically in the context of decision making.</a> <a data-t="202">But across decades of research,</a> <a data-t="203">what we find is that people are rational,</a> <a data-t="206">but in surprisingly predictable ways.</a> <a data-t="208">And so for us as developers, this is actually good news,</a> <a data-t="211">because we can understand that strange user behavior</a> <a data-t="214">of ours and design for it.</a></p>

<p><a data-t="217">So here's a classic study to illustrate what I mean here.</a> <a data-t="220">Researchers set up a tasting test at a grocery store</a> <a data-t="223">and varied the number of jams that they had</a> <a data-t="224">on display at this table.</a> <a data-t="227">And in the big display condition,</a> <a data-t="228">there were 24 jams on the table.</a> <a data-t="230">In the small display condition, there were only six jams.</a> <a data-t="233">And the big display with 24 jams</a> <a data-t="237">actually attracted more shoppers.</a> <a data-t="239">60% of people stopped,</a> <a data-t="240">whereas only 40% of people stopped in the small condition.</a> <a data-t="244">But only 3% of people in that large condition</a> <a data-t="247">actually bought jam,</a> <a data-t="248">whereas 30% did in the small condition.</a> <a data-t="252">So that's a convergent difference of 10 times.</a></p>

<p><a data-t="255">And so this is exactly what's going on</a> <a data-t="257">with our dashboards, right?</a> <a data-t="258">We're seeing more features and more options</a> <a data-t="261">attract attention, but paralyze choice.</a></p>

<p><a data-t="265">And so paradox of choice is one key component here,</a> <a data-t="269">but there's a lot of concepts in behavioral science</a> <a data-t="272">that we can pull from that are applicable to dashboards.</a> <a data-t="275">We don't have time to go through all of them,</a> <a data-t="276">I'm just gonna pull out some common themes here,</a> <a data-t="278">just talk really high level about them,</a> <a data-t="279">because they're not just abstract theories.</a> <a data-t="281">These are engineering principles</a> <a data-t="283">that we can leverage in our dashboards.</a></p>

<p><a data-t="286">Cognitive load, for example, right?</a> <a data-t="286">This is how having too many choices</a> <a data-t="289">can actually overwhelm our mental capacity.</a> <a data-t="291">So think of this like RAM.</a> <a data-t="293">Your users have processing limits,</a> <a data-t="295">and each filter adds a processing cost.</a></p>

<p><a data-t="299">Progressive disclosure is the idea</a> <a data-t="301">that we're kind of hiding complexity</a> <a data-t="303">until it's actually needed, right?</a> <a data-t="305">And so we're showing that vital information first,</a> <a data-t="307">those key points up first,</a> <a data-t="309">and then just revealing more later on.</a> <a data-t="311">It's kind of like lazy loading for human cognition.</a> <a data-t="314">And then for immunoflex,</a> <a data-t="315">I'll show you a more concrete example of this,</a> <a data-t="317">but this is how basically the same exact data</a> <a data-t="319">can show two very different stories.</a> <a data-t="322">So revenue being up 15% might be a true statement,</a> <a data-t="325">but it could also be true</a> <a data-t="325">that it's missing the target by 5%.</a> <a data-t="328">And so unless folks are talking about that,</a> <a data-t="330">they might be coming from two very different places.</a></p>

<p><a data-t="333">And so when we apply this to dashboards,</a> <a data-t="335">this might mean showing three filters</a> <a data-t="337">instead of all 18 at first.</a> <a data-t="339">It might mean showing summaries</a> <a data-t="341">instead of those detailed outputs, right?</a> <a data-t="343">And so this is how we start to engineer</a> <a data-t="346">for the user experience.</a></p>

<p><a data-t="348">But what if you don't know anything</a> <a data-t="350">about behavioral science, right?</a> <a data-t="352">Well, you fall back on things</a> <a data-t="354">that you do know something about.</a> <a data-t="356">And I bet you'd know a little something about road trips,</a> <a data-t="358">especially if you took the road trip here, right?</a> <a data-t="361">And so engineering apps with behavioral science</a> <a data-t="363">is a lot like planning for a road trip.</a> <a data-t="365">You need to know your destination,</a> <a data-t="367">you need to anticipate the hazards,</a> <a data-t="368">and you need to make sure everyone arrives there safely.</a> <a data-t="371">Right, and that's where the behavioral insight design</a> <a data-t="374">or the bid framework, right?</a> <a data-t="376">The five simple stages that guide you through this process</a> <a data-t="378">and help prevent those dashboard disasters</a> <a data-t="380">that we've all seen.</a></p>
