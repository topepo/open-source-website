<h3>What is rayshader?</h3>

<p><a data-t="76">So what is rayshader?</a> <a data-t="81">I could start first by describing it in words like a normal person, but luckily, this year,</a> <a data-t="83">I don't have to.</a> <a data-t="83">Because I have a great visual aid created by the unparalleled RStudio artist in residence,</a> <a data-t="88">Allison Horst.</a></p>

<p><a data-t="91">Rayshader is a package designed to create 2D and 3D visualizations in R, specializing</a> <a data-t="95">in maps.</a> <a data-t="96">It includes a wide variety of functions, here represented by fuzzy functional monsters that</a> <a data-t="102">help you build, color, and annotate and beautify your visualizations.</a> <a data-t="107">Specifically, rayshader uses a technique called raytracing in order to realistically simulate</a> <a data-t="113">how light falls on 3D models and landscapes.</a> <a data-t="116">On the left, you can see how rayshader calculates the shadows cast by buildings in Philadelphia</a> <a data-t="122">using a combination of LiDAR data and the suncalc package to calculate the position</a> <a data-t="127">of the sun in the sky.</a></p>

<p><a data-t="129">Rayshader can take these 2D maps and then project them into 3D, as we see here on the</a> <a data-t="134">right.</a> <a data-t="136">So rayshader also includes functions that you can apply post-processing effects to enhance</a> <a data-t="140">these 3D maps.</a> <a data-t="142">Here I've added a depth of field effect to blur areas away from the focal point, as well</a> <a data-t="146">as a camera vignetting effect to add a subtle darkening around the edges like you'd see</a> <a data-t="151">in a DSLR.</a> <a data-t="153">This 3D visualization was created entirely in R with no external software required.</a></p>

<p><a data-t="160">Rayshader does all this with a straightforward pipeable interface where you use elevation</a> <a data-t="165">data to generate shadows and maps.</a> <a data-t="168">Here the Hobart object at the top of the chain is simply a 2D array of elevation values</a> <a data-t="174">at every point.</a> <a data-t="175">It's just an R matrix.</a> <a data-t="177">We use that first to generate a color layer, add a water layer, add some shadow layers,</a> <a data-t="182">and then finally plot the map.</a> <a data-t="184">To project this map into 3D, we simply replace the plot map function with the plot 3D function</a> <a data-t="190">and include the original matrix of height values.</a> <a data-t="194">This function generates a 3D mesh generated by the Hobart matrix and then paints the surface</a> <a data-t="199">of it with the output of everything above.</a></p>

<h3>Extending rayshader beyond maps</h3>

<p><a data-t="208">Mapping is great, but one thing I always get when I demo Rayshader is people say, I'd love</a> <a data-t="213">to have a reason to use that, but I don't work with geographic data.</a> <a data-t="218">So let's say I wanted to extend Rayshader to work with other types of data.</a> <a data-t="223">Let's first specify what type of 3D plot we want to work on.</a></p>

<p><a data-t="228">Typically when we talk about 3D plots, there are four common types we're referring to.</a> <a data-t="233">First is the 3D scatter plot, which is a bunch of points floating in space.</a> <a data-t="238">Second is the 3D line plot, which is a bunch of points floating in space connected by lines.</a> <a data-t="244">Third is the surface plot, which is a contiguous mesh of points representing a solid surface.</a> <a data-t="250">And finally is the abomination, which is what happens when you take a plot that doesn't</a> <a data-t="257">want to be 3D, but you pull it into the third dimension anyway.</a></p>

<p><a data-t="262">To make sure we don't make any abominations, we'll only use 3D plots when we have three</a> <a data-t="266">continuous variables, variables we map to the X, Y, and Z directions.</a> <a data-t="272">We'll also avoid plots that involve data floating in space, as those suffer from the inability</a> <a data-t="276">of the reader to correctly determine the depth of the points in space, leading to ambiguities</a> <a data-t="280">in interpreting the data.</a> <a data-t="283">Surface plots suffer far less from this problem, since the regular grid gives the reader more</a> <a data-t="286">perceptual cues to the data's spatial ordering.</a></p>

<p><a data-t="291">We will also remove the axes, which might seem heretical in data viz, but axes in 3D</a> <a data-t="295">don't work the same way they do in 2D.</a> <a data-t="298">The viewer can't unambiguously map the data back to their values.</a> <a data-t="302">Putting your 3D visualization in a box rarely improves it, and if a visualization requires</a> <a data-t="308">the axes to be interpretable, it's likely not a good candidate for 3D in the first place.</a></p>

<blockquote><a data-t="302">Putting your 3D visualization in a box rarely improves it, and if a visualization requires</a> <a data-t="308">the axes to be interpretable, it's likely not a good candidate for 3D in the first place.</a></blockquote>

<h3>Designing the API: why ggplot?</h3>

<p><a data-t="315">So we're going to focus on surface plots.</a> <a data-t="317">What's our interface going to look like?</a> <a data-t="319">When you're creating a new software package, one good idea before you start is to survey</a> <a data-t="323">what tools are already out there.</a> <a data-t="325">This serves two purposes.</a> <a data-t="326">One, to figure out whether you even need to write your package, whether the niche is already</a> <a data-t="330">filled.</a> <a data-t="331">And second, to get inspiration for what your software should, or maybe shouldn't, do.</a></p>

<p><a data-t="336">So what's the premier tool out there for 3D visualization currently?</a> <a data-t="341">Blender.</a> <a data-t="343">Blender is a 3D modeling, rendering, sculpting, simulation, visual effects, and everything</a> <a data-t="347">but the Kitchen Sink software package, open source.</a> <a data-t="352">It's an extremely powerful piece of software that produces absolutely breathtaking 3D renderings,</a> <a data-t="357">including 3D data visualizations.</a> <a data-t="359">Here are a few examples taken with permission from artists and cartographers who use Blender</a> <a data-t="365">to visualize data with gorgeous results.</a></p>

<p><a data-t="369">There's a few issues, however.</a> <a data-t="371">First, Blender is primarily a GUI-based application, as you see here, along with the text I have</a> <a data-t="377">added.</a> <a data-t="378">And GUI-based workflows are not reproducible.</a> <a data-t="380">One of the benefits of using R to power your visualization and analysis pipeline is that</a> <a data-t="385">you can reproduce old analyses years after the fact, rerun the same exact analyses on</a> <a data-t="390">new batches of data, and make slight changes to figures without having to redo the entire</a> <a data-t="354">thing.</a> <a data-t="415">There's a Python interface to Blender, but it's primarily aimed at developers, not at</a> <a data-t="420">data scientists and analysts.</a> <a data-t="424">Data is not a first-class citizen.</a></p>

<p><a data-t="428">Additionally, you're here at this conference because you've at least somewhat invested</a> <a data-t="431">yourself in the R ecosystem.</a> <a data-t="431">You've taken workshops, read tutorials, and presumably have spent a good deal of time</a> <a data-t="435">becoming familiar and maybe even an expert in R. And all of those skills will be absolutely</a> <a data-t="442">useless when it comes to learning Blender, except for maybe robust anger management techniques.</a></p>

<p><a data-t="450">Which, if you saw </a><a href="/people/jenny-bryan/">Jenny Bryan</a><a data-t="450">'s keynote this morning, you'll know the frustration of seeing</a> <a data-t="456">Objective-type closure is not subsettable for the first time.</a> <a data-t="460">So it's a stretch to ask most people to learn one complex piece of software.</a> <a data-t="465">Learning two to perform a single task makes that task inaccessible to all but the most</a> <a data-t="470">perseverant of individuals.</a></p>

<p><a data-t="473">So for a surface-plotting API, I have a few goals in mind.</a> <a data-t="477">First and foremost, you should never have to think about vertices or triangles or polygons.</a> <a data-t="482">I want a 3D-plotting interface that's focused around data, not computer graphics.</a> <a data-t="486">Secondly, I want to avoid introducing new interfaces you'll have to learn.</a> <a data-t="490">I want to use as much as what you already know to lower the barrier to entry for using</a> <a data-t="494">the software.</a> <a data-t="496">And finally, I want the interface to be friendly and simple enough that people feel like they</a> <a data-t="500">can play with it.</a> <a data-t="502">Because play and experimentation is the fuel for creativity and learning.</a></p>

<p><a data-t="507">And what fills all these requirements?</a> <a data-t="509">ggplot.</a> <a data-t="510">It's a plotting library that's data-centric, well-known by many people, and easy to experiment</a> <a data-t="516">with.</a></p>

<h3>How plot_gg works</h3>

<p><a data-t="517">So you might be asking, wait, </a><a href="/software/ggplot2/">ggplot2</a><a data-t="517"> is a 2D-plotting library.</a> <a data-t="522">Is there a ggplot3 I don't know about?</a> <a data-t="525">No, but I'll tell you how you can transform into 3D.</a> <a data-t="532">Recall what I said earlier.</a> <a data-t="532">To plot a 3D surface, all you need are three variables that we map to the X, Y, and Z directions.</a> <a data-t="537">The X and Y aesthetics are built into ggplot, and we encode the third aesthetic representing</a> <a data-t="542">height to color.</a> <a data-t="544">Then we've encoded all three variables of our 3D surface into our ggplot.</a></p>

<p><a data-t="550">Images are, generally speaking, several matrices just layered together, each which specifies</a> <a data-t="554">a color intensity.</a> <a data-t="556">So even an image can be a height matrix, for better or for worse.</a> <a data-t="563">Sorry, Hadley.</a></p>

<p><a data-t="565">The plot3D function, which performs all the 3D mapping logic, only cares that it's past</a> <a data-t="570">a matrix of height values in an image.</a> <a data-t="572">If we encode our data into one of those color layers, we can then extract a matrix of height</a> <a data-t="577">values from our ggplot and create a 3D surface onto which we can paint our data.</a> <a data-t="582">By swapping the ggplot's palette to a grayscale black and white palette, it ensures we have</a> <a data-t="586">a one-to-one relationship between our data and our height value.</a> <a data-t="590">We then strip out all the non-color elements and non-data elements, axes, labels, backgrounds,</a> <a data-t="595">text, and our grayscale image now represents a height map of the surface.</a> <a data-t="599">We then drop this into plot3D along with the ggplot texture, and now we have our 3D ggplot.</a></p>

<p><a data-t="607">You might ask, wait, you guarantee me little to no effort on my part.</a> <a data-t="613">Stripping out all these theme elements seems like a lot of effort, and it is, which is</a> <a data-t="618">why I automated the entire process into a single-function plot gg.</a> <a data-t="624">You just pass in the ggplot object, and it transforms it into a 3D surface, no other</a> <a data-t="633">work required.</a> <a data-t="633">No drawing polygons, no fiddling with complex GUIs, just one simple API to call, drop in</a> <a data-t="639">your ggplot, and watch it grow.</a></p>

<p><a data-t="644">PlotGG works with both the color and fill aesthetics, and RayShader automatically detects</a> <a data-t="648">which one you used.</a> <a data-t="650">Here I used the color aesthetic in a point geome, which is then projected to 3D.</a> <a data-t="658">If you have a complex plot that uses both, you can specify in PlotGG which one to use.</a></p>

<h3>Animating and enhancing 3D plots</h3>
