<h3>Who the Carpentries are</h3>

<p><a data-t="49">In order to understand who the Carpentries are, I'm going to tell you a little story</a> <a data-t="53">about myself.</a> <a data-t="55">This is not a recent photo.</a> <a data-t="56">This was from 2014.</a> <a data-t="58">I was teaching my very first conference workshop in Population Genetics in R, giving it to</a> <a data-t="64">plant pathologists.</a> <a data-t="65">If you're wondering what Population Genetics in R is, think 23andMe, but for fungi.</a></p>

<h3>Building the Carpentries Workbench</h3>

<p><a data-t="379">And so I was hired in 2020 to reimagine this infrastructure, because it was clear it was</a> <a data-t="384">not living up to our values.</a> <a data-t="387">And what I realized is that we really need to separate the tools from the content.</a> <a data-t="394">Anything everybody should worry about is just being able to write markdown, or R markdown,</a> <a data-t="400">and get a website out of it, and not having to sift through all the output that's contained</a> <a data-t="406">in the same folder, or having to worry about tools breaking.</a></p>

<p><a data-t="411">And so this is how I created the Carpentries Workbench, and this is a series of packages</a> <a data-t="418">that avoids having a lesson being wrapped around a blog infrastructure, and it keeps</a> <a data-t="424">the tools and outputs separate, and it's based on the philosophy of package down.</a> <a data-t="429">Work where you are.</a></p>

<p><a data-t="432">And this, it consists of three packages.</a> <a data-t="436">The first package that you're going to interact with is sandpaper.</a> <a data-t="440">This is what orchestrates everything.</a> <a data-t="442">The only command you really need to know is sandpaper serve.</a> <a data-t="445">The next package is on the inside, and you don't interact with it so often, but you do</a> <a data-t="451">with it, you do, sandpaper uses it in the backend to help you validate your markdown,</a> <a data-t="458">to make sure that it actually is going to do what you want it to do.</a> <a data-t="463">And then coming in at the end is varnish.</a> <a data-t="466">That gives you the styling of your lessons, to make sure that your lessons are accessible</a> <a data-t="471">and beautiful.</a></p>

<p><a data-t="473">And all of this was designed with what I like to call core values-driven design.</a> <a data-t="479">And so I'm going to talk about a few of our core values and how the design elements fit</a> <a data-t="484">in with them.</a> <a data-t="485">There are a lot more, but I don't have that much time.</a></p>

<h3>Core values-driven design</h3>

<p><a data-t="489">So the first thing I'm going to start with is championing people first, because really</a> <a data-t="495">individuals who make up our community are the most important part of our organization.</a> <a data-t="500">And they are our strongest resource.</a> <a data-t="502">So it only makes sense that when I was looking for a static site generator, something to</a> <a data-t="508">generate our websites with, that I would choose something that had the most on ramps.</a> <a data-t="514">And really, R is the best resource.</a> <a data-t="519">It makes the most sense.</a></p>

<p><a data-t="522">You saw it in Ileana's talk, and you see it here.</a> <a data-t="526">And this tweet from Janita Bellini, and she says that some people laugh when I tell them</a> <a data-t="531">that R's best feature as a language is its community, but I'm very serious.</a> <a data-t="537">Especially Rladies has been such a positive agent of change for so many people.</a> <a data-t="543">That's one of the reasons why y'all are here.</a> <a data-t="545">We are a good community here.</a> <a data-t="548">We know how to make people feel included.</a> <a data-t="551">And so it only makes sense.</a></p>

<blockquote><a data-t="526">And this tweet from Janita Bellini, and she says that some people laugh when I tell them</a> <a data-t="531">that R's best feature as a language is its community, but I'm very serious.</a></blockquote>

<p><a data-t="554">Another way that we're championing people first is that we virtually increase the number</a> <a data-t="559">of instructors in the room when you're teaching a lesson.</a> <a data-t="562">And I'm going to explain what I mean by that.</a> <a data-t="565">So you have your lesson.</a> <a data-t="566">It has code examples.</a> <a data-t="568">And you want to teach someone how to add a column to a data frame.</a> <a data-t="572">Okay?</a> <a data-t="573">So you are working in RStudio, and you show them the code, you run the code, and then</a> <a data-t="579">you have output that looks like this.</a> <a data-t="581">You can't really tell what happened, because there are so many columns in the data frame,</a> <a data-t="585">everything ran off the edge.</a></p>

<p><a data-t="588">If you had the wisdom of the instructors who came before you, and yes, you are teaching</a> <a data-t="594">with another instructor.</a> <a data-t="595">They may know this, but they may not.</a> <a data-t="599">You would know that you have to add a select command.</a> <a data-t="604">And so what we do is we allow instructors to communicate through time and space by allowing</a> <a data-t="612">them to include instructor notes directly into the lessons in the context that is needed.</a> <a data-t="619">So that when you teach this lesson, you can change your behavior, and you can show the</a> <a data-t="624">learners what exactly they need to know and reduce their cognitive load.</a></p>

<p><a data-t="630">And another way we champion people first is that we include automation.</a> <a data-t="635">Oftentimes, as you're building R Markdown, packages will update.</a> <a data-t="641">Things fail.</a> <a data-t="642">You get a lot of errors.</a> <a data-t="643">You get warnings that don't make sense, because the package authors changed their packages.</a> <a data-t="649">We avoid this by embedding </a><a href="/software/renv/">renv</a><a data-t="649"> on the back end of sandpaper.</a> <a data-t="659">And the beauty about this, you don't need to know anything about renv in order to use</a> <a data-t="664">this.</a> <a data-t="664">We lock the packages.</a> <a data-t="665">We give you a way to update them, and we give you a way to pin them so that your lessons</a> <a data-t="672">are always working and you have a way to investigate what happens when packages update.</a> <a data-t="678">And I'm going to show you some of that a little bit later.</a></p>

<p><a data-t="682">And really, this leads me into the fact that we champion access for all, because it shouldn't</a> <a data-t="690">matter what packages you have on your system.</a> <a data-t="692">It shouldn't matter what packages you have in your global library.</a> <a data-t="695">You should be able to build the lesson the same anyone else can build the lesson.</a> <a data-t="700">And so I'm going to show you, like, what happens when you don't have your own computer,</a> <a data-t="707">when you're not on your own computer and you receive a pull request.</a> <a data-t="710">That changes a plot.</a> <a data-t="713">But can anybody tell me what's going to happen when this happens?</a> <a data-t="717">Probably not.</a> <a data-t="718">Because it's R code.</a> <a data-t="721">But seeing the output of the plot is night and day.</a></p>

<p><a data-t="725">And we have embedded pull request previews for the intermediate markdown for these lessons</a> <a data-t="734">so that you can confirm that things actually work, whether or not you have the tools installed.</a></p>

<p><a data-t="752">And furthermore, if you're writing an R lesson, sometimes you may forget to include alternative</a> <a data-t="757">text or you have link descriptions that don't actually describe anything.</a> <a data-t="762">And you say, whoops.</a> <a data-t="764">And it's this kind of preflight validation that's done by Pegboard that allows you to</a> <a data-t="772">take a step back and say, yeah, I made a mistake, and fix it.</a> <a data-t="777">And that's really the key to how we're championing access for all, and we're always learning.</a> <a data-t="785">Because we're humans.</a> <a data-t="787">We forget things.</a> <a data-t="789">You shouldn't be punished.</a> <a data-t="790">You shouldn't have, like, glaring errors when you miss something.</a> <a data-t="796">When you mistype something.</a> <a data-t="799">And it shows that we're always learning.</a></p>

<p><a data-t="801">And I'm going to turn it on myself.</a> <a data-t="803">I'm always learning.</a> <a data-t="805">Sandpaper is not the fastest thing in the world.</a> <a data-t="807">The Workbench is not the fastest thing in the world.</a> <a data-t="810">But I really want to address the pink pachyderm in the premises.</a> <a data-t="817">We don't use </a><a href="/software/quarto/">Quarto</a><a data-t="817"> yet.</a> <a data-t="820">Sorry.</a> <a data-t="822">But the way I designed the system, we're going to include Quarto in the future.</a> <a data-t="830">We just need to get it stable in our system.</a> <a data-t="833">And I built it to be flexible.</a> <a data-t="835">So it works on the exact same principle that Quarto works.</a></p>

<p><a data-t="841">And then finally, I want to get to the fact that we empower one another.</a> <a data-t="845">The Workbench is going to support 168 lessons come January.</a> <a data-t="851">And this is really key, because most of these lessons are in the Carpentries Incubator,</a> <a data-t="857">in which people are actively developing lessons, working on lessons, testing them out, teaching</a> <a data-t="865">them, and getting feedback and revamping them in this iterative development cycle that eventually</a> <a data-t="871">leads to peer review of lessons.</a> <a data-t="874">And this is the Carpentries Workbench makes this model sustainable.</a> <a data-t="881">It makes it much easier for people to do these iterations with all of these safeguards that</a> <a data-t="886">we've built into it.</a></p>
