<h3>The ideal approach: SASPy</h3>

<p><a data-t="330">And the answer is yes, and there are multiple different approaches.</a> <a data-t="333">The most straightforward approach is a simple three-step recipe, and step one of this recipe</a> <a data-t="339">is to gather your required ingredients.</a> <a data-t="342">For this particular recipe, the required ingredients are the languages that you're interested in</a> <a data-t="346">using plus Java.</a> <a data-t="349">And the other ingredient that's not pictured here, but which really is the most important</a> <a data-t="352">ingredient for making this particular recipe succeed is admin privileges.</a> <a data-t="357">And if you don't have admin privileges on your computer, you might not be able to get</a> <a data-t="361">this recipe to work.</a></p>

<p><a data-t="365">Step two of this three-step recipe, assuming you have all of those required ingredients</a> <a data-t="369">from step one, is to install SASPy, which is a Python package that works very similarly</a> <a data-t="374">to how </a><a href="/software/reticulate/">Reticulate</a><a data-t="374"> works.</a> <a data-t="377">So just like Reticulate is an R interface to Python, SASPy is a Python interface to</a> <a data-t="384">SAS.</a></p>

<p><a data-t="412">Now that was definitely the most straightforward approach and the recommended, well, my personal</a> <a data-t="416">recommended way of how to achieve this workflow, but if you have limited control over your</a> <a data-t="421">computer, then it might not be possible for you.</a> <a data-t="423">So in the next few slides, I'm going to present a few alternative approaches that will allow</a> <a data-t="427">you to combine SAS and R in the same script, and sometimes Python.</a> <a data-t="433">The approach that you choose is going to depend heavily on what you need to do and on how much</a> <a data-t="437">control you have over your computer.</a> <a data-t="440">And for the three alternative options that I'm about to present, for all three of them,</a> <a data-t="444">there's going to be a big tradeoff between complexity and usefulness.</a></p>
