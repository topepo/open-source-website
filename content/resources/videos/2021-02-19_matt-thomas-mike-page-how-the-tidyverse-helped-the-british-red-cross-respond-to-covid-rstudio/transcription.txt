<h3>Why the Tidyverse?</h3>

<p><a data-t="210">So why the Tidyverse?</a> <a data-t="214">Every data science problem presents a unique landscape which must be navigated.</a> <a data-t="219">Here at the British Red Cross, we like to think of the Tidyverse as the cognitive superhighway</a> <a data-t="224">through many of these landscapes.</a> <a data-t="228">The signs are clear and easy to follow, and it's easy to show colleagues and your future</a> <a data-t="233">self how to navigate back through the terrain.</a></p>

<p><a data-t="238">But this is not to say the Tidyverse is always the correct path to take for every problem.</a> <a data-t="244">Depending upon the task at hand, other tools and paradigms may be better suited, such as</a> <a data-t="249">Pandas and Datatable.</a> <a data-t="254">However, at the British Red Cross, we often have a requirement to develop and collaborate</a> <a data-t="259">quickly, and this is where we believe the Tidyverse shines most brightly.</a> <a data-t="268">It quickly allows us to translate our ideas to code.</a></p>

<p><a data-t="275">This is because the unifying design principles of the Tidyverse are centred first and foremost</a> <a data-t="280">around cognitive performance, not computational performance.</a></p>

<blockquote><a data-t="275">This is because the unifying design principles of the Tidyverse are centred first and foremost</a> <a data-t="280">around cognitive performance, not computational performance.</a></blockquote>

<h3>Human-centred and inclusive principles</h3>

<p><a data-t="303">First, let's take a look at the human-centred and inclusive principles.</a> <a data-t="308">Together, they acknowledge that data scientists come in all shapes and sizes.</a> <a data-t="313">This means the Tidyverse APIs and documentation are designed to be read and written by everyone,</a> <a data-t="318">irrespective of their background.</a> <a data-t="320">This means it's not just for computer science graduates whose idea of a fun weekend is arguing</a> <a data-t="324">about whether a rate should be 0 or 1 indexed.</a></p>

<p><a data-t="330">This quote from the Tidyverse design principles book sums up the human and inclusive principles</a> <a data-t="335">nicely.</a></p>

<p><a data-t="338">One particularly powerful idea is that of affordance, the exterior of a tool, so suggest</a> <a data-t="343">how to use it.</a></p>

<p><a data-t="346">For example, take a look at this code snippet written by Matt that calculates a mean total</a> <a data-t="351">response time from the fire and rescue authorities using a classic </a><a href="/software/dplyr/">dplyr</a><a data-t="351">-style pipeline.</a></p>

<p><a data-t="363">Without ever having seen the code, and only a little bit of prior Tidyverse knowledge,</a> <a data-t="367">it's quick to read and intuitive to understand what is going on.</a> <a data-t="372">Take the fire and rescue authority stats, filter only for fires in English dwellings</a> <a data-t="378">and create a new variable called year, select only the variables of interest, and finally,</a> <a data-t="384">group by a geographical variable to find the average 3-year fire and rescue response time</a> <a data-t="388">in a given area.</a></p>

<p><a data-t="391">The code almost reads like English prose, listing off a set of clear instructions.</a> <a data-t="396">There is no requirement to reason about things that computers are optimized to do, such as</a> <a data-t="401">indexing and subsetting.</a> <a data-t="401">The result is code that is quicker to write, read, and collaborate on.</a></p>

<h3>Consistency</h3>

<p><a data-t="410">Another important design principle of the Tidyverse is that of consistency.</a> <a data-t="415">Principally, this idea means finding the smallest possible set of key ideas and using</a> <a data-t="420">them again and again.</a> <a data-t="424">This manifests itself in two ways in the Tidyverse, in data structures and in function APIs.</a></p>

<p><a data-t="429">The code snippet I'm about to show demonstrates the power of this idea effectively.</a> <a data-t="432">This code generates a data frame of weights across four domains of vulnerability where</a> <a data-t="436">only two domains should be weighted at any given time, and the sum of the weights must</a> <a data-t="437">equal one.</a> <a data-t="437">The main power of this snippet lies in the first function, crossing, which generates</a> <a data-t="443">all combinations of variables found in the dataset while de-duplicating and sorting the</a> <a data-t="447">inputs.</a></p>

<p><a data-t="449">Here, the yellow arrows demonstrate the consistent tidy data frame structures used throughout</a> <a data-t="456">this example.</a> <a data-t="459">Using one common data structure over and over again reduced the cognitive load required</a> <a data-t="463">to reason about the problem we were challenged with.</a> <a data-t="467">One reason for this is that the consistent data structures mean the APIs of the Tidyverse</a> <a data-t="472">communicate seamlessly.</a></p>

<p><a data-t="478">For example, we can see here how the functions from </a><a href="/software/tidyr/">TidyR</a><a data-t="478"> and BplyR fit together like Lego</a> <a data-t="483">blocks.</a> <a data-t="484">This mostly removes the need to reason about the input and output structure of the data</a> <a data-t="487">being passed throughout the Tidyverse, making it quick to learn new tidy functions and packages.</a></p>

<p><a data-t="475">Consistency also extends to the naming of functions throughout the Tidyverse.</a> <a data-t="478">In general, Tidyverse function names offer imperative verbs over nouns, long names over</a> <a data-t="482">short names, and </a><a href="/software/thematic/">thematic</a><a data-t="482"> unity between similar groups of functions.</a> <a data-t="486">The end result are functions that are easier to remember and implement.</a> <a data-t="490">For this particular code snippet, I recall a deep sense of joy as I intuitively knew</a> <a data-t="494">the name of the functions I needed to recall, despite only using several of them a handful</a> <a data-t="498">of times.</a></p>

<h3>Composability</h3>

<p><a data-t="503">The final design principle of the Tidyverse is that it should be composable.</a> <a data-t="508">This means that complex problems should be solved by combining many smaller pieces, with</a> <a data-t="510">each smaller piece easy to reason about.</a> <a data-t="510">This is realised in the Tidyverse for a predominant functional programming paradigm that uses</a> <a data-t="514">immutable objects.</a> <a data-t="516">This means problems are solved by combining functions that transform data, rather than</a> <a data-t="520">by creating objects whose state changes over time.</a></p>

<p><a data-t="524">We saw the power of this idea previously, where a consistent immutable Tidy DataFrame</a> <a data-t="528">object was used throughout.</a> <a data-t="530">This long code block demonstrates the idea of composability nicely.</a> <a data-t="534">Try not to be drawn in to immediately understand it.</a></p>

<p><a data-t="540">Instead, notice how many smaller pieces of the Tidyverse are used to solve an overall</a> <a data-t="545">more complex problem.</a> <a data-t="547">In some instances, some functions such as mutate are used over and over again.</a> <a data-t="551">The end result is we are able to solve complex problems more quickly.</a> <a data-t="556">In this particular case, it allowed us to assess the vulnerability of different regions</a> <a data-t="562">of the UK, split across different vulnerabilities of the vulnerability index.</a></p>

<h3>Real-world impact</h3>

<p><a data-t="576">Have you noticed how all the principles are linked together?</a> <a data-t="580">The Tidyverse principles are rooted in cognitive theories of mind, and they make it easy for</a> <a data-t="585">humans like me and Mike to pull insights out of datasets and make new tools for people.</a></p>

<p><a data-t="592">One example of how we've used our vulnerability index to design better services for people</a> <a data-t="598">is our COVID-19 Hardship Fund.</a> <a data-t="601">Aviva, an insurance company, donated £5 million to us to give to people who are facing economic</a> <a data-t="607">hardship during the pandemic.</a> <a data-t="610">We used our vulnerability index, alongside other research, to figure out that the people</a> <a data-t="615">most likely to need financial support included those without access to state welfare, older</a> <a data-t="621">isolated people, people facing homelessness or living in temporary accommodation, and</a> <a data-t="627">survivors of gender-based violence.</a></p>

<p><a data-t="631">In one region of England, this mapping, combined with local intelligence from our</a> <a data-t="635">frontline teams, led to more than half of referrals to our Hardship Fund coming from</a> <a data-t="641">the most vulnerable areas.</a> <a data-t="643">As of December 2020, we've helped more than 7,000 people with financial assistance.</a></p>
