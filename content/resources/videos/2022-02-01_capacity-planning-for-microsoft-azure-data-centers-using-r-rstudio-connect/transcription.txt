<h3>Availability zones and demand shaping</h3>

<p><a data-t="2031">Okay, so before I get to the demo itself, let me give a brief introduction</a> <a data-t="2041">to availability zones. So availability zones are physically and logically separated data centers</a> <a data-t="2051">with their own independent power source and pooling. Okay, so the basic idea is that if</a> <a data-t="2067">one of these zones go down, right, perhaps due to some natural disaster or,</a> <a data-t="2095">you know, or maybe somebody accidentally hits the power off switch. So if any of these things</a> <a data-t="2103">happen, then those workloads can fill over to another availability zone, and that workload</a> <a data-t="2111">can continue to function. So it's this fill over capability that gives our customers workloads that</a> <a data-t="2120">high availability, right? It's how we can guarantee these high SLAs for our customers.</a></p>

<p><a data-t="2129">Now, in terms of demand, we actually think of a demand as split into three buckets. So there is a</a> <a data-t="2138">portion of demand that we think of as being pinned to a particular zone. So this workload must</a> <a data-t="2146">happen in this physical location, right? So there is that type of demand. There's a second class</a> <a data-t="2152">of demand where it's just, you know, it just has to be replicated across all three zones,</a> <a data-t="2158">mainly for that resiliency that I was talking about. And the last type of demand is this</a> <a data-t="2166">discretionary demand. So there's this third type of demand where we are basically free to</a> <a data-t="2173">allocate that demand into any or all of these availability zones as we see fit.</a></p>

<p><a data-t="2183">Okay, so now,</a> <a data-t="2190">so we can place this demand, we can allocate this demand however we want. And so then the question</a> <a data-t="2191">is, well, how should we allocate this demand, right? So there are a number of policies that we</a> <a data-t="2198">might want to optimize for, right? So one might be we want to allocate demand in such a way that</a> <a data-t="2208">we try to maximize and equalize the amount of excess supply that we have in any of the three</a> <a data-t="2216">zones, right? So that's perhaps one policy objective. Another policy objective might be</a> <a data-t="2223">we want to push demand to zones where we can get the most cost-effective supply,</a> <a data-t="2230">right? So, and there could be others as well, right? There are other heuristics, other policy</a> <a data-t="2235">decisions or policies that we might want to consider when we're trying to allocate this</a> <a data-t="2242">discretionary demand.</a></p>
