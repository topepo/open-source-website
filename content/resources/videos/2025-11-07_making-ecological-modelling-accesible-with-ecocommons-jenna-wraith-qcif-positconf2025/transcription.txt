<p><a data-t="88">I was able to access the data but then had to deliver that same USB stick to the department of e-research.</a> <a data-t="95">When I delivered it, we had a good laugh.</a> <a data-t="95">Here we were in the 21st century solving some biodiversity problems using snail mail.</a> <a data-t="100">But it was also a light bulb moment for me.</a> <a data-t="106">That little USB stick ended up being a really big turning point.</a> <a data-t="106">It landed me a new role piloting a platform across all states and territories in Australia</a> <a data-t="110">and that work eventually led to a five million dollar investment from the Australian Research Data Commons and the Australian government to what's now EcoCommons.</a></p>

<h3>What EcoCommons is</h3>

<p><a data-t="144">So the question then became, what if we could build something better?</a> <a data-t="144">What if there was one place where data, tools and compute power could all come together and anyone could use it?</a> <a data-t="149">And that's what EcoCommons is.</a> <a data-t="155">EcoCommons is a free, open source, cloud-based platform to remove barriers to ecological modelling.</a> <a data-t="160">It's built using Python with all our models and algorithms developed on R packages.</a> <a data-t="165">It brings together trusted biodiversity and environmental data sets,</a> <a data-t="166">integrates tools for analysis and modelling and makes them available in two key ways.</a></p>

<p><a data-t="171">Firstly as point and click workflows for people who aren't coders and also as R and Python notebooks for those who really love the code.</a> <a data-t="178">This means a student, a government scientist, a conservation NGO can all access the same data sets, the same models and get reproducible results.</a> <a data-t="191">This really helps to level the playing field.</a></p>

<h3>Key features of the platform</h3>

<p><a data-t="195">So let's walk through some of the key features.</a> <a data-t="195">First, EcoCommons brings together thousands of data sets in one place and they're not just the raw files, they've been curated and standardised so they're ready to use, which alone saves weeks of work.</a> <a data-t="212">Secondly, we make data wrangling easier.</a> <a data-t="212">We've built tools to harmonise formats, manage metadata so you don't have to spend your time cleaning data.</a> <a data-t="222">Thirdly, we've embedded trusted models and algorithms and these are the ones that are really widely used and validated by the scientific community.</a></p>

<p><a data-t="229">For our advanced users, we provide Jupyter notebooks so you can customise workflows or run experiments at scale.</a> <a data-t="234">And all of this runs in the cloud so you're not just using your own machine.</a> <a data-t="238">I've seen many a laptop start to crash just by running one species distribution model and in EcoCommons, we can run millions.</a> <a data-t="251">So finally, everything follows fair data principles.</a> <a data-t="251">Our data and workflows are findable, accessible, interoperable and reusable and this ensures that results are high quality, transparent and shareable.</a></p>

<p><a data-t="302">The next step is to select your species occurrence records.</a> <a data-t="307">These are simply records where we know koalas already live here in Australia.</a> <a data-t="315">And we have access to different biodiversity data repositories through APIs so a user can simply search the species and then visualise here on the map.</a> <a data-t="320">The next step is to select absence data.</a> <a data-t="328">Absence data simply is where we know the koalas don't occur and we run this in the back end of the system using different methods depending on user needs and this really just helps improve the model.</a></p>

<p><a data-t="341">The next step is to select the climate and environmental data.</a> <a data-t="347">EcoCommons has access to thousands of data sets all in one place</a> <a data-t="351">and today we're going to use a global climate data set.</a> <a data-t="356">Here we select the different layers within the data set that we want to use in our model and we're selecting a few that we know are important for koalas.</a> <a data-t="365">You can visualise this here on the map.</a> <a data-t="368">We can see Australia and we can zoom out to see what it looks like across the globe.</a> <a data-t="371">The next step is to select the study extent.</a> <a data-t="376">We're going to use a global extent for this model today.</a></p>

<p><a data-t="378">Next we want to select our algorithms.</a> <a data-t="383">We have access to around 20 different algorithms that are all built on R packages.</a> <a data-t="391">A user has the option to change all of the settings they need as if they were actually running the R package themselves.</a> <a data-t="394">The next step we hit start experiment.</a> <a data-t="398">We go grab a coffee and wait for our results.</a> <a data-t="399">Now we can see our job submitted and then we can view our results here.</a></p>

<h3>Why this work matters</h3>

<p><a data-t="488">Which brings me to my next point, why this work really matters.</a> <a data-t="492">Globally, biodiversity is unfortunately declining at a pace that we've never seen before.</a> <a data-t="497">Here in Australia, we have one of the highest rates of mammal extinction in the world.</a> <a data-t="502">We have climate change and we have invasive species and habitat loss reshaping ecosystems faster than we can keep up.</a> <a data-t="513">To respond effectively, we need trusted models and data.</a> <a data-t="513">We need to know where species are now, how they're likely to shift in the future, and what management actions will make the biggest difference.</a></p>

<p><a data-t="522">And that's the role that EcoCommons plays, turning complex data into the evidence and data-driven decisions that we need.</a> <a data-t="533">But getting to this point really wasn't that simple.</a> <a data-t="533">Before EcoCommons, the barriers made this kind of modelling almost impossible.</a></p>

<p><a data-t="541">Data was scattered across institutions and often in inconsistent formats.</a> <a data-t="546">Sometimes only accessible if you knew the right person.</a> <a data-t="549">Running models required high-performance compute, and many ecologists simply don't have access to that, so work becomes stalled.</a> <a data-t="560">Even if you had compute, you needed to have advanced coding skills, which can exclude some really brilliant practitioners who just hadn't been trained in programming.</a> <a data-t="571">And on top of that, you need modelling expertise.</a> <a data-t="571">And even then, the workflows are often black boxes.</a> <a data-t="575">Two people could run what looked to be the same model and end up with really different results.</a> <a data-t="582">So all of this meant that biodiversity modelling could be slow, fragmented, and often inaccessible.</a></p>

<h3>How EcoCommons breaks down barriers</h3>

<p><a data-t="588">EcoCommons solves this first barrier with accessible data.</a> <a data-t="588">We integrate trusted biodiversity data and environmental data sets into the platform.</a> <a data-t="594">They're standardised and curated, so they're ready to use.</a> <a data-t="599">No more USBs, no more hunting down files across multiple institutions.</a> <a data-t="604">This makes modelling faster, but also makes it fairer.</a> <a data-t="609">Everyone works from the same reliable data foundation.</a></p>

<p><a data-t="610">EcoCommons runs in the cloud, so a lot of that heavy lifting happens behind the scenes.</a> <a data-t="615">You don't need to own or rent a supercomputer.</a> <a data-t="620">Whether you're running a small student project with a few models, or a government project running millions of models, the platform scales to fit.</a> <a data-t="625">This opens the doors for non-government organisations, non-for-profits, smaller research groups, and anyone who doesn't have expensive infrastructure.</a></p>

<p><a data-t="635">And skills are another big barrier.</a> <a data-t="640">EcoCommons provides point-and-click workflows for people new to modelling.</a> <a data-t="640">They can select data sets, choose algorithms, and run analyses through an intuitive interface.</a> <a data-t="650">At the same time, our coders can integrate R and Python directly, so they're not limited by the interface.</a> <a data-t="659">And we do this through the Jupyter Notebook library.</a> <a data-t="660">We also run a lot of training and provide documentation, helping people build their confidence and skills over time.</a> <a data-t="670">And the result here is instead of locking people out of modelling, we're really helping to bring them in.</a></p>

<p><a data-t="676">And finally, methods.</a> <a data-t="676">Too often, ecological modelling, or modelling as a whole, has been inconsistent or opaque.</a> <a data-t="681">You don't always know what steps have been taken, and you can't always reproduce someone else's results.</a> <a data-t="686">So, we've made these workflows open, transparent, and reproducible.</a> <a data-t="692">Every step is documented, and the methods can be shared and modifiable.</a> <a data-t="698">This transparency builds trust and also helps accelerate science, because the results can be repeated, compared, and improved.</a></p>

<p><a data-t="702">So, what does all this add up to?</a> <a data-t="707">EcoCommons is helping conservation groups plan on-ground actions, it's helping governments assess risk and make policy decisions, it's supporting NGOs and researchers who want to test new ideas and scale up models.</a> <a data-t="718">And beyond individual projects, it's building a community of practice.</a> <a data-t="723">People aren't just using EcoCommons in isolation, they're sharing their workflows, contributing data sets, and really learning from one another.</a></p>

<h3>Lessons learned and community</h3>

<p><a data-t="788">We've really learned a lot in building EcoCommons.</a> <a data-t="793">Scaling our workflows to run millions of models was definitely technically challenging.</a> <a data-t="798">We had to rethink our performance, our reproducibility, and how to handle large data sets.</a> <a data-t="804">And we're consistently improving on this as well.</a> <a data-t="809">We've had to balance the needs of really different user groups, so we've had advanced coders who want to have full flexibility, and newcomers to the industry who just want something intuitive.</a> <a data-t="824">And we've really learned the importance of community feedback.</a> <a data-t="835">Many of our improvements come directly from what users are telling us what worked, what didn't, and what they needed next.</a></p>
