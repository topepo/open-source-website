<h3>Using the litmus test as a design tool</h3>

<p><a data-t="497">So the real value of this little thing, though, I think, is that you can flip it and it can be a design tool.</a> <a data-t="503">So if you're working on something that you want to imbue with more magic, like, I think you just have to, like, shove as much stuff into that last section as possible, right?</a> <a data-t="511">But you need some ideas of, like, what you're going to do.</a></p>

<p><a data-t="549">So those are a bit abstract.</a> <a data-t="551">Like let's think about some concrete ways you could, like, implement these.</a> <a data-t="554">So saving typing, like, here's the continuum.</a> <a data-t="557">You could, like, do the pipeable functions with the nonstandard evaluation.</a> <a data-t="560">You could use the autocomplete friendly names to, like, you know, only type part of it.</a> <a data-t="564">Or you could take the typing away completely.</a> <a data-t="566">And that's what I did with data pasta.</a> <a data-t="568">I really tried to bank heavily on that for magic and drill into, like, minimizing the keystrokes to the absolute.</a></p>

<p><a data-t="579">And saving reading, well, we've talked about the intuitive names.</a> <a data-t="583">Conforming becomes really important when you are making a model or a method, right?</a> <a data-t="588">Because what your users fundamentally want to do with a model or a method is compare it to one they actually understand, yeah?</a> <a data-t="594">And to do that, it needs to have, like, it needs to look a certain way.</a> <a data-t="597">Like, it needs to take a data frame, use a formula interface and return an object that implements certain S3 methods.</a> <a data-t="604">And if it doesn't do that, like, you're just foisting a whole bunch of reading and learning onto your user to allow them to make the comparison that you know they want to make.</a> <a data-t="612">So don't do that.</a> <a data-t="613">Conform.</a></p>
