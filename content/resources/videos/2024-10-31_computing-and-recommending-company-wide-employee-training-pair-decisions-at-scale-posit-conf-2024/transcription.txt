<h3>A flexible data science framework</h3>

<p><a data-t="340">So how do we achieve this scalable outcome?</a> <a data-t="343">Well, we did it using data science, of course.</a> <a data-t="345">We used a flexible framework strategy in which data science is the key to the impactful AI</a> <a data-t="348">that we use to accelerate everything.</a> <a data-t="351">And so if one way of thinking about data science is starting at the end and working backwards,</a> <a data-t="356">particularly if you want to be impactful, and as we all are data scientists here, if</a> <a data-t="360">you don't want to waste your time.</a></p>

<p><a data-t="363">So one definition of data science is the use of computational approaches, automation,</a> <a data-t="367">AI, ML, to enable specific decisions towards desired outcomes.</a> <a data-t="374">And so what do I mean by an outcome?</a> <a data-t="376">Well, some sort of improvement in a context or area that you can only achieve by taking</a> <a data-t="381">an action, by expending resources.</a> <a data-t="384">But you only expend the resources that you decide to expend and not the ones that you</a> <a data-t="389">don't.</a> <a data-t="389">And you only make that decision once you have achieved sufficient confidence by reducing</a> <a data-t="393">the ambiguity surrounding all of your options.</a> <a data-t="396">And you only do that once you've gotten enough information about what likely cause-effect</a> <a data-t="401">relationships could exist.</a> <a data-t="403">And you only do that once you ask the right questions, attempt to eliminate those gaps</a> <a data-t="408">that can make it easier to understand potential causality.</a></p>

<p><a data-t="412">So with that in mind, data science and artificial intelligence have this software intersection</a> <a data-t="420">that define the relationship between the two of them.</a> <a data-t="423">So whereas data science is using data to figure out which decisions to make, artificial intelligence</a> <a data-t="428">is simulating autonomous, basically execution of those decisions.</a> <a data-t="433">So simulating human thinking, reasoning, and behavior.</a> <a data-t="437">So you can have artificial intelligence systems in which you directly tell the computer what</a> <a data-t="442">the decision-making rules are, but you can have other artificial intelligence systems</a> <a data-t="446">that use machine learning in which the computer figures out the decision-making rules by understanding</a> <a data-t="451">the data itself.</a> <a data-t="453">And then deep learning, of course, is making computer use multiple layers of thought in</a> <a data-t="459">order to figure out what the decision-making rules should be.</a></p>

<p><a data-t="463">So in this data science framework, you basically start at the end and you define what that</a> <a data-t="468">outcome is.</a> <a data-t="469">And then you work towards identifying the obstacles towards that outcome and then eliminating</a> <a data-t="474">those obstacles.</a> <a data-t="476">And that's one of the ways that you can maximize your efficiency and also have staying power</a> <a data-t="480">in terms of the impact of what you do.</a></p>

<p><a data-t="483">But basically this means that data science is essentially decision science, which really</a> <a data-t="489">if you think about it is just minimizing ambiguity.</a> <a data-t="492">So if you just do a little mini quick exercise in your head right now, and you think about</a> <a data-t="497">if you were to try to pick between the two roads that are shown here, so just pick the</a> <a data-t="502">right one, like what would you pick?</a> <a data-t="505">You can't because you don't really know anything about either of the two roads.</a> <a data-t="509">There's too much ambiguity.</a> <a data-t="510">And so none of us are confident enough to pick one road over the other.</a> <a data-t="516">We don't have enough justification to decide on either.</a></p>

<p><a data-t="519">Now if the decision threshold were pretty low, so maybe some of you did pick, and that</a> <a data-t="523">might be because you're thinking, well, it doesn't really matter, nothing bad is going</a> <a data-t="526">to happen to me.</a> <a data-t="526">But if it's a higher risk, if something goes sideways and that it impacts something you</a> <a data-t="530">really care about, then you're like, whoa, whoa, whoa, I actually need more information.</a> <a data-t="535">So if you find out, for example, that down one of the roads, there's some sort of danger,</a> <a data-t="540">well, at least now you've reduced the ambiguity about one of the roads and increased the confidence</a> <a data-t="546">that you may have about what you should do.</a> <a data-t="548">So now you have a little bit more confidence.</a> <a data-t="550">And if there's only two roads, one of them is definitely dangerous, well, maybe you could</a> <a data-t="554">go down the other road because now you have at least enough knowledge to decide on what</a> <a data-t="558">you want to do.</a> <a data-t="559">And so that takes you from being stuck, where you had that analysis paralysis in the absence</a> <a data-t="563">of sufficient data, to being unstuck.</a> <a data-t="567">And so that's the goal of data science, to help to reduce those obstacles so you can</a> <a data-t="571">go from one to the other.</a></p>

<h3>How the decision recommendation engine works</h3>

<p><a data-t="716">So how did we apply this to scale mentor matching to Dr. Strange, our problem?</a> <a data-t="725">The real question, actually, was what outcome did Magnetron AI need to be able to prioritize</a> <a data-t="730">decision recommendations towards?</a> <a data-t="732">And that outcome really is that employees in need of specific skills are matched with</a> <a data-t="737">others who are best suited to help them develop those skills.</a> <a data-t="741">So if we think about it, it's really that the decision is to match each mentee with</a> <a data-t="745">the mentor that can provide the best help.</a></p>

<p><a data-t="786">So then the question for you all that you might be wondering is, how does the decision</a> <a data-t="791">recommendation engine work?</a> <a data-t="793">Well, fundamentally, it's two steps, filter and then sort.</a> <a data-t="797">And the filtering is using a lot of the approaches that I mentioned earlier, and the sorting</a> <a data-t="801">is using the other approaches that I mentioned earlier.</a> <a data-t="805">And there's two parts to the filtering and one part to the sorting.</a></p>

<p><a data-t="808">So the first part of the filtering is cohort eligibility inference.</a> <a data-t="812">And you can think of that as like a bouncer at the front door of a club.</a> <a data-t="816">The second one is compatibility or match compatibility inference.</a> <a data-t="821">So you can think of that as the bodyguard of a person sitting at a table inside of the</a> <a data-t="826">club.</a> <a data-t="827">So one is, are you allowed to even get into the cohort?</a> <a data-t="828">And the other is, are you allowed to be a match?</a></p>

<p><a data-t="831">So the way cohort eligibility inference works is identifying eligible users, which are users</a> <a data-t="837">that have any inclusion criteria minus users that have any exclusion criteria.</a> <a data-t="842">And so the inclusion exclusion criteria membership is defined on the basis of username information,</a> <a data-t="847">username lists, and reporting structure hierarchy.</a> <a data-t="852">So these are taken into consideration when a cohort is designed by an administrator.</a></p>

<p><a data-t="858">In terms of the pair compatibility or match compatibility inference, this is asymmetric</a> <a data-t="863">because you can think of it in terms of in the dating world outside of corporate America,</a> <a data-t="868">there could be people who are interested in others.</a> <a data-t="871">But just because one person is interested in 10 people doesn't mean that each of those</a> <a data-t="875">10 people is interested in the one person.</a> <a data-t="877">So there's asymmetry.</a> <a data-t="879">And so this is what is taken into account with R and Shiny here.</a></p>

<p><a data-t="885">So the asymmetric compatible applicants are applicants with any inclusion criteria minus</a> <a data-t="892">applicants with any exclusion criteria.</a> <a data-t="894">And that's from the perspective of each applicant.</a> <a data-t="897">So then if you take the perspective from each applicant and you have your asymmetric list,</a> <a data-t="901">if you integrate these two lists, now you have your symmetric pair of eligible compatible</a> <a data-t="908">people.</a></p>

<p><a data-t="910">And so this is computed in parallel and asynchronously via APIs, as was mentioned a bit earlier.</a> <a data-t="918">And so it's using the same categories as before for the eligibility for applying to the cohort,</a> <a data-t="926">but then additional things like years of experience and personal reporting line are allowed to</a> <a data-t="930">be submitted by the applicants.</a></p>

<p><a data-t="935">And then the ranking part.</a> <a data-t="943">So what's done here is a space symmetric calculation of skill similarity and activity distance</a> <a data-t="948">between all pairs that have been applied to the cohort.</a> <a data-t="952">And then the next step is to do an asymmetric calculation of the rank.</a> <a data-t="956">So from each person's perspective, how much are the other people good matches for each</a> <a data-t="959">member of a given pair?</a> <a data-t="963">And then this is all brought together because then you calculate the average asymmetric</a> <a data-t="968">rank and then rank that asymmetric rank that has been averaged.</a> <a data-t="976">So now you have a global rank.</a></p>
