<h3>ggplot2's philosophy and the grammar of graphics</h3>

<p><a data-t="135">So what do I mean by ggplot2's philosophy?</a> <a data-t="140">ggplot2 is a data visualization package</a> <a data-t="140">that embodies a whole theory of visualization called the layered grammar</a> <a data-t="145">of graphics, which is based on Leland Wilkinson's grammar of graphics, but it</a> <a data-t="150">has some specific things that are, you know, tailored to its implementation in</a> <a data-t="155">R.</a> <a data-t="161">Lots of visualization software gives you a bunch of different charts to choose</a> <a data-t="161">from.</a> <a data-t="166">So you have your pie charts and your scatter plots and your line charts</a> <a data-t="166">and your bar charts, and you take the end result that you want to achieve and sort</a> <a data-t="172">of work backwards to fit your data into it.</a> <a data-t="179">ggplot2 is not like this.</a> <a data-t="179">ggplot2 is</a> <a data-t="179">a more data-first approach to building visualizations, starting with, you know,</a> <a data-t="186">principles of essentially a grammar for how to convey data in visual form.</a></p>

<p><a data-t="194">So</a> <a data-t="194">when we speak or write, we don't normally start with a sentence structure that we</a> <a data-t="199">want to use and then fit our message into that structure.</a> <a data-t="206">Instead, we start with</a> <a data-t="206">some idea and we use our knowledge of grammar and syntax and vocabulary to</a> <a data-t="211">craft the message we want to express that idea.</a> <a data-t="216">And that is very analogous to</a> <a data-t="216">how ggplot2 works.</a> <a data-t="223">ggplot2 lets us express our data in visual forms that might be</a> <a data-t="223">totally novel and not fit into common, you know, bar charts and line charts and</a> <a data-t="228">common types of visualizations.</a> <a data-t="235">This gives us a really flexible tool and a</a> <a data-t="235">really powerful tool that we can use to create the intentional art like you see</a> <a data-t="241">on the slide, which was built with Daniel Navarro's flametree package, which</a> <a data-t="245">itself uses ggplot2.</a> <a data-t="251">So having this flexibility gives us a lot of power and</a> <a data-t="251">control.</a> <a data-t="257">And it also opens up room for, you know, mistakes and for things to not</a> <a data-t="257">always work out the way we had in mind the first time around.</a></p>

<h3>Mapping mishaps</h3>

<p><a data-t="263">There are certain</a> <a data-t="263">foibles and mistakes that come up a lot with ggplot2 that demonstrate pretty</a> <a data-t="268">common misunderstandings of how ggplot2, the package, works.</a> <a data-t="273">And that's what I'm</a> <a data-t="273">going to talk about today.</a> <a data-t="277">The first one that we're going to talk about are</a> <a data-t="277">mapping mishaps.</a> <a data-t="282">These don't have to do with geographical maps.</a> <a data-t="282">What we're</a> <a data-t="282">talking about here is mapping visual properties in the plot to variables in</a> <a data-t="287">our data.</a> <a data-t="294">In the parlance of ggplot2, these mappings are called aesthetics, and</a> <a data-t="294">they're really at the heart of ggplot2's grammar of graphics.</a></p>

<p><a data-t="300">So let's take this</a> <a data-t="300">plot.</a> <a data-t="305">This is a plot of flipper and bill length for different species of penguins</a> <a data-t="305">in Antarctica.</a> <a data-t="310">And in this plot, we have several different aesthetics.</a> <a data-t="310">We have</a> <a data-t="310">position along the x-axis, which corresponds to flipper length.</a> <a data-t="315">We have</a> <a data-t="315">position along the y-axis, which corresponds to bill length.</a> <a data-t="320">And then we</a> <a data-t="320">have both color and shape, which correspond to the penguin species.</a> <a data-t="326">So one</a> <a data-t="326">thing that we might want to do with a plot like this is label these different</a> <a data-t="330">clusters.</a> <a data-t="334">We have sort of well-isolated clusters of these data points, and maybe</a> <a data-t="334">instead of the legend at the bottom, we want to label those clusters with which</a> <a data-t="339">species they correspond to.</a></p>

<p><a data-t="345">And what someone will often do to achieve this is</a> <a data-t="345">add a layer to the plot where they map the text that they want to appear to the</a> <a data-t="351">x and y positions where they want to place it.</a> <a data-t="358">So here we have x equals 195</a> <a data-t="358">and y equals 55.</a> <a data-t="365">We're placing the label chinstrap.</a> <a data-t="365">And you can see that that</a> <a data-t="365">does work in that it places text on the plot.</a> <a data-t="371">If we look closely, though, it</a> <a data-t="371">doesn't actually look that good.</a> <a data-t="376">So it's pretty subtle in this example, but the</a> <a data-t="377">quality of the text is a little bit pixelated.</a> <a data-t="382">It doesn't look as good as the</a> <a data-t="382">points and lines that are around it.</a> <a data-t="388">And if we were running this code live, it</a> <a data-t="388">would actually be a little bit slow to plot.</a></p>

<p><a data-t="394">Now, like I said, it's a pretty</a> <a data-t="394">subtle change or subtle issue, but we actually get a lot of ggplot2 issues</a> <a data-t="402">opened because of this specific thing where people run into their, you know,</a> <a data-t="409">with their own plots, that the plot is really slow, that the text looks really</a> <a data-t="413">bad.</a> <a data-t="419">And they come to us to say, is this a bug in ggplot2? Why does it look so</a> <a data-t="419">bad?</a> <a data-t="426">We can illustrate what's going on here another way by using the ggrepel</a> <a data-t="426">package, which will ensure that text on the plot does not overlap.</a> <a data-t="431">And when we</a> <a data-t="431">do that, we can see that we actually haven't just put a single label on the</a> <a data-t="435">plot, we've put the whole this whole giant starburst of the same text</a> <a data-t="440">repeated hundreds of times.</a></p>

<p><a data-t="447">What's going on here is that we've actually written</a> <a data-t="447">our label the same number of times as there are rows in our data set.</a> <a data-t="453">In this</a> <a data-t="453">case, 344 times.</a> <a data-t="461">By putting the text label that we want inside this AES</a> <a data-t="461">function, where AES is stands for aesthetics, we've declared a mapping</a> <a data-t="467">between our data and the label, causing that label to repeat as many times as</a> <a data-t="473">there are data points.</a> <a data-t="480">So that's not great.</a> <a data-t="480">And we can avoid this by actually</a> <a data-t="480">using a different function called annotate, which is designed to put labels</a> <a data-t="484">on plots just once.</a> <a data-t="488">So it's, it's designed to do exactly what we want to do</a> <a data-t="488">just stick that label on there once not map it to the underlying data.</a> <a data-t="493">So we tell</a> <a data-t="493">annotate that we want to place a text annotation, again, give it the position</a> <a data-t="498">and the label.</a> <a data-t="505">And it places that text on the plot for us.</a> <a data-t="505">And when we zoom in, it</a> <a data-t="505">looks much better, it would be much faster to plot as well.</a></p>

<h3>Scale snafus</h3>

<p><a data-t="511">So that was</a> <a data-t="511">mapping mishaps, where we misapplied these aesthetic mappings between visual</a> <a data-t="516">elements of the plot and our data.</a> <a data-t="523">The next common mix up is what I'll call</a> <a data-t="523">scale snafus.</a> <a data-t="530">In ggplot2, scales convert values from the data space to the</a> <a data-t="530">aesthetic space.</a> <a data-t="534">So in with our aesthetics that we just talked about, we</a> <a data-t="534">declared a mapping between visual elements of the plot and values in the</a> <a data-t="538">data, you know, shape to species, for instance, the scale controls how those</a> <a data-t="544">data points get translated into what we see on the plot.</a></p>

<p><a data-t="550">We're going to talk</a> <a data-t="550">about coordinate systems too, which are what draw the axes and panels in the</a> <a data-t="555">plot and generally kind of set up the 2d environment of the plot.</a> <a data-t="560">So most plots,</a> <a data-t="560">including the one that we're using for our example here, use a simple Cartesian</a> <a data-t="565">coordinate system.</a> <a data-t="570">But other plots can use, for example, like a geographical</a> <a data-t="570">coordinate system or polar coordinate system in the case of pie charts.</a></p>

<p><a data-t="578">So</a> <a data-t="578">let's return to our flipper and bill length visualization.</a> <a data-t="584">And this time, in</a> <a data-t="584">addition to the data points, I've added the smoothed conditional means for each</a> <a data-t="588">species.</a> <a data-t="592">Now let's say that we want to zoom in this plot to highlight one</a> <a data-t="592">species and sort of focus our attention in on just one of these areas of</a> <a data-t="598">the plot.</a> <a data-t="604">There are two ways that we can accomplish this.</a> <a data-t="604">One is by setting limits</a> <a data-t="604">on the scale.</a> <a data-t="610">Here we're setting the x minimum to 200 and the x maximum we just</a> <a data-t="610">let be whatever it normally was.</a> <a data-t="615">And then the other way that we can zoom in this</a> <a data-t="615">plot is by setting limits on the coordinate system.</a> <a data-t="620">Again, setting the x</a> <a data-t="620">minimum to 200 and letting the x max be whatever it is.</a></p>

<p><a data-t="627">When we limit the scale,</a> <a data-t="627">this is the plot that we end up with.</a> <a data-t="634">But when we limit the coordinate system, it</a> <a data-t="634">actually looks quite different.</a> <a data-t="642">The data looks the same, the individual points are</a> <a data-t="642">the same, but the statistical summary looks drastically different.</a> <a data-t="647">And that is</a> <a data-t="647">because when we set the scale limits, any data that's outside of those limits will</a> <a data-t="654">by default be converted to NAs.</a> <a data-t="662">And that this scale transformation happens before</a> <a data-t="662">any statistical summaries.</a> <a data-t="667">So before we calculate what those smooth lines and</a> <a data-t="667">error bands look like, that data has already been converted to NAs.</a> <a data-t="674">The</a> <a data-t="674">coordinate system, on the other hand, when you set limits there, it really just</a> <a data-t="679">zooms in the plot.</a> <a data-t="685">So all the other data that was originally present is still</a> <a data-t="685">getting used, even though you can't see it on the plot.</a></p>

<p><a data-t="692">So in the first plot</a> <a data-t="692">where we set the limits on the scale, statistical summaries look different</a> <a data-t="696">because they're based on just the few data points that are visible for the</a> <a data-t="700">species in red and purple.</a> <a data-t="705">In the second plot though, you can imagine that all the</a> <a data-t="705">rest of the data is still there and it's still getting used and those smooth</a> <a data-t="709">lines are extending out towards where the rest of the data is.</a> <a data-t="715">So this is</a> <a data-t="715">clearly a case where the different ways of setting these limits have</a> <a data-t="720">major implications for the statistical summaries that we're actually displaying</a> <a data-t="724">on the plot.</a> <a data-t="732">And it's important to know this.</a> <a data-t="732">So ggplot2 does alert us to this</a> <a data-t="732">fact and if we return to the code that generated these plots, you can actually</a> <a data-t="737">see that the former version will warn you about what's going on.</a> <a data-t="741">It says</a> <a data-t="741">removed 192 rows containing non-finite values.</a> <a data-t="748">So we've gotten this warning</a> <a data-t="748">message, but it is at times easy to gloss over these messages or ignore them and</a> <a data-t="755">so it's useful to know how scales and coordinates handle data that's out of</a> <a data-t="760">bounds.</a></p>

<h3>Theme threats</h3>

<p><a data-t="765">The last type of ggplot2 mishap that I'm going to talk about are issues</a> <a data-t="765">with themes, theme threats.</a> <a data-t="772">Now ggplot2's theme system is not strictly part of</a> <a data-t="772">the grammar of graphics because it really controls all of the non-data</a> <a data-t="776">aspects of the plot, whereas the grammar mostly deals with how we translate</a> <a data-t="782">our data into visual form.</a> <a data-t="786">But the theme system is really important because it is</a> <a data-t="786">how you customize a plot to look exactly how you want it, all the way up from the</a> <a data-t="792">text font and the background color down to small details like the length of the</a> <a data-t="798">ticks on the axes, for example.</a> <a data-t="804">There are dozens and dozens of things that you can</a> <a data-t="804">customize about plots in ggplot2, but it would be really cumbersome to have to</a> <a data-t="810">specify every single one of those every single time.</a> <a data-t="816">So ggplot2 comes with a</a> <a data-t="816">set of default themes that are built in that you can use and a system for</a> <a data-t="821">theme elements to inherit their values and how they look from other</a> <a data-t="828">aspects of the theme.</a></p>

<p><a data-t="836">So let's look at an example.</a> <a data-t="836">In our theme hierarchy of</a> <a data-t="836">different elements that you can customize, text is one of the ones that's</a> <a data-t="839">way up at the top.</a> <a data-t="845">In theory, this applies to all the text elements of the plot.</a> <a data-t="845">So</a> <a data-t="845">if we look again at our penguin plot, this plot has a number of different text</a> <a data-t="850">elements.</a> <a data-t="858">It has title, it has axis titles and axis text, it has legend which</a> <a data-t="858">has some text on it as well.</a> <a data-t="865">So all of these are within the umbrella of text.</a> <a data-t="865">If</a> <a data-t="865">you wanted to, for example, set the font of all the text in this plot, you</a> <a data-t="871">would want to do it up at that text level rather than having to say I want</a> <a data-t="875">the title to be this font and the axis labels to be this font and so on.</a></p>

<p><a data-t="881">At the</a> <a data-t="881">next level of the hierarchy is, in this example, is axis text.</a> <a data-t="886">So we went from</a> <a data-t="886">text down to just the text that labels the axes.</a> <a data-t="892">That's everything that is</a> <a data-t="892">outlined in blue on this slide, all the axis text elements.</a> <a data-t="898">And in this</a> <a data-t="898">plot, we have axes on the left, right, and bottom.</a> <a data-t="906">At the next level of the theme</a> <a data-t="906">hierarchy, there's axis text X and axis text Y.</a> <a data-t="913">So this allows us to, for example,</a> <a data-t="913">customize the Y axis while leaving the X axis alone and vice versa.</a> <a data-t="920">And there's</a> <a data-t="920">actually one more level, axis text X bottom, axis text X top, axis text Y left</a> <a data-t="928">and axis text Y right.</a> <a data-t="935">Try saying that 10 times fast.</a> <a data-t="935">Where we can customize</a> <a data-t="935">axes for potentially on one side of the plot but not the other.</a></p>

<p><a data-t="959">So let's return to our plot.</a> <a data-t="959">So again, we're looking at these</a> <a data-t="959">penguin measurements.</a> <a data-t="964">And this time I've added a second axis on the right-hand</a> <a data-t="964">side of the plot that just duplicates the axis on the left-hand side.</a> <a data-t="970">So they're</a> <a data-t="970">exactly the same.</a> <a data-t="975">And now let's say that we wanted to customize the labels</a> <a data-t="975">on this axis, on our two Y axes, so that they're maybe rotated 90</a> <a data-t="981">degrees and right justified.</a> <a data-t="986">So we might write something like this to customize</a> <a data-t="986">both of these axes, where we're setting axis text Y to angle equals 90,</a> <a data-t="995">horizontal justification equals 1.</a> <a data-t="999">And since we want to apply this to both of</a> <a data-t="999">our Y axes, this seems like the logical way to do it.</a> <a data-t="1006">But when we look at the</a> <a data-t="1006">plot, something is not quite right.</a> <a data-t="1012">The rotation has taken effect, but the</a> <a data-t="1012">justification has not.</a> <a data-t="1019">So only on the left-hand side are our axis labels</a> <a data-t="1019">justified.</a> <a data-t="1026">On the right-hand side, they don't match.</a></p>

<p><a data-t="1026">So why would one</a> <a data-t="1026">of these changes take effect, but not the other one?</a> <a data-t="1031">And the reason for this has to</a> <a data-t="1031">do with the base theme that I'm using.</a> <a data-t="1036">So I built the theme for these plots on</a> <a data-t="1036">the default ggplot theme, theme gray.</a> <a data-t="1042">And if we printed out the whole theme, it</a> <a data-t="1042">would be like super long, so we're not going to look at all of it.</a> <a data-t="1046">But I've</a> <a data-t="1046">just put on the slide the parts of it that have to do with axis text.</a> <a data-t="1052">You</a> <a data-t="1052">can see that theme gray sets the margins and the justification of axis text Y.</a> <a data-t="1058">And</a> <a data-t="1058">then it actually adjusts those details for axis text Y right.</a> <a data-t="1066">And in</a> <a data-t="1066">particular, theme gray is setting the justification of text in axis text Y</a> <a data-t="1070">right to 0, corresponding to a left-justified text.</a></p>

<p><a data-t="1080">This brings us to the</a> <a data-t="1080">crucial piece of the theme system, which is that when you're customizing theme</a> <a data-t="1084">elements, the most specific element wins.</a> <a data-t="1090">If the theme that you're using sets a</a> <a data-t="1090">specific element, like the justification of axis text Y right, then if you want to</a> <a data-t="1095">change it, you need to edit that very specific element and not any of the ones</a> <a data-t="1099">above it.</a> <a data-t="1105">So in our case, theme gray sets the justification, but not the angle of</a> <a data-t="1105">axis text Y right.</a> <a data-t="1112">So when we change axis text Y, the justification that we</a> <a data-t="1112">provided gets overridden by the theme that we're using.</a> <a data-t="1117">This is something that</a> <a data-t="1117">has thrown off so many people.</a> <a data-t="1122">It's thrown off me, it's thrown off other</a> <a data-t="1122">contributors to ggplot, it comes up all of the time, because it is a subtle and</a> <a data-t="1126">slightly confusing aspect of how this theme inheritance works.</a> <a data-t="1132">But when you</a> <a data-t="1132">think about the fact that it's working by specificity, and not by the order in</a> <a data-t="1137">which you provide, you know, customizations to the theme, it does make</a> <a data-t="1144">sense.</a></p>

<blockquote><a data-t="1080">This brings us to the</a> <a data-t="1080">crucial piece of the theme system, which is that when you're customizing theme</a> <a data-t="1084">elements, the most specific element wins.</a></blockquote>

<p><a data-t="1150">So to get a working plot, we would need to edit axis text Y right like</a> <a data-t="1150">this.</a> <a data-t="1155">So instead of setting our customizations to the axis text Y</a> <a data-t="1155">element, where they'll be overridden by the theme that we're using, we customize</a> <a data-t="1161">axis text Y left and axis text Y right.</a> <a data-t="1166">And that makes sure that our</a> <a data-t="1167">customizations get applied the way we want them to.</a> <a data-t="1173">And now everything looks</a> <a data-t="1173">good on the plot.</a> <a data-t="1178">These these two axes are looking exactly the same on the left</a> <a data-t="1178">and the right.</a></p>

<h3>Recap and closing thoughts</h3>
