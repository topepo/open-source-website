<h3>The request-response cycle</h3>

<p><a data-t="291">Now let's talk about the request response cycle.</a> <a data-t="295">This is the core idea of how Ambiorix works.</a> <a data-t="298">In fact, if you understand this, you'll be 50% there.</a> <a data-t="303">And it's in fact the core idea of how every interaction on the web works, from loading a page to submitting a form.</a></p>

<p><a data-t="310">This cycle is initiated by the client.</a> <a data-t="313">The client sends a request to the server.</a> <a data-t="317">The client in this case can basically be a web browser or another application sending a request to yourself.</a> <a data-t="326">The request is basically a message detailing what the client wants.</a> <a data-t="331">For example, give me the homepage.</a> <a data-t="334">Now the server gets the request, processes it and gives back a response.</a> <a data-t="342">The server in this case is your Ambiorix application.</a> <a data-t="347">Ambiorix creates a web server.</a> <a data-t="349">That is the server that we are talking about.</a></p>

<p><a data-t="352">And the response that the server gives back could be the requested data, an HTML page or an action confirmation.</a> <a data-t="360">And this cycle keeps going on and on and on.</a> <a data-t="363">The client sends a request to the server and the server gives back a response.</a></p>
