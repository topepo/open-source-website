<p><a href="/software/quarto/">Quarto</a><a data-t="0"> Dashboards is a new feature that is coming to Quarto in the 1.4 release, and it's</a> <a data-t="10">something that we really decided to add on the heels of the most recent Posit conference,</a> <a data-t="16">and so it wasn't originally on the plan for 1.4, but we decided to take a little bit of</a> <a data-t="20">extra time and see if we could squeeze it in.</a></p>

<p><a data-t="23">Obviously the goal of Quarto Dashboards is pretty simple, make it easy to build compelling</a> <a data-t="29">interactive dashboards using R, Python, Julia, Observable, and very much sort of heritage</a> <a data-t="37">of Flex Dashboard, so very much sort of bring Flex Dashboard and that type of experience</a> <a data-t="42">into Quarto, incorporate everything we've learned, everything the company has built,</a> <a data-t="47">and try to bring that together into a really great experience for our users.</a></p>

<p><a data-t="53">Dashboards in Quarto are pretty straightforward.</a> <a data-t="56">They're basically composed of cards, so you can think about cards as just sort of the</a> <a data-t="60">units, the atomic units that you're arranging in a dashboard, and you arrange those cards</a> <a data-t="65">into rows and columns to lay them out, and then you can use things like pages, tab sets,</a> <a data-t="71">and sidebars to sort of incorporate more advanced layout, to incorporate controls and other</a> <a data-t="76">things that might need to appear in the dashboard.</a></p>

<p><a data-t="79">But with those simple building blocks, it's actually pretty easy to build some cool dashboards.</a></p>
