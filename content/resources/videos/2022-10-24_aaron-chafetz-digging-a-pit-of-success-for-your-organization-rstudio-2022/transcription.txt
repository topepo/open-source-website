<p><a data-t="26">My name is Aaron Chafetz.</a> <a data-t="32">I work for a federal agency called USAID as an economist, and I</a> <a data-t="32">work in the Office of HIV AIDS.</a> <a data-t="38">Our work in the Office of HIV AIDS feeds into a larger</a> <a data-t="38">interagency effort known as PEPFAR, the President's Emergency Plan for AIDS Relief.</a> <a data-t="44">PEPFAR was</a> <a data-t="44">started in 2004 with the goal of ending the global HIV epidemic.</a> <a data-t="52">And we work in interagency</a> <a data-t="52">fashion with a host of other agencies like CDC, Department of Defense, and Peace Corps.</a></p>

<p><a data-t="59">So one of the success factors, one of the factors that has made PEPFAR so successful</a> <a data-t="65">is the reliance and the push to collect measurement and evaluation data across the world.</a> <a data-t="71">So what</a> <a data-t="71">does that data look like?</a> <a data-t="73">We're collecting data on HIV status, treatment status, and</a> <a data-t="76">prevention indicators, upwards of maybe 30 different indicators collected at 30,000 health</a> <a data-t="84">facilities all across the globe.</a> <a data-t="90">This data also is coming in each quarter to us.</a> <a data-t="90">So as</a> <a data-t="90">you can imagine, this adds up very quickly, and we are swimming in data.</a></p>

<p><a data-t="96">What makes this</a> <a data-t="96">even more challenging is when you don't have the right tools for the job.</a> <a data-t="101">So when I started</a> <a data-t="101">in the Office of HIV-AIDS back in 2015, the primary means of doing analysis was Excel.</a> <a data-t="109">Excel is great, and there's, you know, clearly there are useful uses for it.</a> <a data-t="116">But when you're</a> <a data-t="116">working with data that come in so frequently and are so large, it's probably not the right</a> <a data-t="121">tool for the job.</a> <a data-t="127">And at that point, you kind of have to recognize that there are better</a> <a data-t="127">things out there, much like R.</a> <a data-t="133">And so that's when I started in 2016, started realizing</a> <a data-t="133">that there was a use case here for using R.</a> <a data-t="139">I had been using Stata as an economist, but</a> <a data-t="139">even Stata, Stats package, didn't quite cut it in terms of the scale of the data we were</a> <a data-t="144">working with.</a> <a data-t="151">And I got really excited about R and really spent a long time learning this</a> <a data-t="151">and tried then to convince other people in our organization that R was a great path forward</a> <a data-t="157">and we needed to learn it.</a></p>

<p><a data-t="164">Problem, though, you know, R has a steep learning curve, and</a> <a data-t="164">so trying to convince other people to learn R is much like telling them just go out and</a> <a data-t="169">run a marathon, right?</a> <a data-t="174">It's not easy to do, especially when you don't have a training</a> <a data-t="174">plan in place.</a> <a data-t="181">And so after a few unsuccessful years of proselytizing R, just talking about</a> <a data-t="181">it, what I realized our organization needed was a pit of success.</a> <a data-t="187">We needed something</a> <a data-t="187">that users didn't have to go it alone when they were learning new skill like R.</a></p>

<h3>What is a pit of success?</h3>

<p><a data-t="194">The problem, I think, in terms of this idea of success is we often frame it as one of</a> <a data-t="199">climbing a mountain, right?</a> <a data-t="205">We have to fight this uphill battle, and only when we have</a> <a data-t="205">summited this mountain after all of this work, we have achieved something.</a> <a data-t="211">The idea, though,</a> <a data-t="211">a pit of success is flipping that on its head.</a> <a data-t="217">It's saying why can't we create an infrastructure</a> <a data-t="217">so that in order to succeed, it's as easy as falling into that pit?</a> <a data-t="224">And in order to</a> <a data-t="224">do so, we kind of lay out the guide rails and the framework to make this easy for other</a> <a data-t="229">people to do.</a></p>

<blockquote><a data-t="217">It's saying why can't we create an infrastructure</a> <a data-t="217">so that in order to succeed, it's as easy as falling into that pit?</a></blockquote>

<p><a data-t="235">This is by no means a new concept or one that</a> <a data-t="235">I came up with.</a> <a data-t="242">Hadley was talking about it way back in 2016 in relation to the </a><a href="/software/tidyverse/">tidyverse</a><a data-t="242">.</a> <a data-t="247">And the tidyverse definitely was the primary pit of success for me in learning R, and I</a> <a data-t="247">think this framework also applies to an organization.</a> <a data-t="255">Problem is, if Hadley's talking about it,</a> <a data-t="255">why aren't we all already doing this already?</a> <a data-t="259">And I think there are definitely some clear</a> <a data-t="259">barriers in the way of doing this.</a> <a data-t="265">Obviously, it's resource intensive to set up this pit.</a> <a data-t="272">You need people to kind of shine that light to show others the path to move this forward.</a> <a data-t="276">And it's time intensive, both for those people who are leading the way and for new people</a> <a data-t="276">to come on and learn the skill.</a> <a data-t="281">And then lastly, as part of the government,</a> <a data-t="281">our hiring practices are just definitely not as nimble as the private sector, and our data</a> <a data-t="285">stacks are a little slower to innovate.</a></p>

<h3>Three ways to dig the pit</h3>

<p><a data-t="292">So I want to talk about three ways in which</a> <a data-t="292">we were able to dig this pit of success at our organization.</a> <a data-t="298">The first, the primary means</a> <a data-t="298">of doing so was developing packages to create that infrastructure, then creating buy-in</a> <a data-t="303">at multiple levels, and then lastly, fostering community of support for our users.</a></p>

<p><a data-t="310">So starting things off, I think the foundations of this pit of success were through packages.</a> <a data-t="317">Our work in PEPFAR, we, as I mentioned earlier, have lots and lots of data, measurement and</a> <a data-t="322">valuation data, that's coming in every quarter.</a> <a data-t="327">And this is a good means to automate something,</a> <a data-t="327">and R is a great tool for that.</a> <a data-t="332">So if we have this data that's similar and coming in, we</a> <a data-t="332">can start writing scripts that we can automate.</a> <a data-t="337">Oh, and those scripts, as we get more comfortable,</a> <a data-t="337">we can turn into functions.</a> <a data-t="342">So that those functions are useful to you, but they're also</a> <a data-t="342">can be useful to others.</a> <a data-t="348">And slowly, started developing packages.</a> <a data-t="348">Initially by myself,</a> <a data-t="348">but I saw that this is a means to help others enter the field and learn R.</a></p>

<p><a data-t="355">So as I became</a> <a data-t="355">more proficient in R, and we started bringing on more people who learned R, and we had more</a> <a data-t="360">expertise there, we were able to grow out this infrastructure to allow new people to</a> <a data-t="364">enter more easily.</a> <a data-t="370">So we've developed over time a ton of different packages, whether</a> <a data-t="370">they are around solving a problem for a particular project that we have to repeat, or they're</a> <a data-t="375">more general utility functions so that people don't have to recreate the wheel every time</a> <a data-t="379">they encounter a problem.</a> <a data-t="385">So an example of this is one of our utility packages called</a> <a data-t="385">Glamour, where we do something as simple as setting up a folder structure, right?</a> <a data-t="389">Doing</a> <a data-t="389">this manually is very easy to do, but when you're talking about having to repeat this</a> <a data-t="393">process across lots and lots of different projects and doing this across lots of users,</a> <a data-t="399">having something that allows you to just fall into that pit of success is important.</a> <a data-t="403">And</a> <a data-t="403">we do this through a lot of different ways.</a> <a data-t="408">So like loading secrets, securely storing</a> <a data-t="408">and loading secrets to make it easy for people to have all their credentials in one place</a> <a data-t="413">and to make sure that they're not posting this to GitHub and posting their credentials.</a></p>

<p><a data-t="419">So packages really founded the foundation of this infrastructure, because they created</a> <a data-t="425">reproducibility for our work and consistency across all of our users.</a> <a data-t="433">And more importantly,</a> <a data-t="433">this created an approachable way for new users to come on without having to know the full</a> <a data-t="438">R ecosystem, but they had tools that were specific to our workflows that they could</a> <a data-t="443">use and take advantage of.</a></p>

<h3>Creating buy-in</h3>

<p><a data-t="450">So while we had those packages, we didn't really have the</a> <a data-t="450">buy-in.</a> <a data-t="455">We didn't have people clamoring to be at the table to use R.</a> <a data-t="461">And where we started</a> <a data-t="461">to be able to turn things around was really convincing leadership that R was the right</a> <a data-t="461">direction to go.</a> <a data-t="467">Leadership is really critical, because they're the ones who are ultimately</a> <a data-t="467">creating that tradeoff structure, telling analysts whether they should be focusing their</a> <a data-t="472">attention on training versus having to turn around products, right?</a> <a data-t="477">Because there's always</a> <a data-t="477">that need for those concrete products.</a> <a data-t="484">So that they can create that influence both in</a> <a data-t="484">the workforce and as they're hiring new staff on, can see this as a marker when they're</a> <a data-t="489">looking at CVs in order to bring that on.</a></p>

<p><a data-t="496">And the way we were able to lure in our leadership</a> <a data-t="496">was through concrete products.</a> <a data-t="502">We were able to, and specifically around visualizations.</a> <a data-t="502">We were able to build off GG plot by creating our own package glitter that used our own</a> <a data-t="507">themes and colors that allowed us to quickly and efficiently create high-quality products</a> <a data-t="513">that our leadership could use to understand where our progress was at and use these as</a> <a data-t="518">communication pieces.</a> <a data-t="525">And they wanted to see more and more of these more quickly.</a></p>

<p><a data-t="525">We also</a> <a data-t="531">had to convince our colleagues that R is a worthwhile investment, right?</a> <a data-t="535">And so I think</a> <a data-t="535">it's clear automation makes a ton of sense to all of us in this room, but there is that</a> <a data-t="535">learning curve.</a> <a data-t="541">And so we needed to convince our colleagues that that upfront investment</a> <a data-t="541">cost is worthwhile.</a> <a data-t="547">Spending anywhere from two hours to two weeks to invest that process</a> <a data-t="547">so then in a year, when you're rerunning all of these things, it only takes seconds to</a> <a data-t="552">run those analyses, as opposed to having to do that comfort in a point-and-click world</a> <a data-t="558">where, yes, it's easy to do, that you have spent 40 minutes on it instead of two weeks,</a> <a data-t="564">but you have to spend that 40 minutes every single time that you're doing this.</a></p>

<p><a data-t="569">And so</a> <a data-t="569">the way we were able to influence change for our colleagues was to curate support to their</a> <a data-t="575">different workflows and intervene in specific instances.</a> <a data-t="581">So maybe it was working with a</a> <a data-t="581">colleague to show them how to set up and work with them to support an API to bring their</a> <a data-t="586">data in without having to do a manual download of data.</a> <a data-t="591">Or maybe it was showing them how</a> <a data-t="591">to create calculations before they moved their data into Tableau.</a> <a data-t="598">Or maybe they felt comfortable</a> <a data-t="598">in the point-and-click world for all of their munging, but then wanted to have a high-quality</a> <a data-t="602">product using R for the visualization side.</a></p>

<h3>Fostering community</h3>

<p><a data-t="608">So it's great.</a> <a data-t="608">We started to have some buy-in</a> <a data-t="608">from our users, but we needed then this last piece of community.</a> <a data-t="613">We needed to support them</a> <a data-t="613">in some way.</a> <a data-t="618">And I think everybody here is familiar, as you've started R or some new</a> <a data-t="618">programming language, there's lots of resources out there, and it's easy to be inundated and</a> <a data-t="623">overwhelmed by everything and not sure where you should start learning.</a> <a data-t="628">And so we realized</a> <a data-t="628">early on that we needed to curate an approach for them, a path that they could move forward</a> <a data-t="634">so that they knew what direction to go and what was applicable for our work.</a></p>

<p><a data-t="639">So as a result, we created a community of practice in our office called Core, which</a> <a data-t="643">is really founded on the ideas of being able to share our workflows and best practices</a> <a data-t="648">between inexperienced users and the ones who had a lot more experience.</a> <a data-t="654">And as we developed</a> <a data-t="654">more and more R users, we could do things like having one-on-one mentorship that we</a> <a data-t="660">had people who had been around and knew what was going on.</a> <a data-t="664">We were able to develop classroom-like</a> <a data-t="664">settings to have more people able to kind of know the one-on-ones of R.</a> <a data-t="672">Setting up a</a> <a data-t="672">Slack channel was really important as well, because it created a forum that people could</a> <a data-t="676">feel like they could ask questions very easily, and other people could share new things that</a> <a data-t="681">they'd learned.</a> <a data-t="686">And it didn't have to be a one-way street like a classroom setting.</a> <a data-t="691">And then lastly, just kind of setting up infrequent makeover challenges so people became</a> <a data-t="691">more familiar with ggplot.</a></p>

<p><a data-t="696">So here's kind of an example of how we tailored our approach</a> <a data-t="696">in the classroom setting.</a> <a data-t="703">So myself and other colleagues set up a 12-part series that's</a> <a data-t="703">based on R for Data Science.</a> <a data-t="708">The core principles from R for Data Science are there, and many</a> <a data-t="708">of these chapters are almost copied verbatim.</a> <a data-t="713">But the difference, though, is we tailored</a> <a data-t="713">the data to what our analysts were familiar with.</a> <a data-t="719">We used our PEPFAR data so that an analyst</a> <a data-t="719">understood the funda—could focus on the fundamentals of R rather than having to worry</a> <a data-t="725">about new data.</a> <a data-t="731">We also tailored all of the examples to things that worked in our workflow,</a> <a data-t="731">so that they understood the data, they understood these workflows, and could go from the classroom-like</a> <a data-t="735">setting to apply it the very next day in their work, which was quite successful.</a></p>
