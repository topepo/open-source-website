<h3>Literate programming with R Markdown</h3>

<p><a data-t="102">All right, so R Markdown then is, it's kind of a package in the realm of literate programming.</a> <a data-t="112">And the idea is that rather than embedding comments in a string of code that you have,</a> <a data-t="120">that instead you would actually embed the code within the document that you're writing.</a> <a data-t="124">So you can write kind of this narrative and sort of create this prose that describes the</a> <a data-t="128">analysis that you're doing, and then within that just embed chunks of R code.</a> <a data-t="133">And obviously there are a variety of different ways that you can leverage the tool, but that's</a> <a data-t="137">one of the more popular ones we see in terms of reproducible research and literate programming.</a></p>

<p><a data-t="141">But the idea is that you're going to render the textual output and the graphical output</a> <a data-t="145">and anything else that R is creating just in line within your document.</a> <a data-t="149">So this is an example here.</a> <a data-t="150">So on the left you can see that this would be the input format.</a> <a data-t="154">So there's this convention of three backticks, and then you specify that using the R language.</a> <a data-t="158">And then in here you just write any R code that you want, and then you close that with</a> <a data-t="162">three more backticks.</a></p>
