<p><a data-t="9">So I'm going to be talking about using LLMs to hack productivity,</a> <a data-t="12">and some experiments I've been doing over the past 18 months.</a> <a data-t="15">I was thinking recently</a> <a data-t="18">about a job I did during my career.</a> <a data-t="21">I was working on a project, and as part of the project,</a> <a data-t="25">I had the tedious task of taking a database export as a CSV,</a> <a data-t="27">converting it to Excel,</a> <a data-t="30">applying various sorts and filters,</a> <a data-t="33">getting them just right, and sending it to the stakeholders.</a></p>

<p><a data-t="101">And so, as Cat Hicks wrote in her keynote the other day,</a> <a data-t="103">it's not just what these tools can do,</a> <a data-t="106">but actually our experience of interacting with them.</a> <a data-t="108">And that's something that has really resonated with me.</a> <a data-t="109">I've spent a lot of time,</a> <a data-t="111">over the past 18 months,</a> <a data-t="113">experimenting with LLMs,</a> <a data-t="116">and trying to increase productivity in different ways.</a> <a data-t="117">Some of that has been really helpful,</a> <a data-t="119">and some of it I've committed whole libraries</a> <a data-t="122">to something that hasn't worked.</a></p>

<p><a data-t="134">So the first question is,</a> <a data-t="137">how difficult will the task be?</a> <a data-t="138">I'm going to illustrate this</a> <a data-t="140">by talking about an R package</a> <a data-t="142">I've been working on recently.</a> <a data-t="144">So the package itself</a> <a data-t="147">is called Daisy UI Widget.</a> <a data-t="149">It's an HTML widgets package,</a> <a data-t="150">and that means it uses</a> <a data-t="153">HTML, JavaScript, CSS and R,</a> <a data-t="155">and the users interact with the package</a> <a data-t="157">in R.</a> <a data-t="159">To work with the package,</a> <a data-t="161">the user creates a data frame</a> <a data-t="163">with dates and comments.</a> <a data-t="165">They call the function timeline,</a> <a data-t="166">and they get a timeline,</a> <a data-t="168">like the one on the screen here.</a></p>

<p><a data-t="170">So I decided,</a> <a data-t="171">when I was starting this project,</a> <a data-t="172">to see if I could build it</a> <a data-t="174">with LLM help.</a> <a data-t="176">So I used GitHub Copilot,</a> <a data-t="179">but I ended up having to pump or work to get the code.</a> <a data-t="182">So how did the process go?</a> <a data-t="202">So if the task is fairly easy or straightforward,</a> <a data-t="205">you can get started and generate some code,</a> <a data-t="208">but when we get to the point</a> <a data-t="210">of getting to the quick question,</a> <a data-t="212">things will work fine there.</a></p>

<p><a data-t="214">So at this point,</a> <a data-t="216">I had some of the package,</a> <a data-t="218">but a friend was helping me</a> <a data-t="220">to develop it,</a> <a data-t="223">and I wanted to see</a> <a data-t="225">if LLMs could help.</a> <a data-t="227">So I started moving forward</a> <a data-t="229">on task two.</a> <a data-t="231">So the AI slot</a> <a data-t="233">I'd gotten out of GitHub Copilot</a> <a data-t="235">had come up with this way</a> <a data-t="237">of working with the package,</a> <a data-t="239">where the function Daisy</a> <a data-t="241">was taking a list of lists as input.</a> <a data-t="243">But actually,</a> <a data-t="245">for things like this,</a> <a data-t="247">it's more conventional</a> <a data-t="249">to have a data frame as input.</a> <a data-t="251">So what I was doing here</a> <a data-t="253">was changing the function from list of lists to a data frame.</a> <a data-t="255">So it felt like a task.</a> <a data-t="257">I could understand how to do it myself.</a> <a data-t="259">It wasn't going to take a lot of time,</a> <a data-t="261">but I wanted to see what would happen</a> <a data-t="263">if I got an LLM to do it.</a></p>

<p><a data-t="265">So, how did it go?</a> <a data-t="267">It was pretty good.</a> <a data-t="269">I ended up</a> <a data-t="271">with this incredible code</a> <a data-t="273">that was full of paths</a> <a data-t="275">I'd never seen before.</a> <a data-t="277">The code came with a lot of tests.</a> <a data-t="279">For that small amount of code,</a> <a data-t="281">I'd never seen so many tests associated with it.</a> <a data-t="283">The code felt really important</a> <a data-t="285">to get to fit together.</a> <a data-t="287">The code was also interesting.</a> <a data-t="289">So the code worked</a> <a data-t="291">and it did what it said.</a> <a data-t="293">But when I went through the code,</a> <a data-t="295">there was a lot to take in.</a> <a data-t="297">I feel like code review is</a> <a data-t="299">more of a social exercise.</a> <a data-t="301">It can be enjoyable to connect with people.</a> <a data-t="303">But I kept coming back to asking,</a> <a data-t="305">was the code going to be used in the right way?</a> <a data-t="307">Was the other model going to use it?</a> <a data-t="309">Was the data it was trained on right?</a> <a data-t="311">Was that code really good?</a></p>

<p><a data-t="405">So if the task is fairly easy,</a> <a data-t="407">it feels like</a> <a data-t="409">it takes more time,</a> <a data-t="411">if the task is fairly hard,</a> <a data-t="413">I can't really engage with the code,</a> <a data-t="415">but actually if it's more difficult,</a> <a data-t="417">that's an interesting place.</a> <a data-t="419">And actually, when I framed it this way,</a> <a data-t="421">I realized a model</a> <a data-t="423">to develop and grow.</a> <a data-t="455">So I was going to see if I could use an LLM</a> <a data-t="457">to teach me how to do it myself,</a> <a data-t="459">which brings me to the first question.</a></p>

<h3>What role should the LLM play?</h3>

<p><a data-t="461">What role</a> <a data-t="463">would the LLM play?</a> <a data-t="465">So at this point, I went and looked up</a> <a data-t="467">some of the literature</a> <a data-t="469">on using LLMs in education.</a> <a data-t="471">And I found a paper</a> <a data-t="473">by Dr. C. <p><a data-t="501">So to go back</a> <a data-t="503">to that task,</a> <a data-t="505">the stretch zone task.</a> <a data-t="507">It felt like the perfect stretch task</a> <a data-t="509">to use here, because I had experience</a> <a data-t="511">of the </a><a href="/software/dt/">DT</a><a data-t="511"> task. <p><a data-t="543">So task 1.</a> <a data-t="545">Establishing learning goals.</a> <a data-t="547">I like to think about this in terms of how I would have done it</a> <a data-t="549">if I didn't have LLM help.</a> <a data-t="551">So I think I would have said,</a> <a data-t="553">okay, it's obvious to use the DT package,</a> <a data-t="555">okay, and just gone with that.</a> <a data-t="557">But actually,</a> <a data-t="559">when I asked ChatGPT,</a> <a data-t="562">it asked me a number of questions.</a> <a data-t="564">It asked me exactly what I was trying to achieve,</a> <a data-t="566">how long I wanted to spend on it,</a> <a data-t="568">and what past experience I had.</a> <a data-t="570">And from my answers to the questions,</a> <a data-t="573">it ended up coming with this really clear and concise goal.</a> <a data-t="575">And it broke it down into three practical steps.</a> <a data-t="578">So, capturing the event in Javascript,</a> <a data-t="580">then passing that to Shiny,</a> <a data-t="582">and then testing it in an app.</a> <a data-t="583">Now, could I have done this myself?</a> <a data-t="584">Sure.</a> <a data-t="585">But was I likely to?</a> <a data-t="587">Yeah, probably not.</a> <a data-t="588">So it just left me feeling like</a> <a data-t="591">I really knew exactly what I was trying to achieve here.</a></p>

<p><a data-t="593">So the second step, then,</a> <a data-t="595">locating and accessing resources.</a> <a data-t="597">So, again, like, doing this on my own,</a> <a data-t="598">I'd probably end up Googling things</a> <a data-t="600">and have, like, 50 browser tabs open</a> <a data-t="601">and end up down a rabbit hole</a> <a data-t="604">of kind of reading about a slightly different component.</a> <a data-t="606">But with ChatGPT help,</a> <a data-t="608">I had a couple of questions.</a> <a data-t="610">So how do I prefer learning?</a> <a data-t="612">I said I prefer learning by doing.</a> <a data-t="613">I like having kind of snippets of code</a> <a data-t="616">that I can run and see what happens and play with.</a> <a data-t="619">And it found me really quickly just two resources</a> <a data-t="621">that were perfect for the task at hand.</a> <a data-t="623">So, firstly, the book Javascript for R,</a> <a data-t="625">which has a free copy available online.</a> <a data-t="629">And secondly, a blog post from 2016 by Dina Tali</a> <a data-t="632">that's all about more or less exactly what I was trying to do.</a> <a data-t="634">So, great.</a> <a data-t="636">And I guess this had stopped me kind of having to switch</a> <a data-t="639">between kind of planning and organizing and coding,</a> <a data-t="642">and I could just focus on reading up and writing some code.</a></p>

<p><a data-t="646">So then, finally, the third step for this</a> <a data-t="650">was this idea of adopting and executing learning activities.</a> <a data-t="651">So there wasn't much back and forth</a> <a data-t="653">between me and ChatGPT at this point,</a> <a data-t="654">because we kind of talked a lot</a> <a data-t="656">about kind of my past experience and things,</a> <a data-t="658">but it came up with this three-step process</a> <a data-t="660">I was going to follow.</a> <a data-t="662">Firstly, I was going to have a look in the DT package</a> <a data-t="665">and kind of try to reverse-engineer the code in there.</a> <a data-t="667">So seeing if I could find the code</a> <a data-t="669">that actually implemented that row selection</a> <a data-t="672">to learn more about exactly how that worked.</a> <a data-t="674">Secondly, build a minimal working demo,</a> <a data-t="676">so in a tiny little Shiny app,</a> <a data-t="679">just showing capturing a user event.</a> <a data-t="682">And then thirdly, add it to the actual package.</a></p>

<p><a data-t="686">But it wasn't just like this three-step process that was useful.</a> <a data-t="689">Around this, it did things like it gave me time boxes,</a> <a data-t="693">so I didn't end up spending too long on each step.</a> <a data-t="695">It also had useful little bits and pieces,</a> <a data-t="698">so with the bit around reverse-engineering DT,</a> <a data-t="701">it actually gave me a link to the exact place in the package</a> <a data-t="703">that I needed to find the code to read,</a> <a data-t="704">which was really helpful,</a> <a data-t="706">not having to track that down myself.</a> <a data-t="708">And it just feels like a lot of these things</a> <a data-t="710">are all about reducing the level of frustration</a> <a data-t="711">that we can end up having</a> <a data-t="713">when we're trying to figure out a new thing.</a></p>

<p><a data-t="715">But even better than that,</a> <a data-t="718">it also suggested ways that I could use ChatGPT</a> <a data-t="720">to help me with these.</a> <a data-t="723">So for example, when I was reverse-engineering DT,</a> <a data-t="726">it mentioned, okay, any code you don't understand,</a> <a data-t="728">paste it in, and we'll go through step by step.</a> <a data-t="731">And again, that was super useful.</a> <a data-t="733">When I was building my minimal demo,</a> <a data-t="735">the first time I ran it, it wasn't quite right,</a> <a data-t="737">and just being able to paste it straight in</a> <a data-t="739">and figure out what wasn't working,</a> <a data-t="741">again, just reduced that frustration.</a> <a data-t="744">And then when I added the event tracking to the app,</a> <a data-t="747">we ended up in a situation where it wasn't working,</a> <a data-t="749">and I was asking why,</a> <a data-t="750">and the LLM actually asked me,</a> <a data-t="753">well, what do you think you should need to click on</a> <a data-t="754">in this situation?</a> <a data-t="756">And it really felt like a really great</a> <a data-t="760">just learning activity and being tutored on something.</a></p>

<p><a data-t="764">And so I deem this definitely a success.</a> <a data-t="766">There's a write-up on my blog as well</a> <a data-t="768">with more detail if you're interested.</a> <a data-t="770">And I feel like, for me,</a> <a data-t="772">it was just a really enjoyable learning experience</a> <a data-t="775">because I feel like I'd done the work myself.</a> <a data-t="777">It wasn't the LLM doing the work for me.</a> <a data-t="779">It was scaffolding my learning</a> <a data-t="781">and pointing me in the right direction</a> <a data-t="783">and figuring out what questions I needed to be asking</a> <a data-t="785">and what the right answers were,</a> <a data-t="786">but just not taking over.</a> <a data-t="791">And I genuinely felt quite empowered as a learner.</a></p>

<blockquote><a data-t="779">It was scaffolding my learning</a> <a data-t="781">and pointing me in the right direction</a> <a data-t="783">and figuring out what questions I needed to be asking</a> <a data-t="785">and what the right answers were,</a> <a data-t="786">but just not taking over.</a> <a data-t="791">And I genuinely felt quite empowered as a learner.</a></blockquote>

<h3>When is a human the better choice?</h3>

<p><a data-t="793">So equipped with these ideas</a> <a data-t="796">of the kind of stretch zone projects</a> <a data-t="798">and using the LLM as a tutor,</a> <a data-t="800">I decided to take these ideas</a> <a data-t="803">and apply them to a completely different project.</a> <a data-t="806">And this brings me to my third and final question.</a> <a data-t="809">When is a human the better choice?</a> <a data-t="812">So I'm one of the maintainers of the arrr package,</a> <a data-t="814">and I decided I wanted to have a go</a> <a data-t="819">at contributing to PyArrow, the Python arrow implementation.</a> <a data-t="822">I decided to work on a task related to magic methods.</a> <a data-t="823">Now, if you don't know what they are,</a> <a data-t="825">it doesn't matter too much for this,</a> <a data-t="828">but it felt like the perfect stretch zone task</a> <a data-t="831">in that I've got experience of the arrr package,</a> <a data-t="833">I can write some Python,</a> <a data-t="835">but I didn't know much about PyArrow</a> <a data-t="838">and very little about the internals of Python libraries in general.</a> <a data-t="840">So it kind of felt in the right place.</a></p>

<p><a data-t="845">So next, I needed to figure out what role I wanted the LLM to play.</a> <a data-t="848">So I decided that I was going to write most of the code.</a> <a data-t="851">In this case, you know, this task for me was about learning.</a> <a data-t="853">So I needed to do that myself, really.</a> <a data-t="855">I also wanted to write the tests,</a> <a data-t="856">just so I could be really clear</a> <a data-t="859">about what the intentions of the code that I was writing.</a> <a data-t="863">And I was going to get the LLM to help me with any code tweaks.</a> <a data-t="865">Maybe there's bits of how things are done in Python</a> <a data-t="868">that I don't really kind of get with package internals.</a> <a data-t="871">So I was going to use it to help me understand errors</a> <a data-t="873">and to explain any new concepts, like magic methods,</a> <a data-t="876">which I had no idea what they were at the time.</a> <a data-t="880">But also, I had some other help from my friend, Alenka,</a> <a data-t="882">who is a PyArrow maintainer,</a> <a data-t="885">and she was just going to jump in anywhere I got stuck.</a></p>

<p><a data-t="887">Okay, so how did it go?</a> <a data-t="890">Well, actually, I feel like this is almost like a kind of fight</a> <a data-t="892">between LLMs and human knowledge,</a> <a data-t="896">almost like a kind of retro video game with different levels.</a> <a data-t="898">So the first level, then, rubber duck.</a> <a data-t="900">So I think probably a lot of people in this room</a> <a data-t="903">have experience of that kind of rubber duck debugging thing,</a> <a data-t="905">where, you know, you've got a problem with some code,</a> <a data-t="907">and in explaining it to an inanimate object,</a> <a data-t="910">you actually find yourself getting closer to the solution</a> <a data-t="911">than you were in the first place,</a> <a data-t="913">without even getting an answer back.</a> <a data-t="915">And I found here, actually,</a> <a data-t="919">when I was trying to create the prompt to kind of put into the LLM,</a> <a data-t="921">half the time, trying to get something specific enough</a> <a data-t="923">that it wasn't going to come back with nonsense</a> <a data-t="925">actually helped me figure out a lot of the problem myself,</a> <a data-t="928">which was great.</a></p>

<p><a data-t="931">The second level, the LLM as an assistant.</a> <a data-t="933">So there were points at which I was trying to build PyArrow,</a> <a data-t="935">and it's got C++ under it,</a> <a data-t="937">and I'm not super experienced in C++,</a> <a data-t="939">and I ended up with various errors in the build logs,</a> <a data-t="941">and instead of having to troll through</a> <a data-t="944">and figure out what was the problem and what was happening,</a> <a data-t="950">having an LLM just help me pinpoint it really quickly was fantastic.</a></p>

<p><a data-t="953">So the third level, the LLM as a tutor.</a> <a data-t="955">So as I mentioned, I had no idea</a> <a data-t="957">what magic methods were before this,</a> <a data-t="959">so I got it to come up with a very quick</a> <a data-t="961">sort of mini-tutorial-type thing,</a> <a data-t="964">give me a really quick, simple example of exactly what it was,</a> <a data-t="967">and it just made me feel way more prepared</a> <a data-t="969">for the task I was trying to complete.</a> <a data-t="972">But actually, there was a point at which LLM help</a> <a data-t="975">didn't really take me any further.</a></p>
