<h3>AWS and Amazon SageMaker</h3>

<p><a data-t="210">We'll start with AWS or Amazon Web Services.</a> <a data-t="213">We've partnered with Amazon SageMaker so that it's possible for administrators</a> <a data-t="217">to configure an Amazon SageMaker domain to have access to </a><a href="/software/rstudio/">RStudio</a><a data-t="217">.</a> <a data-t="223">In this process, an administrator goes in, they either create a new domain</a> <a data-t="226">or they go to an existing domain that exists in their SageMaker environment</a> <a data-t="230">and they make sure that there's a Posit license in place</a> <a data-t="233">and they can configure access to Posit Workbench and RStudio within that environment.</a></p>

<p><a data-t="238">Once that's been done, individual users can come into the SageMaker platform</a> <a data-t="242">and can request RStudio sessions from within that platform.</a> <a data-t="246">When users make this request, they're brought to the familiar Posit Workbench homepage,</a> <a data-t="250">but there's a few key differences from what you might expect out of a traditional installation.</a> <a data-t="255">One of the biggest things here is that users can request a specific compute instance type</a> <a data-t="260">for their session to run in.</a> <a data-t="262">This means that every user gets an isolated EC2 instance within AWS for their particular session.</a></p>

<p><a data-t="268">This accomplishes two distinct things.</a> <a data-t="271">One, users can request resources based on the workload they anticipate doing.</a> <a data-t="276">If I have a huge data set and I plan on bringing it into memory and analyzing it,</a> <a data-t="280">I know I'm going to need a lot of memory.</a> <a data-t="281">So, I can make a choice when I start my session to have an instance</a> <a data-t="285">that provides me with adequate memory for my analysis.</a> <a data-t="288">On the other side, maybe I have an analysis that's going to require a lot of parallelization</a> <a data-t="292">across multiple CPU cores.</a> <a data-t="294">To improve efficiency, I can request a compute instance that has a high CPU count</a> <a data-t="298">so that my workload finishes faster.</a></p>

<p><a data-t="301">The other advantage of this architecture is that everybody's sessions run independent of one another.</a> <a data-t="306">This is hugely advantageous if you're like me</a> <a data-t="308">and you occasionally do something a little bit silly in RStudio</a> <a data-t="311">and all of a sudden the entire server is locked up.</a> <a data-t="314">I'm sure I'm not the only one who's tried to read a data set into memory</a> <a data-t="317">that far exceeded the available memory in my environment.</a> <a data-t="321">If that happens in a traditional single-server install,</a> <a data-t="324">I've now brought the server down for everyone.</a> <a data-t="326">I had to buy more than one round of drinks to make up for that.</a></p>

<p><a data-t="329">Now, if you do the same thing here in the SageMaker environment,</a> <a data-t="335">you need to reset your session, but other users are unaffected.</a> <a data-t="339">If I do something in my session that causes me to exceed the available resources,</a> <a data-t="343">I'll need to start over and make sure that my new session contains adequate resources,</a> <a data-t="347">but I am not interrupting anyone else's workflow.</a></p>

<p><a data-t="350">The advantage here, and this is true for all of these solutions that we talk about,</a> <a data-t="354">is that this comes without additional IT administrative burden.</a> <a data-t="358">The IT office does not need to worry about managing this environment.</a> <a data-t="361">SageMaker handles the orchestration of the resources and everything behind the scenes.</a></p>
