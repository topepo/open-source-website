<h3>What is MLOps and why it's intimidating</h3>

<p><a data-t="381">So my first encounter with MLOps, well, what is it, first of all?</a> <a data-t="386">So it stands for Machine Learning Operations.</a> <a data-t="389">And the idea is that it streamlines the machine learning lifecycle.</a> <a data-t="394">So the idea is we train our models.</a> <a data-t="399">We want that model to be usable, so we deploy it to the cloud,</a> <a data-t="403">where users can query it.</a> <a data-t="405">So we'll have some sort of model API.</a> <a data-t="409">And then we also want to monitor the model while it's in production.</a> <a data-t="415">So we want to check that as the data changes,</a> <a data-t="417">is the model still maintaining an acceptable accuracy?</a> <a data-t="420">And if it's not, then we need to make sure we're retraining it</a> <a data-t="423">at a regular frequency.</a></p>

<p><a data-t="427">Another big part of this, as well, is because this is like a cycle,</a> <a data-t="430">and we're constantly retraining our models,</a> <a data-t="432">it becomes very important to version and to store our models,</a> <a data-t="435">because we want to be able to retrieve a model from any point in time in the past.</a> <a data-t="440">If these models are leading to important decision-making in businesses,</a> <a data-t="443">it's very important to see why a certain decision was made,</a> <a data-t="446">and what was it about that model that led to that prediction.</a> <a data-t="450">So that's pretty much all there is to it, really, in words.</a> <a data-t="453">It sounds simple enough.</a> <a data-t="455">Until we see our first architecture diagram.</a></p>

<p><a data-t="459">So this is like a very typical sort of architecture diagram.</a> <a data-t="463">I actually just used a diagram that my friend put together.</a> <a data-t="465">He'll not mind.</a> <a data-t="467">But essentially, the way these work is they basically summarize</a> <a data-t="472">the full infrastructure of like a machine learning sort of lifecycle or workflow.</a> <a data-t="477">So usually there's some version control stuff,</a> <a data-t="480">in this case with GitHub, sort of around the top.</a> <a data-t="483">Then you're going to have some stuff to do with like development environment.</a> <a data-t="486">There's going to be a staging environment somewhere, production environment.</a> <a data-t="491">But, yeah, for like a beginner data scientist, when I look at this,</a> <a data-t="494">it might as well just be squiggly lines, you know, just spaghetti.</a> <a data-t="497">You know, I don't really know what's going on.</a></p>

<p><a data-t="508">So there is just an uncomfortable reality for a lot of beginners in data science</a> <a data-t="510">that, and this was also touched on in Flavia's brilliant talk just before mine,</a> <a data-t="514">there's an overwhelming choice when it comes to data science.</a> <a data-t="518">So particularly with modeling frameworks, you know, do I go tidy models?</a> <a data-t="522">Do I go R or Python?</a> <a data-t="524">You know, which package am I going to use?</a> <a data-t="526">What type of model will I use?</a> <a data-t="528">Then there's cloud platforms.</a> <a data-t="530">Do I use Azure? Do I use AWS?</a> <a data-t="532">You know, it's an endless choice.</a> <a data-t="534">And there's so many different options for creating environments and containers,</a> <a data-t="539">and the list just goes on.</a> <a data-t="541">It's also very multidisciplinary.</a> <a data-t="543">So, you know, this demands expertise in data science on the modeling side,</a> <a data-t="548">but also in like cloud infrastructure.</a> <a data-t="550">So actually setting up, you know, compute environments,</a> <a data-t="553">setting like the, you know, the optimal computing power, that sort of thing.</a> <a data-t="559">So unless you've got a big team around you,</a> <a data-t="561">it's very difficult to know how to jump into this area.</a> <a data-t="563">It's also very expensive.</a> <a data-t="565">You know, all these cloud platforms charge money.</a> <a data-t="567">You know, none of these are free.</a> <a data-t="569">There might be a few free trials out there,</a> <a data-t="571">but, you know, it's very difficult to jump into this.</a></p>
