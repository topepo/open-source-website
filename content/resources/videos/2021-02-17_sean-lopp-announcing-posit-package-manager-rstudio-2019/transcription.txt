<h3>Package management as a supply chain problem</h3>

<p><a data-t="193">This was made even worse by the fact that the user then tried to install the latest</a> <a data-t="198">version of dplyr onto their Windows machine, and CRAN was serving an out-of-date binary,</a> <a data-t="204">so the problems got worse and worse.</a> <a data-t="208">So these are kind of the tip of the iceberg.</a></p>

<p><a data-t="215">There are two things that maybe you've encountered yourself, but I want to do a bit of fear mongering</a> <a data-t="220">here, which is even if you're a single data scientist, there's a lot more that can go</a> <a data-t="225">wrong, and the problem is it can go wrong further down the road.</a> <a data-t="230">So I want to discuss a few problems that if you have the problem, it's really too late.</a> <a data-t="234">It's kind of like a smoke detector and a fire.</a> <a data-t="238">If you're burning in the fire, you probably wish you had invested in the smoke detector,</a> <a data-t="243">and there's a similar issue around package management.</a> <a data-t="246">It's critical that you think of your strategy for managing packages before you're in the</a> <a data-t="251">fire.</a></p>

<p><a data-t="249">So what does that fire look like?</a> <a data-t="254">Well, three things we've seen time and time again, unfortunately, with some of our customers,</a> <a data-t="259">is that the first one here is you make a critical decision based on an analysis, and then you're</a> <a data-t="264">not able to rerun the analysis, because when you click go on your code, you get a package</a> <a data-t="269">error, and that's a really tough place to be in, and it can be quite hard to try to</a> <a data-t="275">figure out what you need to back out or reinstall or change to get the analysis to run again.</a></p>

<p><a data-t="280">The second one here is maybe a bit more mundane, but if you're an R user today who does all</a> <a data-t="285">their work on a desktop and you can install packages from wherever you want, congratulations,</a> <a data-t="290">you're in a happy spot.</a> <a data-t="292">You won't be there for long.</a> <a data-t="294">As your analysis leaves the organization or permeates the organization, you'll likely</a> <a data-t="300">be asked to move your analysis into other places, other servers that might not have</a> <a data-t="304">as easy access to the internet, and at that point, if you haven't thought at all about</a> <a data-t="308">what packages you're using, you're going to have a hard time working with IT to get those</a> <a data-t="312">packages into a happy spot, and then finally, even something as simple as sharing your code</a> <a data-t="317">with a colleague, if you don't have a plan for package management now, it can be really</a> <a data-t="321">hard, and you'll end up in a situation where maybe the new junior R user you've brought</a> <a data-t="326">on board is spending their first day looking at your session info output trying to reinstall</a> <a data-t="331">packages, so none of these things are where we want to be, but the good news is that we</a> <a data-t="336">can come up with strategies that will prevent these problems from happening in the first</a> <a data-t="341">place.</a></p>

<p><a data-t="342">And so that's what we're going to talk about today, is one role or the role that</a> <a data-t="347">package manager can play in those types of strategies, and so what package manager allows</a> <a data-t="352">you to do is instead of having every R client install packages from the various places they</a> <a data-t="357">live, you know, in GitHub or on CRAN or maybe in your own Git enterprise server, instead</a> <a data-t="364">of having all of the R clients install on their own, you can have a central place or</a> <a data-t="369">your own repository that organizes, collects, and governs what packages are available, and</a> <a data-t="375">then have your actual R clients install from there, and there are many ways to create these</a> <a data-t="380">repositories, there's open source packages like mini CRAN, and we believe that package</a> <a data-t="385">manager provides a great end-to-end solution for organizations that are interested in solving</a> <a data-t="390">this type of problem.</a></p>

<p><a data-t="392">To maybe help you wrap your head around this, I'm going to try an analogy with bad clip</a> <a data-t="397">art.</a> <a data-t="397">It's worked for me before, and I hope it works again, but I love to cook, but I wouldn't</a> <a data-t="402">consider myself a professional, I'm kind of a hobbyist, and so I typically, you know,</a> <a data-t="407">whatever I have in my pantry, I throw into a pot, hope for the best, I use, you know,</a> <a data-t="411">my home kitchen appliances, and I look for great deals at the grocery store, and I think</a> <a data-t="416">that's maybe similar to when I started in R, you know, I had the laptop that was available</a> <a data-t="421">to me, I had the tools that were available to me, and I installed, you know, whatever</a> <a data-t="425">I needed to try to get the job done, but a professional chef would never take the approach</a> <a data-t="430">I take to cooking.</a></p>
