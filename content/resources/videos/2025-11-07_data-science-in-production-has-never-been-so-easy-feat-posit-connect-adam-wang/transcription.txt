<h3>Why the team loves Posit Connect</h3>

<p><a data-t="210">So why does my team love Posit Connect?</a> <a data-t="210">The first thing that we like</a> <a data-t="210">is that scheduling and monitoring just comes out of the box.</a> <a data-t="214">We don't have to worry about that.</a> <a data-t="215">So no more waking up early and running reports by hand, and more importantly, we can have</a> <a data-t="221">proactive alerting.</a> <a data-t="226">So we built on top of the Posit Connect API to have this automation status</a> <a data-t="226">report, which summarizes every single day our entire catalog of reports and finds out how many</a> <a data-t="234">ones failed and there's even conditional logic to alert certain people if certain reports failed.</a></p>

<p><a data-t="241">Secondly, and I think this is my favorite aspect, is that we don't have to change how we work.</a> <a data-t="248">So we can develop in our favorite ID of choice, whether that's RStudio, VS Code, or </a><a href="/software/positron/">Positron</a><a data-t="248">,</a> <a data-t="255">as long as we all agree to push back to our Git repository at least once in a while.</a> <a data-t="262">And we get to use our favorite code-first tools like R and Python, </a><a href="/software/shiny-r/">Shiny</a><a data-t="262"> applications,</a> <a data-t="267">Jupyter Notebooks, </a><a href="/software/quarto/">Quarto</a><a data-t="267">, just to name a few.</a></p>

<h3>Architecture and the data science lifecycle</h3>

<p><a data-t="416">So those are some reasons that we really</a> <a data-t="416">enjoy Posit Connect, but how does it fit into our architecture or more generally the data science</a> <a data-t="422">lifecycle?</a> <a data-t="429">And so this is probably not a surprise, but it almost always starts with an idea or</a> <a data-t="429">problem that you're trying to solve.</a> <a data-t="434">And so for us, that was trying to predict the likelihood that a</a> <a data-t="434">donor is willing and able to donate.</a> <a data-t="441">And we call that the donor readiness score.</a> <a data-t="441">And so once we</a> <a data-t="441">have that idea, then we do what we do best as data scientists.</a> <a data-t="448">We do some exploration, exploratory</a> <a data-t="448">data analysis locally, we train a model, we run some validation, make sure that the model is</a> <a data-t="453">performing as we would expect, and we do it with our favorite IDs of choice.</a></p>

<p><a data-t="461">But once we start</a> <a data-t="461">getting to the step where we have code that looks like, take our model, take some new data,</a> <a data-t="467">calculate some new predictions, this is when you want to start thinking about deploying to</a> <a data-t="472">production so you don't have to run this four times a day every single day for a long time.</a> <a data-t="478">And that's where Posit Connect fits in for our architecture.</a> <a data-t="485">So it's hooked up to our GitLab</a> <a data-t="485">repository through a Git integration.</a> <a data-t="492">And this is a one-time setup that basically gives the Posit Connect</a> <a data-t="492">server read access to everything that we push to Git.</a> <a data-t="499">And so things like our data science storefront,</a> <a data-t="499">it's a portal document in our GitLab repository, and Posit Connect has access to that along with</a> <a data-t="508">the usual open-source packages.</a></p>

<p><a data-t="512">And the one change you have to make to your workflow using</a> <a data-t="512">Posit Connect, unlike Docker or Lambdas, is you just have to create this one manifest.json file,</a> <a data-t="519">and you can do that automatically through one of these two commands, depending on if you're</a> <a data-t="522">using Python or R, and the manifest is basically an instruction manual for the server to know what</a> <a data-t="529">version of Python or R you're using, what packages you're using, and their versions,</a> <a data-t="533">so it can help set up that environment.</a> <a data-t="539">And once you have that manifest.json file,</a> <a data-t="539">you go to the Posit Connect UI, you find the file or the folder where the manifest is in,</a> <a data-t="546">you click the blue button to deploy your content, and then you have your awesome report that you can</a> <a data-t="551">link to and share right away.</a></p>

<p><a data-t="559">And it's really helpful for maintenance, too, because Posit Connect</a> <a data-t="559">automatically checks all your content with a manifest every 15 minutes to see if there's</a> <a data-t="563">updates, so we don't have to implement any extra CICD on our end to make sure that the changes we</a> <a data-t="569">pushed gets reflected to the latest version of the report.</a> <a data-t="574">That's all done under the hood,</a> <a data-t="574">all that complexity is abstracted away.</a></p>

<h3>Connecting to the database</h3>

<p><a data-t="583">So the next critical piece to have in our architecture</a> <a data-t="584">is introducing our database, because you can work with CSVs locally, you can work with pins for a</a> <a data-t="590">time, but you always want to connect to your database because you want to have a live connection</a> <a data-t="596">in production.</a> <a data-t="603">So let's zoom in there a little bit.</a> <a data-t="603">So the database set up to Posit Connect is,</a> <a data-t="603">again, a one-time connection, and there are Posit professional drivers that are included</a> <a data-t="608">that support most common databases, around 15 the last time I checked.</a> <a data-t="615">And once you have this</a> <a data-t="615">two-way connection where you can read data from Snowflake and also write back to data</a> <a data-t="620">to Snowflake, you can do things like score millions of records per day,</a> <a data-t="626">and schedule this to run as much as you need.</a></p>

<p><a data-t="633">And one kind of interesting use case that we</a> <a data-t="633">would recommend for your database of choice is to be able to leverage that database compute when</a> <a data-t="640">you need to for the occasional job that doesn't fit into memory in your Posit Connect server.</a> <a data-t="646">And databases are a great place to do that because they're used to dealing with big data all the time</a> <a data-t="652">and your data already lives there, so your security department's going to be really,</a> <a data-t="655">really happy that there's one less transfer of data moving around.</a> <a data-t="661">And for us, we use Snowflake</a> <a data-t="661">and there's Snowpark functionality, so we can actually write Python user-defined functions</a> <a data-t="667">that run natively in the warehouse.</a> <a data-t="673">So when we have to occasionally do a huge compute job,</a> <a data-t="673">we just scale up the warehouse, use some Python, and there you go.</a> <a data-t="679">You can get more creative for</a> <a data-t="679">other databases that don't support Python or Python-like languages by, you know, converting</a> <a data-t="684">your model objects into SQL.</a> <a data-t="691">If you're interested in that, you can look into Orbital, but there's</a> <a data-t="691">creative ways to leverage your warehouse compute when your server isn't doing the job for you.</a></p>
