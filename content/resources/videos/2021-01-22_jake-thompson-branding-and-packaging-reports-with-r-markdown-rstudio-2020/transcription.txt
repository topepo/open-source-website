<h3>Why R Markdown for reports</h3>

<p><a data-t="64">So I probably don't need to sell this crowd on R Markdown too much, but I do think it's</a> <a data-t="68">worth repeating why we wanted to use R Markdown for these reports.</a> <a data-t="73">The first reason is that R Markdown reports are reproducible, and I mean that both within</a> <a data-t="78">and across years.</a> <a data-t="80">So if we have an analysis and someone says, where did this number come from?</a> <a data-t="84">Because we have the code, it's easy for us to say exactly where that analysis happened</a> <a data-t="90">and how we got those results.</a></p>

<p><a data-t="92">But then it's also reproducible across years.</a> <a data-t="94">So if you think about educational assessment, students test every single year, which means</a> <a data-t="100">we have to redo those reports every year with the updated data, and so by using R Markdown,</a> <a data-t="105">we can just drop in the new data files and then get the updated report for that year.</a> <a data-t="110">So it ends up saving us a lot of time.</a></p>

<p><a data-t="113">Secondly, these reports are dynamic.</a> <a data-t="114">So I think we've all probably experienced an email that says, oh no, we have an updated</a> <a data-t="120">data file.</a> <a data-t="121">The one you used is wrong.</a> <a data-t="122">But by using R Markdown, we can just drop in that new data file and get corrected results</a> <a data-t="127">really quickly.</a> <a data-t="128">And the other reason is that we can get multiple output formats with R Markdown.</a> <a data-t="133">So we can write reports out to Microsoft Word or PDF if we're using LaTeX, or you can even</a> <a data-t="138">make Flash Textboard or some slides or a website.</a></p>

<h3>The branding problem</h3>

<p><a data-t="145">So the problem that we faced was that the default look of Markdown reports that you</a> <a data-t="153">see over here on the left does not really match what our normal reports look like on</a> <a data-t="157">the right.</a> <a data-t="159">And so often organizations, Atlas included, have specific branding.</a> <a data-t="164">So we say if we're going to make a report, this is what it needs to look like.</a> <a data-t="168">So in order for us to use R Markdown for our technical reports, we had to make sure that</a> <a data-t="173">it matched our brand guidelines.</a> <a data-t="175">So today what I want to talk about is how we went about making these branded reports</a> <a data-t="180">in a reproducible and scalable way.</a></p>

<h3>Step one: find your brand</h3>

<p><a data-t="185">So the first step is to find your brand.</a> <a data-t="189">And that can mean many things.</a> <a data-t="191">Some organizations have a style guide or brand guidelines that will specify what fonts you're</a> <a data-t="197">supposed to use, the colors, spacing of the margins.</a> <a data-t="200">So anything that is really annoying and gives you a headache if you have to look at it for</a> <a data-t="204">too long, that's probably what's in the brand guidelines.</a> <a data-t="208">If you don't have that, you might have a marketing team who puts together your reports for final</a> <a data-t="213">distribution, or you might even just have an editing team.</a> <a data-t="217">So if you have someone who edits your final reports right before they go out, they are</a> <a data-t="221">probably doing the little editing stuff so you don't have to.</a> <a data-t="225">So you find them, and that will help you figure out what your final result needs to look like.</a></p>

<h3>Step two: build your Word template</h3>

<p><a data-t="232">Step two is to then build your template, which is obviously not that easy.</a> <a data-t="237">It's an iterative process that we're going to talk a little bit more about.</a> <a data-t="241">So basically you'll start a template, you'll render your document, realize it's not exactly</a> <a data-t="245">what you wanted, make some tweaks, re-render, and so on and so forth until you get to that</a> <a data-t="250">final version that you want.</a> <a data-t="252">So today we're going to talk about how to do this with a Microsoft Word template, because</a> <a data-t="256">I think it's the most straightforward, but you could do this with any type of R Markdown</a> <a data-t="260">output.</a></p>

<p><a data-t="263">So I think the easiest way to do this with Microsoft Word is just to start writing your</a> <a data-t="267">document in R Markdown.</a> <a data-t="269">So you can start it just like you would any document, and then once you get a little bit</a> <a data-t="275">into it, you can hit that knit button, and what you'll get is an output that looks like</a> <a data-t="281">this.</a> <a data-t="282">And this is the default Microsoft Word template if you don't specify any changes.</a></p>

<p><a data-t="294">And so what you'll do is you can click your little cursor into some section of the document,</a> <a data-t="300">so for example if we were in the title, and then you'll click the styles pane up top,</a> <a data-t="306">and there you'll see exactly what style the default template has chosen for you.</a> <a data-t="311">So if we are in the title, it'll say that we are in the title style, and then if we</a> <a data-t="316">hit this arrow, we can modify that style.</a> <a data-t="319">So when we modify, we can change the font that's being used, the color, the size, the</a> <a data-t="323">spacing around the paragraph, basically anything that you want to change for that style, you</a> <a data-t="328">can modify.</a></p>

<p><a data-t="330">And then once you've done that, you save that Word document, and that will be your template.</a> <a data-t="335">And so I normally save it as template.docx, just to make it very clear what I'm doing,</a> <a data-t="340">and then in your R Markdown YAML header, you still specify that you want to use the</a> <a data-t="347">Word document, but then you include this extra option that says reference.docx equals template.docx,</a> <a data-t="354">so that template file that you just saved.</a> <a data-t="357">And so now when you knit that R Markdown document, it's going to use all those styles that you</a> <a data-t="362">defined when you were modifying those styles in your Word document.</a></p>

<p><a data-t="367">So there are a lot of different ways that you can modify styles in a Word document to</a> <a data-t="372">do some really cool things.</a> <a data-t="374">So these are some resources that I found helpful when we were developing our Word templates.</a> <a data-t="379">So Daniel Hadley actually gave a talk at RStudioConf a couple years ago called Branding and Automating</a> <a data-t="384">with R Markdown, where he gives some really good advice about how to do this in Word and</a> <a data-t="389">some different tricks about how to get page breaks by using a level six heading, I think.</a> <a data-t="395">There's also a post on the </a><a href="/software/rstudio/">RStudio</a><a data-t="395"> website from Richard Layton called Happy Collaboration</a> <a data-t="400">with R Markdown to DocX.</a> <a data-t="403">And then there's a chapter in R Markdown, The Definitive Guide, about Word documents.</a></p>

<h3>Polishing reports: figures and chunk options</h3>

<p><a data-t="425">So once we have our templates, you're not done.</a> <a data-t="429">So your reports probably need a little bit of polishing.</a> <a data-t="432">So for example, if you're including figures, you might want your figures to also match</a> <a data-t="436">your brand guidelines.</a> <a data-t="437">So use the same fonts as your report, maybe have a color palette that matches your brand's</a> <a data-t="441">colors.</a></p>

<p><a data-t="444">And so there are some really good examples about how to do this.</a> <a data-t="447">I'm not going to talk about </a><a href="/software/ggplot2/">ggplot2</a><a data-t="447"> themes and how to create them today.</a> <a data-t="452">That's kind of beyond the scope of what I want to talk about here.</a> <a data-t="455">But I did include some packages that I think give some really good examples for how to</a> <a data-t="460">do this.</a> <a data-t="461">So the HRBR themes package, and then also Artistic and bbplot, I think do a really good</a> <a data-t="467">job of creating custom themes.</a></p>
